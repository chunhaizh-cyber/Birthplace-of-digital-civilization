# 本能动作管理类返回信息规范 20260430

状态：定义修订中，当前代码仍处于兼容迁移期。

适用范围：`本能动作管理类`、本能动作函数、执行桥、任务管理界面线程读取本能动作返回信息的方式。

本规范用于替换“本能动作函数返回混合结果结构”的旧思路。后续实现不得继续把运行结果、执行结果、结构结果和动作动态混在同一个返回结构中。

本能函数本体的四段式实现、条件场景与输入参数场景边界、变化类动作输入和输出账本，按 [本能函数实现规范20260430.md](本能函数实现规范20260430.md) 执行。

## 1. 核心结论

本能动作函数执行后产生两类直接返回信息：

| 信息 | 权威承载 | 主要消费者 |
| --- | --- | --- |
| 运行结果 | 本轮运行虚拟存在 | 任务管理任务或任务管理界面线程 |
| 执行结果 | 输出结果场景 | 任务管理任务或任务管理界面线程 |

本能动作函数的返回值应表达“这次方法有没有正常运行起来”，即运行结果。

运行结果应返回为一个虚拟存在。这个虚拟存在表示本轮本能函数调用形成的运行事实，可视为任务虚拟存在的同类运行账本，但作用域限定在本轮方法调用。

```cpp
存在节点类* 运行虚拟存在
```

执行结果不作为函数返回值返回，而是写入输出结果场景。

## 2. 四类结果边界

一次方法执行中可能产生四类信息：

| 类别 | 含义 | 落点 | 是否由本能动作管理类解释 |
| --- | --- | --- | --- |
| 运行结果 | 方法入口、参数、执行过程是否正常 | 本轮运行虚拟存在 | 否，只返回给任务管理侧 |
| 执行结果 | 方法对目标、场景、特征产生了什么功能结果 | 输出结果场景 | 否，只提供场景入口 |
| 结构结果 | 本轮条件和结果对是否应沉淀到方法树 | 方法树的条件节点与结果节点 | 否，由执行桥或方法维护逻辑处理 |
| 动作动态 | 本轮是否发生真实方法调用及其过程 | 动态信息 | 否，由执行桥或动作执行体处理 |

本能动作管理类只负责登记、查询、检查和调用。它不根据这些结果直接推进任务，也不直接生成需求。

## 3. 运行结果的表达方式

运行结果必须是虚拟存在，不是抽象特征定义本体，也不只是单个实例特征。

推荐形态：

```text
宿主：方法虚拟存在或本轮任务虚拟存在
运行虚拟存在：本轮本能函数运行账本
运行虚拟存在特征：
  方法_最近运行状态：I64
  方法_最近错误码：I64
  方法_最近输入场景：指针
  方法_最近结果场景：指针
  返回参数信息：结构化参数绑定结果或缺失参数说明
  返回场景信息：输出结果场景、缺口场景或补位场景入口
  方法_最近开始时间：时间戳
  方法_最近结束时间：时间戳
```

说明：

1. 运行虚拟存在表达某个方法本轮运行事实。
2. 运行状态、错误码、输入/输出场景、耗时等作为该虚拟存在的特征。
3. `I64` 运行状态用于排序、比较和简单门控。
4. 任务管理侧不应再读取自然语言备注作为权威判断依据。
5. 兼容迁移期可以保留从运行虚拟存在读取结果值的适配函数，替代旧的单特征返回读取方式。
6. 确保阶段只负责补齐方法头、方法虚拟存在和运行参数声明，不作为成功出口。
7. 本能函数必须继续进入解析绑定、执行和输出阶段；只有在无法形成有效运行虚拟存在时，才允许返回空。
8. 当解析绑定或执行无法继续时，应在运行虚拟存在中写入返回参数信息、返回场景信息和错误状态，再由输出阶段返回该运行虚拟存在。

运行结果示例：

| 抽象特征语义 | I64 值 | 任务管理含义 |
| --- | ---: | --- |
| 运行成功 | 0 | 可以读取输出结果场景并继续结算 |
| 运行结果未知 | -1 | 不足以推进，记录事件后等待下一轮判断 |
| 输入参数不满足 | -20009 | 任务退回筹办或派生补条件需求 |
| 方法首节点缺失 | -20005 | 筹办结果无效，任务退回筹办 |
| 方法节点类型错误 | -20006 | 逻辑错误，任务挂起并提交问题 |
| 动作入口缺失 | -20007 | 需要补动作骨架或重新筹办 |
| 动作入口不归本类执行 | -20008 | 交由其他动作执行器或重新筹办 |
| 本能动作未注册 | -20001 | 方法不可执行，提交方法可用性缺口 |
| 执行异常 | -20003 | 标记方法不稳定，按策略重试或改筹办 |
| 执行未知异常 | -20004 | 标记方法不稳定，任务可重试或改筹办 |

具体数值不是安全值或服务值，不参与需求权重结算。它只服务任务管理侧的运行状态判断。

## 4. 运行虚拟存在的落点

优先落点：

```text
方法虚拟存在
```

原因：

1. 运行结果描述的是方法自身这次是否运行正常。
2. 方法稳定性、可调用性和错误统计都属于方法虚拟存在。
3. 同一个方法多次运行可以在方法虚拟存在上形成稳定记录。

补位落点：

```text
输出结果场景
```

仅在以下情况使用：

1. 检查阶段尚未得到有效方法首节点。
2. 无法解析方法虚拟存在。
3. 本轮返回信息只作为任务管理侧临时消费入口。

补位落点不是长期运行账本。任务管理侧消费后，应根据任务状态决定是否提交需求、挂起任务或重新筹办。

## 5. 执行结果的表达方式

执行结果必须写入输出结果场景。

输出结果场景可包含：

| 内容 | 用途 |
| --- | --- |
| 目标特征变化结果 | 判断需求是否满足或部分满足 |
| 实际输出状态 | 供任务管理结算完成度 |
| 功能缺口事实 | 供任务管理提交派生需求 |
| 相关方法或目标引用 | 供任务管理追溯任务来源 |
| 质量、成本、置信度等功能评价 | 供任务管理选择后续动作 |

执行结果不直接改变任务状态。任务状态必须由任务管理界面线程读取运行结果和输出结果场景后统一改变。

## 6. 任务管理侧消费规则

任务管理任务或任务管理界面线程读取两类信息后，负责做以下判断：

| 运行结果 | 执行结果 | 任务管理侧动作 |
| --- | --- | --- |
| 运行成功 | 目标满足 | 标记任务完成，结算需求满足，唤醒父任务 |
| 运行成功 | 目标部分满足 | 计算满足程度，继续推进或派生剩余需求 |
| 运行成功 | 产生缺口 | 提交派生需求给自我线程 |
| 输入参数不满足 | 任意 | 退回筹办或提交补条件需求 |
| 动作入口缺失 | 任意 | 提交方法可用性或动作骨架缺口 |
| 本能动作未注册 | 任意 | 提交方法可用性缺口或重新筹办 |
| 执行异常或超时 | 任意 | 记录方法不稳定，按任务策略重试、挂起或改筹办 |

本能动作管理类不得在内部决定：

1. 任务是否完成。
2. 需求是否满足。
3. 是否创建派生需求。
4. 是否唤醒父任务。
5. 是否增加安全值或服务值。

## 7. 目标接口方向

本能动作函数目标函数形态：

```cpp
using 本能动作函数 = std::function<存在节点类*(
    方法类::节点类* 方法首节点,
    场景节点类* 输入参数场景,
    场景节点类* 输出结果场景)>;
```

返回值：

```text
存在节点类* 运行虚拟存在
```

输入规则：

1. `方法首节点` 是本次要执行的方法。
2. `输入参数场景` 是参数来源。
3. `输出结果场景` 是执行结果写入位置。
4. 本能动作函数不得读取任务、需求、控制面板或线程调度状态。

本能动作管理类目标执行入口：

```cpp
存在节点类* 执行功能函数(
    枚举_本能动作ID 动作ID,
    方法类::节点类* 方法首节点,
    场景节点类* 输入参数场景,
    场景节点类* 输出结果场景) noexcept;
```

返回值为空表示执行入口自身无法形成有效运行虚拟存在。任务管理侧应将其视为运行结果未知，并记录事件。

## 8. 可调用检查

“本能方法可执行”只检查以下内容：

1. 方法首节点存在。
2. 方法首节点中有动作函数或动作指针。
3. 动作入口可以归属到当前本能动作管理类或其他明确执行器。
4. 输入参数满足方法条件。

可调用检查不检查：

1. 任务状态。
2. 需求状态。
3. 方法是否应该被调度。
4. 控制面板状态。
5. 安全值或服务值。

检查失败时也应形成运行虚拟存在，供任务管理侧处理。优先挂在方法虚拟存在下；没有有效方法虚拟存在时，可临时挂在输出结果场景下。

## 9. 与 `结构_本能动作步骤结果` 的关系

`结构_本能动作步骤结果` 当前把运行结果、执行结果、动作动态入口和诊断文本混在一起。

后续应删除该结构在核心接口中的使用。

删除后：

| 旧字段 | 新落点 |
| --- | --- |
| `成功码` | 运行虚拟存在中的结果值、错误码或命中状态 |
| `置信度` | 方法虚拟存在或输出结果场景中的功能评价特征 |
| `质量` | 输出结果场景或方法虚拟存在中的评价特征 |
| `成本_us` | 方法虚拟存在中的运行成本特征 |
| `结果存在集场景` | 调用方传入的输出结果场景 |
| `结果事件集场景` | 输出结果场景内的事件结果区或后续专门场景 |
| `动作动态` | 动态信息，必要时在方法虚拟存在或输出结果场景中保存引用 |
| `备注` | 日志事件，不作为结构判断依据 |

## 10. 结构结果和动作动态

结构结果不是返回值。

条件节点和结果节点必须成对维护。若本轮产生了可以沉淀的方法结构样本，应由执行桥或方法维护逻辑写入方法树。

动作动态不是返回值。

真实调用方法函数后，应由执行桥或动作执行体生成动作动态。任务管理侧可以通过输出结果场景或方法虚拟存在读取相关引用，但不得把任务消息本身当作真实动作动态。

## 11. 维护约束

后续修改本能动作管理类时必须遵守：

1. 不新增执行上下文结构。
2. 不新增混合结果结构。
3. 不把任务、需求、控制面板字段塞入本能动作函数形态。
4. 不由本能动作管理类创建需求。
5. 不由本能动作管理类推进任务。
6. 运行结果返回运行虚拟存在。
7. 执行结果写入输出结果场景。
8. 任务管理侧统一解释运行结果和执行结果。
