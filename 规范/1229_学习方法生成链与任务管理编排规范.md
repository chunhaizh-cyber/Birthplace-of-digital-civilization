# 1229 学习方法生成链与任务管理编排规范

## 1. 目标

本文用于固定：

- 学习任务怎样围绕 `目标方法可用` 这件事补方法
- 任务管理任务怎样为学习任务筹办步骤


## 2. 核心结论

必须固定以下结论：

1. 学习主链的顶层需求，统一是：`让当前目标方法可用`。
2. `样本收集 / 差异比较 / 共同条件提取 / 候选动作生成 / 结果预测 / 沙箱验证 / 风险评估 / 边界标记` 这些都只是学习步骤语义，不再是顶层任务目标。
3. 任务管理任务在这条链上的职责，是为学习任务筹办当前学习步骤。
4. 学习任务在这条链上的职责，是执行当前学习步骤，并把结果写回目标方法。


## 3. 学习方法的最小成长链

统一固定为：

```text
目标方法
-> 学习任务筹办
-> 补动作步骤
-> 补条件步骤
-> 补结果步骤
-> 验证步骤
-> 尝试学习步骤
-> 方法可用
```


## 4. 任务管理任务的编排职责

任务管理任务必须：

1. 创建或复用目标方法头。
2. 若方法尚不可用，则创建或复用学习任务。
3. 按目标方法当前缺口，为学习任务生成下一学习步骤。

也就是说，任务管理任务编排的是：

- 学习任务树
- 目标方法骨架

而不是单独编排一条脱离任务树的程序链。


## 5. 目标方法缺口到学习步骤的映射

| 目标方法当前缺口 | 应生成的学习步骤 |
| --- | --- |
| 无动作骨架 | 补动作步骤 |
| 有动作但无条件 | 补条件步骤 |
| 有动作和条件但无结果 | 补结果步骤 |
| 基本骨架齐但尚未验证 | 验证步骤 |
| 已可验证且允许进入试探 | 尝试学习步骤 |


## 6. 禁止项

禁止以下做法：

1. 让学习任务在没有步骤节点时直接进入尝试学习。
2. 把候选学习程序当成独立于任务树的主流程对象。
3. 把“目标方法为空”继续当成稳定运行口径保留。
