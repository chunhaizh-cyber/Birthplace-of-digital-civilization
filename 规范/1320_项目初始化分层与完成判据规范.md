# 1320 项目初始化分层与完成判据规范

## 1. 目标

本文用于把当前工程中的项目启动链收敛成统一的分层初始化规则，回答以下问题：

- 项目初始化应分成哪几层
- 哪些内容必须在自我线程启动前完成
- 哪些内容必须由自我线程自己完成
- `初始化完成` 与 `进入运行态` 的边界在哪里
- 哪些内容不得在热路径或子线程中补做首次初始化

本文的目标不是发明新的运行结构，而是把现有共识固定成项目级初始化口径，避免“线程活着就算启动完成”或“谁先用到谁就顺手初始化”的漂移实现。

## 3. 核心结论

项目初始化必须固定分为以下四段：

1. `数据层初始化`
2. `自我锚点初始化`
3. `自我运行环境初始化`
4. `进入运行态`

必须固定：

- `数据层初始化` 放在 `自我线程运行之前`
- `自我锚点初始化` 与 `自我运行环境初始化` 放在 `自我线程内`
- `进入运行态` 必须晚于前三段全部完成
- `初始化完成` 不得简化成 `线程对象已创建`、`joinable()`、`running=true` 或 `对象非空`


## 4. 总顺序

项目启动链必须按以下顺序理解：

1. 初始化公共数据底座。
2. 创建自我对象。
3. 启动自我线程。
4. 由自我线程完成 `自我锚点初始化`。
5. 由自我线程完成 `自我运行环境初始化`。
6. 标记 `初始化完成`。
7. 启动自我的直属子线程或工作者承载。
8. 切换到 `运行中` 并允许主循环进入正式治理。

必须固定：

- `直属子线程` 不得早于 `初始化完成` 启动。
- `初始化完成` 不等于 `健康运行`；健康运行还要求运行状态正确且核心未消亡。
- 若任一初始化层失败，不得继续宣称系统已完成初始化。


## 5. 数据层初始化

### 5.1 定位

`数据层初始化` 是全项目公共底座的初始化，不是某一个具体自我的业务真值初始化。

### 5.2 必须初始化的内容

至少包括：

1. 数据仓库全局链与锁体系进入可用态。
2. 基础数据环境进入可用态：
   - `特征值集`
   - `语素集`
   - `特征类型定义`
   - `语言集`
3. 世界骨架进入可用态：
   - `世界树`
   - 默认世界节点
   - 概念域根
4. 因果公共根进入可用态。
5. 项目日志与故障显性化通道进入可用态。
6. 项目级公共环境单例进入可调用态，例如：
   - `世界树`
   - `需求集`
   - `方法集`
   - 其它不依赖具体自我上下文的公共环境对象

### 5.3 不得在本层完成的内容

以下内容不得归入 `数据层初始化`：

- 某个具体自我的 `当前主需求 / 当前主任务 / 当前主方法`
- 某个具体自我的 `安全值 / 服务值 / 待机状态 / 时序步长`
- 根需求与根任务的具体实例
- 默认本能注册中依赖具体自我上下文的运行态写回

### 5.4 一句话收口

`数据层初始化 = 建公共底座，不建具体自我真值`


## 6. 自我锚点初始化

### 6.1 定位

`自我锚点初始化` 用于回答“这个自我是谁、挂在哪里、以谁为正式承载”。

它必须由自我线程自己完成，不得长期依赖外层线程代做。

### 6.2 必须初始化的内容

至少包括：

1. 当前活动自我锚点。
2. `世界树.自我指针` 对应的自我存在。
3. 自我所在的宿主现实场景。
4. 自我内部世界。
5. 自我存在与内部世界之间的归属关系。

### 6.3 完成判据

只有在以下条件同时成立时，才可认为 `自我锚点初始化完成`：

- 当前活动自我已可被查询
- 自我存在可读写
- 自我内部世界可读写
- 后续自我特征写入已拥有正式承载面

### 6.4 不得在本层完成的内容

- 正式主循环治理
- 直属 worker 启动
- 学习推进
- 执行推进
- 结果结算


## 7. 自我运行环境初始化

### 7.1 定位

`自我运行环境初始化` 用于回答“这个自我是否已经具备进入正式运行闭环的前置条件”。

它建立的是“可运行态”，不是“已经开始正式治理”。

### 7.2 必须初始化的内容

至少包括以下几类：

1. 双值核与核心自我特征默认承载：
   - `安全值`
   - `服务值`
   - `物理安全`
   - `风险安全`
   - `情绪ID / 情绪强度 / 外显情绪ID / 外显情绪强度`
   - `待学习方法数量`
   - `外设可用性`
   - `尝试学习状态`

2. 主链镜像入口：
   - `当前主需求`
   - `当前主任务`
   - `当前主方法`

3. 运行控制入口：
   - `待机状态`
   - `时序正向步长`
   - `时序反向步长`
   - `服务时序衰减步长`

4. 默认模板或等价基础依赖。

5. 默认本能动作集注册：
   - 函数注册
   - 方法首节点创建或复用
   - 方法信息回填

6. 根需求建立：
   - `安全根需求`
   - `服务根需求`

7. 根任务建立：
   - `安全根任务`
   - `服务根任务`

8. 常驻治理任务建立。

9. 内部学习治理所需的最小常驻对象与运行时容器进入初始态，例如：
   - 学习调度运行时
   - 尝试学习参数队列
   - 写回重试队列
   - 治理临时记忆

10. 初始运行镜像与初始化完成状态。

### 7.3 必须固定

- 根需求和根任务属于 `自我运行环境初始化` 的正式组成部分，不得长期拖到“首轮主循环跑到哪里算哪里”。
- 默认本能注册属于初始化前置条件，不得延迟到任务热路径、学习推进路径或周期维护路径再补做首次注册。
- 初始化阶段可以写默认真值，但正式业务真值必须继续追溯到 `自我存在及其特征`。

### 7.4 一句话收口

`自我运行环境初始化 = 把自我从“有承载”推进到“可运行”`


## 8. 进入运行态

### 8.1 定位

`进入运行态` 不是一次普通字段写入，而是项目对外语义的切换点。

### 8.2 进入条件

只有以下条件同时成立，才允许进入运行态：

1. 数据层初始化完成。
2. 自我锚点初始化完成。
3. 自我运行环境初始化完成。
4. 根需求已建立。
5. 根任务已建立。
6. 默认本能注册已完成。
7. 启动主链所依赖的核心对象已进入可工作态。

### 8.3 进入运行态后才允许发生的内容

- 启动直属 worker 或其它直属子线程
- 开放正式治理提交
- 开放正式学习提交
- 对外宣称 `初始化完成`
- 对外进一步判断 `健康运行`


## 9. 初始化完成、已启动与健康运行的关系

必须固定以下区分：

1. `已启动`
   只表示线程真实进入主生命周期。

2. `初始化完成`
   表示项目主链所需的初始化条件全部满足。

3. `健康运行`
   表示：
   - 运行状态正确
   - 初始化完成
   - 自我核心未消亡

因此必须明确：

- `已启动` 不等于 `初始化完成`
- `初始化完成` 不等于 `健康运行`
- `对象已创建`
- `selfThread 已存在`
- `running=true`
- `joinable()`

都不得单独作为 `初始化完成` 判据


## 10. 线程放置与启动顺序

必须固定：

1. 数据层初始化在自我线程之前完成。
2. 自我锚点初始化由自我线程完成。
3. 自我运行环境初始化由自我线程完成。
4. 父线程必须先完成自己的最小初始化，再启动直属子线程。
5. 一旦进入 `停止请求中`，不得再启动新的直属子线程。

项目级推荐顺序固定为：

`宿主准备公共底座 -> 启动自我线程 -> 自我线程完成锚点与运行环境初始化 -> 标记初始化完成 -> 启动直属 worker -> 进入正式运行`


## 11. 禁止项

以下做法必须禁止：

1. 把“全局单例已存在”误当成“数据层初始化完成”。
2. 把“自我对象已创建”误当成“自我锚点初始化完成”。
3. 把“安全值/服务值已写默认值”误当成“自我运行环境初始化完成”。
4. 把“首轮主循环顺手补齐根需求/根任务”当成正式初始化方案。
5. 让 worker、学习线程、执行线程承担首次本能注册职责。
6. 让子线程在父线程尚未完成最小初始化时抢先运行。
7. 在初始化失败后仍对外展示为正常运行。
8. 把结构体缓存扩张成平行于 `世界树中的自我特征` 的长期真值层。


## 12. 失败与幂等要求

必须固定：

1. 三类初始化都应支持幂等重入；重复初始化不得重复造根节点、根任务或默认本能方法头。
2. 任一层初始化失败时，必须留下：
   - 最近故障摘要
   - 可见日志
   - 未完成初始化状态
3. 失败后不得自动跳过未完成层直接进入运行态。
4. 若后续支持热重载，必须明确“只重做运行环境初始化”与“重建自我锚点”的边界，二者不得混用。
