# 0567 最终裁决层与外层重判边界规范

## 1. 目标

本文用于统一回答以下问题：

- `继续 / 停止 / 待机 / 收束 / 回到重筹办 / 转入学习 / 允许升格 / 允许回流` 这类词到底属于哪一层
- 哪些只是候选去向，哪些才算最终裁决
- 最终裁决是由谁做出的，建立在什么结果之上
- 为什么任何会影响主闭环走向的结论都必须回外层重判

## 2. 在五层体系中的位置

本文所说的这些结论，统一属于：

`第 5 层：最终裁决层`

它们应统一理解为：

`外层在消费条件判定结果、二次特征和根层重判结果后，真正决定主闭环往哪里走的结论`


## 3. 候选去向与最终裁决的区别

### 3.1 候选去向

候选去向指的是：

- 在当前管理上下文中
- 基于条件判定结果与局部证据
- 得到的“当前更适合往哪条路走”的候选结论

典型例子：

- `继续下一步`
- `补条件后重试`
- `回到重筹办`
- `转入尝试学习`
- `挂起等待`
- `进入收束`

它属于：

- `第 3 层：条件判定结果层`

它回答的是：

- `当前从工程承接上更像该往哪边走`

### 3.2 最终裁决

最终裁决指的是：

- 在候选去向已经形成后
- 再结合二次特征和根层重判结果
- 由外层真正做出的主闭环结论

典型例子：

- `允许继续推进`
- `停止`
- `待机`
- `进入收束`
- `允许回到重筹办`
- `允许转入学习`
- `允许升格`
- `允许回流`

它回答的是：

- `系统这一轮最终准不准这样走`


## 4. 最终裁决层的最小输入

最终裁决层至少应消费以下三类输入：

1. `条件判定结果`
2. `二次特征`
3. `根层重判结果`

必须固定：

- 缺少这三类输入中的关键项时，不得直接形成强最终裁决。
- 此时最多只能停留在：
  - `待进一步比较`
  - `待进一步观察`
  - `候选未定`
- 更细的 `根层重判结果` 定义、执行前用途和禁止项入口，按 [0315_根层重判结果与执行前允许边界规范.md](D:\鱼巢\规范\0315_根层重判结果与执行前允许边界规范.md) 执行。


## 5. 八类核心最终裁决

### 5.1 `允许继续推进`

必须固定：

- 它必须建立在 `条件上能做 + 方向上该做 + 根层未禁止` 的共同前提下。

### 5.2 `停止`

必须固定：

- `停止` 是强最终裁决，不等于局部缺口或局部失败。

### 5.3 `待机`

必须固定：

- `待机` 不是普通挂起等待。
- 它通常与安全/服务边界、临界保底态相关，不能由局部承接层自行决定。

### 5.4 `进入收束`

必须固定：

- `进入收束` 可以先在候选去向层出现。
- 但只有回到外层重判后，才能变成真正的最终裁决。

### 5.5 `允许回到重筹办`

必须固定：

- `回到重筹办` 在管理上下文里首先只是候选去向。
- 只有外层确认后，才是“允许回到重筹办”的最终裁决。

### 5.6 `允许转入学习`

必须固定：

- `当前下一步去向候选 = 转入尝试学习`，不自动等于“允许转入学习”。

### 5.7 `允许升格`

必须固定：

- `稳定复现`、`结果确认`、`来源判定` 只是升格证据，不自动等于“本轮允许升格”。

### 5.8 `允许回流`

必须固定：

- `有学习产出` 不等于 `允许回流`
- `产出看起来有用` 也不等于 `允许回流`


## 6. 标准顺序

最终裁决层的标准顺序应固定为：

```text
一次特征 / 状态 / 返回值
-> 条件判定结果
-> 二次特征
-> 候选去向
-> 外层根层重判
-> 最终裁决
```

必须固定：

1. `候选去向` 在 `最终裁决` 之前。
2. `外层根层重判` 在 `最终裁决` 之前。
3. 任一承接层若会影响主闭环走向，都必须把结果送回这条链，而不是在本层直接拍板。


## 7. 与各承接层的边界

### 7.1 与任务管理承接层

必须固定：

- 四域、调度器、执行器可以给出 `候选去向`。
- 但它们不直接拥有 `最终裁决权`。

### 7.2 与学习承接层

必须固定：

- 学习线程和学习任务可以给出“建议回流”“建议继续验证”“建议升格”等承接层结果。
- 但它们不直接拥有 `允许回流 / 允许升格 / 允许继续推进` 的最终裁决权。

### 7.3 与方法成长承接层

必须固定：

- 来源判定、成熟度阶段判定、稳定复现统计都只是升格证据和结构承接结果。
- 它们不直接拥有“本轮最终允许升格”的裁决权。

### 7.4 与自我主循环

必须固定：

- 自我主循环是当前运行时里最接近最终裁决层的工程承载者。
- 它负责在每轮承接层返回后，重新做 `根层重判 -> 上下文刷新 -> 再决定是否提交下一轮请求`。


## 8. 禁止事项

以下做法统一禁止：

1. `当前下一步去向候选 = 继续下一步 -> 直接继续推进`
2. `当前下一步去向候选 = 回到重筹办 -> 直接跳回筹办`
3. `当前下一步去向候选 = 转入尝试学习 -> 直接视为学习已获批准`
4. `学习产出已生成 -> 直接允许回流`
5. `方法看起来稳定了 -> 直接允许升格`
6. `局部线程想继续跑 -> 直接继续推进主闭环`
7. 承接层绕过外层重判，自己决定 `停止 / 待机 / 收束 / 升格 / 回流`

## 10. 最终结论

可收敛为一句话：

`候选去向回答“现在更像往哪走”，最终裁决回答“这一轮最终准不准这样走”；两者之间必须经过外层根层重判。`
