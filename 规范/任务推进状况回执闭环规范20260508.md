# 任务推进状况回执闭环规范 20260508

## 1. 总原则

任务闭环固定为：

```text
本能函数产出状况；
任务管理线程解释状况；
自我线程改树、结账、重调度；
子任务完成后父任务待重筹办。
```

筹办和执行都不直接结束问题。它们只暴露本轮状况；状况被任务管理线程和自我线程吸收后，才推动下一轮筹办、子任务承接、方法执行或价值结算。

本规范中的“推进执行单元”包括：

```text
任务筹办普通函数
任务执行普通函数
本能动作函数
学习 / 练习 / 尝试学习函数
```

这些单元只能产出状况、事实、状态变化、条件结果对样本和执行回执。它们不得直接替父需求改树、结账、收束或把父任务标记为最终完成。

2026-05-12 修订：

```text
2026-05-13 修订：
任务筹办 / 任务执行已改为任务管理生命周期普通函数。
它们不进入方法树，不拥有方法虚拟存在，不生成方法动作动态，只返回任务层过程回执。
```

必须固定以下边界：

```text
动作执行成功 ≠ 任务目标达成
任务目标达成 ≠ 治理结果可接受
任务目标达成 ≠ 父需求可直接结算
子任务完成 ≠ 父任务完成
```

任务完成只能由当前任务目标结果符合确认。父需求结算只能由自我线程在吸收任务目标结果后进行。

## 2. 完整闭环

```text
父需求 D
    ↓
父任务 T
    ↓
筹办
    ↓
返回筹办状况
    ↓
任务管理线程处理状况
    ↓
若需要子需求 / 子任务，交给自我线程改需求树和任务树
    ↓
子任务执行
    ↓
子任务完成
    ↓
父任务标记为待重筹办
    ↓
父任务重新筹办
    ↓
若已可执行，进入执行
    ↓
执行返回执行状况
    ↓
任务管理线程归并条件结果对
    ↓
自我线程根据任务目标结果结算需求满足
```

这个过程不是固定两轮，而是多轮。每完成一个子任务，父任务都重新筹办一次。

## 3. 三层职责边界

### 3.1 本能函数

本能函数负责：

```text
1. 检查或执行一个最小动作。
2. 返回本轮状况。
3. 产生状态变化、动态、条件结果对样本。
4. 说明缺了什么。
```

本能函数不负责：

```text
1. 直接创建父需求的子需求。
2. 直接给父需求结算安全值 / 服务值。
3. 直接判断整个需求树下一步怎么长。
4. 直接把父任务标记为最终完成。
```

任务筹办普通函数、任务执行普通函数、学习函数和练习函数按同一边界执行：可以产生事实和回执，不能替自我线程改树、结账或完成父需求。其中学习函数/练习函数若作为方法运行，仍按方法动作动态硬规则落账；筹办/执行普通函数不作为方法动作来源。

### 3.2 任务管理线程

任务管理线程负责解释状况：

```text
1. 判断本状况能否在任务内部继续处理。
2. 判断是否需要等待子任务。
3. 判断是否需要父任务重筹办。
4. 判断是否需要把缺口上交自我线程生成子需求。
5. 判断是否可以进入执行阶段。
6. 执行后触发条件结果对归并。
```

任务管理线程不直接改需求树，不直接结算父需求价值。

### 3.3 自我线程

自我线程负责全局结构变化：

```text
1. 需求树新增、更新、满足、删除。
2. 子需求去重。
3. 子任务创建或复用。
4. 父任务唤醒。
5. 任务结果归类。
6. 安全值 / 服务值结算。
7. 禁止结果和风险结果治理。
```

## 4. 筹办状况

筹办必须返回结构化状况，不能只返回“找到 / 没找到”。

| 筹办状况 | 说明 | 处理者 | 后续 |
| --- | --- | --- | --- |
| 父需求已满足 | 父需求目标状态已经达成 | 任务管理线程，自我线程 | 父任务目标达成，交自我线程结算 |
| 存在活动子需求 | 子需求仍阻塞父任务 | 任务管理线程 | 父任务等待或调度子任务 |
| 方法查找未命中 | 没有方法能产生父任务目标结果；常规路径中只作为筹办内部诊断 | 筹办阶段 | 创建或复用待学习目标方法头，对外返回已建立方法头待重入 |
| 方法命中但缺动作骨架 | 方法有结果声明但不能执行 | 自我线程 | 创建或复用方法动作骨架状态子需求 |
| 方法命中但缺可执行入口 | 方法没有可调用动作入口 | 自我线程 | 创建或复用方法可执行入口子需求 |
| 方法命中但缺条件节点 | 方法结构不完整 | 自我线程 | 创建或复用方法条件节点存在子需求 |
| 方法命中但缺结果节点 | 方法结构不完整 | 自我线程 | 创建或复用方法结果节点存在子需求 |
| 方法命中但缺条件结果对 | 方法缺可验证条件结果经验 | 自我线程 | 创建或复用方法条件结果配对状态子需求 |
| 参数不可配 | 实际场景缺少方法所需参数特征，当前任务输入未闭合 | 自我线程或任务管理线程 | 创建或复用参数配对状态 / 任务参数绑定状态子需求，或标记等待 |
| 方法运行账不足 | 方法可执行性需要更多运行样本支撑 | 自我线程 | 创建或复用方法运行账完整性子需求，由后续筹办按结果能力选择承接方法 |
| 方法结果不稳定 | 方法已有结果能力但复现不稳定 | 自我线程 | 创建或复用方法稳定状态子需求，由后续筹办按结果能力选择承接方法 |
| 方法可执行 | 方法、条件、参数均已闭合 | 任务管理线程 | 进入执行阶段 |

方法查找未命中不是创建子需求的入口。筹办必须先让“应该存在的方法”出生：创建或复用一个待学习方法头，把父任务目标写入方法头的结果目标，然后父任务待重筹办。

`方法查找未命中` 是筹办阶段的内部事实；对外稳定回执优先是：

```text
已建立方法头待重入
```

也就是：

```text
查找未命中
    ↓
创建或复用待学习目标方法头
    ↓
返回已建立方法头待重入
```

筹办返回的可执行性缺口必须转换成“让目标方法头可执行”的子需求，不得复制父需求原目标。

结构化回执必须携带缺口特征类型、缺口目标宿主和当前方法。这里的缺口目标宿主是该缺口特征依附的存在或上下文，不表示要整体改变该存在。方法结构、运行账和稳定性缺口的 `缺口目标宿主` 应指向当前方法头或其方法虚拟存在；参数不可配缺口应指向当前任务输入包 / 参数绑定上下文。自我线程创建子需求时必须使用：

```text
缺口目标宿主 + 缺口特征类型
```

不得沿用父需求的业务目标宿主和业务目标特征类型。

新建待学习目标方法头后，除写入 `方法是否可用 = 0` 外，还必须补写：

```text
方法成熟度阶段 = 待学习
方法结构是否完整 = 0
方法运行账完整性 = 0
方法稳定状态 = 未稳定
方法允许自动召回 = 0
方法允许默认执行 = 0
```

## 5. 执行状况

执行阶段也必须返回结构化状况，不得只用“成功 / 失败”吞掉细节。

| 执行状况 | 说明 | 是否更新结果符合统计 | 后续 |
| --- | --- | ---: | --- |
| 动作执行失败 | 方法没真正产生可比较结果 | 否 | 记运行失败账，不记结果符合 / 不符合 |
| 动作执行成功无可比较结果 | 有动作事实但不能验证目标 | 否 | 归并事实，父任务重筹办或继续检查 |
| 目标结果符合 | 实际结果匹配当前任务目标特征 | 是，符合次数 +1 | 任务目标达成，需求满足检查 |
| 目标结果不符合 | 执行成功但实际结果未覆盖当前任务目标 | 是，不符合次数 +1 | 归并条件结果对，可能生成学习子需求 |
| 出现需要附带结果 | 非当前目标，但命中其他有效需求 | 不计入当前目标统计 | 路由到对应需求账 |
| 出现不需要附带结果 | 非当前目标、非禁止 | 不计入当前目标统计 | 只沉淀事实 |
| 出现禁止结果 | 触犯禁止项或风险约束 | 单独记风险账 | 自我线程风险治理 |

只有在方法可执行、动作函数执行成功、产生可比较结果之后，才允许根据目标结果是否符合当前任务目标更新结果符合 / 不符合统计。

可比较目标结果定义为：

```text
实际结果状态集里存在与当前任务目标特征类型相同、
可由特征比较服务比较、
且可按当前任务满足关系判断符合 / 不符合的状态变化。
```

没有可比较目标结果时，即使动作函数执行成功，也只能返回：

```text
动作执行成功无可比较结果
```

不得返回任务完成，不得结算父需求，不得更新目标结果符合 / 不符合账。

## 6. 子任务完成后的父任务处理

硬规则：

```text
子任务完成
    ↓
父任务不直接完成
    ↓
父任务标记为待重筹办
    ↓
父任务重新走筹办
```

子任务只解决一个阻塞点，不能证明父需求目标已经达成。父任务完成只能由父需求自身目标满足确认。

## 7. 活动子需求

父任务是否等待不能看：

```text
父需求是否有子节点
```

必须看：

```text
父需求是否有活动子需求
```

活动子需求定义：

```text
未满足
未截止
未取消
仍然阻塞父任务执行
```

历史子需求可以留在树上，但不得阻止父任务重新筹办并回到自身目标。

## 9. 缺口状况到子需求目标

方法查找未命中的处理固定为：

```text
查找方法返回未命中
    ↓
筹办创建或复用待学习方法头
    ↓
写入父需求 / 父任务目标结果能力
    ↓
父任务标记待重筹办
    ↓
返回：已建立方法头待重入
```

这里不创建子需求，不标记父需求满足，也不把未命中伪装成命中。

待学习方法头至少要写入：

```text
来源需求
来源任务
目标特征类型
目标状态
目标状态关系 / 方向
成熟度 = 待学习
可执行状态 = 不可执行
方法结构是否完整 = 0
方法运行账完整性 = 0
方法稳定状态 = 未稳定
方法允许自动召回 = 0
方法允许默认执行 = 0
```

下一轮筹办时，方法查找可以按结果能力命中这个方法头，但仍要继续检查动作骨架、可执行入口、条件节点、结果节点、条件结果对和参数绑定。

方法结构补齐任务必须遵守：

```text
被补方法是目标对象；
当前执行方法必须是能产出该缺口目标的方法，可由学习 / 练习 / 尝试学习承接，但不能硬性指定；
不得把被补方法本身绑定成当前执行方法。
```

学习承接只是普通方法匹配的一个结果，不是缺口路由的硬编码终点。底层规则仍然统一为：筹办查找能产出目标状态的方法，执行按同一套任务推进闭环分解子任务。

方法头存在但不可执行时，缺口状况必须转换成可执行性子需求目标：

| 返回状况 | 子需求目标 |
| --- | --- |
| 方法缺动作骨架 | 方法动作骨架状态 |
| 方法缺可执行入口 | 方法可执行入口状态 |
| 方法缺条件节点 | 方法条件节点存在状态 |
| 方法缺结果节点 | 方法结果节点存在状态 |
| 方法缺条件结果对 | 方法条件结果配对状态 |
| 参数不可配 | 参数配对状态 / 任务参数绑定状态 |
| 方法运行账不足 | 方法运行账完整性 |
| 方法结果不稳定 | 方法稳定状态 |

子需求目标宿主应指向任务、方法或方法虚拟存在，不得复制父需求目标。实际场景缺参不落到“方法是否可用”，而是生成参数配对状态 / 任务参数绑定状态一类的输入闭合目标。

同一方法根下不得重复创建相同目标结果能力的待学习方法头。去重口径仍按结果目标：

```text
目标特征类型
目标状态关系 / 方向
目标宿主类型或目标宿主范围
来源需求或适用范围
```

这不违反“找不到就是找不到”：第一次查找确实未命中；随后筹办根据未命中事实创建新的待学习方法头，下一轮再重新查找和检查。

## 10. 最终定型

```text
筹办负责发现能不能执行；
执行负责产生事实结果；
推进执行单元负责返回状况；
任务管理线程负责解释状况和推进任务状态；
自我线程负责改需求树、任务树、价值账和风险账；
每个子任务完成后，父任务必须重新筹办。
```

补充定型：

```text
找不到方法 -> 建方法头；
方法头不可执行 -> 学习子任务补方法；
方法可执行 -> 父任务执行目标。
```
