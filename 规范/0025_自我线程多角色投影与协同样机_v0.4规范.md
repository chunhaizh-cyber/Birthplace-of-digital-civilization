## 1. 目标

本文用于把 [0024_自我线程权限贴场景与人机确认闭环样机_v0.3规范.md](D:\鱼巢\规范\0024_自我线程权限贴场景与人机确认闭环样机_v0.3规范.md) 中已经定下的 `权限贴场景 + 人机确认 + 审计回写`，继续补成一份 `单自我 / 同一世界 / 多角色投影 / 共享事实与共享回写` 的 `v0.4` 协同样机规范。

本文当前只证明一件关键工程事实：

`在同一自我、同一根目标、同一世界快照之下，可以生成多个角色投影线程，但最终裁决权仍然只属于同一个主循环。`

- 同一轮上下文包如何投影出多个角色视图

- 冲突何时必须回根层主循环仲裁
- 多角色动作如何写回同一动态链、同一结算面和同一因果候选池

- `6` 个工程工作项
- `6` 个工程文件
- 一套最小角色投影模板
- 一套角色视图投影器
- 一种跨角色协同任务单
- 一套跨角色冲突回根层仲裁纪律
- 一套共享审计与统一回写纪律
- 一组只在 `一个场景 + 两到三个角色` 内闭环的测试面

本文不负责：

- 做多个彼此独立的自我
- 打开自由多智能体对话
- 扩成跨全部岗位的自动闭环执行
- 直接扩到跨场景全面复制
- 让角色线程夺走根层方向裁决权

一句话收口：

`0025 不是做多个自我，而是让同一个自我在同一世界快照上生成多个职责受限的角色投影，并把协同治理写回同一条事实链。`

## 3. v0.4 的定位与硬边界

### 3.1 v0.4 的本质

必须固定：

`v0.4` 的本质不是多智能体放养，而是单自我下的多角色协同治理。

它当前主要验证：

- 同一轮上下文包能否投影出多个角色视图
- 多个角色能否围绕同一事实底盘协同，而不是各建私有世界
- 多角色协同链能否统一回写到同一动态链和因果候选链
- 多角色冲突能否在上升时返回同一个主循环仲裁
- 根层主循环能否随时打断、收束、撤销全部角色链

### 3.2 v0.4 不做什么

本版明确不做：

- 自由多智能体闲聊
- 多个角色各自维护独立事实库
- 跨全部岗位的自动执行闭环
- 大规模跨场景复制
- 让角色线程之间互判最终优先级

### 3.3 角色线程的工程含义

必须固定：

`本版中的角色投影线程，首先是运行视图和协同边界，不强制等于操作系统线程。`

工程上允许：

- 先在同一个自我线程内，顺序模拟多个角色投影
- 先把角色差异落在 `读范围 / 权限边界 / 关注重点 / 协同票据` 上
- 先证明协同票据、冲突回流和统一回写能闭起来

### 3.4 一句硬边界

必须固定：

`多视图，不多世界；多角色，不多根层。`


## 4. 当前只做六个工程工作项

### 4.1 角色投影模板

先定义角色，不先定义聊天人格。

每个角色当前只写六样：

- `角色名`
- `局部职责`
- `局部目标`
- `读范围`
- `动作权限边界`
- `确认 / 升级对象`

### 4.2 角色视图投影器

从同一个 `线程轮次上下文包` 出发，投影出不同角色看到的局部世界。

必须固定：

- 同一事实底盘不变
- 不同角色看到的关注重点不同
- 投影结果只增 `角色过滤 / 角色关注 / 角色可见性`，不改底层事实

### 4.3 跨角色协同任务单

`v0.4` 不新增第四个大包，也不让角色之间自由喊话。

当前只增加一种 `线程内局部协同票据`，用于承接：

- `请求`
- `让渡`
- `确认`
- `拒绝`
- `升级`

必须固定：

`跨角色协同任务单` 只是当前轮次和当前治理步骤内的局部协同票据，不是新的根层裁决包。

### 4.4 跨角色冲突仲裁器

多角色上场后，最容易出现：

- 局部目标冲突
- 权限边界冲突
- 时序冲突
- 安全与服务判断不一致
- 两个角色都想让对方先动

这里必须补一个硬规则：

`角色之间不能互判最终优先级。`

一旦冲突上升，必须回根层主循环仲裁。

### 4.5 共享审计与统一回写器

多角色协同最怕的不是争论，而是各记各账。

因此当前必须固定：

- 所有角色线程写回同一动态链
- 所有角色线程写回同一结果结算面
- 所有角色线程写回同一因果候选池
- 所有角色动作都要带上 `角色来源 / 确认者 / 执行者 / 让渡链`

### 4.6 多角色金路径测试

仍然坚持 `小切口、强闭环`。

本版只在：

- `一个场景`
- `两到三个角色`
- `一轮上下文包`
- `一条协同回写链`

内完成验证。

## 6. 角色视图投影器

### 6.1 定位

角色视图投影器用于从同一个 `线程轮次上下文包` 投影出多个角色视图。

它回答的是：

- 同一事实对不同角色意味着什么
- 同一轮差额对不同角色各自该关注什么
- 同一轮里哪些对象、状态、关系对当前角色可见、可建议、可升级

### 6.2 投影结果必须包含

- `来源轮次主键`
- `来源上下文包主键`
- `角色主键`
- `角色可见对象集`
- `角色关注状态摘要`
- `角色关注关系摘要`
- `角色建议入口`
- `角色升级入口`

### 6.3 投影器禁止越权

不得：

- 修改上下文包原文
- 修改根层差额判定
- 修改主导需求
- 私造新的世界事实

必须固定：

`投影器只改视角，不改世界。`

### 6.4 同一事件的多角色投影示例

同一个异常事件进入同一轮上下文包后：

- `协调角色` 看全局影响链和各角色等待关系
- `资源角色` 看资源冲突、占用关系和替代位
- `服务角色` 看对象影响、告知对象和建议动作

这三者共享同一事件底盘，但关注点不同。


## 7. 跨角色协同任务单

### 7.1 定位

`跨角色协同任务单` 是附着在当前轮次和当前一步治理上的局部协同票据。

它不是新的大包，不替代：

- `线程轮次上下文包`
- `一步治理任务包`
- `回流结算包`

它只负责在角色之间传递本轮局部协同意图和处理结果。

### 7.2 最小动作

当前只开放五种协同行为：

1. `请求`
2. `让渡`
3. `确认`
4. `拒绝`
5. `升级`

### 7.3 最小字段

最小字段固定为：

- `协同单主键`
- `来源轮次主键`
- `来源任务包主键`
- `来源角色主键`
- `目标角色主键`
- `协同行为`
- `协同理由`
- `目标对象主键`
- `期望输出摘要`
- `时限`
- `当前状态`
- `结算要求`

### 7.4 当前状态

当前状态最小固定为：

- `待响应`
- `已接受`
- `已拒绝`
- `已升级`
- `已超时`
- `已结算`

### 7.5 硬边界

必须固定：

- 协同任务单只能在当前轮次内流动
- 协同任务单不能绕过主循环直接生成新的根任务
- 协同任务单一旦升级，必须回根层主循环仲裁


## 8. 跨角色冲突仲裁器

### 8.1 定位

跨角色冲突仲裁器负责识别并归类角色之间的冲突，并判断是否必须上升到根层主循环。

### 8.2 最小冲突类型

当前最小固定为：

1. `局部目标冲突`
2. `权限边界冲突`
3. `时序冲突`
4. `安全与服务判断不一致`
5. `角色互相等待`

### 8.3 最小裁决纪律

必须固定：

- 角色可以提交理由
- 角色可以发起升级
- 角色不拥有最终优先级裁决权
- 最终优先级仍由根层主循环决定

### 8.4 升级规则

以下情况必须升级回主循环：

- 两个角色都声称当前自己应先动
- 服务推进与安全边界结论冲突
- 让渡链超过本轮上限
- 协同单超时且影响主导需求推进
- 局部拒绝已经影响本轮根需求闭环


## 9. 共享审计与统一回写器

### 9.1 定位

共享审计与统一回写器负责把多角色协同行为写回到同一条事实链上。

### 9.2 必须写回的统一位置

当前必须固定写回到：

- 同一 `动态链`
- 同一 `结果结算面`
- 同一 `因果候选池`

### 9.3 最小审计钩子

任一角色动作至少必须留下：

- `角色来源`
- `发起者`
- `确认者`
- `执行者`
- `回写者`
- `让渡链`
- `升级链`

### 9.4 最小回写纪律

必须固定：

- 任一 `拒绝 / 升级 / 超时` 都必须生成结算
- 任一角色动作都不得写到私有动态面再由后续人工合并
- 因果候选池看到的必须是一条可追踪的协同链，而不是多份无来源碎片

## 11. 先打小切口的多角色金路径

### 11.1 当前最小金路径

当前最小金路径固定为：

`同一轮上下文包`
-> `投影出协调角色 / 资源角色 / 服务角色视图`
-> `协调角色发起协同请求`
-> `资源角色接收并给出局部建议或拒绝`
-> `服务角色补充告知或确认建议`
