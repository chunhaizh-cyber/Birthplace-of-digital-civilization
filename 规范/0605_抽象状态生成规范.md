## 3. 基本定义

### 3.1 实例状态

`实例状态` 指的是：

- 某个具体存在
- 在某个具体时间或时间窗口
- 具有某个具体特征值

它是状态事实，不是抽象概念。

### 3.2 抽象状态

`抽象状态` 指的是：

- 从多个实例状态中抽出来的可复用状态模式
- 它表达的是“某类状态如何成立”，而不是“某次具体状态何时发生”

抽象状态可以分两类：

1. `单特征抽象状态`
- 由单一特征类型下的稳定值域、稳定区间或稳定类别生成

2. `复合抽象状态`
- 由多个单特征抽象状态的稳定交集或稳定组合生成

必须固定：

- 抽象状态不是某个具体存在
- 抽象状态不是某次具体状态事实
- 具体存在只能命中抽象状态，不能定义抽象状态


## 4. 核心边界

### 4.1 抽象状态来自“抽象动作”

必须固定：

`抽象状态不是天然存在于实例状态上的标签，而是对实例状态执行抽象动作后的产物。`

这个抽象动作至少包括：

1. 去掉单次实例时间
2. 去掉单次实例身份对概念身份的直接绑定
3. 保留可复用的状态模式身份
4. 聚合同类证据统计

### 4.2 抽象时删除时间

这里的“删除时间”固定解释为：

- 抽象状态不再保留“这一次是什么时候发生的”这一单次实例语义
- 实例状态上的 `发生时间 / 收到时间` 不进入抽象状态的身份定义

但必须同时固定：

- 抽象动作会删除“单次实例时间”
- 不等于系统内部完全不再保留“最近命中时间 / 首次命中时间 / 最近验证时间”

也就是说：

- `实例状态时间` 属于事实层
- `抽象状态统计时间` 属于概念维护层

两者不得混成一类。

### 4.3 “时间为空”不能作为抽象状态判据

必须硬性固定：

`不是所有时间为空的状态节点，都是抽象状态节点。`

原因至少有 4 条：

1. 时间为空可能只是未填充
2. 时间为空可能只是脏数据或中间态
3. 时间为空的实例状态仍可能绑定具体存在，因此仍是实例事实，不是抽象概念
4. 抽象状态的成立依赖重复命中、稳定区间或稳定交集，而不是依赖“时间字段恰好为 0”

因此统一禁止以下误判：

```text
发生时间 = 0
或 收到时间 = 0
=> 直接视为抽象状态
```

这是错误的。

正确口径必须是：

```text
先对实例状态执行抽象动作
-> 删除实例时间
-> 形成稳定模式候选
-> 再由候选升格为抽象状态
```


## 5. 抽象状态的最小生成条件

一个候选状态要想被生成成 `抽象状态候选`，至少应满足以下条件：

1. 存在可识别的 `特征类型`
2. 存在可比较的 `状态值 / 状态区间 / 状态类别`
3. 至少出现过 `2` 次及以上同类命中
4. 这些命中可以被归并到同一个稳定模式
5. 该模式不依赖某个单一实例时间才能成立

如果是 `复合抽象状态候选`，还必须额外满足：

6. 多个组成状态在观察期内反复共同出现
7. 共同出现不是偶然拼接，而具有稳定交集意义


## 6. 生成流程

## 6.1 单特征抽象状态生成流程

单特征抽象状态的标准生成链固定为：

```text
实例状态样本
-> 按特征类型归桶
-> 对状态值做稳定区间或稳定类别归并
-> 删除单次实例时间
-> 生成抽象状态候选
-> 统计命中次数 / 稳定度 / 适用条件
-> 达到阈值后升格为正式抽象状态
```

必须固定：

1. 先按 `特征类型` 归桶
2. 再按 `值域或类别` 归并
3. 只有归并稳定后才允许生成抽象状态候选

例如：

- 多个实例反复命中 `安全值低位`
- 则可形成单特征抽象状态候选：`安全值低位`

### 6.1.1 单特征抽象状态的身份字段

单特征抽象状态至少应由以下字段共同决定：

- `概念域 = 状态`
- `来源特征类型`
- `值域 / 区间 / 类别`
- `父状态概念主键`
- `适用条件主键`

禁止以下错误身份：

- 只用名字当身份
- 只用某次实例时间当身份
- 只用某个具体存在当身份


## 6.2 复合抽象状态生成流程

复合抽象状态的标准生成链固定为：

```text
多个已存在的单特征抽象状态
-> 观察它们是否稳定共同命中
-> 形成交集候选
-> 删除单次实例时间
-> 统计共同命中次数 / 稳定度 / 适用条件
-> 达到阈值后升格为复合抽象状态
```

必须固定：

1. 复合抽象状态优先建立在“已有单特征抽象状态”之上
2. 不建议直接从裸实例状态一次性跳生成复合抽象状态
3. 复合抽象状态的身份来自“组成状态集合 + 适用条件”，而不是来自某个具体样本

例如：

- `安全值稳定带`
- `服务值中位区`
- `风险安全稳定`

若这三者在相近条件下长期共同命中，才允许形成更高层复合抽象状态候选。


## 7. 抽象动作后保留什么、删除什么

## 7.1 必须删除的实例字段

抽象动作后，以下字段不得继续作为抽象状态身份的一部分：

- `发生时间`
- `收到时间`
- `某次样本的唯一实例主键`
- `某次样本的单次变化原因`

### 7.2 必须保留的抽象字段

抽象动作后，以下字段应保留为概念身份或统计维护字段：

- `来源特征类型`
- `值域 / 类别 / 区间`
- `组成状态集合`
- `适用条件主键`
- `命中次数`
- `首次命中时间`
- `最近命中时间`
- `稳定度`
- `置信度`

必须固定：

- `首次命中时间 / 最近命中时间` 是抽象状态的统计时间
- 它们不等于实例状态的 `发生时间 / 收到时间`


## 8. 世界树挂载规则

抽象状态不应继续按“运行期实例状态节点”挂载。

推荐挂载结构固定为：

```text
世界树根
-> 概念域根
  -> 状态域根
    -> 特征类型状态概念根
      -> 单特征抽象状态
      -> 更细单特征抽象状态
    -> 复合抽象状态分支
```

必须固定：

1. `单特征抽象状态` 挂在对应特征类型根下
2. `复合抽象状态` 挂在状态域聚合分支下
3. 具体存在只保留“命中该抽象状态”的关系，不得反向把抽象状态挂成该存在的私有子节点


## 9. 频次、稳定度与淘汰规则

抽象状态不是一旦生成就永久保留。

必须固定以下规则：

### 9.1 命中增强

当新的实例状态继续命中某个抽象状态时，应至少更新：

- `命中次数`
- `最近命中时间`
- `稳定度`

### 9.2 低频衰减

若某抽象状态长期不再被命中，或命中频率过低，应逐步衰减：

- `稳定度下降`
- `活跃度下降`
- `推荐度下降`

### 9.3 淘汰条件

满足以下任一条件时，可将抽象状态降级或淘汰：

1. 长期未命中
2. 命中频率持续低于最小阈值
3. 与上位状态概念相比不再具备独立区分价值
4. 组成状态交集不再稳定

### 9.4 淘汰后的处理

淘汰不等于直接物理删除。

建议优先分三种处理：

1. `降级为候选`
2. `保留为冷概念`
3. `最终清理`

若系统要保留成长轨迹，则优先降级，不直接硬删。


## 10. 禁止事项

后续必须明确禁止以下做法：

1. 把 `发生时间 = 0` 的实例状态直接认定为抽象状态
2. 把某个具体存在的私有长期状态直接提升为抽象状态
3. 不经过重复命中和稳定性检查，就直接把一次状态升格成抽象状态
4. 用自然语言名字代替抽象状态身份
5. 把复合抽象状态误挂到单一特征细分主干上


## 11. 一句话收口

可收敛为一句话：

`抽象状态不是“时间为空的状态节点”，而是“对多个实例状态执行抽象动作、删除单次实例时间后，经稳定归并形成的可复用状态模式”；时间为空只是抽象动作后的表现之一，不是抽象状态成立的充分条件。`
