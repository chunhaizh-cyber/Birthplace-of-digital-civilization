# 动作动态即因果账本规范 20260512

## 0. 权威修订

本规范受以下硬规则约束：

```text
规范/方法动作动态硬规则规范20260512.md
```

凡是生成“方法动作动态”的变化，都必须绑定具体方法。线程只能调度和调用方法，不能作为方法动作动态的动作来源。

## 1. 定义

系统内部不再单独创建“因果记录”结构。自我内部生成的因果信息统一表达为：

```text
动作动态节点
```

其中“方法动作动态”表示：

```text
某个方法，在某个场景下，
对某个主体的某个特征，
造成了一次可记录事实。
```

方法动作动态不以“成功 / 失败”作为根概念。`成功 / 失败` 只能作为普通结果特征、兼容字段或派生判断。动作动态本体应优先记录：

```text
方法运行了什么。
作用主体是什么。
作用特征是什么。
输入场景是什么。
输出场景是什么。
事实类型是什么。
相位是什么。
如果中断，中断原因是什么。
```

如果变化没有对应方法，只能落为：

```text
观察动态
外部事件动态
环境动态
未知来源动态
```

不能伪装成自我方法动作动态。

## 2. 动作来源规则

旧口径废止：

```text
哪个线程 / 层真实提交特征变化，哪个线程 / 层生成动作动态。
```

新口径：

```text
哪个域拥有提交权限，哪个域的线程负责编排调用对应提交类方法；
动作动态的动作来源必须是该方法。
```

| 域 | 编排者 | 必须对应的方法来源 | 变化内容 |
| --- | --- | --- | --- |
| 世界域 | 执行方法 / 世界树提交层 | 具体执行方法，如 `改变特征值` | 真实世界事实、特征值、状态节点、动态事实 |
| 任务域 | 任务管理界面线程 | `提交任务状态变化`、`同步筹办回执到任务`、`同步执行回执到任务` | 任务状态、当前方法、任务回执 |
| 需求域 | 自我线程 | `回写需求满足`、`派生需求入树` | 需求状态、需求树、需求结算 |
| 方法域 | 学习 / 方法管理流程 | `提交方法状态变化`、`建立条件结果对` | 方法状态、条件结果对、学习产物 |

禁止：

```text
1. 新增任务完成因果记录结构。
2. 新增需求满足因果记录结构。
3. 新增世界事实因果记录结构。
4. 用输出结果场景承载任务状态动作动态。
5. 由任务管理工作线程生成已提交的任务状态方法动作动态。
6. 由自我线程生成任务状态方法动作动态。
7. 由线程直接作为方法动作动态来源。
```

## 3. 三通道边界

本能函数继续遵守三通道：

```text
运行存在 / 函数返回回执：
    给任务管理层使用，承载任务层含义。

输出结果场景：
    给世界树提交层使用，只承载世界事实。

方法虚拟存在：
    给方法自身使用，只承载方法记忆和运行账。
```

动作动态生成位置不改变三通道语义。

任务筹办 / 任务执行是任务管理生命周期普通函数，只产出过程结果和任务层回执。任务管理界面线程读取过程结果后，必须调用同步回执 / 提交类本能方法，才能把任务域变化落成任务状态方法动作动态。

## 4. 动作动态链

动作动态之间应通过特征形成链式解释能力，而不是创建新因果结构。

建议复用或补充以下特征入口：

```text
来源动作动态
上游动作动态
满足证据动态
触发任务
来源需求
来源运行存在
来源输出结果场景
```

标准链路：

```text
世界事实动作动态
    -> 任务状态动作动态
        -> 需求状态动作动态
```

示例：

```text
D0 = 改变服务值动作动态
D1 = 提交任务状态变化动作动态
D2 = 回写需求满足动作动态

D1.来源动作动态 = D0
D2.来源动作动态 = D1
D2.满足证据动态 = D0
```

## 5. 命名收口

| 旧说法 | 新说法 |
| --- | --- |
| 任务完成因果 | 任务状态动作动态 |
| 需求满足因果 | 需求状态动作动态 |
| 世界事实因果 | 世界特征动作动态 |
| 方法学习因果 | 方法状态 / 方法结构动作动态 |
| 因果证据 | 来源动作动态 / 上游动作动态 / 输入输出场景 |
| 成功日志 / 失败日志 | 事实类型 / 结果特征 / 中断原因 |

## 6. 函数级注释要求

后续实现本规范时，新增或修改的函数必须写函数级注释，至少说明：

```text
1. 本函数属于哪个域：世界 / 任务 / 需求 / 方法。
2. 本函数是否调用或实现明确方法。
3. 本函数生成的动作动态以哪个方法为动作来源。
4. 本函数生成的动作动态记录哪一个特征变化。
5. 本函数生成的动作动态记录哪一种事实类型。
6. 本函数不得写哪些域。
7. 本函数的输入动态 / 输出动态链路字段来源。
```

注释示例：

```cpp
// 任务域提交动作：
// 本函数由任务管理界面线程调用，动作来源是“提交任务状态变化”本能方法。
// 它只提交任务状态旧值到新值，并生成任务状态方法动作动态。
// 不写需求树，不写世界事实，不替代工作线程回执。
```

## 7. 最终口径

```text
动作动态就是因果账本。
系统不再另造“因果信息”结构。
任务完成、需求满足、世界事实变化、方法学习结果，都落成各自域的动作动态。
其中凡是方法动作动态，都必须绑定一个可调用方法作为动作来源。
动作动态记录事实，不以成功 / 失败作为根分类。
```
