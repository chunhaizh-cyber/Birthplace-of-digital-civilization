- 状态信息到底在语言上描述什么

本文服从以下硬边界：

1. `状态信息` 仍是真值层中的结构化状态事实，不是语言层主观修辞。
2. 自然语言只是显示层和解释层，不得反向改写状态真值。
3. `状态信息` 可以在语言上看作形容词性表达，但不得因此丢失 `主体 / 特征 / 时间 / 状态值` 这四类最小结构。
4. 问“现在什么样”时，优先使用 `状态信息`；问“正在怎么变”时，必须回到 `动态信息`。


## 3. 核心口径

必须固定：

1. `状态信息` 描述的是：`某存在在某时间的某个特征处于什么状态`。
2. 在自然语言中，`状态信息` 可以优先理解成一种 `形容词性描述` 或 `表语描述`。
3. 但 `状态信息` 不等于“只有形容词”。
4. 当状态值本身是对象、指针、枚举、数值或路径时，语言层仍必须把它组织成完整状态句，而不能只截成单个词。

一句话固定：

`状态信息是真值层中的状态事实，在语言层通常落成“主体当前怎样”的形容词性或表语表达。`


## 4. 依据的字段面

当前 `状态节点主信息类` 可直接提供的关键字段包括：

- `状态域`
- `收到时间`
- `发生时间`
- `状态主体`
- `状态特征`
- `状态值`
- `对应信息节点`
- `是否变化`
- `变化原因类别`
- `变化原因说明`

当前统一理解为：

- `状态主体`：回答“谁处于该状态”
- `状态特征`：回答“处于哪一项状态维度”
- `状态值`：回答“现在是什么样”
- `发生时间 / 收到时间`：回答“在什么时候是这样”
- `对应信息节点`：回答“这条状态事实对应哪一个对象或来源信息”
- `是否变化 / 变化原因类别 / 变化原因说明`：回答“这条状态是否带有变化背景及原因说明”


## 5. 状态信息与形容词性表达

### 5.1 为什么可以看作形容词

状态信息最常回答的是：

- `它现在是什么样子`

这类问题在自然语言里最常落成：

- `X 是 Y`
- `X 处于 Y`
- `X 的 A 为 B`

其中的 `Y / B` 往往就是形容词、状态词、表语短语或结果短语，例如：

- `运行中`
- `未启动`
- `完成`
- `待机`
- `偏高`
- `可用`

因此，状态信息在语言层可以优先视作形容词性表达来源。


### 5.2 为什么不能简化成“只有形容词”

必须继续固定：

1. 单个形容词只能表达“怎样”，不能表达“谁怎样”。
2. 真实状态表达仍要保留主体。
3. 若状态挂在某个特征上，还要保留特征维度。
4. 若状态带时间，还要保留时间信息。

例如：

- 可以说：`安全根任务运行中`
- 更完整应说：`安全根任务当前处于运行中`
- 若涉及特征维度，应说：`自我的当前主任务为安全根任务`


## 6. 最小语义骨架

状态信息最小骨架固定为：

`状态主体 + 状态特征 + 状态值 + 状态时间`

其中：

- `状态主体` 必须回答“谁”
- `状态特征` 必须回答“哪一维”
- `状态值` 必须回答“现在怎样”
- `状态时间` 在需要时回答“什么时候这样”

若缺少特征维度，可退化为：

`状态主体 + 当前状态值`

若缺少主体，只能作为局部摘要，不得长期替代完整句。


## 7. 表达主链

状态信息的自然语言主链固定为：

`状态节点 -> 状态表达单元 -> 状态句型 -> 自然语言`

其中状态表达单元最少应包含：

- `主体名称`
- `特征名称`
- `状态值文本`
- `是否当前`
- `发生时间文本`
- `是否变化`
- `变化原因文本`

必须固定：

1. `状态节点` 不直接等于最终句子。
2. 语言模块先抽出状态表达单元，再决定用哪种状态句型。
3. 控制面板、日志、调试页统一消费语言模块结果，不各自手拼状态句。


## 8. 句型层

### 8.1 形容词性主句

适用条件：

- 状态值本身可直接当状态词或形容词性结果
- 特征维度已经很明确或可以省略

推荐模板：

- `X 当前处于 Y`
- `X 目前为 Y`
- `X 现在是 Y`

示例：

- `安全根任务当前处于运行中`
- `服务根任务目前为未启动`


### 8.2 特征型状态句

适用条件：

- 必须保留特征维度
- 状态值不是裸形容词，而是对象名、数值、枚举值或方向值

推荐模板：

- `X 的 A 为 B`
- `X 当前的 A 为 B`

示例：

- `自我的当前主任务为安全根任务`
- `任务管理的当前下一步去向为 3`


### 8.3 带时间状态句

适用条件：

- 需要保留发生时间或收到时间
- 不是纯“当前快照”，而是审计、回放、解释或日志语境

推荐模板：

- `在 T 时，X 的 A 为 B`
- `X 在 T 处于 Y`

示例：

- `在上一周期，自我的服务衰减步长为 1`
- `安全根任务在该时刻处于运行中`


### 8.4 带原因状态句

适用条件：

- 状态值本身是指针/对象，但当前节点已携带更自然的变化原因说明
- 或需要在解释层说明该状态是如何来的

推荐模板：

- `X 的 A 为 B`
- `X 的 A 由于 C 变为 B`

示例：

- `任务管理的当前方法指针为停止当前试探并进入保守态`
- `任务管理的当前主任务由于治理结果变为安全根任务`


## 9. 状态值文本规则

### 9.1 值为状态词或枚举

优先落成：

- `运行中`
- `未启动`
- `完成`
- `待机`

推荐句型：

- `X 当前处于 Y`


### 9.2 值为对象名或指针指向对象

必须先转成对象的自然语言名称，再进入状态句。

推荐句型：

- `X 的 A 为 B`

不得长期显示成：

- `ptr#...`


### 9.3 值为数值

数值状态默认落成特征型状态句：

- `X 的 A 为 1`
- `X 的 A 为 80`

若已有上位语义可转成定性状态，可在解释层额外生成：

- `X 的 A 偏高`
- `X 的 A 偏低`

但不得拿解释层定性词替代真值层数值。


## 10. 时间规则

必须固定：

1. `当前面板` 优先省略时间，默认按“当前状态”理解。
2. `日志 / 回放 / 审计` 场景中，可显式补入 `发生时间` 或 `收到时间`。
3. 若 `发生时间` 与 `收到时间` 都存在，优先用 `发生时间` 表达状态成立时刻。

推荐理解：

- 当前显示：`自我的当前主任务为安全根任务`
- 回放显示：`在上一周期，自我的当前主任务为安全根任务`


## 11. 与特征信息、动态信息的分工

### 11.1 与特征信息的关系

- `特征信息` 更偏底层真值槽位
- `状态信息` 更偏“该特征在某一时刻的状态快照”

一句话固定：

`特征` 回答“有什么槽位”，`状态` 回答“这个槽位现在怎样”。`


### 11.2 与动态信息的关系

- `状态信息` 负责“现在什么样”
- `动态信息` 负责“刚刚怎样变化 / 正在怎样变化”

必须固定：

- 不得拿 `动态信息` 冒充当前状态
- 不得拿 `状态信息` 冒充动作过程


## 12. 推荐示例

### 12.1 当前状态

- `安全根任务当前处于运行中`
- `服务根任务当前处于未启动`
- `自我的当前主任务为安全根任务`


### 12.2 数值状态

- `自我的服务时序衰减步长为 1`
- `自我的安全增加步长为 1`


### 12.3 对象状态

- `任务管理的当前方法指针为停止当前试探并进入保守态`
- `任务管理的最近结果指针为治理结果_维护_任务待启动_进入运行_运行中_停止当前试探并进入保守态`


### 12.4 带时间状态

- `在该周期，自我的当前主方法为停止当前试探并进入保守态`
- `安全根任务在该时刻处于运行中`


## 13. 工程落点

当前工程中，这部分能力应归属于 `语言模块`，而不是散落在控制面板或各服务类中直接手拼。

最小接口建议固定为：

- `生成状态信息名称(...)`
- `生成状态信息自然语言(...)`
- `生成带时间状态表达(...)`

必须固定：

1. 控制面板只消费语言模块结果，不自行拼装状态句。
2. 同一套状态语言规则应同时服务控制面板、日志与后续调试输出。
3. 状态值若是对象指针，必须优先解析为对象自然语言名称。
4. 若当前只实现最小命名链，也应保证状态节点优先落成 `X 的 A 为 B` 或 `X 当前处于 Y` 这两类句型。
