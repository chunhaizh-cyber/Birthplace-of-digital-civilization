## 3. 总定义与总纲

### 3.1 总定义

必须固定：

`工程口语中的二次特征，可以作为从2次特征到n次特征的高次特征泛称；其中严格2次特征不是原始特征值，也不是动态事实本身，而是建立在状态或动态之上的第一层高次语义结果。`

补充固定：

- `比较型二次特征` 是严格2次特征的重要子类。
- 但严格2次特征不只来自“两个值相减”；只要某种第二层区分是稳定、可复用、可命名、可回指证据的，它都可以是合法二次特征。

它负责回答的不是“这个值是多少”，而是：

- `现在是什么`
- `它是怎么变的`
- `相对什么成立`
- `这个过程在结构上属于哪一类`

### 3.2 两大主域

以下“状态/存在型 / 动态型”分域，默认优先描述 `严格2次特征` 的来源组织与用途；`3次` 及以上的高次特征继续继承这些来源语义，但算术递推按 [0575_二次特征泛称与n次特征递推生成规范.md](D:\鱼巢\规范\0575_二次特征泛称与n次特征递推生成规范.md) 执行。

二次特征最终固定分成两大主域：

1. `状态/存在型二次特征`
   - 描述当前态、当前关系、当前满足性、当前偏离、当前约束状态
   - 回答的是：`现在是什么`

2. `动态型二次特征`
   - 描述变化量、方向、持续、速率、恢复/衰减、路径、转折、峰谷、回返、闭环、周期/相位
   - 回答的是：`它是怎么变的`

当前代码中的 `枚举_二次特征域` 仍使用：

```cpp
enum class 枚举_二次特征域 : std::uint8_t {
    未定义 = 0,
    存在 = 1,
    动态 = 2,
};
```

这里的 `存在` 语义上承接本文所说的 `状态/存在型二次特征`。

### 3.2.1 两个正式来源

二次特征的正式来源固定只有两类：

1. `状态信息来源`
   - 直接从两个可比较状态值相减得到差值
2. `动态信息来源`
   - 先把动态标准化成可比较的前后状态或对齐时间窗，再回到减法生成

进一步细则按 [0574_二次特征双来源与通用减法生成规范.md](D:\鱼巢\规范\0574_二次特征双来源与通用减法生成规范.md) 执行。

### 3.3 总纲

本文必须固定以下总纲：

1. `二次特征在工程口语里可作为2次到n次的高次特征泛称；其中严格2次特征是建立在状态或动态之上的第一层高次语义结果。`
2. `状态/存在型二次特征描述当前态语义，动态型二次特征描述过程语义。`
3. `比较方式在比较型2次特征中必须明确；但高阶特征空间并不只包含比较型生成法。`
4. `动态型二次特征必须建立在已经成立的动态之上；没过阈值，连动态都不生成，也就没有动态型二次特征。`
5. `非相邻不直接比较，必须先降成低层动态，再聚合成多级动态，然后在同级相邻上继续生成二次特征。`
6. `二次特征可以进入因果、任务、方法链，但运行时不得裸判二次特征；必须先经原语层转译，再由选择函数消费。`
7. `“是否是目标”属于任务/方法上下文角色，不属于二次特征本体固有属性。`

## 5. 统一生成主链与硬规则

### 5.1 统一生成主链

二次特征的统一生成主链固定为：

```text
特征比较
-> 状态
-> 动态
-> 二次特征
-> 原语判定
-> 任务 / 方法 / 因果 / 学习 / 选择函数继续消费
```

分层职责固定为：

- `特征类` 管值、区间、稳态、误差、比较核
- `状态类` 管某时刻某主体的状态
- `动态类` 管状态变化、时间、来源、路径、层级
- `二次特征类` 管二阶语义结果与引用
- `原语层` 管运行时条件、方向、结果、冲突、稳定性与升阶判定
- `因果 / 任务 / 方法` 管条件、结果、目标、动作、评估与绑定

### 5.2 阈值硬规则

必须固定：

1. `没过阈值，连动态都不生成。`
2. `未形成合法动态，就不得生成动态型二次特征。`
3. `未感知微差不得事后累计补生成动态。`
4. `累积/聚合只能作用于已经成立的低层动态，不得作用于未成动态的微小变化。`

一句话收口：

`动态型二次特征只能建立在已经成立的动态之上。`

### 5.3 非相邻硬规则

必须固定：

`非相邻不直接比较。必须先降成低层动态，再聚合成多级动态，然后在同级相邻上继续生成二次特征。`

因此必须继续固定：

1. 不允许对非相邻状态直接做首尾硬差分，就宣称得到了完整动态型二次特征。
2. 若确实需要跨更长过程取过程语义，唯一合法路径是：

```text
原子动态
-> 聚合动态
-> 同级相邻动态对
-> 动态型二次特征
```

3. 路径、回返、闭环、周期/相位等高级语义，优先建立在聚合动态之上，不得越过动态层从裸状态序列硬算。

### 5.4 多节点过程规则

当动态节点数大于 `2` 时，必须同时保留两层结果：

1. `分步动态型二次特征`
   - 相邻节点对生成

2. `整体动态型二次特征`
   - 首尾整体、总时长、整体速率、整体路径摘要等

并且必须固定：

- `分步特征` 与 `整体特征` 并存，不能互相替代
- 当节点数大于 `2` 时，`路径特征优先于首尾特征`
- 首尾特征只能描述结果，不代表完整过程形态


## 6. 状态/存在型二次特征生成规范

### 6.1 定义

状态/存在型二次特征描述的是：

- 当前关系
- 当前满足性
- 当前偏离
- 当前约束状态

它回答的是：

`现在是什么`

它不是：

- 动态时长
- 变化率
- 首尾差分
- 路径形态

### 6.2 生成前提

状态/存在型二次特征必须基于当前有效状态，至少满足：

1. 同一场景
2. 同一层级
3. 比较对象特征类型可比较
4. 比较方式由 `特征类` 负责，不在二次特征层重复定义

### 6.3 生成规则

状态/存在型二次特征的生成规则固定为：

1. 抽取当前状态锚点
2. 按特征类型执行合法比较或判定
3. 生成当前态语义
4. 填充主体、客体、左右对象与来源场景
5. 回写到场景 `二次特征索引`

### 6.4 适合承载的内容

状态/存在型二次特征适合表达：

- 占用
- 冲突
- 接近
- 包含
- 相交
- 是否满足某条件
- 相对稳态、目标态或边界的当前偏离

当前仓库第一阶段已落地的代表项是：

- `接近程度`
- `冲突`

### 6.5 不应承载的内容

状态/存在型二次特征不应直接承载：

- 持续时间
- 变化率
- 路径形态
- 首尾过程差分

一句话收口：

`状态/存在型二次特征是当前态语义。`


## 7. 动态型二次特征生成规范

### 7.1 定义

动态型二次特征描述的是：

- 变化量
- 变化方向
- 持续时间
- 变化率
- 恢复/衰减
- 路径形态
- 转折、峰谷、回返、闭环
- 周期、相位、重复模式

它回答的是：

`它是怎么变的`

### 7.2 生成前提

动态型二次特征必须以合法动态为前提，至少满足：

1. 先有动态，后有动态型二次特征
2. 动态的起止状态必须可比较
3. 时间段必须有效
4. 动态必须已经被感知成立

### 7.3 基本生成规则

动态型二次特征的统一生成规则固定为：

1. 获取动态信息中的全部相关特征类型
2. 获取动态起始和结束的特征值，同类型比较得到变化量与方向
3. 若是多节点动态，则每两个相邻节点都应生成分步二次特征
4. 加入时间因素，得到变化率等时间归一语义
5. 首尾整体特征和分步特征并存，不能互相替代

### 7.4 动态二次特征母式总则

必须固定：

`动态二次特征的母式，不是“每种二次特征各有一套算法”，而是“全体动态共用一套固定生成流程”；任何动态对象都应能通过统一母式函数，直接生成其适用的二次特征集合。`

动态母式的详细条文、可比性、阈值过滤、分步/整体/路径并存、路径优先、置信度继承与异常处理，统一按 [0576_动态二次特征统一生成规范.md](D:\鱼巢\规范\0576_动态二次特征统一生成规范.md) 执行。

这里的“通用”必须同时包含两层含义：

1. `入口通用`
   - 所有动态都走同一个生成入口
2. `输出按条件裁剪`
   - 不是每个动态都生成全部二次特征，而是统一母式函数根据动态自身条件输出适用子集

因此必须固定以下条文：

1. 动态二次特征的母式应为全局通用母式。
2. 母式的获取流程应固定，不因具体动态对象而改变。
3. 任何动态对象，只要满足动态成立条件，均应能通过统一母式函数直接获取其适用的二次特征集合。
4. 母式函数应先将动态标准化为同级相邻可比较动态单元，再执行统一生成流程。
5. 母式函数统一负责原子级、整体级、路径级二次特征的生成。
6. 不适用的二次特征不生成，不得因母式通用而强行补齐。
7. 不同比较方式由特征类负责提供，母式函数只调用标准比较接口，不在二次特征层重复定义比较规则。
8. 非相邻动态不得直接生成跨层二次特征，必须先降动态并聚合成同级动态后，再经母式函数生成。
9. 二次特征实例应保存生成结果、来源信息与版本信息，不必把母式本身作为长期本体字段保存。

推荐把统一入口固定理解为：

```cpp
std::vector<二次特征节点类*> 获取动态二次特征(
    const 动态节点类& 动态,
    const 二次特征生成配置& 配置 = {});
```

在当前 `V1` 阶段，统一入口固定只要求稳定生成：

- `变化量`
- `变化方向`
- `持续时间`
- `变化率`

推荐把 `V1` 伪代码固定为：

```cpp
获取动态二次特征(动态 D, 配置 C):
    结果集 = {}

    if D 无效:
        return {}

    标准单元集 = 标准化为同级相邻动态单元(D)
    if 标准单元集为空:
        return {}

    for 单元 in 标准单元集:
        if 单元.初始状态无效 or 单元.结果状态无效:
            continue
        if 单元.动态特征无效:
            continue

        比较结果 = 获取特征比较结果(单元.初始状态, 单元.结果状态)
        if !比较结果.可比较:
            continue

        Δv = 比较结果.变化量
        dir = 比较结果.方向
        Δt = 单元.时间.时长()

        if C.生成变化量:
            结果集 += 生成_变化量(单元, Δv)
        if C.生成变化方向:
            结果集 += 生成_变化方向(单元, dir)
        if C.生成持续时间 and Δt > 0:
            结果集 += 生成_持续时间(单元, Δt)
        if C.生成变化率 and Δt > 0:
            结果集 += 生成_变化率(单元, Δv / Δt)

    if D 具备聚合来源链:
        结果集 += 生成_整体变化量(D)
        结果集 += 生成_整体持续时间(D)
        if D.总时长 > 0:
            结果集 += 生成_整体变化率(D)

    将结果集写回所属场景索引
    用结果集参与因果模板匹配
    return 结果集
```

统一母式函数的推荐流程固定为六步：

1. `动态标准化`
   - 先把输入动态转换成同级、可比较、相邻的标准动态单元
2. `提取可比较特征`
   - 找到该动态涉及的特征，并调用特征类自己的比较核
3. `生成原子级二次特征`
   - 变化量、变化方向、绝对幅度、持续时间、变化率、状态转移等
4. `生成整体级二次特征`
   - 首尾整体变化量、总时长、总体平均变化率、分步数量、最大分步变化、最小分步变化等
5. `生成路径级二次特征`
   - 转折点数量、峰值偏移、谷值偏移、回返类型、闭环类型、重复模式、相位模式、周期/频率等
6. `分类映射与输出裁剪`
   - 统一处理适用性、无效值、概念映射与输出筛选

可收敛为一句话：

`动态二次特征母式 = 一套对所有动态通用、对输入条件固定、对输出按适用性裁剪的统一生成协议。`

### 7.5 速率统一口径

动态型二次特征里的速率类结果统一收口为一句母式：

`速率度量 = 某种状态差分或过程偏移 ÷ 某种时间尺度`

因此：

- 在动态规范里可叫 `变化率 / 速率二次特征`
- 在业务模块里可派生为 `恢复速度 / 衰减速度 / 步长 / 收敛速度`

它们在二次特征层的共同归位，应统一落到：

- `域 = 动态`
- `当前代码兼容期可暂落到：主母式 = 时间归一`
- `时间归一方式 = 分步时长 / 总时长 / 频率窗口`

### 7.6 适合承载的内容

动态型二次特征适合表达：

- 变化量
- 方向
- 持续时间
- 变化率
- 恢复速度 / 衰减速度
- 峰谷偏移
- 转折数
- 回返类型
- 闭环类型
- 周期 / 相位 / 重复模式

当前仓库第一阶段已落地的代表项是：

- `变化量`
- `变化方向`
- `变化率`

### 7.7 不允许的动态型偷算

以下做法必须禁止：

1. 没有合法动态，却直接生成动态型二次特征
2. 拿未感知到的微差做事后累计，补生成高层动态
3. 直接跨非相邻状态做首尾硬比较，冒充完整过程语义
4. 用首尾结果替代多节点路径语义

一句话收口：

`动态型二次特征是过程语义。`


## 8. 母结构与字段口径

### 8.1 有效母结构与结果分类族

必须固定：

这里列出的 `存在关系 / 差分 / 时间归一 / 路径 / 参照偏离 / 状态转移 / 周期相位 / 边界跨越 / 结构变更 / 层级聚合`，
不应再理解为“每一种二次特征各有一套独立母式算法”。

更稳的口径是：

- `全局统一母式` 属于生成器层、规范层、函数层
- 这些枚举项属于统一母式函数内部使用的 `结果分类族 / 生成语义分支 / 兼容期标签`

在“未感知微差不得补算成动态”的修正后，二次特征层保留下来的有效母结构固定为：

1. `存在关系`
2. `差分`
3. `时间归一`
4. `路径`
5. `参照偏离`
6. `状态转移`
7. `周期/相位`
8. `边界跨越`
9. `结构变更`
10. `层级聚合`

当前代码已正式落地的母结构枚举包括：

- `存在关系`
- `差分`
- `时间归一`
- `路径`
- `参照偏离`
- `周期相位`
- `状态转移`

后续仍需补齐的扩展项包括：

- `边界跨越`
- `结构变更`
- `层级聚合`

### 8.2 比较方式不进二次特征主信息

必须固定：

`比较方式不进二次特征主信息。`

原因是：

1. 不同特征类型的比较核，已经由 `特征类` 负责。
2. 二次特征层只消费比较后的合法结果，不重复保存一份平行比较真值。
3. 若把比较核复制到二次特征层，会造成双真值源与后续漂移。

### 8.3 统一母式属于生成器层，实例层保存结果与追溯

必须固定：

`母式` 属于规范层、生成器层、函数层；`二次特征实例` 主要保存生成结果、来源信息与版本信息。

因此更合理的实例层最小承载，应优先包括：

- `种类`
- `域`
- `粒度`
- `基准类型`
- `时间归一方式`
- `来源状态 / 来源动态 / 来源场景`
- `来源动态层级 / 来源动态路径签名`
- `值形态`
- `值`
- `置信度`
- `规则版本`

不建议再把以下信息视为实例固有本体属性：

- `比较方式`
- `每种特征自己的生成函数名`
- `把主母式当成每条实例私有算法身份`

必须继续固定：

1. 当前代码里保留的 `主母式` 字段，只能视为兼容期的结果分类标签或生成语义缓存。
2. `主母式` 不得再被解释为“这条实例自带的一套独立生成算法”。
3. 当前代码中的 `规则版本`，应视为 `生成版本` 的正式承载。
4. 长期更稳的追溯口径是：`来源链 + 粒度 + 基准 + 值形态 + 规则版本`。

### 8.4 生成语义字段的解释优先级

在兼容当前代码字段的前提下，二次特征的生成语义应优先按以下顺序解释：

- `种类`
- `域`
- `粒度`
- `基准类型`
- `时间归一方式`
- `来源状态 / 来源动态 / 来源场景`
- `来源动态层级 / 来源动态路径签名`
- `值形态`
- `规则版本`

`主母式` 若仍存在，只作为兼容期的分类辅助字段使用。

因此必须固定：

1. `度量签名_链键` 只适合作为索引壳与链式定位壳
2. `度量签名_链键` 不应单独承担完整语义
3. 真正的生成语义应以显式字段组与 `规则版本` 为准
4. `主母式` 不再是规范层的最高解释入口

## 9. 二次特征与因果、任务、方法的关系

### 9.1 标准用途

二次特征至少有五种正式用途：

1. `观察事实`
2. `因果条件`
3. `因果结果`
4. `任务目标 / 方法目标`
5. `方法评估依据`

必须补充固定：

`二次特征的主要用途之一，是找不同。`

这里的“找不同”至少包括：

- 找两个对象之间的不同
- 找同一对象前后状态之间的不同
- 找同样时间段内变化方式的不同
- 找多个特征类型之间联动关系的不同

若严格 `2次特征` 已经不足以把差异继续拆清、压细、追深，则允许继续递推到 `3次 / 4次 / ... / n次特征` 做进一步差异分析。

必须补充固定：

- `2次特征` 先负责判方向。
- `3次特征` 及更高 `n次特征` 不再重复判方向，而主要负责在 `同一需求轴` 上对 `同向二次特征候选` 做排序，判断谁更接近需求。

### 9.2 能否成为任务或方法目标

必须固定：

`某些二次特征可以成为任务或方法的目标。`

典型例子包括：

- 消除冲突
- 降低压缩程度
- 提高恢复速度
- 维持某种关系

### 9.3 但“目标”不是本体固有属性

必须继续固定：

1. `二次特征` 负责回答 `它是什么`
2. `任务 / 方法` 负责回答 `拿它来做什么`
3. “目标角色”不应被焊进 `二次特征主信息类` 作为长期本体字段

因此：

- `来源任务主键 / 来源方法主键` 只负责来源追溯
- 它们不自动等于“当前这条二次特征就是目标”

### 9.4 推荐的目标绑定结构

若任务或方法需要把二次特征作为正式目标，推荐在任务/方法层使用独立绑定结构，最小可固定为：

```cpp
enum class 枚举_目标对象类型 : std::uint8_t {
    未定义 = 0,
    状态 = 1,
    动态 = 2,
    二次特征 = 3,
};

enum class 枚举_目标作用 : std::uint8_t {
    未定义 = 0,
    达到 = 1,
    维持 = 2,
    消除 = 3,
    降低 = 4,
    提高 = 5,
};

struct 任务目标绑定 {
    枚举_目标对象类型 目标对象类型 = 枚举_目标对象类型::未定义;
    可解析引用<基础信息节点类> 目标对象{};
    枚举_目标作用 目标作用 = 枚举_目标作用::未定义;
    std::optional<I64区间> 目标区间{};
    bool 硬约束 = false;
    std::int32_t 优先级 = 0;
};
```

必须固定：

- `目标绑定` 属于任务/方法层，不属于二次特征本体层
- 任务侧与方法侧可复用同构结构
- 目标成立与否，最终仍必须回到重新生成的状态、动态、二次特征与因果验证


## 10. 落地优先级

实现推进必须坚持“小闭环优先”，按以下顺序推进：

1. 先做最基本的 `状态/存在型二次特征`
   - 接近、冲突、满足性、偏离

2. 再做最基本的 `动态型二次特征`
   - 变化量、方向、持续、变化率

3. 先打通：

```text
生成
-> 模板匹配
-> 场景回写
-> 因果 / 任务 / 方法消费
```

4. 最后再扩展高级过程项：
   - 路径
   - 转折
   - 峰谷
   - 回返
   - 闭环
   - 周期 / 相位

### 10.1 `V1` 最小样例

`统一母式函数 V1` 的最小验收样例固定为三组：

1. `样例 A：两点动态，线性上升`
   - 期望生成：`变化量 / 变化方向 / 持续时间 / 变化率`
2. `样例 B：三点动态，先升后降`
   - 期望生成：相邻分步结果与首尾整体结果并存
   - 首尾结果不得替代路径事实
3. `样例 C：非相邻跨段输入`
   - 期望行为：不得直接生成跨段动态型二次特征
   - 必须先形成合法聚合动态，再进入统一入口

### 10.2 `V1` 最小验收

当前阶段只要求以下五条闭环成立：

1. 两点合法动态能稳定产出 `变化量 / 方向 / 持续时间 / 变化率`。
2. 三点及以上过程能同时保留分步结果与整体结果。
3. 非相邻输入不会绕过聚合层直接生成跨段结果。
4. 生成结果能回写 `场景.二次特征索引`。
5. 生成结果能直接参与 `因果模板` 的 `条件模板 / 结果模板` 匹配。

## 11. 一句话总括

可收敛为一句话：

`二次特征在工程里可作为2次到n次的高次特征泛称；其中严格2次特征建立在状态或动态之上，且1次转2次时必须先知道比较方法；一旦进入2次层，正式数值统一按int承载，3次及以上继续通过同阶相减递推生成；状态/存在型描述当前态语义，动态型描述过程语义，二次特征可以进入因果、任务、方法链，但“目标”属于任务/方法对它的使用角色，不属于二次特征本体的固有属性。`
