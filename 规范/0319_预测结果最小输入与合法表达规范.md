# 0319 预测结果最小输入与合法表达规范

## 1. 目标

本文用于把 `预测结果` 在当前体系中的最小输入面和合法表达方式固定下来。

本文要回答的是：

- `预测结果` 到底是什么
- 它最少要怎样表达，才允许进入根层重判或执行前门控
- 为什么不能把自然语言预期、方法名、主键字段组合、摘要直接当预测结果
- 预测结果如何与当前值比较并生成 `预测结果方向`

## 3. 核心结论

必须固定以下结论：

1. `预测结果` 不是一句“这个方法大概会成功”的口头预期，而是一组可比较的 `带值特征`。
2. `预测结果` 若要进入根层重判、执行前门控或任务执行比较链，必须先满足合法表达条件。
3. `方法名`、`动作名`、`结果主键`、`自然语言说明`、`摘要文本` 都不能直接充当预测结果本体。
4. 合法预测结果至少必须能回答：
   - 它预测了哪个特征类型
   - 该特征预测值是多少
   - 这个预测值如何与当前值比较
5. `预测结果方向` 只能来自 `预测值 - 当前值` 的比较结果，不能直接从“看起来会变好/会变坏”的文字判断中生成。
6. `预测结果` 可以进入根层重判，也可以进入方法/任务层的候选比较，但它本身不是最终裁判依据。
7. 若预测结果不能落成可比较的带值特征，则它最多只能保留为候选描述或说明，不得直接进入正式判定链。

一句话收口：

`预测结果只有在被结构化表达为一组可比较的带值特征之后，才允许进入正式判定链；否则它只是描述，不是判断输入。`


## 4. 定义

`预测结果` 指的是：

- 当前方法、步骤或治理路径对“执行后结果特征会怎样变化”的结构化预期表达

它回答的是：

- 哪个特征会变化
- 变化后预计到哪个值
- 预计变化方向是什么

它不是：

- 方法名字
- 结果主键
- 结果摘要
- 成功/失败口号
- 任意自然语言描述


## 5. 合法表达的最小形式

### 5.1 最小合法单位

一个最小合法预测结果项，至少必须包含：

- `特征类型`
- `预测值`

若要正式进入比较链，还必须能取得：

- `当前值`

因此最小比较单位应为：

```text
特征类型 + 当前值 + 预测值
```

### 5.2 最小合法结果组

一个合法预测结果，最少应表达为：

- `一个或多个预测结果项`

每一项都应是：

- `特征类型 + 预测值`

若某一项不能补齐 `当前值`，则该项当前不允许进入正式方向比较。


## 6. 非法表达清单

以下内容都不能直接当 `预测结果`：

1. `方法名`
2. `动作名`
3. `结果主键`
4. `成功率高`
5. `看起来会更好`
6. `预计能解决问题`
7. `经验上通常有效`
8. 任何未落成带值特征的自然语言描述

这些内容最多只能作为：

- 召回辅助
- 说明文字
- 候选证据

不得直接进入：

- 根层重判
- 执行前门控
- 任务推进方向判定


## 7. 预测结果方向的生成规则

`预测结果方向` 必须按以下链条生成：

```text
预测结果项
-> 取得同特征类型的当前值
-> 比较 (预测值 - 当前值)
-> 生成 大于 / 小于 / 等于
```

必须固定：

1. `预测结果方向` 只能落在：
   - `大于`
   - `小于`
   - `等于`
2. `预测结果方向` 不得直接手写成抽象词，例如：
   - `变好`
   - `变坏`
   - `可能有利`
3. 若无法取得 `当前值`，则当前不允许正式生成该项方向。


## 8. 与需求方向的关系

`预测结果` 进入正式判定链时，必须和 `需求方向` 对照。

最小链条应固定为：

```text
需求值 - 当前值 -> 需求方向
预测值 - 当前值 -> 预测结果方向
比较两者方向是否一致
```

必须固定：

1. 若 `需求方向 = 等于`，则 `预测结果方向` 只能是 `等于`。
2. 若 `需求方向 ≠ 等于`，则 `预测结果方向` 必须等于 `需求方向`，才允许继续作为正向候选。
3. `预测结果方向` 与 `需求方向` 不一致时，当前预测结果不得直接作为放行依据。

更细的逐项判定表，按 [0320_预测结果方向与需求方向对齐判定表规范.md](D:\鱼巢\规范\0320_预测结果方向与需求方向对齐判定表规范.md) 执行。


## 9. 与根层重判的关系

`预测结果` 进入根层重判时，必须先满足：

1. 已被表达为 `一组带值特征`
2. 已能与当前值比较
3. 已生成 `预测结果方向`

然后才允许作为以下输入之一进入根层重判：

- `预测结果方向`
- `预测结果特征摘要`

必须固定：

1. 进入根层重判的，不是原始自然语言预测，而是比较后的结果。
2. 根层重判消费 `预测结果方向`，不是直接消费“方法看起来会成功”。

- 预测结果必须先以带值特征表达
- 再通过比较生成方向
- 再由方向进入候选筛选或执行前判断


## 11. 严格禁止的混层

以下做法必须禁止：

1. 用 `方法名` 直接代表预测结果。
2. 用 `结果主键` 直接代表预测结果。
3. 用“预计有效”直接代表预测方向。
4. 未取得当前值就直接生成预测方向。
5. 把预测结果描述直接当成根层重判输入。
6. 把预测结果本体直接当成最终裁决依据。
