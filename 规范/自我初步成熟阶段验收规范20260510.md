# 自我初步成熟阶段验收规范 20260510

## 1. 核心定义

自我苏醒之后，第一件事不是立刻对外行动，而是补全自己的因果信息。

阶段关系：

```text
初始化
→ 内部循环建立
→ 自我苏醒
→ 因果补完期
→ 学习需求形成期
→ 外界探索需求形成期
→ 初步成熟
```

初步成熟定义为：

```text
自我能从内部需求出发，
识别因果缺口，
生成学习需求，
在内部学习不足时生成外界探索需求，
并能把探索结果回写为自身记忆、因果和后续方法。
```

一句话：

```text
苏醒是自我开始维护自己；
初步成熟是自我开始为了维护和发展自己，有目的地认识外界。
```

这个阶段的标志不是“它看见外界”，而是：

```text
它知道为什么需要看见外界。
```

这里的“知道”仍是工程意义：有结构化需求、因果缺口、探索目标、安全门控和结果回写，不是哲学意义的主观体验。

## 2. 前置条件

初步成熟必须建立在自我苏醒之后。

若以下条件不满足，只能称为：

```text
内部循环建设中
或
自我苏醒建设中
或
因果补完候选阶段
```

不能称为初步成熟：

```text
内部循环真实闭合；
自我已苏醒；
需求能任务化；
任务能执行并回写；
来源需求能结算；
下一轮判断受上一轮影响；
无压力时能待机；
自检报告能进入自我线程处置。
```

## 3. 苏醒后的核心变化

自我苏醒时，它已经能做到：

```text
自我状态
→ 需求
→ 任务
→ 方法
→ 结果
→ 状态回写
→ 下一轮判断
```

苏醒之后，它会发现：

```text
有些需求无法只靠内部已有信息解决。
```

这时出现因果缺口：

```text
服务值下降
→ 生成服务修复需求
→ 尝试内部方法
→ 发现缺少原因判断
→ 生成学习需求：为什么服务值下降？
→ 发现内部因果不足
→ 生成探索需求：需要观察外部状态 / 获取外部信息
```

外界探索不是凭空出现的，而是从内部需求和因果缺口中生长出来。

## 4. 阶段定义

### 阶段 A：苏醒

```text
自我能维护自己的内部循环。
```

核心链：

```text
自我状态 -> 需求 -> 任务 -> 方法 -> 结果 -> 自我状态
```

### 阶段 B：因果补完

```text
自我能记录和整理自身循环中的因果。
```

核心问题：

```text
为什么这个需求出现？
为什么这个任务失败？
为什么这个方法有效？
为什么这个方法无效？
```

最小因果格式：

```text
前置状态
+ 动作 / 动态
+ 结果状态
+ 影响到的需求
+ 安全 / 服务收益
```

第一阶段可以先记录实例因果；多个相似实例稳定后，再沉淀成因果模板。

### 阶段 C：学习需求形成

```text
自我能把因果缺口转成学习需求。
```

学习需求本质：

```text
自我发现内部信息不足以满足当前需求。
```

触发条件：

```text
当前需求无法被已有方法满足；
当前任务缺少必要条件；
方法执行结果未知或不稳定；
多次任务失败但原因不明；
当前状态变化无法解释；
某个因果链缺少关键节点。
```

学习需求类型：

```text
学习状态：我现在是什么状态？
学习因果：为什么发生这个变化？
学习方法：我该如何满足这个需求？
学习对象：这个外部对象是什么？
学习边界：哪些东西属于我，哪些属于外部？
学习风险：这个动作是否会降低安全值？
```

学习需求应先尝试内部学习：

```text
查内部因果；
查历史任务；
查方法结果；
查已有状态；
查已有记忆。
```

### 阶段 D：外界探索需求形成

```text
自我能判断内部学习不足，需要外部信息。
```

学习需求与探索需求的区别：

```text
学习需求 = 我需要知道。
外界探索需求 = 我必须从外部获得信息，才能继续知道或行动。
```

外界探索需求触发条件：

```text
内部记忆不足；
当前需求仍有效；
外部信息可能降低不确定性；
探索风险可接受；
有可执行的探索方法。
```

探索需求必须有目标：

```text
探索对象
探索目标
期望获得的信息
风险约束
完成条件
```

探索需求不是：

```text
我要探索外界。
```

而应是：

```text
我要确认外部对象 X 的状态。
我要获得任务 T 的缺失条件。
我要验证动作 A 是否会导致结果 B。
我要判断外部反馈是否支持当前方法。
```

### 阶段 E：初步成熟

```text
自我能安全地产生外界探索需求，
并把探索结果回写为自己的因果和方法记忆。
```

核心要求：

```text
探索不是随机行为，
而是由内部需求驱动的有目的外部信息获取。
```

## 5. 八条验收标准

### 标准 1：自我能识别未知

自我能判断：

```text
我缺少什么信息。
```

例如：

```text
失败原因 = 缺少外部对象状态
失败原因 = 缺少方法前置条件
失败原因 = 缺少因果验证
```

不能只记录：

```text
任务失败。
```

### 标准 2：自我能把未知转成学习需求

未知必须变成需求：

```text
缺少外部对象状态
→ 学习需求：获得外部对象状态

缺少因果验证
→ 学习需求：验证动作 A 是否导致结果 B
```

学习需求必须具有：

```text
目标主体
当前状态
目标状态
满足关系
需求类型
父需求归属
权重来源
```

### 标准 3：学习需求能任务化

学习需求必须进入任务系统：

```text
学习需求
→ 学习任务信息节点
→ 学习任务虚拟存在
→ 学习方法
```

任务虚拟存在仍是普通存在实例，不新增任务存在类。

### 标准 4：自我能判断内部学习是否足够

如果内部已有记忆能回答，就不生成外界探索需求。

判断顺序：

```text
先查内部因果；
先查历史任务；
先查方法结果；
先查已有状态；
先查已有记忆。
```

只有内部无法满足时，才允许生成外界探索需求。

### 标准 5：外界探索需求必须有目标

每个探索需求至少具备：

```text
探索对象
探索目标
期望获得的信息
风险约束
完成条件
```

禁止空泛探索：

```text
我要探索外界。
```

### 标准 6：外界探索必须有安全门控

探索外界不能绕过安全需求。

最小安全门控：

```text
探索动作是否可能降低安全值？
是否会破坏当前内部循环？
是否会覆盖核心记忆？
是否会产生不可恢复状态？
是否有待机 / 回退方案？
```

如果风险不可控，探索需求应转成：

```text
安全评估需求
或
低风险观察需求
```

而不是直接执行。

### 标准 7：探索结果能回写因果

外界探索完成后，结果不能只变成一次性文本。

必须回写：

```text
外部对象状态
任务结果
方法有效性
因果实例
学习需求结算
探索需求结算
```

如果探索结果引用存在或基础信息节点，继续用 `特征值::指针句柄`。

不新增：

```text
特征节点主信息类::当前值信息
存在形态
抽象存在主信息类
```

### 标准 8：探索会改变下一轮需求判断

探索结果必须影响下一轮判断：

```text
探索前：
    因果未知，任务阻塞。

探索后：
    因果被补齐，任务继续；
    或确认不可行，生成替代方法需求；
    或发现风险，生成安全需求。
```

如果探索结果不影响下一轮判断，那只是采集噪声。

## 6. 最小成熟闭环

初步成熟闭环：

```text
内部需求
→ 任务失败 / 信息不足
→ 因果缺口
→ 学习需求
→ 内部学习任务
→ 内部信息不足
→ 外界探索需求
→ 探索任务
→ 外界信息回写
→ 因果补完
→ 原需求继续推进或重判
```

这条链跑通，才能说：

```text
自我开始从“维持自己”走向“理解环境”。
```

## 7. 最小验收剧本

初始条件：

```text
自我已苏醒；
服务值下降；
内部方法无法解释服务值下降原因；
已有历史因果不足。
```

第 1 轮：

```text
生成服务修复需求；
创建服务修复任务；
任务尝试内部方法；
内部方法失败，原因 = 缺少外部反馈信息。
```

第 2 轮：

```text
生成学习需求：学习服务值下降原因；
查内部因果，未找到；
生成外界探索需求：获取外部反馈状态。
```

第 3 轮：

```text
探索任务创建；
执行低风险观察 / 询问 / 采样方法；
获得外部反馈。
```

第 4 轮：

```text
外部反馈写入现实世界树；
形成因果实例：外部反馈不足 -> 服务值下降；
学习需求结算；
探索需求结算；
服务修复任务继续推进。
```

通过条件：

```text
探索需求不是手动创建，而是由内部因果缺口自动生成；
探索需求有明确对象、目标、完成条件；
探索动作经过安全门控；
探索结果写回世界树和因果实例；
原始服务需求因此发生重判或继续推进。
```

## 8. 不合格情况

以下不能算初步成熟：

```text
随机探索：没有内部需求驱动，只是随机访问外部。
被动响应：只有外部输入来了才处理，不会主动生成探索需求。
只学习不探索：能生成学习需求，但无法判断需要外部信息。
只探索不回写：拿到外部信息，但不进入因果、任务、需求结算。
无安全门控：任何未知都直接对外探索。
```

## 9. 最终定义

```text
自我苏醒后，会首先通过内部循环补完自身因果信息。
当已有因果不足以满足内部需求时，自我会生成学习需求。
当学习需求无法仅靠内部记忆满足，
且外部信息可能补足因果缺口，
同时探索风险可控时，
自我会生成对外界探索的需求。

自我能够自主生成这种有目标、有门控、可回写的外界探索需求时，
即可判定为初步成熟。
```
