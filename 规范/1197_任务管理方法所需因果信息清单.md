# 1197 任务管理方法所需因果信息清单

## 1. 目标

本文用于把 `任务管理任务` 后续真正要生成的 `因果信息`，单独整理成一张清单。

本文只回答一件事：

- 如果暂时不补代码，先从规范上看，`任务管理方法` 到底需要哪些因果信息

本文不再重复展开“方法需求位是什么”或“状态动态总表是什么”，而是只保留与因果生成直接相关的信息项。

### 2.1 在统一主链中的位置

本文默认处于以下主链中段：

`根层方向裁决`
-> `形成任务管理上下文`
-> `按本文准备方法因果所需信息`
-> `生成候选实例因果 / 轻量因果链`
-> `返回外层重判与成长确认`

必须固定：

1. 本文只统计 `根层已允许继续推进` 之后的任务管理方法因果信息。

3. 本文列出的因果信息，是任务管理治理证据面，不是新的根语义裁决层。

## 4. 共用必备因果信息

不管属于哪一个方法需求位，以下因果信息都应视为 `共用必备`：

### 4.1 共用必备条件信息

- `最近根层裁决结果`
- `当前功能域`
- `当前缺口类型`
- `当前下一步去向`
- `输入条件主键`
- `治理态型`
- `结构完备度`
- `方法可用度`
- `风险门控`

### 4.2 共用必备动作信息

- 当前管理方法 `开始运行`
- 当前管理方法 `完成 / 失败`
- `来源方法首节点`
- `动作相位`

### 4.3 共用必备结果信息

- `条件缺口分类`
- `可执行性判断`
- `推荐分解方向`
- `是否需要补条件任务`
- `是否建议等待学习`
- `任务_状态`
- `任务_当前方法`
- `任务_当前步骤`
- `任务_当前子任务`
- 若涉及外部动作，还必须有 `外部结果确认状态`

### 4.4 共用必备桥接信息

- `宿主任务`
- `输入条件主键`
- `当前管理方法首节点`
- `结果主键`
- `主动作类别`
- 若涉及外部动作，还必须有 `外部结果确认来源`
- `链最弱跳可信度`

### 4.5 外部动作与链可信度规则

若某条实例因果或因果链包含 `外部动作动态`，则必须额外补齐：

- `外部结果确认状态`
- `外部结果确认来源`

其中确认来源至少应来自以下之一：

- `观测确认`
- `回执确认`
- `反馈确认`
- `结果状态确认`

必须固定：

1. 没有外部结果确认时，`外部动作已发出` 只能视为弱果侧证据。
2. 没有外部结果确认时，不得把该跳记为强确认实例因果。
3. 因果链整体可信度，不得高于 `链最弱跳可信度`。
4. 若最弱跳是“未确认外部结果跳”，则整条链不得被记为高可信稳定链。

### 4.6 与外层重判的边界

必须继续固定：

1. 本文列出的因果信息，只用于支撑 `任务管理上下文`、`方法需求位`、`候选方法成长` 和 `外层重判所需证据`。
2. 本文列出的因果信息，不得单独替代 `根层方向裁决`。
3. 即使某条方法因果信息已经稳定，也只能作为外层重判的输入证据之一，不能越权给出最终继续/停止结论。

### 4.7 果的值方向记法

为了避免只写“有这个状态”却不知道“值该往哪边走”，本文统一使用以下方向记法：

- `0 -> 1`
表示布尔或开关态从“不成立”转成“成立”

- `0 -> 非0`
表示从“未绑定/未定义/空指针/空主键”转成“已绑定/已定义/非空”

- `低 -> 中/高`
表示分档值向更完整、更可用、更稳定方向收敛

- `未定义 -> 已定义`
表示枚举态、结果码或结果主键从“未判定”转成“已判定”

- `旧值 -> 新值`
表示发生了明确切换，且 `新值` 必须与 `旧值` 不同

必须固定：

- 对某个方法需求位来说，`果` 不要求下表中的所有状态同时变化
- 但至少应命中其中一组主结果状态，并且方向必须与该方法位语义一致

## 6. 第一批最该补的因果信息

如果只先做第一批，建议优先盯下面两组：

### 6.1 补结构方法位

最该先明确列出来的因果信息是：

- 条件：
`依赖缺失`
`任务链未成形`
`无可试方法`
`结构完备度=低`

- 动作：
`管理_筹办器完成动态`

- 结果：
`管理任务_建议方法首节点指针: 0->非0`
`任务_当前步骤ID: 0->非0`
`任务_当前子任务头指针: 0->非0`
`任务_当前方法首节点指针: 0->非0`
`管理结构推进成立: 0->1`
`管理结构完备度: 低->中/高`

- 链桥接：
`宿主任务`
`输入条件主键`
`管理_筹办器`
`结果主键`
`主动作类别=内部动作`
`链最弱跳可信度`

这里的 `结果主键` 仍然只承担 `链桥接辅助字段` 职责，不替代上面已经列出的具体结果状态。

### 6.2 失败策略方法位

最该先明确列出来的因果信息是：

- 条件：
`最近执行失败`
`连续失败次数`
`关键方法不再可用`
`失败原因码`

- 动作：
`管理_失败策略器完成动态`

- 结果：
`管理任务_失败原因码: 0->非0`
`管理任务_是否建议等待学习: 0->1`
`管理任务_是否建议回到筹办: 0->1`
`管理任务_建议方法首节点指针: 0/旧值->非0/新值`
`任务_转入学习次数: n->n+1`
`任务_当前方法首节点指针: 旧失败方法->替代方法/0`

- 链桥接：
`宿主任务`
`输入条件主键`
`失败原因码`
`结果主键`
`主动作类别=内部动作`
`链最弱跳可信度`

这里的 `结果主键` 仍然只承担 `链桥接辅助字段` 职责，不得因为结果主键相近就直接把失败策略判成已成立。

### 6.3 若后续链延长到外部动作

当前 `任务管理方法` 本身主要还是内部治理动作，但若后续因果链继续延长到：

- 叶子执行动作
- 外设控制动作
- 对外输出动作

则还必须补齐以下因果信息：

- `外部结果确认状态`
- `外部结果确认来源`
- 该外部跳的 `单跳可信度`

必须固定：

1. 没有 `外部结果确认状态` 时，只能说明该外部动作已发出。
2. 没有 `外部结果确认来源` 时，不得把该跳升格为强确认果跳。
3. 一旦因果链包含外部跳，整条链都必须受 `链最弱跳可信度` 上限约束。
