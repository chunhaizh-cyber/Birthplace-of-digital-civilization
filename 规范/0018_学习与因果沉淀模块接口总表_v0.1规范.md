## 3. 定位

必须固定：

`学习模块不是第二个主循环，也不是隐藏国王。`

它只负责把 `已经结算过` 的运行结果，转成可复用的路径经验，并把这些经验回写到：

- `方法评价`
- `因果实例`
- `因果模板`
- `黑名单样本`
- `待验证样本`

它只能：

- 调路径
- 补证据
- 调评价
- 归档不稳定样本

它不能：

- 改方向
- 改根目标
- 改退出边界
- 直接决定下一轮主导需求
- 绕过主循环直接续跑任务

一句话收口：

`学习负责把已结算结果沉成可复用证据，但没有权力改写宪法。`


## 4. 核心结论

必须固定以下结论：

1. 学习模块只消费 `已经完成结算` 的结果，不消费未结算的原始过程噪声。
2. 学习模块的主结果应优先落在 `方法资产增长` 与 `因果证据沉淀` 上，而不是落在根目标改写上。
3. 学习模块在语义上属于 `内部治理路径优化模块`，不是新的根任务。
4. 学习模块只能调路径，不能改方向。
5. 单次样本最多只能进入 `待验证 / 候选 / 黑名单` 这类中间层，不得直接跃迁成稳定规则。
6. 失败样本、超时样本、回滚样本、误判样本与止损样本都属于正式学习原料，不只是“垃圾日志”。
7. 学习模块必须对 `证据不足 / 证据冲突 / 风险过高 / 结果未验证` 这四类情况保守处理。
8. 安全接管链中的学习默认只记录 `止损证据` 与 `风险样本`，不得借机反向恢复服务推进。
9. 学习更新完成后，控制权必须回到主循环下一轮重判，而不是由学习模块直接宣布“以后这样做”。

同时必须立四条硬规则：

1. `先结算，再学习`
2. `先验证，再固化`
3. `先调路径，不改方向`
4. `单轮可修正，不能单轮定终身`

## 6. 学习模块内部最小链条

学习模块内部主链固定为：

`样本入池 -> 证据对齐 -> 有效性筛查 -> 因果抽取 -> 方法评价更新 -> 稳定/黑名单归档`

各段回答的问题固定为：

1. `样本入池`
   - 这次有哪些已结算结果值得进入学习候选面

2. `证据对齐`
   - 动态、场景差异、因果候选、人工纠偏、止损记录之间能否对齐到同一事件链

3. `有效性筛查`
   - 这批样本是否足够可信、是否已验证、是否存在冲突、是否属于高风险误学来源

4. `因果抽取`
   - 这次到底能形成实例因果、模板候选，还是只能形成弱样本

5. `方法评价更新`
   - 这次样本对当前方法的可靠度、适用条件、失败边界意味着什么

6. `稳定/黑名单归档`
   - 这次样本最终进入稳定候选、待验证池，还是进入黑名单/风险标签

其中：

必须固定：

`这些字段当前先服务于样本筛查、因果抽取和方法评价更新，而不是直接跳成根层依据。`


## 8. 与主循环和结算模块的顺序关系

学习模块与 `结果回流、结算与主循环重判` 的关系必须固定为：

```text
执行结果进入回流
-> 先完成本轮结算
-> 形成已结算证据包
-> 学习模块消费该证据包
-> 学习结果写回方法/因果/样本池
-> 控制权返回主循环下一轮正式重判
```

因此必须固定：

1. 学习不得在 `本轮结算完成前` 先行落库。
2. 学习不得把未结算原始过程直接上升为稳定经验。
3. 学习写回之后，仍要回到主循环下一轮正式重判，而不是直接宣布“以后默认这样做”。

这也意味着：

`0015 / 0016` 中的“结果回流、结算与主循环重判”在运行顺序上至少要理解成：先回流结算，再允许学习消费，再进入下一轮正式重判。


## 9. 两条完整金路径

### 9.1 金路径 A：服务主导闭环

```text
边界检查
-> 状态刷新
-> 服务/安全差额评估
-> 生成服务需求
-> 仲裁确定服务主导
-> 任务管理一步治理
-> 授权/配额/限幅
-> 方法匹配与选择
-> 执行调度与监督
-> 结果回流与结算
-> 学习与因果沉淀
-> 主循环重判
```

这条链证明的是：

- 系统不只是会看见世界
- 还能把 `解释 / 建议 / 执行 / 学习` 串成一条服务推进链
- 本轮结果会沉成下一轮可复用证据，而不是只变成一次性痕迹

这条链的关键闸门固定为：

- `安全差额未越接管阈值`
- `授权未放开，执行不得启动`
- `未结算，学习不得落库`
- `学习更新后，仍要回到主循环重判`

### 9.2 金路径 B：安全接管闭环

```text
边界检查
-> 状态刷新
-> 物理安全 / 风险安全差额突增
-> 仲裁触发安全接管
-> 任务管理生成止损一步治理
-> 最小必要授权
-> 安全守卫打断 / 降级 / 回滚 / 收束
-> 结果回流与结算
-> 生命周期切到降级 / 收束 / 待机
-> 学习仅记录止损证据
-> 主循环重判是否恢复服务推进
```

这条链证明的是：

- 安全不是墙上的告示，而是在边界上真的能夺回控制权
- 止损链完成后，系统不会偷偷直接跳回服务链
- 学习在安全链中只负责记录 `止损证据 / 风险样本`，不负责替系统恢复服务推进

这条链的关键闸门固定为：

- `安全守卫可以中断当前执行`
- `止损动作只拿最小授权`
- `安全链完成后不能直接跳回服务链`
- `必须经主循环根层重判后才允许恢复推进`


## 10. 到这里，12 模块如何收口

当前 `12` 模块可统一压成一句话：

`根层判方向，治理层发一步，承接层安全落地，回流层做结算，学习层沉淀证据，但任何局部模块都不得改写根目标与边界。`

再压到底层能力链，就是：

`看见世界 -> 理解语义 -> 形成需求 -> 拆解任务 -> 调用方法 -> 记录结果 -> 沉淀因果`

其中：

- `看见世界` 主要由 `世界与状态刷新` 承接
- `形成需求` 主要由 `差额评估 + 根需求生成 + 仲裁` 承接
- `拆解任务` 主要由 `任务管理一步治理` 承接
- `调用方法` 主要由 `授权 + 选法 + 执行监督` 承接
- `记录结果` 主要由 `回流结算` 承接
- `沉淀因果` 主要由 `学习与因果沉淀` 承接


## 11. 当前阶段最小验收口径

当前若要承认 `学习与因果沉淀模块` 已最小成立，至少必须满足：

1. 学习模块只消费 `已结算结果`，而不提前消费未结算过程。
2. 样本至少能分流到：
   - `待验证样本池`
   - `候选因果实例/模板`
   - `黑名单 / 风险样本`
3. 方法评价调整与因果沉淀能够回写，但不会直接改写根层方向。
4. 单次样本不会直接上升为稳定规则。
5. 服务主导闭环和安全接管闭环都已经把学习模块正确接入末端，而不是把学习放到裁决前面。

最终收口：

`到 0018 为止，12 模块的最小强闭环已经完整：先看见、再治理、再承接、再结算、再学习，但根裁决始终只属于主循环。`
