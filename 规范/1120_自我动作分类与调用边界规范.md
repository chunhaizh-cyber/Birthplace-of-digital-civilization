# 1120 自我动作分类与调用边界规范

## 1. 目标

本文用于固定“自我动作”在当前工程中的三类划分、调用边界、归类判据与当前动作对齐口径，避免以下混乱：

- 先天维持动作被误当成普通任务可主动调用动作
- 自我内部动作与外部动作混用
- 对虚拟存在特征的操作动作没有统一归类口径
- 同一个通用动作因目标不同而跨域混淆


## 2. 分类判据

自我动作的分类，必须同时看以下两个维度：

1. `谁可以触发`
- 只能由自我运行时被动触发
- 还是允许被任务/方法主动选中与调用

2. `主要作用对象是谁`
- 自我内部状态
- 自我内部世界中的虚拟存在及其特征
- 现实世界对象、外设对象、对外输出通道

若两个维度冲突，优先以“主要作用对象”和“最终结果落点”为准。


## 3. 三类动作

### 3.1 内部先天动作

定义：

- 这类动作属于自我的先天维持、被动回归、运行时保底动作。
- 它们只能由 `自我线程`、周期维护、被动事件链或等价运行时入口触发。
- 它们不得作为普通任务候选动作、普通方法候选动作或对外服务动作被主动点名调用。
- 纳入固定先天机制清单的内部先天动作，必须由 `自我线程` 每个周期自动调用一次。
- 每次调用都视为一次独立的内部先天动作事实，并应生成一条对应的 `本能动作动态`。

允许：

- 维护自我的基础时序平衡
- 回归安全/服务相关内部值
- 维护自我所在场景的基础安全状态
- 在内部账本、因果、状态链中登记其事件边界与结果

禁止：

- 作为普通任务叶子动作直接筹办
- 作为外部服务动作暴露给对话/设备控制路径
- 与外设控制动作合并成一个不可拆分的原子动作

说明：

- 测试、自检、回归用例中允许显式调用这类动作，但这不改变其生产语义分类。
- `自我` 不能把这类动作当作普通方法主动点名调用；它只能在运行时读取这些动作的结果状态和动作动态，并据此组织状态、动态和因果。
- 对内部先天动作，当前正式最小记录单位是“每周期一次调用 = 一条本能动作动态”，而不是默认拆成多条普通动作边界。

典型动作：

- `定时衰减服务值`
- `按双值重算服务衰减步长`
- `按服务值重算安全增加步长`
- `按服务值重算安全减少步长`
- `安全低位回升`
- `安全高位回落`
- `风险安全回归`
- `场景安全度回归`
- `服务归零待机保护`


### 3.2 自我内部动作

定义：

- 这类动作允许由自我内部任务或方法主动调用。
- 它们主要用于读取、更新、组织或修正自我内部状态、自我内部世界、虚拟存在及其特征。
- 这类动作的主结果必须主要落在内部域，而不是现实外设域。

自我内部动作至少分为两组：

1. `内部状态/认知动作`
- 读取核心状态
- 观察场景
- 识别、组织、修正内部认知结构
- 根据内部风险或预算切换保守态、试探态等内部运行姿态

2. `虚拟存在操作动作`
- 对 `自我虚拟存在`
- 对 `任务虚拟存在`
- 对 `方法虚拟存在`
- 对 `自我内部世界` 中的内部对象
进行存在创建、特征确保、特征读写、特征删除、状态回填等操作

允许：

- 被内部任务、内部方法、学习兜底路径主动调用
- 写入自我内部状态
- 写入自我内部世界
- 写入任务虚拟存在与方法虚拟存在

禁止：

- 直接承担外设控制副作用
- 直接替代对外表达动作
- 在一次原子动作中同时跨内部域和外部域完成主结果写入


### 3.3 外部动作

定义：

- 这类动作允许由任务主动调用。
- 它们用于执行自我对外动作，或调用外部设备、改变现实世界对象状态、向外输出可观察结果。
- 其主结果主要落在外部域。

允许：

- 对外确认、请求、通知
- 外设启动、关闭、取数、调参
- 现实世界对象或设备事实更新

禁止：

- 把自我内部治理动作伪装成外部服务动作
- 把内部虚拟存在操作直接当成外设动作对待

典型动作：

- `确认收到`
- `请求补充信息`
- `相机启动`
- `相机关闭`
- `获取帧信息`
- `设置相机参数`


## 4. 内部域与外部域

### 4.1 内部域

以下对象默认视为内部域：

- `自我` 及其内部状态
- `自我内部世界`
- `自我虚拟存在`
- `任务虚拟存在`
- `方法虚拟存在`
- 学习沙盒、内部推演场景、内部输出场景


### 4.2 外部域

以下对象默认视为外部域：

- 现实世界场景中的对象
- 外设对象
- 相机等设备实例
- 对人类显式可见的外部输出通道


## 5. 通用动作的归类规则

并不是所有动作都能只看名字归类。

以下“通用操作动作”必须按目标对象所属域归类，而不是按函数名固定归类：

- `世界_确保虚拟存在`
- `世界_确保特征节点`
- `世界_读特征值`
- `世界_写特征值`
- `世界_删除特征节点`
- `世界_创建场景`
- `世界_删除存在`
- `世界_迁移存在`

归类规则固定如下：

1. 当目标是 `自我/任务/方法虚拟存在` 或 `自我内部世界` 对象时，它们属于 `自我内部动作`。
2. 当目标是 `现实世界对象`、`外设对象`、`对外输出对象` 时，它们属于 `外部动作`。
3. 同一个原子动作不得同时承担“内部主结果写入”和“外部主结果写入”两种职责。
4. 若一个高层流程同时需要内部写回和外部执行，必须拆成两个可追踪动作，由方法链或任务链组合，不得混成一个黑盒原语。


## 6. 调用边界表

| 动作类别 | 允许触发者 | 是否可被普通任务主动选中 | 主结果落点 | 典型目标 |
| --- | --- | --- | --- | --- |
| 内部先天动作 | 自我线程、周期维护、被动运行时入口 | 否 | 自我基础状态、内部账本、内部场景 | 服务值、安全值、场景安全度 |
| 自我内部动作 | 自我内部任务、内部方法、学习兜底 | 是 | 自我内部状态、内部世界、虚拟存在 | 自我、任务虚拟存在、方法虚拟存在 |
| 外部动作 | 任务、方法、外设调用链、对外交互链 | 是 | 外部设备、现实对象、对外输出 | 相机、对话输出、现实世界对象 |

## 7. 动作动态的因果可信度分层

以后凡是由 `自我的动作动态` 参与生成实例因果或因果链，统一按以下可信度分层处理：

### 7.1 内部先天动作动态

- `动作事实` 绝对可信
- `动作边界` 绝对可信
- 若结果本身就是该先天动作直接写回的内部结果，则该结果可视为 `直接可信`
- 生成因果时，不再重复做“动作是否真的发生过”的真实性校验

但必须继续服从：

- `同锚点`
- `同层级`
- `时间有序`

### 7.2 自我内部动作动态

自我内部动作动态的可信度固定分三层：

1. `直接可信`
- 动作确实发生
- 动作名称、开始边界、结束边界可信

2. `高可信但仍需校验`
- 动作对应的结果变化是否属于同批次、同锚点、同层级

3. `不能直接跳过验证`
- 整条因果已经稳定
- 这条因果已经等同于方法

### 7.3 外部动作动态

- 直接可信的只是：`自我确实发出了该外部动作`
- 不直接等于：`外部世界已经按预期改变`

因此必须固定：

- 外部动作的果侧必须优先依赖 `观测 / 回执 / 反馈 / 结果状态确认`
- 在没有外部结果确认时，不得把“动作已发出”直接写成“外部结果已成立”
- 外部动作参与因果生成时，其动作证据默认低于 `内部先天动作` 与 `自我内部动作`

### 7.4 对因果链的约束

这套可信度分层不只作用于单跳实例因果，也作用于所有因果链。

必须固定：

1. 因果链中的每一跳，都必须按其动作类别分别应用上述可信度规则。
2. 因果链整体可信度，不得高于其中最弱的一跳。
3. 若某一跳以外部动作作为主动作，但没有外部结果确认，则该跳不得冒充强确认因果。
4. 若某一跳的动作事实本身不成立，则整条链不得继续向后组织。

## 9. 对当前实现的收口要求

### 9.1 对内部先天动作

- 不应进入普通任务的常规候选动作集
- 不应作为学习兜底的普通单步试探候选
- 若因测试原因暂时保留显式调用入口，调度器仍应把它们视为“不可常规主动筹办”
- 自我只能读取其运行结果与动作动态，不得把它们误当成普通可主动调用方法


### 9.2 对自我内部动作

- 允许进入内部任务、内部方法、学习兜底路径
- 若动作只操作虚拟存在特征，不得误记为外设控制动作


### 9.3 对外部动作

- 必须明确标注其外部目标
- 必须与内部状态修正动作分开记账、分开记因果、分开记事件边界
