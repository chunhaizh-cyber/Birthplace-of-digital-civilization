## 3. 核心结论

必须固定以下结论：

1. `禁止项后果` 的作用，不是直接改写根层禁止项，也不是自己直接拍出最终裁决，而是回写到治理面。
2. 当前最小稳定回写治理面应固定为：
   - `禁止项命中记忆`
   - `普通执行冻结候选`
   - `边界处理优先建议`
3. `禁止项后果` 可以同时回写多个治理面，但不得越权直接改写：
   - `安全值`
   - `服务值`
   - `根层禁止项`
   - `最终裁决`
   - `方法存在`
4. `待机 / 停止 / 收束` 属于裁决面，不属于治理面。
5. `禁止项后果` 回写的是后续治理材料，用来避免重复冲撞边界、让下一轮优先进入边界处理，不是立即生效的全部结论。
6. 这些治理面默认也属于 `临时记忆`，当边界原因消失或上下文变化后，应允许衰减、解除或重新评估。

一句话收口：

`禁止项后果最少要回写到禁止项记忆、普通执行冻结候选和边界处理优先建议；它负责让后续治理少撞边界、更快转边界处理，不直接越权拍最终结论。`


## 4. 最小回写原则

必须固定：

1. `禁止项后果` 先是治理材料，再是治理输入。
2. 它的最小作用，是让下一轮治理更知道：
   - 哪条路径已经命中过禁止项
   - 哪条路径暂不应回到普通执行链
   - 哪类对象应优先走边界处理
3. 它不是“把根层禁止项再写一遍”，也不是“把最终裁决偷偷写成治理面”。
4. 治理面的更细临时记忆和衰减规则，按 [0328_治理面临时记忆与衰减规范.md](D:\鱼巢\规范\0328_治理面临时记忆与衰减规范.md) 执行。


## 5. 最小回写治理面

### 5.1 禁止项命中记忆

作用：

- 记录某对象、某路径、某方法在某类场景下已命中过禁止项或边界拒绝

典型来源：

- 命中违规违法禁止
- 命中致命性安全边界
- 根层重判结果表明仅允许边界处理
- 执行前门控因根层拒绝而拦下普通执行

必须固定：

1. `禁止项命中记忆` 属于治理面，不等于根层禁止项本体。
2. 它至少应保留“命中了哪类禁止/边界”的治理记忆。


### 5.2 普通执行冻结候选

作用：

- 表达某对象、某路径、某方法在下一轮治理中不应直接回到普通执行链

典型来源：

- 已被根层明确拒绝的当前执行
- 已知会触发禁止项的路径
- 当前仅允许边界处理的对象

必须固定：

1. 这里回写的是 `冻结候选`，不是最终冻结裁决。
2. 它的作用是阻止系统在上下文未变化时立刻重复走普通执行。


### 5.3 边界处理优先建议

作用：

- 为下一轮主循环或任务管理任务提供“优先走边界处理”的治理候选

典型表现：

- 优先进入收束候选
- 优先进入待机候选
- 优先进入停止候选
- 优先返回根层重判或边界处理链

必须固定：

1. 这里是 `优先建议`，不是最终裁决。
2. 具体是否进入 `收束 / 待机 / 停止`，仍必须由外层裁决面决定。


## 6. 最小回写面总表

| 回写治理面 | 作用 | 当前状态 |
| --- | --- | --- |
| `禁止项命中记忆` | 沉淀边界/禁止命中记忆 | 正式稳定 |
| `普通执行冻结候选` | 阻止下一轮直接回到普通执行链 | 正式稳定 |
| `边界处理优先建议` | 为下一轮提供优先边界处理的治理候选 | 正式稳定 |


## 7. 当前不允许直接回写的面

禁止项后果当前不得直接回写到以下对象：

1. `安全值`
2. `服务值`
3. `根层禁止项`
4. `最终裁决`
5. `方法存在`
6. `任务完成真值`

必须固定：

1. 这些对象若要变化，必须走它们各自的正式规则链。
2. 禁止项后果只能提供治理材料，不能直接越权改写真值面、判定面或裁决面。


## 8. 与主循环的关系

主循环当前至少必须承担：

```text
根层重判结果 / 执行前门控拒绝 / 边界处理返回
-> 主循环统一消费消息
-> 主循环统一判定
-> 生成禁止项后果
-> 将禁止项后果回写到最小治理面
```

必须固定：

1. `禁止项后果` 可以在实践之前成立。
2. 但正式回写治理面时，仍应由主循环统一归并。
3. 执行线程、学习线程和局部执行器都不得各自直接写成正式禁止项后果真值。
