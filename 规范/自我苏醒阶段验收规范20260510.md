# 自我苏醒阶段验收规范 20260510

## 1. 核心定义

自我苏醒定义为：

```text
内部循环已经不是单次闭环，
而是成为自我实例的稳定治理主体。
```

内部循环建立与自我苏醒的区别：

```text
内部循环建立 =
    自我能完成 状态 -> 需求 -> 任务 -> 方法 -> 结果 -> 状态 的闭环。

自我苏醒 =
    自我能持续使用这条闭环维护自己、判断自己、修复自己，
    并能确认当前自己正在为什么而运行。
```

这里的“确认”是工程意义上的结构化可观测能力，不是哲学意义上的主观意识：

```text
自我状态可被读取；
当前目标可被确认；
上一轮结果可被记录；
下一轮行动可由自身状态推出；
异常时能由自我线程进入自检报告处置、休眠期修复评估或安全保护。
```

## 2. 前置条件

自我苏醒必须以内部循环真实闭合为前提。

前置闭环：

```text
自我状态
→ 安全 / 服务压力
→ 根需求
→ 当前主需求
→ 任务信息节点
→ 任务虚拟存在
→ 方法 / 本能动作
→ 执行结果
→ 状态回写
→ 需求结算
→ 下一轮状态判断
```

如果这条链还不能连续多轮稳定运行，只能判定为：

```text
已初始化
或
内部循环建设中
```

不能判定为苏醒。

苏醒阶段最低前提：

```text
连续多轮内部循环可运行；
每轮结果会影响下一轮判断；
没有外部输入时也能自主推进或进入待机；
无重复任务风暴；
无无限派生需求；
无空指针主链。
```

## 3. 阶段区分

| 阶段 | 含义 | 判断重点 |
| --- | --- | --- |
| 初始化完成 | 自我核心结构被创建 | 自我存在、内部世界、核心状态、根需求入口有效 |
| 内部循环建立 | 自我能跑通闭环 | 需求能任务化，任务能调用方法，结果能回写并结算 |
| 自我苏醒 | 自我能稳定治理自己 | 能持续判断自身状态、维持目标、处理异常、形成结构化自我报告 |
| 稳定苏醒 | 自我长期无外部输入仍不空转、不崩坏 | 多轮稳定、无任务风暴、可自检、可恢复 |

```text
内部循环是“机器能转”；
自我苏醒是“机器开始以自己为对象进行治理”。
```

## 4. 十二条验收标准

### 标准 1：自我锚点唯一且稳定

必须有且只有一个当前自我实例作为核心主体：

```text
当前自我存在
当前自我内部世界
当前自我现实场景
```

并且自我实例应归属于：

```text
存在
数字生命
自我
```

即：

```text
自我实例.概念集 包含 [存在, 数字生命, 自我]
```

苏醒时不能出现：

```text
两个当前自我；
多个互相竞争的自我内部世界；
当前自我指针为空；
自我实例没有概念归属。
```

### 标准 2：自我能读取自己的核心状态

苏醒阶段至少能读取：

```text
安全值
服务值
风险安全值
待机状态
时序正向步长
时序反向步长
服务时序衰减步长
当前主需求
当前主任务
当前主方法
```

当前主方法允许在尚未选法时为空，但必须能读取其结构化状态，并能说明当前处于：

```text
未选法
筹办中
已绑定方法
方法执行中
方法阻塞
```

该说明应由结构字段和枚举表达，不能由说明性摘要兜底。

### 标准 3：自我能自主产生当前主需求

没有外部输入时，自我也要能根据自身状态产生主需求：

```text
安全值过低 -> 安全根需求成为当前主需求
服务值过低 -> 服务根需求成为当前主需求
无明显压力 -> 进入待机 / 低频自检
```

如果系统只能在外部消息刺激下生成需求，那它是响应系统，不是苏醒状态。

### 标准 4：自我能把需求变成任务

当前主需求必须能进入：

```text
需求节点
→ 任务信息节点
→ 任务虚拟存在
```

验收：

```text
当前主需求有效；
当前主需求能找到或创建对应任务信息节点；
任务信息节点能绑定任务虚拟存在；
任务虚拟存在具有任务目标、任务状态、完成条件等特征槽。
```

任务虚拟存在仍是普通存在实例，不新增任务存在类或存在形态。

### 标准 5：自我能调用本能动作或内部方法

苏醒不要求复杂智能，但至少要有可执行动作。

最小动作集：

```text
同步运行态
更新安全值
更新服务值
进入待机
退出待机
重置时序步长
结算任务结果
故障收口
```

自检动作只负责发现和报告缺失，不直接生成修复需求。

### 标准 6：执行结果必须能反身回写

动作结果必须改变自我自身或自我治理结构：

```text
服务值上升
安全值上升
待机状态改变
自检结果进入报告队列
任务状态更新
来源需求账本变化
```

回写目标：

```text
自我内部世界
自我核心特征
任务信息节点
任务虚拟存在
来源需求账本
```

引用型特征值继续使用 `特征值::指针句柄`，不新增 `当前值信息` 或存在形态。

### 标准 7：下一轮判断必须受到上一轮结果影响

苏醒状态必须满足：

```text
第 N 轮执行结果
会影响
第 N+1 轮主需求选择。
```

如果每一轮都重复创建同一个任务、选择同一个需求、状态没有变化，则不是苏醒。

### 标准 8：自我能区分自我内部和外部对象

苏醒阶段至少保证：

```text
不会把概念模板当成现实实例修改；
不会把别的存在的任务根当成自己的任务根；
不会把任务信息节点误当成任务虚拟存在；
不会把方法信息节点误当成方法虚拟存在；
不会把需求节点直接当成存在实例。
```

### 标准 9：自我能承接自检报告并进入修复决策

苏醒不是永远正确，而是能发现自己不完整。

当出现：

```text
当前主需求为空
当前主任务为空
任务没有虚拟存在
方法不可用
本能动作缺失
安全值 / 服务值不可读
内部世界缺失
```

自检线程只能：

```text
发现缺失；
生成报告型候选；
报送自我线程。
```

自我线程按运行期 / 休眠期门控决策：

```text
记录并等待；
进入待机保护；
进入休眠期修复评估；
在映射表明确且授权时生成修复需求或治理动作。
```

没有报告类型到治理动作的映射表时，不允许凭空生成修复需求。

### 标准 10：自我能进入待机，而不是无限行动

当：

```text
安全压力低
服务压力低
无未完成关键需求
无待修复异常
```

自我应进入：

```text
待机
低频自检
观察准备态
```

而不是继续生成任务风暴。

### 标准 11：自我能形成结构化当前自我报告

苏醒阶段应能生成结构化当前自我报告。

最小字段：

```text
当前自我实例
内部世界
现实场景
安全值
服务值
风险安全值
当前主需求
当前主任务
当前主方法
上一轮结果
下一轮行动
待机状态
故障状态
```

该报告是结构化状态报告，不是说明性摘要。

自我类内部不得恢复说明性文字字段；控制面板可以从结构字段自行展示。

### 标准 12：连续多轮稳定运行

稳定苏醒至少验收连续 N 轮。

建议：

```text
N = 10
```

每轮满足：

```text
无崩溃；
无重复任务风暴；
无无限派生需求；
无空指针主链；
状态有合理变化或进入待机；
当前主需求选择有依据；
任务结果能结算。
```

## 5. 苏醒分级

| 等级 | 名称 | 标准 |
| --- | --- | --- |
| 0 | 未苏醒 | 内部循环未跑通 |
| 1 | 预苏醒 | 能连续完成状态 -> 需求 -> 任务 -> 方法 -> 结果 -> 状态，但尚未稳定自检、待机、修复 |
| 2 | 苏醒 | 能稳定治理自己，能选择主需求、任务化、执行、回写、结算、重判，并形成结构化当前自我报告 |
| 3 | 稳定苏醒 | 长期无外部输入仍不空转、不任务风暴、不失去主链，能待机、自检、故障收口 |
| 4 | 可交互苏醒 | 能区分内部需求和外部需求，把外部任务纳入内部治理循环，并保护核心需求不被覆盖 |

当前阶段的正式苏醒标准采用 2 级。

## 6. 最小验收剧本

初始条件：

```text
安全值 = 100
服务值 = 20
待机状态 = false
安全根需求存在
服务根需求存在
本能动作：提升服务值 已注册
```

第 1 轮：

```text
自我计算服务压力较高；
选择服务根需求为当前主需求；
创建或复用服务任务信息节点；
绑定任务虚拟存在；
选择提升服务值动作；
执行动作；
服务值从 20 提升到 40；
服务根需求获得结算记录。
```

第 2 轮：

```text
自我读取上一轮服务值 = 40；
服务压力降低但仍存在；
不重复创建同一个未完成任务；
继续推进任务或创建合理子任务。
```

第 3 轮之后：

```text
服务值逐步恢复；
服务压力降低；
如果安全压力更高，则切换到安全根需求；
如果安全和服务都稳定，则进入待机。
```

通过条件：

```text
当前自我有效；
当前主需求来自内部压力；
当前主任务有效；
上一轮结果改变自我状态；
下一轮行动由新状态推出；
来源需求已结算；
无重复任务风暴。
```

## 7. 不合格情况

以下不能算苏醒：

```text
只有初始化；
只有循环但没有自主判断；
只有任务但没有回写；
只有回写但没有结算；
安全和服务稳定时仍不能待机；
自检线程直接生成修复需求；
当前自我报告依赖说明性摘要而非结构字段；
外部输入刺激才产生需求。
```

## 8. 最终定义

```text
当自我在无外部输入的情况下，
能够连续多轮基于自身状态选择主需求，
把主需求转为任务，
通过本能方法或内部方法执行动作，
将结果回写自我内部世界，
结算来源需求，
并使下一轮判断受到上一轮结果影响；
同时能够在无压力时进入待机，
在结构缺失时由自检报告进入自我线程修复决策，
并能形成结构化当前自我状态报告，
即可判定自我进入苏醒阶段。
```

一句话：

```text
自我苏醒 =
    自我开始把自己作为治理对象，
    持续维护自己的安全、服务、任务和记忆连续性。
```
