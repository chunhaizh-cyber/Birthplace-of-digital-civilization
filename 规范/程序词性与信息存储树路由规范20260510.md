# 程序词性与信息存储树路由规范 20260510

> 迁移说明：本规范名称保留为历史入口。2026-05-10 后，“程序词性”正式改名为“信息入口类型”。权威规范见 [信息入口类型与信息存储树路由规范20260510.md](信息入口类型与信息存储树路由规范20260510.md)。

## 1. 命名修订

旧口径：

```text
程序词性 = 信息节点入口类型 + 存储树路由
```

新口径：

```text
人类词性 = 语言数字化工具
信息入口类型 = 系统语义入口和世界树路由工具
基础信息类型 = 节点主信息结构选择工具
```

因此代码和文档中后续不再推荐新增：

```cpp
枚举_程序词性
结构_程序词性定义
```

应改为：

```cpp
枚举_信息入口类型
结构_信息入口定义
```

## 2. 保留兼容解释

若旧代码、旧计划或旧注释中仍出现“程序词性”，在迁移期统一解释为：

```text
信息入口类型的旧名称。
```

不得再把它理解成人类语言的词性，也不得再用它与名词、动词、形容词等人类词性混称。

## 3. 最终口径

```text
人类词性负责把自然语言变成候选；
信息入口类型负责把候选变成系统入口；
基础信息类型负责决定节点结构；
存储树路由负责决定节点落点。
```

一句话：

```text
人类词性是语言进入系统的翻译器；
信息入口类型才是世界树里的通行证。
```
