# 0020 自我线程运行三包规范 v0.1规范

## 1. 目标

本文用于把 `自我线程` 的运行态，正式收口成三张线程运行票据：

- `线程轮次上下文包`
- `一步治理任务包`
- `回流结算包`

本文要固定的是：

- 三包各自的定位、生命周期和写入归属
- 三包的字段组、封口时机和核心规则
- 三包之间怎样把 `消息输入 -> 需求树/需求队列治理 -> 方法禁止项更新 -> 一步治理 -> 工作线程承接 -> 结果回写` 串成真正可跑的数据合同链
- 为什么包必须是短生命运行态，而不能和世界/知识节点混成一类对象

本文不负责：

- 发明新的模块
- 改写 `12` 模块边界
- 定义节点级字段全集
- 复制世界对象的大副本

若当前继续把三包数据合同进一步落成 `统一枚举 / C++ 结构体 / 字段拥有者 / 封口规则 / 持久化策略 / 包到节点映射器`，按 [0021_自我线程运行包结构与映射规范_v0.1规范.md](D:\鱼巢\规范\0021_自我线程运行包结构与映射规范_v0.1规范.md) 执行。

## 3. 硬边界：包不是节点

必须先定一条硬边界：

`包不是节点。`

这意味着：

- `包` 是线程运行态，短生命
- `节点` 是世界 / 知识持久态，长生命

因此必须固定：

1. 包里尽量只放：
   - `主键`
   - `可解析引用`
   - `快照摘要`
   - `结算结果`
2. 包里默认不复制：
   - 大对象副本
   - 整条链表
   - 场景全量状态
3. 包是线程轮次的数据合同，不是新的知识实体。
4. 包可以引用节点，但不能冒充节点。

这也与当前底座口径一致：

- 场景节点主要保留 `状态 / 动态 / 关系 / 实例因果索引`
- 动态节点主要记录 `来源动作 / 输入输出场景 / 执行结果`
- 因果实例主要记录 `来源任务 / 来源方法 / 置信度 / 已验证`


## 4. 三包总览

三包在主链中的角色，固定为：

- `线程轮次上下文包`
  - 相当于本轮宪法
  - 说明“这一轮为什么能跑、当前世界是什么样、需求树与需求队列是什么样、哪些方法当前被禁止、全局队列怎么排、根层怎么裁决”

- `一步治理任务包`
  - 相当于本轮执照
  - 说明“这一轮唯一允许落地的那一步是什么、任务入口是什么、能做到哪、准备用什么方法做，以及哪些方法已被过滤”

- `回流结算包`
  - 相当于本轮账本
  - 说明“这一轮实际发生了什么、派生需求有哪些、子结果该路由给谁、缩差了没有、越界了没有、哪些方法该继续禁止/解除禁止、学习候选是什么”

三包之间的最短主链固定为：

`唤醒事件 -> 生成线程轮次上下文包 -> 自我线程基于需求树生成需求队列并更新方法禁止项摘要 -> 自我线程创建任务入口并调度任务管理工作线程 -> 任务管理任务生成一步治理任务包并补齐方法/执行策略摘要与过滤结果 -> 工作线程承接 -> 生成回流结算包 -> 主循环做派生需求裁决 / 结果路由 / 最终结算 -> 生成下一轮上下文包，或进入待机 / 收束 / 停止 / 退出`


## 5. 公共包头

为便于后续 C++ 落地，建议先固定一层公共包头：

`结构_线程运行包头`

最小字段建议如下：

| 字段 | 作用 |
| --- | --- |
| `包主键` | 当前包的唯一标识 |
| `轮次主键` | 当前线程轮次标识 |
| `线程主键` | 当前线程实例标识 |
| `宿主存在主键` | 当前统一主体标识 |
| `父轮次主键` | 上一轮来源 |
| `追踪签名` | 全链路跟踪 |
| `开始时间` | 本包开始时间 |
| `结束时间` | 本包结束时间；未封口前可空 |

必须固定：

- 三包都应复用同一类包头语义
- `轮次主键 + 追踪签名` 是三包串链的主轴
- 包头只负责追踪，不负责业务裁决


## 6. 《线程轮次上下文包》

### 6.1 定位

它不是任务包，而是：

`本轮主循环已经判好的运行上下文。`

它装的不是“准备做什么”，而是：

- 这一轮为什么能跑
- 当前世界是什么样
- 差额在哪里
- 根层怎么裁决

### 6.2 谁写

它必须按段写入：

- `生命周期与边界控制` 写 `边界段`
- `状态刷新` 写 `世界快照段`
- `差额评估` 写 `差额段`
- `根需求生成` 写 `根需求段`
- `冲突仲裁` 写 `仲裁段`

### 6.3 什么时候封口

`仲裁完成后封口。`

一旦进入任务管理，这个包只读，不允许局部模块回头改根判定。

### 6.4 字段组

| 字段组 | 最小内容 | 主要写入者 |
| --- | --- | --- |
| `包头` | `轮次主键 / 线程主键 / 宿主存在主键 / 父轮次主键 / 唤醒原因 / 追踪签名 / 开始时间 / 截止时间` | 主循环入口 |
| `生命周期边界段` | `当前生命周期状态 / 运行模式（正常、降级、待机、收束）/ 停止标志 / 死亡退出标志 / 安全接管标志` | 生命周期与边界控制 |
| `世界快照段` | `当前场景引用 / 关键存在引用 / 输入场景引用 / 关键状态索引摘要 / 关系索引摘要 / 异常变化摘要` | 状态刷新 |
| `差额段` | `服务值目标/当前/差额/趋势；物理安全值目标/当前/差额/趋势；风险安全值目标/当前/差额/趋势` | 差额评估 |
| `需求治理段` | `需求树摘要 / 父需求对子需求的权重调制摘要 / 需求队列摘要 / 综合权重摘要` | 优先级与权重治理 |
| `根需求段` | `服务需求候选 / 直接服务或准备性服务标记 / 安全需求候选 / 物理安全或风险安全标记` | 根需求生成 |
| `方法禁止项段` | `当前生效方法禁止项摘要 / 触发条件摘要 / 禁止原因摘要 / 解禁条件摘要` | 优先级与权重治理 |
| `全局队列段` | `可运行队列摘要 / 等待队列摘要 / 回流队列摘要 / 收束停止队列摘要` | 优先级与权重治理 |
| `仲裁段` | `主导需求 / 从属需求 / 冲突结论 / 推进等级 / 安全是否接管` | 冲突仲裁 |
| `资源与权限上限段` | `本轮资源预算 / 时间预算 / 并发上限 / 可写范围 / 最大授权等级` | 主循环与仲裁结果汇总 |
| `回流要求段` | `本轮必须回写什么 / 必须留下什么证据 / 允许哪些副作用 / 失败时必须如何结算` | 主循环与仲裁结果汇总 |

### 6.5 核心规则

必须固定：

1. `线程轮次上下文包` 代表 `根层本轮共识`。
2. 局部线程、单个方法和学习候选沉淀接口都只能读取，不能改写。
3. 一轮只允许一个有效 `线程轮次上下文包`。
4. 若需要重判，必须开新轮，不得在旧轮里偷偷改宪法。


## 7. 《一步治理任务包》

### 7.1 定位

它是任务管理中轴把 `根需求` 压成的：

`本轮唯一有效一步。`

它不是：

- 长期规划
- 整棵任务树总表
- 任意扩张的执行许可证

### 7.2 谁写

这个包必须分四段归属：

- `自我线程` 写 `来源段 / 任务入口段`
- `任务管理任务` 写 `治理定义段`
- `最小授权接口` 写 `授权约束段`
- `任务管理任务（含内部策略器）` 写 `方法段`

### 7.3 什么时候封口

`方法选定后封口。`

进入执行阶段后，不再改任务语义和授权边界；只能按包执行，或者被安全守卫打断。

### 7.4 字段组

| 字段组 | 最小内容 | 主要写入者 |
| --- | --- | --- |
| `包头` | `任务包主键 / 轮次主键 / 治理序号 / 任务节点主键 / 需求节点主键 / 父任务主键` | 任务管理中轴 |
| `来源段` | `来自哪个根需求 / 触发事实 / 这一步要缩哪个差额 / 预期缩到什么程度` | 任务管理中轴 |
| `任务入口段` | `任务根节点主键 / 任务入口主键 / 任务根只读标记 / 调度工作线程主键` | 自我线程 |
| `治理定义段` | `任务语义键 / 动作类型 / 目标对象 / 目标状态 / 目标关系 / 预期结果 / 完成条件` | 任务管理中轴 |
| `授权约束段` | `动作白名单 / 资源配额 / 时间窗 / 并发上限 / 停止条件 / 回滚许可 / 是否必须人工确认 / 是否允许真实写回` | 最小授权接口 |
| `方法段` | `方法需求描述 / 候选方法列表 / 选定方法主键 / 前置条件 / 备选顺位 / 预期输入场景 / 预期输出场景` | 任务管理中轴（含内部策略器） |
| `方法门控段` | `方法禁止项快照 / 被过滤方法摘要 / 被过滤原因 / 解禁前提摘要` | 任务管理中轴（含内部策略器） |
| `监督段` | `要求记录哪些动态 / 要求保留哪些证据 / 失败回调路由 / 安全监测等级` | 任务管理中轴 + 工作线程接口汇总 |

其中，`动作类型` 当前建议先固定为以下枚举语义：

- `观察`
- `解释`
- `预警`
- `建议`
- `执行`
- `回滚`
- `收束`

### 7.5 核心规则

必须固定：

1. `一步治理任务包` 只做一步。
2. `一步治理任务包` 只在本轮有效。
3. `一步治理任务包` 不得借执行之名扩权。
4. 一个 `上下文包` 最多只允许派生一个活跃的 `一步治理任务包`。
5. `任务根节点` 在任务包里只能以 `只读任务入口` 方式被引用，不得被局部线程改写。
6. `任务管理任务` 的“只能调路径，不能改方向”，在这里必须落实成字段边界，而不是口头原则。


## 8. 《回流结算包》

### 8.1 定位

它不是日志，而是：

`根层结算依据。`

任何一步治理，不论：

- 成功
- 失败
- 未命中
- 被打断
- 授权失败

都必须产出结算包。

### 8.2 谁写

也必须分段归属：

- `工作线程承接接口` 写 `执行事实段`
- `边界守卫接口` 写 `安全异常段`
- `结果回流 / 结算模块` 写 `缩差结算段`
- `学习候选沉淀接口` 只写 `学习候选段`

### 8.3 什么时候封口

`根层完成结算并给出下一轮入口后封口。`

### 8.4 字段组

| 字段组 | 最小内容 | 主要写入者 |
| --- | --- | --- |
| `包头` | `结算包主键 / 轮次主键 / 任务包主键 / 执行尝试号 / 开始时间 / 结束时间` | 回流结算模块 |
| `执行事实段` | `选定方法主键 / 来源输入场景 / 来源输出场景 / 来源执行成功 / 来源错误码 / 动态路径签名 / 是否超时 / 是否被中断` | 工作线程承接接口 |
| `状态变化段` | `状态变化列表 / 关系变化列表 / 副作用列表 / 资源消耗 / 权限消耗 / 是否越过预期范围` | 工作线程承接接口 + 回流结算模块 |
| `安全异常段` | `是否降级 / 是否回滚 / 是否触发收束 / 是否建议停止 / 异常来源 / 止损原因` | 边界守卫接口 |
| `派生需求候选段` | `派生需求候选 / 来源任务主键 / 来源步骤摘要 / 是否建议实例化` | 任务管理任务返回面 |
| `结果路由段` | `父任务主键 / 路由目标 / 路由原因 / 是否已路由父层任务` | 结果回流、结算与重判 |
| `缩差结算段` | `服务差额前后值 / 物理安全差额前后值 / 风险安全差额前后值 / 净缩差或净恶化或无效推进 / 本轮是否算有效推进` | 结果回流、结算与重判 |
| `方法禁止项更新段` | `新增/续期/解除禁止建议 / 命中的触发条件 / 禁止原因 / 来源方法主键 / 解禁条件摘要` | 结果回流、结算与重判 |
| `学习候选段` | `因果实例候选 / 来源任务主键 / 来源方法主键 / 置信度 / 已验证默认 false / 样本类别 / 黑名单提示` | 学习候选沉淀接口 |
| `下一轮建议段` | `继续下一轮 / 待机 / 收束 / 停止 / 死亡退出 / 人工接管 / 是否建议重试 / 是否需要人工确认` | 结果回流、结算与重判 |

### 8.5 核心规则

必须固定：

1. 每个 `一步治理任务包` 必须对应一个 `回流结算包`。
2. 哪怕授权失败、方法未命中、执行被打断，也要结算。
3. `先结算，再学习`。
4. 学习只能消费结算包，不能直接消费裸执行过程。
5. `学习候选段` 必须默认 `已验证 = false`，与已验证因果分开。
6. `方法禁止项更新段` 只能在结算后生成，不能直接从裸执行过程跳写成正式禁止项。
7. `派生需求候选` 只能在结算后由主循环决定是否实例化为新任务。
8. `子结果` 必须先进入 `回流结算包`，再由主循环路由给父层任务。


## 9. 三包之间的主链

三包主链固定为：

`唤醒事件 -> 生成线程轮次上下文包 -> 自我线程创建任务入口并调度任务管理工作线程 -> 任务管理任务生成一步治理任务包 -> 工作线程承接 -> 生成回流结算包 -> 主循环做派生需求裁决 / 结果路由 / 最终结算 -> 生成下一轮上下文包，或进入待机 / 收束 / 停止 / 退出`

把它再压成线程时序，就是：

`上下文包决定本轮共识 -> 任务包决定本轮唯一一步 -> 结算包决定这一步到底算什么`

必须固定：

1. 只有主循环能生成下一轮 `线程轮次上下文包`。
2. 方向永远在根层，不在局部链路手里。
3. `上下文包` 必须带出 `需求树摘要 / 需求队列摘要 / 方法禁止项摘要`。
4. `任务包` 必须带出 `方法过滤结果`。
5. `结算包` 必须带出 `方法禁止项更新建议`。
6. `派生需求` 和 `子结果路由` 都必须经由结算包回到主循环。
7. 三包串起来之后，自我线程才算从“会转的状态机”变成“有血流的数据合同”。


## 10. 五条不变量

必须立下以下五条不变量：

1. `一轮只允许一个有效上下文包。`
   - 需要重判，就开新轮
   - 不允许在旧轮里偷偷改宪法

2. `一个上下文包最多派生一个活跃的一步治理任务包。`
   - 防止局部线程自己长枝杈

3. `每个任务包必须对应一个结算包。`
   - 无论成功、失败、未命中还是被打断

4. `学习候选默认未验证。`
   - 不允许执行结果直接跳过结算和校验，写成稳定知识

5. `只有主循环能生成下一轮上下文包。`
   - 根方向永远不下放
