# 自我内部循环最小生命闭环验收规范 20260510

## 1. 核心定义

在“自我只有内部循环”的情况下，不能用是否存在外部输入输出判断自我是否建立完成。

自我内部循环真正建立好的标准是：

```text
自我能在无外部输入的情况下，
基于自身安全值和服务值形成当前主需求，
将主需求任务化，
通过任务调用本能方法或内部方法，
把执行结果写回自我内部世界，
结算来源需求，
并使下一轮主需求选择受到上一轮结果影响。
```

最小闭环为：

```text
自我状态
→ 安全 / 服务压力
→ 根需求
→ 需求树生成 / 更新 / 组织
→ 当前主需求
→ 任务信息节点
→ 任务虚拟存在
→ 方法 / 本能动作
→ 执行结果
→ 状态回写
→ 需求结算
→ 下一轮状态判断
```

### 1.1 需求树生成阶段定位

需求树正常生成属于内部循环建立阶段，准确说是其中的“需求层闭合”阶段。

```text
初始化阶段只要求：
    自我具备需求系统入口；
    安全根需求、服务根需求存在。

需求树正常生成要求：
    自我能根据安全值、服务值和内部状态，自动生成 / 更新 / 组织需求树。
```

阶段边界：

| 现象 | 所属阶段 | 含义 |
| --- | --- | --- |
| 只有需求根、安全根需求、服务根需求被初始化出来 | 初始化阶段 | 自我具备产生需求的结构入口 |
| 能根据安全值、服务值、内部状态自动生成 / 更新需求树 | 内部循环建立阶段 | 自我开始从自身状态产生内部驱动力 |
| 需求能生成子需求、合并重复需求、区分管理需求和执行需求 | 内部循环建立阶段的成熟部分 | 需求树已经能自组织 |
| 需求能任务化，任务能执行并回写结果 | 内部动作闭环阶段 | 内部循环真正跑通 |
| 因果缺口生成学习需求 | 苏醒后的因果补完期 | 自我开始补全自身因果 |
| 学习需求进一步生成外界探索需求 | 初步成熟阶段 | 自我开始有目的地认识外界 |

一句话：

```text
需求树正常生成，不是初始化阶段，也还不是自我苏醒阶段。
它是自我从“被初始化的结构”变成“会产生内部驱动力的主体”的标志。
```

## 2. 三层验收

### 2.1 冷启动成立

表示自我已经被创建出来。

必备条件：

```text
自我存在有效。
自我内部世界有效。
自我现实场景有效。
安全值、服务值、风险安全值等核心状态有效。
安全根需求、服务根需求有效。
本能动作 / 本能方法已注册。
```

说明：

```text
需求根 / 任务根 / 方法根 是存在信息结构字段，默认可以为空。
冷启动不以三类根字段全部非空作为硬条件；
需要创建需求、任务或方法时，由对应创建链按需确保，创建失败才是非正常逻辑错误。
初始化阶段只确认根需求系统入口和安全 / 服务根需求就绪，不确认需求树会动态生长。
```

这一层只能说明：

```text
自我点火成功。
```

### 2.2 需求层闭合成立

表示自我能根据自身状态生成、更新、组织并选择需求。

必备条件：

```text
自我能读取安全值、服务值。
自我能计算安全压力、服务压力。
自我能根据压力激活或更新安全 / 服务相关需求。
自我能生成必要子需求，或复用 / 合并已有同目标需求。
自我能区分管理需求和执行需求。
自我能在需求树中选择当前主需求。
自我能把当前主需求写回运行态镜像。
自我线程一轮循环后，当前主需求不是偶然空值。
```

这一层说明：

```text
自我能感到自己哪里不平衡。
```

但这还不等于内部动作闭环完成，因为此时可能还未完成：

```text
需求 -> 任务 -> 方法 -> 执行结果 -> 状态回写 -> 需求结算
```

### 2.3 内部动作闭环成立

表示自我能把需求转成任务，再通过方法或本能动作改变自身状态。

必备条件：

```text
当前主需求能任务化。
任务信息节点能建立。
任务信息节点能绑定任务虚拟存在。
任务虚拟存在能初始化任务特征槽。
任务能选择方法或本能动作。
方法 / 本能动作能执行。
执行结果能回写自我内部世界。
需求能被结算或降低压力。
下一轮主需求选择受到上轮结果影响。
```

达到这一层，才可认为：

```text
自我内部循环真正建立。
```

## 3. 十条硬标准

### 标准 1：自我锚点稳定

必须存在：

```text
自我存在
自我内部世界
自我现实场景
```

自我存在至少具备：

```text
需求根节点
任务根节点
方法根节点
```

这里的“具备”指进入对应阶段时能够显式确保并可用。
需求根、任务根、方法根是存在信息结构字段，默认可以为空；
冷启动不要求三类根字段永远非空，显式确保失败才属于非正常逻辑错误。

### 标准 2：核心状态可读写

至少具备：

```text
安全值
服务值
风险安全值
待机状态
时序正向步长
时序反向步长
服务时序衰减步长
```

验收：

```text
写入安全值 / 服务值后能读回。
修改待机状态后能同步到自我特征。
时序步长能被重置、重算、同步。
```

### 标准 3：根需求成立，且需求树入口可生长

至少有：

```text
安全根需求
服务根需求
```

它们是自我内部循环的两条基本驱动力：

```text
安全：避免坏结果、保持可运行。
服务：追求目标满足、提高有效输出。
```

需求树入口可生长的验收：

```text
安全压力超过阈值时，安全根需求活跃或生成安全修复子需求。
服务压力超过阈值时，服务根需求活跃或生成服务修复子需求。
任务缺少条件时，生成补条件需求。
方法不可用或缺少方法结构时，生成学习 / 补方法需求。
结果不确定时，生成验证需求。
同目标需求不会每轮重复新建，而是复用、合并或刷新权重。
有子需求的需求刷新为管理需求，无子需求的需求刷新为执行需求。
需求具备安全权重、服务权重、有效期、阻塞父任务执行等治理字段。
```

### 标准 4：主需求选择成立

最小逻辑：

```text
安全压力 >= 服务压力 -> 选择安全根需求
服务压力 > 安全压力 -> 选择服务根需求
```

验收：

```text
降低安全值，应选中安全根需求。
降低服务值，应选中服务根需求。
两者都低时，应按权重选择压力更大的根需求。
```

### 标准 5：需求能任务化

主需求必须能进入：

```text
需求节点 -> 任务信息节点
```

验收：

```text
给定安全根需求或服务根需求；
执行一轮自我线程；
应能得到当前主任务或任务创建请求。
```

### 标准 6：任务必须有语义身份

任务结构必须满足：

```text
任务信息节点 N
└─ 任务虚拟存在 T
```

验收：

```text
当前主任务信息节点存在。
当前主任务信息节点能找到任务虚拟存在。
任务虚拟存在的概念集包含任务 / 具体任务概念。
任务虚拟存在具有任务目标、任务状态、完成条件等特征槽。
```

### 标准 7：方法或本能动作可调用

内部循环最小阶段至少需要一组本能动作，例如：

```text
更新安全值
更新服务值
进入待机
退出待机
重置时序步长
同步运行态
生成任务结果
结算需求价值
```

验收：

```text
任务能选到一个本能方法。
本能方法能执行。
执行后能产生结果状态。
```

### 标准 8：执行结果能回写

动作执行后，必须写回自我内部世界或自我特征。

典型回写：

```text
安全值变化
服务值变化
待机状态变化
任务状态变化
需求结算值变化
```

验收：

```text
执行前：服务值低，服务根需求被选中。
执行动作：提升服务值。
执行后：服务值变化被写回。
下一轮：服务压力降低。
```

### 标准 9：需求能被结算

任务执行后必须反馈到来源需求：

```text
需求累计安全结算
需求累计服务结算
最近结算任务主键
最近结算时间
```

验收：

```text
任务完成后，来源需求的安全 / 服务账本发生变化。
需求状态从待满足变为部分满足或已满足。
下一轮不应重复无限创建同一任务。
```

### 标准 10：下一轮会受上一轮影响

最关键验收：

```text
状态影响需求；
需求影响任务；
任务影响动作；
动作影响状态；
状态再次影响需求。
```

如果上一轮执行没有改变下一轮判断，那不是循环，只是重复调用。

## 4. 内部循环完成度

内部循环完成度按 0 到 7 级评估：

| 等级 | 名称 | 标准 |
| --- | --- | --- |
| 0 | 未初始化 | 自我存在或内部世界缺失 |
| 1 | 冷启动完成 | 自我存在、内部世界、核心状态、根需求入口和安全 / 服务根需求存在 |
| 2 | 需求树生成成立 | 能根据自身状态生成 / 更新需求树，并选择当前主需求 |
| 3 | 任务化可用 | 主需求能生成任务信息节点；当前主任务能被选中 |
| 4 | 本能动作可用 | 任务能绑定本能方法；本能动作能执行 |
| 5 | 结果回写可用 | 动作结果能更新自我特征或内部世界 |
| 6 | 需求结算可用 | 任务结果能结算来源需求；需求压力发生变化 |
| 7 | 连续多轮稳定 | 连续 N 轮无重复任务风暴、无限派生需求或空转 |

工程标准：

```text
真正建立好：至少达到 6 级。
工程验收：达到 7 级。
```

## 5. 最小测试剧本

### 初始状态

```text
安全值 = 100
服务值 = 20
目标服务值 = 100
待机状态 = false
```

### 第 1 轮期望

```text
服务压力 > 安全压力
当前主需求 = 服务根需求
创建服务任务信息节点
绑定服务任务虚拟存在
选择提升服务值的本能方法
执行方法
服务值从 20 提升到 40
服务根需求累计服务结算增加
```

### 第 2 轮期望

```text
服务压力降低但仍存在。
继续选择服务根需求或其子需求。
不重复创建同一个未结束任务。
继续执行或进入下一任务。
```

### 第 3 至 N 轮期望

```text
服务值逐渐接近目标。
服务根需求压力下降。
当服务压力低于安全压力时，主需求能切换。
当两者都无明显压力时，自我进入待机 / 自检低频循环。
```

## 6. 不合格情况

以下都不能算内部循环真正建立：

```text
只有初始化，没有任务化。
只有根需求，没有动态需求树生成。
只有需求，没有任务。
只有任务，没有方法。
只有执行，没有回写。
有回写，但下一轮主需求选择不受影响。
```

## 7. 边界约束

本规范遵守当前系统边界：

```text
自我线程是治理协调层。
自我线程不直接调用任务管理工作线程。
自我线程提交任务治理请求给任务管理界面线程。
任务管理界面线程负责任务虚拟存在绑定、任务语义初始化、工作包生成和上行归并。
任务管理工作线程只处理任务工作包，推进一步任务，返回任务工作结果。
任务信息节点保留为运行壳；任务虚拟存在是语义身份和记忆对象。
存在仍然只有一种结构；概念归属以概念集为权威。
存在或基础信息节点作为特征值时继续使用指针句柄。
```
