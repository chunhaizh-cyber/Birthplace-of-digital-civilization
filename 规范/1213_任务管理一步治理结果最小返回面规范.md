## 3. 核心结论

必须固定以下结论：

1. `一步治理结果` 是 `任务管理任务` 单轮推进后的最小返回面，不是最终裁决，也不是整包执行现场转储。
2. `一步治理结果` 的第一职责，是把 `本轮到底推进了什么、显性化了什么、建议往哪走` 可靠交回外层主循环。
3. `一步治理结果` 必须至少能支撑：`外层根层重判 -> 主链调度 -> 再提交` 这一整条链不断。
4. `一步治理结果` 中的 `当前下一步去向候选`、`收束建议`、`重筹办建议`、`学习子任务候选` 都只是候选或建议，不得直接跳升成最终裁决。
5. `一步治理结果` 必须优先返回稳定治理中间态，不要求把所有原始分析过程、原始消息包、原始场景快照整包带回。
6. `一步治理结果` 可以挂带管理证据和摘要，但证据挂带不等于结果本体。
7. `一步治理结果` 若不能回指 `当前任务 / 当前步骤 / 当前方法 / 最近结果` 中至少必要的一组锚点，外层不得据此继续推进主链。
8. `一步治理结果` 不允许直接给出“需求已满足”“主链已结束”“允许升格”“允许回流”这类越权结论。
9. 叶子完成结果回到外层后，应先回自我主循环，再结合需求列表与需求树做结果回挂，而不是由局部任务自己直接写成整棵树最终真值。
10. 若系统启用了主体层，`一步治理结果` 允许额外挂带 `主体回并摘要 / 当前方法需求位 / 当前本能能力缺口类型 / 输入条件主键 / 治理态型` 等桥接字段，但这些字段仍然只是治理证据和承接锚点，不得越权冒充最终裁决。

一句话收口：

`一步治理结果是任务管理任务把“本轮治理改变了什么、形成了什么中间结论、建议下一步往哪走”交回外层的最小稳定返回面。`


## 4. 定义

`一步治理结果` 指的是：

- `任务管理任务` 在完成一次单步治理推进后，交给外层主循环、调度器、恢复层继续消费的最小返回结构

它回答的是：

- 本轮到底作用在谁身上
- 本轮落在哪个治理域
- 本轮显性化了什么缺口
- 本轮建议下一步往哪走
- 本轮有没有写回结构、方法挂点、结果节点、等待态
- 本轮是否额外产生学习子任务候选或收束/重筹办建议

它不是：

- 根层重判本体
- 需求满足结论
- 叶子执行结果整包
- 调度器内部瞬时状态全集

### 5.2 可选挂带字段

以下字段允许挂带，但不属于“最小稳定返回核心”：

- `学习子任务候选`
- `收束建议`
- `重筹办建议`
- `结构写回摘要`
- `方法候选写回摘要`
- `管理证据摘要`
- `原因说明`
- `实例因果锚点`
- `动作动态锚点`
- `主体回并摘要`
- `当前方法需求位`
- `当前本能能力缺口类型`
- `输入条件主键`
- `治理态型`

必须固定：

1. 这些字段主要服务解释、审计、成长与下轮更快承接。
2. 即使这些字段全部缺失，只要核心字段完整，主链仍应能继续。
3. 这些字段若存在，也不得越过外层重判直接变成最终结论。


## 6. 与任务管理上下文的关系

`任务管理上下文` 与 `一步治理结果` 的关系，应固定为：

```text
上一轮任务管理上下文
-> 任务管理任务单步推进
-> 形成一步治理结果
-> 外层消费一步治理结果并刷新下一轮任务管理上下文
```

因此必须固定：

1. `一步治理结果` 不是 `任务管理上下文` 的简单原样返回。
2. 它是“上下文被消费一次之后的新产物”。
3. 下一轮 `任务管理上下文` 中的稳定字段，很多都应来自上一轮 `一步治理结果` 的镜像更新。
4. 其中 `当前方法需求位 / 当前本能能力缺口类型 / 输入条件主键 / 治理态型`，通常来自上一轮的 `治理特征面`，允许作为下一轮上下文稳定字段继续承接。


## 7. 与消息、请求、恢复的关系

### 7.1 与返回消息的关系

`一步治理结果` 与 [0950_任务调度器消息驱动规范.md](D:\鱼巢\规范\0950_任务调度器消息驱动规范.md) 中的 `返回消息` 关系如下：

- `返回消息` 更偏向执行器与调度器的通信载体
- `一步治理结果` 更偏向任务管理任务交给外层主循环的稳定治理返回面

两者允许重叠，但不应混成同一个概念。

### 7.2 与任务管理请求的关系

`任务管理请求` 是输入，`一步治理结果` 是输出。

必须固定：

1. 请求里允许带 `当前场景 / 调度时间 / 特征快照整包` 这类补充材料。
2. 结果里优先返回稳定中间态，不要求把这些补充材料原样带回。

### 7.3 与恢复快照的关系

`恢复快照` 只应优先抽取 `一步治理结果` 中：

- `当前功能域判定结果`
- `当前缺口类型`
- `当前下一步去向候选`
- `当前治理状态迁移`
- `当前步骤 / 当前方法 / 最近结果` 的镜像
- `最近总控结果`
- `最近反馈类型 / 最近反馈摘要`
- 可选的 `当前方法需求位 / 当前本能能力缺口类型 / 输入条件主键 / 治理态型`

而不是整包保存原始治理返回对象。


## 8. 严格禁止的混层

以下做法必须禁止：

1. 把 `当前下一步去向候选` 直接当成最终裁决。
2. 把 `收束建议 / 重筹办建议 / 学习子任务候选` 直接当成已获批准的动作。
3. 把 `最近结果节点主键` 当成 `最近总控结果` 或 `最近结果节点` 的替代品。
4. 把 `一步治理结果` 直接写成“需求已满足”“允许升格”“允许回流”。
5. 把原始反馈包整包复制进稳定治理结果面。
