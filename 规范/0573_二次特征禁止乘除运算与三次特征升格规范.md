## 3. 核心结论

必须固定以下 12 条：

1. 本文中的 `二次特征` 作为泛称时，适用于 `2次特征` 到 `n次特征` 整个高次特征族。
2. 这个高次特征族的核心数值母式，只允许建立在 `两个同层真值的显式比较与相减` 之上。
3. 这个高次特征族的核心数值结果，只允许是 `差值` 或由差值直接得到的 `区间归属`。
4. `2次特征` 到 `n次特征` 中禁止出现 `乘法`。
5. `2次特征` 到 `n次特征` 中禁止出现 `除法`。
6. 任何以 `乘法 / 除法` 为核心步骤得到的结果，都不得写成正式 `2次- n次特征` 结果。
7. `差异度 / 差值` 本身可以是合法高次特征结果。
8. `方向` 只能由差值落入 `正区间 / 零区间 / 负区间` 三大区间得到。
9. `变化率 / 速率 / 比例 / 归一化Q / 加权值 / 综合分` 不属于正式 `2次- n次特征` 本体。
10. 严格 `3次特征` 不是乘除结果，而是 `两个2次特征(int)` 再次相减。
11. 高次特征应优先保留 `左值 / 右值 / 差值 / 方向区间` 这组可回溯字段，而不是把乘除产物写死成新的“特征结果”。
12. 对当前阶段而言，`除法` 还属于更高层派生概念，不应进入正式 `2次- n次特征` 母式。

一句话收口：

`2次到n次特征都只负责“同层两个值相减得到差值，再由三大区间判断方向”；除法仍属于更高层派生概念，一旦进入乘除，就已经越过高次特征族边界。`


## 4. 二次特征的合法算术边界

### 4.1 合法核心母式

二次特征的正式核心母式固定为：

```text
左值 = A
右值 = B
差值 = 右值 - 左值
方向区间 = sign(差值) -> 正区间 / 零区间 / 负区间
```

这里必须强调：

1. `左值 / 右值` 必须显式可回指，不能只剩匿名结果。
2. `差值` 必须显式可回推到这两个输入值。
3. `方向区间` 只能来自 `差值的符号归属`，不能另造第四种方向算式。
4. 不同业务可以改变 `左值 / 右值` 的语义角色，但不能跳过“两个值相减”这一步。

例如：

- `目标值 - 当前值`
- `结果值 - 起始值`
- `右对象值 - 左对象值`
- `同样时间段内的 ΔF(后窗) - ΔF(前窗)`
- `固定 ΔA 后，用减法得到 ΔB`

都可以成立，但都必须显式保存“到底谁是左值、谁是右值、差值按什么顺序得到”。

### 4.2 合法附属操作

二次特征允许出现的附属操作，仅限于不改变其“比较 + 相减 + 落区间”本质的工程动作，例如：

- 空值检查
- 类型检查
- 同类值比较
- 区间包含判断
- 正负零三分
- 名称映射
- 版本标记

这些动作只允许承担：

- 生成前提检查
- 区间归属落点
- 工程兼容与字段写回

它们不得把二次特征偷偷扩写成 `比率 / 权重分 / 复合评分`。


## 5. 明确禁止的算式

以下结果一律不得作为正式二次特征写回：

1. `A * B`
2. `A / B`
3. `(B - A) / T`
4. `(B - A) / Range`
5. `(B - A) * Weight`
6. `Σ(二次特征 × 权重)`
7. `A / Target`
8. `A * Gate`
9. `差值 × 系数`
10. `差值 ÷ 时间窗`

必须固定：

- 只要公式主核里有 `乘` 或 `除`，就已经不是“两个值相减后的直接二阶关系”。
- 即使该结果看起来仍很“接近比较”，也不得继续叫二次特征。
- 不允许因为它“看起来很方便”或“暂时无处可放”，就把乘除产物降格塞回二次特征。


## 6. 越界结果去向规则

以下旧口径自本文生效后废止：

- `变化率是三次特征`
- `归一化差异是三次特征`
- `加权差异是三次特征`

它们都不再按“严格3次特征”理解。

凡满足以下任一条件，结果都已越出 `2次- n次特征族`：

1. 需要把 `差值` 再与 `时间` 做除法。
2. 需要把 `差值` 再与 `量程` 做除法。
3. 需要把 `差值` 再与 `权重` 做乘法。
4. 需要把多个二次特征做 `加权汇总`。
5. 需要把二次特征再与控制门、预算、风险系数等乘除耦合。

典型升格例子固定如下：

| 结果名 | 公式 | 正式归属 |
| --- | --- | --- |
| `变化率` | `差值 ÷ 时间` | `派生指标 / 统计结果` |
| `归一化差异度Q` | `差值 ÷ 量程` | `派生指标 / 统计结果` |
| `加权差异` | `差值 × 权重` | `派生指标 / 裁决辅助结构` |
| `综合评分` | `Σ(二次特征 × 权重)` | `裁决辅助结构` |
| `收益密度` | `收益差值 ÷ 成本` | `派生指标 / 统计结果` |

必须固定：

- 严格 `3次特征` 可以消费 `2次特征`，但消费方式仍是 `int 相减`。
- 乘除结果不得反向伪装成 `3次特征` 或更高 `n次特征`。
- “差异度本身是二次特征结果，它可以继续生成三次特征”仍成立，但这里的 `三次特征` 仅指 `两个2次特征继续相减得到的结果`。


## 7. 字段与承载建议

二次特征的正式最小字段组，至少应优先包括：

- `比较左值`
- `比较右值`
- `差值`
- `方向区间`
- `区间归属`

若当前实现需要继续兼容旧字段，也必须固定：

1. `标量值` 可以承载 `差值`。
2. `离散编码 / 离散语义键` 可以承载 `方向区间`。
3. 但二次特征主信息里不应再把 `变化率 / 比例 / 加权值` 当成正式本体字段写回。
4. 若确需保留这些结果，应新建 `派生指标` 承载面，或明确落到更高层的治理/裁决辅助结构。


## 8. 对旧口径的受控修订

对仓内既有文档与实现，本文作如下受控修订：

1. 旧口径中凡把 `变化率` 视为二次特征的，现统一改读为：`n次特征外的派生指标`。
2. 旧口径中凡把 `归一化差异度Q`、`相对比例`、`加权差异` 直接写在二次特征层的，现统一改读为：`高一层派生结果`。
3. `时间归一` 若实质依赖除法，不再属于正式二次特征母式。
4. 当前代码中若仍保留旧字段或旧命名，只能作为兼容过渡，不得再新增同类旧口径实例。


## 9. 审计清单

后续审计二次特征实现时，至少检查以下 8 项：

1. 是否显式保存了 `左值 / 右值 / 差值`。
2. 方向是否只通过 `正区间 / 零区间 / 负区间` 得到。
3. 是否在二次特征生成函数中出现了 `*`。
4. 是否在二次特征生成函数中出现了 `/`。
5. 是否把 `变化率 / 归一化 / 比例 / 加权值` 写成了正式二次特征节点。
6. 是否存在“没有左值右值，只剩一个匿名评分”的伪二次特征。
7. 是否把派生指标结果错误回写到了二次特征主信息。
8. 若出现旧口径兼容代码，是否明确标注了迁移目标。


## 10. 最终收口

必须固定以下最终收口句：

`2次到n次特征的数值真身，只能是同层两个值之间的差值及其三区间方向；凡乘除产物，都已越出这条正式高次特征链，不得继续伪装成3次特征或其他n次特征。`
