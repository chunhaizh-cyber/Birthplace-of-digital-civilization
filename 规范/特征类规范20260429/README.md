# 特征类规范 20260429

状态：当前权威规范。

适用范围：特征类、特征节点主信息、抽象特征、实例特征、特征区间、稳态升格、特征实例化、结构体语义替代、特征与存在/场景/任务/方法虚拟存在的关系。

本规范覆盖 2026-04-29 讨论后收敛的新逻辑。旧的特征类说明、特征值类说明和归档规范只作为历史实现参考，不再作为特征系统的本体定义依据。

## 1. 总原则

特征不是结构体字段，也不是日志标签。特征是系统发现、命名、复用和治理的一种区分能力。

项目中的长期语义承载应逐步从结构体字段迁移到抽象特征与实例特征。结构体可以继续作为无语义的临时参数、返回值、消息包、锁状态或局部算法缓存存在，但不得再作为世界模型中的权威语义本体。

特征系统分为两层：

| 层 | 节点 | 挂载位置 | 当前值 | 区间 | 职责 |
| --- | --- | --- | --- | --- | --- |
| 定义层 | 抽象特征 | 世界树根下的抽象特征细化链 | 无 | 有或待完善 | 定义特征维度、抽象状态、区间和细化描述。 |
| 实例层 | 实例特征 | 存在、场景或虚拟存在下 | 有 | 默认引用抽象区间，可局部覆盖 | 记录某个宿主的当前值、观测、稳态和命中状态。 |

硬规则：

1. 抽象特征不挂在存在或场景下面。
2. 抽象特征没有当前值。
3. 抽象特征只用区间、名称、类型和子抽象特征表达定义。
4. 实例特征挂在存在、场景、任务虚拟存在、方法虚拟存在等宿主下面。
5. 实例特征拥有当前值，并通过当前值命中抽象特征区间。
6. 创建实例特征前，必须已经存在对应抽象特征壳。
7. 抽象特征细化链是细化描述链，不是动态观察层级。
8. 同一宿主下的实例特征在运行层视为一层；父特征是粗略口径，子特征是详细口径。
9. 特征使用优先选择子特征；子特征缺失、不可比较或不可用时，父特征才能补位。
10. 状态信息不落在特征节点下，而落在特征宿主中，并引用实际采用的实例特征口径。

## 2. 抽象特征

抽象特征是世界树根下的特征本体节点。它表示一个可复用的特征定义或抽象特征状态。

本规范不区分“维度节点”和“状态节点”。它们统一叫抽象特征。区别只体现在区间是否存在：

| 抽象特征形态 | 区间 | 含义 |
| --- | --- | --- |
| 未定区间抽象特征 | 空 | 系统知道有这个区分维度，但区间尚未沉淀。 |
| 区间抽象特征 | 有效 I64 区间 | 系统已经确认某段值域应视为同一个抽象状态。 |
| 点区间抽象特征 | `[x,x]` | 离散枚举、布尔或唯一状态。 |

示例：

```text
抽象特征: 任务状态
  -> 抽象特征: 当前阶段
      -> 抽象特征: 筹办，区间 [1,1]
      -> 抽象特征: 执行，区间 [2,2]
      -> 抽象特征: 已完成，区间 [3,3]
      -> 抽象特征: 阻塞，区间 [4,4]
  -> 抽象特征: 是否阻塞
      -> 抽象特征: 未阻塞，区间 [0,0]
      -> 抽象特征: 已阻塞，区间 [1,1]
```

这里的 `任务状态`、`当前阶段`、`筹办`、`执行` 都是抽象特征，不再拆出另一套节点类型。`当前阶段` 的长期语义落在任务虚拟存在的实例特征上，代码结构体字段只是运行期缓存或回写来源。

## 3. 实例特征

实例特征是某个宿主存在、场景或虚拟存在实际拥有的特征。实例特征引用抽象特征，不深度复制抽象特征本体。

实例特征至少回答：

1. 属于哪个宿主。
2. 实例化自哪个抽象特征。
3. 当前值是多少。
4. 当前值命中了哪个抽象特征区间。
5. 是否有局部区间覆盖。
6. 是否形成了新的实例稳态。

实例化示例：

```text
抽象特征:
  任务状态
    -> 当前阶段
       -> 执行中 [2,2]

任务虚拟存在 A:
  实例特征: 任务状态，抽象来源=任务状态，当前值=2
  实例特征: 当前阶段，抽象来源=当前阶段，当前值=2
     当前命中抽象特征=执行中
```

这里 `当前阶段` 是 `任务状态` 的细化口径，但在任务虚拟存在的运行层仍然同层挂载，不形成真正的实例父子观察关系。

实例化不深度复制抽象特征子树。细化实例特征采用同层懒创建：只有当写入、读取或治理流程实际需要某个细化特征时，才在同一宿主下创建对应实例特征。

## 4. 区间

区间是抽象特征的核心定义字段。它表示某个抽象特征状态覆盖的值域。

硬规则：

1. V1 阶段只正式支持 `I64区间`。
2. `U64` 若要进入区间系统，必须先映射、量化或编码为可比较的 `I64` 值。
3. `VecU` 不直接使用值域区间。VecU 通常自身已经承载足够的数组特征信息，使用时要么作为指针集合/证据集合，要么先比较生成 I64 差值，再由差值进入区间系统。
4. 指针句柄不使用值域区间。需要比较时先转成相等/不等、集合差异或其他 I64 结果。

区间不是简单的显示范围，而是“当前值到抽象状态”的归并规则。

```text
实例特征当前值 = 2
抽象特征区间 = [2,2]
=> 当前值命中该抽象特征状态
```

连续值和离散值使用同一机制：

```text
连续值: [70,80] = 安全值偏高
枚举值: [2,2] = 执行中
布尔值: [0,0] = 否，[1,1] = 是
```

### 4.1 标准抽象特征：特征值差值

`特征值差值` 是用于表达两个 I64 特征值之间差异的标准抽象特征。典型计算口径为：

```text
特征值差值 = 执行后值 - 执行前值
```

它的基础子抽象状态为：

| 抽象状态 | I64 区间 |
| --- | --- |
| 等于零 | `[0,0]` |
| 大于零 | `[1,I64_MAX]` |
| 小于零 | `[I64_MIN,-1]` |

`不等于零` 不是单个连续区间，不得伪造为一个 I64 区间。V1 支持单个 `I64区间`，所以 `不等于零` 应表达为组合抽象特征：

```text
不等于零 =
  任一命中(
    小于零,
    大于零
  )
```

执行、查找或条件匹配需要使用 `不等于零` 时，应把它展开为区间集合：

```text
[I64_MIN,-1] ∪ [1,I64_MAX]
```

这条规则同样适用于 `改变特征值` 的差值约束包、方法能力包中的执行差值范围，以及任务需求条件包中的执行差值要求。

### 4.2 标准抽象特征：自我能力

`自我能力` 是用于表达自我已经具备哪些本能方法能力的标准抽象特征。它不是方法名称索引，而是一个枚举型抽象特征：

```text
自我能力.当前值 = static_cast<I64>(枚举_本能方法ID)
```

每个子抽象状态使用点区间表达一个本能方法 ID：

```text
抽象特征: 自我能力
  -> 未定义 [0,0]
  -> 读取核心状态 [6020,6020]
  -> 读取最新人类输入 [6021,6021]
  -> ...
```

规则：

1. `自我能力` 的特征值只能使用 `枚举_本能方法ID` 对应的 I64 值。
2. 子抽象状态的显示名称可以来自本能方法元信息，但逻辑匹配必须使用点区间中的枚举值。
3. 初始化时应从默认本能方法元信息表生成状态，不另建手写清单。
4. 新增本能方法 ID 并加入默认本能方法表后，应自动形成新的 `自我能力` 子状态。
5. 方法虚拟存在应保存 `自我能力 = 对应本能方法ID`，表达该方法承载的本能能力。
6. 任务虚拟存在长期保存 `选中自我能力值 = 对应本能方法ID`；方法首节点和方法虚拟存在只允许作为运行期缓存。
7. 任务、方法或需求需要表达“需要某项自我能力”时，应写入能力枚举值，而不是依赖动作名称、方法名称或显示文本。

### 4.3 标准抽象特征：特征类型

`特征类型` 可以理解为抽象特征树本身。它的值不是名称字符串，也不是有限枚举，而是特征引用。

规则：

1. 任意抽象特征、实例特征都可以成为 `特征类型` 这个特征的值。
2. 抽象特征作为值时，表示抽象特征树中的对应节点。
3. 实例特征作为值时，先归一到它引用的抽象特征；若抽象引用缺失，则退回该实例特征自身的类型词。
4. 需求条件包、方法能力包、形参包和条件结果对中的 `特征类型` 项都应保存特征引用，而不是只保存显示文本。
5. 匹配时应以抽象特征树的父子关系判断覆盖关系：需求特征类型可以沿抽象特征父链上升到能力特征类型。
6. 名称、语素文本和函数名只能用于显示、日志和调试，不能作为 `特征类型` 的权威值。

## 5. 稳态与区间生成

稳态属于实例特征，不属于抽象特征。

实例特征负责记录当前值、近帧候选、稳态列表和当前稳态索引。抽象特征只接收已经从实例稳态升格出来的区间。

主链固定为：

```text
实例特征当前值
-> 连续观测
-> 候选稳态
-> 实例稳态
-> 稳态区间
-> 升格/修正抽象特征区间
-> 形成或完善抽象特征
```

硬规则：

1. 抽象特征不保存运行中的稳态列表。
2. 稳态列表、近帧候选、近帧时间属于实例特征。
3. 抽象特征区间来自实例稳态的升格、合并、拆分或人工初始化。
4. 抽象特征壳可以先存在，区间后完善。
5. 多个存在的实例稳态允许共同修正同一个抽象区间。
6. 抽象区间允许合并。

## 6. 抽象特征生成流程

生成特征时必须先生成抽象特征壳，再创建实例特征。

标准流程：

```text
发现可区分维度
-> 创建抽象特征壳，区间可为空
-> 将抽象特征实例化到某个宿主
-> 实例特征写入当前值
-> 实例特征形成稳态
-> 稳态升格为抽象区间
-> 抽象特征区间被完善、拆分或合并
```

抽象特征壳表示“系统知道有这个区分维度”。抽象特征区间表示“系统知道这个维度中的某个稳定状态范围”。

因此允许：

```text
抽象特征存在，但区间为空。
```

这表示该抽象特征仍处于待观测、待学习、待归并阶段。

## 7. 区间重叠、覆盖、拆分与合并

同级抽象特征区间允许重叠。重叠不是非法状态，而是特征系统发现边界尚未细分的证据。

示例：

```text
状态A: [1,3]
状态B: [3,5]
当前值: 3
```

当前值同时命中两个抽象特征时，系统应记录重叠命中，而不是直接报错或强行只保留一个。

重叠治理规则：

1. 允许同级区间重叠。
2. 每次实例特征当前值命中多个抽象区间时，应记录重叠次数。
3. 应统计重叠频率、最近重叠时间、重叠涉及的抽象特征。
4. 当重叠次数或频率达到治理阈值时，应触发区间细分。
5. 细分可以生成新的更窄抽象特征区间。
6. 若多个区间长期等价、边界无治理意义，也允许合并。

实例特征允许局部覆盖抽象区间。局部覆盖用于表达某个宿主的特殊阈值、特殊误差范围或正在完善中的个体边界。

局部覆盖规则：

1. 默认使用抽象特征区间。
2. 实例特征可以保存局部区间覆盖。
3. 局部覆盖不直接改写抽象本体。
4. 局部覆盖形成稳定事实后，可以反向参与抽象区间拆分、扩展或合并。

## 8. 子特征与结构体替代

结构体的语义应转化为抽象特征树。

这里的子特征只表示特征的细化描述，不表示运行层真实父子关系。

父特征表示粗略描述，子特征表示详细描述。

原结构体：

```cpp
struct 任务状态 {
    int 当前阶段;
    int 是否阻塞;
    int 是否完成;
};
```

新表达：

```text
抽象特征: 任务状态
  -> 抽象特征: 当前阶段
  -> 抽象特征: 是否阻塞
  -> 抽象特征: 是否完成
```

实例表达：

```text
任务虚拟存在
  -> 实例特征: 任务状态，当前值=2
  -> 实例特征: 当前阶段，当前值=2
  -> 实例特征: 是否阻塞，当前值=0
  -> 实例特征: 是否完成，当前值=0
```

这里 `当前阶段`、`是否阻塞`、`是否完成` 是 `任务状态` 的细化口径，但在任务虚拟存在下仍然是一层实例特征。

硬规则：

1. 有长期语义的结构体字段应迁移为抽象子特征。
2. 结构体字段值应迁移为实例特征当前值。
3. 枚举、布尔、状态常量应迁移为点区间抽象特征。
4. 连续观测稳定态应迁移为范围区间抽象特征。
5. 字段嵌套应迁移为抽象特征细化链。
6. 不含世界语义的临时结构体可以继续使用。
7. 实例层不得照搬抽象细化链形成真实实例父子树；同一宿主下的实例特征按一层管理。
8. 读取和使用特征时，优先查找可用子特征；没有可用子特征时，父特征作为粗略补位。
9. 父特征补位不能替代子特征的精确语义；补位结果必须保留实际采用的特征口径。

## 9. 抽象特征实例化

抽象特征实例化是把世界树根下的抽象特征定义投影到某个宿主下面，形成该宿主自己的实例特征。

实例化分两类：

| 类型 | 含义 | 典型用途 |
| --- | --- | --- |
| 定义实例化 | 先在宿主下创建实例特征壳，当前值可为空 | 初始化任务、方法、需求、存在模板。 |
| 观测实例化 | 因写入当前值或观测事实而创建实例特征 | 运行结果、感知结果、学习结果自动生成。 |

实例化规则：

1. 实例特征必须引用抽象特征。
2. 实例特征不深度复制抽象定义。
3. 细化实例特征按需同层懒创建。
4. 实例特征当前值必须回到抽象特征区间中解释。
5. 实例特征若没有命中任何抽象区间，应保留观测并等待稳态升格或区间扩展。
6. 实例特征若命中多个抽象区间，应记录重叠命中。

推荐关系：

```text
实例特征.抽象来源 -> 抽象特征
实例特征.当前命中抽象特征 -> 抽象特征中的某个区间状态
实例特征.当前值 -> 当前观测值或写入值
```

## 10. 多抽象特征组合

多个抽象特征可以组合生成一个新的抽象特征。组合关系不使用子链表达，而使用“组合来源引用 + 组合规则”表达。

硬规则：

1. 细化链表示结构组成，例如 `任务状态 -> 当前阶段 -> 执行中`。
2. 组合规则表示来源依赖，例如 `方法是否可用 + 条件是否齐备 + 风险是否阻断 -> 任务是否可执行`。
3. 组合生成的新抽象特征仍然是普通抽象特征，仍挂在世界树根下的抽象特征细化链上。
4. 组合生成的新抽象特征没有当前值。
5. 组合规则只定义如何从多个来源抽象特征计算或判断出新特征值。
6. 组合结果只有实例化到宿主后，才形成实例特征当前值。
7. 新抽象特征自己的细化链只表达自身状态，不表达来源依赖。
8. 抽象特征组合依赖图必须无环。V1 阶段按 DAG 处理。

示例：

```text
抽象特征: 任务是否可执行
  组合规则:
    输入1: 方法是否可用，期望区间 [1,1]
    输入2: 条件是否齐备，期望区间 [1,1]
    输入3: 风险是否阻断，期望区间 [0,0]
    组合方式: 全部命中
    输出策略: 布尔值
    成立输出值: 1
    不成立输出值: 0

  子抽象特征:
    不可执行 [0,0]
    可执行 [1,1]
```

这里 `方法是否可用`、`条件是否齐备`、`风险是否阻断` 是组合来源，不是 `任务是否可执行` 的子抽象特征。

### 10.1 组合数据结构

迁移期内，抽象特征节点可用 `基础信息节点类 + 抽象特征主信息类` 表示。后续若新增独立 `抽象特征节点类`，可将引用类型替换为更具体的节点类型。

```cpp
using 抽象特征引用 = 可解析引用<基础信息节点类>;

enum class 枚举_抽象特征组合方式 : std::uint8_t {
    未定义 = 0,

    全部命中 = 1,
    任一命中 = 2,
    至少N项命中 = 3,

    命中计数 = 10,
    加权求和 = 11,
    有符号差值 = 12,

    方法计算 = 50,
};

enum class 枚举_组合输入角色 : std::uint8_t {
    未定义 = 0,
    主输入 = 1,
    条件输入 = 2,
    否决输入 = 3,
    权重输入 = 4,
    上下文输入 = 5,
};

enum class 枚举_组合缺失策略 : std::uint8_t {
    失败 = 0,
    视为未命中 = 1,
    忽略该项 = 2,
    使用默认值 = 3,
};

enum class 枚举_组合输出策略 : std::uint8_t {
    布尔值 = 0,
    原始计算值 = 1,
    方法返回值 = 2,
};

struct 结构_抽象特征组合输入 {
    抽象特征引用 来源抽象特征{};
    抽象特征引用 期望抽象状态{};
    std::optional<I64区间> 期望区间{};

    枚举_组合输入角色 角色 = 枚举_组合输入角色::主输入;
    枚举_组合缺失策略 缺失策略 = 枚举_组合缺失策略::失败;

    I64 默认值 = 0;
    I64 权重 = 10000;
    std::int32_t 方向 = 1;

    std::string 角色键{};
    std::uint32_t 输入序号 = 0;
};

struct 结构_抽象特征组合输出 {
    枚举_组合输出策略 输出策略 = 枚举_组合输出策略::布尔值;

    std::optional<I64区间> 成立区间{};
    I64 成立输出值 = 1;
    I64 不成立输出值 = 0;
    std::optional<I64区间> 输出值限制{};
};

struct 结构_抽象特征组合规则 {
    std::string 规则主键{};
    std::string 规则名称{};

    std::vector<结构_抽象特征组合输入> 输入{};
    枚举_抽象特征组合方式 组合方式 = 枚举_抽象特征组合方式::未定义;
    结构_抽象特征组合输出 输出{};

    std::uint32_t 最少命中数 = 0;
    可解析引用<基础信息节点类> 计算方法{};

    std::uint32_t 规则版本 = 1;
    std::int32_t 优先级 = 0;
    bool 已启用 = true;

    std::uint64_t 命中次数 = 0;
    std::uint64_t 失败次数 = 0;
    时间戳 最近命中时间 = 0;
    时间戳 最近失败时间 = 0;
};
```

字段说明：

| 字段 | 用途 |
| --- | --- |
| `来源抽象特征` | 组合输入来自哪个抽象特征。 |
| `期望抽象状态` | 输入应命中的抽象状态，可用于点区间或范围区间状态。 |
| `期望区间` | 对期望抽象状态区间的局部覆盖；存在时优先使用。 |
| `角色` | 标记该输入是主输入、条件输入、否决输入、权重输入或上下文输入。 |
| `缺失策略` | 来源实例特征缺失时如何处理。 |
| `权重` | 加权求和、排序、评分类组合使用。 |
| `方向` | `1` 表示正向，`-1` 表示反向，`0` 表示只作上下文。 |
| `组合方式` | 定义多个输入如何得到中间结果。 |
| `输出` | 定义中间结果如何变成新抽象特征实例值。 |
| `计算方法` | `方法计算` 时引用的计算方法或原函数入口。 |

### 10.2 组合求值语义

组合求值发生在实例层。

当某个宿主需要实例化组合抽象特征时，流程固定为：

```text
读取组合规则
-> 按输入引用找到宿主下对应实例特征
-> 读取输入实例特征当前值
-> 判断是否命中期望抽象状态或期望区间
-> 按组合方式计算中间结果
-> 按输出策略得到新实例特征当前值
-> 写入组合抽象特征对应的实例特征
-> 用新当前值命中该抽象特征自己的区间状态
```

缺失输入必须按 `缺失策略` 处理：

| 缺失策略 | 语义 |
| --- | --- |
| `失败` | 本轮组合求值失败，不写新值。 |
| `视为未命中` | 该输入参与计算，但判定为未命中。 |
| `忽略该项` | 该输入不参与本轮计算。 |
| `使用默认值` | 使用输入项的 `默认值` 继续计算。 |

组合结果仍必须回到抽象特征自己的区间中解释。例如 `任务是否可执行` 的组合输出为 `1` 后，应命中其子抽象特征 `可执行 [1,1]`。

## 11. 主信息类型拆分

现有 `特征节点主信息类` 同时承担了定义层和实例层的部分职责。新规范要求逐步拆分为两个主信息类型：

```text
抽象特征主信息类
实例特征主信息类
```

### 11.1 抽象特征主信息类

抽象特征主信息类用于世界树根下的抽象特征细化链。

最小字段：

```cpp
class 抽象特征主信息类 : public 基础信息基类 {
public:
    std::optional<I64区间> 区间{};
    std::vector<结构_抽象特征组合规则> 组合规则{};

    枚举_特征生命周期 生命周期 = 枚举_特征生命周期::候选;

    std::uint64_t 重叠命中次数 = 0;
    时间戳 最近重叠命中时间 = 0;

    可解析引用<基础信息节点类> 来源实例特征{};
    时间戳 升格时间 = 0;
};
```

说明：

1. 没有 `当前值`。
2. 没有 `稳态列表`。
3. 子抽象特征通过基础信息树细化链表达。
4. 区间可以为空。
5. 组合规则表达来源依赖，不替代细化链。
6. 重叠统计可先采用最小计数字段，后续再升格为更完整的重叠事件节点。

### 11.2 实例特征主信息类

实例特征主信息类用于存在、场景或虚拟存在下面的实例特征。

最小字段：

```cpp
class 实例特征主信息类 : public 基础信息基类 {
public:
    可解析引用<基础信息节点类> 抽象特征{};
    可解析引用<基础信息节点类> 当前命中抽象特征{};

    特征值 当前值{};

    static constexpr std::size_t 候选窗口容量 = 60;
    环形缓冲<特征值, 候选窗口容量> 近帧候选{};
    环形缓冲<时间戳, 候选窗口容量> 近帧时间{};
    std::vector<特征稳态> 稳态列表{};
    int 当前稳态索引 = -1;

    std::optional<I64区间> 局部区间覆盖{};
    I64区间 连续测量误差区间{ 0, -1 };
};
```

说明：

1. 有 `当前值`。
2. 有 `稳态列表`。
3. 引用抽象特征。
4. 可缓存当前命中的抽象特征。
5. 可保存局部区间覆盖。

### 11.3 旧主信息兼容

迁移期内：

```text
旧 特征节点主信息类 = 当前实现中的实例特征兼容壳。
```

未来应逐步收敛为：

```text
特征主信息基类
  -> 抽象特征主信息类
  -> 实例特征主信息类
```

写当前值的接口只能接受实例特征。抽象特征不得进入写当前值流程。

## 12. 与现有特征类实现的关系

当前 `特征类` 已具备以下可复用能力：

1. 在基础信息树中创建、查找、删除特征节点。
2. 写入和读取当前值。
3. 维护近帧候选、稳态列表和当前稳态索引。
4. 比较特征值和特征集合。
5. 生成度量签名。

新规范对现有实现的调整方向：

1. 新增抽象特征创建和查找入口。
2. 实例特征创建必须引用抽象特征。
3. `创建特征_按类型` 应逐步降级为兼容入口。
4. `取或创建子特征_按类型` 应逐步升级为 `取或创建实例特征_按抽象特征`。
5. 写入实例特征当前值后，应执行抽象区间命中。
6. 稳态变化后，应允许触发抽象区间升格、拆分、合并或局部覆盖。
7. 比较特征集合时，特征类型相等应以抽象特征引用或稳定主键为准，不应只按裸词性指针建表。
8. 高阶抽象特征应通过组合规则求值，不应把组合来源挂成子抽象特征。

## 13. 与世界树的关系

世界树应提供统一入口：

```text
取或创建抽象特征(...)
取或创建实例特征(...)
写入实例特征值(...)
读取实例特征值(...)
解析实例特征命中抽象特征(...)
升格实例稳态为抽象区间(...)
求值组合抽象特征(...)
```

世界树根下应有抽象特征细化链，作为全部抽象特征定义的权威挂载位置。

存在、场景、任务虚拟存在、方法虚拟存在等宿主只挂实例特征，不挂抽象特征定义。

## 14. 最小迁移顺序

建议按以下顺序落地：

1. 新增 `抽象特征主信息类`。
2. 明确旧 `特征节点主信息类` 作为实例特征兼容壳。
3. 在世界树根下建立抽象特征细化链。
4. 新增抽象特征取或创建接口。
5. 新增实例特征按抽象特征创建接口。
6. 写入实例特征后执行区间命中。
7. 实例稳态可升格为抽象区间。
8. 支持区间重叠统计。
9. 支持局部覆盖、区间拆分与合并。
10. 新增组合规则数据结构。
11. 支持组合抽象特征的实例层求值。
12. 将任务状态、方法状态、需求状态等语义结构体逐步迁移为抽象特征树。

## 15. 验收口径

本规范的 V1 验收至少包括：

1. 能在世界树根下创建抽象特征，且抽象特征无当前值。
2. 能将抽象特征实例化到某个存在或虚拟存在下。
3. 实例特征能写入当前值。
4. 当前值能命中抽象特征区间。
5. 点区间能表达枚举和布尔状态。
6. 实例特征形成稳态后，能补全或修正抽象特征区间。
7. 同级抽象区间重叠时，能记录重叠次数或频率。
8. 细化实例特征采用同层懒创建。
9. 旧 `特征节点主信息类` 不再被解释为抽象特征本体。
10. 有语义的结构体字段有明确迁移到抽象子特征的路径。
11. 能保存组合规则，并能区分组合来源和子抽象特征。
12. 能用 `全部命中` 组合方式生成一个布尔型组合实例特征值。

## 16. 一句话收口

特征系统的新本体是：

```text
世界树根下的抽象特征定义
  + 宿主下的实例特征当前值
  + 实例稳态升格形成的抽象区间
  + 抽象特征细化链替代语义结构体
  + 组合规则表达高阶抽象特征来源依赖
```

抽象特征负责定义，实例特征负责观测；稳态属于实例，区间属于抽象；结构体语义最终应迁移为抽象特征细化链；高阶组合关系进入组合规则，不进入细化链。
