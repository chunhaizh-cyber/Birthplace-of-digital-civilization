# 0576 动态二次特征统一生成规范

## 1. 目标

本文用于把 `动态二次特征` 的统一母式、统一入口、统一输出和统一异常处理单独固定下来，重点回答以下问题：

- 动态二次特征的 `母式` 到底是什么
- 为什么母式应是一套 `万能的生成二次特征函数`
- 哪些动态输入可以进入正式生成
- 哪些结果属于正式动态二次特征，哪些结果只能落到派生指标
- 可比性、时间尺度、路径层级、阈值过滤、可信度继承应如何统一收口

本文只讨论 `严格2次特征` 中的 `动态来源生成规则`，不单独讨论 `2次以后` 的递推规则，也不把除法类派生指标写回正式动态二次特征本体。

## 3. 核心结论

必须固定以下 13 条：

1. `动态二次特征的母式` 不是一堆散开的专用算法，而是一套 `对所有动态统一适用的生成函数`。
2. 这套统一函数的正式任务，是从 `可比的时间序列动态输入` 中，生成适用的动态二次特征集合。
3. 动态二次特征的正式真身，仍必须建立在 `可比较值对的差分` 之上。
4. 时间在正式动态二次特征里承担 `排序 / 对齐 / 分段 / 持续时间 / 时间尺度绑定` 作用，不承担除法分母作用。
5. `变化率 / 速率 / 差值除以时间` 这类结果若确有业务价值，只能落到 `派生指标`，不得写成正式动态二次特征。
6. 动态二次特征必须先分清 `原始计算层` 与 `分类映射层`，不得一上来只保留标签词。
7. 差值至少应拆成 `带符号变化量 / 方向 / 绝对变化幅度` 三层，而不是揉成一个匿名值。
8. 动态二次特征必须同时支持 `分步特征`、`整体特征` 与 `路径特征`，三者不得互相替代。
9. 当节点数大于 `2` 时，`路径特征优先于首尾特征`；首尾只描述总体结果，不代表过程形态。
10. `可比性判定`、`阈值过滤`、`可信度继承`、`异常时间处理` 必须进入统一函数本体，而不是交给调用方各自发挥。
11. 不是每条动态都要生成全部动态二次特征；统一函数只输出 `当前输入真正适用的子集`。
12. 对当前阶段而言，`除法` 还属于更高层派生概念，因此正式动态二次特征母式先不引入它。
13. 因此，动态二次特征的正式一句话收口应为：

`动态二次特征 = 对同一主体、同一特征类型、在可比时间序列上的差分、分段差分、路径抽象与分类映射。`

补充说明：

- 若涉及多个特征类型，必须先分别对各自特征做内部差分，再组织跨特征关系。
- 若某结果必须依赖除法、乘法、加权或归一化，统一视为 `动态派生指标`，不再继续叫正式动态二次特征。


## 4. 统一输入前提

### 4.1 同型可比规则

动态二次特征不得对任意两个字段直接硬算，必须先满足可比性。

至少应检查：

- `主体相同`
- `特征类型相同`
- `度量单位相同`
- `值域语义一致`
- `比较基准一致`
- `语义层级一致`

必须固定：

1. 先判定可比，再计算变化。
2. 不可比输入不得直接生成正式动态二次特征。
3. 枚举型、序数型、概念型输入若没有显式序关系，不得伪装成可减标量。

### 4.2 序列抽取与标准化规则

统一函数接收的正式输入，应先被标准化为：

```text
动态输入 = {(t0, v0), (t1, v1), ... , (tn, vn)}
```

其中必须明确：

- 节点时间顺序
- 节点来源
- 节点值口径
- 是否存在插值
- 是否存在异常时间戳

必须固定：

1. 母式函数不直接对原始脏输入计算。
2. 必须先排序、清洗、标准化，再进入正式生成。


## 5. 正式输出分层

### 5.1 原始计算层与分类层分离

动态二次特征的正式输出至少分三层：

1. `原始计算层`
   - 带符号变化量
   - 方向
   - 绝对变化幅度
   - 持续时长
   - 分步数量
   - 转折数量
   - 峰谷位置
   - 回返类型
   - 闭环类型
2. `区间映射层`
   - 微小 / 小 / 中 / 大
   - 平稳 / 波动 / 剧烈波动
   - 持续短 / 持续中 / 持续长
3. `概念命名层`
   - 小幅上升
   - 缓慢回稳
   - 剧烈波动
   - 回返起点附近

必须固定：

1. 先算数，再贴标签。
2. 阈值变化优先重算 `区间映射层 / 概念命名层`，不应破坏原始计算层真值。

### 5.2 差值分拆规则

变化量至少应拆成：

- `带符号变化量`
- `方向`
- `绝对变化幅度`

必须固定：

1. 方向不能替代带符号变化量。
2. 绝对变化幅度不能被方向字段隐式承载。
3. 需要只看方向、只看幅度、或同时看两者的下游，不应再重复拆值。


## 6. 统一母式函数

### 6.1 正式母式

推荐把正式母式固定理解为：

```text
生成动态二次特征(主体 O, 特征类型 F, 时间序列 S, 配置 C)
```

其正式处理主链固定为：

```text
输入动态
-> 可比性判定
-> 序列抽取与标准化
-> 异常时间处理
-> 生成相邻分步差分
-> 生成首尾整体差分
-> 生成路径抽象
-> 阈值过滤与稳定带处理
-> 置信度继承
-> 区间映射
-> 概念命名
-> 输出适用子集
```

### 6.2 推荐伪代码

```cpp
生成动态二次特征(主体 O, 特征类型 F, 序列 S, 配置 C):
    if !可比(O, F, S):
        return {}

    序列 = 标准化并排序(S)
    序列 = 处理异常时间戳(序列, C)
    if 序列.无效():
        return {}

    结果集 = {}
    分步集 = 生成相邻分步(序列)

    for 分步 in 分步集:
        Δv = 分步.后值 - 分步.前值
        dir = 方向(Δv)
        absΔ = 绝对值(Δv)
        Δt = 分步.后时刻 - 分步.前时刻
        if !通过阈值(Δv, Δt, C):
            continue
        结果集 += 写入分步结果(Δv, dir, absΔ, Δt)

    总Δv = 序列.末值 - 序列.首值
    总dir = 方向(总Δv)
    总absΔ = 绝对值(总Δv)
    总Δt = 序列.末时刻 - 序列.首时刻
    结果集 += 写入整体结果(总Δv, 总dir, 总absΔ, 总Δt)

    路径结果 = 抽象路径形态(序列, 分步集, C)
    结果集 += 路径结果

    结果集 = 继承并下调置信度(结果集, 序列, C)
    结果集 = 映射区间与概念(结果集, C)
    return 裁剪不适用项(结果集)
```

必须固定：

1. 统一函数是正式入口。
2. 输出必须是 `适用子集`，而不是固定全量套餐。
3. `速率 / 变化率 / 差值除以时间` 不得混入这条正式二次特征主链。


## 7. 时间规则

### 7.1 时间尺度显式绑定

虽然正式动态二次特征不得做 `差值 ÷ 时间`，但时间尺度仍必须显式绑定。

至少应保存：

- `分步时长`
- `总时长`
- `窗口时长`
- `时间粒度`

必须固定：

1. 时间尺度是解释动态过程的上下文，不是正式二次特征里的除法分母。
2. 同样的差值，若时间尺度不同，其语义标签和阈值判定可以不同。

### 7.2 不等时间间隔规则

当序列的相邻间隔不相等时，必须固定：

1. 每段时长逐段独立保存。
2. 不允许用节点数替代真实时间。
3. 路径形态判断应基于真实时间顺序，不应假设等间隔采样。

### 7.3 零时长与重复时间戳规则

必须固定：

1. `Δt = 0` 的相邻节点，不得直接生成正式持续类分步特征。
2. 时间倒序节点不得参与正常动态计算。
3. 重复时间戳必须按预设策略处理：
   - 合并
   - 覆盖
   - 丢弃
   - 标记异常后阻断正式生成
4. 处理策略必须固定，不能由调用点临时决定。


## 8. 分步、整体与路径规则

### 8.1 分步特征与整体特征并存

当节点数大于 `2` 时，必须同时生成：

- `分步特征`
- `整体特征`

必须固定：

1. 分步特征保留过程细节。
2. 整体特征保留总体结果。
3. 两者并存，不能二选一。

### 8.2 路径优先于首尾

必须固定：

1. 当节点数大于 `2` 时，`路径特征优先于首尾特征`。
2. 首尾差分只描述总体变化，不代表中间过程。
3. 任何剧烈波动、先升后降、先降后升、回返起点的过程，不得被首尾差分抹掉。

### 8.3 过程形状特征

对完整路径，推荐至少支持：

- `单调段数量`
- `转折点数量`
- `首次峰值位置`
- `首次谷值位置`
- `回返类型`
- `闭环类型`

必须固定：

1. 这类特征属于 `过程形状二次特征`。
2. 它们不是首尾结果的别名，而是路径事实的抽象。


## 9. 参照规则

动态二次特征不应只支持“和起点比”，推荐至少支持以下基准：

- `相邻基准`
- `起始基准`
- `稳态基准`
- `目标基准`

必须固定：

1. 相邻基准回答 `是否继续恶化 / 是否继续改善`。
2. 起始基准回答 `相对起点整体变了多少`。
3. 稳态基准回答 `是否回到稳定带附近`。
4. 目标基准回答 `是否达标 / 仍有多少偏离`。


## 10. 阈值、稳定带、置信度与异常处理

### 10.1 阈值过滤与稳定带

必须显式定义：

- `最小可感知变化量`
- `最小有效时长`
- `连续稳定区间`

必须固定：

1. 只有超过阈值的变化，才生成正式动态二次特征。
2. 未过阈值的结果，可记为 `微扰 / 稳定带内波动`，但不得冒充正式变化。

### 10.2 置信度继承规则

必须固定：

1. 动态二次特征的置信度不得高于其上游输入真值。
2. 分步结果的置信度应受两端状态节点约束。
3. 路径结果的置信度应受整条路径的最低可信段约束。
4. 异常时间、重复时间戳、插值节点、缺失修复节点应下调置信度。

### 10.3 缺失值与插值规则

必须固定：

1. 缺节点时是否允许插值，必须由统一配置决定。
2. 插值结果若允许参与生成，也必须显式降权。
3. 连续缺失超过阈值时，应阻断正式路径特征生成。


## 11. 统一输出与链签名规则

### 11.1 输出并存规则

每个动态序列至少应支持两套结果并存：

1. `原子结果`
   - 每段的带符号变化量
   - 每段方向
   - 每段绝对变化幅度
   - 每段持续时长
2. `聚合结果`
   - 首尾整体变化量
   - 首尾整体方向
   - 首尾整体绝对变化幅度
   - 总时长
   - 路径形状摘要

### 11.2 统一链签名建议

建议 `度量签名_链键` 至少由以下显式字段自动生成：

- `主体类型`
- `特征类型`
- `基准类型`
- `时间尺度`
- `粒度`
- `路径层级`
- `单位`
- `规则版本`

必须固定：

1. 链签名应由显式字段推导，不应再手写成黑箱字符串。
2. `相邻分步结果` 与 `整体路径结果` 不得共用同一签名。


## 12. 除法结果的去向

若业务确实还需要：

- `变化率`
- `平均速率`
- `局部速率`
- `速率波动度`

则统一按以下口径处理：

1. 它们可以存在，但不属于正式动态二次特征。
2. 它们应单独落到 `动态派生指标` 或更高层辅助结构。
3. 它们不得回写成正式 `2次特征`，也不得伪装成 `3次 / n次特征`。
4. 在当前阶段，调用方不应把这类除法结果当作动态二次特征母式的一部分。


## 13. 最终收口

`动态二次特征的母式，应被固定为一套对所有动态统一适用的万能生成函数；它先做可比性判定与序列标准化，再统一生成分步、整体与路径结果，并在阈值、稳定带、置信度和异常处理的约束下，输出原始计算层、区间映射层与概念命名层的适用子集；凡涉及除法的速率类结果，都只属于动态派生指标，不属于正式动态二次特征本体。`
