# 0575 二次特征泛称与n次特征递推生成规范

## 1. 目标

本文用于把 `0阶 -> 1阶 -> 2阶 -> 3阶 -> ... -> n阶` 的特征递推正式固定下来，回答以下问题：

- `二次特征` 到底是严格二阶，还是高阶特征的泛称
- 原始经验、直接特征、二次特征、三次特征之间怎样分层
- 高阶特征是否只能来自“比较与相减”
- 为什么 `从缺口反推特征` 不再是总模型

说明：

- 本文中的代码块或流程块都只表示语义，不表示现有实现中的直接接口或函数签名。

## 3. 核心定义

必须固定：

1. 工程口语里的 `二次特征`，可以继续作为 `高阶特征族` 的泛称。
2. 严格落文时，必须尽量写清楚具体阶次。
3. 高阶特征不再被限定为“比较后相减再排序”的唯一形式。
4. `比较型二次特征` 只是高阶特征族中的一个重要子族，不是全部定义。

### 3.1 阶次

正式阶次固定为：

```text
0阶 = 原始经验信号
1阶 = 直接特征
2阶 = 二次特征
3阶 = 三次特征
n阶 = 更高阶治理特征
```

### 3.2 0阶：原始经验信号

0阶不是正式治理特征，只是原始材料。

至少包括：

- 用户消息
- 任务状态变化
- 方法调用结果
- 失败日志
- 学习反馈
- 代码变化
- 计划变更历史

### 3.3 1阶：直接特征

1阶特征直接从原始材料提取。

例如：

- `task_has_acceptance_criteria`
- `task_has_external_dependency`
- `method_exists_for_task_type`
- `task_has_learning_signal`

### 3.4 2阶：二次特征

2阶特征由低阶材料进一步组织出来，开始承担 `治理解释` 职责。

它可以来自：

- 比较
- 聚合
- 关系压缩
- 反复共现
- 稳定模式发现
- 权重组合

例如：

- `task_learning_gap_pressure`
- `method_reuse_confidence`
- `goal_visibility`
- `dependency_readiness`

### 3.5 3阶及以上

3阶及以上特征继续由低阶特征组合，开始承担：

- 策略判断
- 成长价值判断
- 系统治理态判断

例如：

- `task_self_growth_priority`
- `method_future_reuse_value`
- `system_governance_stagnation_index`


## 4. 正式递推链

后续统一按以下递推链理解：

```text
Raw Experience
-> Direct Feature
-> Second-Order Feature
-> Third-Order Feature
-> N-Order Governance Feature
```

必须固定：

1. 原始经验先于直接特征。
2. 直接特征先于二次特征。
3. 二次及更高特征可以继续显现新的可见缺口与可见可能。
4. 需求形成晚于高阶特征显现，不得倒置。


## 5. 高阶特征的合法生成方式

高阶特征至少允许以下正式生成方式：

1. 比较生成  
   例如：目标值与当前值比较、前后状态比较、左右对象比较。

2. 聚合生成  
   例如：多个一阶特征聚合成 `learning_gap_pressure`。

3. 关系生成  
   例如：由主体-客体、前置-后果、依赖-就绪关系压缩出来。

4. 模式生成  
   例如：失败簇、复用簇、回退簇、等待簇。

5. 权重组合生成  
   例如：由多项低阶特征经稳定权重组合形成治理解释。

必须固定：

1. `比较` 是重要生成法，但不是唯一生成法。
2. `排序` 是高阶特征的一种合法子类，但不是全部三次特征或 n 次特征的定义。
3. 只要某种区分是稳定、可复用、可命名、可回指证据的，它就可以形成更高阶特征。


## 6. 特征派生图与使用边界

高阶特征在结构上应理解为一张有向无环派生图。

最小语义固定为：

```text
0阶原始信号
-> 1阶直接特征
-> 2阶二次特征
-> 3阶三次特征
-> n阶高阶治理特征
```

必须固定：

1. 高阶特征的 `父特征` 只要仍被下游高阶特征正式依赖，就算被使用。
2. 因此高阶特征父节点不能因为“没有直接选步骤”就被误判成无用。
3. 高阶特征图应优先理解成 `派生图 + 使用边` 的组合，而不是一堆孤立字段。

## 9. 废止口径

以下旧口径自本文生效后降级：

1. `1次 -> 2次` 是全部高阶特征的唯一入口。
2. `3次特征` 只能解释为“同需求轴同向候选排序”。
3. `从缺口反推特征` 是新特征的总模型。

它们若继续保留，只能作为：

- 比较型子族说明
- 某条局部实现路径
- 历史兼容描述

## 10. 最终收口

`二次特征` 在工程里可以继续作为高阶特征族泛称；严格分层时，0阶是原始经验信号，1阶是直接特征，2阶是治理解释开始出现的第二层稳定区分，3阶及以上继续承担策略、成长与系统治理判断；比较、聚合、关系、模式和权重组合都可以成为合法生成方式。
