# 0870 统一控制规则（安全服务双值核与自组织）

## 1. 目标

本文保留根层控制语义，但任务主流程统一改按：

`目标 -> 筹办 -> 执行`


## 2. 核心结论

必须固定：

1. 根层方向仍由安全值与服务值约束。
2. 任务层主流程不再使用 `四域` 作为统一主逻辑。
3. 根层负责决定是否允许继续推进。
4. 一旦允许继续推进，任务侧统一进入 `目标 -> 筹办 -> 执行`。


## 3. 任务侧统一控制口径

当前工程承接层只保留三段：

1. `目标`
2. `筹办`
3. `执行`

其中：

- `目标` 回答想改变谁、改什么、改到什么程度
- `筹办` 负责完善任务树
- `执行` 负责消费当前步骤并写结果


## 4. 对旧四域口径的处理

旧文中的：

- `筹办`
- `维护`
- `纠偏`
- `收束`

不再作为任务统一主流程保留。

若后续仍出现这些词，只允许退为：

- 步骤语义
- 结果分流标签
- 日志观察标签
