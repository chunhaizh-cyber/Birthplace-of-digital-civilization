# 1295 动作动态因果与反推-组合方法升格规范

## 1. 目标

本文用于固定以下问题：

- `自我的动作动态信息` 在因果与方法体系里应被放在哪一层
- `从外界获取的因果信息` 应如何进入学习与方法出生链
- `因果链筛选逻辑` 在当前体系里的主要职责是什么，不是什么
- `稳定任务树` 何时允许抽象为 `组合方法`
- `组合方法` 的主要价值是什么，不能被误写成什么

本文要收口的是：

- `动作动态因果`
- `外界因果`
- `反推候选方法`
- `组合方法升格`

这四者之间的分层与升格边界，而不是某一个具体模块的局部实现细节。

### 2.1 在统一主链中的位置

`根层方向裁决`

-> `沉淀动作动态与因果证据`

-> `运行期验证与结构复现`
-> `外层重判后再决定是否继续升格`

必须固定：

2. 动作动态因果、外界因果、反推候选和组合方法升格，全部只能作为外层继续/停止/收束判断的证据与候选来源，不得越权替代根层裁决。
3. 当这条链服务于 `任务管理方法` 分支时，仍须先在外层显式化 `最近根层裁决结果`、`当前缺口类型`、`当前下一步去向`。


## 3. 核心结论

必须固定以下 15 条：

1. `自我的动作动态信息` 可以视为最小单位的高可信内部因果证据。
2. `从外界获取的因果信息` 默认属于待验证外部证据，不得直接当成已验证方法。
3. `自我动作动态` 与 `外界因果` 属于 `因果证据来源层`，不是正式 `方法来源` 标签。
4. `方法头是否出生` 与 `方法是否可信/是否成熟` 必须分开判定。
5. 多次同桶的 `自我动作动态` 可以直接支持低成熟度候选方法头出生，但不得仅凭动作记账次数直接宣称成熟。
6. 运行时新方法的首次出生，更适合先落到低成熟度阶段，如 `运行期临时` 或 `反推候选`。
7. `因果链` 应降级为“生成方法树时按需展开的筛选/倒推轨迹”，默认先做单步倒推，不再作为常驻核心知识层单独维护。
8. `稳定复现` 裁判的是“相似条件下方法结构是否反复得到同向结果”，不是“某个动作动态是否重复出现过几次”。
9. 某个需要反复执行的任务，其任务路径或任务子树在相似条件下长期稳定后，才允许继续抽象为 `组合成熟` 方法。
10. 任一高层成熟方法最终都必须能分解回既有元函数、既有本能函数或已沉淀的方法片段链。
11. `反推候选`、`运行期临时`、`已验证稳定` 与 `组合成熟` 的任何升格，都只能发生在根层仍允许继续推进的前提下。
12. 任一候选方法或组合方法在当前治理轮被消费后，若它将影响继续/停止/收束路径，仍必须回到外层重新做 `根层重判 -> 上下文刷新`。
13. 所有由 `自我动作动态信息` 生成的因果，在进入正式学习前，都允许先进入 `方法首节点出生判定`；若不存在适配旧方法，则可直接出生低成熟度方法首节点；学习只负责后续补条件、补结果、做验证与升格。
14. 若已存在适配旧方法树能够承接同批 `自我动作动态因果`，则应优先回补旧方法树，而不是强制并行出生平行新方法树。
15. 对 `自我动作动态因果` 支持的可执行方法树，若 `动作不同` 或 `条件组合不同`，则应独立生成另一棵方法树；不得为减少树数量而强制并树。

可收敛为一句话：

`动作动态和外界因果先作为证据进入方法头出生与学习链；其中自我动作动态因果允许先直接触发低成熟度方法树出生；学习不负责方法头出生，只负责后续补全和验证；真正的成熟与升格仍要靠稳定复现统一裁判。`


## 4. 四层分工

必须先区分四层：

### 4.1 因果证据来源层

回答的是：

- 这条因果原料从哪里来
- 当前可信度大概处于什么层级

当前最少包括：

- `内部先天动作动态`
- `自我内部动作动态`
- `外界因果`
- `外部反馈/回执/观察补证`

### 4.2 因果筛选与倒推层

回答的是：

- 在生成方法树时，哪些单跳因果值得优先拿来倒推
- 当单步不够时，怎样按需补出多步桥接筛选轨迹

当前最少包括：

- `单跳实例因果`
- `单步倒推筛选`
- `按需多步桥接筛选轨迹`

### 4.3 方法出生层

回答的是：

- 这些证据第一次沉淀时，方法头应落成什么初始成熟度阶段

当前最少包括：

- `运行期临时`
- `反推候选`

### 4.4 方法升格层

回答的是：

- 哪些已经出生的方法，何时可以继续升格成成熟结构

当前最少包括：

- `运行期临时 -> 反推候选`
- `运行期临时 / 反推候选 -> 已验证稳定`
- `已验证稳定 -> 组合成熟`

### 4.5 与任务管理治理分支的边界

必须固定：

1. 本文描述的是全域“证据到方法升格”的共性链路。
2. 若服务对象是 `任务管理方法` 分支，则必须额外携带：
- `最近根层裁决结果`
- `当前缺口类型`
- `当前下一步去向`
3. 这三项属于治理上下文，不属于所有方法来源统一必填字段。


## 5. 动作动态因果与外界因果的可信度分层

### 5.1 自我动作动态

`自我动作动态` 的口径固定为：

- 自我内部一次明确发生过的动作事件
- 具备较清晰的开始、结束和局部结果边界

因此必须固定：

1. 动作事实和动作边界可以直接视为可信动作证据。
2. 若结果是该动作直接写回的内部结果，可视为高可信果侧证据。
3. 若在相似条件桶下，多次出现同向动作-结果配对，可直接支持：
   - `运行期临时` 方法头出生
   - `反推候选` 方法头出生
   - 已有适配方法的局部模板补齐
4. 这类证据更适合形成：
   - `运行期临时方法`
   - `局部方法补齐`
   - `反推候选的强证据起点`
5. 若当前因果已经足以给出 `身份锚点 + 主作用表达 + 最小结构承载`，则不必先停留在裸原料层；可直接经方法注册链出生正式低成熟度方法首节点，再把该新头交给学习链继续补条件、补结果和做验证。

### 5.2 外界因果

`外界因果` 的口径固定为：

- 来自外部世界、外部设备、外部回执、外部反馈或外部观测的因果原料

因此必须固定：

1. 直接可信的只是“外界给出了某条信息或某次反馈”。
2. 外界结果是否成立，仍需依赖：
   - `观测`
   - `回执`
   - `结果确认`
   - `重复出现`
3. 外界因果在未完成验证前，不得直接产出高成熟度方法。

### 5.3 二者与正式方法来源的边界

必须固定：

- `自我动作动态` 和 `外界因果` 都不是正式方法来源标签
- 它们只是支撑方法出生和升格的证据来源
- 正式方法侧只记录来源与成熟度结果，而不直接记录原料出处
- 正式方法来源的精确分类，优先按 [0569_方法来源边界与正式来源分类规范.md](D:\鱼巢\规范\0569_方法来源边界与正式来源分类规范.md) 落成 `因果抽象 / 观察抽象 / 路径提炼`

### 5.4 动作动态复现与稳定复现的边界

必须固定：

1. `动作动态复现` 回答的是：同类动作证据是否重复出现。
2. `稳定复现` 回答的是：同一方法结构在相似条件下，是否反复得到同向、同边界、同结果桶的成功效果。
3. `动作动态复现` 可以直接支持候选方法头出生或候选成熟度上升。
4. `动作动态复现` 不自动等于 `稳定复现`；只有补齐条件归桶、结果确认和边界一致性后，才允许计入稳定复现裁判。
5. 如果已有适配方法树能承接这些证据，应优先回补旧方法树，而不是机械并行生成平行新树。
6. 这里“是否已有适配旧方法可以承接”的最小唯一性判定，优先按 `动作键 + 条件主键 + 主结果特征类型 + 主结果方向或主结果语义` 做。
7. 因此，`自我动作动态因果 -> 方法首节点出生` 与 `方法首节点出生后再进入学习补全/验证` 是允许且推荐的最小承接顺序；但这个顺序的主裁判仍是“是否已有适配旧方法树可以承接”，而不是学习线程。


## 6. 因果链筛选逻辑的职责

`因果链` 在当前口径下应理解为：

- 生成方法树时按需展开的筛选/倒推轨迹
- 不是常驻核心知识层
- 不是高成熟度方法的直接等价物

其职责应固定为：

1. 当单跳实例因果不足以直接生出方法头时，补出一条可解释的倒推筛选轨迹。
2. 为学习线程提供 `反推候选方法` 的中间桥接原料。
3. 帮助找出复杂方法树里“不好直接从一次成功片段看出来”的中间步骤或前置条件。
4. 服务较复杂的方法树候选生成，而不是直接替代方法树。

因此必须固定：

- 默认优先 `单步倒推`
- 只有单步不够时，才继续展开 `按需多步桥接筛选轨迹`
- `因果链` 偏“当前这次怎样筛、怎样倒推”
- `方法树` 偏“最终怎样执行和复用”

更具体地说：

- `因果链筛选逻辑` 适合补出：
  - 需要补的中间步骤
  - 可能存在的前置条件
  - 可能存在的结果桥接
- `因果链筛选逻辑` 不适合直接产出：
  - 高成熟度方法
  - 直接可执行的正式组合枝

必须禁止：

- 把一条外界因果链直接抬成成熟 `组合方法`
- 把“因果链很长”误写成“方法已经稳定”
- 把因果链筛选轨迹误当成常驻成熟知识对象


## 7. 从反推到复杂方法树

当前应固定的最小升格链是：

```text
自我动作动态 / 外界因果
-> 单跳因果证据
-> 方法首节点出生判定
-> 新生低成熟度方法头 或 回补适配旧方法
-> 单步倒推筛选
-> （必要时）按需多步桥接筛选轨迹
-> 运行期验证与补齐
-> 稳定方法片段
```

说明：

- `反推` 主要解决“复杂方法树怎么长出来”
- 它更适合先长出候选枝、桥接枝、补缺枝
- 它不是成熟 `组合方法` 的直接同义词

因此必须固定：

1. `反推` 偏证据推导。
2. `组合` 偏结构稳定后的收敛与压缩。
3. `复杂方法树` 的首次出生可以主要靠 `反推`，而 `因果链` 只是此时按需展开的筛选轨迹。
4. `复杂方法树` 的长期稳定与简化，则更多依赖后续结构复现与组合升格。
5. `反推` 与 `组合升格` 产出的候选结构，在当前治理轮被实际采用后，仍必须返回外层重新做继续/停止判断。


## 8. 稳定任务树到组合方法的升格

当一个需要反复执行的任务，其树形结构比较稳定后，允许继续抽象为 `组合方法`。

但必须同时满足以下条件：

1. 该任务路径或任务子树在相似条件下反复出现。
2. 该结构能脱离当前宿主任务，抽出可复用的通用步骤骨架。
3. 该结构的每个关键动作都能分解回既有本能、既有元函数或已沉淀方法片段。
4. 该结构不是纯粹依赖一次偶然成功或一次外部回执拼出来的临时链。
5. 抽象后确实能减少筹办、搜索、回忆和推演成本。

### 8.1 组合方法的主要价值

`组合方法` 的主要价值固定为：

- `简化思考过程`

具体表现为：

- 减少重复的显式筹办
- 压缩多步显式搜索
- 把反复出现的结构折叠成稳定方法块
- 让任务管理与学习调度更快命中高层复用结构

### 8.2 组合方法不是什么

必须禁止把 `组合方法` 理解成：

- 新增底层原语
- 无需再解释的黑箱
- 只因为路径很长就直接升格的“大方法”


## 9. 默认落账路径

建议后续统一按以下路径落账：

| 原料主导类型 | 更适合的首次出生阶段 | 后续主要成熟方向 |
| --- | --- | --- |
| `自我动作动态` | `运行期临时` 或 `反推候选` | `已验证稳定 -> 组合成熟` |
| `外界因果 + 已验证补证` | `反推候选` 或 `运行期临时` | `已验证稳定 -> 组合成熟` |
| `按需多步桥接筛选轨迹` | `反推候选` | `已验证稳定 -> 组合成熟` |
| `稳定任务路径 / 稳定任务子树` | `运行期临时` | `已验证稳定 -> 组合成熟` |

必须固定：

- `反推` 是复杂候选树的优先出生位
- `组合` 是稳定结构的优先收敛位
- `组合` 不应替代 `反推` 的出生职责
- 默认顺序仍应是 `证据采集 -> 候选出生 -> 运行期验证 -> 外层重判 -> 再升格`


## 10. 工程对齐结论

按现有体系收口，当前最自然的工程分工应固定为：

1. `动作动态记录层`
- 负责沉淀自我动作动态
- 作为最小高可信内部因果原料

2. `因果信息模块`
- 负责沉淀和检索因果证据
- 负责外界因果与内部因果的统一候选入口

3. `学习/反推链路`
- 负责把单跳因果和按需倒推筛选轨迹提炼成 `反推候选方法`

4. `任务结构稳定化链路`
- 负责把长期稳定的任务路径或任务子树继续抽象为 `组合成熟` 方法

5. `方法来源展示层`
- 只展示正式方法来源标签
- 不把“动作动态”或“外界因果”误展示成来源标签
 - 低成熟度出生与后续可信度信息，工程上优先落在既有 `方法虚拟存在` 特征类型，不另造平行结构


## 11. 硬约束

1. `动作动态因果` 是证据来源，不是正式方法来源标签。
2. 多次 `动作动态复现` 可以支持低成熟度方法头出生，但不得直接等价为 `稳定复现`。
3. `外界因果` 在未验证前，不得直接产出成熟方法。
4. `因果链` 的主要职责是服务生成方法树时的筛选/倒推，不得直接越权成为成熟组合方法。
5. `组合成熟` 只能来自稳定结构抽象，不得由一次成功或一条长因果链直接跳成。
6. `组合成熟` 的核心目标是简化思考与复用结构，不是引入新的底层原语。
7. 若服务于任务管理治理链，外层仍必须先具备 `最近根层裁决结果 / 当前缺口类型 / 当前下一步去向`，不得让方法升格链越权冒充治理上下文。
8. 若需要把证据侧出生结果写成正式方法来源，应优先写成 `因果抽象 / 观察抽象 / 路径提炼` 这样的正式来源值，不得把 `自我动作动态 / 外界因果` 直接抬成来源标签。


## 12. 一句话收口

`自我动作动态给出最小高可信内部因果，外界因果提供待验证外部证据；这些证据可以推动低成熟度方法头出生，而因果链只在生成方法树时按需充当筛选/倒推轨迹，最终仍只有稳定复现才能把候选裁判成可信成熟方法。`
