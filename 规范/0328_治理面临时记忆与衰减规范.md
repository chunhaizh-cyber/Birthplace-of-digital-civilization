# 0328 治理面临时记忆与衰减规范

## 1. 目标

本文用于把治理面的“临时记忆”性质和最小衰减规则单独固定下来。

本文要回答的是：

- 为什么治理面应视为临时记忆
- `学习压力 / 重试压力 / 风险记忆 / 普通执行冻结候选 / 边界处理优先建议` 怎样衰减
- 哪些治理面只应在条件变化后重新激活

## 3. 核心结论

必须固定以下结论：

1. `治理面` 默认属于 `临时记忆`，不是永久真值。
2. `治理面` 应随着时间、使用情况和条件变化而衰减、冻结、解除或清空。
3. `学习压力` 当前应优先理解为 `待学习方法数量` 或等价待补方法数量。
4. `重试压力` 不是一直有效；只有在条件发生变化或重新符合后，才应重新激活。
5. `风险记忆` 当前应按 `时间流逝 + 后续重复验证次数` 逐步衰减。
6. `普通执行冻结候选` 与 `边界处理优先建议` 默认不永久保留；当上下文变化后，应允许重新评估。

一句话收口：

`治理面是主循环用来调整下一轮姿态的临时记忆，不是永久真值；它们必须随时间、使用和条件变化而衰减或解除。`


## 4. 学习压力

必须固定：

1. `学习压力` 当前优先按 `待学习方法数量` 或等价待补方法数量理解。
2. 新增待补方法、待补条件、待补观察、待补解释时，学习压力可上升。
3. 当对应方法缺口被补齐、学习结果被吸收或方法不再待学时，学习压力应下降。
4. 学习压力不是永久累加器。


## 5. 重试压力

必须固定：

1. `重试压力` 表示当前对象在条件重新变化后，是否值得再次尝试。
2. `重试压力` 不是“无条件继续重试授权”。
3. 只有当：
   - 条件发生变化
   - 或原本缺失条件重新满足
   - 或外部环境重新允许
   时，重试压力才应重新激活。
4. 在条件未变化前，重试压力应优先保持冻结或衰减，而不是持续推动重复执行。


## 6. 风险记忆

必须固定：

1. `风险记忆` 属于治理面的临时风险记忆，不等于根层禁止项。
2. 风险记忆当前应同时考虑：
   - `时间流逝`
   - `后续重复验证次数`
   两条衰减轴。
3. 若后续在相同或相近场景中持续得到稳定正向结果，则风险记忆应逐步衰减。
4. 若再次命中负向实践结果，则风险记忆可重新上升。


## 7. 禁止项相关治理面

### 7.1 普通执行冻结候选

必须固定：

1. `普通执行冻结候选` 不是永久封门。
2. 当根层边界、条件状态、上下文结构发生变化后，应允许重新评估是否解除冻结。

### 7.2 边界处理优先建议

必须固定：

1. `边界处理优先建议` 是临时治理姿态，不是永久标签。
2. 当边界原因消失、上下文刷新后，应允许降低其优先级或撤销。


## 8. 严格禁止的混层

以下做法必须禁止：

1. 把治理面当成永久真值保存不衰减。
2. 把学习压力直接写成根层禁止项。
3. 把重试压力直接写成“必须继续执行”。
4. 把风险记忆直接写成永久不可用结论。
