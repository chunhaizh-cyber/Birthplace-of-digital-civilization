# 0327 多特征需求权重汇总与否决项裁决规范

## 1. 目标

本文用于固定多特征需求在当前阶段的最小综合裁决规则。

本文要回答的是：

- 多特征需求在逐特征比较之后，当前如何综合
- `否决项` 为什么优先级更高
- 普通特征为什么要按权重和裁决
- 这套规则与根层安全/服务边界如何衔接

## 3. 核心结论

必须固定以下结论：

1. 多特征需求当前仍必须先 `逐特征比较`，不得先做黑箱总比较。
2. 在逐特征比较之后，当前综合裁决应按两步进行：
   - 先判 `否决项`
   - 再对非否决项按 `权重和` 做综合
3. `否决项` 的优先级高于普通权重和。
4. 即使普通特征加权结果看起来正向，只要命中否决项，当前仍不得判为成立。
5. 多特征综合裁决仍不得越过根层安全/服务边界；否决项优先之后，最终仍要回到外层重判。

一句话收口：

`多特征需求当前先逐特征，再先判否决项，最后对其余特征按权重和综合；否决项永远先于普通加权结果。`


## 4. 最小综合顺序

当前固定顺序为：

```text
逐特征生成方向与对齐结果
-> 标出否决项特征
-> 若任一否决项不成立，则当前整体不成立
-> 若否决项全部通过，再对普通特征按权重和综合
-> 形成当前多特征综合候选结论
-> 回到外层重判
```


## 5. 否决项

### 5.1 什么是否决项

`否决项` 指的是：

- 一旦不成立，就足以使当前整体候选不应被判为成立的关键特征

当前常见否决项包括：

- 直接触及安全生死边界的特征
- 直接命中根层禁止、违规违法、致命边界的特征
- 明确约定为“必须先满足”的关键门槛特征

### 5.2 否决项裁决

必须固定：

1. 否决项不参与普通加权求和抵消。
2. 否决项若为 `反向` 或 `未满足`，则当前整体不得判为成立。
3. 否决项若要求 `等于`，则实际结果也必须保持 `等于`。


## 6. 普通特征的权重和裁决

在未命中否决项时，普通特征当前按 `权重和` 综合。

必须固定：

1. 每个普通特征仍先独立判为：
   - `成立`
   - `无推进`
   - `反向`
2. 再结合该特征在当前需求中的权重，形成普通特征综合权重和。
3. 当前综合权重和只回答：
   - 本轮普通特征整体更偏正向
   - 本轮普通特征整体更偏无推进
   - 本轮普通特征整体更偏反向

必须补充固定：

- 这里的 `权重和` 是多特征需求内部的综合材料，不直接替代外层最终裁决。


## 7. 与安全/服务根边界的关系

必须固定：

1. 多特征需求内部的 `否决项优先 + 权重和综合`，只是需求内部综合规则。
2. 它不能越过：
   - `安全值 = 0` 的生死边界
   - 根层禁止项
   - 根层仅允许收束/待机/停止的约束
3. 因此即使内部综合结果偏正向，外层仍可因根层边界而拒绝执行或拒绝继续。


## 8. 严格禁止的混层

以下做法必须禁止：

1. 跳过逐特征比较，直接拿总分或总权值做结论。
2. 用普通权重和抵消否决项失败。
3. 让多特征需求内部的综合结果直接替代外层根层重判。
4. 把主键命中或方法成功率直接折算成多特征裁决结果。
