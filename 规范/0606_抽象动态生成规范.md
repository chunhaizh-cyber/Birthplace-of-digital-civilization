## 3. 基本定义

### 3.1 实例动态

`实例动态` 指的是：

- 某个具体主体
- 在某个具体时间段
- 发生的一次具体动作或变化

它是动态事实，不是抽象概念。

必须固定：

- 实例动态在结构上是 `连续出现的有序状态集合`
- 一个实例动态最少由 `2` 个状态构成，最多不设上限，可由 `n` 个状态构成，其中 `n >= 2`
- 当只存在 `2` 个状态时，可视为最小动态
- 当存在多个中间状态时，中间状态共同构成该动态的过程路径
- `初始状态 = 结果状态` 仍然允许成立，只要中间状态序列或过程片段可识别
- 对只记录动作边界的动态，这里的“状态”允许表现为 `开始相位状态 -> 结束相位状态`

### 3.2 抽象动态

`抽象动态` 指的是：

- 从多个实例动态中归并出来的可复用动态模式
- 它描述的是“某类变化如何发生”，而不是“某次变化在什么时候发生”

必须固定：

- 抽象动态不是某次单独事件
- 抽象动态不是某个具体主体私有的动态
- 具体主体只能命中抽象动态，不能定义抽象动态


## 4. 核心边界

### 4.1 抽象动态必须建立在抽象状态之上

必须固定：

`抽象动态不应绕过抽象状态直接出生。`

推荐依赖链固定为：

```text
实例状态
-> 抽象状态
-> 实例动态
-> 抽象动态
```

原因是：

1. 动态天然包含一个 `连续出现的状态序列`
2. 这个序列最少要有 `初始状态` 与 `结果状态`
3. 若这些状态还停在实例层，则抽象动态身份会过度依赖具体样本
4. 先有抽象状态，才能把动态压缩成“状态模式之间的迁移或闭环”

### 4.2 抽象动态不是只看动作名

必须固定：

`抽象动态不能只保留动作名或动作短语。`

抽象动态至少要同时保留：

1. `初始状态` 的抽象面
2. `结果状态` 的抽象面
3. `状态序列` 的变化面
4. `过程路径` 的抽象面
5. `时间因素` 的抽象面

一句话收口：

`抽象动态 = 抽象状态链 + 动态差分链 + 过程路径二次特征链 + 时间二次特征链`

### 4.2c 动态最少两态，最多 n 态

必须固定：

- 一个动态最少由 `2` 个状态构成
- 一个动态最多不设上限，可由 `n` 个状态构成
- 动态应统一理解为 `连续出现的有序状态集合`
- 动态的规范表达不应被锁死为只有 `初始状态 -> 结果状态` 两态

因此动态在结构上应统一理解为：

```text
S0 -> S1 -> ... -> S(n-1)
```

其中：

- `n >= 2`
- `S0` 是起始状态
- `S(n-1)` 是结果状态
- `S1 ... S(n-2)` 为可选中间状态

一句话收口：

`两态动态是最小形式，多态动态是完整形式。`

### 4.2a 首尾同态不等于无动态

必须固定：

若某次动态的初始抽象状态与结果抽象状态相同，但中间存在可识别的局部变化过程、相位切换、往返路径或片段动态序列，
则该动态仍可形成抽象动态候选。

判断“是否有动态”时，不只看首尾状态是否不同，还要看：

- 是否存在中间状态序列
- 是否存在局部变化片段
- 是否存在完整边界
- 是否存在稳定的时间因素模式

因此必须固定：

`首尾同态 != 无动态`

只有在以下两点同时成立时，才可退化为“无有效动态”：

- `初始抽象状态 = 结果抽象状态`
- 中间过程也不可识别

### 4.2b 抽象动态分两类

抽象动态至少分为两类：

1. `首尾迁移型抽象动态`
- `初始抽象状态 != 结果抽象状态`
- 重点表达“从哪迁到哪”

2. `过程闭环型抽象动态`
- `初始抽象状态 = 结果抽象状态`
- 但存在可识别的 `中间状态序列 / 相位序列 / 局部变化片段`
- 重点表达“虽然首尾同态，但中间过程真实存在且可复用”

### 4.3 抽象动态也要删除单次实例时间身份

和抽象状态一样，抽象动态在抽象动作后，不再保留单次实例时间作为身份字段。

这里的“删除时间”固定解释为：

- 不把实例动态的 `开始时间 / 结束时间 / 单次相位时刻` 带入抽象动态身份
- 改为把时间压缩成 `时间因素二次特征`

因此必须固定：

```text
实例时间 -> 时间二次特征
实例时间 -/-> 抽象动态身份
```

### 4.4 “时间为空”不能作为抽象动态判据

必须明确：

`不是所有时间为空的动态节点，都是抽象动态节点。`

原因与抽象状态同理：

1. 时间为空可能只是未填充
2. 时间为空可能只是中间态
3. 时间为空的动态仍可能绑定某次具体主体和具体动作，因此仍是实例事实
4. 抽象动态的成立依赖重复命中、状态迁移模式与时间模式的稳定归并，而不是依赖“时间字段为 0”


## 5. 抽象动态比抽象状态多出的部分

相比抽象状态，抽象动态多出的内容固定为三大组。

## 5.1 状态差分二次特征组

这是抽象动态最核心的第一组额外输入。

至少包括：

1. `初始状态二次特征`
2. `结果状态二次特征`
3. `状态迁移二次特征`

其中：

- `初始状态二次特征` 回答“这次变化从什么抽象状态出发”
- `结果状态二次特征` 回答“这次变化落到什么抽象状态”
- `状态迁移二次特征` 回答“整个状态序列如何演进，方向如何，差分有多大，属于哪类迁移”

### 5.1.1 状态迁移二次特征至少应能表达

- `上升`
- `下降`
- `回升`
- `回落`
- `切换`
- `迁移`

以及必要时的：

- `小幅 / 中幅 / 大幅`
- `同层微调 / 跨区间跃迁 / 终态进入`

## 5.2 时间因素二次特征组

这是抽象动态比抽象状态多出的第二组关键输入。

至少应包括：

1. `持续时间区间`
- 极短
- 短窗
- 中窗
- 长窗

2. `起止节奏`
- 立即完成
- 持续推进
- 长时拖延
- 中断失败

3. `发生频率`
- 高频
- 中频
- 低频
- 偶发

4. `重复模式`
- 单次型
- 周期型
- 连续型
- 间歇型

5. `相位模式`
- 开始型
- 完成型
- 失败型
- 完整边界型

必须固定：

- 抽象动态中保留的是“时间因素的二次特征”
- 不是实例动态的原始时间戳本身

## 5.3 过程路径二次特征组

这是抽象动态从“首尾差值模型”升级为“过程模型”的关键输入。

至少应包括：

1. `中间抽象状态序列签名`
2. `相位序列签名`
3. `转折点数量`
4. `峰值/谷值偏移`
5. `回返类型`
6. `闭环类型`

其中：

- `中间抽象状态序列签名` 回答“中间经过了哪些抽象状态”
- `相位序列签名` 回答“开始 / 推进 / 失败 / 完成如何排布”
- `转折点数量` 回答“过程里出现了几次方向切换或阶段转折”
- `峰值/谷值偏移` 回答“过程里是否曾经显著偏离起点状态”
- `回返类型` 回答“离开原状态后是如何返回的”
- `闭环类型` 回答“这是不是一个完整闭环，以及属于哪类闭环”

### 5.3.1 回返类型至少应支持

- `试探回返`
- `扰动恢复`
- `振荡回返`
- `失败回退`
- `维护闭环`

### 5.3.2 闭环类型至少应支持

- `无闭环`
- `单次闭环`
- `周期闭环`
- `多相闭环`


## 6. 抽象动态的最小生成条件

一个候选动态要想被生成成 `抽象动态候选`，至少应满足以下两路之一，并额外满足共同条件。

### 6.1 A路：首尾迁移型

1. 存在可识别的 `初始抽象状态`
2. 存在可识别的 `结果抽象状态`
3. `初始抽象状态 != 结果抽象状态`
4. 能生成 `状态迁移二次特征`
5. 能生成 `时间因素二次特征`

### 6.2 B路：过程闭环型

1. 存在可识别的 `初始抽象状态`
2. 存在可识别的 `结果抽象状态`
3. `初始抽象状态` 允许与 `结果抽象状态` 相同
4. 存在至少一个可识别的 `中间状态` 或 `中间相位`
5. 能生成 `过程路径二次特征`
6. 能生成 `时间因素二次特征`

### 6.3 共同条件

1. 存在可识别的 `动作核心类型` 或 `变化类型`
2. 单次动态内部至少能识别出 `2` 个状态
3. 至少出现过 `2` 次及以上同类命中
4. 这些命中可被归并到同一动态模式


## 7. 生成流程

抽象动态的标准生成链固定为：

```text
实例动态
-> 切成简单动态片段
-> 为每个片段匹配状态序列（至少 `初始状态 / 结果状态`，可带多个中间状态）
-> 状态序列中的各状态分别命中抽象状态
-> 生成状态迁移二次特征
-> 生成过程路径二次特征
-> 生成时间因素二次特征
-> 聚合为复杂动态
-> 删除单次实例时间身份
-> 形成抽象动态候选键
-> 更新或创建抽象动态候选
-> 达到阈值后升格为正式抽象动态
```

必须固定：

1. 没有初始/结果抽象状态，不应直接生成正式抽象动态
2. 没有 `状态迁移二次特征` 且没有 `过程路径二次特征`，不应把动态只靠动作名压成抽象动态
3. 没有时间因素二次特征，不应宣称已完成抽象动态
4. 当 `初始抽象状态 = 结果抽象状态` 时，若中间过程也不可识别，则不应宣称已形成有效抽象动态
5. 不得把只含单个状态点的记录冒充为动态


## 8. 世界树挂载规则

抽象动态不应继续按“运行期动态节点”挂载。

推荐挂载结构固定为：

```text
世界树根
-> 概念域根
  -> 动态域根
    -> 动作/变化类型根
      -> 抽象动态
        -> 更细抽象动态
    -> 复合抽象动态分支
```

必须固定：

1. `抽象动态` 挂到 `概念域::动态`
2. 单条抽象动态优先挂到对应动作/变化类型根下
3. 多个抽象动态构成的更高层模式，后续可挂到复合抽象动态分支
4. 具体主体只保留“命中该抽象动态”的关系，不得把抽象动态挂成某个主体的私有子节点


## 9. 概念身份规则

抽象动态身份不得主要依赖：

- 名称字符串
- 某次实例动态的主键
- 某次实例动态的开始时间
- 某次实例动态的结束时间
- 某个具体主体主键

抽象动态的最小身份建议由以下字段共同决定：

- `概念域 = 动态`
- `动作核心类型`
- `来源业务语义类型`
- `初始抽象状态主键`
- `结果抽象状态主键`
- `状态迁移签名`
- `过程路径签名`
- `时间因素签名`
- `适用条件主键`

必须固定：

- `概念主键` 只负责身份、归桶与去重
- 抽象动态是否成立，仍需依赖重复命中、稳定性与适用条件检查
- 当 `初始抽象状态主键 = 结果抽象状态主键` 时，`过程路径签名` 成为区分不同抽象动态的必要身份字段


## 10. 命中、升格、衰减与淘汰

## 10.1 命中增强

当新的实例动态继续命中某个抽象动态时，应至少更新：

- `命中次数`
- `最近命中时间`
- `稳定度`

## 10.2 升格条件

一个抽象动态候选要升格为稳定抽象动态，至少应满足：

1. `命中次数 >= 最小阈值`
2. `最近命中时间` 仍在活跃窗口内
3. `状态迁移模式` 或 `过程路径模式` 相对稳定
4. `时间因素模式` 没有频繁漂移
5. 当 `初始抽象状态 = 结果抽象状态` 时，`过程路径模式` 具备独立区分价值
6. 相比父抽象动态具备独立区分价值

## 10.3 低频衰减

若某抽象动态长期不再被命中，或命中频率过低，应逐步衰减：

- `稳定度下降`
- `活跃度下降`
- `推荐度下降`

## 10.4 淘汰条件

满足以下任一条件时，可将抽象动态降级或淘汰：

1. 长期未命中
2. 命中频率持续低于阈值
3. 状态迁移模式或过程路径模式不再稳定
4. 时间因素模式持续漂移
5. 当 `初始抽象状态 = 结果抽象状态` 时，过程路径已失去独立区分价值
6. 相比父抽象动态已失去独立区分价值

## 10.5 淘汰后的处理

淘汰不等于直接物理删除。

建议优先分三种处理：

1. `稳定 -> 冷却`
2. `冷却 -> 候选`
3. `候选 -> 清理`


## 11. 一句话收口

可收敛为一句话：

`抽象动态的生成方式和抽象状态类似，但它必须额外吸收“连续出现、且在多次观察中会重复出现的状态序列”“过程路径的二次特征”和“时间因素的二次特征”，最终形成“某类状态模式在某类时间模式下如何沿某条过程路径迁移或闭环”的可复用动态概念。`
