## 3. 核心结论

必须固定以下 12 条：

1. 学习对象的实现，后续统一采用 `对象选择器 + 对象适配器 + 原料结构` 三层模型。
2. `学习目标` 先于 `学习对象`，所以学习对象的实现不得绕过目标直接自由选择。
3. 每一类学习对象都必须回答 4 个工程问题：
   - 它存在哪里
   - 如何被选中
   - 如何抽取成统一原料
   - 默认产出什么
4. 当前最先应落实的不是 6 类都同时做满，而是先收口 `方法直接学习对象 + 方法原料抽象入口`。
5. 当前最先应收口的 4 类是：
   - `本能方法`
   - `现有方法/方法片段`
   - `因果信息`
   - `成功执行片段`
6. `稳定任务路径` 与 `稳定任务子树` 在工程里定位为 `方法原料`，不是直接学习执行对象；它们必须先经过 `方法抽象适配器`。
7. 任何学习对象或方法原料进入学习闭环前，都必须先被标准化成统一 `学习对象原料`，并最终落到 `关联方法首节点 / 待学习方法骨架`。
8. 若当前只有 `需求信息 / 目标特征类型`，则该头仅属于 `需求位方法头`，不得直接被标成 `可直接进入学习`。
9. 不允许直接在学习主流程里到处散写“如果对象是 A 就这样、对象是 B 就那样”的无序分支。
10. 本文定义的 `当前具备状态` 统一分成：`已具备 / 半具备 / 待补`。
11. 后续每次落实学习对象时，都应先回看本文，再回看 [1215_学习任务目标-对象-产出-回流规范.md](D:\鱼巢\规范\1215_学习任务目标-对象-产出-回流规范.md)。
12. 当学习对象落实服务于 `任务管理方法` 分支时，外层仍须显式化 `最近根层裁决结果`、`当前缺口类型`、`当前下一步去向`。
13. 任一由学习对象适配结果推动的学习产出，一旦在当前治理轮被消费，仍必须回到外层重新做 `根层重判 -> 上下文刷新`。

可收敛为一句话：

`学习对象的落实，不是把 6 类对象分别硬塞进学习线程，而是先用统一原料结构收口，再逐类接适配器。`


## 4. 统一实现模型

后续学习对象统一按以下三层实现：

### 4.1 学习对象选择器

职责：

- 根据 `学习目标`
- 在所有合法学习对象里
- 选出当前最合适的对象

输入至少包括：

- `来源宿主任务`
- `来源步骤`
- `来源方法需求位`
- `学习类型`
- `目标约束强度`

输出至少包括：

- `学习对象类型`
- `原对象指针`
- `选择理由`


### 4.2 学习对象适配器

职责：

- 把某类具体对象抽成统一学习原料

每个适配器最少要提供：

- `是否支持当前目标`
- `从当前上下文选择候选对象`
- `把对象抽成统一原料`
- `给出默认产出建议`


### 4.3 学习对象原料结构

职责：

- 作为所有对象进入学习主流程前的统一表达

最小建议字段：

```cpp
结构_学习对象原料_v0
{
    枚举_学习对象类型 类型 = 未定义;
    void* 原对象 = nullptr;
    std::string 对象主键{};

    方法节点类* 关联方法首节点 = nullptr;
    任务节点类* 关联任务头结点 = nullptr;
    任务节点类* 关联步骤节点 = nullptr;

    std::vector<状态节点主信息类> 条件摘要;
    std::vector<状态节点主信息类> 结果摘要;

    bool 仅需求位方法头 = false;
    bool 已具动作骨架 = false;
    bool 已具条件侧 = false;

    bool 可直接进入学习 = false;
    std::string 默认产出建议{};
};
```

必须固定：

- 后续所有学习对象都先进入这层
- 学习主流程只消费这层，不直接消费原始对象
- 若原始对象不是方法，而是因果、片段、路径或子树，则必须先通过 `方法抽象适配器` 补出 `关联方法首节点` 或等价 `待学习方法骨架`
- 若 `仅需求位方法头 = true`，则 `可直接进入学习` 必须固定为 `false`
- 只有当 `已具动作骨架 = true` 且 `已具条件侧 = true` 时，才允许把 `可直接进入学习` 置为 `true`
- 若方法条件特征类型为 `I64`，允许按类型值域或样本值域生成 `最小 / 中间 / 最大` 试探值
- 若方法条件特征类型为 `VecIU64`，不生成最大最小值，统一走相似度比较
- 若方法条件特征类型为 `指针`，不得直接学习原指针值，必须先转成二次特征；二次特征统一使用 `I64`

### 4.4 与任务管理治理分支的边界

必须固定：

1. 本文描述的是全域“学习对象选择器 + 对象适配器 + 原料结构”的共性实现模型。
2. 若服务对象是 `任务管理方法` 分支，则必须额外携带：
- `最近根层裁决结果`
- `当前缺口类型`
- `当前下一步去向`
3. 这三项属于治理上下文，不属于所有学习对象原料的统一强制字段。
