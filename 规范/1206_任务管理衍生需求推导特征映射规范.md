# 1206 任务管理衍生需求推导特征映射规范

## 1. 目标

本文用于固定以下问题：

- `任务管理任务` 可以从哪些现有特征中推导出衍生需求
- 哪些衍生需求属于 `单轮即可判定`
- 哪些衍生需求必须依赖 `跨轮聚合`
- `扩选 / 健壮性 / 覆盖 / 恢复 / 安全边际 / 降本` 等方向分别由哪些特征支撑
- 这些衍生需求在统一主链中的上浮位置是什么
- 后续应如何从这些推导量生成 `学习子任务 / 储备学习动机 / 方法资产扩充动机`

本文不负责规定具体学习算法，也不负责规定具体线程实现。

## 3. 核心结论

必须固定以下 8 条：

1. `扩选 / 健壮性 / 覆盖 / 恢复 / 安全边际 / 降本 / 冗余 / 可验证` 这些方向，都不属于新的原生根需求。
2. 这些方向统一属于 `任务管理任务衍生的内部治理需求`。
3. 它们必须能够从 `安全/服务根任务` 的真实推进过程里自动上浮，而不是由人工先验硬定义。
4. `单轮控制输入 + 管理派生产出 + 管理二次特征` 已经足以直接推导出一部分衍生需求。
5. `任务多样性 / 覆盖面 / 冗余度 / 迁移性` 这类方向，通常不能只看单轮，必须增加跨轮聚合。
6. `泛化学习` 不是“无需求学习”，而是某类 `方法需求位` 的压力长期偏高后，上浮出来的 `扩选/健壮/覆盖` 动机。
7. `新增可用方法`、`提升方法稳定性`、`补齐条件或结果` 都属于内部治理结果，因此属于 `任务管理任务` 可处理范畴。
8. 后续若要生成真正的学习子任务，必须先把特征压缩成 `压力维度`，再由压力维度生成 `衍生需求`。
9. 衍生需求只能从“根层已允许推进后的治理运行事实”中上浮，不得绕过根层裁决直接独立冒头。
10. 若根层已经判定当前路径应停止或收束，则普通扩选/覆盖/健壮等衍生需求不得继续按常规上浮。

### 3.1 在统一主链中的位置

按当前最新体系，衍生需求的上浮位置必须固定为：

```text
根层方向裁决
-> 形成任务管理上下文
-> 单步治理运行并产生产出/二次特征/结果
-> 压缩成压力维度
-> 上浮衍生需求
-> 必要时转成学习子任务或储备动机
```

必须固定：

1. 衍生需求不属于根层原生需求。
2. 衍生需求也不直接等于学习任务本体。
3. 衍生需求首先是任务管理层对持续治理缺口的结构化表达。

可收敛为一句话：

`任务管理衍生需求，应由任务管理特征自动上浮出来，而不应由系统外部硬塞一个“长期学习需求”名称。`


## 4. 推导所依赖的三层特征

## 4.1 原始控制输入

这层回答：

- 当前是否还要继续推进
- 当前是否缺方法、缺条件、缺结构
- 当前风险是否允许继续探索

最关键的现有字段包括：

- `任务链已成形`
- `已有可尝试方法`
- `最近结果稳定`
- `最近执行失败`
- `风险变坏`
- `检查点通过`
- `依赖缺失`
- `需要新子任务或新步骤`
- `关键方法不再可用`
- `服务收益不足以继续`
- `安全要求必须停止`
- `已满足完成判据`
- `已确定无法继续`

这些字段当前来源于根任务控制输入，见实现入口：

- [自我线程模块.ixx](D:\鱼巢\自我线程模块.ixx)
- `私有_组装根任务控制判定输入(...)`

必须补充固定：

- 这层特征应理解为“根层已裁决后，任务管理层本轮可用的控制输入和上下文输入”。
- 它们是衍生需求的输入原料，不是衍生需求本身。


## 4.2 管理派生产出

这层回答：

- 当前到底缺什么
- 当前推荐往哪个方向走
- 是否已经需要学习、观察、恢复、补条件

最关键的现有字段包括：

- `管理任务_条件缺口分类`
- `管理任务_可执行性判断`
- `管理任务_候选方法评分`
- `管理任务_推荐分解方向`
- `管理任务_是否需要补条件任务`
- `管理任务_是否需要观察任务`
- `管理任务_是否需要恢复任务`
- `管理任务_是否建议等待学习`
- `管理任务_是否建议回到筹办`
- `管理任务_评分证据Q`
- `管理任务_失败原因码`
- `管理任务_二次特征证据`
- `管理任务_结果节点主键`


## 4.3 管理二次特征

这层回答：

- 当前局势在治理语义上像什么
- 当前可复用性、稳定性、价值性是否已形成

最关键的现有字段包括：

- `管理任务_治理态型`
- `管理任务_结构完备度`
- `管理任务_方法可用度`
- `管理任务_风险门控`
- `管理任务_结构推进成立`
- `管理任务_治理闭环成立`
- `管理任务_稳定复现级别`
- `管理任务_治理可复用性`
- `管理任务_治理价值成立`
- `管理任务_价值闭环类型`
- `管理任务_条件价值映射已出现`


## 5. 压力维度定义

后续衍生需求，不应直接从零散特征一跳生成，而应先压缩成 `压力维度`。

当前最小压力维度固定为：

- `扩选压力`
- `健壮性压力`
- `覆盖压力`
- `恢复压力`
- `安全边际压力`
- `降本压力`
- `冗余压力`
- `可验证压力`

说明：

- `迁移性` 暂并入 `覆盖压力`
- `单点依赖` 暂并入 `冗余压力`
- 后续若要细拆，必须仍保持这些上层维度不变


## 6. 特征到压力的映射

## 6.1 扩选压力

含义：

- 当前这类任务的可选方法范围过窄
- 一旦现有少量方法不可用，就容易卡死

最直接依赖的现有特征：

- `已有可尝试方法 = 0`
- `管理任务_方法可用度 = 低`
- `管理任务_失败原因码 = 缺少方法 / 等待学习`
- `管理任务_是否建议等待学习 = 1`
- `管理任务_结构完备度 = 低`
- `管理任务_治理态型 = 补结构型`

建议上浮规则：

- 若 `已有可尝试方法 = 0` 且 `是否建议等待学习 = 1`，则本轮 `扩选压力` 至少为 `高`
- 若 `方法可用度 = 低` 且 `失败原因码` 连续多轮落在 `缺少方法 / 等待学习`，则 `扩选压力` 持续累积

可派生需求：

- `扩选需求`
- `弱定向学习需求`
- `扩选型泛化学习需求`


## 6.2 健壮性压力

含义：

- 当前不是完全没方法，而是已有方法不够稳、容易分裂或失败

最直接依赖的现有特征：

- `最近结果稳定 = 0`
- `最近执行失败 = 1`
- `风险变坏 = 1`
- `检查点通过 = 0`
- `管理任务_风险门控 = 需谨慎 / 需纠偏`
- `管理任务_治理闭环成立 = 0`
- `管理任务_稳定复现级别 = 低`
- `管理任务_治理可复用性 = 低`

建议上浮规则：

- 若 `已有可尝试方法 = 1` 但 `最近结果稳定 = 0` 且 `最近执行失败 = 1`，则本轮优先上浮 `健壮性压力`
- 若 `稳定复现级别` 长期停留在 `临时`，且 `治理可复用性 = 低`，则 `健壮性压力` 应继续累计

可派生需求：

- `健壮性需求`
- `方法加固学习需求`
- `稳态修复学习需求`


## 6.3 覆盖压力

含义：

- 当前方法并非完全不可用，但覆盖不了越来越分散的任务条件桶

直接原料：

- `管理任务_成长证据_输入条件主键哈希`
- `管理任务_结果节点主键`
- `管理任务_治理态型`
- `管理任务_推荐分解方向`
- `管理任务_当前功能域`
- `管理任务_失败原因码`

关键说明：

- `覆盖压力` 不能靠单轮直接判定
- 它必须依赖跨轮聚合

最小聚合口径：

- 同一 `方法需求位` 下，最近一段时间出现了多少不同 `输入条件主键哈希`
- 这些条件桶对应的 `结果主键` 是否高度离散
- 相同需求位下是否经常走到不同 `推荐分解方向`

建议上浮规则：

- 若同一需求位下 `输入条件桶` 持续增多，而 `稳定复现级别 / 治理可复用性` 没同步上升，则 `覆盖压力` 升高
- 若多轮都在相近需求位进入 `等待学习`，但每轮具体条件桶不同，则 `覆盖压力` 高于 `单点扩选压力`

可派生需求：

- `覆盖需求`
- `任务多样性适应需求`
- `扩覆盖型泛化学习需求`


## 6.4 恢复压力

含义：

- 主方法链一旦偏离或失败，缺少足够好的纠偏、回退、替代路径

最直接依赖的现有特征：

- `最近执行失败 = 1`
- `关键方法不再可用 = 1`
- `管理任务_是否需要恢复任务 = 1`
- `管理任务_风险门控 = 需纠偏`
- `管理任务_治理态型 = 偏离修复型`

建议上浮规则：

- 只要 `是否需要恢复任务 = 1`，本轮就至少产生 `恢复压力`
- 若同一需求位连续多轮都以 `偏离修复型` 结束，则应上浮 `恢复学习需求`

可派生需求：

- `恢复需求`
- `纠偏替代学习需求`
- `回退路径储备需求`


## 6.5 安全边际压力

含义：

- 当前方法虽然可能可用，但安全余量偏低，不适合继续冒险

最直接依赖的现有特征：

- `安全要求必须停止 = 1`
- `风险变坏 = 1`
- `检查点通过 = 0`
- `管理任务_风险门控 = 需谨慎 / 需停止`

建议上浮规则：

- 若 `风险门控 = 需停止`，则不再上浮学习需求，而应优先收束或停止
- 若长期处于 `需谨慎` 且每次推进都高度依赖少量方法，则可上浮 `安全边际学习需求`

可派生需求：

- `安全边际需求`
- `保守替代方法储备需求`


## 6.6 降本压力

含义：

- 当前方法能完成，但成本、收益比或服务代价不理想

最直接依赖的现有特征：

- `服务收益不足以继续 = 1`
- `管理任务_评分证据Q` 长期偏低
- `管理任务_治理闭环成立 = 1` 但 `治理价值成立 = 0`

关键说明：

- 当前实现对 `降本压力` 的支撑还偏弱
- 它已经有原料，但还没有独立成本统计面

建议上浮规则：

- 若 `治理闭环成立 = 1`，但 `评分证据Q` 长期偏低，且 `服务收益不足以继续` 反复出现，可上浮 `降本需求`

可派生需求：

- `降本需求`
- `低成本替代方法储备需求`


## 6.7 冗余压力

含义：

- 当前需求位过度依赖少数方法，一旦失效就没有替代路径

直接原料：

- `管理任务_方法可用度`
- `管理任务_关键方法不再可用`
- `管理任务_稳定复现级别`
- `管理任务_治理可复用性`
- 同需求位下的 `建议方法首节点指针` 历史分布

关键说明：

- `冗余压力` 也必须依赖跨轮聚合
- 单轮里通常只能看出“当前关键方法不可用”，还看不出“是否长期单点依赖”

建议上浮规则：

- 若同一需求位的历史命中方法长期集中在极少数方法首节点，且出现 `关键方法不再可用 = 1`，则 `冗余压力` 应上升

可派生需求：

- `冗余需求`
- `替代方法储备需求`


## 6.8 可验证压力

含义：

- 当前方法可能有效，但结果不容易被确认，导致不敢高置信复用

最直接依赖的现有特征：

- `管理任务_治理闭环成立 = 0`
- `管理任务_治理价值成立 = 0`
- `管理任务_条件价值映射已出现 = 0`
- `管理任务_结果节点主键` 高度波动

建议上浮规则：

- 若方法多轮运行都能推进结构，但长期难以形成闭环或价值证据，则可上浮 `可验证压力`

可派生需求：

- `可验证需求`
- `观测/回执增强学习需求`


## 7. 哪些需求能直接推，哪些必须聚合后再推

### 7.1 单轮可直接上浮

以下方向可以直接由当前单轮特征上浮：

- `扩选需求`
- `健壮性需求`
- `恢复需求`
- `安全边际需求`
- `部分降本需求`

它们依赖的主要是：

- 原始输入
- 当前轮派生产出
- 当前轮二次特征

但必须补充固定：

- “可直接上浮”仍然建立在 `根层允许继续推进` 的前提下。
- 若本轮已进入强停止、强收束或停机边界处理，则这类普通上浮应让位于收束、止损和记录动作。

### 7.2 必须跨轮聚合后再上浮

以下方向不应只看单轮：

- `覆盖需求`
- `冗余需求`
- `迁移性需求`
- `强降本需求`
- `扩覆盖型泛化学习需求`

它们至少需要聚合：

- `输入条件主键哈希` 分布
- `结果主键` 分布
- `建议方法首节点` 分布
- `治理态型` 分布
- `失败原因码` 分布
- `等待学习 / 恢复 / 回筹办` 的重复模式


## 8. 从压力维度到衍生需求的最小规则

后续统一按以下顺序生成：

```text
任务管理特征
-> 压缩为压力维度
-> 判断压力是否超过上浮阈值
-> 生成任务管理衍生需求
-> 再决定是定向学习、弱定向学习还是泛化学习
```

必须禁止以下错误跳跃：

- 不能从某个失败样本直接跳成“长期需求”
- 不能从某次等待学习直接跳成“泛化学习”
- 不能把“没有方法”与“方法不稳”混成同一种衍生需求
- 不能把“覆盖不够”与“可选范围太窄”完全等同


## 9. 当前实现已具备与仍需补充的部分

### 9.1 已具备

- 推 `扩选需求` 的直接特征已经足够
- 推 `健壮性需求` 的直接特征已经足够
- 推 `恢复需求` 的直接特征已经足够
- 推 `安全边际需求` 的直接特征已经足够
- 推 `部分降本需求` 已有原料

### 9.2 仍需补充

- `覆盖压力` 的跨轮桶统计
- `冗余压力` 的方法分布统计
- `迁移性/任务多样性` 的离散度统计
- `降本压力` 更细的成本账本
- `输入条件主键原文` 的任务侧镜像


## 10. 收口结论

`任务管理任务` 当前已经有足够的特征基础，用来自动推导一批内部衍生需求。  
其中：

- `扩选 / 健壮 / 恢复 / 安全边际` 已可直接由现有单轮特征上浮
- `覆盖 / 冗余 / 迁移 / 强降本` 还需要跨轮聚合

因此，后续真正需要补的重点，不是再凭空定义新的“长期需求”名称，而是：

1. 固定 `压力维度`
