# 1315 任务管理任务三段式流程规范（保留原文件名兼容）

## 1. 说明

原 `五模块流程` 口径已废止。

当前 `任务管理任务` 的正式主流程统一为：

`目标 -> 筹办 -> 执行`


## 2. 任务管理任务的目标

`任务管理任务` 的标的是 `任务`。

它的目标固定为：

`使宿主任务达到完成状态`


## 3. 任务管理任务的筹办

任务管理任务的筹办职责固定为：

1. 读取当前宿主任务、当前步骤、当前结果以及任务/方法虚拟存在特征。
2. 判断“当前少了什么”，形成 `筹办需求集合`。
3. 把筹办需求提交给 `自我线程`，等待 `是否可满足 + 返回节点` 的回执。
4. 把回执返回的步骤、结果、方法、子任务挂回 `任务管理任务树` 与 `宿主任务树`。
5. 当当前特征类型无法表达治理缺口时，创建新的虚拟存在特征类型。

也就是说，任务管理任务的筹办不是“判断五段”，而是：

`把任务管理任务树和宿主任务树补到可继续执行，并把执行入口闸门交给当前执行步骤`


## 4. 任务管理任务的执行

任务管理任务的执行职责固定为：

1. 消费筹办回给当前步骤的 `执行入口闸门`。
2. 执行当前治理步骤。
3. 推动宿主任务当前步骤前进一步。
4. 把结果写到当前治理结果节点。

执行后只允许：

1. 宿主任务完成
2. 回到筹办继续长树


## 5. 任务管理任务树的显式步骤

虽然主流程只保留三段式，但 `任务管理任务` 本体树允许显式长出多个治理步骤。

当前显式步骤可以包括：

1. `受理请求`
2. `宿主锚点确认`
3. `绑定宿主方法`
4. `宿主步骤树筹办`
5. `前置缺口外化`
6. `执行当前治理步骤`
7. `学习承接与业务派生`
8. `写回结果`
9. `组装一步结果`

必须固定：

- 这些是 `任务管理任务树上的步骤语义`
- 不是和 `目标 / 筹办 / 执行` 并列的一级主流程


## 6. 对旧五模块口径的处理

旧文中的：

`筹办 -> 可执行判断 -> 能执行判断 -> 执行 -> 反馈`

不再作为任务管理任务统一主流程保留。

若后续仍需保留，只允许把它们降级为：

- 任务管理步骤语义
- 日志观察切片
- 执行细分标签
- 执行入口闸门
