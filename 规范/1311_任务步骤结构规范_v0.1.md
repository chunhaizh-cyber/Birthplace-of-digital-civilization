## 3. 核心结论

必须固定以下结论：

1. 本文定义的 `任务步骤结构`，不是新的外层流程；它只放在 `执行当前域 -> 当前域任务子树 -> 当前步骤节点` 这一层。
2. 外层骨架仍固定为：`判域 -> 选树 -> 执行 -> 接收结果`；本文只定义其中“执行当前域”内部真正消费的步骤结构。
3. 每个步骤都必须先回答两个问题：`想让谁的哪个特征进入什么目标状态`，以及 `为了得到这个状态，需要哪些前置状态和动作路径`。
4. 每个步骤当前正式固定为 `8` 组字段：`步骤标识 / 目标特征状态 / 前置状态集 / 动作集 / 达成判据 / 失败去向 / 结果写回锚点 / 一步治理返回映射`。
5. `当前方法需求位` 是步骤的 `能力标签`，不是具体方法本体；它回答“当前主要缺哪类治理能力”，不回答“唯一该调用哪个方法”。
6. `动作集` 回答的是可用状态-动作路径；结构差异明显的候选方法或候选路径必须拆成并列步骤，同一步骤内只保留当前方法与少量备用方法。
7. 当前步骤未达成时，顶层先只分 `4` 类：`本地继续推进 / 局部等待 / 上报派生需求 / 失败或收束或回退`；若实现层需要更细分，再在这四类之下展开。
8. `步骤达成` 只说明当前步骤命中了本步目标状态或其候选成立面，不得越权写成 `需求已满足`、`主链已结束` 或 `允许升格`。
9. 步骤可以记录 `外部依赖引用`，但不能直接拥有 `正式子任务创建权`；需要外部支撑时，只能通过 `派生需求消息` 上浮给外层。
10. `结果方向二次特征`、`前置条件二次特征`、`结果摘要二次特征` 在步骤结构里统一属于证据层；运行时必须经绑定原语转译后，才能进入下一步选择。
11. `推进一步` 读步骤结构时，必须按 `目标特征状态 -> 前置状态集 -> 动作集 -> 达成判据 -> 失败去向` 的顺序推进；其稳定回传面仍以 [1213_任务管理一步治理结果最小返回面规范.md](D:\鱼巢\规范\1213_任务管理一步治理结果最小返回面规范.md) 为准。
12. 主链是否正式收口，以头结点完成判据或目标特征状态是否成立为准，不以结果节点与头结点目标字段字面相同为准。

可收成一句话：

`任务步骤是“围绕目标特征状态组织的局部推进壳”，而不是“先决定做哪个动作”的黑盒。`


## 4. 放置层级与最小原理

### 4.1 正式放置层级

当前必须把 `任务步骤结构` 放在下面这条层级链上理解：

```text
判域
-> 选树
-> 执行当前域
   -> 当前域任务子树
      -> 当前步骤节点
```

因此必须固定：

1. `任务步骤结构` 不替代 `判域`、`选树` 或 `接收结果`。
2. 外层只决定“当前由哪个域、哪棵树来承接本轮”；步骤结构只决定“当前这一步内部怎么推进”。
3. `任务管理任务` 的一步执行，本质上就是在当前前沿步骤上消费这套结构。

### 4.2 任务步骤最小原理

任务步骤的最小解释模板固定为：

```text
目标存在 E
目标特征 F
目标状态 G

当前状态 C
前置状态集 P
可选动作集 A

若 P 满足：
    选择 A 中的一条或组合推进
若 P 不满足：
    先判断是局部等待，还是派生需求

执行后得到结果状态 R
若 F 已达到 G：
    本步成立
若 F 尚未达到 G：
    判断是本地继续推进、局部等待、派生需求，还是失败/收束/回退
```

因此必须固定：

1. `目标状态` 是步骤的第一定义。
2. `前置状态集` 是步骤能不能进入动作的门槛面。
3. `动作集` 是手段，不是步骤本体。
4. `达成判据` 决定本步能不能记为成立。
5. `失败去向` 决定本步未达成时如何继续治理，而不是把失败当成死终点。

### 4.3 步骤首先回答的两个问题

每个步骤当前都必须先能回答下面两个问题：

1. `为什么要这么做`
   因为当前还没有得到某个关键特征的目标状态。
2. `这么做需要什么`
   需要一组前置状态满足，再执行一组动作路径。

这两个问题压平后，步骤就不再是“为了做某个动作”，而是“为了让目标特征状态成立而组织的一次局部推进”。

### 4.4 每个步骤都必须能解释成六件事

任务树里的每个步骤，最少都应能解释成下面 `6` 件事：

| 解释面 | 最小回答 |
| --- | --- |
| `目标是什么` | 让谁的哪个特征，变成什么目标状态 |
| `当前为什么还没达到` | 当前状态与目标状态之间还差什么 |
| `需要哪些前置状态` | 前序、资源、确认、方法可调用、依赖结果、风险门控等是否满足 |
| `需要哪些动作` | 当前有哪些可选动作或方法路径可把状态推进过去 |
| `怎样判定达成` | 看哪个特征到什么值算完成 |
| `如果没达成怎么办` | 本地继续推进、局部等待、上报派生需求、失败/收束/回退中的哪一类 |


## 5. 正式八组字段

### 5.1 八组字段总表

| 字段组 | 回答的问题 | 最少必须写 |
| --- | --- | --- |
| `步骤标识` | 当前在管哪一步、属于谁、落在哪个域、主要缺哪类能力 | `步骤主键`、`所属任务主键`、`所属域`、`当前方法需求位` |
| `目标特征状态` | 这一步到底想让谁的哪个特征变成什么状态 | `目标主体`、`目标特征`、`目标状态` |
| `前置状态集` | 达到目标状态前，最低还要先成立什么 | `前置状态列表`、`来源层`、`是否硬前置`、`是否可部分满足` |
| `动作集` | 前置满足后，可以走哪些动作或方法路径 | `候选动作集`、`主动作`、`备用动作`、`是否需要外部确认`、`是否允许兜底` |
| `达成判据` | 怎么确认这一步真的成立 | `主达成判据`、`次达成判据`、`结果确认方式`、`是否必须等待外部确认` |
| `步骤分流与失败去向` | 本步没达成时，下一步往哪分流 | `步骤分流类型`、`细分失败去向` |
| `结果写回锚点` | 本步的状态、动态、因果与结构结果写回哪里 | `结果状态写回锚点`、`结构推进写回锚点`、`动作动态写回锚点`、`因果实例写回锚点` |
| `一步治理返回映射` | 本步执行后，哪些稳定字段要带回外层 | `一步治理返回字段映射` |

### 5.2 步骤标识

`步骤标识` 回答的是：`当前是谁的哪一步，在什么域里，以什么能力位承接。`

最少必须写：

- `步骤主键`
- `所属任务主键`
- `所属域`
- `当前方法需求位`
- `步骤语义`

补充固定：

1. `所属域` 当前应与 `枚举_任务管理功能域` 对齐。
2. `当前方法需求位` 当前应与 `枚举_任务管理方法需求位` 对齐。
3. `当前方法需求位` 只是能力标签；更细的历史桥接解释，按归档 [1195_任务管理方法需求位与因果到方法映射规范.md](D:\鱼巢\规范\归档\1195_任务管理方法需求位与因果到方法映射规范.md) 与 [1196_任务管理方法需求位最小状态动态因果清单.md](D:\鱼巢\规范\归档\1196_任务管理方法需求位最小状态动态因果清单.md) 理解，不重新升格为当前 `P0` 外层。

### 5.3 目标特征状态

`目标特征状态` 回答的是：`这个步骤到底想把谁的哪个特征推到什么状态。`

最少必须写：

- `目标主体`
- `目标特征`
- `目标状态`
- `当前状态摘要`
- `状态差距摘要`

典型例子：

- 让 `任务_当前步骤` 进入 `就绪`
- 让 `服务对象_可用性` 进入 `可用`
- 让 `任务_当前方法` 绑定到某个方法首节点
- 让 `依赖满足` 变成 `true`

### 5.4 前置状态集

`前置状态集` 回答的是：`要达到目标状态，最低还要哪些状态先成立。`

每个前置状态项最少必须写：

- `前置状态名`
- `前置状态来源层`
- `是否硬前置`
- `是否可部分满足`
- `当前是否满足`
- `缺口摘要`

`前置状态来源层` 当前建议固定为：

- `任务状态`
- `管理派生产出`
- `二次特征`
- `外部确认状态`
- `依赖结果`
- `场景只读事实`

补充固定：

1. `前置状态集` 是步骤进入动作前的最小资格面。
2. [1214_任务管理上下文最小字段表规范.md](D:\鱼巢\规范\1214_任务管理上下文最小字段表规范.md) 中的 `最近根层重判结果 / 当前缺口类型 / 当前下一步去向 / 输入条件主键 / 治理态型 / 结构完备度 / 方法可用度 / 风险门控 / 当前方法需求位` 等，都是前置判断的重要原料。
3. `前置状态不满足` 时，不得直接跳过检查进入执行。

### 5.5 动作集

`动作集` 回答的是：`当前前置状态满足后，可以用哪些路径把目标状态推进出来。`

最少必须写：

- `候选动作集`
- `主动作`
- `备用动作`
- `当前选中方法首节点`
- `备用方法首节点列表（按需）`
- `该动作是否需要外部确认`
- `是否允许兜底`

补充固定：

1. `动作集` 可以是动作、方法枝、方法组合片段或合法模板，不强迫一开始就只有唯一动作。
2. `主动作` 与 `备用动作` 只表达当前优先顺序，不表达最终裁决。
3. `动作集` 必须服从当前 `方法需求位` 的能力边界，不得把“目标结果”直接当成“动作本体”。
4. 若候选方法对应不同前置条件、不同结果链或不同收口去向，必须先拆成并列步骤，而不是继续留在同一步骤的方法集合里。
5. 只有目标、条件面与结果链基本一致时，替代方法才允许作为 `备用方法首节点列表` 留在同一步骤内。

### 5.6 达成判据

`达成判据` 回答的是：`怎么知道这个步骤真的达成了。`

最少必须写：

- `主达成判据`
- `次达成判据`
- `结果确认方式`
- `是否必须等待外部确认`
- `是否允许仅记为候选成功`

`结果确认方式` 当前建议固定为：

- `内部状态命中`
- `结果节点命中`
- `二次特征判定`
- `外部确认回流`

补充固定：

1. 达成判据优先看 `目标特征状态是否成立`，而不是“动作是否调用过”。
2. 若只命中内部候选成功、尚未拿到外部确认，也只能记为 `本步达成候选`。
3. 最终是否可继续主链，仍必须回到外层重判，并按 [1213_任务管理一步治理结果最小返回面规范.md](D:\鱼巢\规范\1213_任务管理一步治理结果最小返回面规范.md) 收口。
4. 任务是否正式收口，必须看头结点完成判据是否已成立，不得要求当前结果节点与头结点目标字段字面相同。

### 5.7 步骤分流与失败去向

`步骤分流与失败去向` 回答的是：`当前步骤没达成时，下一步往哪走。`

顶层当前建议先固定为 `4` 类：

- `本地继续推进`
- `局部等待`
- `上报派生需求`
- `失败或收束或回退`

若实现层需要更细分，当前可在这 `4` 类之下继续挂接：

- `切备用动作`
- `回筹办`
- `转纠偏`
- `转收束`

必须固定：

1. `步骤分流` 不是单纯错误码，而是本步的治理分流器。
2. 一步内部若未达成，顶层必须先落到至少一个正式分流类型。
3. `上报派生需求` 表达的是本地无法满足关键前置状态，需要外层决定是否创建新任务入口。
4. `本地继续推进` 表达的是目标状态尚未命中，但当前仍有本地动作路径可继续尝试，不要求立刻把本步打成失败。

### 5.8 结果写回锚点

`结果写回锚点` 回答的是：`本步的执行事实应写回哪里。`

最少必须写：

- `结果状态写回锚点`
- `结构推进写回锚点`
- `动作动态写回锚点`
- `因果实例写回锚点`
- `外部依赖引用`

补充固定：

1. `外部依赖引用` 只表达“已被外层批准并回挂”的依赖事实，不等于本步骤直接创建了子任务。
2. `结构推进写回锚点` 只写局部结构推进事实，不直接改写正式需求树。
3. `因果实例写回锚点` 只承载本轮因果证据，不重新升格历史成长链为当前 `P0` 主驱动。
4. 步骤节点建立后，应立即创建初始化结果节点，并把结果写回锚点对准该结果节点或其可回填承接面。
5. 初始化结果节点时，必须先根据 `步骤目标状态` 与 `当前状态` 生成 `结果方向二次特征`；它是结果特征状态的二次特征，不是额外的自由文本说明。
6. `结果方向二次特征` 生成后，后续 `是否继续 / 是否达标 / 是否冲突 / 是否升阶` 必须交给 `result_primitives / conflict_primitives / escalation_primitives` 或同级正式原语解释，不得直接裸判这条二次特征。

### 5.9 一步治理返回映射

`一步治理返回映射` 回答的是：`本步执行完后，要把哪些稳定字段带回外层。`

当前应直接对齐 [1213_任务管理一步治理结果最小返回面规范.md](D:\鱼巢\规范\1213_任务管理一步治理结果最小返回面规范.md)，最少包括：

- `当前功能域判定结果`
- `当前缺口类型`
- `当前下一步去向候选`
- `当前治理状态迁移`
- `当前步骤指针镜像`
- `当前方法首节点指针镜像`
- `最近结果节点指针镜像`
- `最近总控结果`
- `最近反馈类型`
- `最近反馈摘要`

补充固定：

1. `一步治理返回映射` 是步骤结构的一部分，不是执行结束后临时想到再补的摘要。
2. `返回映射` 只负责把稳定字段带回外层，不复制整包局部运行态。
3. 即使步骤本轮只进入等待或只上浮派生需求，也仍应产生稳定返回映射。


## 6. 结构草案

以下草案用于固定字段分组与命名方向，未要求与当前代码逐字同名，但语义必须对齐：

```cpp
enum class 枚举_前置状态来源层 : std::uint8_t {
    未定义 = 0,
    任务状态 = 1,
    管理派生产出 = 2,
    二次特征 = 3,
    外部确认状态 = 4,
    依赖结果 = 5,
```

## 7. 与“推进一步”的关系

有了这套结构后，`推进一步()` 当前必须按下列顺序读步骤：

1. 读取 `目标特征状态`
2. 检查 `前置状态集`
3. 确保当前步骤已具备初始化结果节点，且已按 `步骤目标状态 -> 当前状态` 生成 `结果方向二次特征`
4. 若满足，选择 `动作集`
5. 执行动作或方法
6. 用 `达成判据` 检查本步是否成立
7. 若不成立，先按 `本地继续推进 / 局部等待 / 上报派生需求 / 失败或收束或回退` 四类分流
8. 把结果写回到当前结果节点或新生成的实际结果节点，并生成 `一步治理结果`

因此必须固定：

1. `推进一步` 不是在黑盒里自由发挥，而是在消费步骤结构。
2. `步骤结构` 是 `推进一步` 的最小可解释输入壳。
3. `推进一步` 的自由度，只允许存在于 `动作选择 / 参数调整 / 模板替换 / 经验增强`，不允许越过步骤结构改写边界。


## 8. 三类样板步骤

当前建议先用以下 `3` 类步骤做样板：

| 样板步骤 | 典型目标状态 | 典型前置状态 | 典型动作 | 典型失败去向 |
| --- | --- | --- | --- | --- |
| `补结构步骤` | `任务_当前步骤` 进入 `可承接` 或 `就绪` | `结构完备度不足`、`方法挂点缺失`、`依赖关系待补` | `补步骤`、`补方法挂点`、`补局部结构` | `回筹办 / 派生需求 / 局部等待` |
| `分解补条件步骤` | `依赖满足` 进入 `true` 或 `条件缺口已外送` | `关键前置状态无法本地满足`、`分解方向已明确` | `上浮派生需求`、`回挂依赖引用`、`切换到补条件路径` | `派生需求 / 局部等待 / 转纠偏` |
| `求证判定步骤` | `完成判据` 进入 `成立` 或 `不成立已明确` | `结果已回流`、`检查点可读`、`判据已明确` | `读取结果`、`命中判据`、`形成候选成功或失败` | `转收束 / 转纠偏 / 局部等待` |


## 9. 严格禁止的混层

以下做法必须禁止：

1. 把 `动作` 直接当成 `步骤目标`。
2. 把 `当前方法需求位` 直接当成 `唯一具体方法`。
3. 在步骤内部直接创建正式子任务，而不是只记录 `外部依赖引用` 并上浮 `派生需求消息`。
4. 把 `本步达成` 直接写成 `需求已满足` 或 `主链已结束`。
5. 步骤执行完后不生成 `一步治理返回映射`，只留下局部日志或临时状态。

当前阶段统一按以下口径落地：

1. `1311` 负责冻结步骤的正式字段结构。
