# 0325 经验项后果最小回写治理面规范

## 1. 目标

本文用于把 `经验项后果` 最少应回写到哪些治理面单独固定下来。

本文要回答的是：

- `经验项后果` 生成后，最少要回写到哪些治理面
- 哪些治理面属于正式稳定回写面
- 哪些对象不能被经验项后果直接改写

## 3. 核心结论

必须固定以下结论：

1. `经验项后果` 的作用，不是直接改写双值，也不是直接形成禁止项，而是回写到治理面。
2. 当前最小稳定回写治理面应固定为：
   - `学习压力`
   - `重试压力`
   - `风险记忆`
   - `方法谨慎度候选`
   - `方法可靠度候选证据`
   - `更保守治理路径建议`
3. `经验项后果` 可以同时回写多个治理面，但不得越权直接改写：
   - `安全值`
   - `服务值`
   - `根层禁止项`
   - `最终裁决`
   - `方法存在`
4. `经验项后果` 回写的是后续治理材料，不是立即生效的全部结论。
5. `方法可靠度` 若受经验项后果影响，当前也应优先理解为“形成可靠度候选证据，并通过谨慎度起作用”，不得直接把方法存在面抹掉。
6. 这些治理面默认属于 `临时记忆`，应随时间、使用情况和条件变化衰减，而不是永久真值。

一句话收口：

`经验项后果最少要回写到学习、重试、风险与方法谨慎治理面；它影响后续怎么更谨慎地治理，不直接越权改写双值、禁止项或最终裁决。`


## 4. 最小回写原则

必须固定：

1. `经验项后果` 先是治理材料，再是治理输入。
2. 它的最小作用，是让下一轮治理更知道：
   - 哪类方法更该谨慎
   - 哪类场景更该观察
   - 哪类路径更该重试或转学习
3. 它不是“立刻做最终决策”的捷径。
4. 治理面的更细临时记忆和衰减规则，按 [0328_治理面临时记忆与衰减规范.md](D:\鱼巢\规范\0328_治理面临时记忆与衰减规范.md) 执行。


## 5. 最小回写治理面

### 5.1 学习压力

作用：

- 表达当前补方法、补条件、补解释、补边界的压力是否上升

典型来源：

- 无推进
- 反向
- 普通失败
- 方法缺口明显

必须固定：

1. `经验项后果` 可以提高学习压力。
2. 学习压力提高，不等于立刻跳过主循环直接学习。


### 5.2 重试压力

作用：

- 表达当前对象是否仍需再次尝试、再次验证或有限重试

典型来源：

- 一次失败但尚未证明路径完全失效
- 存在波动、偶发失败、待观察情形

必须固定：

1. `经验项后果` 可以提高或维持重试压力。
2. 重试压力不等于无限重试授权。


### 5.3 风险记忆

作用：

- 记录“某路径在某类场景下曾产生负向实践结果”的治理记忆

典型来源：

- 反向结果
- 异常
- 高风险失败
- 明显不稳定的实践结果

必须固定：

1. 风险记忆属于经验治理面，不直接等于根层禁止项。
2. 风险记忆可作为后续更保守路径选择的依据。


### 5.4 方法谨慎度候选

作用：

- 表达“该方法后续是否应更小步、更密检查、更快回退”

典型来源：

- 连续失败
- 结果漂移
- 无推进
- 反向

必须固定：

1. `经验项后果` 可以回写到 `方法谨慎度候选`。
2. 更细的“可靠度只调谨慎度，不直接封门”，按 [0990_方法可靠度与谨慎度规则.md](D:\鱼巢\规范\0990_方法可靠度与谨慎度规则.md) 执行。


### 5.5 方法可靠度候选证据

作用：

- 给后续可靠度更新提供负向候选证据

典型来源：

- 连续失败
- 结果不稳定
- 明显漂移
- 高风险异常

必须固定：

1. 这里回写的是 `候选证据`，不是立刻完成全部可靠度定稿。
2. 可靠度下降也不直接否定“方法存在”。


### 5.6 更保守治理路径建议

作用：

- 为下一轮主循环或任务管理任务提供更保守的治理候选

典型表现：

- 优先观察
- 优先补条件
- 优先转学习
- 优先回到重筹办
- 提高收束倾向

必须固定：

1. 这是 `建议面`，不是最终裁决面。
2. 它只影响下一轮更倾向怎么走，不直接代替外层裁决。


## 6. 最小回写面总表

| 回写治理面 | 作用 | 当前状态 |
| --- | --- | --- |
| `学习压力` | 提升补缺需求 | 正式稳定 |
| `重试压力` | 约束有限重试与再次验证 | 正式稳定 |
| `风险记忆` | 沉淀负向实践记忆 | 正式稳定 |
| `方法谨慎度候选` | 让后续尝试更谨慎 | 正式稳定 |
| `方法可靠度候选证据` | 给可靠度更新提供负向证据 | 正式稳定 |
| `更保守治理路径建议` | 为下一轮治理提供更保守候选 | 正式稳定 |


## 7. 当前不允许直接回写的面

经验项后果当前不得直接回写到以下对象：

1. `安全值`
2. `服务值`
3. `根层禁止项`
4. `最终裁决`
5. `方法存在`
6. `任务完成真值`

必须固定：

1. 这些对象若要变化，必须走它们各自的正式规则链。
2. 经验项后果只能提供治理材料，不能直接越权改写真值层。


## 8. 与主循环的关系

主循环当前至少必须承担：

```text
实践结果回流
-> 生成经验项后果
-> 将经验项后果回写到最小治理面
-> 再进入下一轮根层重判与治理上下文装配
```

必须固定：

1. 回写治理面属于主循环统一动作。
2. 执行线程、学习线程不得各自直接写成正式治理后果真值。


## 9. 与方法谨慎度和可靠度的关系

经验项后果与方法治理面的关系当前应固定为：

1. 先形成 `方法谨慎度候选`
2. 再形成 `方法可靠度候选证据`
3. 后续按 [0990_方法可靠度与谨慎度规则.md](D:\鱼巢\规范\0990_方法可靠度与谨慎度规则.md) 消费这些材料

必须固定：

1. 不得把一次失败直接解释成“方法不存在”。
2. 不得把经验项后果直接解释成“永久禁用该方法”。


## 10. 严格禁止的混层

以下做法必须禁止：

1. `经验项后果 -> 直接扣减安全值`
2. `经验项后果 -> 直接扣减服务值`
3. `经验项后果 -> 直接生成根层禁止项`
4. `经验项后果 -> 直接生成最终裁决`
5. `经验项后果 -> 直接删除方法存在面`
6. `经验项后果 -> 直接宣布任务永远失败`
