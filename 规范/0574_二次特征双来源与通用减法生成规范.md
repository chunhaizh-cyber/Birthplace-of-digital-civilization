# 0574 二次特征双来源与通用减法生成规范

## 1. 目标

本文用于把 `1次 -> 2次` 这一步的正式来源与统一生成方法单独收口，重点回答以下问题：

- 二次特征到底有哪两个正式来源
- `状态信息来源` 的二次特征应如何生成
- `动态信息来源` 的二次特征为什么更复杂，但仍必须回到减法
- 同特征类型动态、涉及时间的动态、多特征类型动态，分别应如何落地
- 为什么这些情况都可以统一归到一个 `通用减法生成方法`

本文只讨论 `严格2次特征的来源与生成路径`，不单独讨论自然语言表达、UI 展示和 `2次以后` 的递推结构。

## 3. 核心结论

必须固定以下 13 条：

1. 本文所说的两个正式来源，主要用于约束 `1次 -> 2次` 这一步。
2. 严格 `2次特征` 只有两个正式来源：`状态信息` 与 `动态信息`。
3. `状态信息来源` 的二次特征，核心母式就是两个状态值直接相减。
4. `动态信息来源` 的二次特征，虽然组织形态更复杂，但算术真身仍必须回到减法。
5. `1次 -> 2次` 时必须已知该一次特征的 `比较方法`，并按固定比较方法取出可减基值。
6. 一旦生成出正式 `2次值`，它应统一收口为 `int`。
7. `3次` 及以上不再新增来源类型，而是直接继承下层来源链。
8. `3次` 及以上的正式算术，不再回到原始一次特征比较，而是直接对同阶 `int` 值相减。
9. 同特征类型的动态变化，应直接复用状态来源的减法方法。
10. 任何涉及时间的动态描述，都必须建立在 `同样时间段` 或 `对齐时间窗` 之上。
11. 时间在二次特征里只承担 `对齐窗口 / 来源时间段 / 过程边界` 作用，不承担除法分母作用。
12. 状态来源与动态来源虽有两条入口，但最终都应汇到同一套 `通用减法生成方法`。
13. 若某结果无法回到这套通用减法方法，就不应继续叫正式高次特征。

一句话收口：

`状态信息和动态信息这两个正式来源，主要用于把1次特征生成成严格2次特征；一旦进入2次层，正式数值统一收口为int，3次及以上继续直接相减递推。`


## 4. 两个正式来源

### 4.1 状态信息来源

状态信息来源指的是：

- 当前状态与目标状态
- 起始状态与结果状态
- 左对象状态与右对象状态
- 同一场景下两个可比较状态点

它的正式母式固定为：

```text
左值 = 状态值A
右值 = 状态值B
差值 = 右值 - 左值
方向区间 = 差值落入 正区间 / 零区间 / 负区间
```

必须固定：

1. 状态来源是二次特征里最基本、最直接的来源。
2. 只要两个状态值可比较，就优先用这套方法。
3. 状态来源生成的二次特征，应优先保留 `左值 / 右值 / 差值 / 方向区间`。

### 4.2 动态信息来源

动态信息来源指的是：

- 一个合法动态中的前后状态
- 同一动态内的相邻分步
- 同样时间段内的两个变化片段
- 同一动态里多个特征类型的联动变化

必须固定：

1. 动态来源不是一种新的算术类型，而是一种更复杂的 `组织来源`。
2. 动态来源最终仍必须还原成一个或多个 `可比较的前后值对`。
3. 动态来源的复杂性来自 `时间、顺序、路径、多特征联动`，不是来自更高阶算术。
4. 动态来源内部若需要统一母式函数、阈值过滤、路径优先、置信度继承与异常时间处理，统一按 [0576_动态二次特征统一生成规范.md](D:\鱼巢\规范\0576_动态二次特征统一生成规范.md) 执行。


## 5. 动态来源的三类正式生成路径

### 5.1 同特征类型变化

当动态里关注的是 `同一个特征类型` 的前后变化时，必须固定：

1. 直接抽取该特征在 `起点` 与 `终点` 的状态值。
2. 复用状态来源的生成法：

```text
ΔF = F(后) - F(前)
```

3. 再由 `ΔF` 生成：
   - `变化量`
   - `变化方向`
   - `区间归属`

这类情况虽然来源于动态，但生成方法仍与状态来源完全同构。

### 5.2 涉及时间的变化

当动态信息里涉及 `时间` 时，必须固定：

1. 时间只用于确定 `同样时间段`、`对齐时间窗`、`分步边界`。
2. 不允许在二次特征层直接做 `差值 ÷ 时间`。
3. 合法做法只有两类：

第一类：`单段描述`

```text
在时间段 T 内：
左值 = F(T.起)
右值 = F(T.止)
差值 = 右值 - 左值
```

第二类：`同窗比较`

```text
窗口 W1 与 W2 时长相同
ΔF(W1) = F(W1.止) - F(W1.起)
ΔF(W2) = F(W2.止) - F(W2.起)
窗口差值 = ΔF(W2) - ΔF(W1)
```

必须固定：

- 时间在这里提供的是 `同窗可比性`。
- 二次特征回答的是“在这个时间段里怎么变了”或“两个同样时间段里哪个变化更多”。
- 若进一步要求“每单位时间变化多少”，就已经越出正式高次特征链，应改由派生指标承接。

### 5.3 多特征类型前后关系

当同一动态里同时存在多个特征类型时，可以按需要生成不同特征类型之间的前后关系。

必须固定：

1. 先分别对每个特征类型做自己的差值：

```text
ΔA = A(后) - A(前)
ΔB = B(后) - B(前)
```

2. 再根据业务需要生成关系型二次特征，例如：
   - `A 上升时 B 也上升`
   - `A 上升时 B 下降`
   - `在同一时间段内，A 的变化先于 B`
   - `在 ΔA 固定时，B 的变化量为 ΔB`

3. 若要把某特征类型作为锚点，也只能固定它的 `差值` 或 `差值区间`，不得把它变成乘除系数。

一句话收口：

`多特征动态关系不是直接算综合分，而是先分别求出各特征差值，再按需要组织它们之间的前后关系。`

### 5.4 与3次及以上的衔接

当严格 `2次特征` 已经生成完成后，后续 `3次 / 4次 / ... / n次` 不再重新讨论“来源类型”，而统一按 [0575_二次特征泛称与n次特征递推生成规范.md](D:\鱼巢\规范\0575_二次特征泛称与n次特征递推生成规范.md) 递推：

```text
3次值 = 2次右值 - 2次左值
4次值 = 3次右值 - 3次左值
...
```

必须固定：

1. `0574` 负责回答 `1次怎么进2次`。
2. `0575` 负责回答 `2次以后怎么继续往上递推`。
3. 两者不得混写成两套互相冲突的高次特征母式。


## 6. 通用减法生成方法

必须固定一套统一口径：

```text
输入 = 状态信息或动态信息
-> 标准化为可比较单元
-> 为目标特征类型取前值与后值
-> 差值 = 后值 - 前值
-> 生成方向区间与关系描述
-> 写回二次特征
```

推荐把通用方法理解为：

```cpp
生成二次特征(来源 S, 目标特征集 F, 生成配置 C):
    单元集 = 标准化(S)
    for 单元 in 单元集:
        for 特征类型 f in F:
            左值 = 取前值(单元, f)
            右值 = 取后值(单元, f)
            if 不可比较:
                continue
            Δf = 右值 - 左值
            输出 += 生成同特征二次特征(f, 左值, 右值, Δf)

        if C.需要多特征关系:
            for 每对(锚点特征 a, 关联特征 b):
                Δa = 已生成差值(a)
                Δb = 已生成差值(b)
                输出 += 生成跨特征关系(a, Δa, b, Δb)

    return 输出
```

这里必须强调：

1. 通用性不等于“一次生成所有可能特征”。
2. 通用性指的是“所有来源都走标准化，再回到减法核”。
3. 状态来源和动态来源的差别，在 `标准化步骤`，不在 `算术真身`。


## 7. 字段建议

若二次特征要稳定承接这两类来源，建议至少具备以下字段：

- `来源类型`：状态 / 动态
- `比较左值`
- `比较右值`
- `差值`
- `方向区间`
- `来源时间段`
- `主特征类型`
- `锚点特征类型`
- `关联特征类型`

必须固定：

1. `来源类型` 负责回答“这条二次特征是从状态还是动态来的”。
2. `主特征类型` 负责回答“当前主要在看哪个特征”。
3. `锚点特征类型 / 关联特征类型` 只在多特征关系场景下启用。


## 8. 审计清单

后续审计二次特征实现时，至少检查以下 10 项：

1. 是否明确区分了 `状态来源` 和 `动态来源`。
2. 状态来源是否直接走 `相减`。
3. 动态来源是否先标准化成可比较单元。
4. 同特征动态是否复用了状态来源的减法方法。
5. 涉及时间时，是否使用了 `同样时间段 / 对齐时间窗`。
6. 涉及时间时，是否错误出现了 `差值 ÷ 时间`。
7. 多特征关系是否先分别得到 `ΔA / ΔB`。
8. 多特征关系是否错误直接进入了乘除或加权。
9. 是否存在“动态来源很复杂，所以另外发明一套非减法二次特征”的情况。
10. 是否可以把最终结果回溯到明确的 `前值 / 后值 / 差值`。


## 9. 最终收口

必须固定以下最终收口句：

`二次特征虽有状态信息与动态信息两种来源，但它们最终都必须汇到同一套减法核；动态的复杂性属于来源组织复杂，不属于算术真身复杂。`
