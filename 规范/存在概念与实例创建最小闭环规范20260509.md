# 存在概念与实例创建最小闭环规范 20260509 修订版

## 1. 总原则

存在概念先不做复杂本体系统。它只有两个最小用途：

```text
1. 自然语言理解：
   名词、专名、抽象名词等语素，指向一个“概念存在”。

2. 新建实例存在：
   根据一个或多个“概念存在”创建实例存在，并把概念中的基础记忆扩充到实例上。
```

核心关系固定为：

```text
存在概念 = 用于解释词义、辅助创建实例的存在模板
实例存在 = 实际世界、场景、内部世界中的具体存在
```

存在只有一种结构：

```cpp
存在节点主信息类
```

不新增：

```text
抽象存在主信息类
枚举_存在形态
复杂的概念归属记录
置信度
来源
状态
命中特征
冲突特征
```

能力、需求、方法、用途、行为模板等，不进入概念归属关系。它们后续若需要使用，只作为“新建实例时的记忆扩充内容”处理。

## 2. 最简结构

`存在节点主信息类` 使用 `概念集` 表达一个实例对应的多个概念存在：

```cpp
class 存在节点主信息类 : public 基础信息基类 {
public:
    // 主概念 / 默认概念 / 历史兼容字段。
    可解析引用<存在节点类> 概念模板{};

    // 新逻辑权威字段：实例可以对应多个概念存在。
    // 概念存在本身通常为空，也可以包含更上位概念。
    std::vector<可解析引用<存在节点类>> 概念集{};

    可解析引用<场景节点类> 内部世界{};

    需求节点类* 需求根节点 = nullptr;
    任务节点类* 任务根节点 = nullptr;
    方法节点类* 方法根节点 = nullptr;

    std::vector<可解析引用<因果实例节点类>> 实例因果{};
};
```

兼容规则：

```text
概念模板 = 主概念 / 默认概念 / 兼容字段
概念集 = 新逻辑权威字段
```

同步要求：

```text
1. 旧代码写入 概念模板 时，必须同步保证 概念集 包含该概念。
2. 新代码添加第一个概念时，如果 概念模板 为空，应同步写入 概念模板。
3. 读取概念归属时优先读 概念集。
4. 如果 概念集 为空但 概念模板 有效，可视为单概念集合。
5. 删除或替换 概念模板 时，不得误删 概念集 中的其他概念。
```

推荐增加轻量工具函数，而不是让业务代码直接改字段：

```cpp
bool 添加实例概念(
    存在节点类* 实例,
    存在节点类* 概念,
    bool 是否扩充记忆 = true
);

std::vector<存在节点类*> 获取有效概念集(
    const 存在节点类* 实例
);
```

## 3. 概念与实例的区分

概念存在和实例存在不靠枚举区分。

不用加：

```cpp
枚举_存在形态
```

判断规则：

```text
在“存在概念根”下面的存在节点，就是概念存在。
在“世界根 / 场景根 / 内部世界”下面的存在节点，就是实例存在。
```

示例：

```text
世界根
├─ 存在概念根
│  ├─ 飞机
│  ├─ 交通工具
│  ├─ 机场资源
│  ├─ 机位
│  └─ 航班
│
└─ 当前场景根
   ├─ B-1234
   ├─ 101机位
   └─ MU1234航班
```

概念集示例：

```text
B-1234.概念集 = [飞机, 交通工具, 机场资源]
101机位.概念集 = [机位, 机场资源]
MU1234航班.概念集 = [航班, 运行任务]
```

## 4. 唯一性规则

概念存在和实例存在的唯一性规则不同。

### 4.1 概念存在

概念存在用于自然语言和实例创建，因此应尽量唯一。

第一阶段规则：

```text
同一 存在概念根 下，名称 + 类型 相同的概念存在只保留一个。
```

因此：

```cpp
确保存在概念(...)
```

应优先查找已有概念，找不到才创建。

### 4.2 实例存在

实例存在可以同名多实例。

示例：

```text
多个场景里都可以有 “101机位”
多个内部世界里都可以有 “当前任务”
多个观察过程中都可以先生成临时实例
```

因此：

```cpp
创建存在实例(...)
```

默认创建新实例，不因为同名而自动复用。是否复用实例，由调用方根据场景、主键、来源或匹配策略决定。

## 5. 自然语言入口

语素层关系固定为：

```text
语素词性节点
  -> 对应基础信息类型
  -> 对应基础信息节点
```

对于名词性入口：

```text
“飞机” + 名词
  -> 类型：存在
  -> 节点：存在概念根 / 飞机

“机位” + 名词
  -> 类型：存在
  -> 节点：存在概念根 / 机位

“航班” + 名词
  -> 类型：存在
  -> 节点：存在概念根 / 航班
```

所以自然语言里出现“飞机”，系统先找到的是：

```text
飞机这个概念存在
```

不是某一架具体飞机。

对于专名或实例名：

```text
“B-1234” + 专有名词
  -> 类型：存在
  -> 节点：实例存在 / B-1234
```

专名是否指向实例，应由调用方在有场景或实例上下文时显式绑定；语素层不默认猜测。

## 6. 新建实例入口

当语言或程序要求：

```text
新建一架飞机
```

流程为：

```text
找到“飞机”概念存在
创建一个实例存在
把“飞机”加入实例的概念集
把“飞机”概念上的基础记忆扩充到实例
```

示例：

```text
创建 B-1234

B-1234.概念集 += 飞机
B-1234.概念集 += 交通工具
```

然后执行记忆扩充：

```text
飞机概念上的默认特征、默认需求、默认方法、默认能力
复制 / 实例化到 B-1234
```

能力等不属于概念归属关系，只属于新建实例时的记忆扩充功能。

## 7. 概念特征扩充

简化规则：

```text
新建实例时，把概念存在的特征实例化到实例存在上。
```

第一阶段可以只扩充特征，需求、任务、方法、能力以后再接入。

不做：

```text
动态继承
逻辑合并
冲突集
复杂置信度治理
```

示例：

```text
飞机
├─ 特征：位置
├─ 特征：速度
├─ 特征：航向
├─ 特征：运行状态
└─ 特征：占用机位
```

创建实例：

```text
B-1234
```

扩充后：

```text
B-1234
├─ 概念集：飞机、交通工具、机场资源
├─ 特征：位置
├─ 特征：速度
├─ 特征：航向
├─ 特征：运行状态
└─ 特征：占用机位
```

多个概念的扩充内容合并到实例上：

```text
飞机      -> 位置、速度、航向、运行状态
交通工具  -> 可移动、载体、路径
机场资源  -> 可调度、占用状态、资源状态
```

重复特征按以下优先级去重：

```text
1. 抽象特征引用相同。
2. 特征类型相同。
3. 名称 + 类型 相同。
```

简单合并规则：

```text
1. 实例已有特征优先，不被概念扩充覆盖。
2. 多个概念都提供同一特征时，先加入的概念优先。
3. 如果已有特征无值、概念特征有默认值，可以补值。
4. 不做冲突判断，只做补缺。
```

候选窗口、统计信息、观测时间等运行期痕迹不从概念复制到实例。

## 8. 指针值复制边界

概念特征里如果有 `指针句柄` 值，实例化时要区分：

```text
1. 指向概念节点、稳定字典节点、单位节点、枚举值节点的指针，可以复制。
2. 指向场景实例、临时动态、当前任务、运行期对象的指针，不应从概念复制到实例。
```

第一阶段可以采用更保守规则：

```text
概念扩充时只复制特征类型，不复制指针类当前值。
```

如果业务明确需要默认指针值，再由具体方法显式填充。

## 9. 实例在先，概念在后

允许：

```text
实例可以先独立存在。
概念可以后创建。
概念创建后，可以用于之后的新实例创建。
旧实例是否补充概念，由额外的整理 / 归类过程处理。
```

示例：

```text
先有 B-1234、B-5678、B-9999
后面总结出“飞机”概念
```

后续整理过程可以执行：

```text
B-1234.概念集 += 飞机
B-5678.概念集 += 飞机
B-9999.概念集 += 飞机
```

并可选择：

```text
按“飞机”概念补充缺失记忆
```

但这不是核心结构的一部分，只是后续整理函数。

说明：

```text
“实例在先，概念在后”是认识论原则，不要求工程上禁止预置概念字典。
工程实现可以预先创建常用概念，但概念仍只作为词义锚点和实例创建模板使用。
```

## 10. 集合表达

集合优先用特征和查询条件表达。

不要建：

```text
存在形态::集合
```

也不要默认创建：

```text
可用机位集合节点
```

例如：

```text
101机位
├─ 概念集：机位、机场资源
├─ 占用状态：空闲
└─ 可分配：是
```

查询“可用机位”：

```text
概念集 包含 机位
且 占用状态 = 空闲
且 可分配 = 是
```

如果某个集合需要被调度、命名或记录因果，例如：

```text
今日第一保障波次
```

它可以是普通存在，成员关系仍然用特征表达：

```text
B-1234飞机.所属波次 = 今日第一保障波次
加油任务A.所属波次 = 今日第一保障波次
```

## 11. 最小接口

### 11.1 确保存在概念

```cpp
存在节点类* 确保存在概念(
    const 词性节点类* 名称,
    存在节点类* 父概念 = nullptr
);
```

用途：

```text
自然语言中的名词 -> 找到或创建概念存在
```

要求：

```text
同一 存在概念根 下 名称 + 类型 唯一。
```

### 11.2 创建实例存在

```cpp
存在节点类* 创建存在实例(
    const 词性节点类* 名称,
    const std::vector<存在节点类*>& 概念集,
    场景节点类* 所属场景 = nullptr,
    bool 是否扩充记忆 = true
);
```

用途：

```text
根据一个或多个概念存在，创建实例存在。
```

要求：

```text
默认创建新实例，不按名称全局复用。
```

### 11.3 扩充实例记忆

```cpp
void 根据概念扩充实例记忆(
    存在节点类* 实例,
    const std::vector<存在节点类*>& 概念集
);
```

职责：

```text
复制 / 实例化概念特征
复制 / 实例化默认需求
复制 / 实例化默认任务
复制 / 实例化默认方法
后续再复制能力
```

第一阶段可以只复制 / 实例化概念特征。

### 11.4 给实例追加概念

```cpp
bool 添加实例概念(
    存在节点类* 实例,
    存在节点类* 概念,
    bool 是否扩充记忆 = true
);
```

用途：

```text
实例已经存在，后来发现它也属于另一个概念。
```

示例：

```text
B-1234 原来只是“飞机”
后来又被标记为“机场资源”
于是补充“机场资源”的相关特征
```

### 11.5 判断概念与实例

```cpp
bool 是存在概念(const 存在节点类* 节点) const;
bool 是存在实例(const 存在节点类* 节点) const;
```

判断依据：

```text
节点所在根树，而不是存在形态枚举。
```

## 12. 与特征值的关系

`概念集` 是实例概念归属的第一阶段权威结构。

普通对象关系仍然可以用特征表达：

```text
B-1234飞机.当前位置 = 101机位
101机位.占用对象 = B-1234飞机
B-1234飞机.所属波次 = 今日第一保障波次
```

当特征值是另一个存在或基础信息节点时，按“存在作为特征值”规范使用：

```text
特征值::指针句柄
```

概念归属不强制改写成特征多值；第一阶段以：

```text
存在节点主信息类::概念集
```

为权威入口。

如果为了统一查询需要生成：

```text
B-1234飞机.概念 = 飞机
```

这类特征节点，它们只作为：

```text
概念集 的镜像 / 索引 / 查询辅助
```

不得替代 `概念集`。

## 13. 禁止项

禁止：

```text
1. 新增 抽象存在主信息类。
2. 新增 枚举_存在形态。
3. 新增复杂概念归属记录。
4. 把能力、需求、方法、用途、行为模板塞进 概念集。
5. 把集合建成默认存在形态。
6. 在概念特征扩充第一阶段做动态继承、冲突集或复杂合并。
7. 因为同名而自动复用实例存在。
8. 把概念特征镜像当作 概念集 的替代权威结构。
```

允许：

```text
1. 保留 概念模板 作为主概念 / 默认概念 / 兼容字段。
2. 使用 概念集 表达多个概念归属。
3. 使用存在概念根区分概念存在。
4. 使用世界根 / 场景根 / 内部世界区分实例存在。
5. 新建实例时把概念特征复制 / 实例化到实例。
6. 为查询方便，从 概念集 生成概念特征镜像。
```

## 14. 验收标准

实现应满足：

```text
1. 存在只有一种结构：存在节点主信息类。
2. 不增加存在形态枚举。
3. 不增加抽象存在主信息类。
4. 概念存在和实例存在靠所在树区分。
5. 名词性语素通常指向概念存在。
6. 实例存在可以对应多个概念存在。
7. 概念存在当前用途只有两个：自然语言理解、新建实例存在。
8. 新建实例时，把概念存在的特征、需求、方法等扩充到实例。
9. 第一阶段扩充可以只做特征。
10. 集合优先用特征和查询条件表示。
11. 能力、用途等后续归入“创建实例时的记忆扩充”。
12. 概念集 是概念归属权威结构。
13. 概念特征节点如存在，只是镜像或索引。
14. 概念存在按 名称 + 类型 唯一，实例存在允许同名多实例。
```

## 15. 最终定稿

```text
概念存在先不要做成复杂本体系统。
它只是词义锚点 + 实例创建模板。
实例创建出来以后，概念里的东西通过记忆扩充变成实例自己的记忆。

概念归属第一阶段由 概念集 承载。
特征系统可以镜像概念归属，但不替代 概念集。
```
