# 0033 控制域与一次特征字典 v0.1

## 1. 目标

本文用于冻结一份可以直接落到代码字段和字典枚举的最小规范，回答四个问题：

- 当前先承认哪 6 类核心存在
- 这 6 类存在各自拥有哪些 `一次特征`
- `控制域` 应拆成哪些正式取值组
- 哪些最小 `动作语义` 允许修改哪些一次特征

本文只服务当前阶段的“小切口、强闭环”目标：

`先把存在、控制域、一次特征和最小动作语义定住，再回头收消息字段和线程判定。`

本文不负责：

- 扩张 `消息类型`
- 扩写 `自我线程` 主循环逻辑
- 把资源、外部对象、学习样本、审计记录一次性铺满
- 定义完整任务调度器、完整优先级系统或完整学习语义

## 3. 范围与硬边界

本版只先冻结 6 类核心存在：

- 自我
- 自我线程
- 任务管理任务
- 任务
- 方法
- 工作线程

必须固定：

1. 本版先不把 `资源 / 外部对象 / 学习样本 / 审计记录 / 结果回流实例` 展开成正式核心存在字典。
2. 若后续判定需要引用这些对象，只允许先作为 `外部条件输入` 或 `客体占位输入` 使用。
3. 一次特征固定解释为：`可直接记录为状态真值，不依赖额外比较即可落成状态节点的特征。`
4. 二次特征固定不得混写进一次特征字典。
5. `任务控制态` 虽不是本版新增的第 `7` 类核心存在，但必须作为 `附着在任务上的控制面状态簇` 单独列出，不得继续和 `任务` 或 `任务管理任务` 混写。


## 4. 核心结论

必须固定以下结论：

1. 自我是 `真实存在`，自我线程、任务管理任务、任务、方法、工作线程都必须视为有正式特征面的合法存在。
2. 项目内每个受治理线程都应被视为 `线程子存在`；`自我线程` 与 `工作线程` 只是两类线程子存在特化。
3. 线程子存在至少应具备三类核心标准特征：`生命周期状态 / 控制意图 / 实际运行状态`。
4. `生命周期状态` 是线程生命过程特征，不属于控制域；控制域至少拆成两组：
   - `控制意图特征`
   - `实际运行状态特征`
5. 线程子存在可按适用性继续挂接 `健康状态 / 当前预算 / 绑定任务 / 阻塞状态` 四类通用扩展特征。
6. `启动 / 暂停 / 继续 / 打断 / 收束 / 停止` 不再视为彼此平级的消息原语，而应视为 `控制域特征` 的取值。
7. `请求收束` 和 `已收束` 不得写入同一个值桶。
8. 动作语义优先回答“谁做了什么，改了谁的哪些一次特征”，消息类型只负责后续路由归类。
9. 后续任何消息字段、变化项表和路由矩阵，都必须从本文的一次特征字典和动作语义表反推，不得脱离本文另造字段世界。
10. 与任务相关的一次特征，必须按 `任务根语义 / 任务控制态 / 任务管理任务局部推进态` 三层归属。
11. `当前步骤前沿 / 当前等待原因 / 派生需求挂起状态` 不属于 `任务` 根语义；`优先级 / 队列状态 / 预算 / 派发许可` 不属于 `任务管理任务` 局部推进态。

### 5.1 自我

| 一次特征键 | 含义 | 推荐值类型 | 主写入者 | 备注 |
| --- | --- | --- | --- | --- |
| 生命周期状态 | 自我当前是否存在、健康、故障、退出边界 | 枚举 | 自我初始化链、自我线程 | 是根存在状态面 |
| 待机状态 | 自我当前是否处于待机保持 | 布尔或枚举 | 自我线程 | 是根层重判输入 |

| 服务值 | 当前服务面真值 | 数值 | 自我本体、自我线程结算 | 是根边界输入 |

| 当前主任务 | 当前承接的主任务引用 | 主键或指针 | 自我线程 | 是任务主链回填输入 |

### 5.2 线程子存在（通用标准面）

| 特征层级 | 一次特征键 | 是否必备 | 适用范围 | 备注 |
| --- | --- | --- | --- | --- |
| 核心标准 | 生命周期状态 | 是 | 全部线程子存在 | 表示线程处于哪一段生命过程 |
| 核心标准 | 控制意图 | 是 | 全部线程子存在 | 表示想让线程进入什么控制目标 |
| 核心标准 | 实际运行状态 | 是 | 全部线程子存在 | 表示线程现在事实上处于什么运行态 |
| 通用扩展 | 健康状态 | 否，按适用性 | 建议长期存在线程 | 表示线程稳定性、故障、恢复 |
| 通用扩展 | 当前预算 | 否，按适用性 | 受预算门控线程 | 表示线程当前可用预算 |
| 通用扩展 | 绑定任务 | 否，按适用性 | 承接任务的线程 | 表示路由和回流定位 |
| 通用扩展 | 阻塞状态 | 否，按适用性 | 可能阻塞的线程 | 表示门控和收束输入 |

必须固定：

1. 每个线程子存在至少具备上述 `3` 类核心标准特征，不得再少。
2. 上述 `4` 类通用扩展特征按适用性挂接，不要求所有线程挂满。
3. 线程类型专属特征可继续追加，例如 `运行模式 / 邮箱深度 / 心跳健康`。
4. 当前阶段不设置线程特征理论上限，但新增特征不得混入 `任务 / 结果 / 场景 / 审计` 等非线程本体语义。
5. 时间、决策者、原因、旧状态、新状态属于 `状态变化记录` 或 `动作动态` 字段，不直接冒充线程本体标准特征。

### 5.3 自我线程

`自我线程` 是在线程子存在通用标准面之上追加的线程特化。

| 一次特征键 | 含义 | 推荐值类型 | 主写入者 | 备注 |
| --- | --- | --- | --- | --- |
| 生命周期状态 | 线程是否未启动、运行中、停止中、已停止、故障 | 枚举 | 自我线程 | 是线程壳状态 |
| 运行模式 | 普通推进、待机保持、收束优先等运行模式 | 枚举 | 自我线程 | 不等于控制意图 |
| 控制意图 | 想让线程进入什么控制目标 | 枚举 | 自我线程或宿主控制面 | 与实际运行状态分离 |
| 实际运行状态 | 线程现在实际上处于什么运行状态 | 枚举 | 自我线程 | 是事实状态 |
| 邮箱深度 | 当前 mailbox 消息数 | 数值 | 自我线程 | 是运行观测值 |
| 健康状态 | 线程健康、故障、恢复中 | 枚举 | 自我线程 | 是线程稳定性真值 |
| 当前预算 | 当前线程可用预算 | 数值 | 自我线程 | 是上位控制输入 |

### 5.8 工作线程

`工作线程` 是在线程子存在通用标准面之上追加的执行载体特化。

| 一次特征键 | 含义 | 推荐值类型 | 主写入者 | 备注 |
| --- | --- | --- | --- | --- |
| 生命周期状态 | 线程是否已启动、已停止、故障等 | 枚举 | 工作线程、自我线程 | 是执行载体状态面 |
| 控制意图 | 工作线程当前收到的控制请求 | 枚举 | 自我线程 | 是控制链入口 |
| 实际运行状态 | 工作线程现在事实状态 | 枚举 | 工作线程回报、自我线程确认 | 与控制意图分离 |
| 健康状态 | 工作线程当前健康、故障、恢复中 | 枚举 | 工作线程、自我线程 | 是统一健康真值面 |
| 绑定任务 | 当前绑定的宿主任务 | 主键或指针 | 自我线程、工作线程 | 是路由和回流定位键 |
| 心跳健康 | 心跳是否正常 | 枚举或数值 | 工作线程、自我线程 | 是健康输入 |
| 阻塞状态 | 当前是否阻塞以及阻塞级别 | 枚举 | 工作线程回报 | 是门控和收束输入 |

| 自我线程 | 下发收束指令 | 工作线程 | 控制意图 | 否 | 是 | 是 | 是 |
| 任务 | 提交结果 | 任务、任务管理任务 | 结果状态、局部完成度、实际运行状态 | 是 | 是 | 是 | 是 |
| 任务管理任务 | 上报派生需求 | 任务 | 当前等待原因、派生需求挂起状态 | 是 | 是 | 是 | 是 |
| 自我线程或确认链 | 确认条件满足 | 任务管理任务 | 条件满足状态、依赖等待状态 | 是 | 是 | 是 | 是 |
| 任务管理任务 | 完成步骤推进 | 任务管理任务、任务 | 当前步骤前沿、前序完成状态、局部完成度 | 是 | 否 | 是 | 是 |

必须固定：

1. 动作语义优先由 `动作主体 + 动作语义键 + 被修改的一次特征` 定义。
2. 同一动作允许改多个一次特征。
3. 动作先定语义，再由消息层决定如何路由。
4. 若某动作无法明确指出“改了谁的哪些一次特征”，则该动作不得进入正式协议层。


## 8. 推荐落码约束

为保证后续能直接落代码，本版推荐至少形成以下字典或枚举：

- `枚举_核心存在类型`
- `枚举_一次特征键`
- `枚举_控制意图值`
- `枚举_实际运行状态值`
- `枚举_最小动作语义键`

必须固定：

1. 一次特征键命名应优先贴合状态节点写回面，不应混入裁判结论词。
2. `可执行 / 依赖满足 / 有效优先级` 这类判断词不得出现在一次特征键字典中。
3. 动作语义键应优先采用“动词 + 对象”结构，例如 `下发收束指令 / 提交结果 / 确认条件满足`。


## 9. 与 0034 的接口面

`0033` 输出给 `0034` 的最小一次特征输入固定为：

- `任务.结果状态`
- `任务控制态.是否允许继续派发`
- `任务管理任务.当前步骤前沿`
- `任务管理任务.当前等待原因`
- `任务管理任务.前序完成状态`
- `任务管理任务.条件满足状态`
- `任务管理任务.依赖等待状态`
- `工作线程.控制意图`
- `工作线程.实际运行状态`

一句话收口：

`0033` 先把“谁有什么真值特征”钉死，并通过 `0039` 把任务相关特征分到 `任务 / 任务控制态 / 任务管理任务` 三层；`0034` 再在这些真值特征上做最小判断层。
