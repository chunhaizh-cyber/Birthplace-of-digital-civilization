# 2060 动态信息自然语言表达规范

## 1. 目标

本文用于固定当前工程中：

- `动态节点主信息类` 如何转换为自然语言
- 为什么 `动态信息` 在语言层应同时支持 `动词` 与 `动作短语` 两类输出
- `动态信息` 与 `状态信息`、`因果信息` 在表达层如何分工
- 动态信息在“事件事实”“过程变化”“带场景动作”几种场景下应如何落句

本文重点回答：

- 动态信息到底在语言上描述什么
- 动词与动作短语分别承担什么职责
- 动态节点最少要提供哪些语言槽位
- 输入场景、输出场景、动作相位、执行结果如何进入动态表达

本文服从以下硬边界：

1. `动态信息` 仍是真值层中的动作、事件或变化事实，不是语言层主观修辞。
2. 自然语言只是显示层和解释层，不得反向改写动态真值。
3. 问“刚刚发生了什么 / 正在怎样变化”时，优先使用 `动态信息`。
4. 问“现在是什么样”时，必须回到 `状态信息`，不得用动态句偷代当前状态。


## 3. 核心口径

必须固定：

1. `动态信息` 描述的是：`某主体在某时间执行了什么动作、发生了什么变化、从什么状态走向什么状态`。
2. 在自然语言中，`动态信息` 应同时支持两类输出：
   - `动词`
   - `动作短语`
3. `动词` 负责给出事件核心，回答“做了什么”。
4. `动作短语` 负责给出更完整的动作表面，回答“具体做了什么”。

一句话固定：

`动态信息是真值层中的事件与变化事实，在语言层可同时落成动词核和动作短语，其中动词给出动作核心，动作短语给出完整事件表述。`


## 4. 依据的字段面

当前 `动态节点主信息类` 可直接提供的关键字段包括：

- `初始状态`
- `结果状态`
- `动态主体`
- `动态特征`
- `时间`
- `来源类型`
- `来源动作`
- `来源动作名`
- `来源动作相位`
- `来源输入场景`
- `来源输出场景`
- `来源执行成功`
- `来源错误码`
- `动态路径签名`

当前统一理解为：

- `动态主体`：回答“是谁在动”
- `来源动作 / 来源动作名`：回答“做了什么动作”
- `初始状态 / 结果状态`：回答“从哪变到哪”
- `时间`：回答“何时发生”
- `来源输入场景 / 来源输出场景`：回答“在哪发生、输出到哪”
- `来源动作相位`：回答“开始、进行中、完成、失败”这类阶段
- `来源执行成功 / 来源错误码`：回答“动作结果如何”


## 5. 动词与动作短语的分工

### 5.1 动词

`动词` 负责回答：

- 做了什么
- 发生了什么
- 变化方向是什么

它是动态信息的谓语核心，常见形式包括：

- `读取`
- `推进`
- `回升`
- `回落`
- `衰减`
- `切换`
- `观察`

必须固定：

1. 动词优先用来给出事件核。
2. 动词可用于摘要、标题、统计项、时间线短标签。
3. 若当前动作无法稳定收敛到单个动词，可退回到动作短语，不得硬切出错误动词。


### 5.2 动作短语

`动作短语` 负责回答：

- 具体做了什么
- 动作包含了哪些补语、对象、方向或结果补充

它是动态信息的完整表述面，常见形式包括：

- `停止当前试探并进入保守态`
- `按双值重算服务衰减步长`
- `按服务值重算安全增加步长`
- `任务管理任务推进当前步骤`

必须固定：

1. 动作短语不是另一条真值，而是同一动态事实的更完整表述。
2. 动作短语优先用于控制面板正文、日志正文、因果动作面。
3. 当动词过于抽象、无法独立表达具体动作时，应优先显示动作短语。


### 5.3 二者关系

二者关系必须固定为：

- `动词 = 动态核心`
- `动作短语 = 动态展开`

同一条动态信息可以同时拥有：

- 一个动词核
- 一个动作短语面

但不要求必须都存在。若只拿到动作短语，允许直接显示动作短语；若只拿到稳定动词，也允许只显示动词核。


## 6. 最小语义骨架

动态信息最小骨架固定为：

`动态主体 + 动作核心 + 动作展开 + 时间 + 状态变化`

其中：

- `动态主体` 必须回答“谁在动”
- `动作核心` 对应动词
- `动作展开` 对应动作短语
- `时间` 回答“何时发生”
- `状态变化` 回答“从哪到哪”

若缺少状态变化，可退化为：

`动态主体 + 动作短语`

若缺少主体，只能作为局部动作摘要，不得长期替代完整句。


## 7. 表达主链

动态信息的自然语言主链固定为：

`动态节点 -> 动态表达单元 -> 动词 / 动作短语 -> 句型 -> 自然语言`

其中动态表达单元最少应包含：

- `主体名称`
- `动作动词`
- `动作短语`
- `初始状态短语`
- `结果状态短语`
- `输入场景短语`
- `输出场景短语`
- `动作相位短语`
- `执行结果短语`

必须固定：

1. `动态节点` 不直接等于最终句子。
2. 语言模块先抽出动态表达单元，再决定选“动词句”还是“动作短语句”。
3. 控制面板、日志、调试页统一消费语言模块结果，不各自手拼动态句。


## 8. 句型层

### 8.1 动词型动态句

适用条件：

- 只需要给出事件核
- 用于摘要、标题、时间线列表、统计标签

推荐模板：

- `X 在执行 V`
- `X 执行了 V`
- `X 正在 V`

示例：

- `自我执行了读取`
- `任务管理任务正在推进`
- `风险安全正在回落`


### 8.2 动作短语型动态句

适用条件：

- 需要完整表达动作语义
- 动作自带补语、对象、方向或复合过程

推荐模板：

- `X 执行了 P`
- `X 正在 P`

示例：

- `自我执行了停止当前试探并进入保守态`
- `自我正在按双值重算服务衰减步长`


### 8.3 状态迁移型动态句

适用条件：

- 初始状态与结果状态都存在
- 需要明确“从哪到哪”的变化

推荐模板：

- `X 从 A 变到 B`
- `X 的 T 从 A 变为 B`

示例：

- `任务管理任务从未启动变为运行中`
- `自我的当前主任务从服务根任务变为安全根任务`


### 8.4 带场景动态句

适用条件：

- 输入场景或输出场景需要保留

推荐模板：

- `X 在 S 中执行了 P`
- `X 从 S1 进入 S2`

示例：

- `自我在自我内部世界中执行了读取核心状态`
- `信息从输入场景进入输出场景`


### 8.5 带结果动态句

适用条件：

- 需要保留动作是否成功
- 或需要保留错误信息

推荐模板：

- `X 执行了 P，并成功`
- `X 尝试 P，但失败`

示例：

- `自我执行了读取核心状态，并成功`
- `自我尝试读取输入，但失败`


## 9. 动词生成规则

### 9.1 来源优先级

动词来源推荐优先级固定为：

1. `来源动作名` 中可稳定抽出的动词核
2. `来源动作` 对应方法名中可稳定抽出的动词核
3. `初始状态 -> 结果状态` 中可归纳出的变化动词
4. 若都无法稳定得到，则不强行生成动词


### 9.2 动词的职责边界

必须固定：

1. 动词只负责给出动作核心，不负责承载全部动作语义。
2. 不能为了得到一个单字或双字动词，把完整动作短语切坏。
3. 若动作短语是稳定的、动词核不稳定，则应以动作短语为主。


## 10. 动作短语生成规则

动作短语来源推荐优先级固定为：

1. `来源动作名`
2. `来源动作` 对应方法名
3. `状态迁移句`
4. `场景 + 动词 + 结果` 组装句

必须固定：

1. 动作短语优先保留原动作语义，不随意压缩。
2. 若来源动作名本身已经是完整短语，应原样优先使用。
3. 动作短语可以承接对象、方向、补语、阶段信息。


## 11. 时间与相位规则

必须固定：

1. `当前面板` 可默认省略绝对时间，优先显示“正在 / 已完成 / 已执行”。
2. `日志 / 回放 / 审计` 场景中，可显式补入时间和相位。
3. `来源动作相位` 应优先落成：
   - `开始`
   - `进行中`
   - `完成`
   - `失败`

推荐示例：

- `自我开始执行读取核心状态`
- `自我正在执行读取核心状态`
- `自我完成了读取核心状态`


## 12. 与状态信息、因果信息的分工

### 12.1 与状态信息的关系

- `状态信息` 负责“现在什么样”
- `动态信息` 负责“刚刚怎样变化 / 正在怎样动作”

一句话固定：

`状态` 是截面，`动态` 是过程。`


### 12.2 与因果信息的关系

- `动态信息` 负责记录动作事实
- `因果信息` 负责组织“条件 + 动作 + 结果”

因此：

- 动态句不能直接冒充因果句
- 因果句可以消费动态中的动作短语或动词核


## 13. 推荐示例

### 13.1 动词型

- `任务管理任务正在推进`
- `风险安全正在回落`
- `服务值正在衰减`


### 13.2 动作短语型

- `自我执行了停止当前试探并进入保守态`
- `自我执行了按双值重算服务衰减步长`
- `自我执行了读取核心状态`


### 13.3 状态迁移型

- `任务管理任务从未启动变为运行中`
- `当前主任务从服务根任务变为安全根任务`


### 13.4 带场景与结果

- `自我在自我内部世界中执行了读取核心状态，并成功`
- `输入信息从输入场景进入输出场景`


## 14. 工程落点

当前工程中，这部分能力应归属于 `语言模块`，而不是散落在控制面板或各服务类中直接手拼。

最小接口建议固定为：

- `生成动态信息动词(...)`
- `生成动态信息动作短语(...)`
- `生成动态信息自然语言(...)`

必须固定：

1. 控制面板只消费语言模块结果，不自行拼装动态句。
2. 同一套动态语言规则应同时服务控制面板、日志与后续调试输出。
3. 当前若先实现最小命名链，也应保证动态节点至少能落成 `X 执行了 P` 或 `X 从 A 变为 B` 这两类句型。
4. 后续因果表达若需要动作面，应优先复用动态语言模块的动词核与动作短语结果。
