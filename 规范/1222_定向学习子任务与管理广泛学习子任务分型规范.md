# 1222 定向学习子任务与管理广泛学习子任务分型规范

## 1. 目标

本文用于固定以下问题：

- 学习任务是否只有一种形态
- `任务管理任务` 生成的独立学习任务应怎样定义
- `任务子任务` 生成的学习任务应怎样定义
- 两大类学习任务的父子归属、目标粒度、回流边界应如何区分

本文要收口的是 `学习子任务分型`，不是具体学习算法实现。

## 3. 核心结论

必须固定以下 12 条：

1. 学习任务后续统一只分成两大类：
   - `任务管理任务生成的独立学习任务`
   - `任务子任务生成的学习任务`
2. `任务子任务生成的学习任务` 对应当前实现里的 `定向学习子任务`。
3. `任务管理任务生成的独立学习任务` 对应当前实现里的 `任务管理广泛学习子任务`。
4. `任务子任务生成的学习任务` 的父任务是某个真实业务任务；它解决的是该父任务当前暴露出的具体缺口。
5. `任务管理任务生成的独立学习任务` 的父任务是对应的 `任务管理任务`；它解决的是某个需求位上的储备、扩选、健壮、覆盖类压力。
6. `任务子任务生成的学习任务` 的目标必须更细，不允许只写成笼统的 `新增可用方法`；至少要落到 `哪个需求位 / 哪种缺口 / 缺多少`。
7. `任务管理任务生成的独立学习任务` 默认可以是 `泛化学习`，但仍必须带清楚 `来源根任务 / 来源功能域 / 来源方法需求位 / 上浮压力类型`。
8. 两大类学习任务允许共用一个共享学习执行器，但语义归属、目标、回流范围不得混淆。
9. `缺失方法数量` 是 `任务子任务生成的学习任务` 出生的第一条显式缺口特征；它不能单独代表 `任务管理任务生成的独立学习任务`。
10. 后续代码不得再把“所有学习都压成一条共享泛化学习流”。
11. 当学习子任务分型服务于 `任务管理方法` 分支时，外层仍须显式化 `最近根层裁决结果`、`当前缺口类型`、`当前下一步去向`。
12. 任一学习子任务的执行结果在当前治理轮被消费后，仍必须先回到外层重新做 `根层重判 -> 上下文刷新`，再决定是否继续调度、召回或升格。

一句话收口：

`学习任务统一分为“任务管理独立学习”和“任务子任务学习”；二者都可共用执行器，但不能共用语义。`

并且：

`两类学习任务都应先独立落成任务实体，再统一提交给学习调度器执行。`


## 4. 两类学习子任务定义

### 4.0 与任务管理治理分支的边界

必须固定：

1. 本文描述的是全域“学习子任务分型”的共性层。
2. 若服务对象是 `任务管理方法` 分支，则必须额外携带：
- `最近根层裁决结果`
- `当前缺口类型`
- `当前下一步去向`
3. 这三项属于治理上下文，不属于所有学习子任务字段的统一必填字段。

## 4.1 任务子任务生成的学习任务

含义：

- 由某个真实业务任务、步骤或子任务在推进过程中派生
- 目标是补它当前这一步、这一片需求位上的缺口

父子归属：

- `来源根任务`
- `来源父任务`
- `学习子任务`

最小目标粒度：

- `来源方法需求位`
- `学习目标类型`
- `缺口类型`
- `缺失方法数量`

常见目标：

- `新增可用方法`
- `补齐条件`
- `补齐结果`
- `补结构`
- `提升健壮性`

回流边界：

- 默认优先回流到 `来源父任务`
- 若证据足够，再扩展到 `同需求位任务`


## 4.2 任务管理任务生成的独立学习任务

含义：

- 由任务管理任务在长期推进中识别出的储备学习动机所派生
- 目标不是立刻补某个单一任务，而是扩充某个需求位附近的方法资产

父子归属：

- `来源根任务`
- `对应根任务的任务管理任务`
- `任务管理广泛学习子任务`

最小目标粒度：

- `来源功能域`
- `来源方法需求位`
- `上浮压力类型`
- `学习目标类型`

常见目标：

- `扩大可选范围`
- `扩大覆盖面`
- `提升健壮性`
- `提升可验证性`
- `新增储备方法`

回流边界：

- 默认先回流到 `当前根任务方法资产`
- 再按需求位匹配，回流到同类任务


## 5. 出生规则

## 5.1 任务子任务生成的学习任务出生

当满足以下条件时，应优先出生 `任务子任务生成的学习任务`：

- 当前父任务仍需继续推进
- 当前父任务在当前需求位 `无可用方法`
- 或当前父任务存在明确的 `补条件 / 补结果 / 补结构 / 健壮性` 缺口

最小写口：

- `缺失方法数量`
- `来源方法需求位`
- `学习目标类型`
- `学习类型=定向/弱定向`


## 5.2 任务管理任务生成的独立学习任务出生

当满足以下条件时，应出生 `任务管理任务生成的独立学习任务`：

- 由任务管理任务统计出的 `扩选 / 健壮 / 覆盖 / 冗余 / 可验证` 压力持续偏高
- 但当前又不能压缩成某个单一父任务的精确缺口

最小写口：

- `来源功能域`
- `来源方法需求位`
- `上浮压力类型`
- `学习目标类型`
- `学习类型=泛化`


## 6. 目标粒度规则

### 6.1 任务子任务生成的学习任务

目标必须细化到以下层级之一：

- `在需求位 X 上新增 1 个可用方法`
- `为父任务当前步骤补齐条件`
- `为父任务当前步骤补齐结果`
- `为父任务当前路径补一个结构方法`
- `为父任务当前方法提升健壮性`

禁止只写：

- `新增可用方法`
- `学习一下`


### 6.2 任务管理任务生成的独立学习任务

目标可以相对宽，但必须至少细化到：

- `在需求位 X 上扩大可选范围`
- `在需求位 X 上提升健壮性`
- `在需求位 X 上扩大覆盖面`
- `在需求位 X 上补充储备方法`


## 7. 共享执行器规则

允许：

- 两类学习子任务共用共享学习执行器

但必须：

- 在执行前明确当前消费的是哪一类子任务
- 在执行中保留 `来源根任务 / 来源父任务 / 来源功能域 / 来源方法需求位`
- 在回流时按子任务类型走不同回流路径

禁止：

- 只因为共用执行器，就把所有学习都记成同一类 `泛化学习`


## 8. 字段最小集

### 8.1 任务子任务生成的学习任务最少新增/保留

- `学习子任务_类型 = 定向`
- `学习子任务_来源根任务头指针`
- `学习子任务_来源父任务头指针`
- `学习子任务_来源方法需求位`
- `学习子任务_学习目标类型`
- `学习子任务_缺失方法数量`
- `学习子任务_缺口类型`


### 8.2 任务管理任务生成的独立学习任务最少新增

- `学习子任务_类型 = 管理广泛`
- `学习子任务_来源根任务头指针`
- `学习子任务_来源任务管理任务头指针`
- `学习子任务_来源功能域`
- `学习子任务_来源方法需求位`
- `学习子任务_上浮压力类型`
- `学习子任务_学习目标类型`

## 10. 后续实现顺序

建议顺序固定为：

1. 先补 `学习子任务_类型`
2. 再把当前显式学习子任务收成真正的 `任务子任务生成的学习任务`
3. 再新增 `任务管理任务生成的独立学习任务`
4. 最后让共享学习执行器按类型分流


## 11. 结论

后续统一按以下口径实现：

- `任务本身缺口` -> `定向学习子任务`
- `任务管理长期储备动机` -> `任务管理广泛学习子任务`

不允许再把两者混成同一种学习任务语义。

并且始终保持：

- `学习子任务出生 -> 调度 -> 执行 -> 回流` 只能作为承接层运行。
- 每轮消费结果后，仍必须回到外层重新做 `根层重判 -> 上下文刷新`。
