# 1207 学习任务作为任务管理内部治理子任务规范

## 1. 目标

本文用于固定学习任务的统一定位。

当前唯一正式口径是：

- 标的：`方法`
- 目标：`使方法可用`
- 主逻辑：`目标 -> 筹办 -> 执行`


## 2. 核心结论

必须固定以下结论：

1. `学习任务` 不是第三根任务。
2. `学习任务` 是由 `任务管理任务` 派生并继续治理的内部治理子任务。
3. `学习任务` 的标的是 `方法`，不是现实对象。
4. `学习任务` 的目标是 `使目标方法可用`。
5. `学习任务` 的筹办职责，是完善学习任务树，并把待学习方法补到可执行。
6. `学习任务` 的执行职责，是执行当前学习步骤，把结果写回待学习方法树和学习结果节点。


## 3. 学习任务的任务树

学习任务必须遵循和普通任务一致的结构：

```text
学习任务头
-> 学习步骤
-> 学习结果
-> 下一学习步骤
```

不得再把学习任务长期理解成“只有任务头、靠另一套状态机推进”。


## 4. 学习任务的筹办

学习任务的筹办至少要补齐：

1. 当前学习步骤节点
2. 当前学习结果节点
3. 当前步骤绑定的学习方法
4. 待学习方法的动作骨架
5. 待学习方法的条件侧
6. 待学习方法的结果侧

也就是说，学习任务在真正尝试学习之前，必须先让学习对象可执行。


## 5. 学习任务的执行

学习任务执行的是 `学习步骤`，不是抽象“学习状态机”。

当前学习步骤可以是：

1. `补动作`
2. `补条件`
3. `补结果`
4. `验证`
5. `尝试学习`

其中最后一步才是 `尝试学习`。


## 6. 学习任务与任务管理任务的关系

任务管理任务负责：

1. 确认是否需要学习
2. 创建或复用学习任务头
3. 为学习任务筹办当前学习步骤

学习任务负责：

1. 执行当前学习步骤
2. 把本轮学习结果写回待学习方法树


## 7. 禁止项

禁止以下做法：

1. 把学习任务写成与普通任务不同的另一套主流程。
2. 在没有学习步骤节点的情况下直接进入尝试学习。
3. 把“方法不可用”直接等同“可以空转学习”。
