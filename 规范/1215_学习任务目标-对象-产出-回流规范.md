# 1215 学习任务目标-对象-产出-回流规范

## 1. 目标

本文用于在学习任务内部，把 `目标 -> 筹办 -> 执行` 进一步解释清楚。


## 2. 核心结论

必须固定以下结论：

1. 学习任务的目标，是 `使目标方法可用`。
2. 学习任务的对象，是 `目标方法`，以及围绕该方法生成的任务实例节点。
3. 学习任务的产出，不是泛化摘要，而是写回到方法树和学习结果节点的结构化结果。
4. 学习任务的回流，必须先经过学习结果节点，再返回任务管理任务消费。


## 3. 目标

学习任务头必须至少能回答：

1. 目标方法是谁
2. 什么叫“该方法可用”
3. 当前还缺什么


## 4. 筹办

学习任务的筹办，不是“直接开始学”，而是：

1. 为学习任务创建当前学习步骤。
2. 为学习步骤创建当前学习结果。
3. 为当前学习步骤绑定学习方法。
4. 为目标方法补动作、补条件、补结果。

因此当前顶层学习需求统一收口为：

`让当前目标方法可用`

而不是把：

- 样本收集
- 差异比较
- 候选动作生成

这些直接升格成顶层任务目标。


## 5. 执行

学习任务执行当前学习步骤后，至少应写回：

1. 当前学习结果节点
2. 目标方法的动作骨架补齐结果
3. 目标方法的条件侧补齐结果
4. 目标方法的结果侧补齐结果

必要时还可继续长出下一学习步骤。


## 6. 回流

学习任务的结果回流顺序固定为：

```text
学习方法执行结果
-> 当前学习结果节点
-> 目标方法树写回
-> 返回任务管理任务
```

不得直接跳过学习结果节点，把方法写回结果直接冒充任务管理最终结论。
