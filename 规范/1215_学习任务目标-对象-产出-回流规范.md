必须固定：

3. 本文不是新的根裁决层，而是学习闭环的业务承接层。


## 3. 核心结论

必须固定以下 21 条：

1. `学习对象` 只是原料视角下的承接对象；正式进入学习执行态时，直接对象统一收口为 `方法`。
2. `学习目标` 决定“这次为什么学、想学出什么”，优先于对象选择。
3. `学习对象` 决定“拿什么方法来学”；若上游只有因果、片段或任务结构，则必须先抽象出 `待学习方法`。
4. `学习产出` 不是单一的“新方法”，而可能是：候选方法、临时方法、禁用条件、补齐模板、可靠度增量、可选范围增量。
5. `学习回流` 不总是“直接唤醒原任务”；它可以是 `直接回流 / 匹配后回流 / 仅沉淀资产`。
6. 没有 `学习目标`，学习对象无法被正确选择。
7. 没有 `学习产出定义`，学习结果无法进入任务管理与方法成长闭环。
8. 没有 `学习回流规则`，学习只能停留在“学到了点什么”，不能转化为治理收益。
9. 学习任务的最小业务闭环必须固定为：

```text
衍生需求
-> 学习目标
-> 学习对象
-> 学习产出
-> 学习回流
-> 方法成长/任务继续推进
```

10. 后续所有学习实现，都不得跳过这 5 层中的任一层。
11. 当学习服务于 `任务管理方法` 分支时，外层仍须显式化 `最近根层裁决结果`、`当前缺口类型`、`当前下一步去向`，再进入目标/对象/产出/回流闭环。
12. 任一学习产出在当前治理轮被消费后，仍必须先回到外层重新做 `根层重判 -> 上下文刷新`，再决定是否继续验证、召回或升格。
13. 学习结果允许在每次尝试后先直接写入对应方法或方法账本，但这种直接写入首先只表示“进入方法存在面/证据面/补全面”，不自动等于“已经允许默认召回、默认执行或正式升格”。
14. `学习回流` 在本规范中首先属于承接层结果；只有外层最终裁成 `允许回流`，学习产出才算正式接回当前主闭环。
15. 学习不仅负责提升已有方法的可用性与可靠性，也负责把原本尚未进入方法存在面的结构化方法补进来。
16. `无方法` 或 `方法不可用` 不自动等于“应立即反复进入学习”；默认应先挂起对应 `待学习方法`，等待再次观察、条件变化或再次命中后再重判。
17. 学习进入正式执行态前，至少应已有 `动作骨架 + 条件侧`；其中 `条件侧` 允许由学习任务给出，且方法条件特征类型必须已知；若当前只有 `目标特征类型`，则该头仍属于 `需求位方法头`，不得直接进入学习。
18. 方法条件特征若为 `I64`，则按值域或样本域试探；若为 `VecIU64`，则按相似度比较；若为 `指针`，则不得直接学习原值，必须先转成二次特征，且二次特征统一以 `I64` 表达。
19. 学习执行必须按方法主影响面分型；若方法主结果只影响 `自我的真实特征值或自我运行真值`，则学习阶段默认只能做影子验证、镜像验证或等价验证，不得直接改写真实自我状态。
20. 上述受保护的 `内部状态`，只包括 `自我的真实特征值` 与 `自我运行真值`，不包括 `方法条件 / 方法结果 / 方法账本 / 方法可靠度 / 方法稳定度 / 候选方法 / 临时方法 / 方法虚拟存在` 等方法侧资产变化；这些方法侧变化仍允许作为学习主结果正常写入。
21. 若方法主结果会影响 `现实对象 / 外设 / 对外输出`，则学习阶段默认只能在 `可控范围` 内执行；至少应显式限制 `作用对象`、`作用幅度`、`执行时长`、`执行次数` 与 `停止条件`。

可收敛为一句话：

`学习不是“拿着原料直接学”，而是先定目标，再把原料收口成待学习方法，再定义产出；每次尝试的结果可以先直接写入方法与记忆整理，再由外层决定如何正式回流。`

并且必须继续固定：

- `决定怎样回流` 不等于 `已经允许回流`
- `允许回流` 的最终边界，按 [0567_最终裁决层与外层重判边界规范.md](D:\鱼巢\规范\0567_最终裁决层与外层重判边界规范.md) 执行
- 学习与方法四层分工的统一边界，按 [0568_方法存在可用可执行成熟分层规范.md](D:\鱼巢\规范\0568_方法存在可用可执行成熟分层规范.md) 执行
- 每次尝试后直接写入方法与记忆整理的口径，优先落在 `方法账本 / 条件节点 / 结果节点 / 方法虚拟存在`，不另造平行学习账本
- 学习不能直接污染 `自我的真实特征值与运行真值`，但允许正常写入 `方法资产`


## 4. 五层闭环定义

## 4.1 学习目标

回答的是：

- 这次为什么学
- 想补哪类能力
- 想让哪一片内部治理状态发生变化

最小应包含：

- `来源根任务`
- `来源宿主任务`
- `来源方法需求位`
- `学习类型`
- `目标约束强度`
- `目标作用域`

常见目标包括：

- `补方法存在面`
- `新增可用方法`
- `提升方法健壮性`
- `扩大可选范围`
- `扩大覆盖面`
- `补齐条件`
- `补齐结果`
- `提升可验证性`


## 4.2 学习对象

回答的是：

- 这次准备用什么原料来学习

最小直接对象集合以后续统一按 [1209_学习对象与方法来源规范.md](D:\鱼巢\规范\1209_学习对象与方法来源规范.md) 为准，包括：

- `本能方法`
- `现有方法/方法片段`
- `待学习方法骨架`

上游可用原料包括：

- `因果信息`
- `成功执行片段`
- `稳定任务路径`
- `稳定任务子树`

必须固定：

- 对象选择不是自由发挥
- 必须受 `学习目标` 约束
- `因果信息 / 成功执行片段 / 稳定任务路径 / 稳定任务子树` 必须先抽象为 `待学习方法`，不得裸进入正式学习执行
- 只有当学习对象已能组织出某个 `待学习方法` 的 `动作骨架` 时，才允许继续进入正式学习执行态。
- 若当前只有 `需求信息 / 目标特征类型`，还没有动作骨架与条件侧，则它只能停留在 `需求位方法头 / 方法需求位 / 待补动作与条件` 状态，不得直接进入正式学习。
- 若当前只有“缺方法/方法不可用”结论，但还没有可挂起的待学习方法骨架，则应先保留为缺口和压力，不得强行发出可执行学习请求。
- 若方法条件特征类型为 `I64`，允许按合法值域或样本值域生成 `最小 / 中间 / 最大` 三组试探值，并每组三次重复验证稳定性。
- 若方法条件特征类型为 `VecIU64`，不生成最大最小值，统一按相似度分层比较与重复验证稳定性。
- 若方法条件特征类型为 `指针`，不得直接学习原指针值；必须先转成二次特征，而二次特征统一使用 `I64`。

### 4.2.1 学习执行影响面边界

必须固定：

- 学习执行不是普通执行；它首先用于获取方法证据、验证边界与补齐方法资产。
- 若某方法的主结果只影响 `自我的真实特征值` 或 `自我运行真值`，则学习阶段默认只能做 `影子验证 / 镜像验证 / 等价验证`，不得直接改写真实自我状态。
- 这里受保护的 `内部状态`，只包括自我的真实特征值与运行真值，例如：`安全值`、`服务值`、`待机状态`、`风险安全值`、`当前主需求 / 当前主任务 / 当前主方法` 等主链真值。
- `方法条件节点 / 方法结果节点 / 方法账本 / 方法可靠度 / 方法稳定度 / 候选方法 / 运行期临时方法 / 方法虚拟存在 / 学习证据` 等方法侧资产，不属于这条保护限制，仍允许作为学习结果正常写入。
- 若某方法的主结果会影响 `现实对象 / 外设 / 对外输出`，则学习阶段默认只能在 `可控范围` 内执行，不得把学习试探直接放大成正式执行。
- 这里的 `受控`，应先由 `任务管理任务` 完成治理判定；学习执行器不得自行越权决定“是否允许试探、允许到什么范围”。
- `可控范围` 至少应显式限制：`作用对象`、`作用幅度`、`执行时长`、`执行次数`、`停止条件`；若可回退或可补偿，也应优先纳入控制边界。
- 这里的 `受控`，还必须包含对 `方法对自我影响是否可接受` 的综合判定；不得只限制次数和时长，却不判断方法结果本身是否可接受。
- 这里的 `试探`，不是影子占位，而是真实执行；只是该真实执行必须服从上一步由 `任务管理任务` 给出的受控边界。
- 受控微试探的可接受性，应基于 `整次试探执行的结果状态集合` 做综合评判，而不得只看 `最后一步返回的单个结果状态`。
- 这里的 `结果状态集合`，除最后一步结果状态外，还必须包括执行过程中 `中间产生且最终保留下来的结果状态`；凡进入本次试探最终结果面的状态，都应纳入综合评判。
- 对每个结果状态，应按其方向与权重形成综合分，再将结果状态综合分叠加。
- 当前工程可先按以下最小口径收敛：`综合分 > 0` 视为可接受，`综合分 < 0` 视为不可接受，`综合分 = 0` 不得直接视为可接受，应继续补证或交由外层重判。

可收敛为两句话：

`学习不能随意改写真实自我状态，但可以正常沉淀方法资产。`

`会影响现实的方法，只能做受控微试探，不能直接按正式执行放开。`

`受控微试探是否可接受，不看口号，而看整次试探结果状态集合权重叠加后的综合分。`

`受控` 由任务管理任务先判，`试探` 由学习执行器按受控边界做真实执行。`


## 4.3 学习产出

回答的是：

- 这一轮学习最少能产出什么可被后续消费的东西

最小产出类型固定为：

- `候选方法`
- `运行期临时方法`
- `禁用条件`
- `条件模板补全`
- `结果模板补全`
- `方法可靠度增量`
- `方法稳定度增量`
- `可选范围增量`
- `覆盖面增量`


## 4.4 学习回流

回答的是：

- 学出来的东西先回到哪里
- 以什么强度影响任务管理和方法成长

最小回流策略固定为：

- `直接回流`
- `匹配后回流`
- `仅沉淀资产不回流`

必须补充固定：

1. `每次尝试直接写入方法` 属于“先写入方法证据面/补全面”。
2. 这种写入可以在每次尝试后立即发生。
3. 但是否因此进入当前轮正式召回、正式执行或成熟度升格，仍需外层重判。
4. 对 `无方法 / 方法不可用` 场景，默认先把对应 `待学习方法` 挂起并标成待观察；只有条件变化、再次观察命中、或明确记录到该方法被使用后，才允许重新进入学习或回流链。
5. 这类挂起状态应可被控制面板或等价任务信息窗口显示。

最小结果归属范围固定为：

- `当前任务`
- `当前根任务`
- `同需求位任务族`
- `全局方法资产`


## 4.5 方法成长与任务继续推进

回答的是：

- 学习结果怎样继续进入治理闭环

最小后续流向固定为：

- `更新当前任务可用方法面`
- `更新等待学习任务唤醒条件`
- `更新方法可靠度/稳定复现账本`
- `新方法进入运行期临时/反推/组合来源链`

### 4.6 与任务管理治理分支的边界

必须固定：

1. 本文描述的是全域学习闭环的共性层。
2. 若服务对象是 `任务管理方法` 分支，则必须额外携带：
- `最近根层裁决结果`
- `当前缺口类型`
- `当前下一步去向`
3. 这三项属于治理上下文，不属于所有学习目标、学习对象、学习产出的统一必填字段。


## 5. 为什么“有了学习对象”还不够

必须明确：

同一个学习对象，在不同目标下，后续动作完全不同。

例如：

- `因果信息`
  - 若目标是 `补当前缺口`
    - 更适合走 `反推候选方法`
  - 若目标是 `提升覆盖`
    - 更适合做 `条件桶扩展`

- `成功执行片段`
  - 若目标是 `先让原任务能继续`
    - 更适合提炼 `运行期临时方法`
  - 若目标是 `扩大可选范围`
    - 更适合先记为 `候选资产`

- `稳定任务子树`
  - 若目标是 `补结构`
    - 更适合沉淀 `组合方法`
  - 若目标是 `提升健壮性`
    - 更适合提炼 `替代路径`

所以必须固定：

`学习对象` 只能回答“可学什么”，不能回答“为什么学、学出什么、怎么用”。


## 6. 学习目标到直接学习对象/方法原料的默认映射

| 学习目标 | 默认优先学习对象 | 说明 |
| --- | --- | --- |
| `新增可用方法` | 本能方法、现有方法片段、成功执行片段、因果信息 | 优先补可调用路径 |
| `提升健壮性` | 现有方法片段、因果信息、稳定任务路径 | 优先看失败与分裂附近 |
| `扩大可选范围` | 现有方法树、稳定任务路径、稳定任务子树 | 优先扩同需求位附近的方法面 |
| `扩大覆盖面` | 因果信息、稳定任务路径、稳定任务子树 | 优先扩条件桶覆盖 |
| `补齐条件` | 现有方法片段、成功执行片段、稳定任务路径 | 优先补条件节点和前置步骤 |
| `补齐结果` | 成功执行片段、因果信息、稳定任务路径 | 优先补结果节点和结果模板 |
| `提升可验证性` | 因果信息、成功执行片段、稳定任务路径 | 优先补可确认果与回执链 |

说明：

- 这是默认优先级，不是硬编码唯一选择
- 若当前目标约束很强，应优先选择与来源宿主任务最近的对象
- 若当前目标是泛化学习，则优先选择跨任务稳定对象
- 表中的 `因果信息 / 成功执行片段 / 稳定任务路径 / 稳定任务子树` 默认都属于方法原料，必须先抽象出 `待学习方法`，再进入正式学习执行


## 7. 学习对象/方法原料到学习产出的默认映射

| 学习对象 | 默认最适合产出 | 默认方法来源倾向 |
| --- | --- | --- |
| `本能方法` | 最小可试探路径、基础候选方法 | `本能 / 运行期临时` |
| `现有方法片段` | 补齐条件、补齐结果、邻近变体方法 | `运行期临时 / 组合` |
| `因果信息` | 待学习方法骨架、反推候选方法、条件摘要、结果摘要 | `反推` |
| `成功执行片段` | 待学习方法骨架、运行期临时方法、局部模板 | `运行期临时` |
| `稳定任务路径` | 待学习方法骨架、组合候选方法、稳定推进模板 | `运行期临时 -> 组合` |
| `稳定任务子树` | 待学习方法骨架、结构化组合方法、失败策略方法、补结构方法 | `运行期临时 -> 组合` |

必须固定：

- `因果信息` 默认不直接产出成熟组合方法
- `成功执行片段` 默认不直接产出正式组合枝
- `任务路径/任务子树` 默认可以产出更高层方法结构，但首次沉淀仍应先记为 `运行期临时`
- 任何原料型对象都必须先收口为 `待学习方法`，再进入正式学习执行与稳定性验证


## 8. 学习产出的最小落账要求

每轮学习至少应显式产出以下之一：

1. `方法类产出`
- `候选方法首节点`
- `运行期临时方法首节点`
- `方法来源`

2. `模板类产出`
- `条件模板补全`
- `结果模板补全`

3. `治理增量产出`
- `可用方法增量`
- `稳定方法增量`
- `稳定度总分增量`
- `可选范围增量`
- `覆盖面增量`

4. `负样本产出`
- `禁用条件`
- `不可用路径`
- `弱证据候选`

若以上都没有，则本轮学习不能记为“有效学习完成”。


## 9. 学习回流的三种标准模式

## 9.1 直接回流

适用：

- 这次学习的目标强约束明确
- 结果直接补中了来源宿主任务的当前缺口

典型情形：

- 为等待学习的原任务新增了一个可用方法
- 为当前步骤补齐了条件/结果

效果：

- 允许直接唤醒原任务
- 允许立即更新当前任务可用方法面


## 9.2 匹配后回流

适用：

- 学到了东西，但不确定是否正好补中原任务

典型情形：

- 弱定向学习得到邻近替代方法
- 泛化学习得到新候选方法

效果：

- 先沉淀到候选资产
- 再由任务管理任务按 `方法需求位` 重新匹配


## 9.3 仅沉淀资产不回流

适用：

- 这次学习主要形成的是长期资产
- 当前没有可直接受益的宿主任务

典型情形：

- 扩选型泛化学习
- 覆盖型泛化学习
- 只形成弱证据或条件模板

效果：

- 不直接唤醒原任务
- 只进入方法资产、因果资产或模板资产


## 10. 学习闭环的最小运行顺序

后续统一按以下顺序运行：

```text
1. 任务管理任务上浮衍生需求
2. 生成学习目标
3. 选择学习对象
4. 执行学习
5. 生成学习产出
6. 判定回流策略
7. 写回任务管理状态与方法成长账本
```

并且必须继续固定：

- 第 7 步之后，不得直接在同一层内部继续连跑“召回方法 -> 再学习 -> 再升格”闭环。
- 正确顺序应为：`学习写回 -> 返回外层根层重判/上下文刷新 -> 再决定是否继续筹办、召回或二次学习`。

必须禁止以下跳步：

- 不能跳过 `学习目标` 直接选对象
- 不能跳过 `学习产出定义` 直接回流
- 不能跳过 `回流策略判定` 直接唤醒原任务

## 12. 收口结论

后续统一按以下判断执行：

1. 学习对象已经具备，不代表学习闭环已经具备。
2. 下一步必须把 `目标 -> 对象 -> 产出 -> 回流` 接全。
3. 学习结果只有在经过 `产出定义 + 回流判定` 后，才真正能进入任务管理与方法成长体系。

最终统一收口为：

`学习任务的真正最小闭环，不是“有原料就学习”，而是“有目标、把原料收口成待学习方法、产生产出、决定回流，并把结果接入方法成长与任务继续推进”。`
