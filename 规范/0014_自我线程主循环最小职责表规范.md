## 3. 核心结论

必须固定以下 14 条：

1. `自我线程` 是运行时第一层，且只有一个 `逻辑主控`。
2. `自我线程主循环` 只承担控制面职责，不负责 `展开步骤`，也不直接 `实现方法`。
3. `控制消息`、`全局环境消息`、`工作线程回流`、`外部反馈` 和 `恢复事件` 都必须先进入 `自我线程`，再决定是否下沉。
4. 主循环每轮必须先 `冻结本轮消息批次`，只处理本轮开始时已可见的消息及其同步内部闭包。
5. 主循环必须把本轮输入统一整理成 `特征变化`，不能直接拿局部结果越级当最终结论。
6. 主循环必须基于一次变化 `重算二次特征`，再进入上位判断。
7. `根层重判` 只能由 `自我线程` 发起并持有结果。
8. `自我线程` 对下游输出的是 `一个主调度决议`；该决议下可以包含 `0..N 个彼此不冲突的并发派发动作`，而不是局部步骤执行细节。
9. `派发工作线程` 可以有多个，但都只能发生在 `根层重判` 与同一份 `主调度决议` 之后。
10. `任务管理任务`、`动作线程`、`学习线程` 只负责承接局部推进，不得替代 `自我线程` 做最终运行裁决。
11. `子结果`、`学习反馈`、`异常结果` 必须先回 `自我线程`，再由它统一 `路由结果`。
12. `最终结算` 只能由 `自我线程` 完成，不能散落给局部工作线程自行记总账。
13. 本轮处理中异步新产生的消息必须延后到下一轮；高优先级控制消息可按抢占规则中断本轮。
14. 若缺少 `收消息 -> 冻结批次 -> 整理特征变化 -> 重算二次特征 -> 根层重判 -> 生成调度决议 -> 派发工作线程 -> 路由结果 -> 最终结算` 这条链，规范层就不能宣称主循环最小闭环已经成立。

一句话收口：

`第一层自我线程只做统一收件、统一重判、统一调度、统一路由和统一结算。`

## 4. 第一层：自我线程

只有一个逻辑主控。

它不负责展开步骤，也不直接实现方法。

### 4.1 最小职责表

| 职责 | 回答的问题 | 为什么必须留在自我线程 |
| --- | --- | --- |
| `收消息` | 这一轮主控先看见了什么输入 | 首收件人必须统一，否则控制面会碎裂 |
| `冻结本轮消息批次` | 这一轮到底处理哪一批消息 | 只有先冻结批次，轮次边界和抢占边界才清楚 |
| `整理特征变化` | 本轮输入到底引起了哪些一次变化 | 后续二次特征与根层重判必须建立在统一变化面上 |
| `重算二次特征` | 一次变化综合后形成了哪些上位判断线索 | 二次特征是正式重判输入，不能散落在局部线程各自解释 |
| `根层重判` | 当前应继续、等待、重筹办、收束还是停止 | 这是所有下游推进前的上位裁决 |
| `生成调度决议` | 本轮整体该如何调度、可并发下发哪些不冲突动作 | 推进方向、承接对象、并发集合和预算资格只能由逻辑主控统一拍板 |
| `派发工作线程` | 哪些下游线程来承接本轮局部工作 | 让谁承接、能否并发承接、承接到哪一层都必须由主控明确 |
| `路由结果` | 回来的结果该回给谁、影响哪一层 | 父子关系和全局影响只有主控能统一判断 |
| `最终结算` | 本轮最终如何记账并形成下一轮输入 | 总账、最终去向和下一轮入口必须统一收口 |

必须固定：

- `需求树 / 需求队列 / 方法禁止项 / 任务控制态` 等更细治理材料，若当前工程仍需维护，应视为 `整理特征变化`、`生成调度决议`、`最终结算` 的内部支撑面，而不是第一层额外扩出去的执行职责。
- 第一层只回答 `看见什么`、`如何判断`、`要不要派发`、`结果回到哪里`、`最后怎么算` 这五类问题。
- 任何需要进入 `步骤展开`、`方法组织`、`局部执行` 的工作，都必须向下让渡给 `任务管理任务` 或其他工作线程。


## 5. 统一主循环顺序

从规范层看，主循环至少应按以下顺序运行：

```text
1. 收消息
2. 冻结本轮消息批次
3. 整理特征变化
4. 重算二次特征
5. 根层重判
6. 生成调度决议
7. 派发工作线程
8. 路由结果
9. 最终结算
```

必须固定：

- 不得在第 `5` 步之前先默认“允许继续推进”。
- 不得在第 `6` 步之前先行派发工作线程。
- 不得在第 `8` 步之前让子结果绕过线程直写父层任务或全局状态。
- 不得把第 `9` 步结算下放给动作线程、学习线程或任务管理工作线程自行决定最终真值。
- 本轮冻结消息集处理完之前，不得先行做本轮最终派发或最终去向裁决。
- 本轮中新产生的异步消息，不得为了“清空活队列”而硬追入当前轮次。


## 6. 主循环必须直接完成的事项

### 6.1 收消息与冻结本轮消息批次

主循环必须直接消费至少以下类型的回流：

- `任务执行返回消息`
- `学习线程阶段结果`
- `等待/唤醒结果`
- `宿主输入或外部反馈`
- `恢复重建一次性事件`
- `mailbox` 中待消费的显式治理事件

必须固定：

- 这些回流首先属于主循环输入，不属于执行线程本地真值。
- 回流要先冻结成“本轮消息集”，并处理其同步内部闭包，再进入 `整理特征变化 -> 重算二次特征 -> 根层重判`。
- 正式 `负反馈` 也只能在这一步之后，由主循环统一生成。
- `控制消息` 与 `全局环境消息` 必须直接进入主循环，不得被局部工作线程先行吞掉。
- `任务绑定且仅影响局部流程的外部反馈` 可先进入对应任务管理工作线程，再把升级后的控制面变化上报主循环。

补充固定：

1. 主循环每轮不是“只吃一条消息”，也不是“追到活队列彻底为空”为止。
2. 主循环每轮应处理完本轮冻结消息集，以及这些消息同步引发的内部闭包。
3. 本轮处理中异步新产生的消息，应进入下一轮。
4. `打断 / 收束 / 停止 / 强制优先级刷新 / 强制预算刷新` 这类高优先级控制消息，可按抢占规则中断本轮。
5. `步骤内部小进度 / 临时变量变化 / 内部小重试 / 纯中间状态` 不属于必须上报主循环的消息。
6. `当前步骤阻塞 / 解除阻塞 / 派生需求 / 局部失败 / 异常 / 当前任务完成 / 最终结果返回 / 需要父层任务判断的子结果` 属于应上报主循环的关键推进事件。

### 6.2 整理特征变化

主循环必须直接承担：

- 整理 `状态变化`
- 整理 `结果变化`
- 整理 `阻塞 / 解除阻塞 / 异常` 变化
- 整理 `派生需求 / 学习反馈 / 外部反馈` 带来的控制面变化
- 形成本轮 `一次特征变化集`

必须固定：

- 这里整理的是 `变化`，不是 `步骤`。
- 主循环只负责把变化压成正式输入，不负责在这一层直接补方法、挑方法或展开宿主步骤。
- 纯局部执行痕迹只有在改变控制面判断时才允许上浮到这一层。

### 6.3 重算二次特征

主循环必须直接完成：

- 基于 `一次特征变化集` 重算 `二次特征`
- 把原始事实压成可重判的 `方向线索 / 风险线索 / 门控线索 / 学习或等待线索`
- 形成本轮 `上位判断输入`

必须固定：

- `二次特征` 是原始变化之后、根层重判之前的正式桥层，不得与原始消息或最终结算值混写。
- 二次特征的实现算法可以下沉为模块，但其拥有权和最终解释权仍属于 `自我线程`。

### 6.4 根层重判

主循环必须直接完成：

- 读取当前关键值
- 结合需求值、当前值和二次特征生成方向输入
- 需要时结合预测结果与执行前门控形成上位判断
- 生成 `最近根层裁决结果`

必须固定：

- 这里的方向必须来自比较结果，不能直接拿一次特征或局部成功字段当裁判。
- 根层重判是后续 `调度决议` 的上位输入。
- `最近根层裁决结果` 也是执行前是否允许进入普通执行链的直接依据。
- 更细的 `根层重判结果` 定义、预测型/回收型两种重判时机和禁止项入口，按 [0315_根层重判结果与执行前允许边界规范.md](D:\鱼巢\规范\0315_根层重判结果与执行前允许边界规范.md) 执行。
- 更细的 `根层重判结果` 最小字段表，按 [0316_根层重判结果最小字段表规范.md](D:\鱼巢\规范\0316_根层重判结果最小字段表规范.md) 执行。
- 更细的 `执行前门控` 最小输入与判定顺序，按 [0317_执行前门控最小输入与判定顺序规范.md](D:\鱼巢\规范\0317_执行前门控最小输入与判定顺序规范.md) 执行。

### 6.5 生成调度决议

主循环必须直接完成：

- 生成 `继续 / 等待 / 回到重筹办 / 转入学习 / 进入收束 / 停止` 等去向候选
- 决定是否需要实例化或唤醒 `任务入口`
- 决定本轮可派发给哪些 `工作线程 / 任务入口`
- 决定本轮的 `优先级 / 预算 / 允许边界`
- 决定 `派生需求` 是否升级为新的任务入口
- 把多个彼此不冲突的派发动作收进同一份 `主调度决议`

必须固定：

- `调度决议` 是第一层的正式产物，不是局部线程自己猜出来的默认行为。
- 本轮只生成一个 `主调度决议`；该决议下可以包含 `0..N 个彼此不冲突的并发派发动作`。
- `决议` 仍然只有一个，但 `派发动作` 可以有多个。
- 若本轮所有对象都应等待、收束、待机或停止，则允许 `主调度决议` 下不产生新的执行派发动作。
- 当前工程若仍使用 `需求树 / 需求队列 / 方法禁止项 / 任务控制态` 等运行时结构，应把它们视为这一决议步骤内部使用的治理材料，而不是另外一层执行职责。
- `任务管理任务` 可以提出候选，但不能越过这一层直接实例化新任务或直接宣布全局下一步。
- 更细的 `任务 / 任务控制态 / 任务管理任务` 三层边界，按 [0039_任务-任务控制态-任务管理任务三层结构规范_v0.1.md](D:\鱼巢\规范\0039_任务-任务控制态-任务管理任务三层结构规范_v0.1.md) 执行。

最小例子：

- 若本轮根层重判后得到：`任务 A 可运行`、`任务 B 可运行`、`任务 C 需等待 A 的结果`、`任务 D 因风险超阈值必须收束`
- 则本轮的 `主调度决议` 可以是：`并发派发 A、B`，`向 D 下发收束`，`C 留在等待队列`
- 这里仍然只有一份 `调度决议`，但它包含了多个可并发动作

### 6.6 派发工作线程

主循环必须直接完成：

- 创建或唤醒本轮需要的 `0..N 个任务入口`
- 装配本轮最小 `任务管理上下文`
- 把本轮局部工作按 `主调度决议` 派发给对应 `任务管理工作线程 / 动作线程 / 学习线程`

必须固定：

- 派发发生在 `调度决议` 之后，不得倒序。
- 派发动作可以是并发集合，但这些动作必须 `彼此不冲突`、`受同一份主调度决议约束`，并同时服从本轮预算与门控边界。
- 主循环负责 `让谁承接`，不负责替代下游 `展开步骤`。
- 工作线程只承接一步局部推进，不得在工作线程内越权连续跑多轮主闭环。
- 更细的上下文字段表与一步治理返回面，按 [1214_任务管理上下文最小字段表规范.md](D:\鱼巢\规范\1214_任务管理上下文最小字段表规范.md) 和 [1213_任务管理一步治理结果最小返回面规范.md](D:\鱼巢\规范\1213_任务管理一步治理结果最小返回面规范.md) 执行。

### 6.7 路由结果

主循环必须直接完成：

- 接收 `子结果 / 学习反馈 / 异常 / 等待结果 / 唤醒结果`
- 决定这些结果应回给哪个 `父层任务 / 控制面 / 下一轮输入`
- 决定哪些结果需要升级为 `派生需求 / 阻塞变化 / 优先级变化 / 新任务候选`

必须固定：

- 子任务结果必须先回主循环，再由主循环路由给父层任务；父层任务只判断局部影响，主循环判断全局影响。
- 结果路由属于全局关系处理，不能让局部工作线程直接越权改写父层和根层。
- 若继续把“收到消息之后”的治理骨架细化为八步实现表，按 [0037_自我线程消息后治理八步实现表_v0.1.md](D:\鱼巢\规范\0037_自我线程消息后治理八步实现表_v0.1.md) 执行。

### 6.8 最终结算

主循环必须直接完成：

- 统一结算本轮 `安全值 / 服务值 / 风险影响 / 学习触发`
- 统一写回本轮 `最终去向`
- 形成下一轮的最小输入摘要

必须固定：

- 不能只按“成功/失败”粗暴结算。
- 局部线程不能自行报功；总账只能由主循环统一结算并决定是否触发学习。
- 即使事件来自 `mailbox`，也必须先回到主循环统一拍板，不得让同一宿主越权连续推进两轮。
- 更细的 `实际结果方向` 与 `需求方向` 单特征对齐规则，按 [0321_实际结果方向与需求方向对齐判定表规范.md](D:\鱼巢\规范\0321_实际结果方向与需求方向对齐判定表规范.md) 执行。


## 7. 主循环不直接做的事

以下事项不应直接放在主循环里长期阻塞执行：

1. 展开宿主任务步骤
2. 直接实现或长时间执行叶子方法
3. 大块学习搜索、验证或补缺
4. 阻塞式外设或宿主 I/O
5. 在局部线程内连续跑完整个主闭环

必须固定：

- 主循环负责“决定与归并”，不负责长期占用执行时间片的重活。


## 8. 与下层边界

必须固定：

1. `自我线程` 负责 `收消息 / 重判 / 调度 / 派发 / 路由 / 结算`。
2. `任务管理任务` 负责在允许边界内 `展开一步宿主治理`。
3. `动作线程 / 学习线程` 负责局部执行、验证、采样、补缺，并把结果回流回来。
4. `需求树 / 需求队列 / 方法禁止项 / 任务控制态` 若在当前工程中存在，应服务于第一层决议，不得反过来把第一层膨胀回“全能执行层”。


## 9. 收口结论

后续统一按以下口径执行：

1. 第一层 `自我线程` 只保留九项最小职责：`收消息 / 冻结本轮消息批次 / 整理特征变化 / 重算二次特征 / 根层重判 / 生成调度决议 / 派发工作线程 / 路由结果 / 最终结算`。
2. 任何进入 `步骤展开 / 方法组织 / 局部执行` 的工作，都应继续向下沉给 `任务管理任务` 或其他工作线程。
3. 若这些职责没有在规范层被统一收口，就不能宣称最小治理闭环已经成立。

最终统一收口为：

`自我线程主循环真正要做的，不是亲自做事，而是把本轮输入重组成下一轮可判断、可派发、可结算的统一控制面。`
