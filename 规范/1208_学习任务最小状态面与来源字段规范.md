## 3. 核心结论

必须固定以下 12 条：

1. `学习任务` 的最小状态面至少应覆盖：`身份`、`来源`、`执行阶段`、`目标`、`结果`、`回流` 六层。
2. 当前已经具备的字段，足以支撑 `等待学习 -> 请求学习 -> 推进学习 -> 唤醒原任务` 的基础闭环。
3. 当前最明显的缺口，不是“学习任务没有状态”，而是 `来源根任务 / 来源方法需求位 / 学习类型 / 结果归属范围 / 回流策略` 这批字段还没统一显式化。
4. `学习任务` 允许共享执行器，但共享执行器不能替代来源字段。
5. `学习任务` 的请求字段应既能表达 `显式请求学习`，也能表达 `等待学习来源`。
6. `学习任务` 的结果字段应既能表达 `本轮学习结果`，也能表达 `进化目标是否达成`。
7. `学习任务` 的回流字段应能区分：`直接回流`、`匹配后回流`、`仅沉淀资产不回流`。
8. 所有自然语言名称字段，后续统一通过 `语素集.添加词性词(...)` 获取。
9. 后续新增字段时，不允许打破本文的六层分组。
10. `无方法 / 方法不可用` 默认先落成 `待学习方法挂起`，不自动生成可执行学习任务。
11. 可执行学习任务的最小前提是：能回指出 `待学习方法` 的动作骨架，且条件侧已存在或可由学习任务补给。
12. 若当前只有 `需求信息 / 目标特征类型`，则它仍属于 `需求位方法头`，不得直接生成可执行学习任务；必须待动作骨架与条件侧补齐后，才允许升为可执行学习任务。

可收敛为一句话：

`学习任务必须既能说明“我是谁”，也能说明“我从哪来、在学什么、学出了什么、回到哪里去”。`


## 4. 六层最小状态面

## 4.1 身份层

这层回答：

- 这是哪一个学习任务
- 它是学习树根节点、学习尝试分支，还是某个学习阶段步骤

最小字段：

- `学习任务头结点指针`
- `学习尝试分支头结点指针`
- `学习阶段步骤节点指针`
- `任务_当前子任务头指针`
- `任务_当前步骤ID`
- `学习阶段码`

当前对齐情况：

- `学习任务节点_ / 学习尝试任务节点_ / 学习采样步骤节点_ / 学习更新步骤节点_ / 学习验证步骤节点_ / 学习提交步骤节点_ / 学习回滚步骤节点_` 已存在于运行时
- `任务_当前子任务头指针`、`任务_当前步骤ID` 已有统一任务事实写回
- `学习阶段码` 已作为任务事实写回


## 4.2 来源层

这层回答：

- 这次学习是谁触发的
- 它最初是为哪个根任务、哪个宿主任务、哪个步骤、哪个方法位服务

最小字段：

- `来源根任务头指针`
- `来源宿主任务头指针`
- `来源步骤指针`
- `来源方法指针`
- `来源结果指针`
- `来源优先级`
- `来源错误码`
- `来源方法需求位`

当前已具备：

- `尝试学习_来源任务头指针`
- `尝试学习_来源步骤指针`
- `尝试学习_来源方法指针`
- `尝试学习_来源结果指针`
- `尝试学习_来源优先级`
- `尝试学习_错误码`

当前待补：

- `来源根任务头指针`
- `来源方法需求位`

说明：

- `来源宿主任务` 当前已经具备
- 但 `来源根任务` 还未显式镜像
- 若宿主任务不是根任务，后续回流和统计仍需要 `来源根任务`


## 4.3 执行阶段层

这层回答：

- 当前学习流推进到了哪一步
- 现在是待派发、执行中、提交中还是回滚中

最小字段：

- `任务状态`
- `学习阶段码`
- `学习工作流当前阶段`
- `学习工作流最近阶段时间`
- `任务_当前步骤ID`
- `任务_当前子任务头指针`

当前已具备：

- 学习树中的学习任务头结点会进入 `就绪 / 排队中 / 执行中`
- 学习尝试分支会进入 `执行中`
- 阶段步骤会进入 `执行中 / 完成 / 失败`
- `学习阶段码` 与当前步骤已同步写回

说明：

- 当前执行阶段面已经足以支撑运行时调试和基础回流
- 但尚未补成一套独立的“学习任务镜像状态面”文档接口

## 8. 收口结论

当前学习任务已经具备一套可以运行的基础状态面：

- 能表达来源宿主任务
- 能表达等待学习和显式学习请求
- 能表达学习阶段、学习目标、学习增量结果
- 能表达基础回流

但要真正把它收成“任务管理任务可直接管理的内部治理子任务”，还必须再补齐：

1. `来源根任务`
2. `来源方法需求位`
3. `学习类型`
4. `回流策略`
5. `结果归属范围`

后续所有新增字段，都应优先落入本文定义的六层状态面，而不应零散追加到无归属的临时事实里。
