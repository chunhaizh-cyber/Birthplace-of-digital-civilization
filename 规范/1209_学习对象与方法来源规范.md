- 学习在当前体系里，到底可以把什么当作 `现有学习对象`

本文要收口的是 `学习对象` 与 `方法来源` 的分类口径，不是某一种具体学习算法实现。

## 3. 核心结论

必须固定以下 14 条：

1. 正式进入学习执行态的直接学习对象统一收口为 `方法`。
2. `因果信息` 可以支持方法出生，但 `因果` 不是直接学习执行对象。
3. 当前体系必须把 `直接学习对象`、`方法原料`、`方法来源`、`方法成熟度阶段` 四层分开。
4. `方法来源` 回答“方法是怎么来的”；`方法成熟度阶段` 回答“方法现在有多可信、允许被怎样使用”。
5. `本能方法`、`现有方法树中的可执行方法/方法片段`、`待学习方法骨架` 是当前合法直接学习对象。
6. `因果信息`、`成功执行片段`、`稳定任务路径`、`稳定任务子树` 是合法方法原料，但不得裸进入正式学习执行。
7. `稳定任务路径 / 稳定任务子树` 必须先抽象出 `待学习方法` 或 `方法骨架`，再继续进入学习验证与稳定化。
8. `任务沉淀为方法` 是正式允许的一类方法出生路径，但它不是“整个任务直接变方法”，而是“从可复用任务片段里抽取方法结构”。
9. `任务沉淀为方法`、`因果转方法`、`从成功片段整理方法原料` 不是同一件事；它们只是不同原料驱动的新头出生路径。
10. 如果已有适配方法能承接新证据，应优先回补该方法的适配边界、条件模板和结果模板，而不是强制生成平行新头。
11. 任一新方法，最终都必须能分解回既有元函数、既有原生学习兜底函数，或它们的合法组合片段。
12. 当学习对象选择服务于 `任务管理方法` 分支时，外层仍须显式化 `最近根层裁决结果`、`当前缺口类型`、`当前下一步去向`。
13. 任一由学习补全、验证或升格的方法，一旦在当前治理轮被消费，仍必须回到外层重新做 `根层重判 -> 上下文刷新`。
14. 对 `自我动作动态信息` 生成的因果，允许先直接进入 `方法首节点出生判定`；若不存在适配旧方法，则可直接出生低成熟度方法首节点，再把该新头作为正式学习对象交给学习链。
15. 若某个方法头当前只有 `需求信息 / 目标特征类型`，则它属于 `需求位方法头`；这类头只用于显性化方法位，不属于合法直接学习对象。
16. `需求位方法头` 必须待 `动作骨架` 与 `条件侧` 补齐后，才允许转入正式学习执行。

可收敛为一句话：

`学习的直接对象统一是方法；因果、成功片段与任务结构都只能作为方法原料，必须先抽象出待学习方法，或由独立方法头出生判定先生成方法头，再进入正式学习与稳定化。`


## 4. 直接学习对象、方法原料、方法来源与方法成熟度阶段的区别

必须先区分四层：

### 4.1 学习对象

回答的是：

- 学习时最终拿什么方法来观察、补全、验证和稳定

例如：

- 本能方法
- 已有可执行方法首节点
- 方法片段
- 待学习方法骨架

必须补充固定：

- `因果实例`、`成功执行片段`、`任务路径`、`任务子树` 默认属于 `方法原料层`
- 这些原料必须先抽象出 `待学习方法骨架 / 方法首节点`，再进入正式学习执行
- 对 `自我动作动态因果`，若已足以触发方法头出生判定，应先由独立方法头出生判定生成低成熟度方法首节点；学习只负责在该新头已存在后做补全、验证与稳定化
- 若当前方法头仅有 `需求信息 / 目标特征类型`，则它仍属于 `需求位方法头`，不得直接作为学习执行对象；必须待动作骨架与条件侧补齐后，才转成可学习方法。

### 4.2 方法来源

回答的是：

- 最终沉淀出来的方法，主要是怎么来的

例如：

- 本能
- 外部导入
- 因果抽象
- 观察抽象
- 路径提炼

### 4.3 方法成熟度阶段

回答的是：

- 这个方法当前可信到什么程度
- 允许被怎样召回、执行和升格

例如：

- 运行期临时
- 反推候选
- 已验证稳定
- 组合成熟

必须固定：

- `学习对象` 是原料层
- `方法来源` 是出生归因层
- `方法成熟度阶段` 是可信度/可调用权限层
- 三者不能混用
- `自我动作动态` 与 `外界因果` 属于 `因果证据来源层`，用于支持方法出生与升格，不直接替代正式 `方法来源` 标签
- `需求驱动 / 目标驱动` 若只是表示先根据 `学习目标` 或 `方法需求位` 发起检索，再从状态列表中按同类特征筛出因果证据，则它应优先落在 `学习目标/候选方法出生机制` 层，不单独新增正式 `方法来源`
- 此时最终来源仍按主导原料归为 `因果抽象 / 观察抽象 / 路径提炼`
- 现有展示口径里的 `运行期临时 / 反推 / 组合` 可以视作过渡期的“来源-成熟度混合标签”；新规范讨论时，应优先把它们理解成 `方法成熟度阶段`
- 正式 `方法来源` 的精确分类，按 [0569_方法来源边界与正式来源分类规范.md](D:\鱼巢\规范\0569_方法来源边界与正式来源分类规范.md) 执行

### 4.4 与任务管理治理分支的边界

必须固定：

1. 本文描述的是全域“直接学习对象/方法原料 -> 方法来源/成熟度阶段”的共性口径。
2. 若服务对象是 `任务管理方法` 分支，则必须额外携带：
- `最近根层裁决结果`
- `当前缺口类型`
- `当前下一步去向`
3. 这三项属于治理上下文，不属于所有学习对象和所有方法来源的统一必填字段。

## 5. 当前方法来源与成熟度口径总表

### 5.1 方法来源

| 方法来源 | 含义 | 典型原料 | 当前口径 |
| --- | --- | --- | --- |
| `本能` | 开发者预置或先天固定的方法 | 本能动作、本能方法首节点 | 系统初始可直接调用 |
| `外部导入` | 外部明确注入的方法 | 开发者导入、外部定义 | 不是当前主学习路径 |
| `因果抽象` | 主要由因果证据抽象出的方法头 | 因果实例、稳定因果对、轻量因果链 | 优先回答“为什么这条方法成立” |
| `观察抽象` | 主要由重复观察、对比现象抽象出的方法头 | 观察结果、回执模式、状态对比 | 允许当前尚未形成稳定因果闭合 |
| `路径提炼` | 主要由稳定执行路径、任务路径、任务子树提炼出的方法头 | 成功片段、稳定步骤链、稳定任务子树 | 优先回答“这条方法怎样被组织出来” |
必须固定：

- 旧口径中的 `运行时学习沉淀` 只保留兼容总类意义，不再作为首选正式来源值。
- 正式来源的精确分类，优先写成以上 5 类之一。
- 若当前是“只有目标、还没有现成方法”的需求驱动生成场景，可先命中 `方法需求位`，再按状态列表同类特征筛证据；但最终落账时，仍只写以上 5 类正式来源之一。

### 5.2 方法成熟度阶段

| 方法成熟度阶段 | 含义 | 典型入口 | 调用纪律 |
| --- | --- | --- | --- |
| `运行期临时` | 新近出生、局部可用，但可信度尚低的方法 | 成功片段、自我动作动态首次压缩、初次因果提炼 | 允许局部试用，不应默认高信任召回 |
| `反推候选` | 证据较强、值得优先验证的候选方法 | 稳定因果、轻量因果链、动作动态多次同桶复现 | 允许受控召回和继续补证 |
| `已验证稳定` | 经稳定复现和结果确认，可作为常规方法使用 | 相似条件下重复得到同向成功结果 | 可常规参与召回与执行 |
| `组合成熟` | 已证明可复用并能压缩思考成本的高层成熟方法 | 稳定任务路径、稳定任务子树、稳定方法组合 | 可作为高层稳定结构参与筹办与推演 |

必须固定：

- `任务沉淀为方法` 初次沉淀时，若只是局部可复用，应先落到低成熟度阶段，如 `运行期临时`
- 若后续主要靠稳定任务结构沉淀并反复复现，则应优先升到 `组合成熟`
- 若后续主要靠稳定因果证据抽象，则更偏先进入 `反推候选`，再由验证进入更成熟阶段
- `自我动作动态` 优先作为高可信内部因果原料，支持 `运行期临时` 或 `反推候选` 方法头出生
- 对 `自我动作动态` 生成的因果，默认推荐承接顺序是：`因果形成 -> 方法头出生判定 -> 新生低成熟度方法头或回补旧方法 -> 再进入学习补全/验证`；学习不负责方法头出生本身
- `外界因果` 默认属于待验证外部原料，经确认、复现或结果验证后，才允许推动成熟度上升
- 工程上新增的成熟度信息，优先复用 `方法虚拟存在` 的特征类型表达，不轻易另造独立数据结构


## 6. 当前合法直接学习对象与方法原料总表

当前至少承认以下 `2 类直接学习对象 + 4 类方法原料`：

### 6.1 本能方法

含义：

- 系统已存在的原生可调用方法

作用：

- 作为最小学习起点
- 在没有更具体对象时，承担最小试探

### 6.2 现有方法树中的方法与方法片段

含义：

- 已存在的完整方法首节点、条件节点、结果节点，以及可局部复用的方法片段

作用：

- 复用已有能力
- 补条件、补结果、补局部步骤

补充固定：

- 若某个现有方法头当前只有 `目标特征类型`，还没有动作骨架与条件侧，则它仍按 `需求位方法头` 处理，不计入本条“现有可直接学习的方法”。

### 6.3 因果信息（方法原料）

含义：

- 稳定的 `因 -> 动作 -> 果` 证据

作用：

- 支持 `反推学习`
- 先抽象出候选方法或待学习方法，再进入后续验证与升格

### 6.4 成功执行片段（方法原料）

含义：

- 刚走通的一小段运行链

作用：

- 提炼 `待学习方法骨架`
- 再沉淀 `运行期临时` 方法

说明：

- 这类对象偏局部片段
- 当前 `从成功片段整理方法原料` 就属于这一类

### 6.5 稳定任务路径（方法原料）

含义：

- 某个任务在多个步骤上的稳定推进路径

作用：

- 可抽取步骤顺序
- 可抽取常见补条件顺序
- 可抽取稳定转域路径
- 可先抽象出 `待学习方法骨架`

说明：

- 这类对象比“成功执行片段”更长
- 也比“单条因果”更偏结构

### 6.6 稳定任务子树（方法原料）

含义：

- 某类任务在相似条件下反复长出的局部任务结构

作用：

- 可先抽象出 `待学习方法骨架`
- 再沉淀成较高层的组合方法
- 可复用为补结构方法、失败策略方法、收束复盘方法等

说明：

- 这类对象最接近“任务沉淀为方法”


## 7. 任务沉淀为方法的正式定义

本文正式补充以下来源：

`从任务路径或任务子树沉淀方法`

它的严格含义是：

- 从一段可复用的 `任务步骤链`
- 或一段稳定的 `任务子树`
- 抽取出可脱离当前宿主任务复用的治理结构
- 再将这段治理结构沉淀为候选方法

它不是：

- 整个任务头节点直接改名成一个方法
- 把当前任务所有细节原封不动拷进方法树
- 把一次偶然执行成功直接写成成熟组合方法


## 8. 任务沉淀为方法与相邻概念的边界

### 8.1 与因果转方法的边界

`因果转方法` 偏：

- 从稳定证据里抽象条件和结果
- 更接近“为什么这条方法成立”

`任务沉淀为方法` 偏：

- 从稳定任务结构里抽象步骤和组织方式
- 更接近“这条方法是怎么被组织出来的”

### 8.2 与从成功片段整理方法原料的边界

`成功片段整理方法原料` 偏：

- 局部
- 短链
- 刚运行成功的一小段

`任务沉淀为方法` 偏：

- 更长的步骤链
- 更完整的任务推进结构
- 可能包含多个子任务、多个检查点、多个转域节点

### 8.3 与组合升格的边界

`任务沉淀为方法` 是原料抽取或初次沉淀动作。

`组合成熟` 是后续稳定复现后的成熟度升级结果。

二者不能写成同一个动作。


## 9. 任务沉淀为方法的最小条件

只有同时满足以下条件时，才允许从任务沉淀为方法：

1. 该任务路径或任务子树在相似条件下反复出现。
2. 该结构能脱离当前宿主任务，抽取出通用步骤骨架。
3. 该结构最终能分解回既有元函数、既有本能函数或合法组合片段。
4. 该结构不是纯粹由外部偶然回执拼出来的不可复现链。
5. 若结构里包含外部动作跳，则仍受 `外部结果确认` 与 `最弱跳可信度` 约束。


## 10. 任务沉淀后的建议落账

建议后续统一按以下规则写来源与成熟度：

1. `第一次从任务路径/任务子树抽取出来`
   - 来源优先写成 `路径提炼`
   - 成熟度阶段默认写成 `运行期临时`
2. `如果主要证据来自稳定因果归纳`
   - 正式来源优先写成 `因果抽象`
   - 后续优先成熟为 `反推候选`，再进入 `已验证稳定`
3. `如果主要原料来自重复观察、对比模式或回执模式`
   - 正式来源优先写成 `观察抽象`
   - 成熟度阶段默认仍先保守写低成熟度阶段
4. `如果旧口径曾写成由学习过程综合生成新头`
   - 应回溯主导出生依据，优先改写为 `因果抽象 / 观察抽象 / 路径提炼`
   - 成熟度阶段仍按证据与复现单独裁判
5. `如果主要证据来自稳定任务结构复用`
   - 后续优先成熟为 `组合成熟`

必须禁止：

- 第一次沉淀就直接写成 `组合成熟`
- 只因为任务树很长，就跳过验证直接写成成熟方法
- 候选方法在当前治理轮被采用后，不回外层直接继续连锁升格


## 11. 当前实现对齐判断

当前代码与本文最接近的已有落点有 4 条：

1. `学习_尝试学习` 作为基础本能方法对象
2. `从成功片段整理方法原料`
3. `从因果实例沉淀方法`
4. `从运行机会沉淀候选方法`

其中：

- `从成功片段整理方法原料` 已较接近 `成功执行片段 -> 方法原料/待学习骨架`
- `从因果实例沉淀方法` 已较接近 `因果信息 -> 反推候选`
- `从运行机会沉淀候选方法` 已是 `任务沉淀为方法` 的最近现有口径

因此当前最准确的判断是：

`任务沉淀为方法` 在规范层已经应被正式承认；在实现层，它可先并入“从运行机会沉淀候选方法”的收口方向，再逐步细化为“从稳定任务路径沉淀”和“从稳定任务子树沉淀”。`


## 12. 收口结论

后续统一按以下口径执行：

1. 学习的直接对象统一收口为方法。
2. 方法来源不只包含本能与外部导入，还应精确区分 `因果抽象 / 观察抽象 / 路径提炼`。
3. `因果信息 / 成功执行片段 / 任务路径 / 任务子树` 是正式合法方法原料，但不得裸进入正式学习执行。
4. `任务路径 / 任务子树` 必须先抽象出 `待学习方法`，再继续学习。
5. `从任务沉淀为方法` 是正式允许的方法出生路径。
6. 它第一次沉淀默认应写成 `来源=路径提炼`、`成熟度阶段=运行期临时`，后续再按证据与复现路径成熟为 `反推候选 / 已验证稳定 / 组合成熟`。

最终统一收口为：

`学习执行统一面向方法；因果、成功片段与任务结构都先作为方法原料，经过方法抽象后再进入正式学习，其中“从任务结构学”正式收口为“从任务路径或任务子树沉淀方法”。`
