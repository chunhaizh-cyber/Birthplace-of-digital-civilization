## 3. 核心结论

必须固定以下结论：

1. `执行前门控` 是当前任务/方法/动作进入普通执行链前的最后一道正式检查。
2. `执行前门控` 不能只看一个字段，至少必须共同消费：
   - `根层重判结果`
   - `任务管理上下文`
   - `局部条件判定结果`
3. `是否允许继续推进` 不等于 `是否允许当前执行`；执行前门控必须优先读取后者。
4. `方法当前可用` 不等于 `当前允许执行`；局部可用性只能作为执行前门控的一部分输入。
5. 若命中根层禁止边界或仅允许边界处理，则执行前门控必须拦下普通执行链。
6. 若当前上下文显示应 `回到重筹办 / 转学习 / 挂起等待 / 进入收束`，执行前门控也不得强行继续执行。
7. 执行前门控只决定“当前是否允许进入普通执行链”，不直接承担根层最终裁决全部职责。

一句话收口：

`执行前门控是“根层是否放行 + 当前上下文是否具备 + 局部条件是否允许”三者共同成立后，当前任务才允许真正进入普通执行链的最后门。`


## 4. 定义

`执行前门控` 指的是：

- 在任务管理任务、执行器或方法调用真正进入普通执行链之前
- 对当前轮次是否允许执行所做的一次结构化门控判断

它回答的是：

- 当前是否允许进入普通执行
- 若不允许，是被根层拦下、被上下文分流，还是被局部条件拦下
- 当前应回到 `重筹办 / 学习 / 等待 / 收束` 中的哪一路

它不是：

- 根层重判本体
- 局部条件分析本体
- 最终裁决的全部内容


## 5. 最小输入面

### 5.1 根层输入

执行前门控至少必须直接消费以下根层字段：

- `重判时机`
- `是否允许当前执行`
- `是否命中根层禁止`
- `是否仅允许收束/待机/停止`
- `根层禁止原因类型`

这些字段统一来自 [0316_根层重判结果最小字段表规范.md](D:\鱼巢\规范\0316_根层重判结果最小字段表规范.md)。

### 5.2 任务管理上下文输入

执行前门控至少必须直接消费以下上下文字段：

- `当前管理对象指针`
- `当前步骤指针`
- `当前方法首节点指针`
- `当前缺口类型`
- `当前下一步去向`

这些字段统一来自 [1214_任务管理上下文最小字段表规范.md](D:\鱼巢\规范\1214_任务管理上下文最小字段表规范.md)。

### 5.3 局部条件输入

执行前门控至少必须直接消费以下局部条件结果：

- `当前方法是否存在`
- `当前方法是否可用`
- `当前条件是否满足`
- `是否无权限`
- `是否不可获取`
- `是否值未知需观察`

必须固定：

1. 这些内容首先属于局部条件判定结果，不是根层结果。
2. 它们只负责补充“当前上下文下能不能落执行”。


## 6. 标准判定顺序

执行前门控的标准顺序必须固定为：

```text
1. 先读根层重判结果
2. 再读任务管理上下文
3. 再读局部条件判定结果
4. 最后决定是否允许进入普通执行链
```

更细的最小判定顺序必须固定为：

1. `先检查根层重判结果是否存在`
2. `检查重判时机是否可用于当前执行前判断`
3. `若是否命中根层禁止 = true -> 直接禁止执行`
4. `若是否仅允许收束/待机/停止 = true -> 直接禁止普通执行`
5. `若是否允许当前执行 = false -> 直接禁止普通执行`
6. `检查当前管理对象 / 当前步骤 / 当前方法首节点是否齐备`
7. `若当前下一步去向已明确要求非执行路径 -> 禁止普通执行`
8. `检查局部条件是否满足、是否无权限、是否不可获取、是否值未知需观察`
9. `只有前三层都未拦下，才允许进入普通执行链`

必须固定：

1. 判定顺序以根层为先，上下文为次，局部条件为后。
2. 不允许把顺序倒过来，先看方法可用就直接执行。


## 7. 拒绝执行时的最小输出

执行前门控若拒绝执行，最少应输出：

- `是否允许进入普通执行链 = false`
- `拒绝来源层级`
- `拒绝原因类型`
- `建议回流路径`

其中：

- `拒绝来源层级` 只能是：
  - `根层`
  - `上下文层`
  - `局部条件层`

- `建议回流路径` 最少应能落到：
  - `回到重筹办`
  - `转入学习`
  - `挂起等待`
  - `进入收束/待机/停止边界`

更细的 `拒绝原因类型` 正式值表与建议回流路径映射，按 [0318_执行前门控拒绝原因类型表规范.md](D:\鱼巢\规范\0318_执行前门控拒绝原因类型表规范.md) 执行。


## 8. 三层拦截解释

### 8.1 根层拦截

典型场景：

- 命中 `违规 / 违法 / 致命性安全边界`
- 当前仅允许进入收束/待机/停止链

处理方式：

- 直接禁止普通执行
- 不再进入局部方法执行判定

### 8.2 上下文拦截

典型场景：

- 当前管理对象为空
- 当前步骤缺失
- 当前方法首节点缺失
- 当前下一步去向已明确不是执行链

处理方式：

- 直接返回治理承接层
- 优先走 `重筹办 / 学习 / 等待 / 收束`

### 8.3 局部条件拦截

典型场景：

- 当前方法不可用
- 当前条件不满足
- 当前无权限
- 当前不可获取
- 当前值未知需观察

处理方式：

- 进入补条件、观察、学习或等待链
- 不得伪装成“已经允许执行”


## 9. 严格禁止的简化

以下做法必须禁止：

1. 只看 `是否允许继续推进` 就直接执行。
2. 只看 `方法当前可用` 就直接执行。
3. 只看 `当前下一步去向 = 继续下一步` 就直接执行。
4. 在 `当前方法首节点` 缺失时仍直接进入执行。
5. 命中根层禁止后仍继续做普通执行尝试。
6. 把执行前门控退化成“当前代码能不能跑通”的工程判断。
