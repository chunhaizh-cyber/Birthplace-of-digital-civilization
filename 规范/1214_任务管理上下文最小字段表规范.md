- 哪些属于 `条件判定结果层`

## 3. 核心结论

必须固定以下结论：

1. `任务管理上下文` 不是一个长期保留的黑箱输入包，而是一组可拆、可镜像、可恢复的稳定治理中间态。
2. `任务管理上下文` 的最小字段，必须优先服务 `任务管理任务的一步治理承接`，而不是服务调试方便。
3. `根层重判结果` 属于 `任务管理上下文` 的上位输入，但不等于整个上下文本体。
4. `宿主任务头 / 当前步骤 / 当前方法 / 最近结果` 这类字段，本质上是 `任务侧真值镜像` 进入治理上下文，不是管理者私有真值。
5. `当前缺口类型`、`当前下一步去向`、`最近功能域`、`最近总控结果` 属于治理承接层的稳定中间态。
6. `当前场景`、`任务虚拟存在特征快照`、`原始反馈包`、`调度时间` 等，只是本轮请求补充材料，不应直接冒充稳定上下文字段。
7. `控制输入` 若仍以黑箱形态存在，也必须在语义上拆解回本文定义的字段，不允许长期以黑箱代替字段规范。
8. `任务管理上下文` 中允许携带候选去向和最近治理结论，但这些内容仍不是最终裁决。

一句话收口：

`任务管理上下文是“根层已放行后，任务管理任务继续承接宿主任务所需的最小稳定治理中间态”，不是任意调度材料的打包袋。`


## 4. 定义

`任务管理上下文` 指的是：

- 自我线程在完成 `根层重判`、`需求整理`、`结果归并` 之后，为 `任务管理任务` 组装的一组最小治理字段

它回答的是：

- 当前在管谁
- 当前围绕哪个主需求治理
- 当前任务已经推进到哪里
- 当前最主要的缺口是什么
- 当前更像应该往哪继续
- 最近一次治理和反馈留下了什么

它不是：

- 根层裁决本体
- 原始控制输入黑箱
- 叶子执行返回值整包
- 调度器内部所有瞬时状态

### 5.2 当前阶段建议优先级

若当前工程上只能先补最少一批稳定字段，优先顺序固定为：

1. `最近根层重判结果`
2. `当前管理对象指针`
3. `当前缺口类型`
4. `当前下一步去向`
5. `最近反馈类型`
6. `最近反馈摘要`
7. `当前步骤指针`
8. `当前方法首节点指针`
9. `最近结果节点指针`

原因是：

- 前 6 项最直接决定 `任务管理任务` 这一轮能否正确承接。
- 后 3 项更直接服务结构承接与执行衔接。


## 6. 只属于本轮请求的补充材料

以下内容允许进入 `任务管理请求`，但默认不应直接当成稳定上下文字段：

- `当前场景`
- `宿主任务虚拟存在特征快照`
- `当前功能域判定输入`
- `原始反馈包 / 反馈快照整包`
- `调度时间`
- `提交序号`
- `标签`

必须固定：

1. 这些内容主要服务本轮执行、判域、调度或调试。
2. 它们可以进入请求，也可以只保留在实例堆。
3. 除非经过拆解，不得直接镜像进宿主任务虚拟存在充当长期治理真值。


## 7. 与请求、实例、快照的关系

三者关系应固定为：

```text
根层重判结果
+ 任务管理上下文最小稳定字段
+ 本轮请求补充材料
= 一轮任务管理请求
```

因此必须固定：

1. `任务管理请求` 大于 `任务管理上下文`。
2. `任务管理上下文` 是请求里的稳定语义核心。
3. `恢复快照` 只从 `任务管理上下文` 和少量请求辅助字段中提取最小恢复集，不整包复制请求。
4. `宿主任务虚拟存在` 主要镜像 `任务管理上下文` 中可观察、可恢复、可审计的部分。


## 8. 严格禁止的混层

以下做法必须禁止：

1. 把 `控制输入` 黑箱直接当成 `任务管理上下文` 本体。
2. 把 `当前场景` 或 `特征快照整包` 直接当成稳定上下文字段。
3. 把 `当前下一步去向` 直接当成最终裁决。
4. 把 `最近功能域` 直接当成根层方向。
5. 把 `最近反馈摘要` 直接当成需求已满足结论。
