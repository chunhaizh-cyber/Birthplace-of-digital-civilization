# 1000 任务执行类实现规范（当前顺序执行骨架）

## 1. 目标

本文用于固定 `任务执行器` 的唯一职责：

`执行当前步骤，并把结果写入当前结果节点。`


## 2. 核心结论

必须固定以下结论：

1. `任务执行器` 不负责定义任务目标。
2. `任务执行器` 不负责完善整棵任务树。
3. `任务执行器` 只消费已经被筹办好的 `当前步骤 + 当前结果节点 + 方法实例节点`。
4. `任务执行器` 的输出，必须先写回结果节点，再由外层判断是否完成或回到筹办。


## 3. 最小输入

执行当前步骤至少需要：

1. 当前任务头
2. 当前步骤节点
3. 当前结果节点
4. 当前步骤绑定的方法头
5. 当前步骤上的方法实例条件节点
6. 当前结果上的方法实例结果节点
7. 任务虚拟存在与方法虚拟存在上的运行特征


## 4. 最小执行链

当前统一执行链固定为：

```text
读取当前步骤
-> 读取步骤方法
-> 读取任务虚拟存在 / 方法虚拟存在特征
-> 读取方法实例条件节点
-> 执行方法
-> 写入方法实例结果节点
-> 写入当前实际结果节点（若原节点是预测结果，则新建实际结果节点）
-> 若执行方法运行中发现额外缺口，则提出前置任务需求并等待自我线程实例化回挂
-> 若实际结果偏离命中预测，则返回外层转入待重筹办
-> 返回外层比较目标是否达成
```


## 5. 三类任务的执行区别

### 5.1 普通任务

- 执行现实步骤，推进现实存在的目标特征

### 5.2 任务管理任务

- 执行治理步骤，推进宿主任务向完成态靠近

### 5.3 学习任务

- 执行学习步骤，推进目标方法向可用态靠近


## 6. 禁止项

禁止以下做法：

1. 让 `任务执行器` 直接承担完整筹办逻辑。
2. 让 `任务执行器` 直接生成主步骤任务。
3. 让 `任务执行器` 直接实例化前置任务头；它只能提出前置任务需求，由自我线程创建并挂回当前步骤。
4. 在没有当前结果节点的情况下直接执行。
5. 在没有方法实例条件节点的情况下把方法模板直接当执行实例使用。
6. 把预测结果节点直接覆写成实际结果节点，导致预测与实际无法并存。
7. 把结果节点的下一步继续写在 `后续任务`，而不是显式写到 `下一跳步骤节点`。
