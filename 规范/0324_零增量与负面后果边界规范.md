# 0324 零增量与负面后果边界规范

## 1. 目标

本文用于把 `零增量` 与 `负面后果` 在当前体系中的边界单独固定下来。

本文要回答的是：

- 什么时候应判为 `零增量`
- 什么叫 `负面后果`
- 为什么 `负面后果` 应分成 `禁止项后果` 与 `经验项后果`
- 为什么正式规范里不再使用 `惩罚` 作为上位术语

## 3. 核心结论

必须固定以下结论：

1. `零增量` 属于 `结算层` 结果，回答“本轮不直接增加安全值或服务值”。
2. `负面后果` 属于 `后果层` 总称，回答“这轮结果触发了什么负向后果”。
3. `负面后果` 当前正式分为两类：
   - `禁止项后果`
   - `经验项后果`
4. `禁止项后果` 来自禁止项、根层边界和执行前放行失败，不属于经验积累。
5. `经验项后果` 来自实践结果回流，属于经验积累与治理回写。
6. `零增量` 不等于 `负面后果`。
7. `禁止项后果` 不等于 `经验项后果`。
8. `经验项后果` 只保留一个正式入口：`实践结果回流 -> 主循环统一判定 -> 生成经验项后果`。
9. `惩罚` 不再作为正式上位术语使用；若后续需要具体命名，应落到 `负面后果` 的具体子项里表达。

一句话收口：

`零增量回答“本轮不直接给双值正向增量”，负面后果回答“这轮产生了什么负向后果”；负面后果再分为禁止项后果和经验项后果。`


## 4. 最小定义

### 4.1 零增量

`零增量` 指的是：

- 本轮已完成比较、结算资格检查和增量类型判定
- 但当前不允许直接形成 `安全正向增量` 或 `服务正向增量`

它回答的是：

- 这轮是否应直接给双值正向回写

它不回答：

- 是否命中禁止项
- 是否要增加学习压力
- 是否要记经验项后果


### 4.2 负面后果

`负面后果` 指的是：

- 本轮结果触发的负向后果总称

它不直接等于：

- 双值负增量
- 单一失败布尔值
- 单一惩罚动作

必须固定：

1. `负面后果` 是总称。
2. 具体使用时必须继续分成 `禁止项后果` 与 `经验项后果`。


### 4.3 禁止项后果

`禁止项后果` 指的是：

- 因命中 `根层禁止项`、`违规违法边界`、`致命性边界`、`仅允许收束/待机/停止` 等上位边界而产生的后果

它回答的是：

- 当前路径为什么不允许继续
- 当前是否必须转入收束、待机、停止或其它边界处理

它不是：

- 经验积累结果
- 学习反馈结果


### 4.4 经验项后果

`经验项后果` 指的是：

- 本轮 `实践结果` 在主循环统一判定后形成的负向经验后果

这里的 `实践结果` 当前至少包括：

- 方法实际执行后的返回结果
- 动作实际执行后的返回结果
- 学习验证中的真实试验结果
- 外部观察、宿主回流或环境回流中的真实结果

它回答的是：

- 后续治理上是否应更谨慎
- 是否应增加学习压力、重试压力、失败记忆、风险记忆
- 是否应形成更保守的治理侧回写

必须固定：

1. `经验项后果` 不是预测层概念。
2. `经验项后果` 不是执行前门控概念。
3. `经验项后果` 不能仅由“预计会失败”“当前不放行”“当前候选被淘汰”直接生成。


## 5. 何时判为零增量

当前至少以下情况，应优先落为 `零增量`：

1. 当前需求不具直接结算资格
2. 当前需求不是叶子需求
3. 当前结果仅是局部成功，不是叶子首次满足
4. 对齐结果为 `无推进`
5. 对齐结果为 `反向`
6. 所属根树不明确

必须固定：

1. `零增量` 的意思是“不直接给正向结算”。
2. `零增量` 不自动等于“已经产生禁止项后果”。
3. `零增量` 也不自动等于“已经产生经验项后果”。


## 6. 何时产生禁止项后果

当前至少以下情况，应形成 `禁止项后果`：

1. 命中违规违法或根层禁止边界
2. 命中致命性安全边界
3. 根层重判结果表明当前仅允许收束、待机或停止
4. 执行前门控因上位边界拒绝执行

`禁止项后果` 当前可表现为：

- 拒绝执行
- 强制收束
- 待机
- 停止
- 边界处理

必须固定：

1. `禁止项后果` 可以在实践之前就成立。
2. `禁止项后果` 不是经验项后果。


## 7. 何时产生经验项后果

当前至少以下情况，应形成 `经验项后果`：

1. 方法执行失败
2. 执行结果为 `无推进`
3. 执行结果为 `反向`
4. 发生异常、超时、不可恢复错误
5. 当前虽未命中禁止项，但已显示出明显风险、失败或不可用性

`经验项后果` 当前可服务于：

- 学习压力上升
- 重试压力上升
- 风险记忆积累
- 谨慎度上升
- 方法可靠度下降候选
- 后续更保守的治理路径选择

必须固定：

1. `经验项后果` 可以存在而不伴随直接双值扣减。
2. 上述情况只有在形成 `实践结果回流` 之后，才允许进入正式经验项后果判定。


## 7.1 经验项后果的唯一正式入口

当前 `经验项后果` 只保留一个正式入口：

```text
实践结果回流
-> 主循环统一消费消息
-> 主循环统一判定
-> 生成经验项后果
```

必须固定：

1. 执行线程不得各自生成正式经验项后果真值。
2. 学习线程不得各自生成正式经验项后果真值。
3. 宿主/外部回流也只是 `实践结果消息`，不是正式经验项后果本体。
4. 只有主循环的统一结果归并阶段，才允许把实践结果正式落成 `经验项后果`。


## 8. 三者的最小关系表

| 情况 | 零增量 | 禁止项后果 | 经验项后果 |
| --- | --- | --- | --- |
| 叶子首次满足且方向成立 | 否 | 否 | 通常否 |
| 非叶子或不具资格 | 是 | 否 | 可有可无 |
| 无推进 | 是 | 否 | 是 |
| 反向 | 是 | 否 | 是 |
| 普通失败 | 通常是 | 否 | 是 |
| 命中禁止项/致命边界 | 通常是 | 是 | 可有可无 |

必须固定：

1. 该表是最小边界表，不是所有策略细节表。
2. `命中禁止项/致命边界` 时，是否同时产生经验项后果，取决于是否已有实践结果回流。


## 9. 与双值结算的关系

双值结算当前直接负责的是：

- 判定是否形成正向增量
- 判定是否只能落为零增量

双值结算当前不直接等于：

- 禁止项后果系统
- 经验项后果系统

因此必须固定：

1. `零增量` 可以直接由双值结算层给出。
2. `禁止项后果` 属于上位边界与门控后果。
3. `经验项后果` 属于实践回流后的治理后果。
4. 不得把 `负面后果` 偷换成“负增量就是结算的一部分”，除非后续另立正式规则。


## 10. 与主循环和自我实现的关系

主循环可以统一消费：

- `零增量`
- `禁止项后果`
- `经验项后果`

但必须分别处理：

- `零增量` 用于双值不作正向回写
- `禁止项后果` 用于拒绝执行、收束、待机、停止与边界处理
- `经验项后果` 用于学习、重试、谨慎度与治理侧回写

必须固定：

1. 主循环可以同时看到三者，但不得把三者压扁成一个 `失败` 布尔值。
2. 主循环还必须承担 `实践结果消息 -> 正式经验项后果` 的唯一入口职责。
3. `风险后果` 若要使用，应作为 `负面后果` 的具体后果描述，不单独升为并列总层。


## 11. 严格禁止的混层

以下做法必须禁止：

1. `零增量 = 负面后果`
2. `禁止项后果 = 经验项后果`
3. `预测失败 -> 直接生成经验项后果`
4. `执行前门控拒绝 -> 直接生成经验项后果`
5. `线程本地失败 -> 直接写成正式经验项后果`
6. `普通失败 -> 直接当成双值负增量`
7. 继续把 `惩罚` 当正式上位术语使用
