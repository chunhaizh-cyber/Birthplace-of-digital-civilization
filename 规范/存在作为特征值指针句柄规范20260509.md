# 存在作为特征值指针句柄规范 20260509 修订版

## 1. 总原则

存在、场景、动态、任务、方法等基础信息节点作为**特征值**时，统一使用 `特征值` 中已有的 `指针句柄` 分支承载。

定稿口径：

```text
存在作为特征值时，就用 特征值::指针句柄。
不新增 特征节点主信息类::当前值信息。
不新增 存在形态。
本规范只约束“特征值承载”。
实例概念归属以《存在概念与实例创建最小闭环规范》中的 概念集 为第一阶段权威结构。
```

当前结构已经足够：

```cpp
using 特征值 = std::variant<std::monostate, I64, VecU句柄, 指针句柄>;

class 特征节点主信息类 : public 基础信息基类 {
public:
    特征值 当前值{};

    bool 有当前值() const noexcept;
    void 设置当前值(const 特征值& v);
};
```

因此不再给 `特征节点主信息类` 增加：

```cpp
可解析引用<基础信息节点类> 当前值信息{};
```

该字段会与 `指针句柄` 职责重复，使结构变厚。

## 2. 特征节点结构

特征节点统一表达为：

```text
特征节点 = 特征类型 + 特征值
```

值侧承载规则：

| 值类型 | 承载 |
| --- | --- |
| 无值 | `std::monostate` |
| 数值 | `I64` |
| 向量证据 | `VecU句柄` |
| 存在 / 场景 / 动态 / 任务 / 方法 / 其他基础信息节点引用 | `指针句柄` |

也就是：

```text
数值是值。
向量证据是值。
另一个基础信息节点也是值。
```

只是它们分别落在：

```text
I64
VecU句柄
指针句柄
```

三个分支里。

## 3. 特征类型与特征值的边界

需要区分：

```text
存在作为特征类型
存在作为特征值
```

### 3.1 存在作为特征类型

存在作为特征类型时，它位于特征节点的“类型侧”，不进入 `当前值`。

示例：

```text
颜色、当前位置、占用对象、所属场景、概念
```

这些是特征类型。

第一阶段可继续使用现有结构表达类型侧：

```text
1. 基础信息基类::名称 / 类型 作为语言锚点。
2. 特征节点主信息类::抽象特征 作为规则或语义锚点。
```

如果某个“存在”需要作为严格的特征类型锚点，可以让 `抽象特征` 指向对应的基础信息节点。第一阶段不新增 `特征类型存在` 字段。

### 3.2 存在作为特征值

存在作为特征值时，进入：

```cpp
特征节点主信息类::当前值
```

并使用：

```cpp
指针句柄
```

示例：

```text
B-1234飞机.当前位置 = 101机位
101机位.占用对象 = B-1234飞机
B-1234飞机.所属场景 = 当前机场场景
```

其中：

```text
当前位置、占用对象、所属场景 = 特征类型
101机位、B-1234飞机、当前机场场景 = 特征值
```

## 4. 指针句柄使用规则

`指针句柄` 中的指针必须指向：

```text
基础信息节点类* 或其别名节点类型
```

允许：

```text
存在节点类*
场景节点类*
动态节点类*
状态节点类*
特征节点类*
二次特征节点类*
任务节点类*
方法节点类*
```

禁止：

```text
1. 指向 基础信息基类* 主信息对象。
2. 指向栈上临时对象。
3. 指向外部无生命周期保障的业务对象。
4. 指向已经从世界树 / 仓库中删除的节点。
```

生命周期要求：

```text
被指向节点的生命周期不得短于持有该指针特征值的特征节点。
```

如果节点被删除，删除流程应负责清理、失效或重建相关指针特征值。第一阶段可以只规定约束，不强制实现全局反向清理。

## 5. 推荐工具函数

为避免业务代码到处手写 `reinterpret_cast`，建议提供轻量工具函数。

```cpp
inline 特征值 构造指针特征值(基础信息节点类* 节点) noexcept
{
    return 指针句柄{
        reinterpret_cast<std::uintptr_t>(节点)
    };
}

inline bool 特征值是指针(const 特征值& 值) noexcept
{
    return std::holds_alternative<指针句柄>(值)
        && std::get<指针句柄>(值).有效();
}

inline 基础信息节点类* 取特征值基础信息节点(const 特征值& 值) noexcept
{
    if (!std::holds_alternative<指针句柄>(值)) {
        return nullptr;
    }

    const auto& 句柄 = std::get<指针句柄>(值);
    if (!句柄.有效()) {
        return nullptr;
    }

    return reinterpret_cast<基础信息节点类*>(句柄.指针);
}

template<class T节点>
inline T节点* 取特征值节点(const 特征值& 值) noexcept
{
    return reinterpret_cast<T节点*>(取特征值基础信息节点(值));
}
```

使用方式：

```cpp
特征->设置当前值(构造指针特征值(机位101节点));

auto* 当前机位 = 取特征值节点<存在节点类>(特征->当前值);
```

说明：

```text
当前工程中 存在节点类、场景节点类、特征节点类 等都是 基础信息节点类 的别名，
所以模板函数主要用于表达调用意图。
如果后续节点类型改成真实继承结构，应增加主信息类型检查。
```

## 6. 示例

### 6.1 位置特征

```text
B-1234飞机.当前位置 = 101机位
```

```cpp
auto* 位置特征 = ...;
位置特征主信息->设置当前值(构造指针特征值(机位101节点));
```

### 6.2 占用对象特征

```text
101机位.占用对象 = B-1234飞机
```

```cpp
auto* 占用对象特征 = ...;
占用对象特征主信息->设置当前值(构造指针特征值(B1234飞机节点));
```

### 6.3 所属波次特征

```text
B-1234飞机.所属波次 = 今日第一保障波次
```

```cpp
auto* 所属波次特征 = ...;
所属波次特征主信息->设置当前值(构造指针特征值(今日第一保障波次节点));
```

## 7. 概念归属边界

本规范不规定实例概念归属的权威结构。

第一阶段定稿为：

```text
实例概念归属的权威结构 = 存在节点主信息类::概念集
```

该口径由《存在概念与实例创建最小闭环规范》约束。

为了查询统一性，可以把概念归属镜像成特征节点：

```text
B-1234飞机.概念 = 飞机
B-1234飞机.概念 = 交通工具
B-1234飞机.概念 = 机场资源
```

这些镜像特征的值仍然使用：

```cpp
特征值::指针句柄
```

但要注意：

```text
概念特征节点只是镜像 / 索引 / 查询入口，不是第一阶段权威结构。
概念集 才是实例概念归属的权威结构。
```

因此不得在同一份规范中同时写成：

```text
概念归属既完全由 概念集 决定，又完全不是存在字段。
```

正确口径是：

```text
概念归属第一阶段以 概念集 为权威；
如果需要统一特征查询，可以从 概念集 派生或同步概念特征节点。
```

## 8. 多值特征

如果同一个特征类型允许多个值，第一阶段采用最简单结构：

```text
同一主体下挂多个同类型特征节点。
```

示例：

```text
B-1234飞机
├─ 特征：概念 = 飞机
├─ 特征：概念 = 交通工具
└─ 特征：概念 = 机场资源
```

不在第一阶段强行设计：

```text
1. 多值容器字段。
2. 当前值信息字段。
3. 动态继承表。
4. 冲突特征集。
5. 复杂归属记录。
```

后续如果需要稳定表达多值，可考虑：

```text
1. 同类型多特征节点。
2. 稳态列表 / 候选窗口记录多个指针值。
3. 专用查询函数聚合同类型特征节点。
```

但这些都不改变 `特征节点主信息类` 的基本结构。

## 9. 持久化边界

`指针句柄` 只适合运行期世界树内引用。

第一阶段：

```text
指针句柄 = 运行期引用
不处理跨进程、跨重载、跨仓库持久化
```

第二阶段再处理：

```text
1. 指针句柄 + 节点主键。
2. 保存时把指针值转换成节点主键。
3. 重载时按节点主键重建运行期指针。
```

持久化问题不得提前塞进 `特征节点主信息类`，也不得因此新增 `当前值信息` 字段。

如果 `特征快照值` 中包含 `指针句柄`，则该快照只在当前运行期有效。持久化保存时应转换为节点主键或忽略该指针值。

## 10. 禁止项

禁止：

```text
1. 给 特征节点主信息类 新增 当前值信息 字段。
2. 为“存在作为值”新增专用字段。
3. 为概念 / 实例区分新增 存在形态枚举。
4. 为同一特征多值新增复杂归属记录。
5. 在第一阶段处理指针持久化。
6. 把指针句柄指向主信息对象、临时对象或外部无生命周期保障对象。
7. 把概念特征镜像当作 概念集 的替代权威结构。
```

允许：

```text
1. 用 指针句柄 表示存在 / 基础信息节点作为特征值。
2. 用多个同类型特征节点表示多值。
3. 用工具函数封装 指针句柄 构造和读取。
4. 后续在保存 / 读取层处理指针与主键转换。
5. 为查询方便，从 概念集 同步概念特征节点镜像。
```

## 11. 验收标准

实现应满足：

```text
1. 不新增 特征节点主信息类::当前值信息。
2. 不新增 枚举_存在形态。
3. 可以用 指针句柄 表达 B-1234飞机.当前位置 = 101机位。
4. 可以用 指针句柄 表达 101机位.占用对象 = B-1234飞机。
5. 指针特征值读写通过轻量工具函数完成。
6. 指针句柄指向 基础信息节点类*，不指向主信息对象。
7. 概念归属按存在概念规范使用 概念集 为权威结构。
8. 如生成概念特征节点，只作为 概念集 的镜像或查询索引。
```

## 12. 最终定稿

```text
特征系统统一承载所有值：
数值是 I64；
向量证据是 VecU句柄；
存在和基础信息节点引用是 指针句柄。

概念归属第一阶段以 概念集 为权威；
概念特征节点可以作为镜像或查询索引，但不替代 概念集。
```
