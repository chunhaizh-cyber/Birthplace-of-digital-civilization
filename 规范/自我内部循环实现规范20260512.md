## 1. 核心定义

自我内部循环是自我在没有外部输入时，基于自身状态持续完成治理的最小运行链。

标准链路：

```text
自我核心状态
-> 安全 / 服务压力
-> 根需求
-> 需求树生长
-> 当前主需求
-> 任务发起意图
-> 任务信息节点
-> 任务虚拟存在
-> 任务筹办
-> 当前方法 / 本能动作
-> 任务执行
-> 输出结果场景 / 运行回执
-> 任务状态推进
-> 需求满足与结算
-> 自我核心状态变化
-> 下一轮主需求重判
```

一句话：

```text
自我内部循环 = 自我用自己的状态生成需求，用需求发起任务，用任务调用方法，用方法结果改变状态，再用新状态重判需求。
```

本规范只定义“内部循环如何实现”，不把它等同于自我苏醒。自我苏醒必须在内部循环连续多轮稳定后再判定。

## 2. 分层职责

### 2.1 自我线程

自我线程负责：

```text
1. 消费治理消息。
2. 落任务管理上行消息、派生需求消息、自检报告。
3. 维护需求树。
4. 刷新需求权重。
5. 选择当前主需求。
6. 生成任务发起意图。
7. 生成任务调度意图。
8. 根据任务回执回写来源需求。
9. 结算安全 / 服务价值。
10. 决定下一轮继续推进、待机或休眠期修复门控。
```

自我线程不得直接写：

```text
任务状态
当前方法
任务候选方法集
可执行输入参数场景
筹办缺口类型
执行缺口类型
任务虚拟存在运行镜像
```

### 2.2 任务管理界面线程

任务管理界面线程负责：

```text
1. 接收自我线程任务发起 / 调度 / 状态推进意图。
2. 调用任务域提交本能方法创建任务信息节点。
3. 调用任务域提交本能方法创建并维护任务虚拟存在。
4. 初始化任务语义。
5. 构造任务工作包。
6. 调用任务管理工作线程。
7. 接收任务管理工作线程结果。
8. 调用同步筹办 / 执行回执到任务的提交本能方法。
9. 调用提交任务状态变化本能方法。
10. 上报任务管理消息给自我线程。
```

任务管理界面线程是任务域调度层，不是动作动态来源。

任务状态、当前方法、候选方法集、可执行输入参数场景、筹办缺口和执行缺口等任务域特征，必须由任务域提交本能方法写入；任务管理界面线程不得作为方法动作动态的动作来源。

### 2.3 任务管理工作线程

任务管理工作线程负责：

```text
1. 按工作包推进模式执行一轮。
2. 调用任务筹办生命周期普通函数。
3. 调用任务执行生命周期普通函数。
4. 读取运行存在中的任务层回执。
5. 读取输出结果场景中的世界事实摘要。
6. 生成任务工作结果。
7. 生成派生需求消息和结果状态消息。
```

任务管理工作线程不得裸写任务状态、需求树或世界事实。

任务状态变化必须由任务域提交本能方法完成。工作线程只负责编排一次工作包推进，不得把线程自身当作动作来源。

### 2.4 本能方法

内部循环中的任务层本能方法分为过程本能方法和提交本能方法。

过程本能方法至少包括：

```text
任务筹办
任务执行
改变特征值
检查需求是否满足
查找特征类型
配对参数
尝试学习
定向学习
练习
```

提交本能方法至少包括：

```text
承接需求创建任务
同步筹办回执到任务
同步执行回执到任务
提交任务状态变化
挂起任务等待子需求
唤醒任务重筹办
取消任务
任务超时处理
```

其中：

```text
承接需求创建任务：创建任务信息节点和任务虚拟存在，任务状态 = 未启动。
任务筹办：形成任务可就绪的筹办回执。
同步筹办回执到任务：把筹办回执写入任务虚拟存在。
提交任务状态变化：提交任务状态变化并生成任务状态动作动态。
挂起任务等待子需求：将依赖子需求的任务推进到等待中。
唤醒任务重筹办：在子需求满足、方法可用或参数补齐后，将任务推进到待重筹办。
任务执行：调用当前方法，并在来源需求目标满足时形成任务可完成的执行回执。
同步执行回执到任务：把执行回执写入任务虚拟存在。
取消任务：将任务状态推进到取消。
任务超时处理：将任务状态推进到超时。
改变特征值：产生真实世界事实变化。
检查需求是否满足：判断输出事实是否满足需求目标。
学习类本能方法：补齐目标方法的条件结果对、动作、参数或稳定性。
```

凡生成方法动作动态的变化，必须由明确本能方法或动作骨架入口作为动作来源。

## 3. 需求目标与三通道数据边界

### 3.1 需求主体、目标宿主、治理主体

本规范固定区分三类主体：

```text
需求主体：
    需求在自我内部的细化发起点或视角主体。
    可以是自我存在、自我子存在、自我虚拟存在、任务虚拟存在、方法虚拟存在，
    或外部交互者在自我内部的表征存在。

目标宿主 / 被需求存在：
    目标特征依附在哪个存在上。
    改变的不是该存在本身，而是该存在上的某个特征状态。

治理主体：
    本项目中固定为自我。
```

完整原子目标固定为：

```text
被需求存在
目标特征类型
当前状态
目标状态
满足关系
```

需求目标表达为：

```text
被需求存在.目标特征类型 -> 目标状态
```

若缺少被需求存在，则目标特征类型只是悬空抽象，不能形成可执行需求。若缺少目标特征类型，则被需求存在没有明确可改变项，也不能形成可执行需求。

例：

```text
需求主体 = 交互者表征存在
被需求存在 = 自我回答结果
目标特征类型 = 满足程度
目标状态 = 已满足
```

这不是自我在治理交互者自己的需求树，而是自我在自己的信息域里记录：交互者表征存在需要自我回答结果的满足程度达到已满足。

### 3.2 三通道数据边界

内部循环中的本能函数必须遵守三通道：

```text
运行存在：
    记录本轮方法运行事实、输入参数场景、输出结果场景和运行摘要。

输出结果场景：
    给世界树提交层使用。
    只承载真实世界事实、特征变化、状态节点、动态事实、结果状态。

方法虚拟存在：
    给方法自身记忆使用。
    只承载本方法运行账、最近运行、学习产物、条件结果对索引。

任务虚拟存在：
    给任务域状态和任务语义镜像使用。
    只能由任务域提交本能方法写入任务状态、当前方法、候选方法集、可执行输入参数场景、筹办缺口类型、执行缺口类型、最近运行存在等任务域特征。
```

任务筹办和任务执行不得：

```text
1. 直接生长需求树。
2. 把任务状态写入输出结果场景。
3. 直接写任务状态。
4. 直接改任务虚拟存在。
5. 把世界事实写入任务虚拟存在。
6. 把方法自身运行账写入任务虚拟存在。
```

2026-05-13 修订：

```text
任务筹办 / 任务执行不是本能方法。
它们是任务管理生命周期普通函数，只形成过程回执，不进入方法树和候选方法集。
```

它们的直接结果是过程回执，而不是任务状态特征的最终提交。

能写任务账的是任务域提交本能方法；能写需求树的是需求域本能方法；能写世界事实的是世界域动作方法；能写方法状态和学习产物的是方法域动作方法。

## 4. 主循环阶段

自我线程主循环按现有代码骨架固定为：

```text
1. 冻结本轮消息批次。
2. 规范化消息并落一次特征账。
3. 重算受影响二次特征。
4. 刷新线程上下文并生成治理帧。
5. 整理需求并做根层重判。
6. 生成并执行主派发决议。
7. 执行休眠期自检报告修复门控。
```

当前代码入口：

```cpp
自我线程类::执行主循环一轮_(...)
```

各阶段不得跳过职责边界。

## 5. 消息归并阶段

消息归并阶段处理：

```text
任务管理派生需求消息
任务管理结果状态消息
线程状态变化消息
自检报告
普通治理变化项
```

要求：

```text
1. 任务管理派生需求消息只进入需求树生长入口。
2. 自检报告只记录缺失并报送自我线程，不直接生成修复需求。
3. 任务管理结果状态消息可触发需求回写和结算，但不得由自我线程写任务状态。
```

当前代码入口：

```cpp
步骤_规范化消息并落一次特征账_(...)
私有_落任务管理派生需求到自我内部世界(...)
私有_落任务管理结果到自我内部世界(...)
```

## 6. 需求层闭合

需求层闭合是内部循环的第一道运行门槛。

必须完成：

```text
1. 读取安全值、服务值。
2. 计算安全压力、服务压力。
3. 刷新安全根需求、服务根需求权重。
4. 根据任务回执缺口生长子需求。
5. 去重 / 合并同目标需求。
6. 刷新管理需求 / 执行需求结构角色。
7. 刷新活跃阻塞子需求权重。
8. 选择当前主需求。
9. 写回自我当前主需求镜像。
```

当前代码入口：

```cpp
私有_从需求树构建需求列表(...)
私有_选择当前主需求候选_按根权重(...)
需求类::执行更新指令(...)
需求类::刷新需求及祖先结构角色(...)
```

需求树正常生长的权威规则见：

```text
020003 需求树生长机制规范20260512.md
```

## 7. 任务发起与调度

自我线程不得创建任务头。

当当前主需求有效且没有承接任务时：

```text
自我线程 -> 任务管理界面线程：
    请求类型 = 发起任务
    来源需求 = 当前主需求
    允许创建任务信息节点 = true
    允许创建任务虚拟存在 = true
```

当前代码入口：

```cpp
私有_提交任务发起请求到任务管理界面线程_线程侧(...)
任务管理界面线程::提交任务治理请求(...)
```

任务管理界面线程必须创建：

```text
任务信息节点
任务虚拟存在
任务状态 = 未启动
任务语义特征槽
需求 -> 任务信息节点 绑定关系
```

当当前主任务已经存在且允许派发时：

```text
自我线程 -> 任务管理界面线程：
    请求类型 = 调度任务
    任务信息节点 = 当前主任务
    来源需求 = 当前主任务对应需求
```

## 8. 任务筹办

任务筹办语义目标：

```text
形成“任务可就绪”的筹办回执
```

任务筹办是任务管理生命周期普通函数。它由任务管理线程按任务状态调用，不被自我作为候选方法选择，不具有方法虚拟存在和条件结果对，也不是任务状态提交方法。

任务筹办负责：

```text
1. 根据任务对应需求读取目标宿主、目标特征类型、当前状态、目标状态、满足关系。
2. 按目标特征类型查找能确定性命中的方法头。
3. 在运行存在中形成候选方法集。
4. 按规则选择当前方法并写入运行存在。
5. 判断当前方法是否可用。
6. 构造可执行输入参数场景并写入运行存在。
7. 若条件齐备，写入：
       筹办结果类型 = 可就绪
       建议任务状态 = 就绪
       当前方法 = M
       候选方法集 = [...]
       可执行输入参数场景 = S
       筹办缺口类型 = 无
   并形成筹办运行事实。
```

任务状态进入就绪，由任务管理线程随后调用 `同步筹办回执到任务` 和 `提交任务状态变化` 完成。

任务筹办不能一步形成可就绪回执时，不是失败，而是记录确定的筹办事实或缺口事实。需要等待、派生、重筹办等中间状态时，由对应任务域 / 需求域提交本能方法继续处理。

任务筹办不负责把任务推进到等待中，也不负责直接派生需求。典型事实：

```text
建议任务状态 = 就绪 / 待重筹办 / 无法执行
筹办缺口类型 = 无确定候选方法 / 候选方法不可用 / 参数配对失败
候选方法集 = [...]
当前方法 = M
可执行输入参数场景 = S
```

任务筹办不得直接改需求树。

当前代码入口：

```cpp
任务模块::筹办::筹办任务(...)
任务模块::筹办::筹办任务(...)
```

## 9. 方法可用性与学习链

方法头只用于大范围索引和候选召回。

执行时依据：

```text
条件节点：参数绑定、前置条件、可执行判断。
结果节点：输出槽、结果结构、完成判断。
条件结果对：条件节点与结果节点之间的知识单元。
方法虚拟存在：方法状态、运行账、学习产物索引。
```

方法状态目标：

```text
方法状态 = 可用
```

可用最低条件：

```text
至少存在一组有效条件结果对。
```

学习链规则：

```text
定向学习：缺动作时发生。
尝试学习：有动作后，建立可用所需条件结果对。
练习：在可用基础上让方法更稳定、更细致、更好用。
```

任务筹办发现候选方法不可用时，派生需求应表达为：

```text
方法状态 = 可用
```

不得在缺口不明确时派生：

```text
方法动作骨架状态
```

除非回执明确证明实际缺动作。

## 10. 任务执行

任务执行语义目标：

```text
形成“任务可完成”的执行回执
```

任务执行是任务管理生命周期普通函数。它内部可以调用当前方法，当前方法产生世界事实动作动态；任务执行根据世界事实和来源需求目标判断任务过程是否闭合。它不是候选方法，也不是任务状态提交方法。

任务执行负责：

```text
1. 读取当前任务。
2. 校验任务状态允许执行。
3. 读取当前方法。
4. 读取可执行输入参数场景。
5. 调用当前方法。
6. 读取当前方法运行存在和输出结果场景。
7. 判断需求目标是否满足。
8. 若来源需求目标满足，写入：
       执行结果类型 = 可完成
       建议任务状态 = 完成
       目标是否达成 = true
       当前方法运行存在 = R
       最近输出结果场景 = S
   并形成执行运行事实。
```

只有：

```text
当前方法执行事实成立
并且
来源需求目标满足
```

才可以形成：

```text
建议任务状态 = 完成
```

任务状态进入完成，由任务管理线程随后调用 `同步执行回执到任务` 和 `提交任务状态变化` 完成。任务完成动作动态来源仍是 `提交任务状态变化`，不是任务执行。

当前代码存在第一版保底：

```text
若无显式需求满足证据，使用本能动作运行成功作为任务完成建议。
```

该保底只能视为过渡实现，不得作为最终闭环验收依据。最终必须以明确需求满足证据为准。

若来源需求目标未满足，不叫执行失败。它应形成执行缺口事实，由自我线程决定是否派生更细需求，或由任务管理线程调度重筹办 / 等待方法。

当前代码入口：

```cpp
任务模块::执行::执行任务(...)
判断任务来源需求是否被输出满足(...)
```

## 11. 任务状态推进

任务状态变化必须由任务状态提交本能方法完成，而不是由线程裸写，也不是由任务筹办 / 任务执行直接写。

任务管理线程负责调度任务域动作方法：

```text
承接需求创建任务
任务筹办
挂起任务等待子需求
唤醒任务重筹办
任务执行
取消任务
超时处理
```

状态推进归属固定为：

```text
承接需求创建任务：
    无任务 -> 任务状态 = 未启动

任务筹办：
    形成建议任务状态 = 就绪 的筹办回执

挂起任务等待子需求：
    筹办中 / 待重筹办 / 执行中 -> 等待中

唤醒任务重筹办：
    等待中 -> 待重筹办

任务执行：
    形成建议任务状态 = 完成 的执行回执

取消任务：
    任意可取消状态 -> 取消

超时处理：
    任意超时状态 -> 超时
```

动作动态归属：

```text
任务创建动态 -> 承接需求创建任务
筹办回执同步动态 -> 同步筹办回执到任务
任务就绪动态 -> 提交任务状态变化
任务等待动态 -> 挂起任务等待子需求
任务重筹办动态 -> 唤醒任务重筹办
执行回执同步动态 -> 同步执行回执到任务
任务完成动态 -> 提交任务状态变化
```

任务筹办 / 任务执行不生成方法动作动态。它们形成的过程回执由 `同步筹办回执到任务` / `同步执行回执到任务` 落入任务虚拟存在；任务状态动作动态的动作来源仍是 `提交任务状态变化`。

任务管理界面线程不得作为动作动态来源。任务管理工作线程不得裸写任务状态。若当前实现仍由普通函数直接提交任务状态或生成动作动态，应视为过渡实现，后续必须方法化。

## 12. 需求回写与结算

任务完成不等于需求满足。

任务完成是任务域事实。需求满足是自我治理事实。需求满足不是需求自身主动完成，而是自我在需求治理账中登记：该原子目标已经由任务输出事实满足。

任务完成后：

```text
任务管理线程 -> 自我线程：
    任务完成回执
    当前方法运行存在
    输出结果场景
    任务状态动作动态
    来源需求
```

自我线程必须复核来源需求是否满足。

满足后：

```text
1. 被需求当前状态 = 目标状态 或 最近满足观测状态。
2. 需求有效截止 = 当前时间。
3. 累计安全结算 / 服务结算更新。
4. 最近结算任务主键更新。
5. 最近结算时间更新。
6. 刷新父级需求。
7. 下一轮重判主需求。
```

需求状态变化若生成动作动态，动作来源必须是需求域本能方法：

```text
本能方法_回写需求满足
```

动作动态表达：

```text
方法 = 本能方法_回写需求满足
主体 = 自我存在 / 自我需求治理账
特征 = 需求满足登记 / 结算账
```

## 13. 待机与休眠期修复

当：

```text
无有效主需求
或当前任务等待缺口承接
或安全 / 服务压力低且无未完成关键需求
```

自我可以进入待机或低频自检。

休眠期修复动作不能凭空决定。

当前阶段规则：

```text
1. 自检线程只发现缺失。
2. 自我线程记录缺失。
3. 是否生成修复需求，后续按休眠期修复策略细化。
4. 当前不把休眠期修复作为内部循环闭合必需条件。
```

## 14. 验收分级

实现验收分级：

```text
0 级：未初始化。
1 级：冷启动完成。
2 级：需求层闭合。
3 级：任务化可用。
4 级：任务筹办可用。
5 级：任务执行可用。
6 级：结果回写与需求结算可用。
7 级：连续多轮稳定。
```

内部循环真正建立至少达到：

```text
6 级。
```

工程验收应达到：

```text
7 级。
```

## 15. 非正常逻辑错误

以下必须弹错 / 记录日志 / 拒绝继续，不得兜底：

```text
1. 任务管理派生需求缺父需求。
2. 派生需求目标状态或当前状态缺失。
3. 派生需求目标特征类型缺失。
4. 任务缺自身对应需求却试图用当前主需求兜底。
5. 任务虚拟存在创建失败。
6. 任务筹办在缺口不明确时凭空生成学习方向。
7. 任务执行没有输出事实却建议任务完成。
8. 自我线程直接写任务状态。
9. 任务管理线程直接写需求树。
10. 自检线程直接生成修复需求。
```

动作动态不以成功 / 失败为根概念。若任务未完成，通常应记录为：

```text
需求继续细化
任务等待子需求
任务重筹办
方法需要学习
参数需要补齐
```

只有逻辑崩溃、目标不可解析、输出事实缺失且无可派生路径时，才进入任务失败或无法执行。

## 17. 最终定稿

```text
自我内部循环的实现，不是单个线程循环，也不是单个本能方法。
它是自我线程、任务管理线程、需求域方法、任务过程方法、任务提交方法、方法域学习方法、世界域执行方法共同组成的闭合链。

自我线程负责“为什么做、做哪个”；
任务管理线程负责“调度哪个任务域方法推进任务”；
任务筹办 / 任务执行负责“形成过程事实和任务层回执”；
同步回执 / 提交任务状态变化负责“写任务账并生成任务状态动作动态”；
本能方法负责“具体动作如何发生”；
输出结果场景负责“世界事实发生了什么”；
自我回写负责“这个事实是否改变下一轮需求判断”。

需求是目标描述；
任务是动态过程；
方法是动作能力；
动作动态是方法运行造成事实变化的记录。

任务筹办是任务生命周期普通函数，语义目标是形成任务可就绪回执；
任务执行是任务生命周期普通函数，语义目标是形成任务可完成回执；
任务状态动作动态由提交任务状态变化本能方法生成；
需求树生长是自我把缺口事实转成更细原子目标的过程；
需求满足是自我对目标被事实满足的登记。
```
