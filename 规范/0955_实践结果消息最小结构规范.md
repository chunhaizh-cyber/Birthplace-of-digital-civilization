# 0955 实践结果消息最小结构规范

## 1. 目标

本文用于把 `实践结果消息` 的最小正式结构单独固定下来。

本文要回答的是：

- 哪些结果能叫 `实践结果消息`
- 主循环统一消费前，最少要带哪些字段
- 执行线程、学习线程、外部回流如何统一成一类消息
- 哪些字段属于正式稳定载荷，哪些只是可选挂带

## 3. 核心结论

必须固定以下结论：

1. `实践结果消息` 是主循环消费实践结果的统一入口。
2. `执行线程 / 学习线程 / 外部观察回流` 都只能提交 `实践结果消息`，不能各自直接写最终结论。
3. `实践结果消息` 的核心，不是“描述做了什么”，而是“把实践后真实得到的带值结果特征带回来”。
4. `实践结果消息` 至少必须能回指：
   - 来源宿主任务
   - 当前步骤
   - 当前方法
   - 最近结果节点或结果特征
5. `实践结果消息` 中的 `实际结果特征列表` 必须是一组 `带值特征`，不能用主键字段命中替代比较、方法名或自然语言摘要代替。
6. `实践结果消息` 可以带状态建议和去向建议，但它们都不是最终裁决。

一句话收口：

`实践结果消息是主循环统一消费实践结果的最小正式载体；它最少要把“谁做的、回到了什么结果特征、当前建议往哪走”带回来。`


## 4. 什么算实践结果消息

当前以下回流统一算作 `实践结果消息`：

- 叶子方法执行结果
- 动作执行结果
- 学习验证结果
- 外部观察、宿主通道、环境回流结果

必须固定：

1. 纯预测结果不是实践结果消息。
2. 执行前门控拒绝不是实践结果消息。
3. 主键命中、候选淘汰、规则推测都不是实践结果消息。


## 5. 最小稳定载荷

| 字段名 | 作用 | 说明 |
| --- | --- | --- |
| `来源承接类型` | 区分来自 `执行 / 学习 / 外部回流 / 观察` | 用于主循环归并 |
| `来源宿主任务指针` | 回指这条消息属于哪个宿主任务 | 缺失则不得继续主链 |
| `来源当前步骤指针` | 回指当前步骤 | 用于结果回挂 |
| `来源当前方法首节点指针` | 回指当前方法 | 用于方法证据与结果归并 |
| `最近结果节点指针或镜像` | 提供当前已形成的最近结果锚点 | 可为空，但必须有明确空语义 |
| `实际结果特征列表` | 正式结果本体 | 必须是一组带值特征 |
| `当前步骤后状态建议` | 建议当前步骤进入什么状态 | 只是建议，不是最终裁决 |
| `当前下一步去向建议` | 建议下一步更像往哪走 | 只是建议，不是最终裁决 |
| `是否叶子完成` | 标记本轮是否为叶子完成结果 | 服务叶子/安全叶子结算资格判断 |
| `命中需求指针列表或需求回挂提示` | 供主循环按需求列表与需求树回挂结果 | 可为一个或多个相关需求锚点 |


## 6. 可选挂带字段

以下字段允许挂带，但不属于最小稳定载荷：

- `动作动态锚点`
- `实例因果锚点`
- `原始错误码`
- `超时标记`
- `原始结果摘要`
- `学习对象指针`
- `外部来源标识`

必须固定：

1. 这些字段服务解释、审计和后续成长。
2. 即使缺失，只要最小稳定载荷完整，主循环仍应能继续承接。


## 7. 与主循环的关系

主循环消费 `实践结果消息` 后，当前至少应继续完成：

```text
结果特征归并
-> 需求树与需求列表回挂
-> 实际结果方向生成
-> 双值结算
-> 经验项后果/禁止项后果回写
-> 外层再重判
```

必须固定：

1. `实践结果消息` 只是输入，不是最终解释。
2. `实际结果方向` 必须由主循环基于带值特征重新比较生成。
3. `需求是否满足` 也必须由主循环或外层正式判定链解释，不得由消息本身直接给出。


## 8. 严格禁止的混层

以下做法必须禁止：

1. 用 `方法执行成功` 代替 `实际结果特征列表`。
2. 用 `结果主键` 代替正式结果本体。
3. 用 `动作已执行` 代替叶子结果。
4. 让执行线程、学习线程本地直接写“需求已满足”。
5. 缺少 `来源宿主任务`、`当前步骤`、`实际结果特征` 仍强行推进主链。
