# 1310 任务树结构体字段草案与状态机转移表

## 1. 目标

本文在 [1300_任务树数据结构与步骤子任务执行规范.md](D:/鱼巢/规范/1300_任务树数据结构与步骤子任务执行规范.md) 的基础上，继续收束出一版可直接指导实现的数据结构草案，重点回答三个问题：

- 任务头结点、步骤节点、结果节点分别需要哪些字段
- 步骤、子任务、执行、等待、回接在状态机上如何表达
- 任务管理任务后续接管根任务分解时，应当向任务树写入哪些标准信息

本文不直接替换现有类定义，而是给出与当前工程实现对齐的“规范口径”。后续新增字段、重构代码、补适配层时，都以本文为准。

## 2. 设计口径

先固定四条总原则：

1. 任务树只承认三类主节点：`头结点`、`步骤节点`、`结果节点`。
2. 子任务不是第四类节点。子任务本体仍然是一个新的任务头结点。
3. 真正执行的最小单元固定为：`步骤节点 + 当前方法首节点`。
4. 任务状态不直接修改安全值或服务值；任务树只负责把治理过程沉淀为可回溯的执行、结果、回接记录。

## 3. 规范化数据结构

### 3.1 基础枚举

```cpp
enum class 枚举_任务节点种类 : std::uint8_t {
    未定义 = 0,
    头结点 = 1,
    步骤节点 = 2,
    结果节点 = 3,
};

enum class 枚举_任务状态 : std::uint8_t {
    未定义 = 0,
    未启动 = 1,
    运行中 = 2,
    挂起 = 3,
    完成 = 4,
    失败 = 5,
    取消 = 6,
    超时 = 7,
    就绪 = 8,
    执行中 = 9,
    筹办中 = 10,
    排队中 = 11,
    等待中 = 12,
    无法执行 = 13,
    待重筹办 = 14,
};

enum class 枚举_任务树类型 : std::uint8_t {
    未定义 = 0,
    编排任务 = 1,
    叶子任务 = 2,
};

enum class 枚举_分支选择动作 : std::uint8_t {
    未定义 = 0,
    继续下一步 = 1,
    回跳重试 = 2,
    生成补条件子任务 = 3,
    转入尝试学习 = 4,
    挂起等待 = 5,
};

enum class 枚举_任务结果角色 : std::uint8_t {
    未定义 = 0,
    预测结果 = 1,
    实际结果 = 2,
};

enum class 枚举_任务执行记账结果 : std::uint8_t {
    未定义 = 0,
    未完成 = 1,
    成功 = 2,
    失败 = 3,
};
```

### 3.2 公共基类

这部分直接对应当前项目里的 `任务信息基类`，是三类节点共享的公共头。

```cpp
struct 结构_任务节点公共信息 {
    const 词性节点类* 名称 = nullptr;
    const 自然句节点类* 描述信息 = nullptr;
    const 词性节点类* 类型 = nullptr;
    枚举_任务节点种类 节点种类 = 枚举_任务节点种类::未定义;

    枚举_任务状态 状态 = 枚举_任务状态::未定义;
    时间戳 创建时间 = 0;

    std::int64_t 基准优先级 = 0;
    std::int64_t 局部优先级偏移 = 0;
    std::int64_t 调度优先级 = 0;

    场景节点类* 场景 = nullptr;
    存在节点类* 任务虚拟存在 = nullptr;
    枚举_任务树类型 任务树类型 = 枚举_任务树类型::未定义;

    std::int64_t 运行次数 = 0;
    std::int64_t 成功次数 = 0;
    std::int64_t 失败次数 = 0;
    std::int64_t 连续失败次数 = 0;
    std::int64_t 转入学习次数 = 0;
    std::int64_t 最近错误码 = 0;
    时间戳 最近开始时间 = 0;
    时间戳 最近结束时间 = 0;
    时间戳 最近调度时间 = 0;
    方法节点类* 当前方法首节点 = nullptr;
    方法节点类* 最近方法首节点 = nullptr;
};
```

字段职责固定如下：

- `状态` 是运行期镜像；权威写回在 `任务虚拟存在`。
- `场景` 是当前节点语义的锚点，头结点挂全局条件，步骤节点挂局部执行场景，结果节点挂结果场景。
- `基准优先级 + 局部优先级偏移 -> 调度优先级`，不允许只写最终值不写来源。
- 执行流水账统一写在公共头，避免步骤、头结点、结果节点各自分裂出不同记账口径。

### 3.3 头结点结构

头结点表示一个独立任务单元，也是“子任务”的真实结构载体。

```cpp
struct 结构_任务头结点信息 : 结构_任务节点公共信息 {
    需求节点类* 需求 = nullptr;
    bool 是否真根任务 = false;
    bool 是否常驻任务 = false;
    bool 等待学习唤醒 = false;
    方法节点类* 等待学习方法首节点 = nullptr;

    任务节点类* 父任务头结点 = nullptr;
    任务节点类* 来源父结果节点 = nullptr;
    任务节点类* 来源父步骤节点 = nullptr;

    任务节点类* 当前步骤节点 = nullptr;
    任务节点类* 当前子任务头结点 = nullptr;
    任务节点类* 最近实际结果节点 = nullptr;
    std::vector<任务节点类*> 直属子任务头结点列表{};

    std::vector<std::string> 目标特征类型{};
};
```

头结点必须承载四类信息：

- 目标来源：`需求`
- 纵向来源：`父任务头结点 / 来源父步骤节点 / 来源父结果节点`
- 当前执行位：`当前步骤节点 / 当前子任务头结点 / 最近实际结果节点`
- 横向分解面：`直属子任务头结点列表`

### 3.4 步骤节点结构

步骤节点表示“任务内部当前要做的一个动作位、等待位、检查位、回接位”。

```cpp
struct 结构_任务步骤节点信息 : 结构_任务节点公共信息 {
    任务节点类* 所属任务头结点 = nullptr;

    std::vector<方法节点类*> 可用方法首节点列表{};
    方法节点类* 当前选中方法首节点 = nullptr;

    std::vector<任务节点类*> 并发子任务头结点列表{};
    任务节点类* 当前子任务头结点 = nullptr;

    std::int32_t 步骤序号 = 0;
    std::int32_t 已重试次数 = 0;
    std::int32_t 允许重试次数 = 0;
    时间戳 超时截止时间 = 0;
    bool 允许切换方法 = true;
};
```

步骤节点的规范职责是：

- 绑定当前可执行方法
- 记录是否已经进入执行
- 表达是否正在等待子任务或外部条件
- 承载重试、超时、换方法等局部控制

### 3.5 结果节点结构

结果节点不是装饰节点，而是任务树里的标准回接点、结算前置点和派生点。

```cpp
struct 结构_任务结果节点信息 : 结构_任务节点公共信息 {
    枚举_任务结果角色 结果角色 = 枚举_任务结果角色::未定义;
    任务节点类* 所属任务头结点 = nullptr;
    任务节点类* 来源步骤节点 = nullptr;

    std::vector<任务节点类*> 派生子任务头结点列表{};

    方法节点类* 来源方法首节点 = nullptr;
    方法节点类* 对应方法结果节点 = nullptr;
    任务节点类* 命中预测结果节点 = nullptr;
    枚举_分支选择动作 命中后动作 = 枚举_分支选择动作::未定义;

    bool 是否预测外结果 = false;
    std::int64_t 置信度_Q10000 = 0;
    std::int64_t 错误码 = 0;
    存在节点类* 评估摘要存在 = nullptr;
};
```

结果节点至少承担三件事：

- 记录实际执行产物
- 作为“下一步”或“子任务派生”的挂接点
- 为后续“叶子需求满足判定”和“价值结算”提供可回溯的证据锚点

### 3.6 返回消息结构

任务执行器、子任务回接器、任务管理任务之间，统一通过返回消息汇总一次执行后的最小结论。

```cpp
struct 结构_任务返回消息 {
    枚举_任务返回消息类型 类型 = 枚举_任务返回消息类型::未定义;
    任务节点类* 来源任务 = nullptr;
    任务节点类* 来源步骤 = nullptr;
    任务节点类* 相关子任务 = nullptr;
    需求节点类* 对应需求 = nullptr;
    bool 需求已满足 = false;
    bool 已进入下一阶段 = false;
    bool 需要重筹办 = false;
    std::string 摘要{};
};
```

## 4. 结构关系规范

### 4.1 最小主链

任务树的最小闭环固定为：

```text
任务头结点
-> 步骤节点
-> 结果节点
```

原因有两点：

- 执行必须落在步骤上，不能让头结点直接执行
- 回接必须落在结果上，不能让下一步直接跳过结果

### 4.2 子任务的真实位置

子任务有两种常见挂法：

```text
步骤节点
-> 并发子任务头结点列表

结果节点
-> 派生子任务头结点列表
```

语义区分如下：

- 挂在步骤上：表示“当前步骤直接发起并等待这个子任务”
- 挂在结果上：表示“某个结果导致系统决定再派生一个子任务”

### 4.3 当前指针

所有运行期调度都优先读取以下三个指针：

- 头结点：`当前步骤节点`
- 头结点或步骤节点：`当前子任务头结点`
- 头结点：`最近实际结果节点`

没有这三个指针，任务树虽然能遍历，但不能稳定驱动执行。

## 5. 步骤、子任务、执行的判定规范

### 5.1 什么时候长步骤

满足以下任一条件时，默认长步骤：

- 只是当前任务内部的一个动作
- 只是一次方法执行
- 只是等待某个子任务完成
- 只是对已有结果做检查、确认、回接
- 只是一次重试、改方法、继续下一步

### 5.2 什么时候必须长子任务

满足以下任一条件时，必须长子任务：

- 该工作体有自己的需求来源
- 该工作体需要独立筹办、维护、纠偏、收束
- 该工作体会继续分解出自己的步骤或子任务
- 该工作体完成后需要作为独立结果回接父任务
- 该工作体需要被单独统计、重试、挂起、取消

### 5.3 执行的唯一口径

执行只承认下面这一个最小单元：

```text
执行单元 = 步骤节点 + 当前方法首节点
```

因此：

- 头结点不直接执行
- 子任务不是执行动作，它是独立任务单元
- 结果节点不执行，它只记录执行产物并驱动后续分支

### 5.4 等待的表达

等待不是执行失败，也不是空转。等待应建模为显式步骤状态：

```text
步骤A：发起子任务B
步骤A：进入等待中
子任务B：独立推进
子任务B：完成
步骤A：接收子任务完成消息
步骤A：生成结果节点
父任务：进入下一步
```

因此必须固定一句话：

`等待子任务完成可以作为步骤，但被等待的独立工作仍然必须是子任务。`

## 6. 状态机转移表

### 6.1 头结点状态机

头结点的状态代表整条任务主链当前所处的治理阶段。

| 当前状态 | 可转向状态 | 说明 |
| --- | --- | --- |
| 未启动 | 就绪 | 任务已创建但未进入运行 |
| 就绪 | 筹办中, 排队中, 取消 | 已可被治理器接管 |
| 筹办中 | 排队中, 等待中, 无法执行, 待重筹办, 取消 | 补条件、补方法、补子任务 |
| 排队中 | 运行中, 等待中, 无法执行, 取消 | 等待执行器领取 |
| 运行中 | 等待中, 完成, 失败, 挂起, 待重筹办, 取消 | 主链正在推进 |
| 等待中 | 就绪, 运行中, 挂起, 超时, 取消 | 等子任务、等外部输入、等学习 |
| 无法执行 | 待重筹办, 挂起, 取消 | 条件不足或方法缺失 |
| 待重筹办 | 筹办中, 挂起, 取消 | 等待再次治理 |
| 挂起 | 就绪, 取消 | 人工或系统暂停 |
| 完成 | 无 | 已收束 |
| 失败 | 待重筹办, 挂起, 取消 | 允许回到治理侧 |
| 超时 | 待重筹办, 失败, 取消 | 超时后不直接算成功 |

### 6.2 步骤节点状态机

步骤节点的状态代表一个局部执行位当前是否可运行、在运行、在等待、或已产出结果。

| 当前状态 | 可转向状态 | 说明 |
| --- | --- | --- |
| 未启动 | 就绪 | 步骤已创建 |
| 就绪 | 排队中, 执行中, 等待中, 取消 | 已具备执行条件 |
| 排队中 | 执行中, 等待中, 取消 | 等待调度 |
| 执行中 | 完成, 失败, 无法执行, 超时, 等待中 | 方法正在运行 |
| 等待中 | 就绪, 执行中, 超时, 取消 | 等子任务、等外部信号 |
| 失败 | 就绪, 待重筹办, 取消 | 允许重试或切方法 |
| 无法执行 | 待重筹办, 挂起, 取消 | 当前位缺前置条件 |
| 超时 | 就绪, 失败, 取消 | 由重试策略决定 |
| 挂起 | 就绪, 取消 | 局部暂停 |
| 完成 | 无 | 应立即配套有结果节点 |

### 6.3 结果节点状态口径

结果节点不作为长期运行态节点使用，建议遵守以下口径：

- 创建时可为 `就绪`
- 一旦完成评估和回接，置为 `完成`
- 如结果无效或判定失败，可置为 `失败`

结果节点不承担排队、执行、等待语义。

## 7. 写回规范

### 7.1 创建头结点时必须写

- `需求`
- `任务树类型`
- `父任务头结点 / 来源父步骤节点 / 来源父结果节点`
- `目标特征类型`
- `基准优先级 / 局部优先级偏移 / 调度优先级`

### 7.2 创建步骤节点时必须写

- `所属任务头结点`
- `步骤序号`
- `可用方法首节点列表`
- `当前选中方法首节点`
- `允许重试次数`
- `超时截止时间`

### 7.3 创建结果节点时必须写

- `结果角色`
- `所属任务头结点`
- `来源步骤节点`
- `来源方法首节点`
- `命中后动作`
- `置信度_Q10000`

### 7.4 进入执行前必须写

- `当前步骤节点`
- `当前方法首节点`
- `最近方法首节点`
- `运行次数`
- `最近开始时间`

### 7.5 执行结束后必须写

- `成功次数 / 失败次数 / 连续失败次数`
- `最近错误码`
- `最近结束时间`
- `最近结果节点`
- 若转入学习，则写 `转入学习次数`

### 7.6 绑定子任务时必须写

如果子任务由步骤直接发起：

- 写入步骤节点的 `并发子任务头结点列表`
- 写入步骤节点的 `当前子任务头结点`
- 写入父头结点的 `直属子任务头结点列表`
- 写入父头结点的 `当前子任务头结点`
- 回填子任务的 `父任务头结点 / 来源父步骤节点`

如果子任务由结果派生：

- 写入结果节点的 `派生子任务头结点列表`
- 必要时同步写入来源步骤的 `并发子任务头结点列表`
- 回填子任务的 `父任务头结点 / 来源父结果节点`

## 8. 给任务管理任务的接口约束

后续任务管理任务接管根任务分解时，至少需要遵守以下接口约束：

1. 任务管理任务只能直接创建或调整：
   - 步骤节点
   - 子任务头结点
   - 结果节点
   - 当前指针与状态
2. 任务管理任务不能直接写安全值、服务值。
3. 任务管理任务若要宣称“治理有效”，必须在任务树上留下至少一条可检索证据：
   - 任务进入可执行或前进态
   - 生成了新的子任务
   - 产出了新的结果节点
   - 触发了下一步或回接
4. 任务管理方法的成长证据，应优先从以下链条回溯：

```text
方法应用
-> 步骤状态推进
-> 结果节点产生
-> 子任务完成或叶子需求满足
-> 根任务账本变化
```

## 9. 最小实现建议

如果下一步要开始改代码，最小改造顺序建议固定为：

1. 先让根任务分解统一经过任务管理任务入口。
2. 让任务管理任务只通过本文定义的结构写口去改任务树。
3. 让显式规则分解退化为“筹办域的兜底方法”。
4. 再把“结果节点 -> 叶子需求满足 -> 安全值或服务值变化”的回溯链挂到方法成长判定上。

## 10. 一句话收束

任务树的数据结构规范可以收束为一句话：

`头结点负责承载独立任务，步骤节点负责承载执行与等待，结果节点负责承载产物与回接，子任务始终是新的头结点，执行始终落在步骤加方法上。`
