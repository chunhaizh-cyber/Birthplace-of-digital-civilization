# 0920 任务三段式切换判定规则（保留原文件名兼容）

## 1. 说明

原 `四域切换` 口径已废止。

当前任务切换只围绕：

`目标 -> 筹办 -> 执行`


## 2. 最小切换规则

### 2.1 先有目标

若任务头没有明确目标，则不得进入正式筹办或执行。

### 2.2 什么时候进入筹办

满足任一即可进入 `筹办`：

1. 当前没有步骤节点。
2. 当前步骤没有结果节点。
3. 当前步骤没有可执行方法。
4. 当前执行后目标未达成，且还需要长下一步。
5. 当前路径失效，需要重建任务树。

### 2.3 什么时候进入执行

满足全部才可进入 `执行`：

1. 当前任务已有明确当前步骤。
2. 当前步骤已具备结果节点。
3. 当前步骤已绑定方法。
4. 当前步骤的方法实例条件节点已具备。

### 2.4 执行后的分流

执行后只允许三种分流：

1. 目标命中：任务完成。
2. 目标未命中：回到筹办。
3. 路径失效：回到筹办。


## 3. 三类任务的切换解释

### 3.1 普通任务

- 目标：现实存在特征值达标
- 筹办：完善现实任务树
- 执行：执行当前现实步骤

### 3.2 任务管理任务

- 目标：宿主任务完成
- 筹办：完善任务管理任务树和宿主任务树
- 执行：执行当前治理步骤

### 3.3 学习任务

- 目标：方法可用
- 筹办：完善学习任务树和待学习方法树
- 执行：执行当前学习步骤


## 4. 对旧口径的处理

旧 `四域切换表`、`判域排序`、`域内动作后重判` 全部删除。

后续若需要更细分的运行细节，只能作为：

- 当前步骤类型
- 当前步骤动作
- 当前结果分流

不得再升格回任务统一主流程。
