## 3. 核心结论

必须固定以下结论：

1. `0020` 解决的是“线程里流动哪三张票据”，`0021` 解决的是“这三张票据长什么样、谁能写、何时封口、如何回写到现有节点体系”。
2. `自我线程运行包.ixx` 只放类型定义，不放算法逻辑。
3. 三包都必须复用统一包头；`包状态 = 已封口` 后，局部模块只能读，不能改。
4. 字段拥有权必须按模块分段写死，不能谁手上有引用谁就补字。
5. 包正文默认不整包持久化；长期价值只通过 `包 -> 节点` 映射选择性写回。
6. `线程轮次上下文包` 是本轮共识，`一步治理任务包` 是本轮执照，`回流结算包` 是本轮账本。
7. 每个任务包必须对应一个结算包；哪怕授权失败、方法未命中、执行被打断，也不允许“无账结束”。
8. 第一条落地金路径应先锁在 `解释 / 预警 / 建议`，不先打开真实执行、自动回滚和自动联动改写。

一句话收口：

`到 0021 为止，三包不只是概念合同，而是已经具备结构、所有权、封口纪律和映射纪律的工程合同。`

### 6.5 三包结构的硬纪律

必须固定：

1. `上下文包` 是本轮共识，`任务包` 是本轮唯一一步，`结算包` 是本轮账本。
2. `上下文包` 仲裁完成后封口。
3. `任务包` 在方法阶段结束后封口；可包含 `选定方法为空` 的未命中失败态，但不得在封口后再改任务语义和授权边界。
4. `结算包` 在根层完成结算并给出下一轮入口后封口。
5. `学习候选集` 默认只能记录 `已验证 = false` 的候选，不得直接写成稳定因果。


## 7. 字段拥有者、写入时机、封口时机与持久化规则

### 7.1 组合字段继承规则

为避免重复，先固定一条规则：

`若某字段是 struct / vector / optional 组合字段，则其叶子字段默认继承该组合字段所在段的拥有者、写入时机、封口时机与持久化策略；除非本文另行显式覆盖。`

### 7.2 公共包头字段

| 字段 | 拥有者 | 首次写入时机 | 封口时机 / 可变性 | 是否持久化 / 去向 |
| --- | --- | --- | --- | --- |
| `包主键` | 包创建器 | 创建包时立即写入 | 写入后只读 | `否`；仅调试审计快照 |

## 8. 持久化策略

### 8.1 默认规则

必须固定：

1. 默认不整包持久化 `上下文包 / 任务包 / 结算包` 正文。
2. 默认只在调试或审计开关下保留完整快照。
3. 正式持久化只允许写回有长期价值的：
   - 摘要
   - 追踪键
   - 结果
   - 证据

### 8.2 三包的选择性持久化口径

| 包 | 默认持久化策略 | 推荐写回去向 |
| --- | --- | --- |
| `线程轮次上下文包` | 默认不整包落库；仅持久化 `差额摘要 / 仲裁摘要 / 生命周期摘要` | 轮次摘要、仲裁摘要、调试审计快照 |
| `一步治理任务包` | 默认不整包落库；保留 `任务语义 / 授权摘要 / 选定方法摘要` | 任务链、任务节点摘要、授权记录 |
| `回流结算包` | 默认不整包落库；保留 `执行事实 / 状态变化 / 结算结果 / 学习候选` | 动态节点、场景动态索引、因果候选、样本池 |

### 8.3 必须禁止

必须禁止：

- 把整包正文原样塞进现有世界节点
- 把 `候选方法集`、`根需求候选集` 这种高波动字段默认落成长期知识
- 把 `学习候选集` 直接写成 `已验证因果`


## 9. `自我线程运行包校验器.ixx`

### 9.1 最小接口

`0021` 当前建议把用户提出的 `6` 个最小能力拆到 `2` 个文件里。

其中前 `3` 个校验能力，放在：

`自我线程运行包校验器.ixx`

```cpp
export module 自我线程运行包校验器;

import 自我线程运行包;

export bool 校验上下文包(const 结构_线程轮次上下文包& 包);
export bool 校验任务包(const 结构_一步治理任务包& 包);
export bool 校验结算包(const 结构_回流结算包& 包);

export bool 封口上下文包(结构_线程轮次上下文包& 包, 时间戳 当前时间);
export bool 封口任务包(结构_一步治理任务包& 包, 时间戳 当前时间);
export bool 封口结算包(结构_回流结算包& 包, 时间戳 当前时间);
```

### 9.2 校验器职责

校验器至少要负责：

- 必填字段完整性
- `包状态` 是否允许当前写入
- 封口前后只读纪律
- `任务包 -> 结算包` 是否成对
- `学习候选.已验证` 初始值是否为 `false`

### 9.3 封口规则

必须固定：

1. 只有校验器可以把 `包状态` 从 `草稿` 改成 `已封口`。
2. 只有校验器可以写 `封口时间`。
3. `上下文包` 未经仲裁完成，不得封口。
4. `任务包` 未经方法阶段结束，不得封口；若方法未命中，可带 `选定方法为空` 的失败态封口，但不得缺少后续结算。
5. `结算包` 未完成双值结算与下一轮入口建议，不得封口。


## 10. `自我线程运行包映射器.ixx`

### 10.1 最小接口

用户提出的后 `3` 个最小能力，放在：

`自我线程运行包映射器.ixx`

```cpp
export module 自我线程运行包映射器;

import 自我线程运行包;

export void 从上下文包生成任务包(
    const 结构_线程轮次上下文包& 上下文包,
    结构_一步治理任务包& 任务包);

export void 从结算包写入动态节点(
    const 结构_一步治理任务包& 任务包,
    const 结构_回流结算包& 结算包);

export void 从结算包沉淀因果候选(
    const 结构_一步治理任务包& 任务包,
    const 结构_回流结算包& 结算包);
```

### 10.2 映射纪律

必须固定：

1. 映射器只做 `包 -> 现有节点体系` 的写回，不创建新的世界模型。
2. `上下文包` 默认不直接映射为长期节点，只作为生成 `任务包` 的输入。
3. `任务包` 的长期价值主要挂回 `任务链 / 任务节点摘要 / 授权摘要 / 选定方法摘要`。
4. `结算包` 的长期价值主要写入 `动态节点 / 场景动态索引 / 实例因果候选 / 样本池`。
5. 每个任务包必须对应一个结算包；映射器不得允许“有任务、无账本”的暗河。

### 10.3 从结算包写入动态节点

`从结算包写入动态节点` 至少应完成以下映射：

| 来源 | 目标 |
| --- | --- |
| `任务包.动作类型` | `动态节点.来源动作` |
| `结算包.输入场景主键` | `动态节点.来源输入场景` |
| `结算包.输出场景主键` | `动态节点.来源输出场景` |
| `结算包.执行成功` | `动态节点.来源执行成功` |
| `结算包.错误码` | `动态节点.来源错误码` |
| `结算包.动态路径签名` | `动态节点.动态路径签名` |
| `结算包.状态变化集` | `场景动态索引 / 状态变化记录` |
| `结算包.副作用状态主键集` | `场景动态索引 / 副作用摘要` |

### 10.4 从结算包沉淀因果候选

`从结算包沉淀因果候选` 至少应完成以下映射：

| 来源 | 目标 |
| --- | --- |
| `学习候选.来源任务主键` | `因果实例候选.来源任务主键` |
| `学习候选.来源方法主键` | `因果实例候选.来源方法主键` |
| `学习候选.置信度` | `因果实例候选.置信度` |
| `学习候选.已验证 = false` | `因果实例候选.已验证` |
| `学习候选.样本类别` | `样本池 / 黑名单标签` |
| `任务包.任务语义键` + `结算包.执行成功 / 错误码 / 状态变化集` | `因果候选证据面` |

### 10.5 映射器禁止越权

映射器不得：

- 直接改 `服务值 / 安全值目标`
- 直接决定下一轮主导需求
- 把一次样本直接升格成已验证因果
- 绕过主循环直接续跑下一轮


## 11. 第一条落地金路径

当前最稳的首条金路径固定为：

`状态刷新 -> 差额评估 -> 生成服务需求 -> 任务包：建议动作 -> 方法匹配 -> 人工确认 / 模拟执行 -> 回流结算 -> 学习候选沉淀`

这一条链当前只先开放以下动作类型：

- `解释`
- `预警`
- `建议`

这一条链当前暂不开放：

- `真实执行`
- `自动回滚`
- `自动联动改写`

### 11.1 关键闸门

必须固定：

1. `上下文包` 封口后，任务管理只能读，不能改根裁决。
2. `任务包` 封口后，执行层只能按包执行，不能反向扩权。
3. 未生成 `结算包`，本轮不算结束。
4. 未完成结算，学习不得落库。
5. 学习写回只能产生 `候选`，不能直接产生 `已验证`。
