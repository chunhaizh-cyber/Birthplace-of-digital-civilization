# 自我线程与任务管理线程权责边界规范 20260512

## 0. 权威修订

本规范受以下硬规则约束：

```text
规范/方法动作动态硬规则规范20260512.md
```

线程不是动作来源。线程只能调度和调用方法。凡是生成方法动作动态的任务域、需求域、方法域提交，都必须通过对应本能方法或动作骨架入口完成。

## 1. 总原则

```text
自我线程：
    管需求树。
    管需求生长。
    管需求估权。
    管任务发起意图。
    管调度意图。

任务管理线程：
    管任务虚拟存在。
    管任务状态推进。
    调用任务筹办。
    调用任务执行。
    调用任务域提交类本能方法。
    同步任务回执。
```

一句话：

```text
自我线程决定“为什么要做、做哪个、优先做什么”。
任务管理线程决定“任务状态如何从未启动推进到完成”。
但任务状态动作动态的动作来源必须是任务域提交类本能方法。
```

## 2. 硬规则：任务状态只由任务管理线程编排提交

只有任务管理线程可以编排任务状态变化提交。

这些状态只能由任务管理线程推进：

```text
未定义
未启动
运行中
挂起
完成
失败
取消
超时
就绪
执行中
筹办中
排队中
等待中
无法执行
待重筹办
```

自我线程不得直接写：

```text
任务状态
当前方法
可执行输入参数场景
筹办缺口类型
执行缺口类型
最近筹办回执
最近执行回执
```

自我线程只发送：

```text
任务发起请求
任务调度请求
任务取消请求
任务挂起请求
任务重筹办请求
任务权重更新请求
需求回写请求
派生需求请求
```

任务管理线程可以提交任务状态，但如果提交会生成方法动作动态，必须调用：

```text
本能方法_提交任务状态变化
本能方法_同步筹办回执到任务
本能方法_同步执行回执到任务
```

任务管理线程本身不是动作来源。

## 3. 自我线程职责

自我线程负责需求域和调度意图。

允许负责：

```text
1. 扫描需求树。
2. 发现未满足需求。
3. 生成新需求。
4. 需求估权。
5. 判断需求是否需要任务承接。
6. 生成任务发起请求。
7. 计算任务优先级 / 权重。
8. 发出调度意图。
9. 根据任务回执生长新的需求分支。
10. 判断来源需求是否满足。
11. 调用回写需求满足本能方法。
12. 调用派生需求入树本能方法。
```

自我线程可以表达：

```text
这个需求需要承接。
这个需求权重是 80。
建议生成任务。
建议优先调度任务 T。
任务 T 的缺口需要派生学习需求。
```

自我线程不得表达：

```text
任务 T 状态 = 筹办中。
任务 T 状态 = 执行中。
任务 T 状态 = 完成。
```

如果需求域变化需要生成方法动作动态，动作来源必须是：

```text
本能方法_回写需求满足
本能方法_派生需求入树
```

## 4. 任务管理线程职责

任务管理线程负责任务域和任务生命周期。

必须负责：

```text
1. 接收任务发起请求。
2. 创建任务信息节点。
3. 创建任务虚拟存在。
4. 初始化任务状态 = 未启动。
5. 根据任务状态调用任务筹办普通函数。
6. 根据筹办回执调用同步筹办回执到任务本能方法。
7. 根据任务状态调用任务执行普通函数。
8. 根据执行回执调用同步执行回执到任务本能方法。
9. 调用提交任务状态变化本能方法。
10. 把任务结果回执发送给自我线程。
```

任务管理线程管理的是：

```text
任务虚拟存在
任务状态
任务方法候选集
当前方法
可执行输入参数场景
筹办运行回执
执行运行回执
任务动作动态
任务生命周期
```

任务筹办 / 任务执行是任务生命周期普通函数，由任务管理线程按任务状态调度，不进入方法树和候选方法集。同步筹办回执、同步执行回执、提交任务状态变化，仍归任务管理线程调度，且凡是生成方法动作动态的同步或提交，必须通过对应本能方法完成。

## 5. 任务发起边界

自我线程负责“发起意图”，任务管理线程负责“创建提交”。

流程：

```text
自我线程发现需求 D
    ↓
自我线程生成任务发起请求
    ↓
任务管理线程接收
    ↓
调用 本能方法_承接需求创建任务
    ↓
创建任务信息节点
创建任务虚拟存在
写入任务状态 = 未启动
写入来源需求
写入权重 / 完成判据 / 回写规则
```

`未启动` 是任务管理线程编排的创建初始化结果，不是自我线程直接推进状态。

## 6. 调度边界

调度拆成两层。

自我线程给调度意图：

```text
建议调度任务 T
调度权重
调度顺序
调度原因
```

任务管理线程调用任务域方法提交状态：

```text
就绪 -> 排队中
排队中 -> 执行中
执行中 -> 完成 / 待重筹办 / 失败 / 等待中
```

因此：

```text
自我线程给调度意图。
任务管理线程调用提交类本能方法后提交任务状态变化。
```

## 7. 任务筹办和任务执行归属

任务筹办 / 任务执行是任务管理线程调用的任务生命周期普通函数。

```text
任务管理线程看到：
    任务状态 = 未启动 / 待重筹办
调用：
    任务筹办普通函数

任务管理线程看到：
    任务状态 = 就绪 / 排队中
调用：
    任务执行普通函数
```

任务筹办返回运行存在：

```text
建议任务状态 = 就绪 / 待重筹办 / 无法执行
筹办缺口类型
目标方法头节点
候选方法数量
可执行输入参数场景
```

任务管理线程再调用：

```text
本能方法_同步筹办回执到任务
本能方法_提交任务状态变化
```

任务执行返回运行存在：

```text
建议任务状态 = 完成 / 待重筹办 / 失败 / 等待中
目标是否达成
执行缺口类型
当前方法运行存在
输出结果场景
```

任务管理线程再调用：

```text
本能方法_同步执行回执到任务
本能方法_提交任务状态变化
```

## 8. 需求满足和任务完成的边界

```text
任务状态由任务管理线程通过任务域方法提交。
需求状态由自我线程通过需求域方法提交。
```

所以：

```text
任务完成
```

和：

```text
需求已满足
```

不是同一个状态。

推荐流程：

```text
任务执行产生输出结果场景
    ↓
任务管理线程读取执行回执
    ↓
任务管理线程调用提交任务状态变化方法推进任务状态
    ↓
任务管理线程发送需求满足证据 / 任务完成回执给自我线程
    ↓
自我线程检查来源需求是否满足
    ↓
自我线程调用回写需求满足方法
    ↓
需求状态、需求结算账本和需求树变化落账
```

语义必须保持：

```text
任务完成属于任务域。
需求已满足属于需求域。
```

两者通过回执同步，不是同一个状态。

## 9. 线程消息流

### 9.1 自我线程 -> 任务管理线程

```text
任务发起请求
任务调度请求
任务取消请求
任务挂起请求
任务重筹办请求
任务权重更新请求
```

### 9.2 任务管理线程 -> 自我线程

```text
任务创建回执
筹办回执
执行回执
任务完成回执
任务失败回执
任务缺口回执
任务等待回执
需求满足证据
派生需求提示
```

一句话：

```text
自我线程发“意图”。
任务管理线程回“结果”。
```

## 10. 四层权限与动作来源

| 层 | 编排者 | 动作来源 | 写入内容 |
| --- | --- | --- | --- |
| 需求层 | 自我线程 | 回写需求满足 / 派生需求入树等本能方法 | 需求状态、需求树、派生需求、需求结算 |
| 任务层 | 任务管理线程 | 提交任务状态变化 / 同步回执到任务等本能方法 | 任务状态、当前方法、任务回执、任务动作动态 |
| 方法层 | 学习 / 方法管理 | 提交方法状态变化 / 建立条件结果对等本能方法 | 方法状态、条件结果对、运行账、学习产物 |
| 世界层 | 世界树提交层 / 执行方法 | 改变特征值 / 具体业务方法 | 真实特征变化、状态节点、动态事实 |

禁止跨层直写：

```text
自我线程不写任务状态。
任务管理线程不写需求树。
任务筹办 / 任务执行不直接写任务虚拟存在。
本能函数不直接写任务状态。
世界事实只走输出结果场景。
线程不得作为方法动作动态来源。
```

## 11. 最终定稿

```text
自我线程：
    负责需求树生长、需求估权、任务发起意图、调度意图。

任务管理线程：
    负责任务创建提交、任务虚拟存在维护、任务状态推进、任务筹办、任务执行、任务回执同步。
    其中会生成动作动态的提交动作必须调用对应本能方法。

任务筹办 / 任务执行：
    是任务管理线程调用的任务生命周期普通函数。
    它们只返回过程回执，不直接改任务状态，不作为方法动作动态来源。

任务状态推进：
    只能由任务管理线程编排，并由任务域提交类本能方法作为动作来源。
```

一句话：

```text
自我线程生长需求并发号施令，任务管理线程接令后调用任务域本能方法推进任务状态；
一个管“为什么做”，一个管“做到哪一步”，动作来源永远是方法。
```
