# 0581 自检需求承接本能函数清单与特征合同规范

## 1. 目标

本文承接 `0580_自检发现缺口与需求落点规范`，用于明确：

- 自检结果进入自我线程后，需要哪些本能函数承接。
- 哪些本能函数已存在，哪些仍欠缺。
- 每个本能函数读取哪些输入特征。
- 输入特征的值域能否确定。
- 本能函数应返回哪些结果特征。
- 结果特征应表达方向，还是直接表达枚举/具体值。

自检线程本身仍只发现需求，不处理问题。

本文定义的是自检需求进入任务闭环后，后续任务可能调用或触发的本能函数合同。

## 2. 基础值域约定

### 2.1 布尔型特征

除已有枚举另有定义外，布尔型特征统一使用：

| 值 | 含义 |
| --- | --- |
| `0` | 否、未具备、不可用、无效 |
| `1` | 是、已具备、可用、有效 |

### 2.2 数量型特征

数量型特征统一使用 `I64` 非负整数。

| 范围 | 含义 |
| --- | --- |
| `< 0` | 非法值，不应写入正式状态 |
| `0` | 缺失或数量为零 |
| `1..I64_MAX` | 已存在对应数量 |

自检第一版只需要区分 `0` 与 `>0`。

### 2.3 I64 度量型特征

可靠度、质量、置信度、完成度等度量型值统一使用 `I64`。

不得再引入 `Q10000` 比例语义。

| 范围 | 含义 |
| --- | --- |
| `< 0` | 负向、不可信、失败压力或惩罚值 |
| `0` | 未形成、无增益或中性 |
| `> 0` | 正向、可信、有效增益或成熟度 |

度量选择规则：

| 场景 | 规则 |
| --- | --- |
| 选择更可靠/更优方法 | 取值更大者 |
| 选择风险/失败压力更低对象 | 取值更小者 |
| 提升可靠度/质量/完成度 | 直接 `当前值 + 增量` |
| 降低可靠度/质量/完成度 | 直接 `当前值 - 减量` |
| 判断是否停滞 | 比较时间窗内值是否增长 |

第一版只要求能比较大小和执行加减，不要求归一化到固定上限。

### 2.4 枚举型特征

枚举型特征可直接返回具体枚举值，不要求表达方向。

例如：

- 任务状态：`等待中`、`待重筹办`、`完成`。
- 学习状态：`可调度`、`执行中`、`已失败`。
- 引用状态：`有效`、`无效`、`已挂起待重建`。

若现有枚举尚不存在，可先用词性状态节点表达具体状态，不新增自检专属结构体。

## 3. 自检结果到本能函数承接总表

| 自检结果 | 自我线程处理 | 主要承接本能函数 | 状态 |
| --- | --- | --- | --- |
| 方法结构不完整 | 生成方法结构补齐需求，建学习任务 | `学习_尝试学习`、`学习_补齐方法结构` | 前者已有，后者欠缺 |
| 方法稳定性不足 | 生成方法稳定性/可靠度提升需求 | `学习_验证方法稳定性`、`学习_提升方法可靠度`、`管理_沉淀运行方法证据` | 部分欠缺 |
| 任务完成度停滞 | 生成任务推进恢复需求 | `管理_诊断任务停滞原因`、`管理_将停滞任务转重筹办` | 欠缺 |
| 学习任务可调度但未推进 | 生成学习推进需求 | `学习_唤醒可调度学习任务`、`学习_执行一条可调度学习任务` | 欠缺 |
| 方法运行账不完整 | 生成方法运行账完整性需求 | `管理_检查方法运行账完整性`、`管理_标记运行账不可用于可靠度` | 欠缺 |
| 方法样本缺失一侧 | 生成方法样本补齐需求 | `学习_补齐方法样本`、`管理_验证样本可召回性` | 欠缺 |
| 学习账本引用失效 | 生成学习账本引用修复需求 | `学习_检查学习账本引用`、`学习_重定位学习账本引用`、`学习_挂起失效学习任务` | 欠缺 |
| 方法虚拟存在缺失 | 生成方法虚拟存在完整性需求 | `管理_确保单承载存在`、`管理_确保方法虚拟存在` | 前者已有/后者建议补 |
| 可用方法数量镜像不一致 | 生成可用方法数量同步需求 | `自我_同步可用方法数量镜像` | 欠缺 |
| 待学习方法数量镜像不一致 | 生成待学习方法数量同步需求 | `自我_同步待学习方法数量镜像` | 欠缺 |

## 4. 已有本能函数复用边界

### 4.1 `学习_尝试学习`

现有职责：

- 读取待学习方法。
- 判断方法动作骨架、条件节点、结果节点是否缺失。
- 当缺失时返回结构化需求。
- 当动作、条件、结果均具备时，可写回 `方法是否可用=1`。

自检承接关系：

- 可复用为方法结构不完整的第一处理入口。
- 不应扩展成所有自检问题的通用处理器。

### 4.2 `自我_学习任务`

现有职责：

- 确保学习任务承接。
- 判断学习对象是否已具备可用性。

自检承接关系：

- 可复用为自检结构缺口、稳定性缺口、样本缺口的学习任务入口。

### 4.3 `管理_沉淀运行方法证据`

现有职责：

- 将运行后的方法证据沉淀到方法树、治理面或长期策略。

自检承接关系：

- 可复用为方法稳定性、运行账、样本完整性的证据来源。
- 但它不是完整性检查器；缺少运行账检查和不可恢复标记能力。

### 4.4 `自我_增加可用方法数量`

现有职责：

- 在确认新方法可用后执行 `可用方法数量 + 1`。

自检承接关系：

- 不能用于镜像一致性修复。
- 自检发现的是“镜像值 -> 统计值”，需要绝对同步本能，不是增量本能。

### 4.5 `自我_更新待学习方法数量`

现有状态：

- 已有方法头。
- 运行体是否完整需以代码注册为准。

自检承接关系：

- 若运行体已存在，可用于待学习方法数量同步。
- 若只有方法头，则仍欠缺绝对同步运行体。

## 5. 欠缺本能函数清单

第一版建议补以下本能函数。

### 5.1 `学习_补齐方法结构`

用途：

- 承接方法结构不完整需求。
- 统一处理动作骨架、条件节点、结果节点三类结构缺口。

输入特征：

| 特征类型 | 来源 | 值域 | 必需 |
| --- | --- | --- | --- |
| 待学习方法 | 学习任务/需求对象 | 方法节点引用 | 是 |
| 方法动作骨架数量 | 方法树 | `0` 或 `>0` | 是 |
| 方法条件节点数量 | 方法树 | `0..I64_MAX` | 是 |
| 方法结果节点数量 | 方法树 | `0..I64_MAX` | 是 |
| 方法是否可用 | 方法虚拟存在 | `0/1` | 否 |

返回特征：

| 特征类型 | 返回值或方向 | 说明 |
| --- | --- | --- |
| 方法动作骨架数量 | `0 -> 1` 或保持 `>0` | 缺动作时返回方向 |
| 方法条件节点数量 | `0 -> 1` 或保持 `>0` | 缺条件时返回方向 |
| 方法结果节点数量 | `0 -> 1` 或保持 `>0` | 缺结果时返回方向 |
| 方法是否可用 | 当三项均具备时返回 `1` | 枚举/布尔具体值 |

处理边界：

- 不直接提升可靠度。
- 不直接增加可用方法数量，除非进入“确认方法可用”任务并由对应本能处理。

### 5.2 `学习_验证方法稳定性`

用途：

- 承接方法稳定性不足需求。
- 验证方法是否可继续默认召回或需要降权/再学习。

输入特征：

| 特征类型 | 来源 | 值域 | 必需 |
| --- | --- | --- | --- |
| 方法可靠度 | 方法虚拟存在 | `I64` | 是 |
| 方法最近质量 | 方法运行账 | `I64` | 是 |
| 方法连续失败次数 | 方法运行账 | `0..I64_MAX` | 是 |
| 方法异常次数 | 方法运行账 | `0..I64_MAX` | 是 |
| 方法运行次数 | 方法运行账 | `0..I64_MAX` | 是 |
| 最近错误码 | 方法运行账 | `I64` | 否 |

建议阈值：

| 条件 | 判定 |
| --- | --- |
| 方法运行次数 `< 3` | 稳定性证据不足 |
| 连续失败次数 `>= 2` | 稳定性不足 |
| 异常次数 `> 0` 且最近一次异常未被恢复 | 稳定性不足 |
| 方法可靠度低于同类候选或小于等于 `0` | 稳定性不足 |
| 最近质量小于等于 `0` | 稳定性不足 |

返回特征：

| 特征类型 | 返回值或方向 | 说明 |
| --- | --- | --- |
| 方法稳定状态 | `稳定/不稳定/证据不足` | 枚举值 |
| 方法可靠度 | 不足时返回 `当前值 -> 当前值 + 增量` | 建议第一版增量为 `1` |
| 方法允许默认执行 | `0/1` | 不稳定时建议 `0` |
| 方法允许自动召回 | `0/1` | 证据不足可返回 `0` |

处理边界：

- 不直接删除方法。
- 不直接改任务当前方法；后续由任务治理决定换方法或重筹办。

### 5.3 `学习_提升方法可靠度`

用途：

- 在稳定性验证通过、样本补齐或学习回流成功后，提高方法可靠度。

输入特征：

| 特征类型 | 来源 | 值域 | 必需 |
| --- | --- | --- | --- |
| 方法可靠度 | 方法虚拟存在 | `I64` | 是 |
| 学习回流成功 | 学习结果 | `0/1` | 是 |
| 方法结构完整 | 方法树 | `0/1` | 是 |
| 方法样本完整 | 方法树/方法运行账 | `0/1` | 否 |

返回特征：

| 特征类型 | 返回值或方向 | 说明 |
| --- | --- | --- |
| 方法可靠度 | `当前值 -> 当前值 + 增量` | 建议第一版增量为 `1` |
| 方法稳定状态 | `稳定` 或 `证据不足` | 枚举值 |

处理边界：

- 只有学习回流成功或验证成功时允许提升。
- 不允许因为自检发现问题而直接提升可靠度。

### 5.4 `管理_检查方法运行账完整性`

用途：

- 承接方法运行账不完整需求。
- 判断运行账是否能用于可靠度、样本沉淀和自动召回。

输入特征：

| 特征类型 | 来源 | 值域 | 必需 |
| --- | --- | --- | --- |
| 方法运行次数 | 方法运行账 | `0..I64_MAX` | 是 |
| 最近开始时间 | 方法运行账 | 时间戳或 `0` | 是 |
| 最近结束时间 | 方法运行账 | 时间戳或 `0` | 是 |
| 最近耗时 | 方法运行账 | `0..I64_MAX` | 是 |
| 最近结果场景 | 方法运行账 | 场景引用或空 | 是 |
| 最近错误码 | 方法运行账 | `I64` | 是 |
| 成功次数 | 方法运行账 | `0..I64_MAX` | 是 |
| 失败次数 | 方法运行账 | `0..I64_MAX` | 是 |
| 异常次数 | 方法运行账 | `0..I64_MAX` | 是 |

返回特征：

| 特征类型 | 返回值或方向 | 说明 |
| --- | --- | --- |
| 方法运行账完整性 | `完整/不完整/不可恢复` | 枚举值 |
| 方法可参与可靠度提升 | `0/1` | 不完整时返回 `0` |
| 方法可参与自动召回 | `0/1` | 不完整且无样本时返回 `0` |

处理边界：

- 不伪造缺失的历史事实。
- 对不可恢复的账，只能标记不可用于可靠度提升或自动召回。

### 5.5 `学习_补齐方法样本`

用途：

- 承接方法样本缺失一侧需求。
- 处理有结构但条件样本或结果样本不足的问题。

输入特征：

| 特征类型 | 来源 | 值域 | 必需 |
| --- | --- | --- | --- |
| 方法条件样本数量 | 方法树 | `0..I64_MAX` | 是 |
| 方法结果样本数量 | 方法树 | `0..I64_MAX` | 是 |
| 方法结构完整 | 方法树 | `0/1` | 是 |
| 方法最近结果场景 | 方法运行账 | 场景引用或空 | 否 |

返回特征：

| 特征类型 | 返回值或方向 | 说明 |
| --- | --- | --- |
| 方法条件样本数量 | `0 -> 1` 或保持 `>0` | 缺条件样本时返回方向 |
| 方法结果样本数量 | `0 -> 1` 或保持 `>0` | 缺结果样本时返回方向 |
| 方法样本完整性 | `完整/不完整/证据不足` | 枚举值 |

处理边界：

- 不把方法结构节点数量等同于样本数量。
- 样本缺失时不得直接标记方法稳定。

### 5.6 `管理_验证样本可召回性`

用途：

- 判断方法条件/结果样本是否足以支持自动召回。

输入特征：

| 特征类型 | 来源 | 值域 | 必需 |
| --- | --- | --- | --- |
| 方法条件样本数量 | 方法树 | `0..I64_MAX` | 是 |
| 方法结果样本数量 | 方法树 | `0..I64_MAX` | 是 |
| 条件判定索引数量 | 方法树 | `0..I64_MAX` | 是 |
| 结果方向索引数量 | 方法树 | `0..I64_MAX` | 是 |
| 方法可靠度 | 方法虚拟存在 | `I64` | 否 |

返回特征：

| 特征类型 | 返回值或方向 | 说明 |
| --- | --- | --- |
| 方法可自动召回 | `0/1` | 枚举/布尔具体值 |
| 方法样本可召回性 | `可召回/不可召回/证据不足` | 枚举值 |

处理边界：

- 不直接执行召回。
- 只给任务治理和方法选择提供事实。

### 5.7 `学习_检查学习账本引用`

用途：

- 承接学习账本引用失效需求。
- 判断学习任务、待学习方法、来源任务、来源需求是否可解析。

输入特征：

| 特征类型 | 来源 | 值域 | 必需 |
| --- | --- | --- | --- |
| 学习任务引用 | 学习账本 | 任务引用 | 是 |
| 待学习方法引用 | 学习账本 | 方法引用 | 是 |
| 来源任务引用 | 学习账本 | 任务引用或空 | 否 |
| 来源需求引用 | 学习账本/任务 | 需求引用或空 | 否 |
| 当前学习状态 | 学习账本 | 枚举 | 是 |

返回特征：

| 特征类型 | 返回值或方向 | 说明 |
| --- | --- | --- |
| 学习账本引用有效性 | `有效/无效/部分无效` | 枚举值 |
| 学习任务状态 | `挂起` 或保持原状态 | 引用无效且不可重定位时返回 `挂起` |
| 学习账本需重建 | `0/1` | 无法恢复时返回 `1` |

处理边界：

- 检查函数不直接重定位。
- 重定位由 `学习_重定位学习账本引用` 承接。

| 待学习方法主键 | 学习账本 | 字符串主键 | 是 |
| 来源任务主键 | 学习账本 | 字符串主键或空 | 否 |
| 来源需求主键 | 学习账本 | 字符串主键或空 | 否 |

返回特征：

| 特征类型 | 返回值或方向 | 说明 |
| --- | --- | --- |
| 学习账本引用有效性 | `有效/部分无效/无效` | 枚举值 |
| 学习账本重定位状态 | `已重定位/部分重定位/失败` | 枚举值 |
| 学习任务状态 | `可调度/挂起` | 重定位成功可恢复可调度 |

处理边界：

- 只允许按主键重定位到当前树内节点。

### 5.12 `管理_诊断任务停滞原因`

用途：

- 承接任务完成度停滞需求。
- 只诊断停滞原因，不直接恢复。

输入特征：

| 特征类型 | 来源 | 值域 | 必需 |
| --- | --- | --- | --- |
| 任务状态 | 任务树 | 任务状态枚举 | 是 |
| 任务完成度 | 任务虚拟存在或任务记录 | `I64` | 否 |
| 最近调度时间 | 任务主信息 | 时间戳或 `0` | 是 |
| 最近执行时间 | 任务主信息 | 时间戳或 `0` | 是 |
| 最近完成时间 | 任务主信息 | 时间戳或 `0` | 是 |
| 连续失败次数 | 任务主信息 | `0..I64_MAX` | 是 |
| 等待学习唤醒 | 任务主信息 | `0/1` | 否 |

返回特征：

| 特征类型 | 返回值或方向 | 说明 |
| --- | --- | --- |
| 任务停滞原因 | `缺方法/等待学习/等待结果/连续失败/未知` | 枚举值 |
| 任务推进状态 | `停滞` 或 `可继续推进` | 枚举值 |
| 建议任务状态 | `待重筹办/等待中/挂起` | 枚举值 |

处理边界：

- 不直接改任务状态。
- 后续由任务治理决定重筹办、唤醒或挂起。

## 6. 第一版优先级

第一版建议先补：

1. `学习_补齐方法结构`
2. `学习_验证方法稳定性`
3. `管理_检查方法运行账完整性`
4. `学习_检查学习账本引用`
5. `自我_同步可用方法数量镜像`
6. `自我_同步待学习方法数量镜像`

原因：

- 这些函数直接对应当前第一批自检条目。
- 输入特征大多已存在或可从基础树读取。
- 输出可以表达为现有需求特征或通用状态特征。
- 不会重复当前任务筹办、任务执行一步、学习尝试学习的主职责。
