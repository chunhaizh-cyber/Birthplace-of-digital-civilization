## 4. 归类总原则

固定以下 6 条：

1. `请求快照` 优先属于 `实例堆`，默认不直接镜像整包到宿主任务。
2. `治理结果`、`治理原因`、`等待类型`、`最近治理时刻` 这类可观察状态，应镜像到宿主任务虚拟存在。
3. `锁占有`、`mailbox`、`队列成员关系`、`原始指针绑定后的瞬时引用` 只属于实例堆。
4. `恢复快照` 只保留“重建任务管理实例所需的最小状态”，不得照搬整个请求或整个实例。
5. `最近请求` 允许作为实例堆内部调试/恢复辅助信息保留，但其内部字段必须拆解后才允许进入恢复快照。
6. 若某字段同时满足“可观察”和“可恢复”，则应同时落到 `宿主任务虚拟存在` 和 `恢复快照`，但表达粒度可以不同。
7. `结构_任务管理请求` 必须理解为“根层已裁决后的一轮输入材料”，而不是根裁决来源本体。
8. 请求里承载的 `控制输入`，后续应优先拆成 `根层裁决结果` 与 `任务管理上下文关键字段`，不得长期维持为单一黑箱包。

更细的 `任务管理上下文关键字段` 定义，统一按 [1214_任务管理上下文最小字段表规范.md](D:\鱼巢\规范\1214_任务管理上下文最小字段表规范.md) 执行。


## 5. `结构_任务管理请求` 字段归类

对应结构：

- [自我线程模块.ixx](D:\鱼巢\自我线程模块.ixx#L342)

### 5.1 字段表

| 字段 | 当前作用 | 主归属层 | 是否镜像到宿主任务 | 是否进入恢复快照 | 说明 |
|---|---|---|---|---|---|
| `宿主任务头结点` | 绑定本轮宿主任务 | `实例堆` | 否 | 是 | 恢复时必须知道实例服务哪个宿主任务。 |
| `控制输入` | 本轮总控输入快照 | `实例堆` | 否 | 否 | 原始控制输入属于单轮判定材料，不宜整包镜像；后续应优先拆解出 `根层裁决结果 / 当前功能域输入 / 当前缺口类型 / 当前下一步去向` 等稳定字段。 |
| `最近反馈快照` | 脱离原指针后的反馈快照 | `实例堆` | 否 | 视情况 | 恢复快照只允许保留“最近反馈摘要”，不保留整包原始结构。 |
| `提交时间` | 本轮请求生成时刻 | `实例堆` | 否 | 是 | 可作为恢复后判断“最近一次治理提交”的基准。 |
| `提交序号` | 请求去重与排序 | `实例堆` | 可选 | 是 | 恢复时可用来防止旧请求重放。 |
| `标签` | 运行日志/调试标签 | `实例堆` | 否 | 可选 | 默认不要求镜像；仅在恢复审计需要时保留。 |

### 5.2 结论

`结构_任务管理请求` 整体应被视为：

- `实例堆内部单轮输入材料`

它不应整体写进宿主任务虚拟存在。  
若需要跨退出恢复，只允许拆出：

- `宿主任务头结点`
- `提交时间`
- `提交序号`
- `最近反馈摘要`

这几项进入恢复快照。

### 5.3 `控制输入` 的后续拆解要求

虽然当前代码里 `控制输入` 仍是一个打包字段，但规范上必须先固定其未来拆解方向：

1. `根层裁决结果`
2. `当前功能域输入`
3. `当前缺口类型`
4. `当前下一步去向`
5. 必要的 `最近结果摘要 / 原因说明`

必须固定：

- 这些字段未来若从 `控制输入` 中拆出，其中“可观察、可恢复”的部分应优先镜像到宿主任务虚拟存在和恢复快照。
- 纯单轮判定材料仍只留在实例堆，不得为追求方便整包写入任务事实。


## 6. `结构_任务管理实例` 字段归类

对应结构：

- [自我线程模块.ixx](D:\鱼巢\自我线程模块.ixx#L351)

### 6.1 字段表

| 字段 | 当前作用 | 主归属层 | 是否镜像到宿主任务 | 是否进入恢复快照 | 说明 |
|---|---|---|---|---|---|
| `宿主任务头结点` | 实例绑定宿主 | `实例堆` | 否 | 是 | 恢复时必须保留。 |
| `状态` | 当前实例运行状态 | `实例堆` | 是 | 是 | 运行态保留在实例堆，恢复态应镜像为治理恢复状态。 |
| `等待原因` | 当前等待类型 | `实例堆` | 是 | 是 | 是恢复点判定的关键字段。 |
| `最近请求` | 最近一次请求快照 | `实例堆` | 否 | 拆解后部分进入 | 不应整包镜像或整包持久化。 |
| `最近根层裁决结果` | 最近一轮根层是否允许继续推进 | `宿主任务虚拟存在` | 是 | 是 | 后续若当前代码补出此字段，应优先视为治理状态面和恢复依据。 |
| `最近功能域` | 最近一次判定到的功能域 | `宿主任务虚拟存在` | 是 | 是 | 典型的治理状态面。 |
| `最近总控结果` | 最近一次总控返回结果 | `宿主任务虚拟存在` | 是 | 是 | 恢复时判断下一步的重要依据。 |
| `当前缺口类型` | 当前一轮治理显性化后的主缺口 | `宿主任务虚拟存在` | 是 | 可选 | 对恢复与观察有价值，后续建议补出稳定字段。 |
| `当前下一步去向` | 当前一轮治理建议的下一条工程路径 | `宿主任务虚拟存在` | 是 | 是 | 既属于治理状态面，也应作为恢复后的继续提示。 |
| `最近开始时间` | 最近一次开始处理时间 | `宿主任务虚拟存在` | 是 | 是 | 可观察且可用于恢复判断。 |
| `最近结束时间` | 最近一次结束处理时间 | `宿主任务虚拟存在` | 是 | 是 | 可观察且可用于恢复判断。 |
| `最近唤醒时间` | 最近一次被事件/提交唤醒的时间 | `宿主任务虚拟存在` | 是 | 是 | 恢复时可帮助判断是否存在陈旧等待。 |
| `当前等待开始时间` | 当前等待态开始时刻 | `实例堆` | 是 | 是 | 对等待恢复和界面观察都有价值。 |
| `最近处理序号` | 最近处理过的请求序号 | `实例堆` | 是 | 是 | 用于恢复后防重放，建议也镜像到宿主任务。 |
| `持有推进权` | 当前是否占有推进权 | `实例堆` | 否 | 否 | 严禁持久化。 |
| `最近原因说明` | 最近一次总控/恢复原因说明 | `宿主任务虚拟存在` | 是 | 是 | 典型可观察、可审计、可恢复字段。 |

### 6.2 结论

`结构_任务管理实例` 不应被视为一个整体持久化对象。  
应拆成三类：

- `纯实例堆字段`
- `实例堆 + 宿主任务镜像字段`
- `实例堆 + 宿主任务镜像 + 恢复快照字段`

## 8. 严格只属于实例堆的字段

以下字段不得写入宿主任务，也不得进入恢复快照：

1. `持有推进权`
2. `任务管理待运行队列` 成员关系
3. `任务管理待运行集合` 成员关系
4. `任务管理延后重投队列` 成员关系
5. `任务管理延后重投集合` 成员关系
6. `最近请求.控制输入.最近反馈` 这类绑定后的内部指针
7. 任何锁、条件变量、queue cursor、mailbox 内部游标

一句话说：

`凡“线程当前正拿着什么”和“调度器当前把它放在哪个容器里”这类信息，都只属于实例堆。`


## 9. 恢复快照最小集

退出或重启时，建议最小恢复快照只保留：

1. `宿主任务头结点`
2. `最近根层裁决结果`
3. `最近功能域`
4. `最近总控结果`
5. `等待原因`
6. `当前下一步去向`
7. `最近开始时间`
8. `最近结束时间`
9. `最近唤醒时间`
10. `当前等待开始时间`
11. `最近处理序号`
12. `最近原因说明`
13. `最近提交时间`
14. `最近反馈摘要`

恢复后，应由这些信息重新构造新的 `结构_任务管理实例`，而不是反序列化旧实例对象。
