# 0038 关键特征变化影响项与服务安全聚合规范 v0.1

## 1. 目标

本文用于把 `自我线程` 的长期目标，从“记住所有特征值”进一步收紧成：

`逐步建立关键特征变化 -> 二次特征判断 -> 影响项 -> 服务值/安全值聚合` 的可解释映射图。

本文回答：

- 为什么不能把所有一次特征直接硬连到 `服务值 / 安全值`
- 什么叫 `影响项`
- 影响项至少应包含哪些维度、方向、强度、时延、可逆性与成立条件
- 自我线程应如何把 `一次特征变化 + 二次特征` 继续归结成 `影响项`
- 第一批最值得先覆盖哪些特征域、二次特征和影响维度

本文不负责：

- 一次展开全部特征类型的影响模板全集
- 直接替代 `0310 / 0322` 对正式双值结算的上位规则
- 直接扩写线程代码实现

## 3. 核心结论

必须固定以下结论：

1. `自我线程` 的长期目标，不是直接记住“所有特征值”，而是逐步建立 `关键特征变化 -> 二次特征 -> 影响项 -> 服务值/安全值聚合` 的可解释映射图。
2. 同一个一次特征变化，在不同条件下，对服务和安全的影响可能完全相反；因此不得把“特征类型”直接硬连到“结果方向”。
3. `启动 / 暂停 / 继续 / 打断 / 收束 / 停止` 当然也是真实特征值变化，但它们属于 `控制域特征值变化`，不等于 `业务结果事实`。
4. `影响项` 是线程在消费一次特征变化并重算二次特征之后，对 `服务 / 安全` 影响的正式归结层。
5. 根层重判和双值结算，最终应优先消费 `影响项聚合结果`，而不是直接消费原始消息名字或孤立特征值。
6. 否定项的重要来源，不只是“最终失败”，还包括“关键中间状态变化 + 二次特征判定后生成的负向影响项”。
7. 来自 `任务`、`任务控制态`、`任务管理任务` 三层的变化，对服务和安全的影响路径可能不同；影响模板不得跨层偷混。
8. 其它线程状态变化进入主链后的目标，不是证明“那个线程变了”，而是评估“这件事对自我的执行能力、等待关系、风险和整体治理节奏意味着什么”。
9. `自我线程` 自身的关键生命周期变化属于例外：它们优先落 `关键日志 / 宿主级审计 / 恢复判断材料`，不作为本文默认展开的普通影响模板对象。


## 4. 四层归结主链

推荐固定主链为：

```text
一次特征变化
-> 二次特征判断
-> 影响项
-> 服务值/安全值聚合
```

必须固定：

1. `一次特征变化` 只回答“谁的哪个值变了”。
2. `二次特征` 只回答“这些变化在当前条件下意味着什么”。
3. `影响项` 只回答“这些意义对服务和安全分别贡献了什么方向、强度、时延与可逆性”。
4. `服务值/安全值聚合` 才回答“这一轮在根判定层上总体是缩差、持平还是恶化”。


## 5. 影响项定义

### 5.1 影响维度

影响维度最小固定为四类：

- `直接服务`
- `准备性服务`
- `物理安全`
- `风险安全`

### 5.2 影响方向

影响方向最小固定为三类：

- `提升`
- `降低`
- `中性`

### 5.3 推荐最小结构

```cpp
enum class 枚举_影响维度 : std::uint8_t {
    直接服务 = 0,
    准备性服务 = 1,
    物理安全 = 2,
    风险安全 = 3
};

enum class 枚举_影响方向 : std::uint8_t {
    提升 = 0,
    降低 = 1,
    中性 = 2
};

struct 结构_影响项 {
    主键 主体主键 = 0;
    主键 来源特征主键 = 0;
    主键 来源二次特征主键 = 0;

    枚举_影响维度 影响维度 = 枚举_影响维度::直接服务;
    枚举_影响方向 影响方向 = 枚举_影响方向::中性;

    I64 影响强度 = 0;
    bool 即时生效 = true;
    bool 可逆 = true;
    I64 置信度 = 0;

    std::string 成立条件摘要{};
    std::string 影响说明{};
};
```

必须固定：

1. `来源特征主键` 回答“最初是哪类一次特征变化触发了这条影响”。
2. `来源二次特征主键` 回答“这条影响是被哪一个判断结果确认的”。
3. `影响项` 不是原始事实，也不是最终结算值，而是双值结算前的正式归结层。


## 6. 为什么不能把所有一次特征直接连到双值

### 6.1 控制域例子

`工作线程.控制意图 = 请求暂停`

这条变化并不天然等于“负面”。

- 在故障扩散时，它对 `安全` 可能是正向影响
- 在正常推进中，它对 `服务` 可能是负向影响

所以必须经过：

- 当前控制面条件
- 当前风险条件
- 当前步骤可执行性

之后，才能归结成正式影响项。

### 6.2 资源域例子

`资源 R.可用性 = 可用`

这条变化也不天然等于“正面”。

只有当：

- 某任务当前确实依赖 `R`
- 且该任务之前因 `R` 不可用而阻塞

它才可能进一步转成：

- `依赖满足 = true`
- `当前步骤可执行 = true`
- 再进一步转成对 `准备性服务 / 直接服务` 的正向影响

一句话收口：

`系统最终要懂的，不是“某个值变了”，而是“某个值在什么条件下意味着什么”。`


## 7. 第一批最小覆盖面

### 7.1 第一批一次特征域

建议先覆盖五类：

- 控制域特征
- 线程状态域特征
- 结果域特征
- 依赖域特征
- 资源域特征

### 7.2 第一批二次特征

建议先覆盖四类：

- `依赖满足`
- `当前步骤可执行`
- `反向推进标记`
- `有效优先级候选`

### 7.3 第一批影响维度

建议先覆盖四类：

- `直接服务影响`
- `准备性服务影响`
- `物理安全影响`
- `风险安全影响`

必须固定：

1. 第一批不追求覆盖全世界，只追求 `小切口 + 强闭环`。
2. 任何影响模板，都必须写清成立条件，不得用“特征类型名字”替代条件面。


## 8. 表 1：最小影响模板表

| 一次特征变化 | 关键条件 | 触发的二次特征 | 影响维度 | 可能方向 | 备注 |
| --- | --- | --- | --- | --- | --- |
| `工作线程.控制意图 = 请求暂停` | 正处于异常扩散或高风险 | `风险升高已成立` 或 `继续执行被门控拒绝` | `物理安全 / 风险安全` | `提升` | 控制域变化可形成安全正向影响 |
| `工作线程.控制意图 = 请求暂停` | 正常推进且无异常扩散 | `当前步骤可执行 = false` | `直接服务` | `降低` | 同一控制域变化可形成服务负向影响 |
| `工作线程.生命周期状态 = 运行中` | 自我当前存在待执行任务，且该线程此前未可用 | `可用执行能力上升` 或 `等待/阻塞缓解` | `准备性服务` | `提升` | 线程恢复可用，通常先改善准备性服务 |
| `工作线程.生命周期状态 = 故障` | 该线程承接关键任务、关键能力或共享池容量紧张 | `可用执行能力下降`、`反向推进标记 = 远离目标` | `准备性服务 / 风险安全` | `降低` | 线程故障常同时压低执行准备面并抬高风险 |
| `工作线程.实际运行状态 = 已阻塞` | 阻塞仍未解除，且上层存在等待关系 | `等待/阻塞加重`、`当前步骤可执行 = false` | `准备性服务 / 直接服务` | `降低` | 线程阻塞不只是线程局部事实，也会拖慢自我整体推进 |
| `子任务.结果状态 = 成功` | 父任务等待子结果 | `依赖满足 = true`、`当前步骤可执行 = true` | `准备性服务 / 直接服务` | `提升` | 典型结果域正向模板 |
| `资源.可用性 = 可用` | 任务此前等待该资源 | `依赖满足 = true` | `准备性服务` | `提升` | 若无任务依赖，则可退化为 `中性` |
| `关键中间状态 = 风险升高` | 已确认生效，且后续虽恢复但曾真实出现 | `反向推进标记 = 远离目标` | `风险安全` | `降低` | 是否定项的重要来源 |


## 9. 表 2：最小聚合口径

| 聚合目标 | 最小来源 | 聚合方式 | 输出 |
| --- | --- | --- | --- |
| `服务净变化` | `直接服务影响项 + 准备性服务影响项` | 按方向与强度求和，再结合置信度校正 | `服务净变化摘要` |
| `安全净变化` | `物理安全影响项 + 风险安全影响项` | 按方向与强度求和，再结合置信度校正 | `安全净变化摘要` |
| `否定项候选` | `关键中间状态 + 负向影响项` | 取反向推进、阻塞、副作用、恶化项 | `否定项候选摘要` |

必须固定：

1. `准备性服务` 允许先记为服务正向，但不得直接冒充“正式叶子需求已满足”。
2. `物理安全` 与 `风险安全` 都属于安全面；聚合时可以分别保留，也可以在更高层再汇总。
3. 具体结算规则仍服从 `0310 / 0322`，`0038` 只定义影响项层。


## 10. 自我线程中的最小处理顺序

自我线程收到消息后，推荐固定为：

1. 消息规范化
2. 落一次特征账
3. 重算受影响二次特征
4. 生成影响项
5. 聚合成服务值/安全值影响摘要
6. 再进入根层重判与主派发

若当前沿用 `0037` 的 8 步骨架，则固定解释为：

- 第 `2` 步负责 `一次特征变化`
- 第 `3` 步负责 `二次特征`
- 第 `4` 步负责 `影响项与双值影响摘要`
- 第 `5 / 8` 步负责消费这些影响摘要，进入根层重判与结算
