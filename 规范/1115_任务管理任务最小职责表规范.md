## 3. 核心结论

必须固定以下结论：

1. `任务管理任务` 是任务系统中的特殊治理任务，但它仍然是任务，不是系统外的超级管理员。
2. `任务管理任务` 的职责是：在 `根层已给出本轮允许边界` 的前提下，承接宿主任务的一步治理推进。
3. `任务管理任务` 不负责重新定义根方向，不负责直接结算 `安全值 / 服务值`，也不负责直接生成或合并需求。
4. `任务管理任务` 负责的是：围绕宿主任务，生成管理事实、管理建议、管理证据，并落实一步合法治理状态迁移。
5. `任务管理任务` 的产出，优先属于 `条件判定结果层` 与 `管理上下文层`，不是最终裁决本身。
6. `任务管理任务` 每轮最多只推进一步，推进结果必须回到外层执行 `根层重判 -> 上下文刷新 -> 再决定是否继续`。
7. `任务管理任务` 可以提出 `继续执行 / 补条件 / 转学习 / 挂起等待 / 回到重筹办 / 进入收束` 等去向候选，但这些候选不得越过外层重判直接冒充最终结论。
8. `任务管理任务` 可以出生或补齐任务树结构、方法挂点、学习子任务，但不能伪造叶子执行成功、外部结果成功或需求已满足结论。
9. `任务管理任务` 默认优先沿当前宿主任务树继续治理，不应每轮都对根任务和宿主任务从头重梳；只有当前已无可执行/可继续推进子任务，或系统刚重启/休眠恢复后，才允许重头梳理。
10. `任务管理任务` 可以依附 `任务管理主体` 做跨宿主、跨候选路径和跨学习反馈的成长汇总，但常规治理轮仍然只围绕一个宿主推进一步；主体级多分身不等于同宿主并发多实例。
11. 若需要做主体级的全局宿主筛选、压力重排或巡视治理，必须由主循环显式提交 `全局巡视治理请求` 或等价请求类型，不得在普通宿主治理轮中无边界盲扫。
12. `任务完成` 是任务管理主体的局部运行目标，`安全值 / 服务值` 仍然是上位方向约束。
13. 从自我线程一级模块层级看，真正的一级治理模块应称为 `任务管理中轴`。
14. `任务管理任务` 是 `任务管理中轴` 在本轮承接或生成的运行态治理实体。
15. `策略沙盘与方案演练` 不是和自我线程并列的一级模块，也不是 `任务管理任务` 本身，而是 `任务管理中轴 / 任务管理任务` 内部可调用的方案比较子模块。
16. 文档层统一称 `任务管理中轴`，运行态实体继续称 `任务管理任务`，运行态产物称 `一步治理任务包`；若代码里落类，统一建议称 `任务管理模块类`。

可收敛为一句话：

`任务管理任务负责在根层允许边界内，围绕宿主任务完成一步治理承接；它产出的是治理事实、治理建议和治理证据，而不是跳过外层重判的最终真理。`

更细的 `根任务 / 宿主任务 / 完成树保留 / 结果回挂` 规则，按 [0975_根任务宿主任务与任务树保留复用规范.md](D:\鱼巢\规范\0975_根任务宿主任务与任务树保留复用规范.md) 执行。


## 4. 在统一主链中的位置

当前统一主链应固定为：

```text
自我线程主循环
-> 消费消息并刷新根层输入
-> 根层重判
-> 整理需求与双值
-> 形成任务管理上下文
-> 调用任务管理中轴
-> 任务管理任务推进宿主任务前进一步
-> 返回一步结果、治理证据与去向候选
-> 自我线程再次做根层重判与主链调度
```

因此必须固定：

1. `自我线程主循环` 是总协调者。
2. `任务管理任务` 是治理承接者。
3. `任务执行器` 是叶子执行承载者。
4. `学习子任务` 是缺口补齐承载者。
5. `外层根层重判` 才是最终是否继续本轮主链的上位约束输入。


## 5. 最小职责表

### 5.1 最小职责规范表

| 职责名 | 作用 | 最小直接输入 | 最小直接输出 | 禁止越权 |
| --- | --- | --- | --- | --- |
| `接收宿主任务与管理上下文` | 明确本轮到底在管哪个宿主任务、在什么边界下治理 | `根层重判结果`、`宿主任务头`、`当前步骤`、`当前方法首节点`、`当前缺口类型`、`当前下一步去向`、`最近反馈` | `本轮治理输入快照` | 不得自行全局盲扫决定“今天想管谁” |
| `判当前治理域` | 判断本轮应进入 `筹办 / 维护 / 纠偏 / 收束 / 等待学习处理` 中的哪一域 | `宿主任务当前状态`、`最近结果`、`当前缺口类型`、`根层允许边界` | `当前功能域判定结果` | 不得把域判定冒充根方向裁决 |

| `生成当前下一步去向候选` | 给出本轮更像应该往哪走的治理路径 | `当前功能域`、`主缺口类型`、`最近反馈`、`当前结果` | `下一步去向候选` | 不得把候选直接当最终裁决 |

### 5.2 当前阶段最少必须做到的职责

当前阶段若要承认 `任务管理任务` 已最小成立，至少必须能做到：

1. 明确接收 `宿主任务 + 根层允许边界 + 最小任务管理上下文`。
2. 明确给出 `当前功能域判定结果`。
3. 明确给出 `当前缺口类型` 与 `当前方法需求位`。
4. 明确给出 `当前下一步去向候选` 与 `当前本能能力缺口类型`。
5. 在需要时能推进以下任一治理动作：
   - 出生或切换一步结构
   - 绑定或切换一个方法挂点
   - 下发一次执行
   - 进入一个等待态
   - 生成一个学习子任务候选
6. 本轮结束后能返回 `一步治理结果` 给外层主循环，而不是在内部继续连跑多轮闭环。


## 6. 留在自我线程主循环的职责

以下职责必须留在 [0014_自我线程主循环最小职责表规范.md](D:\鱼巢\规范\0014_自我线程主循环最小职责表规范.md) 所定义的 `自我线程主循环`，不得下沉给 `任务管理任务`：

1. `根层重判`
2. `安全值 / 服务值` 统一结算
3. `需求整理、需求合并、需求权重调整`
4. `根任务与常驻治理任务存在维护`
5. `整个主链最终去向决定`
6. `待机 / 停机边界处理`

原因是：

- 这些职责都直接涉及根层不变量或主链统一协调权。
- `任务管理任务` 只能消费这些结果，不能反向改写这些结果。


## 7. 不属于任务管理任务的职责

以下职责不应归到 `任务管理任务`：

1. 长时间叶子方法执行
2. 外设阻塞式等待
3. 外部世界原始事实采集线程本身
4. 根层安全/服务总裁决
5. 方法最终升格裁决
6. 学习反馈后的最终允许裁决
7. 需求是否满足的最终判断

这些职责要么属于：

- `叶子执行器`
- `学习子任务`
- `自我线程主循环`
- `外层重判`


## 8. 与方法、学习、筹办成长的关系

必须固定：

1. `任务树结构成长` 主要靠 `任务筹办` 完成。
2. `任务管理任务` 负责触发和承接筹办，不负责把所有成长都压成执行器内部动作。
3. `方法头出生` 不要求当前就可执行；只要当前方法已能被结构化表达，就允许作为候选方法挂点进入管理链。
4. `方法可用 / 可执行 / 成熟度` 属于方法后续状态，不属于方法存在本身。
5. `学习` 主要负责补方法存在面、补可用性、补可靠性、补条件面；`任务管理任务` 负责在需要时把这种补齐诉求转成学习子任务候选。
6. `方法升格` 和 `学习反馈后的使用决定` 都不是任务管理任务可以单方定论的；任务管理任务只能提交候选、回收结果并把证据返回外层。
7. `方法需求位` 是成长需求与具体方法枝之间的正式桥层，不得跳过。
8. 当命中的 `方法需求位` 仍缺少最小动作原语时，`任务管理任务` 应显式返回 `本能能力缺口`，并允许把该缺口转成学习承接输入。


## 9. 最小结果面

更细的字段表、分层与是否镜像/恢复，按 [1213_任务管理一步治理结果最小返回面规范.md](D:\鱼巢\规范\1213_任务管理一步治理结果最小返回面规范.md) 执行。

`任务管理任务` 每轮返回外层时，最少应带回以下结果面：

- `当前功能域判定结果`
- `当前缺口类型`
- `当前下一步去向候选`
- `当前治理状态迁移`
- `当前步骤 / 方法 / 结果节点` 的最新镜像
- `本轮产生的管理证据`
- 若有，`学习子任务候选`
- 若有，`收束建议 / 重筹办建议`
- 若有，`当前方法需求位 / 当前本能能力缺口类型 / 输入条件主键 / 治理态型 / 主体回并摘要`

必须固定：

1. 这些结果面首先属于 `条件判定结果层` 或 `管理上下文更新`。
2. 外层仍需基于这些结果重新执行 `根层重判 -> 主链调度`。
3. 不允许把其中任一结果面直接跳升成“需求已满足”或“主链已结束”的最终结论。

2. 本文的作用不是新造一套职责，而是把这些职责重新压回一个统一表述。
