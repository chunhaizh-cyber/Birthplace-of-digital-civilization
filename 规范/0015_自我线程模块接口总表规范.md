## 1. 目标

本文用于按当前最新理念，重新固定 `自我线程` 的模块边界。

本文只回答以下问题：

- 自我线程直接治理什么对象
- 自我线程主线上最少应分成哪些治理模块
- 每个模块最小负责什么、读什么、写什么、能调用谁
- 哪些细化能力当前只保留接口，不继续拆成并列一级模块

本文不负责：

- 具体算法
- 线程池与锁细节
- 叶子执行逻辑
- 细化样机分期

## 3. 核心定义

必须固定：

1. `自我线程` 是面向 `需求信息` 的消息治理中枢。
2. `消息` 是输入，不是直接治理对象。
3. `需求信息` 的最小运行时承载至少包括 `需求树 / 需求队列 / 方法禁止项集`。
4. `需求列表` 是 `需求队列` 的线程持有队列形态；由 `自我线程` 从 `需求树` 的 `链上 / 链下` 生成和维护。
5. `自我线程` 承担的是 `控制面职责`，不是 `任务完成面职责`。
6. `自我线程` 必须先根据最新 `需求树` 按父需求权重调制子需求权重，再生成 `需求队列 / 需求列表` 并计算综合权重。
7. 生成 `需求列表` 时，必须先按 `需求类型` 聚拢，使相同需求类型连续排列，再统计类型组 `权重汇总`，并按权重完成列表排序。
8. 任一需求节点权重变化后，`自我线程` 必须同步刷新 `需求列表` 中对应类型组的权重汇总，并在必要时重排列表。
9. `自我线程` 必须依据双值变化趋势生成和更新 `方法禁止项集`。
10. `自我线程` 统一管理 `可运行队列 / 等待队列 / 回流队列 / 收束停止队列` 这组全局运行队列。
11. `自我线程` 负责创建 `任务根节点 / 任务入口实例`，并调度对应的 `任务管理工作线程`。
12. `任务管理任务` 只负责在 `只读任务根节点` 约束下推进步骤并返回结果。
13. `任务管理任务` 可以提出 `派生需求`，但不能直接生成子任务实例。
14. 子结果必须先回到自我线程，再由自我线程路由给父层任务；父层任务判断局部影响，自我线程判断全局影响。
15. `工作线程` 只负责承接一步治理，不负责改写根方向，也不得绕过 `方法禁止项集` 执行被禁方法。
16. `策略沙盘与方案演练`、`方法匹配 / 方法选择 / 执行策略比较` 不再作为自我线程并列一级模块；它们只属于 `任务管理任务` 内部的策略子模块。
17. `学习`、`安全守卫`、`授权限幅`、`执行监督` 当前只保留接口位，不再单独拉成主线并列一级模块。
18. 自我线程每轮必须先冻结本轮开始时已可见的消息批次，并处理完这批消息及其同步内部闭包，再做一次根层重判与主派发/收束决议。
19. 本轮处理中异步新产生的消息必须延后到下一轮；高优先级控制消息可按抢占规则中断本轮。
20. `控制消息` 与 `全局环境消息` 的首收件人必须是 `自我线程`。
21. `明确绑定到具体任务且仅影响局部流程的外部反馈`，可先进入对应任务管理工作线程；若升级为 `派生需求 / 阻塞变化 / 异常 / 需要改优先级 / 需要实例化新任务`，必须再上报 `自我线程`。
22. `结果回流消息` 必须先回 `自我线程`；`推进消息` 只上报会改变控制面判断的部分。
23. `控制态真值` 由 `自我线程` 持有，任务管理工作线程只接收与自身相关的 `控制投影`。
24. `消息驱动的需求增量刷新` 由 `自我线程` 直接完成；这是线程内部最重的控制面工作。
25. `需求树 / 需求队列` 的定时全局刷新必须任务化，由 `自我线程` 发起为刷新任务，再交给任务管理工作线程执行。

一句话收口：

`自我线程消费消息，形成根需求，治理需求树与需求队列，更新方法禁止项集，实例化任务入口，调度任务管理工作线程，路由结果并做最终结算。`


## 4. 当前一级模块

按当前实现口径，自我线程主线上固定为以下 `6` 个一级模块：

1. `生命周期与边界控制`
2. `状态/事件归并与根需求形成`
3. `全局优先级与队列治理`
4. `任务入口实例化与任务管理工作线程调度`
5. `结果路由与最终结算`
6. `学习触发与恢复接口`

这 `6` 个模块是当前规范允许保留在主线上的粗粒度模块。

以下能力当前不再单列为自我线程并列一级模块：

- `方法匹配与选择`
- `策略沙盘与方案演练`
- `授权 / 配额 / 限幅`
- `执行调度与过程监督`
- `安全守卫与止损`
- `学习与因果沉淀`

它们若仍需存在，只能以：

- `任务管理任务` 内部子模块
- `工作线程接口`
- `回流结算接口`

三种方式继续存在。

## 6. 主线顺序

当前主线顺序固定为：

`生命周期与边界控制 -> 状态/事件归并与根需求形成 -> 全局优先级与队列治理 -> 任务入口实例化与任务管理工作线程调度 -> 结果路由与最终结算 -> 学习触发与恢复接口 -> 下一轮`

必须固定：

1. `状态 / 事件 / 结果回流` 先归并，再形成 `根需求` 并进入 `需求树`。
2. `需求树` 必须先完成父需求对子需求的权重调制，再生成 `需求队列`。
3. `方法禁止项集` 必须先根据 `需求队列` 的综合权重和双值趋势刷新，再进入任务入口实例化与线程调度。
4. `任务根节点` 与 `任务入口实例` 由自我线程创建；局部线程不得直接扩写。
5. `任务管理任务` 只推进步骤，不直接生成子任务实例。
6. `子结果` 必须先回线程，再由线程路由给父层任务。
7. `最终结算` 只能由自我线程完成，不得停留在局部结果里。


## 7. 轮次消息语义

必须固定：

1. 一轮主循环不是“只处理一条消息”。
2. 一轮主循环也不是“追到活队列彻底为空”为止。
3. 一轮主循环应先冻结本轮开始时已可见的消息批次。
4. 本轮必须处理完这批冻结消息，以及它们同步引发的内部消息闭包。
5. 本轮处理中异步新产生的消息，应进入下一轮。
6. 高优先级控制消息可按抢占规则中断本轮。
7. `控制消息` 与 `全局环境消息` 必须直接进入 `自我线程`。
8. `任务绑定且仅影响局部流程的外部反馈` 可先进入对应任务管理工作线程，再把升级后的控制面变化上报线程。
9. `结果回流消息` 必须先回线程；`推进消息` 只把会改变控制面判断的部分回线程。

## 8. 当前保留的接口位

以下能力当前只保留接口位：

- `最小授权接口`
- `最小安全门控接口`
- `最小执行监督接口`
- `最小学习沉淀接口`

这些接口位的存在目的只有两个：

1. 不让主循环在当前版本里重新膨胀成大而全总线。
2. 不让未来扩展时推翻当前主线。


## 9. 当前阶段最小验收口径

当前若要承认 `0015` 已与最新理念对齐，至少必须满足：

1. `自我线程` 被明确定义为面向 `需求信息` 的消息治理中枢。
2. `方法匹配 / 方案演练 / 策略沙盘` 不再作为自我线程并列一级模块。
3. 主线顺序已经统一为：
