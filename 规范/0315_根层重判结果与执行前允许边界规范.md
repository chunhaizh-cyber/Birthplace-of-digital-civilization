# 0315 根层重判结果与执行前允许边界规范

## 1. 目标

本文用于把 `根层重判结果` 的定位、生成时机和使用边界固定下来。

本文要回答的是：

- `根层重判结果` 到底是什么
- 它为什么必须保留
- 它为什么既服务执行前判断，也服务执行后回收
- 为什么它是禁止项的统一入口
- 为什么它是“任务当前是否允许执行”的直接依据

## 3. 核心结论

必须固定以下结论：

1. `根层重判结果` 必须保留，不能被任务管理任务、执行器、学习线程或局部方法结果替代。
2. `根层重判结果` 有两个直接用途：
   - `执行前用途`：判断当前任务/方法/动作是否允许进入执行
   - `执行后用途`：在结果回收后重新判断主链后续是否允许继续
3. `根层重判结果` 既不是原始真值，也不是叶子执行结果；它是根层消费比较结果后形成的上位约束结论。
4. `根层重判结果` 是禁止项的统一入口；凡命中根层禁止边界的，不得进入普通执行链。
5. `局部可执行` 不等于 `根层允许执行`；即使方法当前可用、条件当前满足，只要根层重判未放行，当前任务仍不得执行。
6. `根层重判结果` 可以消费 `预测结果`，但 `预测结果` 本身必须先落成带值特征，再通过比较生成二次特征方向，不能拿自然语言预期直接当裁判。
7. `根层重判结果` 也可以消费 `实际返回结果`，但实际返回结果同样必须先落成带值特征，再经比较形成方向输入。
8. `根层重判结果` 是 `最终裁决` 的上位输入，但它本身不等于整条主链的全部最终裁决。

一句话收口：

`根层重判结果是根层把“当前值、需求方向、预测或实际结果方向、禁止边界”统一重新比较后形成的上位约束结论；它既负责执行前放行/禁止，也负责执行后继续/收束的上位输入。`


## 4. 定义

`根层重判结果` 指的是：

- 自我线程主循环在根层重新消费当前关键值、需求方向、预测或实际结果方向、禁止边界之后，形成的上位约束结论

它回答的是：

- 这一轮当前是否允许继续推进
- 当前任务/方法/动作是否允许进入执行
- 是否已经命中禁止边界
- 是否只允许进入收束、待机或停止链

它不是：

- 一次特征本身
- 主键字段本身
- 局部条件判定结果本身
- 叶子执行结果本身
- 最终裁决的全部内容


## 5. 两种重判时机

### 5.1 预测型根层重判

`预测型根层重判` 指的是：

- 在任务正式执行之前
- 外层基于当前值、需求方向和预测结果，先做一次根层放行/禁止判断

它的主要用途是：

- 作为 `执行前允许边界`
- 作为 `当前任务是否允许执行` 的直接依据

必须固定：

1. 预测型根层重判通过后，任务才允许进入普通执行链。
2. 若预测型根层重判未通过，则当前任务只允许：
   - 放弃执行
   - 回到重筹办
   - 转入学习
   - 进入收束/待机/停止边界处理
3. 不允许先执行再补做这一步。

### 5.2 回收型根层重判

`回收型根层重判` 指的是：

- 在收到 `一步治理结果 / 执行返回结果 / 学习反馈结果 / 外部反馈` 之后
- 外层重新消费这些结果，再决定主链下一轮是否允许继续

它的主要用途是：

- 作为 `执行后继续/收束` 的上位输入
- 刷新下一轮 `任务管理上下文`

必须固定：

1. 回收型根层重判之后，才能形成新一轮稳定上下文。
2. 不允许承接层拿局部结果跳过这一步，直接续跑下一轮闭环。


## 6. 最小输入面

### 6.1 根层真值输入

根层重判至少应消费：

- `当前安全值`
- `当前服务值`
- `当前主需求`
- `需求值`
- `当前值`

### 6.2 比较后方向输入

根层重判至少应消费：

- `需求方向`
- `预测结果方向` 或 `实际结果方向`

必须固定：

1. 这里的方向只能来自比较后的二次特征。
2. 不允许直接拿一次特征、主键字段组合、方法名、动作名充当方向输入。

### 6.3 禁止边界输入

根层重判还应消费：

- `安全值将趋近 0` 的致命性边界信息
- `违规 / 违法 / 根层禁止项` 的命中信息
- 其它根层明确禁止的边界状态

必须固定：

1. `禁止项` 不是根层重判的替代品，而是根层重判的重要输入面。
2. 一旦命中根层禁止边界，当前普通执行路径必须被拦下。


## 7. 预测结果的使用规则

`预测结果` 若进入根层重判，必须先满足以下条件：

1. 预测结果必须表达为 `一组带值特征`。
2. 预测结果必须能与当前值做比较。
3. 比较后必须能生成 `预测结果方向`。
4. 再由 `预测结果方向` 与 `需求方向`、根层禁止边界一起进入根层重判。

必须固定：

1. `预测结果` 可以作为根层重判的重要输入。
2. 但 `预测结果` 本身不是最终判断依据。
3. 最终判断依据仍然必须回到比较后形成的 `二次特征方向 + 禁止边界输入`。
4. 更细的 `预测结果` 最小输入和合法表达方式，按 [0319_预测结果最小输入与合法表达规范.md](D:\鱼巢\规范\0319_预测结果最小输入与合法表达规范.md) 执行。
5. 更细的 `预测结果方向` 与 `需求方向` 单特征对齐规则，按 [0320_预测结果方向与需求方向对齐判定表规范.md](D:\鱼巢\规范\0320_预测结果方向与需求方向对齐判定表规范.md) 执行。

### 7.1 实际返回结果的使用规则

`实际返回结果` 若进入回收型根层重判，必须先满足以下条件：

1. 实际返回结果必须表达为 `一组带值特征`。
2. 实际返回结果必须能与当前值做比较。
3. 比较后必须能生成 `实际结果方向`。
4. 再由 `实际结果方向` 与 `需求方向`、根层禁止边界一起进入回收型根层重判。

必须固定：

1. `实际返回结果` 可以作为回收型根层重判的重要输入。
2. 但 `实际返回结果` 本身不是最终判断依据。
3. 最终判断依据仍然必须回到比较后形成的 `二次特征方向 + 禁止边界输入`。
4. 更细的 `实际结果方向` 与 `需求方向` 单特征对齐规则，按 [0321_实际结果方向与需求方向对齐判定表规范.md](D:\鱼巢\规范\0321_实际结果方向与需求方向对齐判定表规范.md) 执行。


## 8. 最小输出面

`根层重判结果` 最少应能表达以下内容：

- `是否允许继续推进`
- `是否允许当前执行`
- `是否命中根层禁止`
- `是否仅允许收束/待机/停止`
- `根层禁止原因类型`
- `本轮重判时机`（预测前 / 回收后）

必须固定：

1. `是否允许当前执行` 是任务执行前直接要消费的结论。
2. `是否允许继续推进` 是主链继续与否的上位输入。
3. `根层禁止原因类型` 用于解释为什么当前不能执行。
4. 更细的字段表与“为什么不能只保留单 `bool`”，按 [0316_根层重判结果最小字段表规范.md](D:\鱼巢\规范\0316_根层重判结果最小字段表规范.md) 执行。
5. 更细的执行前门控最小输入和判定顺序，按 [0317_执行前门控最小输入与判定顺序规范.md](D:\鱼巢\规范\0317_执行前门控最小输入与判定顺序规范.md) 执行。


## 9. 与最终裁决的关系

`根层重判结果` 与 `最终裁决` 的关系应固定为：

```text
当前值 / 需求值 / 预测或实际结果值
-> 比较形成方向
-> 命中或未命中根层禁止边界
-> 根层重判结果
-> 最终裁决
```

因此必须固定：

1. `根层重判结果` 在 `最终裁决` 之前。
2. `最终裁决` 会消费 `根层重判结果`，但还会结合候选去向、条件判定结果和其它承接层结果。
3. `根层重判结果` 虽然不是全部最终裁决，但它已经足以直接决定“当前任务能不能执行”。 


## 10. 严格禁止的混层

以下做法必须禁止：

1. `局部条件满足 -> 直接允许执行`
2. `方法当前可用 -> 直接允许执行`
3. `候选去向 = 继续下一步 -> 直接允许执行`
4. `预测结果看起来有利 -> 跳过比较直接执行`
5. `命中违规/违法/致命性边界 -> 仍继续普通执行链`
6. `局部返回成功 -> 跳过根层重判直接续跑下一轮`


## 11. 当前阶段工程结论

当前阶段可先按以下口径理解：

1. `根层重判结果` 既要服务执行前放行，也要服务结果回收后的重新判断。
2. `任务管理上下文` 中的 `最近根层重判结果`，主要镜像的就是这份上位输入。
3. `自我线程主循环` 必须在提交治理请求前，先产出可供执行前判断使用的根层重判结果。
4. 任何线程回流后，也必须再次回到根层重判，再决定后续主链。
