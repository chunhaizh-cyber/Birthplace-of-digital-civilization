# 基础信息与语素信息关系整合规范 20260509

> 2026-05-10 修订提示：本规范中的“词性”现在拆分为“人类词性”和“信息入口类型”。人类词性只提供自然语言候选；信息入口类型决定主信息类型、存储树和默认根。详见 [信息入口类型与信息存储树路由规范20260510.md](信息入口类型与信息存储树路由规范20260510.md)。本文后续保留的 `词性` 字样，在新实现中应优先理解为兼容字段或人类词性字段。

## 1. 总原则

语素层只作为语言入口索引，基础信息层才保存真正的语义对象。

固定结构为：

```text
词节点：只保存词面
  └─ 词性节点：保存 词性 + 对应基础信息类型 + 对应基础信息引用
        └─ 基础信息节点：保存真正的存在、特征、动态、二次特征等语义对象
```

也就是：

```text
词面负责可读入口；
词性负责语言分类和语义类型索引；
基础信息负责真实世界、抽象概念、状态特征和动作语义。
```

语素层不得吞掉基础信息层职责。基础信息层也不得复制词面字符串作为自己的权威语义来源。二者通过 `词性节点类*` 和 `可解析引用<基础信息节点类>` 建立稳定桥接。

## 2. 分层职责

### 2.1 词节点

词节点只保存词面，例如：

```text
飞机
安全
严重
移动
的
```

词节点不直接表达：

```text
存在
特征
动态
二次特征
语言功能项
```

同一个词面可以拥有多个词性节点。不同词性节点可以分别指向不同基础信息模板。

### 2.2 词性节点

词性节点是语言入口到基础信息模板的索引节点。

`词性主信息类` 必须至少表达：

```cpp
枚举_词性 词性 = 枚举_词性::未定义;
枚举_主信息类型 对应基础信息类型 = 枚举_主信息类型::未定义;
可解析引用<基础信息节点类> 对应基础信息{};
```

词性节点表达的是：

```text
词 + 词性 -> 一种基础信息类型 -> 一个基础信息模板 / 概念节点
```

词性节点不再维护“一个词性下挂多个语义项”的结构。若同一词面确实需要多个语义对象，应通过不同词性、不同上下文基础信息，或基础信息层自身的实例 / 派生关系表达。

### 2.3 基础信息节点

基础信息节点保存真实语义对象，包括但不限于：

```text
存在
指代
特征
抽象特征
动态
二次特征_修饰
语言信息_仅记录
状态
关系
因果
```

基础信息节点可以用 `主信息基类::名称`、`主信息基类::类型` 引用词性节点作为语言标签，但不得把词面字符串复制为权威语义源。

## 3. 默认映射

语素层必须提供词性到基础信息类型的默认推断入口：

```cpp
语素_推断词性对应基础信息类型(...);
语素_基础信息类型可由词性模板创建(...);
语素_词性是否兼容基础信息类型(...);
```

默认映射如下：

| 词性族 | 默认基础信息类型 |
| --- | --- |
| 名词、专名、抽象名词、地名、时间词、方位词、n、nr、ns、nt、nz、PER、LOC、ORG、TIME | `存在` |
| 代词、r | `指代` |
| 形容词、状态词、a、an | `抽象特征` |
| 数词、量词、m、q | `特征` |
| 副词、介词、ad、d、p | `二次特征_修饰` |
| 动词、vd、vn、v | `动态` |
| 连词、助词、标点、语气词、短语等语言功能项 | `语言信息_仅记录` |

`抽象特征` 与 `特征` 允许兼容绑定。原因是同一个形容词入口既可能表示特征模板，也可能在具体场景中绑定到特征实例。

## 4. 基础信息模板入口

`基础信息类` 必须提供以词性节点为入口的模板查找 / 创建能力：

```cpp
节点类* 查找词性模板(
    const 词性节点类* 词性节点,
    枚举_主信息类型 类型
) const;

节点类* 确保词性模板(
    const 词性节点类* 词性节点,
    枚举_主信息类型 类型,
    节点类* 父节点 = nullptr
);

std::vector<节点类*> 查找引用词性节点的基础信息(
    const 词性节点类* 词性节点,
    const 节点类* 范围根 = nullptr
) const;
```

典型创建关系：

```text
“飞机” + 名词 -> 存在节点主信息类
“安全” + 形容词 -> 抽象特征主信息类
“严重” + 副词 -> 二次特征主信息类
“移动” + 动词 -> 动态节点主信息类
“的” + 助词 -> 语言记录主信息类
```

`确保词性模板` 创建出的基础信息节点必须回写到词性节点的：

```text
对应基础信息类型
对应基础信息
```

若已有兼容模板，不得重复创建同一词性入口下同类型模板。

## 5. 语素桥接入口

`语素类` 必须提供从语言入口到基础信息模板的桥接函数：

```cpp
基础信息节点类* 确保词性对应基础信息(
    词性节点类* 词性节点,
    基础信息类* 基础信息仓库 = nullptr,
    基础信息节点类* 父节点 = nullptr
);

基础信息节点类* 确保词性对应基础信息(
    const std::string& 词值,
    枚举_词性 词性,
    基础信息类* 基础信息仓库 = nullptr,
    基础信息节点类* 父节点 = nullptr
);

基础信息节点类* 查找词性对应基础信息(...) const;
枚举_主信息类型 获取词性对应基础信息类型(...) const;
```

标准调用链为：

```text
语素集
    ↓
词节点 / 词性节点
    ↓
基础信息类::确保词性模板
    ↓
基础信息节点
```

示例：

```cpp
auto* 飞机存在 = 语素集.确保词性对应基础信息("飞机", 枚举_词性::名词);
auto* 安全特征 = 语素集.确保词性对应基础信息("安全", 枚举_词性::形容词);
auto* 严重修饰 = 语素集.确保词性对应基础信息("严重", 枚举_词性::副词);
auto* 移动动态 = 语素集.确保词性对应基础信息("移动", 枚举_词性::动词);
```

## 6. 语言标签引用

基础信息继续使用 `词性节点类*` 作为语言标签。

通用入口：

```cpp
主信息基类::名称;
主信息基类::类型;
```

特殊主信息还必须参与引用检查：

```cpp
指代节点主信息类::代词;
动态节点主信息类::来源动作名;
二次特征主信息类::概念名称;
语言记录主信息类::语言词性;
```

`查找引用词性节点的基础信息` 不只检查 `名称 / 类型`，也必须检查这些特殊字段。

## 7. 语言记录主信息

必须为语言功能项提供轻量基础信息类型：

```cpp
class 语言记录主信息类 : public 基础信息基类
```

其类型为：

```cpp
枚举_主信息类型::语言信息_仅记录
```

用途：

```text
连词
助词
标点
语气词
短语结构入口
其他只承担语言结构功能的项
```

这类节点只保留语言结构信息，不得冒充存在、动态或特征。

## 8. 场景绑定

`绑定词性词到场景基础信息(...)` 不得丢弃 `场景` 参数。

绑定基础信息后，应按基础信息类型写入场景索引：

```text
状态索引
动态索引
关系索引
二次特征索引
实例因果索引
```

`通过词和场景查找基础信息(...)` 的查找顺序固定为：

```text
1. 查找词面对应词节点。
2. 查找词性入口。
3. 优先返回当前场景索引中命中的基础信息。
4. 当前场景没有命中时，返回词性入口绑定的通用基础信息模板。
```

场景绑定表达的是“该语言入口在当前场景下绑定到哪个语义对象”，不是覆盖词性入口的通用模板。

## 9. 二次特征词

`添加二次特征词` 必须有实际实现，不得为空。

它应基于：

```text
参照物
比较对象
比较类型
```

生成组合词，例如：

```text
A_相对_B_距离
```

生成后应：

```text
1. 添加副词 / 修饰类词性入口。
2. 推断为 二次特征_修饰。
3. 确保对应基础信息模板存在。
```

## 10. 去重规则

同一词性节点下，不得重复创建相同基础信息类型的通用模板。

去重判断至少包括：

```text
词节点词面
词性
对应基础信息类型
词性节点主键
基础信息模板主键
```

场景绑定不得破坏通用模板去重。若同一词性入口在不同场景中指向不同具体实例，应通过场景索引和基础信息实例关系表达。

## 11. 禁止项

禁止以下做法：

```text
1. 在词节点直接挂存在、特征、动态等语义实体。
2. 在词性节点中维护多个并列语义项集合。
3. 在基础信息节点中复制词面字符串作为权威语义。
4. 把助词、标点、连词直接创建成存在或动态。
5. 场景绑定时忽略场景参数。
6. 词性默认映射失败时静默创建未定义基础信息。
7. 查找基础信息时只查名称 / 类型，漏查特殊主信息中的词性引用。
```

## 12. 验收标准

实现应满足：

```text
“飞机” + 名词 能创建 / 查找 存在 模板。
“安全” + 形容词 能创建 / 查找 抽象特征 模板。
“严重” + 副词 能创建 / 查找 二次特征_修饰 模板。
“移动” + 动词 能创建 / 查找 动态 模板。
“的” + 助词 能创建 / 查找 语言信息_仅记录 模板。
同一词性入口重复确保时返回同一基础信息模板。
场景绑定后，场景查找优先返回场景内绑定对象。
场景未命中时，回退到词性入口的通用基础信息。
二次特征词能生成组合词并绑定二次特征模板。
基础信息引用检查能覆盖名称、类型、代词、来源动作名、概念名称、语言词性。
```

## 13. 最终口径

本规范定型为：

```text
语素是入口，不是知识本体；
词性是索引，不是语义容器；
基础信息才是语义对象；
场景绑定表达上下文实例；
语言功能项只记录语言结构，不冒充实体。
```
