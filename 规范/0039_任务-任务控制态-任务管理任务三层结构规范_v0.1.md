## 3. 核心结论

必须固定以下结论：

1. 不应只粗分 `任务` 和 `任务管理任务`，而应明确区分三层：
   - `任务的根语义`
   - `任务控制态`
   - `任务管理任务的局部推进态`
2. `任务` 是被完成的对象，回答“要完成什么、为什么存在、朝哪走、何为完成”。
3. `任务控制态` 是附着在任务上的全局控制面状态簇，回答“全局上还能不能推进、怎么排队、是否允许派发、预算和配额如何、是否已进入收束/停止边界”。
4. `任务管理任务` 是围绕任务运行的一步治理执行体，回答“当前怎么推进、卡在哪、下一步准备怎么动、局部结果到了哪”。
5. `自我线程` 位于这三层之上，统一持有并裁决 `任务控制态`，并根据消息、二次特征和影响项决定是否继续推进、让位、实例化依赖任务、路由结果和结算双值。
6. 不是所有“运行中的状态”都属于 `任务管理任务`；`可运行 / 等待 / 回流 / 收束中 / 预算 / 配额 / 有效优先级 / 是否允许继续派发` 这类状态，应归 `任务控制态`，而不是归局部推进态。
7. `当前步骤前沿 / 当前等待原因 / 条件满足状态 / 依赖等待状态 / 派生需求挂起状态 / 局部结果缓存 / 局部重试计数` 这类状态，应归 `任务管理任务` 的局部推进态。
8. `来源需求 / 方向 / 目标结果 / 完成标准 / 父子关系 / 结果状态` 这类状态，应优先归到 `任务` 的根语义层。
9. `任务` 的特征变化多半更接近 `结果层影响`；`任务管理任务` 的特征变化多半更接近 `过程层影响`；`任务控制态` 的特征变化多半更接近 `调度与控制层影响`。
10. `任务管理任务` 不得直接拍板 `任务控制态` 真值；它只能上报局部推进事实，由 `自我线程` 决定是否改写全局控制面。

## 6. 表 3：消息与写权最小归位表

| 触发事件 | 先写哪一层 | 是否必须上浮自我线程 | 线程后续可能改哪一层 | 说明 |
| --- | --- | --- | --- | --- |
| `步骤阻塞 / 解阻` | `任务管理任务` | 是 | `任务控制态` | 局部阻塞先落局部推进态，再由线程决定是否移出或放回可运行队列 |
| `子任务结果回流` | `任务` + `任务管理任务` | 是 | `任务控制态` | 子结果先改结果事实和等待事实，再由线程更新队列、优先级和派发许可 |
| `派生需求提出` | `任务管理任务` | 是 | `任务控制态` + `任务` 关系面 | 任务管理任务只能提出，是否实例化依赖任务由线程决定 |
| `预算刷新 / 优先级刷新 / 收束控制` | `任务控制态` | 是 | `任务控制态` | 属于全局控制链，不属于局部推进态 |
| `目标结果或完成标准正式变更` | `任务` | 是 | `任务` | 属于语义修订，不得由局部推进自行拍板 |

必须固定：

1. 局部推进消息可以先写 `任务管理任务`，但只要会改变 `队列 / 派发 / 优先级 / 预算 / 是否实例化依赖任务 / 最终去向`，就必须上浮到 `自我线程`。
2. `任务控制态` 的正式真值，只能由 `自我线程` 最终确认。
3. `任务` 的根语义变更必须走正式语义修订链，不能被局部推进事件顺手改写。


## 7. 服务 / 安全影响上的最小口径

必须固定：

1. `任务` 的变化更接近 `结果层影响`，例如 `结果状态`、`完成标准满足与否`，更可能直接进入 `0038` 的 `直接服务 / 准备性服务` 影响模板。
2. `任务控制态` 的变化更接近 `调度与控制层影响`，例如 `优先级变化`、`预算收缩`、`是否允许继续派发`，更可能通过 `准备性服务 / 风险安全` 影响模板进入根层判断。
3. `任务管理任务` 的变化更接近 `过程层影响`，例如 `等待原因`、`条件满足状态`、`派生需求挂起`，通常要先经过 `依赖满足 / 当前步骤可执行` 等二次特征，才能进一步生成正式影响项。


## 8. 样板链

### 8.1 等待确认任务

```text
任务 T：完成步骤 S2 所需确认
-> 任务.来源需求 = 完成某项服务
-> 任务.方向 = 拿到确认
-> 任务.完成标准 = 确认返回
-> 任务.父任务 = P

任务管理任务 M(T)
-> 当前步骤前沿 = 等待确认
-> 当前等待原因 = 外部确认未到
-> 派生需求挂起状态 = 请求确认已挂起
-> 局部结果缓存 = 尚未完成

自我线程持有的任务控制态
-> 队列状态 = 等待
-> 当前有效优先级 = 解阻相关优先级
-> 是否允许继续派发 = false
-> 预算 / 配额 = 当前全局控制面结果
```

确认回来之后：

```text
任务管理任务
-> 当前等待原因 = 无
-> 条件满足状态 = 已满足

自我线程
-> 重算依赖满足 / 当前步骤可执行
-> 任务控制态.队列状态 = 可运行
-> 任务控制态.是否允许继续派发 = true
```

必须固定：

1. 这里改变的是 `任务管理任务` 的局部推进事实和 `任务控制态` 的全局调度结论。
2. `任务` 的根语义并没有改变，只是离完成更近了。
