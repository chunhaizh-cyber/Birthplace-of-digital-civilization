# 2030 存在主信息自然语言映射规范

## 1. 目标

本文用于固定当前工程中：

- `存在节点主信息类` 的各个字段，在自然语言层分别承担什么职责
- `存在信息 / 子存在信息 / 特征信息` 应如何结合成可读表达
- `有什么 / 是什么 / 能什么 / 会什么 / 需要什么` 这五类基本问句，应分别由哪些结构字段承接

本文重点回答：

- 为什么 `名称` 和 `类型` 不能混用
- 哪些内容可以仅靠 `存在主信息` 直接表达
- 哪些内容必须借助 `任务 / 方法 / 需求 / 状态 / 动态 / 因果` 才能合法表达
- 子存在信息应来自哪里，而不应伪装成 `存在主信息` 的内嵌字段

本文服从以下硬边界：

## 3. 依据的字段面

当前 `存在节点主信息类` 的关键字段包括：

- 继承字段：`名称`、`类型`、`统计`
- `概念模板`
- `内部世界`
- `需求根节点`
- `任务根节点`
- `方法根节点`
- `有位置历史`
- `上次观测位置_mm`
- `最近观测位置_mm`
- `最近位移_mm`
- `连续静止帧`
- `连续未命中帧`
- `实例因果`

必须固定：

1. 本文只解释这些字段在自然语言里的职责，不新增平行字段。
2. `子存在信息` 不直接存放在 `存在节点主信息类` 中，而来自世界树中的父子结构与作用域路径。
3. 若某条自然语言需要“子存在”，必须回到世界树树边，而不是假装它是 `存在主信息` 的一个普通属性。


## 4. 总原则

必须固定：

1. `存在信息` 决定“谁”。
2. `子存在信息` 决定“谁里面的谁”。
3. `特征信息 / 状态信息` 决定“这个谁现在怎样”。
4. `动态信息` 决定“这个谁正在怎样变化”。
5. `因果信息` 决定“为什么会这样”或“通常会怎样”。

一句话固定：

`存在主信息负责对象锚点，树结构负责层级路径，特征/状态负责当前样貌，动态/因果负责变化与后果。`


## 5. 名称、类型与概念模板的自然语言职责

### 5.1 名称

`名称` 的职责固定为：

- 个体名
- 专名
- 当前最优自然语言指代

它回答的是：

- `它具体叫什么`

示例：

- `自我`
- `现实世界`
- `安全根任务`

### 5.2 类型

`类型` 的职责固定为：

- 类名
- 角色名
- 槽位名
- 缺少专名时的回退名

它回答的是：

- `它属于什么类`
- `它在表达中扮演什么角色`

示例：

- `自我类型`
- `方法虚拟存在`
- `存在`

### 5.3 概念模板

`概念模板` 的职责固定为：

- 上位概念
- 分类归属
- “这是什么”的补充解释来源

它回答的是：

- `它本质上是一类什么对象`

推荐句型：

- `X 属于 Y`
- `X 是一种 Y`
- `X 的概念模板是 Y`

### 5.4 三者组合规则

必须固定：

1. `名称` 优先负责识别具体对象。
2. `类型` 优先负责补足类别和角色。
3. `概念模板` 优先负责“是什么”的上位说明。
4. `获取名称` 一类接口不得只取 `名称`，也不得只取 `类型`，而应按场景组合。

推荐组合口径：

- 有 `名称` 且有 `类型`：
  - `自我（自我类型）`
- 只有 `名称`：
  - `自我`
- 只有 `类型`：
  - `方法虚拟存在`
- 有 `概念模板`：
  - `本能方法_读取核心状态属于方法虚拟存在`


## 6. 子存在信息的来源与表达

### 6.1 子存在不来自主信息字段

必须固定：

1. `子存在信息` 不直接来自 `存在节点主信息类` 的内嵌字段。
2. `子存在信息` 来自世界树中的 `父节点 -> 子节点` 关系。
3. `内部世界` 是特殊的子作用域入口，但不是“全部子存在信息”的代替物。

### 6.2 子存在表达的最小来源

子存在表达最少应结合：

- 父存在节点
- 子存在节点
- 必要时的中间作用域节点，如 `内部世界`

推荐句型：

- `X 中有 Y`
- `X 的内部世界中有 Y`
- `X 下有一个 Y`

示例：

- `自我的内部世界中有本能方法_读取核心状态`
- `安全根任务下有治理步骤`

### 6.3 子存在的语义职责

子存在信息主要承担：

1. 包含关系
2. 领属关系
3. 路径定位

因此必须固定：

- 子存在自然语言不能只显示裸名字，应尽量保留其上级作用域
- 若脱离上级作用域会歧义，应优先说成 `X 中的 Y`


## 7. 特征信息、状态信息、动态信息与存在的结合

### 7.1 特征信息

特征信息主要回答：

- `它具有什么`
- `它的某个量当前是多少`

推荐句型：

- `X 的 A 为 B`
- `X 具有 A`

示例：

- `自我的服务值为 80`
- `自我具有服务时序衰减步长`

### 7.2 状态信息

状态信息主要回答：

- `它现在是什么样子`

更细的“状态信息可视作形容词性/表语表达，但不得丢失主体、特征、状态值与时间槽位”的口径，按 [2050_状态信息自然语言表达规范.md](D:\鱼巢\规范\2050_状态信息自然语言表达规范.md) 执行。

推荐句型：

- `X 的 A 为 B`
- `X 当前处于 B`

示例：

- `当前主任务为安全根任务`
- `安全根任务处于运行中`

必须固定：

- 问“现在什么样”时，优先使用 `状态信息`
- 不得拿 `动态信息` 冒充当前静态状态

### 7.3 动态信息

动态信息主要回答：

- `它刚刚或正在怎样变化`

更细的“动态信息可同时落成动词与动作短语，动词负责事件核心，动作短语负责完整动作表述”的口径，按 [2060_动态信息自然语言表达规范.md](D:\鱼巢\规范\2060_动态信息自然语言表达规范.md) 执行。

推荐句型：

- `X 执行了 A`
- `X 正在从 A 变到 B`

示例：

- `自我执行了停止当前试探并进入保守态`
- `任务管理任务推进了当前步骤`

必须固定：

- 问“正在怎么变”时，优先使用 `动态信息`
- 不得拿 `状态信息` 冒充动作过程


## 8. 五类基本问句的正式映射

### 8.1 有什么

`有什么` 用于表达存在、包含、拥有。

它的合法来源优先为：

- 子存在树边
- `内部世界`
- 直接挂在该存在下的特征、状态、动态索引
- 三棵根：`需求根节点 / 任务根节点 / 方法根节点`

更细的“任务信息主要承担要做什么、现在做到哪、下一步是什么、服务于哪个需求并通过哪个方法执行”这条口径，按 [2090_任务信息自然语言表达规范.md](D:\鱼巢\规范\2090_任务信息自然语言表达规范.md) 执行。

推荐句型：

- `X 有 Y`
- `X 中有 Y`
- `X 的内部世界中有 Y`
- `X 具有 A`

必须固定：

- `有什么` 优先由树结构和直接挂载关系承接
- 不得把“会发生什么”误说成“有什么”

### 8.2 是什么

`是什么` 用于表达身份、类型、概念归属。

它的合法来源优先为：

- `名称`
- `类型`
- `概念模板`

推荐句型：

- `X 是 Y`
- `X 属于 Y`
- `X 是一种 Y`

必须固定：

- `是什么` 不应优先落到任务、方法、需求流程
- `是什么` 的根依据仍是 `名称 / 类型 / 概念模板`

### 8.3 能什么

`能什么` 用于表达能力、可用动作、可调用方法。

更细的“方法信息主要承担能做什么、在什么条件下能做、会产出什么结果、来源于哪项任务并如何挂接执行”这条口径，按 [2100_方法信息自然语言表达规范.md](D:\鱼巢\规范\2100_方法信息自然语言表达规范.md) 执行。

它的合法来源优先为：

- `方法根节点`
- 方法树中的方法首节点、条件节点、结果节点
- 外层允许边界、门控结果、权限状态

推荐句型：

- `X 能 A`
- `在 C 条件下，X 能 A`
- `X 当前可以 / 暂不能 A`

必须固定：

1. `方法根节点` 只提供能力候选来源。
2. `能什么` 不等于 `现在允许做什么`。
3. 真正的“是否允许继续做”仍必须回到 `根层重判 + 执行前门控 + 允许边界`。

### 8.4 会什么

`会什么` 用于表达趋势、后果、预测、规则性结果。

它的合法来源优先为：

- 动态信息
- 实例因果
- 抽象因果
- 明确写入规范的内部先天动作时间规则

推荐句型：

- `X 会 A`
- `在 C 下，X 会 A`
- `通常 X 会导致 B`

必须固定：

1. 仅凭 `存在主信息` 本身，不得直接生成“会什么”。
2. 若没有动态、因果或规则证据，`会什么` 必须缺席，不得编造。
3. 抽象因果必须使用 `通常 / 往往 / 会 / 可能` 这类归纳语气。

### 8.5 需要什么

`需要什么` 用于表达需求、缺口、待补条件、目标方向。

更细的“需求信息主要承担需要什么、为了什么需要、通过什么任务满足”这条口径，按 [2080_需求信息自然语言表达规范.md](D:\鱼巢\规范\2080_需求信息自然语言表达规范.md) 执行。

它的合法来源优先为：

- `需求根节点`
- 需求树
- 缺口类型、下一步去向、任务管理上下文

推荐句型：

- `X 需要 Y`
- `为了 G，X 需要 Y`
- `X 当前还缺少 Y`

必须固定：

1. `需要什么` 主要由需求树承接，不由 `名称 / 类型` 直接给出。
2. 若没有需求节点或治理上下文，不得仅凭猜测生成“需要什么”。


## 9. 位置历史字段的表达规则

### 9.1 可说内容

当 `有位置历史 = true` 时，允许表达：

- 最近位置
- 上次位置
- 最近位移
- 连续静止
- 连续未命中

推荐句型：

- `X 最近位于 P`
- `X 从 P1 移动到 P2`
- `X 最近位移为 D 毫米`
- `X 已连续静止 N 帧`
- `X 已连续未命中 N 帧`

### 9.2 禁止项

必须固定：

1. 若 `有位置历史 = false`，不得伪造位移叙述。
2. 若只有数值而没有坐标解释，不得硬说“向左”“向前”等方向词。
3. `连续未命中帧` 只能表达观测缺失，不得直接表达死亡、消失或不存在。


## 10. 实例因果字段的表达规则

`实例因果` 的职责固定为：

- 作为存在层“会什么 / 为什么 / 最近发生过什么”的证据入口

推荐句型：

- `关于 X，刚刚发生了……`
- `因为……，所以 X ……`
- `通常关于 X，会……`

必须固定：

1. `实例因果` 只提供证据入口，不直接代替当前状态。
2. 若要表达“为什么”，必须回到 `因果信息` 的条件、动作、结果结构。
3. 更细句型仍按 [2020_因果信息自然语言表达规范.md](D:\鱼巢\规范\2020_因果信息自然语言表达规范.md) 执行。


## 11. 最小表达顺序

围绕一个存在生成自然语言时，最小顺序固定为：

```text
1. 先确定主存在锚点
2. 再确定是否需要补子存在路径
3. 再确定当前是在回答“有什么 / 是什么 / 能什么 / 会什么 / 需要什么”中的哪一类
4. 再从对应字段或树结构抽取槽位
5. 最后线性化成句子
```

必须固定：

1. 不得先想好句子，再倒推字段。
2. 不得在 `会什么` 与 `需要什么` 这两类上跳过证据来源。
3. 同一条句子只能服务一个主问句，不得在一句话里混写多个主裁决。


## 12. 当前阶段工程结论

当前阶段先固定以下最小口径即可：

1. `名称` 是个体名，`类型` 是类名/角色名，`概念模板` 是上位说明。
2. `子存在信息` 来自世界树树边和 `内部世界`，不来自 `存在主信息` 的平铺字段。
3. `状态信息` 负责“现在什么样”，`动态信息` 负责“正在怎么变”。
4. `能什么` 必须回到方法树和门控边界。
5. `会什么` 必须回到动态、因果或明确规则证据。
6. `需要什么` 必须回到需求树和治理上下文。

一句话收口：

`存在主信息负责把对象说清楚，但只有把它和树结构、特征/状态、动态、因果、任务/方法/需求根树接起来，才能把“有什么 / 是什么 / 能什么 / 会什么 / 需要什么”完整说成人话。`
