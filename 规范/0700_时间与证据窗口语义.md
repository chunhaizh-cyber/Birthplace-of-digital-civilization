# 0700 时间与证据窗口语义

## 1. 目标

本文用于固定以下内容：

- 什么叫“一次观察”
- 什么叫“同类证据”
- `发生时间` 与 `收到时间` 各自表达什么
- 短窗、中窗、长窗如何使用
- 候选概念与稳定概念的证据如何累计、失效、冲突与修正

本文的作用是让：

- 概念生成不漂
- 因果判断有统一时间语义
- 运行态验收能看懂每次证据是如何算出来的


## 2. 时间语义的基本单位

当前应固定四个基本单位：

1. `状态样本`  
   某主体在某时刻的一个状态记录。

2. `观察单元`  
   由一个或多个状态样本比较后得到的一次结构化观察结果。

3. `证据`  
   能支持某个候选概念或因果关系的一次有效观察单元。

4. `窗口`  
   用于判断证据是否属于同一类、是否可比较、是否还能累计的时间范围。

可以收敛为：

- 样本是原料
- 观察单元是加工结果
- 证据是可入账的观察单元
- 窗口是入账时的时间边界


## 3. 发生时间与收到时间

### 3.1 发生时间

`发生时间` 表示：

- 该状态、事件、动态在系统所描述世界中实际发生的时间

它用于：

- 因果先后判断
- 动态路径排序
- 证据是否真的满足“先因后果”

### 3.2 收到时间

`收到时间` 表示：

- 系统在内部接收到、记录到该信息的时间

它用于：

- 内部处理顺序
- 消息延迟分析
- 线程间消息到达先后统计

### 3.3 优先级规则

在判断因果关系时，应固定：

- `先看发生时间`
- `收到时间只用于补充解释，不用于推翻发生时间`

也就是说：

- 即使某条信息晚收到，只要它发生得更早，也不能仅因为“收到晚”就被判成后果


## 4. 一次观察的定义

为了避免把同一事实重复记账，当前应固定：

- `一次观察 = 在一个评价周期内，对同一主键字段组合关系得到的一次结构化比较结果`

这里的关键是：

- 同一评价周期
- 同一关系主键
- 同一方向结论

### 4.1 不应重复计数的情况

以下情况默认不应算成多次观察：

1. 同一周期内重复读取同一状态
2. 同一条消息被多个模块转手记录
3. 同一个比较结果被不同日志口重复打印
4. 同一个事实在没有新状态变化的情况下被重复推导

### 4.2 可以新增一次观察的情况

以下情况可新增一次观察：

1. 目标状态发生了新的有效变化
2. 同一关系在新的评价周期再次成立
3. 发生了新的因果链样本
4. 适用条件发生变化后，该关系仍再次成立


## 5. 同类证据的定义

当前应固定：

- `同类证据 = 具有相同关系主键、相同作用方向、相同根需求投影，并且处于可比较窗口内的证据`

必须额外固定：

- 这里的 `关系主键` 与 `适用条件主键` 都只承担 `同类证据归桶键 / 候选聚类键` 职责。
- 它们只回答“这些证据能不能先放到一起比较”，不回答“这些证据已经证明同一结论成立”。
- 是否真正同类、是否可以进入稳定累计，仍必须继续结合 `方向 / 根需求投影 / 时间窗 / 适用条件` 等可比较结果共同判断。

因此要判定两条证据是否同类，至少要比较：

1. `关系主键`
2. `作用对象`
3. `被作用对象`
4. `方向`
5. `作用维度`
6. `根需求投影结果`
7. `适用条件主键`
8. `时间窗`

缺少其中关键字段时，不应直接合并统计。

但必须继续固定：

- `关系主键相同` 只表示“关系候选面相近”。
- 不得把 `关系主键相同` 直接等价成“语义完全相同”或“可直接合并成已成立结论”。

### 5.1 抽象因果模板的同类证据判定

当证据用于生成 `抽象因果模板` 时，必须额外固定：

1. 来源样本必须先是 `合法实例因果`
2. 允许这些样本来自：
- 不同时间
- 不同具体场景
- 不同具体存在
3. 但跨具体场景/存在前，不得直接忽略锚点差异，而必须先抽象成可比较的条件模式

因此模板层的“同类证据归桶键”建议扩成：

1. `锚点类型`
2. `状态层级`
3. `状态面类型`
4. `条件模式主键`
5. `动作模式主键`
6. `结果模式主键`
7. `根需求投影结果`
8. `适用条件主键`
9. `可比较窗口 / 证据窗口类型`

必须继续固定：

- 这里的组合主键只承担 `抽象因果模板归桶键 / 候选聚类键` 职责
- 它只回答“这些实例因果能不能先进入同一个模板候选桶”
- 不回答“这些实例因果已经证明同一个抽象因果模板成立”

### 5.2 抽象因果模板的分桶与拆桶规则

对抽象因果模板，必须额外固定：

1. `场景模板` 与 `存在模板` 分通道维护，不得混桶
2. 若具体场景差异导致 `结果模式` 明显分裂，则必须拆成不同模板
3. 若只有 `条件模式` 相近，但 `动作模式` 或 `结果模式` 持续不一致，则不得强行合并
4. `具体锚点主键` 只用于来源追踪，不进入抽象模板身份字段

一句话收口：

- `抽象因果模板` 允许跨时间、跨具体场景/存在归并
- 但只能在 `抽象条件模式 + 抽象动作模式 + 抽象结果模式 + 可比较窗口` 同类时归并


## 6. 时间窗定义

当前推荐沿用现有时间窗三分法：

- `短窗`
- `中窗`
- `长窗`

### 6.1 当前基线

当前基线可固定为：

1. `短窗`  
   绝对时间差 `<= 1 秒`

2. `中窗`  
   绝对时间差 `> 1 秒 且 <= 60 秒`

3. `长窗`  
   绝对时间差 `> 60 秒`

这个基线应与当前 `二次特征` 里的时间窗推断保持一致。

### 6.2 时间窗的使用语义

- `短窗` 更适合判断直接响应、局部立即影响
- `中窗` 更适合判断短时恢复、短时衰减、局部策略效果
- `长窗` 更适合判断长期趋势、稳定偏好、慢变量影响

### 6.3 一条重要约束

- 不同时间窗下得到的证据，默认不能直接混成一个稳定结论，除非已显式做跨窗归一化


## 7. 候选概念的证据累计规则

### 7.1 入账条件

一条观察要成为候选概念证据，至少应满足：

1. 关系主键明确
2. 时间信息完整
3. 方向明确
4. 该观察不是同周期重复记录

### 7.2 累计规则

候选概念的证据累计应固定为：

- `同类证据 +1`
- `反向证据` 记入反例计数或一致性扣分
- `无法判断方向的证据` 只作为背景样本，不直接累计正反票

### 7.3 不确定样本处理

若当前样本存在：

- 时间顺序不清楚
- 干扰过强
- 方向不稳定
- 条件未分离

则应：

- 记录样本
- 不直接提高稳定度
- 只作为候选补充证据


## 8. 稳定概念的证据累计规则

### 8.1 升级前提

候选概念升级为稳定概念时，时间语义上至少要满足：

1. 不只来自单一瞬时样本
2. 不是同一轮重复计数堆出来的次数
3. 存在跨周期的重复成立
4. 至少在一个明确时间窗中稳定成立

### 8.2 稳定概念的维护

稳定概念建立后，后续证据应分为：

1. `支持证据`  
   增强稳定度或置信度

2. `反例证据`  
   降低一致性、触发降级评估

3. `条件外证据`  
   不直接否定，只提示适用边界可能变窄

### 8.3 不允许的错误累计

以下情况不应被当成“稳定成立”的理由：

1. 同一瞬时循环里刷出来很多重复记录
2. 单一短窗内高频重复但没有跨周期复现
3. 完全不同适用条件下的样本被强行合并
4. 只因为名称相同就把不同关系合并


## 9. 因果判断的时间语义

### 9.1 先因后果

当前应固定：

- `因信息的发生时间 <= 果信息的发生时间`

若不满足，则默认不能判为正向因果样本。

### 9.2 时间差过大

若因果间隔过大，应按以下处理：

- 落入当前允许窗口：可继续作为候选或稳定证据
- 超出窗口但仍可能有关：降级为弱相关线索
- 明显超出合理范围：不记为有效因果样本

### 9.3 时间顺序不清楚

如果只有收到时间，没有可信发生时间，则应：

- 降级使用
- 不直接确认为强因果


## 10. 反例、老化与失效

### 10.1 反例

反例指的是：

- 在同类主键、相近条件、可比较时间窗下，出现了与原方向相反的证据

### 10.2 老化

老化指的是：

- 某概念长期没有新的支持证据，其有效性应逐渐下降

### 10.3 失效

失效指的是：

- 在持续反例或长期无新证据后，该概念不再适合继续强参与决策

### 10.4 推荐处理顺序

- 先降稳定度
- 再降决策级别
- 必要时撤销强参与资格
- 最后再考虑撤销稳定概念身份


## 11. 运行态日志建议

为了让时间与证据窗口可验收，建议日志至少输出：

1. `关系主键`
2. `发生时间`
3. `收到时间`
4. `时间窗`
5. `是否新观察`
6. `是否重复计数`
7. `当前证据次数`
8. `当前反例次数`
9. `当前一致性Q`
10. `当前稳定度Q`

这样在验收时才能回答：

- 这次为什么算新证据
- 这次为什么只是重复读取
- 这次为什么进入候选但没升稳定概念
- 这次为什么导致降级而不是升级


## 12. 本文件的约束结论

后续实现中应固定以下结论：

1. 因果判断优先使用 `发生时间`，`收到时间` 只作补充解释。
2. 同一评价周期内，同一关系主键的重复结果不得重复计数。
3. 同类证据必须同时满足主键字段组合、方向、根需求投影、适用条件和时间窗可比较。
4. 不同时间窗下的证据默认不能直接混成一个稳定结论。
5. 候选与稳定概念的证据累计都必须区分支持证据、反例证据和条件外证据。
6. 没有跨周期重复成立的关系，不得直接升级为稳定概念。
7. 时间语义不清楚的样本可以记录，但不应直接用于强因果和强决策。
8. `关系主键 / 适用条件主键` 只负责证据归桶与候选聚类，不得越权替代 `二次特征` 或稳定性判定本身。
