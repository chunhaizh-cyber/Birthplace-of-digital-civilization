# 需求任务单目标与方法多结果归类规范 20260508

## 1. 总原则

当前模型固定为：

```text
需求：一个目标特征类型
任务：一个目标特征类型
方法：可以产生多个结果特征类型变化
```

也就是：

```text
需求 / 任务是单目标。
方法执行结果是多结果。
多目标通过子需求 / 子任务展开。
额外结果通过结果分类处理。
```

需求树和任务树不能在一个节点里藏目标包。子链就是复合目标的显式展开结构。

## 2. 需求单目标

一个执行需求节点只表达一个原子目标：

```text
目标主体
目标特征类型
当前状态
目标状态
满足关系
```

`需求主信息类` 的权威目标字段是：

```text
被需求存在
被需求当前状态
需求状态
满足关系
目标特征类型缓存
```

其中 `目标特征类型缓存` 只用于查重、索引和显示，不是权威语义源。权威语义仍然来自状态节点和特征比较服务。

多目标需求必须展开成管理需求的直接子链：

```text
目标方法头补齐管理需求
├── 方法动作骨架状态需求
├── 方法可执行入口状态需求
├── 方法条件节点存在状态需求
├── 方法结果节点存在状态需求
├── 方法条件结果配对状态需求
├── 参数配对状态 / 任务参数绑定状态需求
├── 方法运行账完整性需求
└── 方法稳定状态需求
```

每个子需求仍然只允许一个目标特征类型。

实际场景缺少目标方法所需参数特征时，不把父需求复制成参数绑定子需求，也不硬性派发学习任务；它说明当前任务输入参数未闭合，应转成“参数配对状态 / 任务参数绑定状态”目标，再由后续筹办按结果能力选择能承接该目标的方法。

方法查找未命中不是这里的子需求展开入口。未命中时由筹办阶段先创建或复用待学习目标方法头，并把父需求 / 父任务目标写入该方法头的结果能力。只有目标方法头已经存在但不可执行时，才展开上述方法结构、参数或运行账子需求。

## 3. 任务单目标

一个任务只承接一个需求，因此也只允许一个目标特征类型。

任务目标来自对应需求：

```text
任务目标主体 = 对应需求.被需求存在
任务目标特征类型 = 对应需求.目标特征类型
任务当前状态 = 对应需求.被需求当前状态
任务目标状态 = 对应需求.需求状态
任务满足关系 = 对应需求.满足关系
```

任务可以缓存目标信息，但缓存不是第二套目标定义。任务主信息不能持久化多目标包，也不能让任务目标脱离对应需求自行扩展成多个目标。

## 4. 管理需求与执行需求

管理需求代表复合目标、组织目标或汇总目标。它负责：

```text
组织子需求
等待子需求
汇总子需求满足状态
分配或回收权重
```

执行需求代表一个可被任务直接推进的原子目标。只有执行需求进入单次方法查找和执行。

因此：

```text
管理需求可以代表复合目标。
执行需求必须是单目标。
任务承接执行需求时也必须是单目标。
```

## 5. 方法多结果

方法和需求 / 任务不同。方法执行可以产生多个实际结果特征变化。

例如：

```text
任务目标：
    方法运行账完整性 上升

方法实际结果：
    方法运行账完整性 上升
    方法稳定状态 上升
    方法新用途发现状态 出现候选
    服务值 下降
```

学习 / 练习方法可以声明并产出多种方法状态，因此可能匹配到运行账完整性、稳定状态、方法可用性等目标；但它仍然是普通方法匹配结果，不是缺口的硬性去向。

这些结果都要沉淀为事实，但不能都当成当前任务目标满足。

## 6. 执行结果四类归类

方法执行后的实际结果按当前任务目标和自我约束分为四类：

```text
1. 当前任务目标结果
2. 自我需要附带结果
3. 自我不需要附带结果
4. 自我禁止结果
```

### 6.1 当前任务目标结果

当前任务目标结果必须满足：

```text
结果特征类型 == 当前任务目标特征类型
结果关系 / 目标状态符合当前需求满足关系
实际结果状态可由特征比较服务和满足关系判定符合 / 不符合
```

它用于：

```text
更新任务满足程度
交自我线程更新需求满足程度
更新当前任务价值结算
更新该条件下针对该目标结果的符合 / 不符合统计
```

只有当前任务目标结果可以吃当前任务权重。

动作执行成功本身不是当前任务目标结果。若方法动作成功但没有产生可比较目标结果，只能归入“动作执行成功无可比较结果”，不得更新任务满足度、需求满足度或目标结果符合 / 不符合统计。

### 6.2 自我需要附带结果

自我需要附带结果是不匹配当前任务目标，但匹配另一个有效需求的结果。

它可以触发对应需求的满足检查，但不能冒领当前任务权重。它只能进入命中需求自己的账本。

### 6.3 自我不需要附带结果

自我不需要附带结果既不匹配当前任务目标，也不匹配当前有效需求，也不触犯禁止项。

处理方式：

```text
记录事实
沉淀条件结果对
更新出现次数
不更新当前任务满足度
不更新安全值 / 服务值结算
不更新目标结果符合 / 不符合次数
```

### 6.4 自我禁止结果

自我禁止结果与安全约束、禁止项或风险约束冲突。

这类结果必须进入治理风险账。目标结果账和治理风险账分开：

```text
任务目标达成 = 是 / 否
治理结果可接受 = 是 / 否
```

一个方法可以在目标特征上有效，但因禁止副作用而不可选或需要降权。

## 7. 条件结果对归并

每次方法动作函数执行成功后，产生条件结果对样本：

```text
条件状态集 -> 实际结果状态集
```

`实际结果状态集` 可以包含多个特征变化。归并时，同一条件下每个实际结果特征都添加或融合成结果节点：

```text
条件C
    -> 结果A
    -> 结果B
    -> 结果C
```

实际结果负责沉淀事实、样本次数和最近样本时间。

目标结果符合 / 不符合只针对本次任务选择方法时关心的单个目标特征类型。其他实际结果只做归类和事实沉淀。

## 8. 任务完成与结算

任务完成判断只看自己的目标特征：

```text
实际结果集中存在当前任务目标结果
    -> 任务目标达成

实际结果集中不存在当前任务目标结果
    -> 任务目标未达成
```

如果目标达成但出现禁止结果：

```text
任务目标达成 = 是
治理结果可接受 = 否
```

这两个结果不能合成一个布尔值。价值结算只围绕当前任务目标结果进行，禁止结果进入治理风险账。

## 9. 方法选择

多个方法都能产生当前任务目标结果时，排序口径为：

```text
1. 能否达成当前任务目标
2. 目标可信成功率
3. 禁止结果风险
4. 自我需要附带结果收益
5. 自我不需要附带结果复杂度
6. 成本
```

附带结果不能替代目标成功。目标不达成的方法，即使附带结果很多，也不能作为当前任务的有效方法。

## 10. 一句话定型

```text
任务只认一个目标；
方法可以留下很多结果；
结果要分类，不能全拿来结账。
```
