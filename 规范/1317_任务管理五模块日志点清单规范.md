# 1317 任务管理任务三段式日志点清单规范（保留原文件名兼容）

## 1. 目标

本文用于把 `任务管理任务` 的日志验收改成：

`目标 -> 筹办 -> 执行`


## 2. 核心结论

必须固定：

1. 日志验收不得再以“五模块”作为一级观察面。
2. 一级观察面只保留：`目标日志`、`筹办日志`、`执行日志`。
3. 若还需要看 `受理请求 / 绑定宿主方法 / 学习承接` 等内容，只能作为步骤级日志。


## 3. 最小日志链

```text
目标日志
-> 筹办日志
-> 执行日志
-> 结果日志
```


## 4. 一级日志要求

### 4.1 目标日志

至少记录：

- 宿主任务
- 当前完成判据

### 4.2 筹办日志

至少记录：

- 本轮形成了哪些筹办需求
- 自我线程是否回执需求可满足
- 本轮补了哪些步骤、结果、子任务和方法
- 是否转入学习任务或继续等待回执

### 4.3 执行日志

至少记录：

- 当前治理步骤
- 当前执行步骤消费到的筹办闸门
- 当前执行方法
- 当前治理结果


## 5. 二级步骤日志

任务管理任务若显式长出治理步骤树，则步骤级日志可继续记录：

- `受理请求`
- `宿主锚点确认`
- `绑定宿主方法`
- `宿主步骤树筹办`
- `前置缺口外化`
- `执行入口闸门`
- `执行当前治理步骤`
- `学习承接与业务派生`
- `写回结果`
- `组装一步结果`

但它们只允许作为二级日志观察口径保留。
