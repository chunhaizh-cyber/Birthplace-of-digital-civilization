# 0577 原语层与选择函数规范

## 1. 目标

本文用于把 `二次特征 -> 运行时判定 -> 下一步选择` 这条链正式收口成一套独立契约，重点回答以下问题：

- 为什么 `二次特征` 不能直接充当运行时控制器
- `原语层` 与 `选择函数` 分别负责什么
- 方法、任务、因果在运行时应如何消费 `二次特征`
- 哪些绑定必须挂在方法节点、任务节点或步骤节点上
- 当出现多个同向 `二次特征` 候选时，何时必须触发 `3次 / n次特征` 排序

本文只负责固定：

- `二次特征是证据层`
- `原语是判定层`
- `选择函数是路由层`
- `运行时不得裸判二次特征`

## 3. 核心结论

必须固定以下结论：

1. `二次特征` 负责比较、分域、细化触发，是运行时判定的正式证据层。
2. `原语层` 负责把 `场景 + 一次特征 + 二次特征 + 因果模板 + 目标范围 + 动作语义` 转成可执行判定。
3. `选择函数` 只消费原语输出，不直接消费裸 `二次特征节点`。
4. `方法目标 / 任务目标 / 因果条件 / 因果结果 / 方法评估` 都不得直接判裸二次特征，而是判与其绑定的原语执行结果。
5. 当同一需求轴上已经出现多个 `同向` 二次特征候选，且当前层还无法继续判定谁更接近需求时，选择函数不得硬判，必须触发升阶原语。

一句话收口：

`二次特征提供证据，原语给出判定，选择函数决定下一步。`


## 4. 三层总纲

运行时正式链路固定为：

```text
场景 / 一次特征 / 状态 / 动态
-> 二次特征
-> 原语层
-> 选择函数
-> 下一执行原语 / 下一方法节点 / 回滚 / 结束 / 升阶
```

三层职责固定如下：

| 层级 | 名称 | 负责什么 | 不负责什么 |
| --- | --- | --- | --- |
| 第 1 层 | 二次特征层 | 比较、分域、方向/对错/上下等三大域结果，并给出是否已形成 `同向候选组` 的证据 | 直接决定下一步执行动作 |
| 第 2 层 | 原语层 | 把证据翻译成 `满足 / 方向 / 冲突 / 结果符合 / 稳定性 / 升阶建议` 等正式判定 | 直接拍最终路由结论 |
| 第 3 层 | 选择层 | 综合原语结果和历史执行上下文，选 `继续 / 切换 / 回滚 / 终止 / 升阶` | 裸读一次特征或裸读二次特征做拍板 |


## 5. 原语层的正式分类

`原语` 必须分成两类：

### 5.1 判定原语

判定原语负责回答运行时问题，最少包括：

- `条件范围原语`
  - 判断因果条件、任务条件、方法前提是否满足
- `方向连续原语`
  - 判断方向是否一直一致
- `结果符合原语`
  - 判断当前结果是否落入目标范围
- `冲突原语`
  - 判断当前方向是否与目标方向相反
- `因果稳定原语`
  - 判断当前 `条件范围 -> 结果范围` 这对关系是否仍稳定成立
- `升阶原语`
  - 判断当前是否已出现多个同向候选，且必须升到 `3次 / n次` 做排序

### 5.2 执行原语

执行原语负责真正做动作，或调用下一步动作。

必须固定：

1. 判定原语回答 `当前怎么判断`。
2. 执行原语回答 `当前实际做什么`。
3. 两者不得混写成同一种返回值，也不得让执行原语直接越权代替选择函数。


## 6. 原语输出合同

原语输出至少必须包含：

- `满足度`
- `方向状态`
- `结果符合度`
- `冲突状态`
- `置信度`
- `下一步建议`

建议统一理解为：

| 字段 | 语义 |
| --- | --- |
| `满足度` | 当前条件或范围满足到什么程度 |
| `方向状态` | 同向 / 反向 / 中界 / 未知 |
| `结果符合度` | 当前结果和目标范围的贴合程度 |
| `冲突状态` | 当前是否与目标方向或安全边界冲突 |
| `置信度` | 本轮判定的可靠程度 |
| `下一步建议` | 建议继续、切换、等待、回滚、结束或升阶 |

必须固定：

1. 选择函数只能读取这类原语输出，不得绕回裸 `二次特征节点` 重新硬判。
2. 原语输出可以引用证据节点，但证据节点不是最终输出本体。


## 7. 运行时硬约束

以下规则必须正式生效：

1. 方法节点、任务节点、因果模板不得直接以裸二次特征作为最终执行条件。
2. 二次特征只能作为原语的输入证据。
3. 选择函数只能消费原语输出，不能直接消费裸二次特征节点。
4. 若运行时还存在 `条件主键命中 -> 直接执行`、`结果摘要命中 -> 直接达标`、`二次特征节点命中 -> 直接切换下一步` 这类写法，统一视为混层。
5. 当原语输出不足以支持继续 / 切换 / 结束时，选择函数必须返回 `待进一步比较 / 待进一步观察 / 触发升阶`，不得硬拍结果。
6. 当只有单个二次特征候选时，不得为了区分 `接近 / 相等 / 超过` 而伪造 `3次 / n次特征`；高阶只在同向候选需要排序时触发。


## 8. 绑定合同

### 8.1 方法节点与任务节点的绑定

每个方法节点和任务节点，运行时至少应带以下绑定：

- `guard_primitives`
  - 前提/条件原语
- `direction_primitives`
  - 方向连续原语
- `result_primitives`
  - 结果符合原语
- `conflict_primitives`
  - 冲突原语
- `escalation_primitives`
  - 升阶原语
- `selection_policy`
  - 根据原语结果选择下一执行原语或下一方法节点

必须固定：

1. 这些绑定是运行时正式入口，不是调试时临时拼装字段。
2. `条件判定索引 / 结果摘要索引 / 条件模板 / 结果模板` 继续保留为证据与模板面，但不得越权替代上述绑定。

### 8.2 因果模板的绑定

因果模板在运行时最少应绑定：

- `guard_primitives`
  - 解释条件范围是否满足
- `result_primitives`
  - 解释结果范围是否命中
- `conflict_primitives`
  - 解释当前结果是否与目标或安全边界冲突
- `escalation_primitives`
  - 解释当前条件/结果对是否需要进一步细化

必须固定：

1. `因果条件模板 / 因果结果模板` 是证据模板，不是执行控制器。
2. 因果是否仍稳定成立，必须由 `因果稳定原语` 或同级正式原语解释。


## 9. 选择函数的标准形态

选择函数的标准形态固定为：

```text
next = select(
  条件原语结果集,
  方向原语结果集,
  结果原语结果集,
  冲突原语结果集,
  因果原语结果集,
  历史执行上下文
)
```

选择函数的正式输出只允许是以下几类之一：

- `继续当前执行原语`
- `切换到另一执行原语`
- `回滚`
- `结束当前方法`
- `标记任务完成`
- `触发三次 / N 次特征同向候选排序`

必须固定：

1. 选择函数是路由层，不是新一套真值面。
2. 选择函数不得直接重算二次特征。
3. 选择函数不得越权把 `局部建议` 写成 `外层根层最终裁决`。


## 10. 与二次特征升阶的关系

当运行时已经出现：

- 同一需求轴上的两个或更多 `二次特征候选`
- 且这些候选在二次层都表现为 `同向`
- 但当前层仍无法继续判断 `谁更接近需求`

则必须固定：

1. 当前二次特征层只完成了方向判定，还没有完成候选排序。
2. `3次特征` 及更高 `n次特征` 的作用，不是重复描述方向，而是对同向候选做进一步排序。
3. 选择函数不得仅凭 `同向` 就宣布达标、继续强推或任选其一。
4. 必须触发 `升阶原语`，把当前问题正式升到 `3次 / n次特征` 继续做同向候选排序。


## 11. 当前阶段收口

当前阶段可以先按以下口径理解：

1. `二次特征` 解决的是“比较后形成了什么方向和关系”。
2. `原语层` 解决的是“这些方向和关系，放到任务 / 方法 / 因果运行时里该怎么判”。
3. `选择函数` 解决的是“判完以后下一步走哪条路”。

本文的主裁判句固定为：

`规范的核心不再只是二次特征怎么存，而是二次特征怎么和原语绑定，原语怎么被选择函数消费。`
