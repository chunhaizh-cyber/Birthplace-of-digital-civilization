# 学习相关本能函数规范 20260505

状态：当前学习语义规范。

## 1. 适用范围

本文定义学习相关本能函数的分类、边界、输入输出和组合方式。

学习不再作为独立任务系统存在。学习是需求树上的知识缺口、方法缺口或稳定性缺口对应的执行能力。任务仍然只是需求节点的运行壳：筹办阶段选择自我能力值，执行阶段配对参数并调用本能函数。

学习相关本能函数不得以方法名称、动作名称、显示文本、语素文本或函数名称作为语义判断依据。逻辑依据只能来自：

```text
特征类型
特征值
特征值区间
抽象特征父子关系
限制包
方法能力包
条件结果对
运行事实
```

名称只用于显示、日志和调试。

## 2. 三大类学习本能函数

学习相关的本能函数归纳为三大类：

| 类别 | 功能 | 输入 | 输出 | 使用场景 |
| --- | --- | --- | --- | --- |
| 尝试学习 | 将已有因果信息内化为自我能力 | 历史或观察到的因果信息、条件、结果、动作主体 | 条件结果对更新、方法树更新、实例特征更新、缺口子需求信号 | 已有因果信息或外部经验，需要转成自我可用能力 |
| 定向学习 | 在只有目标特征或方法骨架时生成动作或条件结果对 | 方法头节点骨架、目标需求、因果链或观察动态 | 初始动作节点、候选条件结果对、预测状态、子需求 | 方法头节点只有骨架，或动作缺失，需要生成候选方法结构 |
| 练习 | 用已有动作完善条件结果对 | 方法头节点、条件结果对、方法虚拟存在、任务传入参数 | 条件结果对更新、参数区间更新、方法统计更新、缺口特征 | 方法动作节点已存在，需要优化参数稳定性和结果可预测性 |

核心原则：

```text
尝试学习 = 自我能力内化。
定向学习 = 生成或虚拟动作。
练习 = 已有动作的条件结果对优化和稳定。
```

当前代码中已经落地的 `自我_练习` 对应第三类“练习”。历史枚举名 `自我_尝试学习`、`学习_尝试学习` 只能作为兼容别名理解，不得继续把旧学习任务系统作为权威结构。

## 2.1 三层学习方法层级

三类学习本能函数形成固定层级：

| 层级 | 条件 | 动作 | 结果 | 功能 |
| --- | --- | --- | --- | --- |
| 练习 | 已有条件 | 已有动作 | 已有结果 | 对已有动作和条件结果对进行优化、稳定化、收窄参数区间，完善方法树 |
| 尝试学习 | 已有条件 | 历史或外部动作 | 已有结果 | 将外部因果信息中的动作主体替换为自我，形成可执行自我能力，再交给练习完善 |
| 定向学习 | 仅目标特征 | 动作未知，需要发现或生成 | 预测结果 | 在缺少动作节点或方法骨架时，通过因果链或观察动态找到可能动作，生成初始方法和条件结果对 |

判定顺序：

```text
目标明确但动作未知
  -> 定向学习

动作存在且有条件结果对，但动作主体不是自我或能力尚未内化
  -> 尝试学习

动作节点和条件结果对均已存在，但结果不稳定或参数区间不稳
  -> 练习
```

层级关系：

```text
目标明确但动作未知
  -> 定向学习
  -> 生成候选动作和候选条件结果对

动作存在且有条件结果对
  -> 尝试学习
  -> 替换动作主体为自我能力

条件结果对需要优化
  -> 练习
  -> 优化条件结果对并收窄参数区间
```

三层不得互相代偿：

```text
练习不得生成新动作。
练习一般不得直接生成新需求，只能写缺口事实、叶子需求状态和建议下一步能力。
尝试学习不得凭空生成动作，只能内化已有因果动作。
定向学习生成的动作和条件结果对只是候选，必须等待尝试学习或练习验证。
```

## 2.2 新需求生成权责

生成新需求只在方法树不完整、动作缺失、条件结果对缺失或参数缺口阻塞闭环时发生。新需求必须由结构化缺口特征触发，不得随机生成。

| 学习方法 | 允许提出新需求 | 权责边界 |
| --- | --- | --- |
| 尝试学习 | 允许 | 内化已有因果信息时，若发现条件结果对缺失、参数缺失、方法稳定性不足或自我动作主体缺口，可以生成缺口子需求信号。它不得凭空生成新动作；动作缺失应转成缺口需求，由定向学习或动作生成机制处理。 |
| 定向学习 | 允许 | 面对目标特征明确但动作未知、方法头只有骨架或条件不足时，可以生成候选方法头、候选动作节点、初始条件结果对，并为无法闭合的条件生成子需求。 |
| 练习 | 一般不允许 | 只能在已有动作节点和已有条件结果对基础上完善参数、区间、统计和稳定性。参数不足、条件不闭合或结果不稳定时，练习写入缺口类型、叶子需求状态、失败原因和建议下一步能力，由筹办阶段或执行治理消费这些缺口特征后生成子需求。 |

判断规则：

```text
动作缺失或方法头只有骨架
  -> 定向学习可生成候选结构和子需求。

已有因果动作需要变成自我能力
  -> 尝试学习可生成内化缺口子需求。

已有动作节点需要参数、条件结果对和稳定性优化
  -> 练习只更新方法树已有分支、方法虚拟存在、实例特征和运行虚拟存在。
```

## 3. 尝试学习

### 3.1 定义

尝试学习用于把已有因果信息、外部经验或观察事实内化为自我能力。

它的关键动作是：

```text
读取因果信息
  -> 校验条件、结果、动作主体
  -> 将可迁移动作主体归一到自我
  -> 生成或更新自我可用的方法能力描述
  -> 写入方法树、条件结果对和实例特征
```

尝试学习不是随机试运行。它的前提是已经存在可解释的因果信息或观察事实。

### 3.2 输入

输入参数场景应提供：

```text
输入条件包
  来源因果信息
  来源观察动态
  条件包
  结果包
  动作主体
  目标宿主
  目标特征类型
  目标值类型
  限制包
```

缺少可解释因果信息时，不得伪造经验，应输出参数绑定失败或知识缺口。

### 3.3 输出

输出结果场景应写入：

```text
学习状态
内化状态
目标自我能力值
更新的方法能力包
更新的条件结果对
更新的实例特征
派生需求
失败原因
```

运行虚拟存在应记录：

```text
来源因果信息
来源观察动态
动作主体归一化结果
条件结果对更新记录
方法树更新记录
运行事实
```

### 3.4 处理流程

尝试学习按以下顺序执行：

```text
1. 选择可内化的因果信息
   -> 读取历史因果、外部因果或观察因果。
   -> 过滤条件、结果、动作主体不完整的因果。
   -> 排除已经属于自我能力的因果。

2. 查找可替换为自我的动作主体
   -> 读取原动作主体的类型特征。
   -> 在方法虚拟存在或方法树中查找动作主体类型相同或兼容的自我动作节点。
   -> 优先选择已经标记为可执行的自我动作节点。
   -> 若不存在，输出动作主体缺口，不在尝试学习中生成新动作。

3. 替换动作主体
   -> 将因果信息中的动作主体归一为自我。
   -> 保留原条件节点和结果节点。
   -> 记录动作主体替换路径和兼容依据。

4. 生成条件结果对
   -> 把替换后的条件、动作、结果写入目标方法虚拟存在。
   -> 更新条件节点、结果节点和统计信息。
   -> 写入自我能力相关实例特征。

5. 输出学习事实
   -> 输出方法树更新、条件结果对更新、运行虚拟存在。
   -> 若动作主体缺失、条件不闭合或结果不稳定，生成子需求信号。
```

尝试学习的关键问题是“如何找到自我的动作”。判定口径如下：

```text
原动作主体类型
  -> 查找相同或兼容的自我动作节点
  -> 校验方法能力包和可执行状态
  -> 命中则内化
  -> 未命中则生成缺口需求，由定向学习或动作生成机制处理
```

### 3.5 边界

尝试学习不得：

```text
凭空生成动作。
在没有因果证据时假设能力存在。
绕过方法能力包和限制包。
把外部动作主体直接当成自我能力。
在未找到自我兼容动作时伪造动作节点。
```

## 4. 定向学习

### 4.1 定义

定向学习用于在目标明确但方法结构不足时生成候选动作或候选条件结果对。

它处理的是：

```text
方法头节点只有骨架。
目标需求明确但动作缺失。
目标特征类型明确但没有可执行方法。
方法能力包、形参包或条件结果对缺失。
```

### 4.2 输入

输入参数场景应提供：

```text
输入条件包
  目标方法头节点
  目标需求节点
  目标宿主
  目标特征类型
  目标值类型
  目标差值约束
  来源因果链
  来源观察动态
  限制包
```

目标特征类型可以理解为抽象特征树节点。任意抽象特征或实例特征都可以作为 `特征类型` 的值；实例特征作为值时，先归一到它引用的抽象特征。

### 4.3 输出

输出结果场景应写入：

```text
学习状态
候选动作节点
候选方法能力包
候选形参包
候选结果包
候选条件结果对
预测状态
派生需求
失败原因
```

方法树更新应标记候选状态，不得把未经验证的候选结构直接升格为稳定事实。

### 4.4 处理流程

定向学习按以下顺序执行：

```text
1. 分析特征间间接变化
   -> 遍历因果链和观察动态。
   -> 找到哪些条件特征变化可以导致目标特征变化。
   -> 记录变化路径、影响方向和抽象层级。
   -> 受限制包约束，不允许无限泛化。

2. 生成候选动作主体
   -> 根据因果分析创建动作节点骨架。
   -> 为动作节点分配输入参数模板。
   -> 写入候选条件值、候选结果值和限制来源。
   -> 标记为候选动作，不标记为稳定能力。

3. 构建初始条件结果对
   -> 条件侧使用因果链中的起始特征及其区间。
   -> 结果侧使用目标特征及预期状态或差值区间。
   -> 写入方法虚拟存在。
   -> 标记条件结果对状态为候选。

4. 输出候选结构
   -> 输出新动作节点骨架。
   -> 输出初始条件结果对。
   -> 输出预测结果状态。
   -> 若部分条件缺失，输出子需求生成信号。
```

定向学习用于“缺动作节点”或“只有方法头骨架”的场景。它可以生成候选动作和初始条件结果对，但生成结果必须等待练习或真实执行验证。

### 4.5 边界

定向学习不得：

```text
把目标名称当成方法依据。
把候选动作当成已验证能力。
跳过后续练习验证。
在没有执行入口时伪造试运行结果。
```

## 5. 练习

### 5.1 定义

练习用于对已有动作节点进行参数试运行、条件结果对收敛和稳定性统计。

它的核心动作是：

```text
读取目标方法头节点
  -> 读取方法虚拟存在、形参包和条件结果对
  -> 根据任务传入参数完整度判断练习模式
  -> 自己生成实参
  -> 可试运行时调用目标方法
  -> 用运行事实更新条件结果对、参数区间和方法统计
```

练习只在动作节点已经存在时生效。它不生成新动作，不替换动作主体，只完善方法树已有分支。

### 5.2 两种练习模式

练习分为两种模式：

| 模式 | 判定依据 | 语义 |
| --- | --- | --- |
| 指定练习 | 任务传入的目标特征类型、练习方向、执行差值或目标执行后值等参数闭合 | 按任务指定方向生成实参并试运行 |
| 随机练习 | 任务传入参数不闭合，但目标方法具备可练习结构 | 由练习自行生成未出现过的候选实参 |

不得新增语义结构体来判断模式。模式由任务传入参数完整度、方法形参包和值域约束共同决定。

### 5.3 输入

输入参数场景应提供或允许推导：

```text
输入条件包
  目标方法头节点
  目标自我能力值
  目标宿主
  目标特征类型
  练习方向
  执行差值
  目标执行后值
  条件结果对
  方法虚拟存在
  采样限制包
```

其中：

```text
目标特征类型 + 练习方向 + 执行差值或目标执行后值闭合
  -> 指定练习

上述参数不闭合，但目标方法可练习
  -> 随机练习
```

练习必须自己生成参数，不得要求任务替它补造随机实参。

### 5.4 实参生成

练习生成实参时只能使用：

```text
形参包
参数值类型
允许值域
历史成功实参
历史失败实参
条件结果对中的稳定区间
采样限制包
运行事实
```

建议策略：

```text
边界值生成：区间下界、上界、0、1、-1、中心值。
历史值生成：使用历史成功实参或失败边界附近值。
稳态值生成：使用条件结果对中的稳定区间。
随机生成：在允许区间内采样未出现过的参数。
收缩生成：在成功区间附近细化采样。
扩展生成：在失败边界外少量探索。
```

随机练习必须尽量避开已经出现过的参数组合。若无法生成新参数，应输出证据不足或参数空间耗尽缺口。练习生成参数属于已有动作分支内的内部补齐，不等价于生成子需求。

### 5.5 输出

输出结果场景应写入：

```text
练习状态
练习模式
候选实参表
试运行次数
成功次数
失败次数
更新的条件结果对
更新的参数区间
更新的方法统计
参数来源
缺口类型
叶子需求状态
建议下一步能力
失败原因
```

运行虚拟存在应记录：

```text
目标方法头节点
目标方法虚拟存在
条件结果对引用
候选实参表
试运行样本集合
运行事实集合
学习状态
开始时间
结束时间
```

### 5.6 边界

练习不得：

```text
替代定向学习生成全新动作。
替代尝试学习内化外部因果经验。
生成新需求节点、方法头节点或动作节点。
在条件结果对完全缺失时从零构造条件结果对。
在方法没有可调用执行入口时伪造试运行结果。
用默认成功遮蔽参数空间耗尽或结构缺口。
深度复制抽象特征树作为实例特征。
```

方法只有头节点、缺少动作节点或缺少可调用执行入口时，练习不得接管结构生长，只能：

```text
输出方法结构自检结果。
写入方法结构缺口或执行入口缺口。
写入叶子需求状态。
写入建议下一步能力，例如定向学习、尝试学习或绑定执行入口。
```

它不能生成候选动作、候选方法头或伪造试运行事实。

### 5.7 处理流程

练习按以下顺序执行：

```text
1. 生成输入参数组合
   -> 读取方法形参包、条件结果对和历史运行事实。
   -> 根据参数完整度选择指定练习或随机练习。
   -> 使用历史执行数据、稳定区间和参数约束生成可尝试参数。

2. 执行已有动作
   -> 调用目标方法已有动作节点。
   -> 检查执行结果是否满足条件结果对。
   -> 若不稳定，记录失败样本。

3. 调整条件结果对
   -> 修正条件区间。
   -> 修正结果区间。
   -> 更新参数稳定区间。
   -> 拆分或收缩不稳定条件结果对。

4. 更新统计和实例特征
   -> 更新成功次数、失败次数、置信度和平均运行信息。
   -> 更新方法虚拟存在状态。
   -> 写入实例特征更新和运行事实引用。

5. 输出练习结果
   -> 输出更新后的条件结果对。
   -> 输出方法虚拟存在状态。
   -> 输出实例特征更新。
```

练习失败不是终点。参数不足、条件不闭合、结果不稳定或实例特征未命中抽象区间时，应输出明确失败事实、缺口类型、叶子需求状态和建议下一步能力。是否把这些缺口事实生长为子需求，由筹办阶段或执行治理裁决；练习本身不直接创建新的需求节点。

## 6. 为什么其他学习形式都归入三类

其他学习形式不需要新增本能函数类别，可以归约为三类学习能力的组合：

| 复杂形式 | 归约方式 |
| --- | --- |
| 参数学习 | 练习 |
| 策略学习 | 练习 + 条件结果对统计 |
| 探索新动作 | 定向学习 |
| 方法生成 | 定向学习 |
| 内化外部经验 | 尝试学习 |
| 预测学习 | 定向学习生成预测结构，再由练习验证 |
| 强化学习 | 定向学习生成候选动作，练习试运行，尝试学习内化稳定因果 |

组合流程：

```text
生成候选动作
  -> 定向学习

执行或试运行
  -> 练习

观察因果规律并内化
  -> 尝试学习
```

因此，新增学习能力时，应优先扩展三类本能函数的输入、输出或策略，不应创造新的学习类别。

整体闭环可表达为：

```text
筹办阶段
  -> 查找方法头节点
  -> 校验方法完整性
  -> 缺动作节点：派生定向学习需求
  -> 条件结果对不稳定：派生练习需求

执行阶段
  -> 参数配对
  -> 执行本能函数
  -> 更新实例特征

练习
  -> 优化已有动作条件结果对
  -> 方法树稳定

定向学习
  -> 生成动作节点和初始条件结果对
  -> 方法树成长

尝试学习
  -> 替换因果信息中的动作主体为自我
  -> 更新方法树和条件结果对
  -> 形成自我能力

需求树生长
  -> 叶子任务完成
  -> 父需求回传
  -> 下一轮筹办
```

该闭环必须保持以下分离：

```text
动作缺失与条件缺失分离。
方法树成长与方法树稳定分离。
候选结构与稳定结构分离。
因果内化与动作生成分离。
```

## 7. 与需求树的关系

学习缺口可以生长为需求节点，但需求生成权责属于需求治理、任务筹办或允许生成子需求的学习方法。学习本能函数必须先写入结构化缺口事实：

```text
方法头骨架或动作节点缺失
  -> 定向学习

已有动作主体的因果信息未内化
  -> 尝试学习

已有动作节点的条件结果对不完整或不稳定
  -> 练习

条件结果对缺失且可由已有因果信息补齐
  -> 尝试学习

参数空间证据不足
  -> 练习写缺口特征，由治理链决定是否生成参数补齐子需求
```

需求节点对应任务，任务按两阶段推进：

```text
筹办
  -> 查找方法
  -> 选中自我能力值
  -> 方法缺口则生成子需求

执行
  -> 配对参数
  -> 调用学习本能函数
  -> 成功则写运行事实
  -> 失败则写知识缺口、参数缺口或方法结构缺口
  -> 缺口是否生成子需求由治理链或允许生成子需求的学习方法裁决
```

学习结果不能直接把父需求标记为满足。父需求是否满足，必须由需求满足检查读取实例特征、运行事实和条件结果对后确认。

## 8. 与方法树的关系

学习相关本能函数只能通过方法树和方法虚拟存在沉淀能力：

```text
方法头节点
方法虚拟存在
自我能力实例特征
方法能力包
形参包
结果包
条件结果对
试运行限制包
运行事实集合
稳定性统计
```

候选结构必须显式标记候选状态。只有经过运行事实验证，并满足稳定阈值后，才能升格为稳定条件结果对或稳定能力描述。

## 9. 特征化要求

学习状态、练习模式、缺口类型、候选状态、稳定状态、成功次数、失败次数、试运行次数、参数区间等长期语义必须写入实例特征。

结构体只能作为：

```text
临时参数
消息包
局部算法缓存
返回包
```

不能把结构体字段作为长期语义权威。

推荐抽象特征：

```text
学习类别
  尝试学习 [1,1]
  定向学习 [2,2]
  练习 [3,3]

练习模式
  指定练习 [1,1]
  随机练习 [2,2]

学习结构状态
  候选 [1,1]
  试用 [2,2]
  稳定 [3,3]
  废弃 [4,4]

学习失败原因
  输入绑定失败
  方法结构缺口
  执行入口缺口
  参数空间耗尽
  条件结果对不稳定
  因果证据不足

参数来源
  任务提供 [1,1]
  自动生成 [2,2]

需求触发方式
  定向学习生成 [1,1]
  尝试学习生成 [2,2]
  治理消费缺口生成 [3,3]
  不生成新需求 [4,4]

叶子需求状态
  可执行 [1,1]
  等待缺口闭合 [2,2]
  阻塞 [3,3]

建议下一步能力
  定向学习
  尝试学习
  练习
  绑定执行入口
  补充参数证据
```

## 10. 日志与运行证据

学习相关本能函数必须使用日志模块记录关键断点：

```text
输入绑定失败
方法结构缺口
执行入口缺口
参数生成失败
试运行失败
条件结果对未更新
实例特征未命中抽象区间
候选结构升格或废弃
```

遇到结构性问题或证据不足时，可以暂停当前任务并提出问题，或写入明确缺口特征。尝试学习和定向学习可以按权责生成缺口子需求；练习只输出缺口事实，由治理链决定是否生成子需求。不得使用默认成功、白名单或兜底通过遮蔽断点。

## 11. 最终规则

```text
学习相关本能函数只有三类：
  尝试学习
  定向学习
  练习

尝试学习负责内化已知经验。
定向学习负责生成候选动作或候选条件结果对。
练习负责优化已有动作的参数、条件结果对和稳定性。

新需求只由定向学习、尝试学习或治理链基于缺口特征生成。
练习不生成新动作、新方法头或新需求节点。
生成子方法或子需求必须通过缺口特征触发，不得随机生成。

其他学习形式必须归约为这三类的策略扩展或组合流程。
```
