# 学习相关本能函数规范 20260505

状态：当前学习语义规范。

## 1. 适用范围

本文定义学习相关本能函数的分类、边界、输入输出和组合方式。

学习不再作为独立任务系统存在。学习是需求树上的知识缺口、方法补齐入口或稳定性缺口对应的执行能力。任务仍然只是需求节点的运行壳：筹办阶段选择自我能力值，执行阶段配对参数并调用本能函数。

学习相关本能函数不得以方法名称、动作名称、显示文本、语素文本或函数名称作为语义判断依据。逻辑依据只能来自：

```text
特征类型
特征值
特征值区间
抽象特征父子关系
限制包
方法能力包
条件结果对
运行事实
```

名称只用于显示、日志和调试。

学习本能方法被查找到的唯一正当入口是结果能力匹配：

```text
需求结果 = 一个目标特征类型 + 一个目标方向 / 目标状态关系
方法能力签名 = 条件面 -> 结果包
匹配核心 = 方法结果包中存在结果项覆盖该需求结果
```

找不到就是找不到。查找方法不得因为名称叫“练习”“尝试学习”“定向学习”，也不得因为动作句柄、本能 ID 或固定编号，就绕过结果能力声明直接取方法。真正找不到时，说明当前方法树没有任何方法声明能满足该结果需求，只能暴露闭环未建立、本能方法缺失或本能方法结果声明缺失。

## 2. 三大类学习本能函数

学习相关的本能函数归纳为三大类：

| 类别 | 功能 | 输入 | 输出 | 使用场景 |
| --- | --- | --- | --- | --- |
| 尝试学习 | 将已有因果信息内化为自我能力 | 历史或观察到的因果信息、条件、结果、动作主体 | 条件结果对更新、方法树更新、实例特征更新、缺口子需求信号 | 已有因果信息或外部经验，需要转成自我可用能力 |
| 定向学习 | 在外界信息或未归因现象中形成观察方向、确认方向和目标方向 | 外界信息、未归因现象、观察动态、确认对象候选 | 学习方向、观察需求、确认需求、信息需求、预测状态 | 外界信息里动作主体或动作函数未知，需要先识别目标、方向、观察对象或确认对象 |
| 练习 | 用已有动作完善并发现条件结果对 | 方法头节点、条件结果对、方法虚拟存在、任务传入参数 | 条件结果对更新、参数区间更新、新用途候选、同条件结果复现/分裂账、缺口特征 | 方法动作节点已存在，需要优化参数稳定性、结果可预测性，或探索同一动作在不同输入下的新结果能力 |

核心原则：

```text
尝试学习 = 自我能力内化。
定向学习 = 外界信息处理链中的方向形成。
练习 = 已有动作的条件结果对优化、稳定和新用途发现。
```

当前代码中已经落地的 `自我_练习` 对应第三类“练习”。`自我_尝试学习` 是正式本能方法；历史学习命名空间下的尝试学习枚举不再作为兼容入口使用，不得继续把旧独立学习系统作为权威结构。

## 2.1 三类学习本能层级

三类学习本能函数形成固定层级：

| 层级 | 条件 | 动作 | 结果 | 功能 |
| --- | --- | --- | --- | --- |
| 练习 | 已有条件或可生成输入参数 | 已有动作 | 已有结果或待发现结果 | 对已有动作和条件结果对进行验证、优化、稳定化、收窄参数区间，并发现同一动作在新输入参数下的新结果能力 |
| 尝试学习 | 已有条件 | 历史或外部动作 | 已有结果 | 将外部因果信息中的动作主体替换为自我，形成可执行自我能力，再交给练习完善 |
| 定向学习 | 外界现象或未归因信息 | 动作主体或动作函数未知 | 预测方向或待确认目标 | 形成观察方向、确认方向、信息需求和外界方向材料 |

判定顺序：

```text
外界信息只有现象，动作主体或动作函数未知
  -> 定向学习

动作存在且有条件结果对，但动作主体不是自我或能力尚未内化
  -> 尝试学习

动作节点已存在，但结果不稳定、参数区间不稳，或需要探索不同输入参数会造成什么结果
  -> 练习
```

层级关系：

```text
外界信息只有现象，动作主体或动作函数未知
  -> 定向学习
  -> 形成观察需求、确认需求、信息需求和外界方向材料

动作存在且有条件结果对
  -> 尝试学习
  -> 替换动作主体为自我能力

条件结果对需要优化
  -> 练习
  -> 优化条件结果对并收窄参数区间
```

三层不得互相代偿：

```text
练习不得生成新动作。
练习一般不得直接生成新需求，只能写缺口事实、叶子需求状态和建议下一步能力。
练习可以发现同一动作的新用途：当不同输入参数产生新的结果变化时，练习应把本次执行产出的条件结果对样本送入条件结果对归并。归并发现没有等价结构时添加新条件或新结果分支；发现已有等价结构时融合统计和证据。不能把所有结果合并进旧用途。
尝试学习不得凭空生成动作，只能内化已有因果动作。
定向学习只在外界信息处理链中形成方向材料、观察需求、确认需求或信息需求；不得在自我内部方法补齐闭环中替代尝试学习。
```

## 2.3 能力签名声明

三类学习本能函数必须在方法首节点声明自己的能力签名。没有能力签名时，它们作为本能动作可以存在，但不能被 `查找方法` 按需求结果稳定命中。

能力签名结构固定为：

```text
能力签名 = 条件面 -> 结果包
条件面 = 多个可用条件组
条件组 = 多个条件项，组内全满足
结果包 = 多个结果项
```

查找时需求和任务都只有一个目标特征类型，某个结果包中任一结果项覆盖它即可命中。若父级目标包含多个特征，必须先展开成多个子需求 / 子任务，再分别查找；不得在查找阶段把多个目标包或多个方法拼接成一个答案。

固定特征字典：

```text
学习方向状态
目标特征类型识别状态
目标变化方向识别状态
候选方法存在状态
方法动作骨架状态
方法条件结果配对状态
方法结构是否完整
方法稳定状态
方法是否可用
方法运行账完整性
方法成熟度阶段
方法新用途发现状态
```

定向学习的能力签名：

```text
条件组：
  外界信息上下文 = 存在
  动作主体 = 未知 或 动作函数 = 未知

结果包：
  学习方向状态 -> 已形成
  目标特征类型识别状态 -> 已识别
  目标变化方向识别状态 -> 已识别
  学习约束状态 -> 从无到有
  方法需求位识别状态 -> 已命中
```

尝试学习的能力签名：

```text
条件组：
  内部因果样本.动作主体 = 已知
  内部因果样本.动作函数 = 已知
  内部因果样本.结果状态 = 已知

结果包：
  目标方法.候选方法存在状态 -> 已存在
  方法动作骨架状态 -> 已形成
  方法条件结果配对状态 -> 已配对
  方法结构是否完整 -> 更完整
  方法最小出生锚点 -> 具备
```

练习的能力签名：

```text
条件组：
  目标方法.候选方法存在状态 = 存在

结果包：
  方法稳定状态 -> 更稳定
  方法是否可用 -> 趋向可用
  方法运行账完整性 -> 更完整
  方法成熟度阶段 -> 上升
  方法新用途发现状态 -> 发现候选
```

因此闭环分派不是“指定某个学习方法”，而是由缺口形成新的结果需求：

```text
需要方法是否可用趋向可用
  -> 查找能造成该结果的方法
  -> 命中练习

筹办发现没有能满足目标的方法
  -> 创建或复用待学习目标方法头
  -> 父任务待重筹办

下一轮筹办命中待学习方法头但发现结构不可执行
  -> 形成方法动作骨架 / 条件结果配对等可执行性子需求
  -> 命中尝试学习或练习

尝试学习读取内部因果样本
  -> 从动作函数、动作主体、动作前状态、动作后状态中抽取方法候选
  -> 补齐目标方法头的动作骨架和条件结果对
  -> 回到练习验证候选方法
```

自我内部方法补齐闭环中不生成 `学习方向状态 从未形成到已形成` 这类需求。内部因果样本已经带有动作主体、动作函数、起始状态、结果状态、结果特征类型和结果变化方向；如果尝试学习不能形成候选方法，原因应解释为内部因果样本缺失、样本与缺口不匹配、结果特征类型无法对应当前需求、条件不足、动作函数无法转成动作骨架，或方法结果方向无法稳定归纳，而不是转去定向学习。

## 2.4 学习本能方法自身环境

三类学习本能方法在处理目标方法、因果样本、观察动态或练习参数之前，必须先确保自己的方法环境。

自身方法环境包括：

```text
自己的方法首节点
自己的方法虚拟存在
自己的本轮运行虚拟存在
自己的输入槽 / 输出槽
自己的条件场景 / 结果场景
自己的能力签名
自己的专属规格特征
```

这与目标方法环境不同。

例如 `自我_尝试学习` 中：

```text
自我_尝试学习 = 当前本能方法自身环境
目标方法头节点 = 被学习 / 被补齐的目标方法环境
```

禁止把目标方法虚拟存在当作当前学习本能方法自己的虚拟存在；也禁止因为目标方法存在就跳过 `自我_尝试学习` 自身规格确认。

三类学习本能方法的差异应集中在专属规格确认函数中：

```text
确认尝试学习规格
确认定向学习规格
确认练习规格
```

这些函数负责建立本方法自己的输入槽、输出槽、条件槽、结果槽、能力签名和专属状态词典。通用的本能方法头、方法虚拟存在、本轮运行虚拟存在由统一的“确保本能方法自身环境”入口处理。

详细规则见：


## 2.5 新需求生成权责

生成新需求只在方法树不完整、动作缺失、条件结果对缺失或参数缺口阻塞闭环时发生。新需求必须由结构化缺口特征触发，不得随机生成。

| 学习本能函数 | 允许提出新需求 | 权责边界 |
| --- | --- | --- |
| 尝试学习 | 允许 | 内化已有内部因果信息时，若发现条件结果对缺失、参数缺失、方法稳定性不足或自我动作主体缺口，可以生成缺口子需求信号。内部因果样本必须已经有动作主体和动作函数；样本缺这两项时不可用于尝试学习，但这不是定向学习入口。 |
| 定向学习 | 允许 | 只属于外界信息处理链。面对外界信息、未归因现象、动作主体未知或动作函数未知的情况，可以形成学习方向、观察需求、确认需求或信息需求。它不参与自我内部方法补齐闭环。 |
| 练习 | 一般不允许 | 只能在已有动作节点和已有条件结果对基础上完善参数、区间、统计和稳定性。参数不足、条件不闭合或结果不稳定时，练习写入缺口类型、叶子需求状态、失败原因和建议下一步能力，由筹办阶段或执行治理消费这些缺口特征后生成子需求。 |

判断规则：

```text
外界信息只有现象，动作主体或动作函数未知
  -> 定向学习可形成观察需求、确认需求、信息需求和外界方向材料。

已有内部因果动作需要变成自我能力
  -> 尝试学习可生成内化缺口子需求。

已有动作节点需要参数、条件结果对、稳定性优化或新用途探索
  -> 练习更新方法树已有分支、方法虚拟存在、实例特征和运行虚拟存在；若发现新用途，只能新增该动作下的条件结果对候选，不得生成新动作。
```

## 3. 尝试学习

### 3.1 定义

尝试学习用于把已有内部因果信息内化为自我能力。内部因果信息不是“无动作来源的信息”，它至少应当有动作主体、动作函数、动作前状态、动作后状态、结果特征类型和结果变化方向。

它的关键动作是：

```text
读取内部因果样本
  -> 校验动作主体、动作函数、动作前状态、动作后状态
  -> 抽取结果特征类型和结果变化方向
  -> 生成候选方法、动作骨架、方法条件和方法结果
  -> 写入方法树、条件结果对和实例特征
```

尝试学习不是随机试运行，也不是方向猜测。它的前提是已经存在可解释的内部因果样本；方向应从内部因果样本的状态差异中抽取，而不是再查找定向学习。

### 3.2 输入

输入参数场景应提供：

```text
输入条件包
  内部因果样本
  来源需求
  方法补齐目标
  已有动作函数
  动作主体
  动作前状态
  动作后状态
  条件包
  结果包
  结果特征类型
  结果变化方向
  限制包
```

缺少可解释内部因果样本时，不得伪造经验，应输出参数绑定失败或知识缺口。内部因果样本缺少动作主体或动作函数时，该样本不可用于尝试学习，但这不是定向学习入口。

### 3.3 输出

输出结果场景应写入：

```text
学习状态
内化状态
目标自我能力值
候选方法存在状态
方法动作骨架状态
方法条件结果配对状态
方法能力签名集
方法条件面
方法结果包
更新的条件结果对
更新的实例特征
派生需求
失败原因
```

运行虚拟存在应记录：

```text
内部因果样本
动作函数解析结果
动作主体解析结果
动作前状态
动作后状态
状态差异
结果特征类型
结果变化方向
条件结果对更新记录
方法树更新记录
运行事实
```

### 3.4 处理流程

尝试学习按以下顺序执行：

```text
1. 选择可内化的内部因果样本
   -> 读取内部动作记录形成的因果样本。
   -> 过滤缺少动作主体、动作函数、起始状态或结果状态的样本。
   -> 过滤与当前方法补齐目标不匹配的样本。

2. 解析动作和方向
   -> 从动作函数抽取方法动作骨架。
   -> 从动作主体抽取执行主体或作用主体。
   -> 从动作前状态抽取方法条件。
   -> 从动作后状态抽取方法结果。
   -> 从状态差异抽取方法结果项方向。
   -> 从结果状态抽取方法结果项特征类型。

3. 形成候选方法
   -> 写入方法动作骨架。
   -> 写入方法条件节点。
   -> 写入方法结果节点。
   -> 写入能力签名集。
   -> 写入条件面。
   -> 写入结果包。

4. 生成条件结果对
   -> 把条件、动作、结果写入目标方法虚拟存在。
   -> 更新条件节点、结果节点和统计信息。
   -> 写入自我能力相关实例特征。

5. 输出学习事实
   -> 输出方法树更新、条件结果对更新、运行虚拟存在。
   -> 若动作主体缺失、条件不闭合或结果不稳定，生成子需求信号。
```

尝试学习的关键问题是“内部因果样本能否形成方法候选”。判定口径如下：

```text
内部因果样本
  -> 动作主体已知
  -> 动作函数已知
  -> 动作前状态已知
  -> 动作后状态已知
  -> 结果特征类型可对应当前需求
  -> 结果方向可由状态差异归纳
  -> 形成候选方法
```

### 3.5 边界

尝试学习不得：

```text
凭空生成动作。
在没有内部因果证据时假设能力存在。
绕过方法能力包和限制包。
把外界未归因现象直接当成自我能力。
在动作主体或动作函数缺失时伪造动作骨架。
把内部因果样本缺失解释成需要定向学习。
```

## 4. 定向学习

### 4.1 定义

定向学习用于外界信息处理链，在外界信息、未归因现象或观察事实里形成学习方向、观察目标、确认目标或信息需求。

它处理的是：

```text
外界信息里可能没有明确的动作函数。
外界信息里可能不知道动作主体。
外界信息里可能只有现象，没有可归因动作。
需要识别目标、方向、观察对象或确认对象。
```

它不处理自我内部方法补齐闭环。内部动作有动作主体、动作函数和因果记录时，应由尝试学习直接从内部因果样本形成候选方法。

### 4.2 输入

输入参数场景应提供：

```text
输入条件包
  外界信息
  未归因外部现象
  来源观察动态
  观察对象候选
  确认对象候选
  目标特征候选
  目标方向候选
  限制包
```

目标特征类型可以理解为抽象特征树节点。任意抽象特征或实例特征都可以作为 `特征类型` 的值；实例特征作为值时，先归一到它引用的抽象特征。

### 4.3 输出

输出结果场景应写入：

```text
学习状态
学习方向状态
观察需求
确认需求
信息需求
目标特征类型识别状态
目标变化方向识别状态
预测状态
派生需求
失败原因
```

定向学习输出的是外界方向材料，不得把未经归因的外界现象直接升格为方法动作骨架或稳定条件结果对。

### 4.4 处理流程

定向学习按以下顺序执行：

```text
1. 分析外界信息和未归因现象
   -> 遍历外界信息、观察动态和候选对象。
   -> 找到需要进一步观察、确认或询问的信息缺口。
   -> 记录可能目标、可能方向和抽象层级。
   -> 受限制包约束，不允许无限泛化。

2. 形成学习方向
   -> 识别目标特征类型候选。
   -> 识别目标变化方向候选。
   -> 识别观察对象或确认对象。
   -> 标记学习方向状态为已形成。

3. 生成外界方向材料
   -> 生成观察需求。
   -> 生成确认需求。
   -> 生成信息需求。
   -> 记录目标和方向的不确定性来源。

4. 输出方向事实
   -> 输出学习方向状态。
   -> 输出目标识别状态。
   -> 输出方向识别状态。
   -> 若信息不足，输出观察、确认或信息需求。
```

定向学习用于外界信息处理链。它可以生成方向材料和后续观察/确认/信息需求，但不在自我内部方法补齐闭环里生成候选方法。

### 4.5 边界

定向学习不得：

```text
把目标名称当成方法依据。
把外界未归因现象当成已验证能力。
在自我内部方法补齐闭环里替代尝试学习。
在已有内部因果样本时重新猜方向。
把方向材料直接写成方法动作骨架。
```

## 5. 练习

### 5.1 定义

练习用于对已有动作节点进行参数试运行、条件结果对收敛、稳定性统计和新用途发现。

它的核心动作是：

```text
读取目标方法头节点
  -> 读取方法虚拟存在、形参包和条件结果对
  -> 根据任务传入参数完整度判断练习模式
  -> 自己生成实参
  -> 可试运行时调用目标方法
  -> 用条件结果对事实样本更新方法树、参数区间和同条件结果复现/分裂账
  -> 当新输入参数稳定地产生不同结果时，沉淀新的条件结果对候选
```

练习只在动作节点已经存在时生效。它不生成新动作，不替换动作主体；但同一个动作可以因为输入参数不同而产生不同结果，练习负责发现和验证这种新用途，并把它表达为同一动作下的新增条件结果对候选。

### 5.2 两种练习模式

练习分为两种模式：

| 模式 | 判定依据 | 语义 |
| --- | --- | --- |
| 指定练习 | 任务传入的目标特征类型、练习方向、执行差值或目标执行后值等参数闭合 | 按任务指定方向生成实参并试运行 |
| 探索练习 | 任务传入参数不闭合，但目标方法具备可练习结构 | 由练习自行生成未出现过的候选实参，观察同一动作在不同输入下是否产生新结果能力 |

不得新增语义结构体来判断模式。模式由任务传入参数完整度、方法形参包和值域约束共同决定。

### 5.3 输入

输入参数场景应提供或允许推导：

```text
输入条件包
  目标方法头节点
  目标自我能力值
  目标宿主
  目标特征类型
  练习方向
  执行差值
  目标执行后值
  条件结果对
  方法虚拟存在
  采样限制包
```

其中：

```text
目标特征类型 + 练习方向 + 执行差值或目标执行后值闭合
  -> 指定练习

上述参数不闭合，但目标方法可练习
  -> 探索练习
```

练习必须自己生成参数，不得要求任务替它补造随机实参。

### 5.4 实参生成

练习生成实参时只能使用：

```text
形参包
参数值类型
允许值域
历史成功实参
历史失败实参
条件结果对中的稳定区间
采样限制包
运行事实
```

建议策略：

```text
边界值生成：区间下界、上界、0、1、-1、中心值。
历史值生成：使用历史成功实参或失败边界附近值。
稳态值生成：使用条件结果对中的稳定区间。
随机生成：在允许区间内采样未出现过的参数。
收缩生成：在成功区间附近细化采样。
扩展生成：在失败边界外少量探索。
```

探索练习必须尽量避开已经出现过的参数组合。若无法生成新参数，应输出证据不足或参数空间耗尽缺口。练习生成参数属于已有动作分支内的内部补齐，不等价于生成子需求。

### 5.5 输出

输出结果场景应写入：

```text
练习状态
练习模式
候选实参表
试运行次数
结果符合次数
结果不符合次数
更新的条件结果对
新条件结果对候选
新用途候选
更新的参数区间
更新的同条件结果复现/分裂账
参数来源
缺口类型
叶子需求状态
建议下一步能力
失败原因
```

运行虚拟存在应记录：

```text
目标方法头节点
目标方法虚拟存在
条件结果对引用
候选实参表
试运行样本集合
运行事实集合
学习状态
开始时间
结束时间
```

### 5.6 边界

练习不得：

```text
替代定向学习生成全新动作。
替代尝试学习内化外部因果经验。
生成新需求节点、缺方法需求或动作节点。
在条件结果对完全缺失时从零构造条件结果对。
在方法没有可调用执行入口时伪造试运行结果。
用默认成功遮蔽参数空间耗尽或结构缺口。
深度复制抽象特征树作为实例特征。
```

方法只有头节点、缺少动作节点或缺少可调用执行入口时，练习不得接管结构生长，只能：

```text
输出方法结构自检结果。
写入执行入口或动作骨架缺失事实。
写入叶子需求状态。
写入建议下一步能力，例如尝试学习或绑定执行入口。
```

它不能生成候选动作、候选方法头或伪造试运行事实。若处在自我内部方法补齐闭环，方法查找未命中由筹办阶段创建或复用待学习目标方法头；练习只处理已经存在的方法头，在发现动作骨架、条件结果对、参数绑定等结构缺失时输出对应事实，由任务管理线程和自我线程展开成可执行性子需求。

### 5.7 处理流程

练习按以下顺序执行：

```text
1. 生成输入参数组合
   -> 读取方法形参包、条件结果对和历史运行事实。
   -> 根据参数完整度选择指定练习或随机练习。
   -> 使用历史执行数据、稳定区间和参数约束生成可尝试参数。

2. 执行已有动作
   -> 调用目标方法已有动作节点。
   -> 动作函数执行成功后产生条件结果对样本。
   -> 样本包含执行前条件状态集和执行后实际结果状态集。
   -> 执行阶段只产样本和运行账，不直接更新结果符合/不符合统计。

3. 归并条件结果对
   -> 查找或添加等价条件节点。
   -> 沉淀实际结果：添加或融合实际结果分支，样本次数 +1。
   -> 同条件下复现已有等价结果时更新结果符合次数；同条件下出现新结果分支时更新结果不符合次数。
   -> 条件不同即使结果相同也要形成新的条件分支；结果不同即使条件相同也要形成新的结果分支。
   -> 新实际结果不是任务失败，它是方法树的新事实。

4. 更新统计和实例特征
   -> 归并阶段只对有效事实样本更新样本次数、结果符合/不符合次数和条件结果对稳定性。
   -> 动作执行失败、中断、超时、参数配对失败等只写运行账或失败原因，不进入结果符合/不符合账。
   -> 更新方法虚拟存在状态。
   -> 写入实例特征更新和运行事实引用。

5. 输出练习结果
   -> 输出更新后的条件结果对。
   -> 输出新条件结果对候选和新用途候选。
   -> 输出方法虚拟存在状态。
   -> 输出实例特征更新。
```

练习失败不是终点。参数不足、条件不闭合、结果不稳定或实例特征未命中抽象区间时，应输出明确失败事实、缺口类型、叶子需求状态和建议下一步能力。是否把这些缺口事实生长为子需求，由筹办阶段或执行治理裁决；练习本身不直接创建新的需求节点。

练习发现新用途时也不是新动作生成。新用途的表达方式是：

```text
同一个动作节点
  -> 新输入参数区间
  -> 新结果特征类型
  -> 新结果方向
  -> 新条件结果对候选
```

只有经过后续练习或真实运行验证后，新条件结果对候选才能升格为稳定用途。

## 6. 为什么其他学习形式都归入三类

其他学习形式不需要新增本能函数类别，可以归约为三类学习能力的组合：

| 复杂形式 | 归约方式 |
| --- | --- |
| 参数学习 | 练习 |
| 策略学习 | 练习 + 条件结果对统计 |
| 探索内部既有动作新用途 | 练习 |
| 从内部因果样本形成候选方法 | 尝试学习 |
| 外界未归因现象方向识别 | 定向学习 |
| 内化外部经验 | 尝试学习 |
| 预测学习 | 外界链由定向学习形成方向材料；内部链由尝试学习形成候选方法，再由练习验证 |
| 强化学习 | 内部因果样本由尝试学习形成候选方法，练习试运行并沉淀稳定条件结果对 |

组合流程：

```text
内部因果样本形成候选方法
  -> 尝试学习

执行或试运行
  -> 练习

观察因果规律并内化
  -> 尝试学习

外界信息缺目标或方向
  -> 定向学习
```

因此，新增学习能力时，应优先扩展三类本能函数的输入、输出或策略，不应创造新的学习类别。

整体闭环可表达为：

```text
筹办阶段
  -> 查找方法头节点
  -> 校验方法完整性
  -> 方法查找未命中：创建或复用待学习目标方法头并待重筹办
  -> 方法头不可执行：派生动作骨架、条件结果对、参数绑定等可执行性子需求
  -> 条件结果对不稳定：派生练习需求

执行阶段
  -> 参数配对
  -> 执行本能函数
  -> 更新实例特征

练习
  -> 验证候选方法或优化已有动作条件结果对
  -> 方法树稳定或发现同一动作新用途

尝试学习
  -> 从内部因果样本抽取动作函数、条件、结果和方向
  -> 形成候选方法
  -> 回到练习验证

需求树生长
  -> 叶子任务完成
  -> 父需求回传
  -> 下一轮筹办
```

该闭环必须保持以下分离：

```text
动作缺失与条件缺失分离。
方法树成长与方法树稳定分离。
候选结构与稳定结构分离。
因果内化与动作生成分离。
```

自我内部方法补齐闭环固定为：

```text
普通需求
  -> 任务查找方法
  -> 找不到
  -> 方法补齐需求
  -> 查找能提升方法可用性的方法
  -> 练习
  -> 没有候选方法
  -> 查找能形成候选方法的方法
  -> 尝试学习
  -> 从内部因果信息形成候选方法
  -> 练习验证候选方法
  -> 方法可用
  -> 方法补齐入口满足
```

外界信息处理链单独存在：

```text
外界信息
  -> 定向学习
  -> 观察需求 / 确认需求 / 信息需求
  -> 外界方向材料
```

两条链不得混用。自我内部循环不应该触发定向学习，因为内部因果已经带着动作主体和动作函数。

## 7. 与需求树的关系

方法补齐事实和知识缺口可以生长为原子需求节点，但需求生成权责属于需求治理、任务筹办或允许生成子需求的学习本能函数。学习本能函数必须先写入结构化事实：

```text
方法查找未命中
  -> 筹办创建或复用待学习目标方法头
  -> 父任务待重筹办

待学习目标方法头缺动作骨架或条件结果对
  -> 尝试学习

已有内部因果信息未内化
  -> 尝试学习

已有动作节点的条件结果对不完整或不稳定
  -> 练习

条件结果对缺失且可由已有因果信息补齐
  -> 尝试学习

参数空间证据不足
  -> 练习写缺口特征，由治理链决定是否生成参数补齐子需求

外界信息缺少目标、方向、观察对象或确认对象
  -> 定向学习
```

需求节点对应任务，任务按两阶段推进：

```text
筹办
  -> 查找方法
  -> 选中自我能力值
  -> 方法补齐入口则生成子需求

执行
  -> 配对参数
  -> 调用学习本能函数
  -> 成功则写运行事实
  -> 失败则写知识缺口、参数缺口或执行缺失事实
  -> 缺口是否生成子需求由治理链或允许生成子需求的学习本能函数裁决
```

学习结果不能直接把父需求标记为满足。父需求是否满足，必须由需求满足检查读取实例特征、运行事实和条件结果对后确认。

## 8. 与方法树的关系

学习相关本能函数只能通过方法树和方法虚拟存在沉淀能力：

```text
方法头节点
方法虚拟存在
自我能力实例特征
方法能力包
形参包
结果包
条件结果对
试运行限制包
运行事实集合
稳定性统计
```

候选结构必须显式标记候选状态。只有经过运行事实验证，并满足稳定阈值后，才能升格为稳定条件结果对或稳定能力描述。

## 9. 特征化要求

学习状态、练习模式、缺口类型、候选状态、稳定状态、同条件结果符合次数、同条件结果不符合次数、试运行次数、参数区间等长期语义必须写入实例特征。

结构体只能作为：

```text
临时参数
消息包
局部算法缓存
返回包
```

不能把结构体字段作为长期语义权威。

推荐抽象特征：

```text
学习类别
  尝试学习 [1,1]
  定向学习 [2,2]
  练习 [3,3]

练习模式
  指定练习 [1,1]
  随机练习 [2,2]

学习结构状态
  候选 [1,1]
  试用 [2,2]
  稳定 [3,3]
  废弃 [4,4]

学习失败原因
  输入绑定失败
  执行入口缺失
  执行入口缺失
  参数空间耗尽
  条件结果对不稳定
  因果证据不足

参数来源
  任务提供 [1,1]
  自动生成 [2,2]

需求触发方式
  定向学习生成 [1,1]
  尝试学习生成 [2,2]
  治理消费缺口生成 [3,3]
  不生成新需求 [4,4]

叶子需求状态
  可执行 [1,1]
  等待缺口闭合 [2,2]
  阻塞 [3,3]

建议下一步能力
  定向学习
  尝试学习
  练习
  绑定执行入口
  补充参数证据
```

## 10. 日志与运行证据

学习相关本能函数必须使用日志模块记录关键断点：

```text
输入绑定失败
执行入口缺失
执行入口缺失
参数生成失败
试运行失败
条件结果对未更新
实例特征未命中抽象区间
候选结构升格或废弃
```

遇到结构性问题或证据不足时，可以暂停当前任务并提出问题，或写入明确缺口特征。尝试学习和定向学习可以按权责生成缺口子需求；练习只输出缺口事实，由治理链决定是否生成子需求。不得使用默认成功、白名单或兜底通过遮蔽断点。

## 11. 最终规则

```text
学习相关本能函数只有三类：
  尝试学习
  定向学习
  练习

尝试学习负责内化已知经验。
定向学习负责生成候选动作或候选条件结果对。
练习负责优化已有动作的参数、条件结果对和稳定性。

新需求只由定向学习、尝试学习或治理链基于缺口特征生成。
练习不生成新动作、新方法头或新需求节点。
生成子方法或子需求必须通过缺口特征触发，不得随机生成。

其他学习形式必须归约为这三类的策略扩展或组合流程。
```
