# 2090 任务信息自然语言表达规范

## 1. 目标

本文用于固定当前工程中：

- `任务主信息类` 如何转换为自然语言
- 任务信息在语言层应承担哪些表达职责
- 任务如何与来源需求、执行方法、目标状态、结果状态、前后任务关系结合
- 任务在“要做什么、现在做到哪、下一步是什么、服务于哪个需求并通过哪个方法执行”这几类表达中应如何落句

本文重点回答：

- 任务信息到底在语言上描述什么
- 任务应优先落成哪些句型
- 节点种类、任务状态、优先级、启动/完成时间在语言层分别有什么意义
- 任务如何作为任务树中的头节点、步骤节点、结果节点进入自然语言

本文服从以下硬边界：

1. `任务信息` 仍是真值层中的执行承接、推进阶段与结果衔接事实，不是语言层主观计划口号。
2. 自然语言只是显示层和解释层，不得反向改写任务真值。
3. 任务表达可以引用需求、方法、状态与结果，但不得把需求、方法或状态本体混同为任务本体。
4. 任务节点种类与任务状态必须保留，不能只剩一个任务名。


## 3. 核心口径

必须固定：

1. `任务信息` 描述的是：`为了满足某需求、在某场景下、准备执行或正在执行的承接动作结构`。
2. 在自然语言中，任务优先承担四类职责：
   - `要做什么`
   - `现在做到哪`
   - `下一步是什么`
   - `服务于哪个需求并通过哪个方法执行`
3. 任务不是需求本身，也不是方法本身，更不是结果状态本身。
4. 任务表达必须优先保留任务名、节点种类和任务状态。

一句话固定：

`任务信息是真值层中的执行承接与推进事实，在语言层主要回答“要做什么、现在做到哪、下一步是什么、这项任务服务于哪个需求并通过哪个方法执行”。`


## 4. 依据的字段面

当前 `任务信息基类 / 任务主信息类` 可直接提供的关键字段包括：

- `名称`
- `描述信息`
- `类型`
- `节点种类`
- `状态`
- `基准优先级`
- `局部优先级偏移`
- `调度优先级`
- `场景`
- `任务虚拟存在`
- `来源需求`
- `父任务`
- `前置任务`
- `后续任务`
- `执行方法`
- `目标状态`
- `结果状态信息`
- `动作句柄`
- `启动时间`
- `完成时间`

当前统一理解为：

- `名称 / 类型`：回答“这项任务叫什么、属于哪类任务”
- `节点种类`：回答“它是头节点、步骤节点还是结果节点”
- `状态`：回答“当前推进到哪种执行态”
- `来源需求`：回答“它服务于哪个需求”
- `执行方法`：回答“通过哪个方法执行”
- `前置任务 / 后续任务 / 父任务`：回答“它前后衔接什么任务”
- `目标状态 / 结果状态信息`：回答“想达到什么、已经得到了什么”
- `场景 / 任务虚拟存在`：回答“这项任务在哪个场景或哪个任务实体中成立”
- `调度优先级`：回答“当前优先级有多高”
- `启动时间 / 完成时间`：回答“何时开始、何时完成”


## 5. 任务信息的四类语言职责

### 5.1 要做什么

这一层回答：

- `当前要做什么`
- `这项任务是什么`

推荐句型：

- `当前要执行 T`
- `这是一项 T`
- `X 当前承担 T`

示例：

- `当前要执行安全根任务`
- `这是一项任务管理任务`


### 5.2 现在做到哪

这一层回答：

- `任务当前处于哪个执行阶段`
- `任务已经推进到头节点、步骤节点还是结果节点`

推荐句型：

- `T 当前处于 S`
- `T 当前推进到 P`
- `T 当前已到结果阶段`

示例：

- `安全根任务当前处于运行中`
- `安全根任务当前推进到治理步骤`


### 5.3 下一步是什么

这一层回答：

- `完成这一步后接什么`
- `当前任务后续衔接什么任务或结果`

推荐句型：

- `T 的下一步是 N`
- `完成 T 后将进入 N`

示例：

- `治理步骤的下一步是治理结果`
- `完成当前步骤后将进入结果节点`


### 5.4 服务于哪个需求并通过哪个方法执行

这一层回答：

- `这项任务服务于哪个需求`
- `通常通过哪个方法执行`

推荐句型：

- `T 服务于 D`
- `T 通过 M 执行`
- `为了满足 D，当前执行 T`

示例：

- `安全根任务服务于安全根需求`
- `任务管理任务通过停止当前试探并进入保守态执行`


## 6. 最小语义骨架

任务信息最小骨架固定为：

`任务名称 + 节点种类 + 任务状态 + 来源需求 + 执行方法 + 前后衔接`

其中：

- `任务名称` 回答“要做什么”
- `节点种类` 回答“处在哪一层”
- `任务状态` 回答“执行到什么状态”
- `来源需求` 回答“为什么要做”
- `执行方法` 回答“怎么做”
- `前后衔接` 回答“接下来或之前是什么”

若字段不全，可退化为：

`任务名称 + 当前状态`

但不得长期只显示孤立任务名。


## 7. 表达主链

任务信息的自然语言主链固定为：

`任务节点 -> 任务表达单元 -> 任务句型 -> 自然语言`

其中任务表达单元最少应包含：

- `任务名称`
- `任务类型`
- `节点种类文本`
- `状态文本`
- `来源需求名称`
- `执行方法名称`
- `目标状态短语`
- `结果状态短语`
- `前置任务名称`
- `后续任务名称`
- `优先级文本`
- `时间文本`

必须固定：

1. `任务节点` 不直接等于最终句子。
2. 语言模块先抽出任务表达单元，再决定用哪类任务句型。
3. 控制面板、日志、调试页统一消费语言模块结果，不各自手拼任务句。


## 8. 句型层

### 8.1 基本任务句

适用条件：

- 只需要表达“要做什么”

推荐模板：

- `当前要执行 T`
- `这是一项 T`

示例：

- `当前要执行安全根任务`
- `这是一项任务管理任务`


### 8.2 状态型任务句

适用条件：

- 需要保留任务状态

推荐模板：

- `T 当前处于 S`
- `T 目前为 S`

示例：

- `安全根任务当前处于运行中`
- `服务根任务目前为未启动`


### 8.3 节点种类型任务句

适用条件：

- 需要保留 `头结点 / 步骤节点 / 结果节点`

推荐模板：

- `T 是一个头任务`
- `T 当前是步骤节点`
- `T 当前是结果节点`

示例：

- `安全根任务是一个头任务`
- `治理步骤当前是步骤节点`
- `治理结果当前是结果节点`


### 8.4 衔接型任务句

适用条件：

- 需要表达前置、后续或父任务关系

推荐模板：

- `完成 T 后将进入 N`
- `T 依赖 P`
- `T 从属于 F`

示例：

- `完成治理步骤后将进入治理结果`
- `当前结果节点从属于治理步骤`


### 8.5 需求与方法型任务句

适用条件：

- 需要保留来源需求和执行方法

推荐模板：

- `T 服务于 D`
- `T 通过 M 执行`
- `为了满足 D，当前执行 T`

示例：

- `安全根任务服务于安全根需求`
- `任务管理任务通过读取核心状态执行`


### 8.6 目标与结果型任务句

适用条件：

- 需要保留目标状态或结果状态

推荐模板：

- `T 目标是 G`
- `T 已得到 R`
- `T 产生了 R`

示例：

- `治理步骤的目标是进入运行`
- `治理结果产生了运行中状态`


### 8.7 带优先级与时间的任务句

适用条件：

- 需要解释调度与时序

推荐模板：

- `T 当前优先级为 P`
- `T 在 T1 启动`
- `T 在 T2 完成`

示例：

- `安全根任务当前优先级为 10000`
- `治理步骤在本周期启动`


## 9. 节点种类规则

必须固定：

1. `头结点` 优先表达为任务主线入口。
2. `步骤节点` 优先表达为任务当前推进步骤。
3. `结果节点` 优先表达为任务产生的结果或阶段性产物。
4. 语言层不得把这三类节点混成同一种普通任务。

推荐理解：

- `头结点` 回答“要做什么”
- `步骤节点` 回答“做到哪一步”
- `结果节点` 回答“产出了什么结果”


## 10. 状态规则

必须固定：

1. `任务状态` 是任务表达中的一级修饰，不是任务本体。
2. 任务状态应优先落成稳定状态词，例如：
   - `未启动`
   - `运行中`
   - `挂起`
   - `完成`
   - `失败`
   - `取消`
3. 当前面板中，任务句优先保留状态。


## 11. 与需求信息、方法信息、状态信息的分工

### 11.1 与需求信息的关系

- `需求信息` 负责“为什么要做”
- `任务信息` 负责“准备做什么、现在做到哪”

一句话固定：

`需求` 说目标，`任务` 说承接。`


### 11.2 与方法信息的关系

- `任务信息` 负责“当前承接结构”
- `方法信息` 负责“具体能力与执行方式”

因此：

- 任务句不能直接冒充方法句
- 方法句可以被任务句引用，说明“通过哪个方法执行”


### 11.3 与状态信息的关系

- `任务信息` 负责“任务推进结构”
- `状态信息` 负责“任务或对象当前是什么状态”

因此：

- `任务处于运行中` 是任务句
- `目标状态为运行中` 是状态句


## 12. 推荐示例

### 12.1 基本任务

- `当前要执行安全根任务`
- `这是一项任务管理任务`


### 12.2 进度与阶段

- `安全根任务当前处于运行中`
- `安全根任务当前推进到治理步骤`
- `治理结果当前是结果节点`


### 12.3 下一步与衔接

- `完成治理步骤后将进入治理结果`
- `治理结果从属于治理步骤`


### 12.4 来源需求与执行方法

- `安全根任务服务于安全根需求`
- `任务管理任务通过停止当前试探并进入保守态执行`


## 13. 工程落点

当前工程中，这部分能力应归属于 `语言模块`，而不是散落在控制面板或各服务类中直接手拼。

最小接口建议固定为：

- `生成任务信息名称(...)`
- `生成任务信息自然语言(...)`
- `生成任务信息摘要(...)`

必须固定：

1. 控制面板只消费语言模块结果，不自行拼装任务句。
2. 同一套任务语言规则应同时服务控制面板、日志与后续调试输出。
3. 若当前只实现最小命名链，也应保证任务节点至少能落成 `T 当前处于 S` 或 `为了满足 D，当前执行 T` 这两类句型。
4. 后续任务树显示、步骤结果链显示与治理摘要，应优先复用本规范的任务表达结果。
