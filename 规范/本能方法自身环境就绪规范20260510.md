# 本能方法自身环境就绪规范 20260510

## 1. 总原则

每个本能方法在执行自己的核心逻辑之前，必须先确保“自己作为方法”已经完整存在。

完整存在不是只创建一个方法首节点，而是建立该方法自身的运行环境：

```text
1. 方法树里有自己的方法首节点。
2. 方法首节点的动作名、动作句柄、本能 ID、来源和能力签名正确。
3. 方法首节点绑定自己的方法虚拟存在。
4. 方法虚拟存在写入本方法的语义说明、长期记忆入口、规格摘要和专属状态槽。
5. 方法自身的条件节点、结果节点、条件场景和结果场景已建立。
6. 条件节点中的输入参数槽 / 匹配规则，以及结果节点中的输出结果槽 / 校验规则已建立。
7. 本轮运行虚拟存在已建立。
```

一句话：

```text
每个方法先把“自己”建完整，再去处理“目标”。
```

补充执行边界：

```text
方法首节点只负责大范围索引和候选召回。
方法能力签名只负责把结果能力、形参规格和条件规格暴露给查找、诊断和预筛选。
真正执行本能函数时，必须以方法条件节点作为参数设置、参数匹配和特征值范围判断的直接依据。
```

也就是说：

```text
方法头命中 != 本轮可执行；
能力签名完整 != 本轮参数已绑定；
条件节点匹配成功，才允许进入本能核心逻辑。
```

## 2. 修正后的四段模式

本能方法实现统一分为四段：

```text
第 1 段：确保自身方法环境
第 2 段：读取输入与绑定上下文
第 3 段：执行本能核心逻辑
第 4 段：写出结果与更新方法账
```

其中第 1 段是硬前置，不允许跳过。

```text
确保自身方法环境 =
    确保方法首节点
  + 确保方法虚拟存在
  + 确认本方法专属规格
  + 确保条件场景 / 结果场景
  + 确保本轮运行虚拟存在
```

已有方法首节点不等于自身方法环境就绪。节点存在后仍要确认：

```text
规格是否齐备；
能力签名是否齐备；
条件节点输入槽 / 结果节点输出槽是否齐备；
条件场景 / 结果场景是否齐备；
本轮运行虚拟存在是否齐备。
```

第 2 段必须遵循：

```text
1. 先读取本方法自身的条件节点。
2. 从条件节点的条件场景 / 条件集合中解析条件槽、参数槽和特征值范围。
3. 再用输入参数场景、输入条件包、任务参数包或现实场景包进行匹配。
4. 条件节点缺失、必需条件缺失、必需范围不匹配时，返回参数绑定失败或条件不足，不进入核心逻辑。
```

禁止在第 2 段直接绕过条件节点，只按输入包字段拼装执行参数。输入包只能作为“实参来源”，不能替代条件节点成为规格权威。

确认规格的精确定义：

```text
确认规格 =
    建好方法虚拟存在中的语义说明和长期记忆入口；
    建好方法条件节点中的输入参数槽和匹配规则；
    建好方法结果节点中的输出结果槽和校验规则。
```

关键结构的职责不得混用：

| 结构 | 职责 | 是否作为执行时参数依据 |
| --- | --- | --- |
| 方法首节点 | 大范围索引、方法身份、候选召回、动作名 / 动作句柄入口 | 否 |
| 方法虚拟存在 | 方法语义、长期记忆、统计、学习产物索引 | 否 |
| 条件节点 | 输入参数槽、前置条件、匹配规则、可执行判断 | 是 |
| 结果节点 | 输出结果槽、目标状态、完成判断、结果校验 | 间接 |
| 条件结果对 | 条件节点和结果节点之间的关联知识单元 | 作为筹办经验依据 |

筹办阶段可以用方法头和能力签名做候选召回与预筛选；执行阶段必须回到条件节点完成本轮输入绑定。

## 3. 通用底座与专属规格

每个本能方法共有的通用底座是：

```text
确保本能方法首节点
确保方法虚拟存在
确保本轮运行虚拟存在
更新方法运行账
```

每个本能方法不同的部分是：

```text
条件节点输入槽集合
结果节点输出槽集合
条件场景槽位
结果场景槽位
方法虚拟存在专属规格特征
方法能力签名结果包
失败原因和状态词典
```

因此，差异应集中在“确认专属规格”函数里，而不是分散在每个本能方法函数体开头。

## 4. 建议结构

建议建立统一环境结构：

```cpp
struct 结构_本能方法自身环境 {
    方法类::节点类* 方法首节点 = nullptr;
    存在节点类* 方法虚拟存在 = nullptr;
    存在节点类* 本轮运行虚拟存在 = nullptr;

    bool 方法头已就绪 = false;
    bool 方法虚拟存在已就绪 = false;
    bool 方法规格已确认 = false;

    const 语素入口节点类* 失败原因 = nullptr;

    bool 成功() const noexcept {
        return 方法首节点
            && 方法虚拟存在
            && 本轮运行虚拟存在
            && 方法头已就绪
            && 方法虚拟存在已就绪
            && 方法规格已确认;
    }
};
```

建议建立统一入口：

```cpp
template<class 规格确认函数>
结构_本能方法自身环境 确保本能方法自身环境(
    方法类::节点类* 输入方法首节点,
    枚举_本能方法ID 本能ID,
    场景节点类* 输出结果场景,
    时间戳 now,
    规格确认函数&& 确认规格) noexcept;
```

该入口只负责“当前正在执行的本能方法自己”。

环境结构应允许记录本方法的首个可用条件节点和条件匹配结果。若代码暂未把条件节点指针放入环境结构，也必须在读取输入前通过统一 helper 从方法首节点解析条件节点。

建议补充：

```cpp
方法类::节点类* 方法条件节点 = nullptr;
bool 方法条件节点已就绪 = false;
```

## 5. 方法自身环境与目标方法环境必须分开

学习 / 练习类方法通常同时涉及两套环境：

```text
A. 当前正在执行的本能方法自身环境
B. 被学习 / 被补齐 / 被练习的目标方法环境
```

例如“尝试学习”：

```text
A = 自我_尝试学习
    方法首节点
    方法虚拟存在
    自己的输入输出规格
    本轮运行虚拟存在

B = 目标方法
    目标方法头节点
    目标方法虚拟存在
    条件结果对
    方法条件节点
    方法结果节点
```

禁止把两者混为一谈：

```text
不得把目标方法虚拟存在当作当前学习本能方法自己的虚拟存在；
不得把当前学习本能方法的条件 / 结果场景当作目标方法的条件 / 结果场景；
不得在确保自身环境阶段修改目标方法结构，目标方法结构补齐属于核心逻辑阶段或输出阶段。
```

## 6. 两类接口

### 6.1 确保方法本体

```cpp
方法类::节点类* 确保本能方法本体(
    方法类::节点类* 方法首节点,
    枚举_本能方法ID 本能ID,
    时间戳 now);
```

职责：

```text
1. 方法树中有这个本能方法的首节点。
2. 动作名、动作句柄、本能 ID、来源等基础字段正确。
3. 方法虚拟存在存在。
4. 通用方法类型、来源、成熟度等特征存在。
5. 基础能力签名存在。
```

### 6.2 确认方法专属规格

```cpp
bool 确认本能方法专属规格(
    枚举_本能方法ID 本能ID,
    方法类::节点类* 方法首节点,
    存在节点类* 方法虚拟存在,
    时间戳 now);
```

职责：

```text
1. 在方法虚拟存在中建立语义说明、规格摘要、长期记忆入口和专属状态槽。
2. 在条件节点 / 条件场景中建立输入参数槽、必需标记、匹配规则和特征值范围。
3. 在结果节点 / 结果场景中建立输出结果槽、目标状态和结果校验规则。
4. 将条件 / 形参 / 结果规格同步到能力签名，供查找、预筛选和诊断使用。
5. 写入方法动作类型、学习阶段、补齐类别等专属规格。
6. 确保必要状态词、结果词、失败原因词已注册。
```

内部分发：

```cpp
switch (本能ID) {
case 枚举_本能方法ID::自我_尝试学习:
    return 确认尝试学习规格(...);
case 枚举_本能方法ID::自我_定向学习:
    return 确认定向学习规格(...);
case 枚举_本能方法ID::自我_练习:
    return 确认练习规格(...);
default:
    return 确认通用本能方法规格(...);
}
```

## 8. 失败处理

自身方法环境确保失败属于非正常逻辑错误或关键结构缺失，不允许静默兜底。

处理规则：

```text
1. 记录日志。
2. 弹错 / 报错。
3. 返回失败回执或空运行虚拟存在。
4. 不创建泛化影子方法。
5. 不把规格缺失伪装成目标方法缺口。
```

业务缺口仍然可以进入状况回执、派生需求或学习承接；但方法自身环境缺失不是业务缺口，而是方法系统自举失败。

条件匹配失败的归类：

```text
自身方法环境缺失：
    非正常逻辑错误，必须报错和记录日志。

方法条件节点缺失：
    本方法规格未就绪。若该方法已被召回执行，应记录错误并返回条件不足。

必需条件参数缺失：
    当前输入不足，返回参数绑定失败 / 条件不足。

必需特征值范围不匹配：
    当前方法在本轮上下文不可用，返回参数绑定失败 / 条件不满足。
```

这四类不得混成“目标方法缺口”，也不得静默兜底成执行成功。

## 9. 禁止项

新增禁止项：

```text
1. 禁止把方法首节点当作执行参数依据。
2. 禁止仅凭能力签名直接执行本能核心逻辑。
3. 禁止在存在方法条件节点时绕过条件节点直接读取输入包执行。
4. 禁止将条件范围读取长期藏在学习函数内部，执行函数必须能通过公共 helper 读取条件节点范围。
```

```text
1. 禁止只因方法首节点存在就跳过规格确认。
2. 禁止在读取输入前未创建本轮运行虚拟存在。
3. 禁止把目标方法环境当作自身方法环境。
4. 禁止把任务筹办、任务执行、自我线程主循环、任务界面线程承接注册成本能方法。
5. 禁止用方法名 / 动作名 / 显示文本判断语义，能力匹配仍以特征类型、状态和能力签名为准。
6. 禁止用静默兜底替代规格失败报错。
```

## 10. 最终定稿

```text
本能方法执行前必须先确保自身方法环境。

通用部分：
    方法首节点、动作句柄、方法虚拟存在、条件节点、结果节点、条件场景、结果场景、本轮运行虚拟存在。

差异部分：
    每个本能方法自己的条件输入槽、结果输出槽、匹配 / 校验规则、能力签名摘要和专属状态词典。

通用部分由确保本能方法本体处理；
差异部分由确认XXX规格处理；
核心逻辑只在自身环境就绪且条件节点完成本轮输入绑定后执行。
```
