# 规范一致性检查与修订说明 20260509

## 1. 检查结论

两份规范的大方向是对的：

```text
1. 存在作为特征值时，使用 特征值::指针句柄。
2. 不新增 当前值信息。
3. 不新增 存在形态。
4. 不新增 抽象存在主信息类。
5. 概念存在只作为自然语言入口和实例创建模板。
6. 实例可以对应多个概念存在。
```

但原文存在一个需要立即修正的冲突：

```text
《存在作为特征值指针句柄规范》末尾写：
    概念归属也是特征，不是存在字段。

《存在概念与实例创建最小闭环规范》正文写：
    概念集 是实例概念归属的第一阶段权威结构。
```

这两个口径不能同时作为权威。

修订后的统一口径：

```text
概念归属第一阶段以 存在节点主信息类::概念集 为权威。
概念特征节点可以作为镜像 / 索引 / 查询辅助。
普通存在关系仍然可以用 特征值::指针句柄 表达。
```

## 2. 已修正的问题

### 2.1 概念归属权威冲突

原问题：

```text
一处说 概念集 是权威。
一处说 概念归属不是存在字段。
```

修正：

```text
概念集 是权威。
概念特征节点只是镜像。
```

### 2.2 指针句柄指向对象不够明确

原文只说指针指向存在或基础信息节点。

修订后明确：

```text
指针句柄必须指向 基础信息节点类* 或其别名节点类型。
不得指向 基础信息基类* 主信息对象。
不得指向栈上临时对象。
不得指向外部无生命周期保障对象。
```

### 2.3 工具函数 const 语义不清

原工具函数接收 `const 基础信息节点类*`，但模板读取返回 `T节点*`，容易出现 const 语义混乱。

修订后改为：

```cpp
inline 特征值 构造指针特征值(基础信息节点类* 节点) noexcept;
inline 基础信息节点类* 取特征值基础信息节点(const 特征值& 值) noexcept;
template<class T节点>
inline T节点* 取特征值节点(const 特征值& 值) noexcept;
```

### 2.4 概念唯一性与实例多实例性未写清

用户已确认：

```text
同名实例可以有很多个。
模板 / 概念通常只能有一个。
```

修订后增加：

```text
同一 存在概念根 下，名称 + 类型 相同的概念存在只保留一个。
实例存在允许同名多实例，创建实例默认不按名称复用。
```

### 2.5 概念扩充复制规则过于粗略

原文只写“复制 / 实例化概念特征”。

修订后补充：

```text
1. 实例已有特征优先。
2. 多概念重复特征按加入顺序补缺。
3. 只补缺，不覆盖。
4. 第一阶段可只复制特征类型，不复制指针类当前值。
5. 运行期观测痕迹不从概念复制到实例。
```

## 3. 建议采用的最终口径

```text
存在只有一种结构：存在节点主信息类。
概念和实例靠所在根树区分，不靠枚举区分。
概念存在 = 词义锚点 + 实例创建模板。
实例存在 = 场景 / 世界 / 内部世界中的具体对象。
实例可以对应多个概念。
概念集 是第一阶段概念归属权威结构。
特征节点可以镜像概念归属，但不替代 概念集。
存在作为特征值时，使用 特征值::指针句柄。
```
