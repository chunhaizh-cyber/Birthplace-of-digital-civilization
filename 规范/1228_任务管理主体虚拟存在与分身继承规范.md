# 1228 任务管理主体虚拟存在与分身继承规范

## 1. 目标

本文用于把 `任务管理主体`、`分身继承`、`方法需求位桥层` 与 `本能能力缺口` 正式补成规范层，回答以下问题：

- `任务管理任务` 之上是否允许存在一个跨宿主成长的主体承载
- “很多分身” 应如何解释，才不会撞上 `每宿主最多一个任务管理实例`
- `理解得到的高阶特征 -> 可见缺口/可见可能 -> 方法需求位 -> 本能能力缺口 -> 最小动作原语` 这条桥如何正式落层
- 主体层、运行时三层、主循环和学习反馈之间应如何接平

本文不替代 [1219_任务管理实例堆与恢复快照规范.md](D:\鱼巢\规范\1219_任务管理实例堆与恢复快照规范.md) 的运行承载三层，本文只补“谁在成长、怎样继承、怎样回并”这条主体主线。

## 3. 核心结论

必须固定以下 12 条：

1. `任务管理任务` 仍然是任务，不是系统外的超级管理员。
2. `任务管理主体虚拟存在` 是一个非运行时的主体级汇总载体，用于汇总跨宿主、跨候选路径、跨学习反馈链的成长特征。
3. `任务管理主体虚拟存在` 不持有推进权，不替代主循环，不替代宿主任务虚拟存在，也不替代恢复快照。
4. “很多分身” 只能解释为 `主体级逻辑分身`，不能解释为“同一宿主上并发跑多个治理实例”。
5. `每个宿主任务最多一个任务管理运行时实例`、`同一宿主同一时刻只有一个实例持有推进权` 这两条运行时纪律继续成立。
6. `分身继承面` 负责承载某条候选治理链、本轮学习反馈链或跨宿主承接链的局部继承与局部增量。
7. 高阶特征显现出来的可见缺口或可见可能，不能直接等价成具体方法，必须先压缩成 `方法需求位`。
8. `方法需求位` 命中后，仍应继续下钻出 `本能能力缺口`，判断当前缺的是方法枝本身，还是支撑该位落地所需的最小动作原语。
9. `任务管理方法因果` 必须晚于 `状态镜像完成 + 动作动态完成`，再回并到主体层作为成长证据。
10. `任务完成` 是任务管理主体的局部运行目标，`安全值 / 服务值` 仍然是上位方向约束。
11. 若需要做主体级的全局宿主筛选、压力重排或巡视治理，必须由主循环显式提交 `全局巡视治理请求` 或等价请求，不能在普通单宿主治理轮里无边界盲扫。
12. 主体层的最终作用，是把“谁在成长、怎样继承、怎样回并”从宿主运行时镜像中独立出来，而不是重写运行时纪律。

一句话收口：

`运行承载三层回答“每宿主的一轮治理怎样跑”，主体层回答“跨多宿主、多路径、多次学习反馈之后，到底是谁在成长”。`


## 4. 与运行承载三层的边界

按 [1219_任务管理实例堆与恢复快照规范.md](D:\鱼巢\规范\1219_任务管理实例堆与恢复快照规范.md)，运行承载三层继续固定为：

| 层 | 作用 |
| --- | --- |
| `任务管理实例堆` | 跑当前宿主的一轮运行时实例 |
| `宿主任务虚拟存在` | 记当前宿主的治理状态面 |
| `恢复快照` | 在退出或恢复时续当前宿主实例 |

本文新增的是 `主体级汇总层`，但必须固定它不是第四个运行时层：

| 层 | 作用 | 不能做什么 |
| --- | --- | --- |
| `任务管理主体虚拟存在` | 汇总跨宿主、跨分身、跨学习反馈的稳定成长特征 | 不持有推进权，不保存线程态 |
| `分身继承面` | 承接某条候选治理链的局部继承、本地增量与回并策略 | 不替代实例队列，不抢宿主推进权 |
| `方法需求位特征面` | 显式表达当前缺的是哪类治理能力位 | 不直接等价于具体方法枝 |
| `本能能力缺口面` | 显式表达该能力位仍缺哪些最小动作原语 | 不替代高层治理目标 |


## 5. 四个正式承载面

### 5.1 任务管理主体虚拟存在

它至少应稳定承载：

- `主体ID`
- `主体版本`
- `局部运行目标`
- `安全服务取向`
- `预算策略`
- `风险偏好`
- `稳定推进次数`
- `完成次数`
- `失败次数`
- `方法位覆盖位图`
- `最近高价值证据摘要`
- `最近高置信轻量因果链摘要`
- `最近融合来源摘要`
- `最近汇总时间`

### 5.2 分身继承面

它至少应稳定承载：

- `分身ID`
- `主体ID`
- `宿主任务指针`
- `宿主任务虚拟存在引用`
- `方法虚拟存在引用`
- `来源学习反馈标识`
- `继承宿主特征位图`
- `继承方法特征位图`
- `本地二次特征摘要`
- `本地增量摘要`
- `本地最近动作动态锚点`
- `本地最近实例因果锚点`
- `回并策略`
- `当前状态`

### 5.3 方法需求位特征面

最小字段至少包括：

- `当前方法需求位`
- `来源功能域`
- `来源缺口类型`
- `来源下一步去向`
- `输入条件主键`
- `治理态型`
- `结构完备度`
- `方法可用度`
- `风险门控`
- `当前准备度`
- `候选方法摘要`
- `是否允许出生候选枝`
- `是否建议学习扩选`

### 5.4 本能能力缺口面

最小字段至少包括：

- `所属方法需求位`
- `来源本能能力缺口类型`
- `所需最小动作原语集合`
- `已具备原语位图`
- `缺失原语位图`
- `来源主观察特征`
- `动作边界要求`
- `动作后回读要求`
- `是否需学习补齐`
- `学习优先级`
- `服务到哪个方法位/方法枝`


## 6. 主体层的正式桥链

后续统一按以下桥链理解：

```text
主体级成长压力
-> 分身继承宿主与方法特征
-> 形成治理特征面
-> 命中当前方法需求位
-> 展开本能能力缺口
-> 映射到最小动作原语
-> 完成单步治理
-> 先写状态镜像
-> 再记动作动态
-> 再生实例因果与轻量因果链
-> 回并到主体
```

必须固定：

1. `方法需求位` 是成长需求和具体方法枝之间的正式桥层。
2. `本能能力缺口` 是方法需求位和最小动作原语之间的正式桥层。
3. `状态镜像 -> 动作动态 -> 实例因果 -> 轻量因果链 -> 主体回并` 的顺序不得倒置。


## 7. 与主循环、学习反馈和结果返回的关系

### 7.1 与主循环的关系

主循环仍然负责：

- 根层重判
- 双值结算
- 需求治理
- 是否提交普通宿主治理请求
- 是否提交 `全局巡视治理请求`
- 最终继续/待机/停止/收束决定

主体层不能替代这些职责。

### 7.2 与学习反馈的关系

当命中的 `方法需求位` 已明确，但其 `本能能力缺口` 仍未补齐时：

- 允许把该缺口显式转成学习承接输入
- 学习结果可以补 `方法枝`，也可以补 `最小动作原语 / 治理本能组合`
- 反馈被消费时必须能回指 `来源方法需求位`、`来源本能能力缺口类型`、`来源主观察特征` 与 `服务目标方法位/方法枝`

### 7.3 与一步治理结果的关系

`一步治理结果` 允许挂带以下主体层摘要：

- `主体ID`
- `分身ID`
- `主体回并摘要`
- `当前方法需求位`
- `当前本能能力缺口类型`
- `输入条件主键`
- `治理态型`

这些字段只用于帮助外层继续承接和回并，不得越权冒充最终裁决。


## 8. 最终结论

后续统一按以下判断执行：

1. `任务管理主体虚拟存在` 是正式规范层，而不是临时实现缓存。
2. `很多分身` 应理解为主体级逻辑分身，而不是宿主级并发治理实例。
3. `每宿主一实例` 的运行时纪律继续保留，不因主体层引入而被推翻。
4. `方法需求位` 与 `本能能力缺口` 必须补成正式桥层。
5. 主体层最终服务的是成长汇总与学习反馈，不是替代主循环做根层裁决。

一句话收口：

`真正需要补的不是“谁来治理”，而是“治理承接者之上有没有成长主体、主体怎样分身、分身怎样继承、特征空间怎样长出方法需求位、方法需求位又怎样继续落到最小本能原语”。`
