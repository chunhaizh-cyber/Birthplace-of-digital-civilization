## 2. 核心术语体系

### 2.1 根层与方向术语

#### 2.1.1 自我

`自我` 指的是：

- 在根约束层之下持续运行的统一主体。
- 负责记录状态、生成需求、组织任务、筛选方法、回收结果、承接学习的总承载者。
- 由 `需求 / 任务 / 方法 / 学习` 形成持续封闭循环的统一主语义对象。

扩写说明：

- `自我` 不是一句自然语言称呼，而是整个系统内部“谁在持续存在、持续比较、持续决策”的统一主体定义。
- `自我` 不是单一线程，也不是单一节点，而是语义主体、运行时主体与世界树锚点共同组成的统一对象。
- `自我` 不是静态容器，而是持续围绕“缺口 -> 推进 -> 反馈 -> 学习 -> 再判断”运行的闭环主体。

代码映射：

- 运行时主入口当前主要承载在 `自我类.ixx` 的 `自我类`。
- 世界中的结构锚点当前主要承载在 `世界树类.ixx` 的 `自我指针` 与相关“取或创建自我存在/场景”接口。
- 因此当前更准确的说法是：`自我` 在代码上由 `自我类` 运行时和 `世界树` 中的自我存在节点共同承载。

#### 2.1.2 根目的

`根目的` 当前固定为：

- `满足人类需求 / 提升服务成立程度`

扩写说明：

- 根目的属于根约束层，不是学习结果，不是概念归纳结果，也不是任务层临时选出来的偏好。
- 所有需求、任务、方法、学习、概念命名与工程改造，最终都必须回投到这个根目的。

代码映射：

- 当前代码里不存在一个“可随时改写的根目的变量”，这本身就是正确的。
- 根目的当前主要通过 `服务值`、`安全值`、根需求生成方向、任务组织方向间接落地。
- 因此代码层应理解为：`根目的不作为可学习字段存在，而作为所有相关字段与流程的上位约束存在。`

#### 2.1.3 服务

`服务` 指的是：

- `对满足人类需求具有正向净贡献的结果`

它包括：

- `直接服务`
- `准备性服务`

扩写说明：

- 直接服务是直接减少人类需求缺口的结果。
- 准备性服务是尚未直接交付最终结果，但能真实提升后续直接服务成功率、稳定性或持续性的结果。
- 只有“正向净贡献”可以计入服务，不能因为“发生了动作”就把动作本身算成服务。

代码映射：

- 当前与服务值直接相关的运行时入口主要在 `自我类.ixx`，例如 `获取服务值`、服务值读写与衰减相关逻辑。
- 自我服务相关特征当前落在世界树的自我存在特征上，而不是散落成无主变量。

#### 2.1.4 安全

`安全` 指的是：

- `维持持续服务能力的保障条件`

扩写说明：

- 安全非常重要，但安全不是高于服务的独立根目的。
- 安全之所以高优先级，是因为没有安全保障，持续服务能力会崩塌。
- 当前必须固定：`安全值 == 0` 才表示生死边界成立。

代码映射：

- 当前与安全值直接相关的运行时入口主要在 `自我类.ixx`，例如 `获取安全值`、`是否已消亡`、安全值回归与更新逻辑。
- 当前实现还细分了 `物理安全`、`风险安全` 等承载点，它们属于安全保障的展开，不高于总安全语义。

#### 2.1.5 权重

`权重` 指的是：

- 在父级方向许可范围内，对先后次序、资源分配、局部优先级进行调节的量。

扩写说明：

- 权重只能调权，不能改向。
- 权重可以让某条路径“先做、少做、多做、暂缓做”，但不能把父级允许的方向改成相反方向。
- 任何把权重当成“目标改写器”的做法，都属于语义越权。

代码映射：

- 权重的使用会落到需求、任务、调度等模块中，但本文只固定其语义边界，不固定具体存放位置。
- 若任何实现把权重直接用于反向改写根方向，应视为违规实现。

#### 2.1.6 根方向

`根方向` 指的是：

- 根目的在当前状态下形成的可持续差值缩减方向。

它至少包括：

- `主方向`
- `保障方向`

扩写说明：

- 主方向回答“最终要更好地服务到哪里”。
- 保障方向回答“是否还能继续服务下去”。
- 根方向不是标签词，也不是口号，而是通过目标值、当前值和边界条件计算出来的结构化方向。

代码映射：

- 当前方向语义的运行时承载，主要依赖 `自我类.ixx` 中的服务值/安全值状态、方向相关概念记录入口，以及需求任务生成模块。
- 当前代码里的“基础方向概念记录”只能视为方向外显入口，不能替代方向本体的根层定义。

#### 2.1.7 持续逼近型需求

`持续逼近型需求` 指的是：

- 有明确目标值。
- 有明确当前值。
- 在动态环境中只能持续逼近，而通常无法永久终结完成的需求。

扩写说明：

- 根需求必须按持续逼近型需求理解，而不能按“一次完成就永远结束”的终结型需求理解。
- 阶段性子需求可以完成，但它们的完成只表示对根需求的局部逼近。

代码映射：

- 该术语的上位语义由 [0400_持续逼近型需求规则.md](D:\鱼巢\规范\0400_持续逼近型需求规则.md) 固定。
- 当前任务和需求模块若出现“把根需求直接建模成永久完成态”的实现，应视为与本文术语冲突。

### 2.2 差值、二次特征与绝对方向

#### 2.2.1 当前值

`当前值` 指的是：

- 某个可度量对象在当前时刻的实际取值。

例如：

- 当前服务值
- 当前安全值
- 当前需求值

#### 2.2.2 目标值

`目标值` 指的是：

- 某个对象在当前评价体系下应被逼近的目标上界、目标线或目标区间中心。

#### 2.2.3 差值

对带目标值的对象，当前规范统一采用：

- `差值 = 目标值 - 当前值`

扩写说明：

- `目标值` 与 `当前值` 都应落在可直接比较的一次特征值上。
- 需求、结果、返回值若不能落成带值特征，就不能稳定进入这一层比较。
- 差值不是自然语言评价，而是结构化量。
- 差值必须可比较、可分区、可进入决策。
- 差值表达的是“还差多少、偏到哪边、是否已经对齐”，而不是“喜欢/不喜欢”。

代码映射：

- 当前通用比较层在 `主信息定义模块.ixx` 和相关比较实现里，经常使用 `左值 - 右值` 作为中性比较载体。
- 这与本文的 `目标值 - 当前值` 不冲突，但必须满足一个映射规则：
  - 当比较语义是“目标差值”时，必须显式约定 `左 = 目标`、`右 = 当前`。
  - 不允许把 `左值 - 右值` 的内部实现习惯，直接偷换成根层差值语义。

#### 2.2.4 二次特征

`二次特征` 指的是：

- 由状态、动态或数值比较后得到的结构化关系结果。

其中最重要的一类是：

- `方向差值型二次特征`

它专门表达：

- 当前值与目标值比较后得到的差值关系。
- 差值所落入的区间。
- 区间对应的方向含义。

扩写说明：

- 二次特征不是原始事实，而是“对事实做过比较和规约之后”的结果。
- 原始状态、动态、数值是输入；二次特征是比较输出。
- 后续很多概念、因果、决策，并不是直接基于原始值，而是基于二次特征组织起来的。

代码映射：

- 二次特征主承载类当前在 `主信息定义模块.ixx` 的 `二次特征主信息类`。
- 当前实现已明确区分 `schema / condition / state` 三种形态，这为“概念模板、条件引用、结果落地”提供了承载基础。
- `动态类.ixx` 当前承担了运行时动态缓存、动作动态创建、状态记录后刷新动态等过程层承载，是“原始过程事实”进入比较与归纳前的重要入口。
- `世界树类.ixx` 当前承担了大量“比较并生成二次特征 state”的编排职责。

#### 2.2.5 正区间、负区间、零区间

方向差值型二次特征必须落入以下三个区间之一：

1. `正区间`
   `差值 > 0`，表示当前值仍低于目标值，方向表现为继续逼近。

2. `负区间`
   `差值 < 0`，表示当前值已越过目标值，或出现反向偏移，方向表现为回调、纠偏或抑制。

3. `零区间`
   `差值 = 0`，表示当前值与目标值重合，属于绝对对齐。

必须固定：

- 正区间与负区间互斥。
- 正区间与负区间都具有方向性。
- 零区间不是“无信息”，而是“已绝对对齐”。
- 方向判断只允许落入这三个固定区间，不允许在根层再发明第四种方向语义。

#### 2.2.6 绝对方向

`绝对方向` 指的是：

- `某一比较值所落入的区间`

也就是说：

- 落入正区间，则该维度当前方向是继续逼近。
- 落入负区间，则该维度当前方向是回调或纠偏。
- 落入零区间，则该维度当前方向是绝对对齐。

扩写说明：

- 绝对方向是区间归属，不是语言词感。
- 绝对方向必须能回溯到差值，而不能只回溯到一句话解释。

#### 2.2.7 多维方向裁决

当同时存在服务、安全、需求等多个维度时，必须：

1. 先分别计算各维度差值。
2. 再分别判断各维度区间与绝对方向。
3. 最后由根目的与安全边界裁决当前总方向。

必须固定：

- 不能跳过分维计算，直接凭模糊语言给出“总体上好像该这样”。
- 不能让某个局部维度未经裁决就偷换成总方向。
- 不同特征类型上的方向，不能只凭名称相似或同时变化就直接判为同向。
- 只有通过合法因果，才允许把一个特征类型上的方向投影为另一个特征类型的方向支撑。

代码映射：

- `世界树类.ixx` 当前在状态比较和条件影响分析中，已经显式产出 `值变化量`、`值变化方向`、`时间窗` 等结构化结果。
- `世界树类.ixx` 的 `尝试计算状态二次特征`、`尝试分析条件影响结果`，是当前“状态/动态 -> 二次特征 -> 方向摘要”链条的重要承载点。
- `因果信息类.ixx` 当前已经出现 `典型条件方向摘要`、`典型结果方向摘要` 之类方向摘要字段，这些字段只能建立在前面的结构化比较结果之上，不能脱离差值语义单独漂浮。

#### 2.2.8 本组实现约束

后续实现中，凡是涉及目标差值的模块，都必须遵守：

1. 根层差值语义统一是 `目标值 - 当前值`。
2. 通用比较层若使用 `左值 - 右值`，必须在模块边界显式注明左右语义。
3. 任何模块不得私自把正区间/负区间的含义反转后继续上送。
4. 若某模块内部使用相反符号约定，必须在输出前完成映射。
5. 任何 `任务 / 方法 / 学习候选` 进入执行前，必须先完成方向一致性判断；只有同向或维持零区间对齐的候选，才允许执行。
6. 若候选动作直接作用于另一种特征类型，则必须先证明其方向能够通过合法因果回投到目标方向，才允许执行。
7. 任务执行阶段若同时存在 `需求值 / 当前值 / 返回值`，则应分别计算：
   - `需求方向 = sign(需求值 - 当前值)`
   - `结果方向 = sign(返回值 - 当前值)`
   再判断两者是否一致。
8. 一次特征、原始状态、原始数值、方法返回值都只是真值输入；真正的 `是否同向 / 是否成立 / 是否继续`，只能由比较后形成的 `二次特征` 裁决。
9. 若某模块跳过比较过程，直接拿一次特征或原始返回值当最终判定依据，应视为违规实现。
10. `一次特征 / 显式键 / 条件判定结果 / 二次特征 / 最终裁决` 的统一分层与允许流向，按 [0565_判定依据分层与裁决顺序规范.md](D:\鱼巢\规范\0565_判定依据分层与裁决顺序规范.md) 执行。

### 2.3 认知与概念术语

#### 2.3.1 基础信息节点

`基础信息节点` 指的是：

- 能长期参与比较、归纳、复用的最小信息单元。

它可以来自：

- 状态事实
- 动态事实
- 二次特征
- 因果信息
- 时间主键或时间摘要字段
- 条件主键
- 根目的投影结果

扩写说明：

- 这里的“基础”不是指“永远不变”，而是指它已经足够独立、可被其它结构复用。
- 基础信息节点是后续概念、因果、证据链的最小积木，而不是最终概念本身。
- `时间主键 / 条件主键 / 路径主键 / 结果主键 / 各类摘要字段` 这类结构字段，在本体系中统一视为 `结构化辅助定位信息`。
- 它们可以服务于 `归类 / 索引 / 去重 / 证据聚合 / 身份定位`，但不能直接充当 `语义真值`、`方向真值` 或 `需求是否满足` 的最终判定依据。
- 若需要判断 `是否成立 / 是否继续 / 是否满足 / 是否同向`，仍必须回到一次特征比较，并由比较后形成的 `二次特征` 裁决。

代码映射：

- 当前基础信息族主承载在 `主信息定义模块.ixx` 的 `基础信息基类` 及其派生结构。
- 当前节点族已明确区分 `存在节点类 / 场景节点类 / 状态节点类 / 动态节点类 / 二次特征节点类 / 因果节点类` 等承载入口。
- 这意味着当前实现已经具备“从基础节点出发构造更高层认知”的结构条件。

#### 2.3.2 概念

`概念` 指的是：

- `由稳定基础信息节点按关系组织而成的结构化认知对象`

概念至少具备：

- `稳定性`
- `结构性`
- `可复用性`
- `可表述性`

扩写说明：

- 概念不是词，而是结构。
- 名称只是概念的外显锚点，不是概念成立的真值来源。
- 概念必须能回溯到基础信息节点与证据链，不能是黑箱标签。

代码映射：

- 当前概念域、概念节点和概念关系的基础承载在 `概念树类.ixx`。
- 当前候选概念账本、证据次数、反例次数、稳定状态、决策参与级别等成长过程承载在 `概念引擎类.ixx`。
- 这说明当前代码已经具备“候选 -> 稳定 -> 落锚”的基本骨架。

#### 2.3.3 状态事实

`状态事实` 指的是：

- 某主体、某特征或某对象在某时刻的一次具体状态记录。

扩写说明：

- 状态事实是运行时事实，不是抽象概念。
- 状态事实可以作为概念证据，但不能直接等同于概念。

代码映射：

- 当前状态事实主承载在 `主信息定义模块.ixx` 的 `状态节点主信息类`。
- 状态比较、状态二次特征生成当前主要由 `世界树类.ixx` 编排。

#### 2.3.4 动态事实

`动态事实` 指的是：

- 某主体、某对象或某动作在一段时间中的过程性变化记录。

扩写说明：

- 状态更偏“时刻切片”，动态更偏“过程展开”。
- 动态事实常常是提取因果和方向摘要的重要中间层。
- 动作执行事实与状态变化事实必须区分：动作发生了，不等于一定发生了状态变化。
- 只要动作确实执行，就应允许存在动作事实动态；但新的变化方向、变化因果，仍以前后状态确实不同为前提。

代码映射：

- 当前动态事实主承载在 `主信息定义模块.ixx` 的 `动态节点主信息类`。
- `动态类.ixx` 的 `动态类` 当前负责动态树根、运行时动态缓存、动作动态和状态刷新后的动态聚合，是动态事实的直接管理入口。
- 自我动作动态当前可通过 `自我类.ixx` 中的 `记录自我动作动态` 等入口进入结构层。

#### 2.3.5 特征状态

`特征状态` 指的是：

- 从具体存在上剥离出来、对单一特征值或特征值区间的抽象描述。

例如：

- 颜色值区间可抽象为 `红色`
- 亮度区间可抽象为 `明亮`
- 力量区间可抽象为 `强`

必须固定：

- 特征状态描述的是 `特征值如何被抽象`
- 特征状态不直接等于某个具体存在
- 某个存在可以命中特征状态，但特征状态本身不属于该存在

#### 2.3.6 状态概念树

`状态概念树` 指的是：

- 从特征类型作为最抽象根开始生长的分层树。

其生成原则固定为：

1. 每个特征类型先作为最抽象的状态概念根。
2. 再按稳定区间逐层细分。
3. 直到当前不再需要细分，或无法稳定细分为止。

扩写说明：

- 状态概念树不是从某个具体存在长出来的。
- 具体存在只负责命中树上节点，不负责定义树根。
- 当前叶子节点应理解为“当前已细分到的最细稳定状态概念”，而不是永恒终叶。

代码映射：

- 当前概念域基础结构在 `概念树类.ixx` 中已经具备。
- 当前二次特征概念模板与状态命中关系，在 `二次特征主信息类` 的 `概念模板`、`概念名称` 等字段上已经有承载基础。
- 当前“专用的状态概念树管理器”并未完全独立成单模块，因此状态概念树的实现承载目前仍是“概念树域 + 二次特征模板 + 证据归纳”的组合落点。

#### 2.3.7 复合状态概念

`复合状态概念` 指的是：

- 由多个特征状态或状态概念，经跨存在重复观察后形成的聚合状态概念。

例如：

- 健康
- 病态

必须固定：

- 复合状态概念仍属于状态概念域。
- 不同存在只提供证据，不构成概念身份本身。
- 只有稳定交集在观察期内可复现，才允许升为正式概念。

#### 2.3.8 复合特征

`复合特征` 指的是：

- 由多个特征状态按一定组合关系形成的更高层特征概念。

例如：

- 美丽
- 强壮

必须固定：

- 复合特征仍属于特征组合。
- 复合特征不属于存在概念。
- 某个存在可以命中某个复合特征，但复合特征本身依然是抽象特征概念。

#### 2.3.9 概念候选

`概念候选` 指的是：

- 值得继续观察、但尚未稳定成立的关系假设。

#### 2.3.10 稳定概念

`稳定概念` 指的是：

- 已有重复证据。
- 已具备较稳定方向。
- 已能作为后续判断依据使用的概念。

#### 2.3.11 概念与自然语言

必须固定：

- 概念不等于自然语言词面。
- 自然语言只是概念的外显描述方式。
- 概念候选可以暂时无名。
- 稳定概念应至少具有一个主名称。

扩写说明：

- 语言负责表达概念，不负责制造概念真值。
- 词面相同不代表概念相同，词面不同也不代表概念不同。

#### 2.3.12 稳定概念的命名要求

必须固定：

- 所有稳定概念都需要命名。
- 命名不是概念成立的真值来源，但属于稳定概念的必要外显工作。
- 命名任务可以后置，但不能被取消为“永不命名”。

#### 2.3.13 私有概念、全局概念、公用概念

`私有概念` 指的是：

- 只在局部任务、局部场景或局部对象范围内稳定成立的概念。

`全局概念` 指的是：

- 在当前自我内部多个任务、多个场景、多个模块中都稳定成立的概念。

`公用概念` 指的是：

- 不只对当前自我成立，还具有稳定跨主体、跨模块或对外复用价值的概念。

#### 2.3.14 本组硬约束

概念生成必须遵守：

- [0600_概念自动生成判据.md](D:\鱼巢\规范\0600_概念自动生成判据.md)

并且必须固定：

- 所有正式概念都必须可追溯到基础信息节点。
- 所有正式概念都必须可追溯到证据链，而不能只追溯到词面。
- 一切不可回溯、不可解释、不可复核的黑箱标签，都不得升为正式概念。

### 2.4 因果、证据与时间术语

在本组术语中，必须先固定一条总规则：

- `因果` 是不同特征类型之间方向能够合法链接、传递与对齐的唯一准则。
- 若没有因果，不同特征类型之间即使同时变化，也只能视为并列观察，不能直接互相替代。

并且必须固定：

- `先有变化，才可能有新的方向判断。`
- 没有变化，就没有新的变化状态，也没有由变化直接生成的新二次特征方向。
- 但“没有变化”不等于“动作没有发生”；动作事实动态仍可独立存在。

#### 2.4.1 锚点容器

`锚点容器` 指的是：

- 单条因果成立时所依附的观察边界。

当前必须固定：

- 锚点容器只允许为 `场景` 或 `存在`。

#### 2.4.2 状态层级

`状态层级` 指的是：

- `观察对象的宿主边界 + 状态展开深度`

它用于约束：

- 哪些状态可以被写成同一条因果的条件与结果。

#### 2.4.3 状态面

`状态面` 指的是：

- 状态所属的解释域或可比域。

例如：

- 几何面
- 能量面
- 功能面
- 评价面
- 任务面

#### 2.4.4 实例因果信息

`实例因果信息` 指的是：

- 一次真实发生过的、具有明确锚点、层级和时间顺序的因果事实。

它的最小表达应理解为：

- `条件状态集合 + 动作/事件 -> 结果状态或结果动态`

代码映射：

- 当前因果实例承载在 `主信息定义模块.ixx` 的 `因果实例主信息类`。
- 其中已经出现 `锚点信息`、动作主体、动作语义、动作信息等关键字段。
- 当前因果生成与整理逻辑则主要落在 `因果信息类.ixx`。

#### 2.4.5 抽象因果模板

`抽象因果模板` 指的是：

- 由多条同层实例因果归纳沉淀出的可复用模式。

必须固定：

- 抽象因果模板不是当前事实。
- 它只能用于解释候选、预测候选或方法检索候选。

代码映射：

- 当前抽象因果模板承载在 `主信息定义模块.ixx` 的 `因果模板主信息类`。
- 任何把模板直接当成当前事实写回的行为，都属于语义越权。

#### 2.4.6 因果链

`因果链` 指的是：

- 多条合法单跳因果按时间顺序和桥接关系组织形成的解释链。

必须固定：

- 单条因果不允许跨层。
- 因果链允许跨层。
- 但链中的每一跳都必须先是合法的同层因果。

#### 2.4.7 状态样本

`状态样本` 指的是：

- 某主体在某时刻的一次状态记录。

#### 2.4.8 观察单元

`观察单元` 指的是：

- 由一个或多个状态样本比较后得到的一次结构化观察结果。

#### 2.4.9 证据

`证据` 指的是：

- 能支持某个候选概念或因果关系的一次有效观察单元。

#### 2.4.10 同类证据

`同类证据` 指的是：

- 关系主键、方向、根需求投影、适用条件和时间窗都可比较的一组证据。

#### 2.4.11 时间窗

`时间窗` 指的是：

- 用于判断证据是否可比较、是否可累计、是否还能保持有效性的时间范围。

代码映射：

- 当前 `二次特征主信息类` 已有时间窗枚举承载。
- `世界树类.ixx` 当前在状态比较与条件影响分析中，已使用短/中/长时间窗作为结构化结果的一部分。

### 2.5 需求时效、优先级与可用性术语

#### 2.5.1 需求有效时间

`需求有效时间` 指的是：

- 某一需求在时间维度上仍然成立、仍应参与响应与调度判断的有效区间。

#### 2.5.2 需求有效截止

`需求有效截止` 指的是：

- 某一需求在当前阶段下可接受的最晚完成时间点。

#### 2.5.3 永远有效

`永远有效` 指的是：

- 不受普通阶段性截止时间约束、在系统持续运行期间始终成立的有效时间语义。

当前根需求应属于：

- `永远有效`

#### 2.5.4 时效要求

`时效要求` 指的是：

- 某一非根需求在产生时同时带入的时间完成要求。

#### 2.5.5 时间余量

`时间余量` 指的是：

- `需求有效截止 - 当前时间 - 方法执行所需时间`

#### 2.5.6 时间紧急度

`时间紧急度` 指的是：

- 由时间余量所落入区间得到的紧急程度判断。

当前至少应包含：

- `已超`
- `临近`
- `正常`

#### 2.5.7 最初优先级

`最初优先级` 指的是：

- 认知尚未充分展开前，系统用于决定先后次序的基础排序规则。

#### 2.5.8 绝对优先级

`绝对优先级` 指的是：

- 认知之后，对对象、需求、方向与后果完成归属判断后形成的稳定排序结果。

必须固定：

- 最初优先级与绝对优先级不得混用。

#### 2.5.9 可获取

`可获取` 指的是：

- 某个对象、特征、状态或方法，当前能否被找到、接触、读取或调用。

#### 2.5.10 使用权限

`使用权限` 指的是：

- 即使对象或值当前可获取，系统规则、角色边界、安全约束与上下文是否允许使用。

#### 2.5.11 自我可用

`自我可用` 指的是：

- 对自我当前任务而言，某个对象、特征、状态或方法同时满足 `可获取` 与 `有权限使用`。

可以固定为：

- `自我可用 = 可获取 + 有权限使用`

## 3. 使用规则

后续规范与实现中，应统一遵守以下规则：

1. 若一个术语已在本文固定含义，则下游文档不得私自改写同名术语的核心语义。
2. 若某模块内部采用与上位规范相反的符号约定，必须在模块边界完成映射，不得把相反符号直接上送为根层语义。
3. 对目标差值类比较，统一语义是 `目标值 - 当前值`；通用比较层若使用 `左值 - 右值`，必须显式约定左右含义。
4. 自然语言层可以命名、解释和输出概念，但不能通过命名本身制造概念真值。
5. 任何概念若无法追溯到基础信息节点与证据链，则不得被视为正式概念。
6. 任何方向判断若无法回到差值与区间归属，则不得被视为稳定方向判断。
7. 任何“状态概念”若仍直接绑定某个具体存在，则应回退为状态事实，而不是抽象概念。
8. 所有稳定概念一旦成立，都应进入命名流程；命名优先级可调，但不得永久无名。
9. 单条因果中的条件与结果，必须先满足同锚点、同层级、时间有序等基本合法性，再讨论是否可归纳。
10. 代码映射只能帮助定位实现入口，不能反向改写术语定义本身。

## 4. 本文件的约束结论

后续实现中应固定以下结论：

1. `自我` 是在根约束层之下持续运行的统一主体，不等于单一变量、单一线程或单一句柄。
2. `根目的` 固定为 `满足人类需求 / 提升服务成立程度`。
3. `服务` 是对满足人类需求具有正向净贡献的结果，包含直接服务与准备性服务。
4. `安全` 是持续服务能力的保障条件，`安全值 == 0` 是根层生死边界。
5. `权重` 只能在父方向许可范围内调节次序和资源，不能改向。
6. `根方向` 是根目的在当前状态下形成的可持续差值缩减方向，至少包含主方向和保障方向。
7. `持续逼近型需求` 具有明确目标值和当前值，但通常只能持续逼近而不能永久终结完成。
8. 对带目标值的对象，统一采用 `差值 = 目标值 - 当前值`。
9. 二次特征是比较后的结构化关系结果，其中方向差值型二次特征是决策主入口之一。
10. 方向差值型二次特征必须落入 `正区间 / 负区间 / 零区间` 之一。
11. 某一比较值所落入的区间，就是该维度上的绝对方向。
12. 多维度方向必须先分维计算，再由根目的与安全边界裁决总方向。
13. `基础信息节点` 是概念、因果、证据链的最小可复用单元。
14. `概念` 是由稳定基础信息节点按关系组织而成的结构化认知对象，不等于自然语言词面。
15. 概念生成必须遵守 `0600`，所有正式概念都必须可追溯到基础信息节点与证据链，避免黑箱。
16. `状态概念树` 应从特征类型作为最抽象根开始生长，具体存在只负责命中节点。
17. `复合状态概念` 仍属于状态域，`复合特征` 仍属于特征组合，不得混成存在概念。
18. `实例因果信息` 是运行时真实发生的因果事实，`抽象因果模板` 是归纳出的可复用模式，两者不得混用。
19. `因果链` 允许跨层解释，但链中的每一跳必须先是合法的同层因果。
20. 根需求的有效时间属于 `永远有效`，非根需求的时效应带入具体截止与时间余量判断。
21. `最初优先级` 属于前认知排序，`绝对优先级` 属于认知后的稳定排序，两者不得混用。
22. `自我可用` 的最小判定是 `可获取 + 有权限使用`，二者缺一不可。
