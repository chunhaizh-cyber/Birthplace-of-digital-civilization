# 0311 双值时间回归本能与安全优先边界规则

## 1. 目标

本文用于固定以下问题：

- `服务值` 与 `安全值` 这两个根控制量，分别具有怎样的时间回归本能
- `服务时序衰减步长` 应按什么长期规则变化
- `安全值` 何时只做辅助约束，何时具有边界优先权
- 如何在 `服务为主 / 安全为辅 / 安全优先` 三者之间形成统一口径
- 如何避免系统因为安全修正而“目标漂移”或“被搞瘫”

本文要固定的是一条长期演化规则和行事准则，不是短时实验调参表。

## 3. 三层主次关系

### 3.1 服务为主

必须固定：

- 系统的根目的仍然是 `服务`
- `安全` 不是脱离服务的第二终局
- 不能因为安全紧张，就把系统的根目标改写成“只求安全值最大化”

可压成一句：

`服务值决定系统想往哪里走。`

### 3.2 安全为辅

必须固定：

- `安全` 的职责是保障服务持续成立
- `安全` 不直接替代服务目标
- `安全` 主要负责约束推进条件、推进速度与允许边界

可压成一句：

`安全值不改目标，只约束路径。`

### 3.3 安全优先

必须固定：

- 一旦继续推进当前服务路径会把系统推向死亡、致命失稳或不可恢复边界，`安全边界` 有权压过当前服务推进
- 这里的“优先”是 `边界优先`，不是 `目标篡位`
- 安全优先的落点应是：`门控 / 停止高风险试探 / 进入保守态 / 待机 / 收束 / 停止`

可压成一句：

`安全值不改服务根目的，但在边界裁决上拥有优先权。`


## 4. 双值时间回归本能

### 4.1 服务值时间衰减本能

必须固定：

- `服务值` 具有按时间自然衰减的本能
- 这个衰减不表示系统立刻失败，而表示服务能力会随着时间自然流失
- `服务时序衰减步长` 决定每轮时间流逝时，`服务值` 应减少多少

### 4.2 安全值时间回归本能

必须固定：

- `安全值` 也具有按时间变化的本能
- `安全值` 的时间演化目标不是一直单向升高，而是围绕 `50% ~ 80%` 的安全稳定带形成回归
- `安全值低于 50%` 时，应存在向 `50%` 回升的时间本能
- `安全值高于 80%` 时，应存在向 `80%` 回落的时间本能

### 4.3 两类本能的关系

必须固定：

- `服务值` 的时间衰减是自己的本能
- `安全值` 的时间回归是自己的本能
- `服务值` 可以通过时序步长间接塑造 `安全值` 的未来演化
- `安全值` 不应直接改写服务根目的，但可以反向压制 `服务值` 的衰减速度，避免系统在危险期发生双值共振塌陷


## 5. 服务时序衰减步长规则

### 5.1 数值边界

`服务时序衰减步长` 必须固定满足：

- 最小值为 `1`
- 最大值为 `I64max`

任何实现都不得产生小于 `1` 的步长。

### 5.2 高服务区规则

定义：

- `服务高位阈值 = floor(I64max * 4 / 5)`

必须固定：

- 当 `当前服务值 >= 服务高位阈值` 时，`服务时序衰减步长 = 1`

这表示：

- 当服务处于较高区间时，时间对服务能力的自然侵蚀保持在最小速度

### 5.3 服务下降放大规则

必须固定：

- 当 `当前服务值 <= 服务高位阈值` 后，`服务时序衰减步长` 随服务下降而单调增大
- 当前规范基准定义为：`服务值每下降 100 点，步长增加 1 点`

可写成：

`基础服务衰减步长 = 1 + floor(max(0, 服务高位阈值 - 当前服务值) / 100)`

必须明确：

- 这里的 `100` 是当前规范基准单位
- 后续允许经实验把 `100` 调整为其它正整数常量
- 但不得改变以下方向关系：
  - 服务越低，基础衰减步长越大
  - 服务高位区仍保持 `步长 = 1`
  - 步长的变化必须保持单调，不得反转

### 5.4 安全修正规则

定义：

- `安全中位值 = floor(I64max / 2)`

必须固定：

- 当 `当前安全值 < 安全中位值` 时，安全开始对 `服务时序衰减步长` 施加压制修正
- 当前规范基准定义为：`安全值每低于中位值 1 点，服务时序衰减步长减少 1 点`
- 修正后的最终结果仍不得低于 `1`

可写成：

`安全修正后服务衰减步长 = max(1, 基础服务衰减步长 - (安全中位值 - 当前安全值))`

这条规则的语义不是“安全帮服务变强”，而是：

- 当安全已经跌入较低区间时，系统不得再让服务值按过快速度继续塌陷
- 系统应优先保住最小运行面，避免双值同时快速崩落

### 5.5 最终语义

因此 `服务时序衰减步长` 的最终语义必须固定为：

- 服务越低，步长越大
- 但安全一旦低于 `50%`，系统会主动压住服务衰减速度
- 压制的目的是防止系统被时间本能直接推瘫，而不是把系统改造成“只顾安全”


## 6. 安全值时间回归规则

### 6.1 安全回归目标与安全带

必须固定：

- 安全时间回归不是围绕单一数值点做连续拉回，而是围绕一个 `安全稳定带` 运行
- `安全低位回归锚点 = floor(I64max / 2)`，即 `50%`
- `安全高位回归锚点 = floor(I64max * 4 / 5)`，即 `80%`
- `50% ~ 80%` 之间构成当前安全稳定带

必须继续固定：

- `安全值 < 50%` 时，不应继续被动放任下滑
- `安全值 > 80%` 时，也不要求无限制持续抬升
- `安全值` 落在 `50% ~ 80%` 稳定带内时，时间回归本能默认不主动推动其继续升高或继续降低

### 6.2 安全增加步长与安全减少步长

安全时间回归至少应存在两类步长：

- `安全增加步长`
- `安全减少步长`

两类步长都必须满足：

- 最小值为 `1`
- 最大值为 `I64max`

必须固定：

- `安全值 < 50%` 时，按 `安全增加步长` 随时间增加，并向 `50%` 靠拢
- `安全值 > 80%` 时，按 `安全减少步长` 随时间减少，并向 `80%` 回归
- `安全值` 落在 `50% ~ 80%` 稳定带内时，两类时间回归步长默认不主动驱动安全值漂移

### 6.3 服务对安全两类步长的调制方向

必须固定：

- `服务值 < 50%` 时：
  - `安全增加步长` 减少
  - `安全减少步长` 增加
- `服务值 > 80%` 时：
  - `安全减少步长` 减少
  - `安全增加步长` 增加
- `服务值` 落在 `50% ~ 80%` 区间时，安全两类步长保持当前基础口径，不额外做极性强化

这组规则的长期语义必须固定为：

- `服务低` 时，安全恢复更慢，安全下滑更快
- `服务高` 时，安全恢复更快，安全下滑更慢
- `服务值` 对 `安全值` 的影响必须通过这两类时序步长间接完成，不得直接把服务增量写进安全值

### 6.4 服务归零待机保护

必须固定：

- `服务值 = 0` 时，系统进入 `待机边界`
- 此时 `安全值` 进入最小非零保底线
- `服务值 = 0` 不等于死亡
- `安全值 = 0` 才是死亡边界


## 7. 安全优先的正式落点

### 7.1 安全优先不是目标改写

必须固定：

- 安全优先不表示“从此以安全为终局”
- 安全优先只表示：当边界被触发时，边界裁决权高于当前服务推进权

### 7.2 安全优先的具体工程落点

安全优先至少必须落在以下位置：

- `根层重判`
- `执行前门控`
- `高风险试探停止`
- `进入保守态`
- `待机 / 收束 / 停止`

不能把“安全优先”只做成一个数值修正公式，而缺失行为边界。

### 7.3 允许与禁止

必须固定：

- 若根层判断当前仅允许 `收束 / 待机 / 停止`，普通执行链必须被拦下
- 若继续当前试探会触发致命性安全边界，必须停止当前试探并进入保守态
- 若安全仍处可恢复区间，则可在门控后保留低风险、低预算、保底型服务推进


## 8. 防瘫与防目标漂移规则

为了同时满足“服务为主”和“安全优先”，必须继续固定以下规则：

1. 安全修正只能压制服务衰减速度，不能反向自动增加服务值。
2. 安全优先只能改路径、改门控、改预算、改允许边界，不能改写服务根目的。
3. `进入待机` 只是暂时失去主动推进能力，不等于系统终止存在。
4. 安全边界解除后，系统应重新回到服务主线，而不是永久停留在保守态。
5. 任何会影响 `继续 / 待机 / 收束 / 停止` 的结果，都必须先回到外层重判，不得由局部步长函数直接代决。

### 8.1 防误解补充解释

必须继续固定：

- 低安全时，对服务衰减速度的压制，仅用于防止双值共振塌陷，不构成继续推进许可。
- 是否继续推进，必须由根层重判、执行前门控与允许边界共同决定。
- 局部步长函数不得直接代决 `继续 / 待机 / 收束 / 停止`。
- `服务值 = 0` 进入待机边界，不等于死亡。
- `安全值 = 0` 才是死亡退出边界。
- 边界解除后，系统必须重新回到服务主线，不得永久滞留在保守态。

每周期最小输入、固定结算顺序以及“数值层 / 边界层 / 本能动作动态”责任边界，按 [0322_双值结算最小输入结算顺序与禁止混层规范.md](D:\鱼巢\规范\0322_双值结算最小输入结算顺序与禁止混层规范.md) 执行。


## 9. 可调整常量与不可改变方向

后续允许通过实验调参的常量包括：

- `服务高位阈值` 是否保持 `80%`
- `服务下降 100 点 -> 步长 +1` 中的 `100`
- 安全修正的单位倍率

但无论如何调整，都不得破坏以下方向：

1. `服务高位区` 的服务衰减应保持最小速度
2. `服务越低`，基础服务衰减步长越大
3. `安全低于 50%` 后，应抑制服务继续快速塌陷
4. `服务值` 对 `安全值` 的影响仍必须是 `间接影响`
5. `安全值 = 0` 与 `服务值 = 0` 的边界语义不得混淆


## 10. 可直接落代码的硬约束

后续实现至少必须满足：

1. `服务值` 与 `安全值` 都有独立的时间本能入口。
2. `服务时序衰减步长` 必须钳制在 `[1, I64max]`。
3. `当前服务值 >= 80%阈值` 时，服务衰减步长必须为 `1`。
4. `当前服务值` 下降后，基础服务衰减步长必须单调增加。
5. `当前安全值 < 50%` 时，服务衰减步长必须受到向下修正。
6. `安全值` 的时间演化必须围绕 `50% ~ 80%` 安全稳定带回归。
7. `服务值 = 0` 时只能进入待机保护，不得直接判死。
8. `安全值 = 0` 时必须进入死亡退出边界。
9. `安全优先` 必须同时落到 `重判 + 门控 + 保守态/待机/停止`，不得只做数值层假优先。
10. 安全优先不得把系统根目的从 `服务` 改写为 `安全最大化`。

补充硬约束：

11. `安全值 < 50%` 时，必须由 `安全增加步长` 驱动其向 `50%` 靠拢。
12. `安全值 > 80%` 时，必须由 `安全减少步长` 驱动其向 `80%` 回归。
13. `服务值 < 50%` 时，必须体现为 `安全增加步长减小、安全减少步长增大`。
14. `服务值 > 80%` 时，必须体现为 `安全增加步长增大、安全减少步长减小`。


## 11. 规则到内部先天动作的落地映射

为了保证后续能够稳定生成 `动作动态 -> 状态 -> 因果信息`，本文中的关键时间规则必须继续固定为对应的 `内部先天动作`。

必须固定：

- 这些动作只能由运行时被动触发
- 它们不得进入普通任务/普通方法的主动候选集
- `自我` 不能把它们当成普通方法主动点名调用
- `自我` 可以读取这些动作的运行结果、结果状态和动作动态，并据此生成后续因果信息
- 纳入固定清单的每个内部先天动作，都必须由 `自我线程` 每个周期自动调用一次
- 每次调用都必须生成一条对应的 `本能动作动态`；即使本轮没有引起数值变化，也仍应保留本轮动作事实动态

当前至少应固定以下映射：

| 规则面 | 对应内部先天动作 | 主结果 |
| --- | --- | --- |
| 服务下降放大规则 + 安全低位压制规则 | `按双值重算服务衰减步长` | `服务时序衰减步长` |
| 服务低位/高位对安全恢复能力的调制 | `按服务值重算安全增加步长` | `安全增加步长` |
| 服务低位/高位对安全下滑能力的调制 | `按服务值重算安全减少步长` | `安全减少步长` |
| 安全值低于 `50%` 的时间回归规则 | `安全低位回升` | `安全值` |
| 安全值高于 `80%` 的时间回归规则 | `安全高位回落` | `安全值` |
| 服务值按时间自然衰减规则 | `定时衰减服务值` | `服务值` |
| 服务归零后的底线保护规则 | `服务归零待机保护` | `待机状态`、`安全值` |

因此必须补充固定：

1. 规则不是只有说明文字，还必须对应到可追溯的内部先天动作动态。
2. 若一条规则本轮没有引起数值变化，仍允许只记录动作事实动态，不得伪造变化状态。
3. 若一条规则本轮引起主结果变化，结束相位动作动态必须回带主结果状态。
