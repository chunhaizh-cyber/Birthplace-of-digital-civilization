# 0890 控制状态机与触发条件表

## 1. 目标

本文用于把任务控制状态机收口到新的统一主逻辑：

`目标 -> 筹办 -> 执行`


## 2. 核心结论

必须固定：

1. `目标` 不是状态枚举，而是任务头固有定义。
2. `筹办` 和 `执行` 才是任务运行中的主阶段。
3. `完成 / 失败 / 取消 / 超时 / 无法执行` 仍然是任务状态，而不是并列主流程。


## 3. 运行阶段表

| 阶段 | 进入条件 | 退出条件 |
| --- | --- | --- |
| 目标已定 | 任务头已创建且目标已明确 | 进入筹办 |
| 筹办 | 当前任务树尚未可执行，或执行后需继续长树 | 当前步骤与结果节点已具备 |
| 执行 | 当前步骤、结果节点、方法实例已具备 | 目标达成，或回到筹办 |


## 4. 典型切换

| 当前阶段 | 触发 | 下一阶段 |
| --- | --- | --- |
| 目标已定 | 需要补步骤/结果/方法实例 | 筹办 |
| 筹办 | 当前步骤已具备执行条件 | 执行 |
| 执行 | 目标已达成 | 终态 |
| 执行 | 目标未达成或路径失效 | 筹办 |


## 5. 对旧模式词的处理

旧 `控制.模式 = 筹办 / 维护 / 纠偏 / 收束` 已废止。

后续如需保留兼容字段，只能解释为：

- 当前步骤语义
- 当前结果分流

不得再解释成任务统一主流程阶段。
