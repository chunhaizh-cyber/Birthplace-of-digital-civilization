# 0566 条件判定结果层与条件裁决边界规范

## 1. 目标

本文用于统一回答以下问题：

- `可获取 / 有权限 / 条件满足 / 缺口类型 / 下一步去向` 到底属于哪一层
- 这些结果为什么不是原始布尔量，也不是最终裁决
- 它们如何从一次特征、规则和模板中生成
- 它们与二次特征、最终裁决之间的正确关系是什么

## 2. 在五层体系中的位置

本文所说的这些结果，统一属于：

`第 3 层：条件判定结果层`

它们不是：

- `第 1 层` 的一次特征
- `第 2 层` 的主键字段组合辅助键
- `第 5 层` 的最终裁决

它们应统一理解为：

`在已有一次特征、规则、模板、权限边界和可获取边界基础上，通过比较、展开、筛选后得到的条件类结构化结论`


## 3. 条件判定结果的统一定义

条件判定结果至少应满足以下四点：

1. 有明确输入
   - 来自一次特征、原始状态、场景模板、权限边界、可获取边界等
2. 有明确判定过程
   - 至少经过比较、展开、筛选或规则匹配
3. 有明确输出类别
   - 输出为结构化枚举、分类结果或候选结果
4. 可被外层复用
   - 可继续进入二次特征层或最终裁决层

因此必须固定：

- 条件判定结果不是“原始值换个名字”
- 条件判定结果不是“局部经验拍脑袋”
- 条件判定结果是 `条件语义上的比较后产物`


## 4. 五类核心条件判定结果

### 4.1 可获取性判定结果

这一类回答：

- `目标状态、目标对象、目标信息当前能不能拿到`

典型输出包括：

- `可获取`
- `不可获取`
- `值未知需观察`

必须固定：

- `可获取 / 不可获取` 不是原始存在位本身，而是对“当前能否拿到所需条件”的判定结果。
- `值未知需观察` 不等于 `不可获取`，它表示“当前还缺观测，不应直接下死结论”。


### 4.2 权限性判定结果

这一类回答：

- `目标状态、目标对象、目标动作当前允不允许使用`

典型输出包括：

- `有权限`
- `无权限`

必须固定：

- 权限性判定结果必须建立在明确的规则边界或上下文边界上。
- `无权限` 不得被误写成“值不存在”或“条件未满足但可补”。
- 一旦判到 `无权限`，它优先属于条件判定结果，不应先被稀释成其它一般缺口。


### 4.3 条件满足性判定结果

这一类回答：

- `当前方法、当前动作、当前步骤在现有条件下能不能直接进入执行`

典型输出应统一为：

- `无条件，可直接执行`
- `条件满足，可直接执行`
- `条件不满足，但能明确列出缺哪些状态`
- `值未知需观察`
- `不可获取`
- `无权限`

必须固定：

- 这是方法执行和步骤生成前最直接的条件判定结果。
- 它不是最终裁决，但可以直接进入“是否需要补条件、观察、学习、挂起”的后续链。


### 4.4 缺口类型判定结果

这一类回答：

- `当前不能直接推进时，主缺口到底属于哪一类`

当前阶段统一值域应为：

- `无缺口`
- `待观察`
- `可补条件`
- `不可获取`
- `无权限`

必须固定：

- `缺口类型` 是对条件满足性结果的进一步提炼。
- 它是“主缺口分类”，不是所有缺口的完整列表。
- 同一轮允许保留详细缺口清单，但主缺口只能提炼一个。

当前工程提炼优先级继续固定为：

`无权限 > 不可获取 > 值未知需观察 > 未满足但可补 > 无缺口`


### 4.5 下一步去向候选结果

这一类回答：

- `基于当前上下文和条件判定，系统下一步更适合往哪条路径走`

当前阶段典型值域包括：

- `未定义`
- `继续下一步`
- `回跳重试`
- `补条件后重试`
- `转入尝试学习`
- `挂起等待`
- `回到重筹办`
- `进入收束`

必须固定：

- `当前下一步去向` 首先属于 `管理上下文中的条件判定结果`。
- 它回答的是“当前更适合往哪里走”，不是“根层已经永远批准这样走”。
- 任何会影响 `继续 / 停止 / 收束 / 待机 / 升格 / 回流` 的去向结果，仍必须回外层重判。


## 5. 生成顺序

当前应统一按以下顺序理解条件判定结果的生成：

```text
一次特征 / 状态 / 上下文 / 规则
-> 可获取性判定
-> 权限性判定
-> 条件满足性判定
-> 主缺口类型提炼
-> 下一步去向候选
-> 外层再结合二次特征做最终裁决
```

必须固定：

1. 不应跳过可获取性与权限性，直接讨论“条件满足”。
2. 不应跳过条件满足性，直接生造“缺口类型”。
3. 不应跳过缺口类型和上下文，直接拍“下一步去向”。
4. 不应把下一步去向候选直接当成根层最终真理。


## 6. 与二次特征的关系

条件判定结果层和二次特征层的关系，必须固定为：

- 条件判定结果回答：
  - `当前条件上能不能做`
  - `当前最主要缺的是什么`
  - `当前更适合走哪条路`
- 二次特征回答：
  - `当前方向对不对`
  - `结果是否同向`
  - `状态差值是否缩小`
  - `这次推进是不是在逼近目标`

因此：

- 条件判定结果层不能替代二次特征层
- 二次特征层也不能替代条件判定结果层

外层最终裁决应统一理解为：

`条件上能不能做`
`方向上该不该做`

两者都成立时，才允许真正推进。


## 7. 与最终裁决的关系

必须固定：

1. `可获取 / 无权限 / 条件满足 / 缺口类型 / 下一步去向` 都不是最终裁决。
2. 它们只是进入最终裁决前必须先形成的一组条件类结果。
3. 最终裁决仍必须综合：
   - 条件判定结果
   - 二次特征
   - 根层重判结果

因此：

- `可直接执行` 不等于 `一定允许执行`
- `回到重筹办` 不等于 `已经最终确定必须重筹办`
- `转入尝试学习` 不等于 `学习一定已被批准启动`


## 8. 禁止事项

以下做法统一禁止：

1. `某值非零 -> 可获取`
2. `某指针非空 -> 条件满足`
3. `主键命中 -> 无缺口`
4. `历史经常这么走 -> 当前下一步去向已确定`
5. `当前下一步去向候选 = 回到重筹办 -> 根层最终也一定回到重筹办`
6. 把 `无权限` 混写成 `不可获取`
7. 把 `值未知需观察` 混写成 `不可获取`
8. 把 `缺口类型` 直接当成最终裁决

`可获取、权限、条件满足、缺口类型、下一步去向，都属于条件判定结果层；它们是比较后的条件类结论，不是原始真值，也不是最终裁决。`
