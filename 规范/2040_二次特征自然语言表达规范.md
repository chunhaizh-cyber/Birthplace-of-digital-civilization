# 2040 二次特征自然语言表达规范

## 1. 目标

本文用于固定当前工程中：

- `二次特征主信息类` 如何转换为自然语言
- `反义词` 与 `介词短语` 在二次特征表达中分别承担什么职责
- 为什么反义词不能按“单节点自带一对词”理解，而必须从同特征类型的方向轴中取得

本文重点回答：

- 二次特征的方向词从哪里来
- 介词短语在二次特征表达中到底说明什么
- 单个二次特征节点在语言层应提供哪些槽位
- 完整表达应如何由“来源说明 + 方向结果”组织出来

本文服从以下硬边界：

1. `二次特征` 仍是比较后的结构化结果，是真值层。
2. 自然语言只是显示层和解释层。
3. 自然语言不得反向发明新的方向真值。
4. 同一个二次特征节点只能给出当前方向落点，不能把整条方向轴伪装成节点私有数据。


## 3. 核心口径

必须固定：

1. `反义词` 不是单个二次特征节点自带的一对词。
2. `反义词` 来自“同特征类型的一条方向轴”。
3. 单个二次特征节点只负责说明：当前比较结果落在这条方向轴的哪一侧。
4. `介词短语` 不是另一套结果词，而是“这个方向结果是如何获得到的”的来源说明。

一句话固定：

`二次特征节点给方向落点，语言模块按同特征类型取反义词，并用介词短语说明该方向是相对什么、在什么维度、通过什么比较得到的。`


## 4. 依据的字段面

当前 `二次特征主信息类` 可直接提供的关键字段包括：

- `形态`
- `种类`
- `主母式`
- `粒度`
- `基准类型`
- `时间归一方式`
- `度量签名_链键`
- `概念名称`
- `所属场景`
- `主体`
- `客体`
- `左对象`
- `右对象`
- `标量值`
- `是否满足`
- `置信度`

当前统一理解为：

- `形态`：回答“当前用在哪里”
- `种类`：回答“当前是什么二次特征”
- `主母式 / 粒度 / 基准类型 / 时间归一方式`：回答“当前二次特征是怎么生成出来的”
- `概念名称 / 度量签名_链键`：决定这是哪条方向轴，以及当前链式索引落点
- `主体 / 客体 / 左对象 / 右对象 / 所属场景`：决定介词短语来源槽位
- `标量值`：决定当前落在正向 / 反向 / 中性 哪一侧
- `是否满足 / 置信度`：只影响语气强弱，不决定方向词本身

必须额外固定：

- `度量签名_链键` 不应单独承担完整生成语义。
- 更稳的口径应是：`显式生成字段是真身，链键只是索引外壳。`


## 5. 反义词与介词短语的分工

### 5.1 反义词

`反义词` 负责回答：

- 最终得到的方向结果是什么

例如：

- `更高 / 更低`
- `更近 / 更远`
- `在内 / 在外`
- `在左侧 / 在右侧`
- `偏高 / 偏低`

必须固定：

- 反义词属于方向结果层
- 反义词必须从同特征类型的方向模板中选出
- 反义词不得直接写死在单个节点中


### 5.2 介词短语

`介词短语` 负责回答：

- 这个方向结果是相对什么得到的
- 在什么维度上得到的
- 通过哪种比较或空间关系得到的

例如：

- `相对于目标值`
- `相对于对象 B`
- `在安全维度上`
- `在内部世界中`
- `依据最近一次比较`
- `按稳态参照`
- `按首尾整体比较`
- `按总时长归一`

必须固定：

- 介词短语属于来源说明层
- 介词短语不得替代方向结果词
- 介词短语应优先由 `主体 / 客体 / 左右对象 / 所属场景 / 概念名称 / 主母式 / 粒度 / 基准类型 / 时间归一方式` 生成


## 6. 方向轴模板规则

### 6.1 模板归属

方向词模板必须归属于：

- `同特征类型`
- 或 `同概念名称 / 同生成母式字段组 / 同度量签名_链键`

而不归属于单个节点。

推荐理解：

- 一条“高低比较轴”共享 `更高 / 更低 / 等高`
- 一条“远近关系轴”共享 `更近 / 更远 / 同距`
- 一条“左右方位轴”共享 `在左侧 / 在右侧 / 重合`


### 6.2 方向判定

当前最小规则固定为：

1. `标量值 > 0`：选正向结果词
2. `标量值 < 0`：选反向结果词
3. `标量值 == 0`：选中性结果词

但必须继续固定：

- `正向 / 反向 / 中性` 只是模板选择结果
- `正向` 不等于“总是好”
- `反向` 不等于“总是坏”
- 价值判断必须回到外层规范，不得由语言层偷做


## 7. 两类输出面

### 7.1 反义词结果面

用于输出：

- `A 比 B 更高`
- `A 比 B 更低`
- `自我安全偏高`
- `自我安全偏低`

这一层只回答：

- 方向结果是什么


### 7.2 介词短语来源面

用于输出：

- `相对于目标值`
- `相对于对象 B`
- `在安全维度上`
- `在自我内部世界中`

这一层只回答：

- 方向结果是如何得出的


### 7.3 完整表达面

完整表达必须由上面两层合成。

推荐顺序：

`主体 + 介词短语 + 反义词结果`

推荐示例：

- `自我的安全值在安全维度上相对于目标值偏低`
- `对象 A 相对于对象 B 更近`
- `杯子在桌子的左侧`


## 8. 按种类的最小落法

### 8.1 比较型二次特征

当 `种类` 属于比较型时，优先生成：

- 介词短语：`相对于右对象` / `在某维度上`
- 反义词结果：`更高 / 更低 / 更近 / 更远`

推荐模板：

- `左对象 相对于 右对象 更 Adj`
- `主体 在 概念维度上 偏 Adj`


### 8.2 关系型二次特征

当 `种类` 属于关系型时，优先生成：

- 介词短语：空间或作用域来源
- 反义词结果：方位或内外方向词

推荐模板：

- `主体 在 客体 内部`
- `主体 位于 客体 左侧`
- `主体 处于 客体 外部`


### 8.3 观察关系

当 `种类 = 观察关系` 时：

- 反义词不是主面
- 介词短语来源说明可以弱化
- 优先直接生成关系句

推荐模板：

- `主体 观察到 客体`


## 9. 工程落点

当前工程中，这部分能力应归属于 `语言模块`，而不是散落在控制面板或各服务类中直接手拼。

最小接口建议固定为：

- `生成二次特征反义词结果(...)`
- `生成二次特征介词短语(...)`
- `生成二次特征自然语言(...)`

必须固定：

1. 控制面板只消费语言模块结果，不自行拼装二次特征句子。
2. 同一套二次特征语言模板应同时服务控制面板、日志与后续调试输出。
3. 不得在多个输出层各自维护一套互相漂移的方向词表。


## 10. 当前统一口径

当前阶段统一按以下顺序实现：

1. 先识别二次特征属于哪条方向轴
2. 再根据 `标量值` 判定正向 / 反向 / 中性
3. 再从同特征类型模板中取反义词结果
4. 再用 `主体 / 客体 / 左右对象 / 所属场景` 生成介词短语来源说明
5. 最后合成为完整自然语言

一句话收口：

`二次特征的反义词来自同特征类型的方向模板；介词短语负责说明这个方向是相对什么、在什么维度、通过什么关系得到的。`
