## 3. 定位与边界

### 3.1 本文负责

本文负责：

- 固定“世界树 -> 表达单元 -> 模板句型 -> 文本”的主链
- 固定世界树节点到表达用途的最小映射表
- 固定自然语言显示最少应保留哪些锚点与证据
- 固定控制面板、日志、自检输出中的自然语言显示口径


### 3.2 本文不负责

本文不负责：

- 发明新的真值字段或新的平行语义结构
- 取代世界树中的状态、动态、因果本体
- 取代语言理解、全文解析、文章级生成
- 强制当前阶段一次性实现完整自然语言系统


### 3.3 总原则

必须固定：

1. 世界树不直接生成句子，世界树先生成表达单元。
2. 表达单元经过句型模板和线性化后，才得到自然语言。
3. 文本只是最终表现，不是唯一输出。
4. 每条自然语言表达都应能回指来源节点与证据节点。

一句话固定：

`世界树先给出结构化表达单元，自然语言层只负责把它线性化成可读句子。`


## 4. 核心术语

### 4.1 表达单元

`表达单元` 指的是：

- 从世界树中抽出的一个最小可说内容
- 它已经确定了“在说什么”，但还没有变成最终句子

表达单元至少应回答：

- 谁
- 在什么场景或上下文里
- 发生了什么 / 具有什么特征 / 与谁有什么关系
- 有哪些证据支持这个表达


### 4.2 最小表达结构

`最小表达结构` 指的是：

- 一条最小可落成自然语言的句型骨架

当前第一批最小表达结构固定为 6 类：

1. 存在陈述
2. 特征描述
3. 事件叙述
4. 关系布局
5. 比较排序
6. 因果解释


### 4.3 表达用途

`表达用途` 指的是：

- 当前这条表达想表达哪一类语义

本规范承接旧口径，当前主要使用：

- `存在_陈述`
- `特征_描述`
- `事件_叙述`
- `关系_布局`
- `比较_排序`
- `因果_解释`


### 4.4 线性化

`线性化` 指的是：

- 把表达单元中的主语、谓语、宾语、修饰项、时间、地点，按句型顺序展开为词序列和文本

当前第一批线性化策略固定为：

1. `SVO`
2. `比较结构`


### 4.5 证据链

`证据链` 指的是：

- 当前这条文本表达所依赖的状态、动态、因果、二次特征或锚点节点集合

必须固定：

- 自然语言文本可以被压缩显示
- 但证据链不得丢失


## 5. 自然语言显示主链

世界树自然语言显示主链固定为：

`世界树节点`
-> `识别表达用途`
-> `抽取表达槽位`
-> `生成表达单元`
-> `选择句型模板`
-> `线性化`
-> `输出文本 + 锚点 + 证据`

必须固定：

1. 先识别用途，再生成句型。
2. 不允许跳过“表达单元”层，直接到处手拼字符串。
3. 同一个世界树节点允许生成多条表达，但每条表达只能有一个主用途。


## 6. 表达单元最小字段

当前世界树自然语言显示的最小表达单元，至少应具备以下字段：

```text
表达用途
锚点场景
主语节点
谓语词
宾语节点或结果值
修饰项列表
来源节点列表
证据节点列表
置信度
```

若映射到工程输出结构，最少应包含：

```text
用途
文本
词序列
锚点场景
来源节点
证据节点
置信度
```

必须固定：

- `文本` 是显示结果
- `来源节点` 是这条表达在说谁
- `证据节点` 是这条表达凭什么成立

## 8. 六类最小表达结构

### 8.1 存在陈述

最小槽位：

- 主语
- 系词或位置词
- 宾语或地点

推荐模板：

- `主语 是 宾语`
- `主语 在 地点`

适用来源：

- 场景
- 存在


### 8.2 特征描述

最小槽位：

- 主语
- 特征名
- 特征值

推荐模板：

- `主语 的 特征 为 值`
- `主语 当前 处于 值`

适用来源：

- 特征
- 状态


### 8.3 事件叙述

最小槽位：

- 主语
- 动作
- 可选宾语
- 可选时间/场景

推荐模板：

- `主语 执行了 动作`
- `主语 在 场景 中 执行了 动作`

适用来源：

- 动态


### 8.4 关系布局

最小槽位：

- 左对象
- 关系词
- 右对象

推荐模板：

- `左对象 在 右对象 内部`
- `左对象 位于 右对象 左侧`

适用来源：

- 二次特征中的关系型表达


### 8.5 比较排序

最小槽位：

- 比较对象 A
- 比较对象 B
- 比较词
- 维度词

推荐模板：

- `A 比 B 更 Adj`
- `A 的 维度 高于 B`

适用来源：

- 二次特征中的比较型表达


### 8.6 因果解释

最小槽位：

- 条件列表
- 动作
- 结果

推荐模板：

- `因为 条件，所以 动作 产生了 结果`
- `在 条件 下，动作 导致了 结果`

默认口径继续固定为：

`条件 + 动作 = 结果`

适用来源：

- 因果实例
- 抽象因果


## 9. 世界树节点如何生成表达单元

### 9.1 通用规则

对任意世界树节点，生成表达单元时必须优先抽取：

1. 锚点场景
2. 当前节点类型
3. 当前节点主语义对象
4. 当前节点最适合的表达用途
5. 当前节点的直接证据


### 9.2 场景/存在的抽取规则

场景或存在生成表达单元时：

- 主语优先取节点名称
- 若存在类型词，则允许补“属于某类型”
- 若存在父场景，则允许补“位于某场景中”
- 若要回答 `有什么 / 是什么 / 能什么 / 会什么 / 需要什么`，必须继续按 [2030_存在主信息自然语言映射规范.md](D:\鱼巢\规范\2030_存在主信息自然语言映射规范.md) 区分字段来源与证据层级


### 9.3 特征/状态的抽取规则

特征或状态生成表达单元时：

- 主语优先取状态主体或特征宿主
- 谓语优先取“特征为”或“当前为”
- 宾语优先取特征值
- 若值没有直接自然语言语义，允许保留结构化值文本


### 9.4 动态的抽取规则

动态生成表达单元时：

- 主语优先取动态主体
- 动作优先取来源动作名
- 若已有显式动作动态，不得回退成状态特征名充当动作名

这一点继续服从 [0850_统一表示规则（对象-关系-事件-状态与路径实例）.md](D:\鱼巢\规范\0850_统一表示规则（对象-关系-事件-状态与路径实例）.md)。


### 9.5 因果的抽取规则

因果生成表达单元时：

- 条件优先取条件状态列表
- 动作优先取动作语义
- 结果优先取主结果状态

若为抽象因果：

- 条件可以允许简写为“这类条件”
- 动作与结果应继续保留结构化锚点


## 10. 线性化规则

### 10.1 `SVO` 线性化

默认顺序固定为：

`主语 -> 谓语 -> 宾语 -> 时间/地点修饰`

适用于：

- 存在陈述
- 特征描述
- 事件叙述
- 因果解释中的动作段


### 10.2 比较结构线性化

默认顺序固定为：

`A -> 比 -> B -> 更/较/最 -> 维度词`

适用于：

- 比较排序


### 10.3 缺槽位回退规则

若句型缺槽位，回退顺序固定为：

1. 优先缩成短语
2. 再缩成标题
3. 最后才回退为结构化标签

不允许：

- 因为缺一个槽位，就整条表达丢失
- 因为缺少自然语言名，就现造一套平行命名系统

## 12. 世界树显示的输出层级

当前世界树自然语言显示应分成三层输出：

### 12.1 节点标题层

用于树结构浏览。

特点：

- 短
- 可扫读
- 能快速定位节点类型和主语义


### 12.2 句子摘要层

用于控制面板中部的自然语言说明。

特点：

- 一到三句
- 尽量完整表达“谁 / 什么 / 结果”
- 优先读得懂，而不是面面俱到


### 12.3 证据展开层

用于调试、自检和高级查看。

特点：

- 回指来源节点
- 回指证据节点
- 可展开条件、结果、关联场景


## 13. 最小输出结构

一条合格的自然语言显示结果，最少应包含：

```text
标题
文本
用途
锚点场景
来源节点
证据节点
置信度
```

这意味着控制面板后续显示不应只有一段字符串，而应是：

- `可读文本`
- `结构化回指`
- `可展开证据`


## 14. 禁止项

以下做法必须禁止：

1. 把自然语言显示结果当成新的真值层写回世界树。
2. 因为显示方便，直接修改节点原有语义。
3. 丢掉来源节点和证据节点，只保留裸文本。
4. 为显示层再造独立词对象池。
5. 用推测补齐不存在的因果、关系、动作。
6. 让显示层字符串反向成为对象主键。


## 15. 当前工程的第一阶段落地顺序

第一阶段只要求补齐以下能力：

1. 节点类型到表达用途的映射
2. 六类最小句型
3. `SVO` 和比较结构两类线性化
4. 文本 + 来源 + 证据的最小输出结构

当前阶段不强求：

1. 文章级自然语言树
2. 17 类原子结构全量实现
3. 完整语言知识库
4. 全量修辞、代词、省略和多句篇章组织


## 16. 当前统一口径

一句话收口：

`世界树自然语言显示不是把节点硬翻成字符串，而是先把结构化世界对象抽成表达单元，再按最小表达结构线性化成自然语言，并始终保留来源与证据。`
