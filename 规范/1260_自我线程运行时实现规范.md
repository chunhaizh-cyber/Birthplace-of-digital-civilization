# 1260 自我线程运行时实现规范

## 1. 目标

本文用于固定当前 `自我线程模块.ixx` 的运行时实现边界，回答以下问题：

- 自我线程当前到底承担哪些运行时职责
- 根层方向裁决、任务管理上下文装配与运行时提交之间如何接平
- 主循环、动作线程、学习线程和延迟写回如何协同
- 队列、锁、结构调整和学习反馈应如何分层
- 哪些属于自我线程运行时，哪些不属于

本文对应当前工程实现：

- `自我线程模块.ixx`
- `任务执行类.ixx`
- `任务调度类.ixx`

补充运行模型：

- `任务管理任务` 的实例化、单步推进、等待/唤醒与线程池承载规则，以 [1205_任务管理任务线程池运行模型规范.md](D:\鱼巢\规范\1205_任务管理任务线程池运行模型规范.md) 为准。
- `任务管理实例堆 / 宿主任务虚拟存在 / 恢复快照` 的三层分工，以 [1219_任务管理实例堆与恢复快照规范.md](D:\鱼巢\规范\1219_任务管理实例堆与恢复快照规范.md) 为准。
- 当前 `结构_任务管理请求 / 结构_任务管理实例` 的字段级归类，以 [1221_任务管理实例字段归类清单.md](D:\鱼巢\规范\1221_任务管理实例字段归类清单.md) 为准。
- `自我线程主循环` 的最小职责表，以 [1266_自我线程主循环最小职责表规范.md](D:\鱼巢\规范\1266_自我线程主循环最小职责表规范.md) 为准。


## 2. 定位

当前 `自我线程类` 的定位应固定为：

`运行时协调器 + 心跳调度器 + 根层重判触发者 + 任务管理上下文装配者 + 动作线程拥有者 + 学习线程拥有者 + 低频事实整理器`

它不是：

- 根约束定义器
- 宿主监督线程
- 后台前端服务线程
- 相机或外设运行时

### 2.1 最新统一主链中的位置

按当前最新体系，自我线程必须服从以下主链：

```text
自我线程心跳
-> 刷新双值与最小根层判断输入
-> 形成或刷新任务管理上下文
-> 选择待处理宿主任务
-> 提交任务管理请求
-> 动作线程 / 线程池单步推进
-> 返回消息、结果与上下文更新
-> 自我线程再次重判根层与当前功能域
```

必须固定：

1. 自我线程可以触发和承载运行时中的 `根层重判`，但它不是根语义定义来源。
2. 自我线程负责把根层结论和任务管理上下文装配到请求里，再交给后续承接层。
3. 自我线程不应绕过 `外层重判 -> 再提交` 这条链，直接在同一轮里替后续线程连续推进多轮治理闭环。


## 3. 当前运行时车道

当前实现中，自我线程至少包含以下几条运行时车道：

- `主消息/心跳车道`
  - 处理主队列
  - 驱动双值刷新、根层重判和根任务就绪判断
  - 装配最小任务管理上下文
  - 决定是否把任务送入动作队列、线程池请求入口或学习队列
- `动作队列车道`
  - 推进普通任务和管理任务
  - 在 `V0` 形态下同步承接“提交后的一步治理或执行落地”
- `学习队列车道`
  - 推进常驻学习任务
  - 执行 `采样 -> 更新 -> 验证 -> 提交 / 回滚`
- `延迟任务事实车道`
  - 暂存热点路径产生的任务事实
  - 在低频路径统一整理并写回
- `根任务结构调整车道`
  - 串行保护根任务结构调整与任务返回消息落地

这几条车道必须理解成：

- 同属一个运行时
- 但职责必须分层
- 不得因为“都在自我线程里”就互相直接越权改结构


## 4. 主循环职责

当前主循环的职责应固定为：

- 处理主队列消息
- 维持心跳驱动
- 刷新最小根层判断输入
- 执行运行时中的根层重判
- 刷新根任务的就绪与门控状态
- 装配和刷新任务管理上下文
- 决定是否提交 `任务管理请求`
- 触发动作线程、学习线程启动
- 协调低频整理、消息回投与学习唤醒

更细的主循环职责项、统一顺序和“哪些必须留在主循环”的边界，按 [1266_自我线程主循环最小职责表规范.md](D:\鱼巢\规范\1266_自我线程主循环最小职责表规范.md) 执行。

必须继续固定：

- 这里的“决定是否提交 `任务管理请求`”，应理解为当前运行时里最接近 `最终裁决层` 的工程承载动作。
- 它必须建立在 `条件判定结果 + 二次特征 + 根层重判结果` 之上。
- 其统一边界按 [0567_最终裁决层与外层重判边界规范.md](D:\鱼巢\规范\0567_最终裁决层与外层重判边界规范.md) 执行。

主循环不应直接承担：

- 叶子步骤的具体执行
- 学习阶段内部每一步的细节
- 宿主重启监督
- 在线程承载层内部偷偷连续推进多轮治理闭环


## 4.1 初始化阶段（新增硬约束）

`自我线程类::自我初始化()` 必须承担以下顺序化动作：

1. 初始化自我与基础特征承载。
2. 统一注册默认本能动作集（含方法首节点创建/复用与方法信息回填）。
3. 初始化初始需求与任务。
4. 刷新初始化完成状态并进入运行态。

必须固定：

1. 本能函数“首次注册”属于初始化阶段职责，不得延迟到任务执行/学习执行/周期维护中再补做。
2. 注册完成是“可执行前置条件”，未完成注册不得标记运行时初始化完成。
3. 注册不等于执行：初始化阶段不要求逐个执行本能函数本体。


## 5. 动作线程职责

当前动作线程入口的职责应固定为：

- 从执行队列中取任务
- 将任务置为 `执行中`
- 在结构调整锁保护下调用当前 `V0` 入口做一步治理或一步执行落地
- 接收 `结构_任务返回消息`
- 驱动需求状态刷新、叶子需求结算和调度器提交流程
- 对 `生成子任务` 的返回消息做直接入队

必须固定：

1. 动作线程只消费 `执行队列`。
2. 动作线程的结构改动入口，统一收束到 `推进一步 + 返回消息`。
3. 动作线程可以把事实排队到延迟写回车道，但不应在热点路径里做重整理。
4. 动作线程不负责根层方向裁决；它只承接已由主循环装配好的请求或已入队的执行工作。
5. 当前若仍通过 `根任务筹办器_.推进一步` 同步直调，也只能视为 `1205` 所述线程池模型的 `V0 单线程退化承载`。
6. `继续提交下一轮请求 / 停止提交 / 转入学习 / 进入收束 / 允许回流 / 允许升格` 这类最终裁决，不属于动作线程职责，统一由主循环在外层重判后决定。


## 6. 学习线程职责

当前学习线程的职责应固定为：

- 只消费 `学习队列`
- 特判常驻 `学习任务节点_`
- 推进学习阶段机
- 在更新阶段调用 `私有_尝试学习一步`
- 在提交阶段唤醒等待学习的方法和任务

当前学习线程应理解为：

- 已经能真正执行
- 但仍属于专用学习工作流
- 还不是完全并入普通叶子步骤执行器的最终形态

因此必须固定：

1. 学习线程内部的阶段推进，是学习专用运行时逻辑。
2. 学习线程的产出，主要是方法树更新、候选补全和等待任务唤醒。
3. 学习线程不直接替代任务调度器。
4. 学习线程回流后，仍应回到主循环做 `根层重判 -> 上下文刷新 -> 再提交`，不得自己越权续推主闭环。


## 7. 延迟事实与低频整理

当前实现已经存在“动作线程只记最小事实，扩展整理移到低频路径”的口径。

因此应固定：

- 热路径允许只写最小回执、关键事实和最小方向特征
- 复杂投影、对齐、账本补齐和大规模同步，应由低频整理路径完成

延迟事实队列的职责应固定为：

- 承接热点路径不宜立刻落地的任务事实
- 在更低争用的时间点统一冲刷到任务虚拟存在


## 8. 本模块锁与顺序

当前模块至少存在以下锁：

- `故障锁_`
- `mtx_`
- `执行队列锁_`
- `学习队列锁_`
- `延迟任务事实锁_`
- `根任务结构调整锁_`

本模块的默认规则应固定为：

- 默认单锁化
- 能不双持就不双持
- `故障锁_` 视为独立故障账本锁，默认不与其它锁同持

若确实无法避免多锁同持，本模块的局部顺序应固定为：

1. `mtx_`
2. `执行队列锁_ / 学习队列锁_`
3. `延迟任务事实锁_`
4. `根任务结构调整锁_`

必须固定：

1. 队列等待时，只允许持有对应队列锁。
2. 持有 `根任务结构调整锁_` 时，不得再去等待条件变量。
3. `try_to_lock` 只能作为受限的拥塞退避手段，不得作为主正确性机制。


## 9. 启停与拥有关系

当前自我线程必须服从以下拥有关系：

- `自治宿主` 拥有 `自我线程`
- `自我线程` 拥有 `动作线程` 与 `学习线程`

因此必须固定：

1. 自我线程只负责启动和回收自己的直属子线程。
2. 动作线程和学习线程不得反向控制宿主。
3. 停机应遵守 [1230_多线程嵌套启停管理规范.md](D:\鱼巢\规范\1230_多线程嵌套启停管理规范.md)。


## 10. 自我线程的硬约束

1. 主循环负责“决定与派发”，动作线程和学习线程负责“消费与推进”。
2. 任何任务结构改动，都应尽量通过统一的返回消息和结构调整入口落地。
3. 热路径事实回写应最小化，重整理必须延后。
4. 学习线程是专用学习运行时，不得误当成普通动作线程的别名。
5. 任何队列等待都必须带谓词，并遵守 [1240_线程内部锁与条件变量使用规范.md](D:\鱼巢\规范\1240_线程内部锁与条件变量使用规范.md)。
6. 本模块的本地锁顺序必须可见，不能依赖“写代码的人自己记住”。
7. 线程生命周期状态变化和关键运行时状态变化必须形成日志。
8. 主循环每轮在提交任务管理请求前，应先完成最小根层重判与任务管理上下文刷新。
9. 动作线程或后续线程池每轮只承接一步，不得在承载层内部连跑多轮治理。
10. 任何一步推进后的结果，都应先回到主循环或外层协调器，再进入下一轮判定与提交。

- 元层学习写回的实体关系定义
