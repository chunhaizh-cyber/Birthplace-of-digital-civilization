# 0940 任务方法树与任务树的出生与成长规则（保留原文件名兼容）

## 1. 说明

原 `四域方法树` 口径已废止。

当前只保留两层：

1. `方法树`：模板层
2. `任务树`：实例层

任务统一按 `目标 -> 筹办 -> 执行` 生长。


## 2. 两层出生

### 2.1 方法树先出生

方法头、条件节点、结果节点属于模板层。

它们回答的是：

- 这条方法是什么
- 它需要什么条件
- 它可能产出什么结果

### 2.2 任务树后出生

任务头、步骤节点、结果节点属于实例层。

它们回答的是：

- 当前这次任务想达成什么目标
- 当前树已经长到了哪一步
- 这一步实际产出了什么


## 3. 筹办负责的出生动作

筹办阶段只负责以下生长：

1. 为任务头长出步骤节点。
2. 为步骤节点长出结果节点。
3. 为步骤绑定方法头。
4. 按方法模板长出任务实例条件节点和任务实例结果节点。
5. 必要时长出子任务头。

必须固定：

- 方法树不直接代替任务树执行。
- 任务树也不复制方法模板本体；它只持有实例引用与实例节点。


## 4. 执行负责的成长动作

执行阶段只负责：

1. 读取当前步骤的方法实例条件节点。
2. 调用当前步骤绑定的方法。
3. 把执行结果写入当前结果节点的方法实例结果节点。
4. 根据结果决定任务是否完成，或是否返回筹办继续长树。


## 5. 三类任务的成长区别

### 5.1 普通任务

- 任务树围绕现实存在的目标特征生长。

### 5.2 任务管理任务

- 任务树围绕宿主任务完成度生长。
- 其步骤表示治理步骤，不再表示旧的四域。

### 5.3 学习任务

- 任务树围绕目标方法可用度生长。
- 其步骤默认依次经历：补动作、补条件、补结果、验证、尝试学习。
