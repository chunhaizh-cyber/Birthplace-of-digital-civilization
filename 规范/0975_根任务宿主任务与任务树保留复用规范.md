# 0975 根任务宿主任务与任务树保留复用规范

## 1. 目标

本文用于固定以下问题：

- `根任务` 和 `宿主任务` 是否需要每轮都由 `任务管理任务` 从头梳理
- 什么时候必须重新从头梳理任务树
- 任务完成后，任务树是立刻删除还是保留结构
- 相同或相近需求再次出现时，旧任务树与新任务树是什么关系
- 叶子任务结果回流后，结果应怎样回挂到相关任务链

## 3. 核心结论

必须固定以下结论：

1. `根任务` 不需要每一轮都由 `任务管理任务` 从头梳理。
2. `宿主任务` 也不需要每一轮都从头梳理；只要当前仍有可执行或可继续推进的子任务链，就应优先沿现有任务树继续。
3. 只有在以下场景，才要求 `任务管理任务` 对当前根任务或宿主任务重新从头梳理：
   - 当前已没有可执行子任务
   - 当前已没有可继续推进的子任务
   - 系统刚重启
   - 系统刚从休眠/待机恢复
4. 一个任务被执行完成后，任务树不应立刻删除；当前正式语义应是：
   - `状态改为完成`
   - `树结构保留`
   - `后续允许抽象生成方法`
5. 当后续再次出现相同或相近需求时，不是直接复活旧任务树继续跑，而是：
   - 以旧任务树作为抽象与复用原料
   - 为新需求形成新的任务树
6. 叶子任务完成后，结果必须先回到 `自我主循环`，再由自我根据 `需求列表 + 需求树` 把结果回挂到相关任务链。

一句话收口：

`任务树默认保留并优先沿原链继续推进；只有链条真的走不动、或系统刚重启/休眠恢复时，才重新从头梳理；旧完成树保留结构，供后续抽象方法和复用新树。`


## 4. 根任务与宿主任务的重头梳理触发条件

### 4.1 不应重头梳理的情况

以下情况当前不应强制从头梳理：

- 当前仍有可执行子任务
- 当前虽暂无叶子执行，但仍有可继续推进的步骤/子树
- 当前只是局部等待外部反馈
- 当前只是等待学习结果回流

必须固定：

1. `任务管理任务` 默认优先“沿现有树继续治理”。
2. 不得为了实现简单，每轮都把根任务和宿主任务重新拆开再建。

### 4.2 必须重头梳理的情况

以下情况当前必须允许或优先进入“重头梳理”：

- 当前已没有可执行子任务
- 当前已没有可继续推进的子任务
- 当前宿主任务进入 `待重筹办`
- 系统重启后的第一轮承接
- 系统从休眠/待机恢复后的第一轮承接

必须固定：

1. 这里的“重头梳理”是治理动作，不等于删除旧树。
2. 旧树可以保留为已完成/已冻结/已收束结构，供复用与审计。


## 5. 任务完成后的保留语义

### 5.1 完成后的最小处理

当一个任务树或子树完成后，当前至少应完成：

1. 将对应任务状态改为 `完成`
2. 保留结构与节点关系
3. 保留与该树相关的关键结果节点和方法挂点
4. 允许后续从这棵树抽象方法

必须固定：

1. `完成` 不等于 `立刻删除`。
2. 任务树保留的第一意义是：为后续方法抽象、路径复用、审计和稳定结构提炼提供原料。

### 5.2 旧树与新树的关系

当后续再次出现相同或相近需求时，必须固定：

1. 自我不直接把旧完成树当成“当前活跃任务树”继续跑。
2. 自我应优先：
   - 从旧树抽象出方法、组合路径或稳定结构
   - 再把这些东西带入新需求
   - 形成新的宿主任务树
3. 因此：
   - `旧树` 是复用原料
   - `新树` 是新一轮治理承接对象


## 6. 叶子结果回流与结果回挂

叶子任务完成后，结果流向必须固定为：

```text
叶子任务结果
-> 返回自我主循环
-> 自我根据需求列表与需求树定位相关需求
-> 将结果回挂到相关任务链
-> 刷新任务上下文与下一轮治理输入
```

必须固定：

1. 叶子结果不直接由叶子任务自己写成整棵树的最终结论。
2. 结果回挂的主裁判是 `自我主循环`，不是单个子任务。
3. 回挂时至少应结合：
   - 当前需求列表
   - 需求树关系
   - 当前宿主任务
   - 最近结果节点
4. 回挂完成后，才能进入：
   - 双值结算
   - 任务状态刷新
   - 是否继续推进 / 重筹办 / 收束 的再判断


## 7. 与方法抽象的关系

必须固定：

1. 完成后的任务树结构，允许作为方法抽象原料。
2. 抽象生成的方法应进入方法树或方法账本，而不是只停在任务树里。
3. “旧树保留” 的主要价值之一，就是让自我能把稳定任务结构转成方法存在面。
4. 这不等于“旧任务树直接等于方法”；仍需经过结构化抽象。


## 8. 严格禁止的混层

以下做法必须禁止：

1. 每轮都强制让根任务重新从头梳理。
2. 每次宿主任务局部推进后，都先删树再重建。
3. 一个任务一完成就立即抹掉结构。
4. 旧完成树直接当成新需求的当前活跃树继续跑。
5. 叶子任务不经主循环，直接把结果写成整棵树最终真值。
