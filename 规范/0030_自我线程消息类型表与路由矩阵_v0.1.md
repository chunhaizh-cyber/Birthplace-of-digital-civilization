## 3. 核心结论

必须固定以下结论：

1. 自我线程只处理 `控制面 / 根层判断面 / 全局调度面` 相关的消息。
2. `控制消息` 与 `全局环境消息` 必须先到 `自我线程`。
3. `局部外部反馈` 若只影响某任务当前步骤，应先到对应工作线程。
4. 工作线程只把 `会改变控制面判断` 的推进事件上浮到线程。
5. `派生需求` 只能上报，不能直接生成子任务。
6. `结果回流` 必须先回线程，再由线程路由给父任务。
7. `学习与审计消息` 后置处理，不抢占当前业务轮次。
8. 一轮消息处理必须遵守：`冻结一批，清完这一批，再做一次决议。`

一句话收口：

`控制消息改控制态，全局环境消息改根需求，关键推进事件改队列和阻塞面，派生需求决定是否新建任务，结果回流决定路由与结算，学习与审计统一后置。`

## 8. 消息闭包最小规则

必须固定以下闭包规则：

1. 一轮主循环先冻结“本轮开始时已可见的消息批次”。
2. 本轮必须处理完这批冻结消息。
3. 本轮还必须处理完这些消息同步引发的内部闭包，例如：
   - 子结果先回线程
   - 线程路由结果给父任务
   - 父任务据此更新局部状态
   - 同步产生派生需求上报
4. 本轮中新产生的异步消息不得追入当前轮次。
5. 高优先级控制消息可按抢占规则打断当前轮次。

最小闭包表如下：

| 触发消息 | 本轮同步会引发什么内部消息 | 是否必须本轮处理完 | 是否允许延后到下一轮 |
| --- | --- | --- | --- |
| 打断 / 收束 / 停止 | 控制事件下发、任务控制态更新、局部结果/中断结果回传请求 | 是 | 否 |
| 当前步骤阻塞 | 父任务退队、等待态更新 | 是 | 否 |
| 当前步骤解除阻塞 | 任务重回可运行队列、重排调度 | 是 | 否 |
| 派生需求消息 | 派生需求审核、批准/驳回、必要时创建依赖任务入口 | 是 | 否 |
| 子任务结果 / 局部结果 / 最终结果 | 结果缓存、路由父任务、父任务局部影响判断、结算候选登记 | 是 | 否 |
| 学习与审计消息 | 后置队列登记 | 否 | 是 |
| 新派发出去的依赖任务异步执行结果 | 新的结果回流消息 | 否 | 是 |


## 9. 六条硬规则

必须固定以下 `6` 条：

1. `控制消息` 和 `全局环境消息` 必须先到 `自我线程`。
2. `局部外部反馈` 若只影响某任务当前步骤，应先到对应工作线程。
3. 工作线程只把 `会改变控制面判断` 的推进事件上浮到 `自我线程`。
4. `派生需求` 只能上报，不能直接生成子任务。
5. `结果回流` 必须先到 `自我线程`，再由线程路由给父任务。
6. `学习与审计消息` 后置处理，不抢占当前业务轮次。
