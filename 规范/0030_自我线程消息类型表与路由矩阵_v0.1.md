## 3. 核心结论

必须固定以下结论：

1. 线程正式输入消息只有 `7` 类，不再保留 `学习与审计消息` 作为正式输入类型。
2. `学习 / 审计 / 复盘` 属于后置队列；若升级为正式事实，必须重新包装成这 7 类中的一种再入队。
3. `需求 / 任务权重 / 否定项成立` 都不是外来消息类型。
4. 7 类消息的首收件人，在进入自我线程正式输入集合之后，恒为 `自我线程`。
5. `派生需求上报消息` 可以触发 `新任务候选`，但不能由工作线程直接实例化子任务。
6. `工作线程状态消息` 当前是兼容名；语义上应按 `线程状态变化消息` 理解，且发件线程只负责上报事实，不负责直接落账。

## 7. 表 4：7 类消息处理要点

| 二级类型 | 最小处理要点 |
| --- | --- |
| `控制域事实消息` | 更新线程控制态与任务控制态，决定是否抢占本轮，并向工作线程下发控制投影 |
| `业务域事实消息` | 刷新线程上下文快照，重算依赖满足、可执行性、风险和影响项 |
| `延迟事实消息` | 把到期事实实体化为一次特征变化，进入重算和根层重判 |
| `工作线程状态消息` | 把线程状态变化规范化为一次特征变化；由自我线程统一更新线程状态账，必要时进入故障收口、恢复重建或队列调整 |
| `任务推进关键事件消息` | 只处理关键事件，不吞局部小进度；阻塞退队，解阻再入队 |
| `派生需求上报消息` | 判断是局部等待还是需要升格；若批准则形成新任务候选 |
| `结果与关键中间状态回流消息` | 先判断全局影响，再路由父任务，再抽取否定项候选并统一结算 |


## 8. 必须排除的非输入项

以下内容不得写进 7 类消息矩阵：

- `需求成立`
- `任务权重变化`
- `优先级应该提高`
- `否定项已成立`
- `学习反馈`
- `审计结论`
- `复盘结论`
- `局部小进度`
