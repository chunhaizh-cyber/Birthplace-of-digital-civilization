# 本能函数实现规范 20260503

状态：当前权威规范。文件名沿用 20260430，内容已按 2026-05-03 的方法配对和特征值改变链路修订。

## 1. 适用范围

本文定义本能函数的标准实现方式，适用于自我动作实现模块、后续本能方法实现模块，以及通过方法树调用的固定本能动作。

本文主要约束本能函数本体；当涉及任务链路时，以“需求节点对应任务的两阶段壳”为前提，不再以独立任务管理任务或学习任务作为长期权威结构。

本能函数不得把方法名称、动作名称、显示文本、语素文本或函数名称作为执行语义依据。名称只能用于显示、日志和调试。执行语义必须落到特征类型、特征值、区间、抽象特征父子关系、限制包和方法能力包。

需求树链路固定为：

```text
需求节点
  -> 对应任务
  -> 筹办：查找方法，写入选中自我能力值或派生方法缺口子需求
  -> 执行：配对参数，调用本能函数或派生条件缺口子需求
  -> 运行事实回写
  -> 需求满足状态向父需求回传
```

本能函数失败原因应尽量输出为可派生需求的缺口事实；缺口不是任务终点，而是需求树继续生长的依据。

任务虚拟存在必须把阶段事实写成实例特征：

```text
任务虚拟存在
  当前阶段 = 筹办 / 执行 / 等待子需求或阻塞 / 已完成
  缺口类型 = 方法缺口 / 方法结构缺口 / 参数条件缺口 / ...
  选中自我能力值 = 枚举_本能方法ID 的 I64 值
```

阶段、缺口和能力值不得只保存在临时结构体或控制分支中。

## 2. 标准函数形态

本能函数的标准函数形态固定为：

```cpp
using 本能动作函数 = std::function<存在节点类*(
    方法类::节点类* 方法首节点,
    场景节点类* 输入参数场景,
    场景节点类* 输出结果场景)>;
```

返回值必须是 `存在节点类*`，表示本轮运行虚拟存在。

返回值不表示单个功能结果。功能结果必须写入 `输出结果场景`，运行过程必须写入本轮运行虚拟存在，并可同步长期统计到方法虚拟存在。

## 3. 标准四模块

每个本能函数内部必须按以下四个模块组织：

| 模块 | 职责 |
| --- | --- |
| 方法规格建立与确认 | 确保方法首节点、方法虚拟存在、本轮运行虚拟存在、形参表、结果表、效果表和方法能力包。 |
| 输入条件包绑定与校验 | 从输入参数场景读取本轮实际参数，并校验输入是否闭合。 |
| 执行或匹配 | 对实际目标执行写回，或对候选方法执行匹配。不得在此阶段补造缺失语义。 |
| 输出结果与运行事实 | 写输出结果场景、本轮运行虚拟存在、方法统计、失败原因和必要的派生需求。 |

确保阶段不得因为节点已经存在就直接返回。确保完成后必须继续进入绑定、执行和输出。

绑定失败、执行失败时，也必须进入输出阶段。只有无法形成有效运行虚拟存在时，才允许返回空。

## 4. 四类场景边界

本能函数中必须区分四类场景：

| 名称 | 所属 | 含义 |
| --- | --- | --- |
| 条件场景 | 方法自身 | 方法构建的抽象输入模板，声明本方法需要哪些输入特征。 |
| 结果场景 | 方法自身 | 方法构建的抽象输出模板，声明本方法会形成哪些功能结果。 |
| 输入参数场景 | 任务或执行方 | 本轮调用提供的实际输入参数。 |
| 输出结果场景 | 本轮执行 | 本轮执行写出的实际功能结果和返回信息。 |

条件场景不是任务提供的实际输入。输入参数场景才是任务提供的实际输入。

结果场景不是本轮实际输出。输出结果场景才是本轮实际输出。

## 5. 标准输入条件包

新本能函数应优先读取：

```text
输入参数场景
  输入条件包
```

兼容期可以回退读取：

```text
输入参数场景
  条件集合
```

但是新规范中的长期语义名称统一为 `输入条件包`。

输入条件包必须只承载本轮已经闭合的执行或匹配参数。若参数尚未闭合，应由上游 `查找方法` 和 `配对参数` 阶段完成，不得由执行函数猜测、默认补齐或按名称推断。

## 6. 方法能力包

可被查找的方法必须在方法虚拟存在中保存 `方法能力包`。

方法能力包不是方法名称。方法能力包只说明该方法能覆盖哪些特征类型、特征值范围、区间和约束。

示例：

```text
方法虚拟存在：改变特征值
  方法能力包
    目标宿主
      特征类型 = 目标宿主
      特征值范围 = 可修改宿主 / 可修改现实场景
    目标抽象特征
      特征类型 = 目标抽象特征
      特征值范围 = 可写实例特征
    目标值类型
      特征类型 = 目标值类型
      特征值范围 = 类型_I64
    执行差值
      特征类型 = 执行差值
      特征值区间 = [I64_MIN, I64_MAX]
```

`查找方法` 只能用 `需求条件包 ⊆ 方法能力包` 判断候选，不得按方法名或动作名匹配。

## 7. 查找方法本能函数合同

`查找方法` 的职责是：

```text
在限制包约束下，
判断任务需求条件包是否被某个方法能力包包含，
并输出方法配对结果存在。
```

它不执行方法、不补全执行参数、不生成可执行输入参数场景。

标准输入：

```text
输入参数场景
  输入条件包
    需求条件包
    查找限制包
    候选范围包，可选
```

必须绑定：

| 输入 | 说明 |
| --- | --- |
| 需求条件包 | 表示任务想要什么。至少应有一个必需条件项。 |
| 查找限制包 | 限制抽象层数、允许值类型、允许目标特征上界、允许宿主上界等。 |

可选绑定：

| 输入 | 说明 |
| --- | --- |
| 候选范围包 | 限定方法搜索根、是否包含禁用方法、是否允许当前方法自身等。 |
| 方法搜索根 | 缺失时可从当前方法首节点推导方法树根，但必须写入运行记录。 |

标准输出：

```text
输出结果场景
  方法查找情况
  选中自我能力值 -> 枚举_本能方法ID 的 I64 值
  方法配对结果 -> 方法配对结果存在
```

`方法配对结果存在` 至少保存：

| 输出 | 说明 |
| --- | --- |
| 配对状态 | 配对成功、配对失败、超出限制或输入绑定失败。 |
| 来源需求条件包 | 原始需求条件包引用。 |
| 使用限制包 | 本次查找使用的限制包。 |
| 选中自我能力值 | 权威业务输出，使用 `枚举_本能方法ID` 的 I64 映射值表示。 |
| 缓存方法首节点 | 运行期缓存；执行前必须按 `选中自我能力值` 重新校验或重解析。 |
| 缓存方法虚拟存在 | 运行期缓存；不能作为任务长期语义主键。 |
| 匹配方法能力包 | 命中的能力包。 |
| 原始需求绑定包 | 原始需求口径，供下一步配对参数使用。 |
| 匹配路径记录 | 类型和值域抽象路径与距离。 |
| 待配对形参表 | 选中方法的形参表。 |

重要规则：

1. 原始需求绑定包必须保留原始口径。
2. 查找时可以向上抽象，但执行时不得把父级能力口径当成最终执行口径。
3. 任务长期信息保存 `选中自我能力值`，不保存方法首节点作为权威字段。
4. 查找方法完成后，下一步是 `配对参数`，不是直接执行选中方法。

## 8. 改变特征值本能函数合同

`改变特征值` 是通用执行壳。它只做一件事：

```text
根据已经闭合的输入条件包，
改变某个宿主下某个实例特征的 I64 当前值，
并记录执行事实。
```

它不查找方法、不配对参数、不按名称理解增加或减少、不默认补全缺失参数、不把抽象特征当成实例特征写当前值。

### 8.1 输入结构

标准输入：

```text
输入参数场景
  输入条件包
    目标宿主
    目标抽象特征
    目标值类型
    差值约束包
    执行差值 / 目标执行后值
    执行前值快照，可选
    特征类型配对结果，可选
    实例缺失策略，可选
    溢出策略，可选
    限制包，可选
```

必需项：

| 输入 | 要求 |
| --- | --- |
| 目标宿主 | 目标实例特征挂载的存在、场景或虚拟存在。 |
| 目标抽象特征 | 要改变的抽象特征。必须是抽象特征，不得是实例特征。 |
| 目标值类型 | 必须为 `类型_I64` 或兼容的 I64 值类型标识。 |
| 差值约束包 | 一个或多个 I64 区间，或可展开为区间集合的组合抽象特征。 |

执行值来源必须满足以下条件之一：

| 输入形态 | 说明 |
| --- | --- |
| 执行差值 | 执行后值 = 执行前值 + 执行差值。 |
| 目标执行后值 | 执行后值 = 目标执行后值。 |
| 差值约束包唯一为 `[0,0]` | 可以缺省执行值来源，执行后值 = 执行前值。 |

若 `执行差值` 和 `目标执行后值` 同时存在，必须校验：

```text
目标执行后值 - 执行前值 == 执行差值
```

不一致时必须参数绑定失败或执行失败，不得任选其一。

### 8.2 当前值读取

`改变特征值` 不得从输入参数场景读取当前值作为计算依据。

执行前值必须从目标宿主下的目标实例特征读取：

```text
目标宿主
  实例特征：目标抽象特征
    当前值 = 执行前值
```

`执行前值快照` 只用于校验输入是否过期，不参与计算。

### 8.3 实例特征写回

抽象特征只定义维度、区间和细化链，没有当前值。执行写回必须落到宿主下的实例特征。

实例特征缺失时的最小安全行为是失败并输出派生需求：

```text
目标实例特征缺失
  -> 派生需求：实例化目标特征
```

若未来允许实例化，也必须由显式 `实例缺失策略` 授权，且当前值初始化不得默认补 `0`。

### 8.4 执行公式

执行阶段固定为：

```text
执行前值 = 从目标实例特征读取
执行后值 = 执行前值 + 执行差值
实际执行差值 = 执行后值 - 执行前值
实际执行差值 必须命中 差值约束包
写回目标实例特征当前值
解析实例特征命中抽象特征
```

若只提供 `目标执行后值`：

```text
执行后值 = 目标执行后值
实际执行差值 = 目标执行后值 - 执行前值
```

I64 加法和差值计算必须做溢出保护。默认溢出策略为不允许溢出；若启用饱和写入，必须在运行事实中记录发生溢出。

### 8.5 差值约束

差值约束用区间表达：

| 语义 | 区间 |
| --- | --- |
| 等于零 | `[0,0]` |
| 大于零 | `[1,I64_MAX]` |
| 小于零 | `[I64_MIN,-1]` |
| 任意差值 | `[I64_MIN,I64_MAX]` |

`不等于零` 不是单一区间，不得强行写成假区间。

V1 应表达为：

```text
不等于零 =
  任一命中(
    小于零 [I64_MIN,-1],
    大于零 [1,I64_MAX]
  )
```

执行时应展开为两个区间：

```text
[I64_MIN,-1] ∪ [1,I64_MAX]
```

### 8.6 输出

成功或失败都必须返回本轮运行虚拟存在。

输出结果场景建议写入：

| 输出 | 说明 |
| --- | --- |
| 方法执行情况 | 执行成功、参数绑定失败或执行失败。 |
| 目标宿主 | 本轮目标宿主。 |
| 目标抽象特征 | 本轮目标抽象特征。 |
| 目标实例特征 | 实际写回的实例特征。 |
| 执行前值 | 从实例特征读取的值。 |
| 执行后值 | 写回后的值。 |
| 实际执行差值 | 执行后值 - 执行前值。 |
| 差值命中状态 | 实际差值是否命中约束。 |
| 是否发生溢出 | I64 计算是否发生溢出。 |
| 当前命中抽象特征 | 写入后命中的抽象区间状态。 |
| 动作动态 | 成功写回时形成的动作动态引用。 |
| 失败原因 | 失败时必须写。 |
| 派生需求 | 可继续推进时尽量写。 |

运行虚拟存在必须保存输入参数场景、输出结果场景、开始时间、结束时间、运行时长、方法虚拟存在和上述执行摘要。

方法虚拟存在只更新长期统计：运行次数、成功次数、失败次数、累计运行时长、平均运行时长、最近运行虚拟存在、最近执行情况。不得把本次执行前值、执行后值当成方法自身固定属性。

## 9. 练习本能函数合同

`练习` 的职责不是泛泛学习，而是让“只有方法头节点或结构不完整的方法”逐步形成可验证的方法结构。

标准目标：

```text
方法头节点
  -> 方法自检结果
  -> 方法结构缺口
  -> 候选方法虚拟存在 / 能力包 / 形参包 / 结果包 / 条件结果对 / 试运行限制包
  -> 试运行事实
  -> 条件结果对统计和值域区间
  -> 稳定条件结果对
```

`练习` 有两个模式：

| 模式 | 适用情况 | 行为 |
| --- | --- | --- |
| 自检补全模式 | 方法只有头节点，或方法虚拟存在、能力包、形参包、结果包、条件结果对缺失。 | 创建或补齐候选结构，并写 `方法自检结果` 与 `方法结构缺口`。这些结构只能标记为候选，不得直接当稳定事实。 |
| 试验学习模式 | 方法已有可调用执行入口、形参包和能力包，但条件结果对缺少事实或不稳定。 | 根据形参包和值域生成实参候选，构建输入参数场景，调用目标方法，收集运行事实，更新条件结果对和区间统计。 |

标准输入：

```text
输入参数场景
  输入条件包
    目标方法头节点
    目标自我能力值，可选
    来源缺口
    来源需求节点
    学习目标类型
    试运行次数
    实参生成策略
    采样限制包
    提交阈值
```

标准输出：

```text
输出结果场景
  学习尝试结果
    学习状态
    目标方法头节点
    目标方法虚拟存在
    目标自我能力值
    方法自检结果
    生成的候选能力包
    生成的候选形参包
    生成的候选结果包
    生成的候选条件结果对
    试运行次数
    成功次数
    失败次数
    派生需求
```

方法自检至少检查：

| 检查项 | 缺失时形成的缺口 |
| --- | --- |
| 方法虚拟存在 | 方法虚拟存在缺失。 |
| 自我能力枚举值 | 自我能力值缺失。 |
| 方法能力包 | 能力包缺失。 |
| 形参包 | 形参包缺失。 |
| 结果包 | 结果包缺失。 |
| 条件结果对 | 条件结果对缺失。 |
| 试运行限制包 | 试运行限制包缺失。 |
| 历史运行事实与统计 | 运行事实缺失或稳定性不足。 |
| 稳定区间或候选区间 | 区间缺失或无法验证。 |

若目标方法没有可调用执行入口，`练习` 只能做自检补全和派生需求，不得伪造试运行结果。可调用入口包括有效本能能力值、外部实现句柄或动作序列句柄。仅有方法名称或动作显示文本不算执行入口。

候选结构的状态必须写为候选或未验证。只有实际运行事实长期支持，条件结果对才能从候选升格为稳定。

执行阶段调用目标方法前也必须做方法结构自检。执行前自检发现目标方法只有头节点或缺少可调用执行入口时，应写 `方法自检结果`、`方法结构缺口`，并把当前任务阶段写为阻塞，缺口类型写为方法结构缺口。执行前自检可以记录能力包、形参包、结果包、条件结果对或试运行限制包缺失，但不应在执行阶段直接补造候选结构；候选结构由 `练习` 负责补全。

## 10. 旧变化动作兼容规则

旧的安全值、服务值专用动作可以保留为交互名称或历史入口，但执行语义应逐步统一进入 `改变特征值`。

旧逻辑：

```text
当前值 + / - 改变量
```

新版执行口径：

```text
执行前值 = 从目标实例特征读取
执行后值 = 执行前值 + 执行差值
实际执行差值 = 执行后值 - 执行前值
实际执行差值命中差值约束区间后写回
```

旧入口若仍接收 `改变量`，必须先在配对参数阶段转换为明确的 `执行差值` 或 `目标执行后值`，再调用通用 `改变特征值`。

## 11. 失败与派生需求

失败不应静默。常见失败原因与派生需求：

| 失败原因 | 派生需求 |
| --- | --- |
| 输入条件包绑定失败 | 重新构建输入条件包。 |
| 目标实例特征缺失 | 实例化目标特征。 |
| 执行前值缺失 | 确定目标实例特征当前值。 |
| 缺少执行差值或目标执行后值 | 确定执行差值或目标执行后值。 |
| 执行差值与目标值冲突 | 重新配对参数或调整目标值。 |
| 实际差值未命中约束 | 重新配对参数或调整目标值。 |
| 目标特征写入失败 | 修复实例特征写入能力。 |
| 抽象区间未命中 | 观测并升格区间或扩展抽象特征区间。 |
| 方法结构缺口 | 由 `练习` 补齐候选结构，或派生建立执行入口、能力包、形参包、条件结果对的子需求。 |

派生需求也必须用特征类型和特征值表达，不用名称表达。

## 12. 禁止事项

1. 本能函数不得返回单个特征节点作为标准结果。
2. 本能函数不得把输出结果直接塞进函数返回值。
3. 本能函数不得把任务提供的输入参数场景当成方法自身条件场景。
4. 本能函数不得把方法自身结果场景当成本轮输出结果场景。
5. 本能函数不得在确保阶段提前返回成功。
6. 本能函数不得跳过输出阶段。
7. `查找方法` 不得按方法名称、动作名称、显示文本、语素文本或函数名称匹配。
8. `查找方法` 不得直接执行选中方法或生成可执行输入参数场景。
9. `改变特征值` 不得从输入条件包读取当前值作为执行前值。
10. `改变特征值` 不得把抽象特征当成实例特征写当前值。
11. `改变特征值` 不得缺省补执行差值或目标执行后值，唯一例外是差值约束包为 `[0,0]` 的保持场景。
12. `不等于零` 不得表达成单个 I64 区间。
13. `练习` 不得在没有执行入口时试运行目标方法。
14. `练习` 不得把候选能力包、候选形参包或候选条件结果对直接升格为稳定事实。

## 13. 最小实现检查表

新增或修改本能函数时，至少检查：

| 检查项 | 要求 |
| --- | --- |
| 函数形态 | 返回 `存在节点类*`，三个入参固定。 |
| 四模块 | 方法规格、输入绑定、执行或匹配、输出事实齐备。 |
| 方法能力包 | 可被查找的方法必须声明能力包。 |
| 输入条件包 | 优先使用 `输入条件包`，兼容期才回退 `条件集合`。 |
| 绑定校验 | 必需输入缺失时写参数绑定失败。 |
| 当前值来源 | 执行前值必须来自目标实例特征。 |
| 差值计算 | 使用执行差值或目标执行后值，计算实际执行差值。 |
| 差值约束 | 实际差值必须命中约束区间集合。 |
| 实例写回 | 只写目标实例特征，不写抽象特征。 |
| 区间命中 | 写回后触发抽象区间命中解析。 |
| 运行账本 | 返回运行虚拟存在并写入输入、输出、执行摘要和失败原因。 |
| 输出场景 | 写入功能结果和执行情况。 |
| 方法统计 | 更新运行次数、成功次数、失败次数、累计和平均运行时长。 |
| 编译验证 | 修改后必须通过工程编译。 |
