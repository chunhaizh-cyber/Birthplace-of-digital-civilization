# 2100 方法信息自然语言表达规范

## 1. 目标

本文用于固定当前工程中：

- `方法主信息类` 如何转换为自然语言
- 方法信息在语言层应承担哪些表达职责
- 方法如何与条件场景、结果场景、来源任务、期望结果状态、动作句柄结合
- 方法在“能做什么、在什么条件下能做、会产出什么结果、来源于哪项任务并如何挂接执行”这几类表达中应如何落句

本文重点回答：

- 方法信息到底在语言上描述什么
- 方法应优先落成哪些句型
- 方法首节点、条件节点、结果节点在语言层分别有什么意义
- 动作名、动作句柄、期望结果状态在语言层分别承担什么职责

本文服从以下硬边界：

1. `方法信息` 仍是真值层中的能力、条件、结果与执行挂接事实，不是语言层主观口号。
2. 自然语言只是显示层和解释层，不得反向改写方法真值。
3. 方法表达可以引用条件、结果、任务与动作句柄，但不得把方法本体混同为任务、动态或门控结论。
4. `能做什么` 不等于 `当前允许做什么`，是否允许仍必须回到重判、门控与允许边界。


## 3. 核心口径

必须固定：

1. `方法信息` 描述的是：`某项可执行能力在什么条件下可挂接、预期会导向什么结果、当前由什么动作句柄承载`。
2. 在自然语言中，方法优先承担四类职责：
   - `能做什么`
   - `在什么条件下能做`
   - `会产出什么结果`
   - `来源于哪项任务并如何挂接执行`
3. 方法不是任务本身，也不是当前动态本身，更不是门控放行结论。
4. 方法表达必须优先保留动作名和方法节点种类。

一句话固定：

`方法信息是真值层中的能力与执行挂接事实，在语言层主要回答“能做什么、在什么条件下能做、会产出什么结果、来源于哪项任务并如何挂接执行”。`


## 4. 依据的字段面

当前 `方法信息基类 / 方法主信息类` 可直接提供的关键字段包括：

- `描述信息`
- `节点种类`
- `方法虚拟存在`
- `条件场景`
- `结果场景`
- `动作名`
- `动作句柄`
- `来源任务`
- `父方法`
- `前置方法`
- `后续方法`
- `期望结果状态`
- `最近执行时间`
- `最近完成时间`

当前统一理解为：

- `动作名`：回答“这个方法能做什么”
- `节点种类`：回答“它是方法首节点、条件节点还是结果节点”
- `条件场景`：回答“在什么条件或场景下适用”
- `结果场景`：回答“通常导向哪个结果场景”
- `期望结果状态`：回答“希望产出什么状态”
- `动作句柄`：回答“当前通过什么方式执行”
- `来源任务`：回答“这个方法来自哪项任务承接或归位”
- `父方法 / 前置方法 / 后续方法`：回答“这个方法在方法树里与谁相连”
- `最近执行时间 / 最近完成时间`：回答“最近何时执行、何时完成”


## 5. 方法信息的四类语言职责

### 5.1 能做什么

这一层回答：

- `这个方法能做什么`
- `当前有哪些能力候选`

推荐句型：

- `X 能 A`
- `X 具有方法 A`
- `当前可用方法为 A`

示例：

- `自我能读取核心状态`
- `自我具有停止当前试探并进入保守态的方法`


### 5.2 在什么条件下能做

这一层回答：

- `这个方法适用于什么条件`
- `这个方法在哪个场景下挂接`

推荐句型：

- `在 C 条件下，X 能 A`
- `A 适用于 C`

示例：

- `在自我内部世界中，自我能读取核心状态`
- `停止当前试探并进入保守态适用于高风险场景`


### 5.3 会产出什么结果

这一层回答：

- `这个方法通常导向什么结果`
- `这个方法预期会得到什么状态`

推荐句型：

- `A 会产生 R`
- `执行 A 后，预期得到 R`

示例：

- `读取核心状态会产生核心状态快照`
- `执行停止当前试探并进入保守态后，预期得到保守态结果`


### 5.4 来源于哪项任务并如何挂接执行

这一层回答：

- `这个方法来源于哪项任务`
- `当前通过什么动作句柄执行`

推荐句型：

- `A 来源于 T`
- `A 当前通过 H 执行`
- `为了支持 T，当前挂接 A`

示例：

- `停止当前试探并进入保守态来源于任务管理任务`
- `读取核心状态当前通过本能动作句柄执行`


## 6. 最小语义骨架

方法信息最小骨架固定为：

`动作名 + 节点种类 + 条件 + 结果 + 来源任务 + 动作句柄`

其中：

- `动作名` 回答“能做什么”
- `节点种类` 回答“处在哪一层”
- `条件` 回答“何时可用”
- `结果` 回答“通常导向什么”
- `来源任务` 回答“为什么有这项方法”
- `动作句柄` 回答“通过什么执行”

若字段不全，可退化为：

`动作名 + 当前方法种类`

但不得长期只显示孤立动作名。


## 7. 表达主链

方法信息的自然语言主链固定为：

`方法节点 -> 方法表达单元 -> 方法句型 -> 自然语言`

其中方法表达单元最少应包含：

- `动作名称`
- `节点种类文本`
- `条件场景名称`
- `结果场景名称`
- `期望结果状态短语`
- `来源任务名称`
- `父方法名称`
- `前置方法名称`
- `后续方法名称`
- `动作句柄文本`
- `时间文本`

必须固定：

1. `方法节点` 不直接等于最终句子。
2. 语言模块先抽出方法表达单元，再决定用哪类方法句型。
3. 控制面板、日志、调试页统一消费语言模块结果，不各自手拼方法句。


## 8. 句型层

### 8.1 基本能力句

适用条件：

- 只需要表达“能做什么”

推荐模板：

- `X 能 A`
- `当前方法为 A`

示例：

- `自我能读取核心状态`
- `当前方法为停止当前试探并进入保守态`


### 8.2 条件型方法句

适用条件：

- 需要显式保留条件场景

推荐模板：

- `在 C 条件下，X 能 A`
- `A 适用于 C`

示例：

- `在自我内部世界中，自我能读取核心状态`
- `安全高位回落适用于高安全场景`


### 8.3 结果型方法句

适用条件：

- 需要显式保留结果场景或期望结果状态

推荐模板：

- `执行 A 后，预期得到 R`
- `A 会产生 R`

示例：

- `执行按双值重算服务衰减步长后，预期得到新的衰减步长`
- `安全低位回升会产生更高的风险安全状态`


### 8.4 方法树衔接句

适用条件：

- 需要表达父、前置、后续方法关系

推荐模板：

- `A 之后接 B`
- `A 从属于 P`
- `A 依赖 Q`

示例：

- `方法条件节点之后接方法结果节点`
- `当前方法从属于本能方法树`


### 8.5 来源任务与执行挂接句

适用条件：

- 需要保留来源任务或动作句柄

推荐模板：

- `A 来源于 T`
- `A 当前通过 H 执行`
- `为了支持 T，当前挂接 A`

示例：

- `读取核心状态来源于任务管理任务`
- `读取核心状态当前通过本能函数句柄执行`


### 8.6 带时间方法句

适用条件：

- 需要保留最近执行或最近完成时间

推荐模板：

- `A 最近在 T 执行`
- `A 最近在 T 完成`

示例：

- `读取核心状态最近在本周期执行`
- `安全高位回落最近在上一周期完成`


## 9. 节点种类规则

必须固定：

1. `方法首节点` 优先表达为能力入口。
2. `方法条件节点` 优先表达为适用条件或约束。
3. `方法结果节点` 优先表达为预期结果或结果分支。
4. 语言层不得把这三类节点混成同一种普通方法。

推荐理解：

- `方法首节点` 回答“能做什么”
- `方法条件节点` 回答“在什么条件下能做”
- `方法结果节点` 回答“会导向什么结果”


## 10. 能力与允许边界规则

必须固定：

1. `方法信息` 主要表达能力与挂接，不直接裁决是否允许执行。
2. `X 能 A` 不等于 `X 当前允许执行 A`。
3. 真正的允许性仍必须回到：
   - `根层重判`
   - `执行前门控`
   - `允许边界`

推荐句型：

- `X 能 A`
- `X 当前可以 / 暂不能 A`

但第二类句型只有在明确读到门控或权限结果后才能使用，不得仅凭方法存在就生成。


## 11. 与任务信息、动态信息、状态信息的分工

### 11.1 与任务信息的关系

- `任务信息` 负责“为了满足需求准备做什么”
- `方法信息` 负责“通过什么能力去做”

一句话固定：

`任务` 说承接，`方法` 说能力。`


### 11.2 与动态信息的关系

- `方法信息` 负责“能力是什么”
- `动态信息` 负责“这项能力刚刚怎样被执行”

因此：

- 方法句不能直接冒充动态句
- 动态句可以引用方法动作名


### 11.3 与状态信息的关系

- `方法信息` 负责“预期得到什么状态”
- `状态信息` 负责“现在已经是什么状态”

因此：

- `期望结果状态` 是方法目标面
- `结果状态` 仍应通过状态信息单独表达


## 12. 推荐示例

### 12.1 基本能力

- `自我能读取核心状态`
- `当前方法为停止当前试探并进入保守态`


### 12.2 条件与结果

- `在自我内部世界中，自我能读取核心状态`
- `执行读取核心状态后，预期得到核心状态快照`


### 12.3 来源与挂接

- `停止当前试探并进入保守态来源于任务管理任务`
- `读取核心状态当前通过本能函数句柄执行`


### 12.4 方法树衔接

- `方法条件节点之后接方法结果节点`
- `当前方法从属于本能方法树`


## 13. 工程落点

当前工程中，这部分能力应归属于 `语言模块`，而不是散落在控制面板或各服务类中直接手拼。

最小接口建议固定为：

- `生成方法信息名称(...)`
- `生成方法信息自然语言(...)`
- `生成方法信息摘要(...)`

必须固定：

1. 控制面板只消费语言模块结果，不自行拼装方法句。
2. 同一套方法语言规则应同时服务控制面板、日志与后续调试输出。
3. 若当前只实现最小命名链，也应保证方法节点至少能落成 `X 能 A` 或 `A 来源于 T` 这两类句型。
4. 后续 `2030` 中的 `能什么` 问句，应优先复用本规范的方法表达结果。
