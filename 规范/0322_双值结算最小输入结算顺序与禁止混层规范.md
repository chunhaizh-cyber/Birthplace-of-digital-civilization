# 0322 双值结算最小输入结算顺序与禁止混层规范

## 1. 目标

本文用于把 `双值时间规则` 从“方向规则”收成“每周期可执行骨架”。

本文只回答以下问题：

- 每个周期最小要拿到哪些输入，才能合法进入双值时间结算
- 双值时间规则必须按什么固定顺序结算
- 哪一步只负责数值计算，哪一步可以触发边界
- 哪一步必须生成 `本能动作动态`
- 双值时间规则与 `根层重判 / 执行前门控 / 外层双值回写` 的责任边界如何切开

本文不负责：

- 具体 Tick 长短
- 具体频率
- 最终常量是否微调
- 执行结果回流后的更细双值增量公式

## 3. 核心结论

必须固定以下结论：

1. 低安全时，对服务衰减的压制，仅用于防止双值共振塌陷，不构成继续推进许可。
2. 是否继续推进，必须由 `根层重判 + 执行前门控 + 允许边界` 共同决定。
3. 局部步长函数不得直接代决 `继续 / 待机 / 收束 / 停止`。
4. 本文中的第 `1 ~ 6` 步属于 `内部先天动作`，必须由 `自我线程` 每周期自动调用一次。
5. 第 `1 ~ 6` 步每次调用都必须生成一条对应的 `本能动作动态`；即使本轮没有引起数值变化，也仍应保留动作事实动态。
6. 第 `7` 步属于 `外层裁决接口`，不是局部步长函数，不得被降格成单纯数值修正。
7. `服务值 = 0` 进入待机边界，不等于死亡。
8. `安全值 = 0` 才是死亡退出边界。
9. 边界解除后，系统必须重新回到服务主线，不得永久滞留在保守态。

一句话收口：

`0322` 固定的是“双值时间规则每周期怎么跑”，不是“局部步长函数是否允许续跑”；续跑与否仍由外层重判和门控裁决。


## 4. 每周期最小输入面

### 4.1 最小稳定输入

双值时间结算每周期最少必须能取得以下稳定输入：

- `当前服务值`
- `当前安全值`
- `当前服务时序衰减步长`
- `当前安全增加步长`
- `当前安全减少步长`
- `当前待机状态`
- `当前是否已命中死亡退出边界`

### 4.2 固定阈值输入

双值时间结算还必须能取得以下固定阈值或等价常量：

- `服务高位阈值`
- `安全低位回归锚点`，即 `50%`
- `安全高位回归锚点`，即 `80%`
- `最小非零安全保底线`

### 4.3 外层交接输入

到达第 `7` 步时，必须能够把当前周期输出交接给外层：

- `根层重判`
- `执行前门控`
- `允许边界`

这些外层输入的更细字段仍按 [0315_根层重判结果与执行前允许边界规范.md](D:\鱼巢\规范\0315_根层重判结果与执行前允许边界规范.md) 和 [0317_执行前门控最小输入与判定顺序规范.md](D:\鱼巢\规范\0317_执行前门控最小输入与判定顺序规范.md) 执行。


## 5. 固定结算顺序

双值时间规则每周期的最小固定顺序必须为：

```text
1. 按双值重算服务衰减步长
2. 按服务值重算安全增加步长
3. 按服务值重算安全减少步长
4. 安全低位回升 / 安全高位回落
5. 定时衰减服务值
6. 服务归零待机保护
7. 根层重判 + 执行前门控
```

不得跳步、换序或把边界裁决提前塞进局部步长函数。

### 5.1 步骤责任表

| 步骤 | 对应动作 / 接口 | 主输入 | 主输出 | 是否只算数值 | 是否允许触发边界 | 是否必须生成本能动作动态 |
| --- | --- | --- | --- | --- | --- | --- |
| 1 | `按双值重算服务衰减步长` | 当前服务值、当前安全值、当前服务时序衰减步长 | 服务时序衰减步长 | 是 | 否 | 是 |
| 2 | `按服务值重算安全增加步长` | 当前服务值、当前安全增加步长 | 安全增加步长 | 是 | 否 | 是 |
| 3 | `按服务值重算安全减少步长` | 当前服务值、当前安全减少步长 | 安全减少步长 | 是 | 否 | 是 |
| 4 | `安全低位回升 / 安全高位回落` | 当前安全值、安全增加步长、安全减少步长 | 当前安全值 | 是 | 否 | 是 |
| 5 | `定时衰减服务值` | 当前服务值、当前服务时序衰减步长 | 当前服务值 | 是 | 否 | 是 |
| 6 | `服务归零待机保护` | 当前服务值、当前安全值、当前待机状态 | 待机状态、最小非零安全保底线 | 否 | 只允许触发待机边界，不允许判死 | 是 |
| 7 | `根层重判 + 执行前门控` | 当前双值、本轮边界状态、外层允许边界输入 | `继续 / 待机 / 收束 / 停止` 等最终允许结果 | 否 | 是 | 否 |

### 5.2 顺序语义

必须固定：

1. 第 `1 ~ 3` 步只负责重算步长，不负责给出继续推进许可。
2. 第 `4 ~ 5` 步只负责推进当前周期的双值时间变化，不负责决定下一轮是否继续推进。
3. 第 `6` 步只负责把 `服务值 = 0` 收口到待机边界，不负责宣告死亡退出。
4. 第 `7` 步才负责把当前双值结果送入外层边界裁决。
5. `安全值 = 0` 的死亡退出语义，必须在第 `7` 步进入外层裁决口，不得被误写成第 `6` 步的待机保护。


## 6. 本能动作动态要求

本文中的内部先天动作清单必须继续固定为：

- `按双值重算服务衰减步长`
- `按服务值重算安全增加步长`
- `按服务值重算安全减少步长`
- `安全低位回升`
- `安全高位回落`
- `定时衰减服务值`
- `服务归零待机保护`

其中必须固定：

1. 每个周期都必须调用一次属于本轮适用域的内部先天动作。
2. 每次调用都必须生成一条 `本能动作动态`。
3. 若本轮没有引起主结果变化，只记录动作事实动态，不得伪造变化状态。
4. 若本轮引起主结果变化，动作动态必须回带主结果状态。
5. 第 `7` 步属于外层裁决接口，应生成对应的重判/门控记录，但不计入 `本能动作动态`。


## 7. 与边界裁决的关系

必须固定：

1. 低安全时压制服务衰减，只是防瘫，不是放行。
2. `服务值 = 0` 时，只能进入待机边界，不得直接判死。
3. `安全值 = 0` 时，必须进入死亡退出边界。
4. 是否允许继续推进，必须交由 `根层重判 + 执行前门控 + 允许边界` 共同裁决。
5. 边界解除后，系统必须重新回到服务主线，不得永久滞留在保守态。


## 8. 与外层双值回写的关系

本文固定的是 `双值时间规则的周期结算接口`，不替代“执行结果回流后的外层双值回写”。

因此必须继续固定：

1. 本文第 `1 ~ 7` 步只负责时间本能、边界保护和外层交接。
2. 执行结果回流后的 `实际结果方向 -> 需求方向 -> 合法双值增量` 仍按 [0321_实际结果方向与需求方向对齐判定表规范.md](D:\鱼巢\规范\0321_实际结果方向与需求方向对齐判定表规范.md)、[0323_双值结算增量合法来源与最小类型表规范.md](D:\鱼巢\规范\0323_双值结算增量合法来源与最小类型表规范.md)、[0324_零增量与负面后果边界规范.md](D:\鱼巢\规范\0324_零增量与负面后果边界规范.md) 执行。
3. 动作线程、学习线程、叶子执行器、局部治理函数，仍不得直接改写最终双值。


## 9. 严格禁止的混层

以下做法必须禁止：

1. `低安全压住服务衰减 -> 直接视为允许继续推进`
2. `服务值 = 0 -> 直接判死`
3. `安全值 = 0 -> 仍按普通待机处理`
4. `局部步长函数 -> 直接决定继续 / 待机 / 收束 / 停止`
5. `第 1 ~ 6 步未跑完 -> 先做外层裁决`
6. `第 6 步服务归零待机保护 -> 越权代替第 7 步`
7. `动作线程 / 学习线程 / 局部治理函数 -> 私自改写最终双值`
8. `执行结果局部成功 -> 跳过外层比较与重判，直接续跑下一轮`


## 10. 当前阶段工程结论

当前阶段先固定两件事即可：

1. 把 `0311` 收成“解释条款 + 固定顺序 + 责任边界”清楚的方向规则。
2. 把本文收成“自我线程每周期该怎么跑双值时间规则”的最小接口规范。

本阶段不先追问“几分钟、几小时、多少 Tick”，而先确保：

- 低安全时，服务衰减被压住
- 服务归零进入待机，不判死
- 安全归零才进入死亡退出边界
- 边界解除后能回到服务主线
