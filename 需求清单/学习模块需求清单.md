## 3. 当前代码已形成的 V0 闭环

按当前代码，学习模块已经形成下面这条 V0 主链：

`任务管理一步治理`
-> `创建或复用学习子任务`
-> `创建或复用学习方法骨架`
-> `同步到学习任务账本`
-> `学习对象适配`
-> `学习调度快照与就绪/等待划分`
-> `共享学习执行器推进 采样/更新/验证/提交/回滚`
-> `提交学习方法资产`
-> `自我线程外层裁决`
-> `控制面板/摘要/日志回读`

当前这条链已经说明：

- 学习不再只是零散钩子，而是进入了 `任务管理 -> 学习账本 -> 调度 -> 执行 -> 写回 -> 外层裁决` 的闭环。
- 但它仍然主要是 `定向学习 + 共享执行器 + V0 账本状态面` 的实现，还没有完整收口到规范要求的全部字段、分型、回流策略和长期学习结构。

## 4. 必须遵守的硬边界

以下 4 条不是“建议”，而是学习模块必须长期服从的硬边界：

1. 学习模块属于 `任务管理内部治理子任务链`，不是第三根任务。
2. 学习结果不能改写根目的、根方向、根评价标准和根生死规则。
3. 学习执行器只有承接权，没有最终裁决权；`允许回流 / 允许升格 / 允许继续推进` 必须回外层重判。
4. 对 `自我真值` 或 `现实对象` 有影响的学习，不得裸执行；只能走 `镜像验证 / 受控微试探 / 审计后提交` 的受控路径。

当前代码对应的已对齐点：

- `学习任务` 已由 [`../任务管理任务模块.cpp`](../任务管理任务模块.cpp) 直接创建和同步，而不是从根层独立平行出生。
- [`../学习执行模块.cpp`](../学习执行模块.cpp) 已按 `影响面` 切分成 `镜像验证 / 受控微试探 / 正式资产提交`。
- [`../自我线程类.cpp`](../自我线程类.cpp) 已对已提交学习结果再做一次 `外层裁决`，没有让学习执行器直接拥有正式消费权。

   - 补齐 `来源根任务 / 来源任务管理任务 / 来源功能域 / 来源方法需求位 / 上浮压力类型`。

3. `学习目标字段`
   - 补齐 `学习目标类型 / 学习类型 / 目标约束强度 / 目标作用域`。

4. `回流策略与结果归属范围`
   - 补齐 `直接回流 / 匹配后回流 / 仅沉淀资产` 和 `当前任务 / 当前根任务 / 同需求位任务族 / 全局方法资产`。

5. `管理广泛学习任务与广泛学习分支`
   - 在已有 `任务管理学习树` 下补出 `任务管理广泛学习子任务` 分支和创建逻辑。

6. `1218 第一批结果字段`
   - 补齐 `结果符合次数增量 / 结果不符合次数增量 / 稳定复现次数增量 / 结果分裂次数增量 / 禁用条件数量 / 回退条件数量 / 条件模板补全数量 / 结果模板补全数量 / 本轮补缺已达成`。

7. `优先级合成升级`
   - 把当前学习优先级从“能不能学”扩到“该不该先学”。

8. `显式合并执行组`
   - 保留独立任务实体前提下，真正落地 `可合并执行`，而不是只写 `可合并键`。

## 8. 中期收口项

在 P0 之后，建议按下面顺序继续收口：

1. `统一学习对象选择器`
   - 把目前散在任务管理和调度里的对象选择，收成显式选择器。

2. `稳定任务路径适配器`
   - 新增路径摘要器，把路径原料收口成待学习方法骨架。

3. `稳定任务子树适配器`
   - 新增子树摘要器，把结构原料收口成待学习方法骨架。

4. `长期储备学习任务`
   - 让 `学习目的 -> 储备学习任务 -> 单轮学习工作` 真正落地。

5. `成长验收指标`
   - 建立“学习后是否减少默认兜底、减少重筹办、提高完成率”的回看指标。

## 9. 建议实现顺序

建议后续实现顺序固定为：

1. 先补字段面
   - 先把 `谁的学习、学什么、学完回给谁` 写清楚。

2. 再补结果面
   - 先把 `证明 / 找边界 / 补缺口` 写成可落账字段。

3. 再补结构分型
   - 把 `定向学习` 和 `管理广泛学习` 分开。

4. 再补调度增强
   - 把优先级、等待唤醒、合并执行组补齐。

5. 最后补长期学习和成长验收
   - 把“会学习”真正推进到“开始成长”。

## 10. 收口结论

当前学习模块已经不是空骨架，而是一个 `能出生、能排队、能执行、能写回、能观察、能外层裁决` 的 V0 学习闭环。

但若要真正达到规范口径，还必须继续补齐三件大事：

1. `把学习任务语义写完整`
   - 重点是分型、来源、目标、回流。

2. `把学习结果结算写完整`
   - 重点是证明、找边界、补缺口。

3. `把成长闭环验收写完整`
   - 重点是方法资产增长后的后续复用验证。

当前最准确的判断是：

`学习模块已经具备 V0 运行能力，但仍处于“定向学习主导、结果字段未收口、广泛学习未独立、成长验收未形成”的阶段。`
