# 特征值类说明

本文说明 `D:\鱼巢\特征值类.h`、`D:\鱼巢\特征值主信息类.h` 以及 `D:\鱼巢\基础数据类型.h` 中与特征值直接相关的类型和函数。

## 1. 文件定位

当前项目里“特征值”分成三层：

- `特征值`：轻量快照值，定义在 `基础数据类型.h`，本质是 `std::variant<std::monostate, I64, VecU句柄, 指针句柄>`。
- `特征值主信息类`：真实承载一段 `VecIU64` 向量值及其统计信息。
- `特征值类`：继承 `链表模板<特征值主信息类>`，作为 VecU 特征值证据池，负责去重、缓存、分层索引和句柄生成。

它和新特征系统的关系是：

- 实例特征的当前值使用 `特征值`。
- 抽象特征的区间只使用 `I64区间`，不直接保存 `特征值`。
- 实例稳态可以保存 `特征值`，并可选保存由该值沉淀出的 `I64区间`。
- `VecU句柄` 和 `指针句柄` 不直接进入抽象区间；需要先比较、量化或映射成 `I64`。

简单说：

- 标量 `I64` 直接放在特征节点的当前值里。
- 大型向量 `VecIU64` 放进 `特征值类` 证据池。
- 特征节点只保存 `VecU句柄`，通过句柄回到证据池读取向量。
- 指针特征用 `指针句柄` 表示运行期指针，不建议作为长期持久化事实。
- 抽象特征没有当前值，只有可选 `I64区间` 和组合规则。

## 2. 基础类型

### 2.1 `VecU句柄`

| 字段/函数 | 用途 | 用法和注意 |
| --- | --- | --- |
| `std::uintptr_t 主信息指针` | 指向 `特征值主信息类` 的地址。 | 这是运行期句柄，不是稳定主键。 |
| `有效() const noexcept` | 判断句柄是否非空。 | `主信息指针 != 0` 时为有效。 |
| `operator==` | 比较两个句柄是否指向同一个主信息。 | 只比较指针地址，不比较向量内容。 |

示例：

```cpp
VecU句柄 h = 世界树.值池().获取或创建句柄(向量值);
if (h.有效()) {
    const auto* v = 世界树.值池().取VecU只读指针(h);
}
```

### 2.2 `指针句柄`

| 字段/函数 | 用途 | 用法和注意 |
| --- | --- | --- |
| `std::uintptr_t 指针` | 保存一个运行期指针地址。 | 用于任务、方法、存在等对象挂点。 |
| `有效() const noexcept` | 判断指针是否非空。 | `指针 != 0` 时为有效。 |
| `operator==` | 比较两个指针句柄是否相同。 | 只比较地址。 |

### 2.3 `特征值`

```cpp
using 特征值 = std::variant<std::monostate, I64, VecU句柄, 指针句柄>;
using 特征快照值 = 特征值;
```

| 形态 | 含义 | 典型用途 |
| --- | --- | --- |
| `std::monostate` | 无值。 | 未设置、无当前值。 |
| `I64` | 内联整数值。 | 安全值、服务值、状态、计数、枚举编码、布尔 0/1。 |
| `VecU句柄` | 向量证据池句柄。 | 轮廓、签名、点阵、体素等较大向量值。 |
| `指针句柄` | 指针地址句柄。 | 任务当前步骤、方法挂点、运行期对象引用。 |

### 2.3.1 `结构_基础信息轻引用` 与 `抽象特征引用`

组合规则中使用轻量引用保存抽象特征来源：

```cpp
struct 结构_基础信息轻引用 {
    std::uintptr_t 指针 = 0;
    std::string 主键{};
};

using 抽象特征引用 = 结构_基础信息轻引用;
```

它的用途是让 `基础数据类型.h` 中的组合规则可以引用基础信息节点，同时避免在基础数据类型层直接包含节点类型。运行期优先用指针解析，指针不可用时可使用主键做恢复或匹配。

| `特征快照有值(const 特征快照值& v) noexcept` | `特征快照值` 的有效性判断。 | 当前直接转调 `特征值有效`。 |

### 2.5 `I64区间`

`I64区间` 是抽象特征 V1 的唯一正式区间类型。

| 字段/函数 | 用途 |
| --- | --- |
| `低值` | 区间下界。 |
| `高值` | 区间上界。 |
| `有效()` | `低值 <= 高值` 时有效。 |
| `是否点()` | `低值 == 高值` 时为点区间。 |
| `点值()` | 返回点区间的值。调用方应先确认是点区间。 |
| `包含(I64 v)` | 判断值是否落在闭区间 `[低值, 高值]` 内。 |
| `I64区间::点(I64 v)` | 创建 `[v, v]` 点区间。 |

使用规则：

- 抽象特征区间只接受 `I64区间`。
- 枚举和布尔状态使用点区间，例如 `[1,1]`。
- 连续标量使用闭区间，例如 `[70,80]`。
- `U64` 需要先编码或量化成可比较的 `I64`。
- `VecU` 通常作为证据数组；若要参与区间命中，应先比较生成 `I64` 差值或相似度分值。

### 2.6 `特征稳态`

| 字段 | 用途 |
| --- | --- |
| `特征值 值` | 当前稳定值或代表值。 |
| `std::optional<I64区间> 区间` | 可选稳定区间。 |
| `std::uint64_t 命中次数` | 稳态命中次数。 |
| `时间戳 最后命中` | 最近命中时间。 |

注意：

- 稳态属于实例特征，不属于抽象特征。
- 抽象特征只接收从实例稳态升格出来的区间。
- 没有区间的稳态仍可作为点值或 VecU 原型存在，但不能直接完善抽象区间。

## 7. 特征值类

`特征值类` 继承 `链表模板<特征值主信息类>`，本质是一个 VecU 证据池。

它提供：

- 按向量内容去重。
- 按 FNV-1a 64 位哈希快速定位候选。
- 用 `VecU句柄` 返回稳定的运行期引用。
- 可选的 2D/3D 轮廓金字塔分层索引。

### 7.1 类型别名

| 名称 | 含义 |
| --- | --- |
| `基类` | `链表模板<特征值主信息类>`。 |

| `写锁守卫` | 基类写锁类型。 |
| `特征值节点类` | `链表模板<特征值主信息类>::节点类` 的全局别名。 |

| `调试清空缓存() noexcept` | 清空哈希缓存和分层索引。 | 调试用。不加锁，调用时应保证没有并发访问。 |

| `添加_分层_已加锁(特征值主信息类&& mi, 枚举_轮廓维度 dim)` | 将主信息添加到证据池，并按需登记分层索引。 | 会确保内容哈希，调用基类 `添加节点_已加锁(nullptr, ...)` 添加到根同层链，再更新哈希缓存。 |

## 8. 典型流程

### 8.1 写入 VecU 特征值

```cpp
VecIU64 原始向量 = ...;
VecU句柄 句柄 = 世界树.值池().获取或创建VecU(
    std::move(原始向量),
    枚举_轮廓维度::二维);

特征值 当前值{ 句柄 };
```

### 8.2 读取 VecU 特征值

```cpp
if (auto* 句柄 = std::get_if<VecU句柄>(&当前值)) {
    const VecIU64* 值 = 世界树.值池().取VecU只读指针(*句柄);
    if (值) {
        // 只读使用
    }
}
```

### 8.3 标量特征值

```cpp
特征值 安全值{ static_cast<I64>(100) };
```

标量不进入 `特征值类` 证据池。

标量也是当前抽象区间命中的唯一直接值形态：

```cpp
I64区间 执行中 = I64区间::点(2);
bool 命中 = 执行中.包含(2);
```

## 9. 生命周期和所有权

- `特征值类` 拥有链表节点和 `特征值主信息类` 值对象。
- `VecU句柄` 不拥有任何对象，只保存 `特征值主信息类` 地址。
- 删除或重建证据池后，旧 `VecU句柄` 可能悬空。
- `取VecU只读指针` 不做归属校验，只根据地址转回指针。
- `指针句柄` 也不拥有对象，必须由上层保证指针有效。

## 10. 常见错误

- 把 `VecU句柄` 当成跨进程、跨存档稳定 ID 使用。它只是运行期地址句柄。
- 修改 `取VecU只读指针` 返回的向量内容。该函数返回 `const VecIU64*`，调用方应保持只读。
- 清空或重载证据池后继续使用旧句柄。
- 认为 `调试清空缓存` 会删除证据池节点。它只清缓存和索引，不删链表节点。
- 认为 `初始化()` 会清空数据。它只重建缓存。
- 只比较 `内容哈希` 就认为向量相同。正确逻辑是先按哈希缩小候选，再比较完整 `VecIU64`。
- 把 `VecU句柄` 直接写成抽象特征区间。抽象区间 V1 只支持 `I64区间`。
- 把运行期 `指针句柄` 当作稳定语义本体。需要长期表达时，应转换成抽象特征或可恢复引用。

## 11. 推荐使用

- 普通 VecU 入池使用 `获取或创建VecU` 或 `获取或创建句柄`。
- 已经持有写锁时使用 `_已加锁` 版本，避免重复加锁。
- 外部读取向量使用 `取VecU只读指针`，不要保存可写指针。
- 加载或恢复证据池后调用 `初始化()` 重建缓存。
- 需要轮廓分层索引时明确传入 `枚举_轮廓维度::二维` 或 `枚举_轮廓维度::三维`。
- 能用 `I64` 表达的状态、枚举、布尔和计数，优先用 `I64`，这样可以直接进入抽象特征区间、稳态升格和组合规则。
- 需要表达复杂数组证据时使用 `VecU`，再通过比较结果生成 `I64` 差值或相似度分值，让该分值参与抽象区间。
