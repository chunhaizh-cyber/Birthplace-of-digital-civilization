# 因果类说明

本文说明 `D:\鱼巢\因果类.h`、`D:\鱼巢\因果类.cpp`，并补充 `D:\鱼巢\基础信息主信息类.h` 中 `因果主信息类` 与相关枚举。

`因果类` 是基础信息树中“因果节点”的服务层。现行主路径是：由动态证据生成或归并抽象因果模板，再由二次特征匹配这些模板。它把“状态如何变化”和“动作或语义是什么”组织成可复用的因果信息。

## 1. 文件定位

当前项目中，因果节点位于动态和方法/任务推断之间：

- 状态节点记录某一刻的事实。
- 动态节点记录状态变化或动作事实。
- 二次特征节点记录状态迁移、动态模式等可比较特征。
- 因果节点把条件模板、动作模板、状态迁移模板和证据动态样本组织在一起。

简单说：

- 动态是事实样本。
- 二次特征是可比较的变化模式。
- 因果是从这些样本中归结出来的“通常什么导致什么”。

当前实现以抽象因果模板为主。`因果实例主信息类` 和 `因果模板主信息类` 只是 `因果主信息类` 的源码层别名，不再是独立运行时主信息壳。

## 2. 因果相关枚举

这些枚举定义在 `基础信息主信息类.h`。

### 2.1 `枚举_因果锚点类型`

| 枚举值 | 值 | 含义 |
| --- | --- | --- |
| `未定义` | `0` | 未指定适用锚点。 |
| `场景` | `1` | 因果模板适用于场景层面的动态或二次特征。 |
| `存在` | `2` | 因果模板适用于某个存在或主体层面的动态。 |

### 2.2 `枚举_因果来源类型`

| 枚举值 | 值 | 含义 |
| --- | --- | --- |
| `未定义` | `0` | 来源未定义。 |
| `观察生成` | `1` | 由观察到的动态生成。 |
| `执行生成` | `2` | 由方法或动作执行过程生成。 |
| `推断生成` | `3` | 由推断过程生成。 |

### 2.3 `枚举_因果形态`

| 枚举值 | 值 | 含义 |
| --- | --- | --- |
| `未定义` | `0` | 形态未定义。 |
| `实例` | `1` | 实例因果。当前没有完整创建和枚举主路径。 |
| `抽象` | `2` | 抽象因果模板。当前主要使用这一形态。 |

## 5. `因果类`职责

`因果类` 封装以下能力：

1. 绑定和访问 `基础信息类`。
2. 读取因果、实例因果、模板因果主信息。
3. 创建抽象因果模板。
4. 按动态创建或归并抽象因果信息。
5. 删除因果节点。
6. 枚举因果模板。
7. 追加条件模板、结果状态、证据动态样本。
8. 绑定动作主体和动作语义。
9. 用二次特征匹配因果模板。
10. 生成因果自然语言。

## 6. 基础信息绑定与主信息读取

### 6.1 `因果类(基础信息类* 基础信息 = nullptr)`

用途：

- 构造因果服务对象。
- 保存基础信息指针。

说明：

- 不拥有 `基础信息类` 生命周期。
- 如果传入空指针，后续大多数操作会直接失败或返回空。

### 6.2 `绑定基础信息(基础信息类* 基础信息)`

用途：

- 给已有 `因果类` 对象重新绑定基础信息。

说明：

- 只写入内部指针。
- 不迁移、不扫描已有因果节点。

### 6.3 `取因果主信息(...)`

用途：

- 读取任意因果节点的 `因果主信息类`。

返回：

- 成功返回主信息指针。
- 基础信息未绑定、节点为空或主信息类型不匹配时返回 `nullptr`。

### 6.4 `取实例主信息(...)`

用途：

- 读取实例因果主信息。

规则：

- 先读取 `因果主信息类`。
- 只有 `形态 == 实例` 时返回。

注意：

- 当前 `枚举因果实例()` 返回空列表。
- 现行代码没有实例因果的完整创建主路径。

### 6.5 `取模板主信息(...)`

用途：

- 读取抽象因果模板主信息。

规则：

- 先读取 `因果主信息类`。
- 只有 `形态 == 抽象` 时返回。

## 7. 创建和删除

### 7.1 `创建因果模板(父节点, 主信息)`

用途：

- 在指定父节点下创建一个抽象因果模板节点。

执行流程：

1. 检查 `基础信息_` 和 `主信息`。
2. 如果 `主信息->形态` 不是抽象，则改成 `抽象`。
3. 调用 `基础信息_->添加子节点(父节点, 主信息)`。
4. 返回新建的 `因果模板节点类*`。

注意：

- 这个函数不会自动挂到“因果信息”根节点下。
- 调用方需要自己传入合适父节点。
- 主信息交给基础信息树接管。

### 7.2 `删除因果(节点)`

用途：

- 删除因果节点。
- 如果父节点是场景，则从场景主信息的 `实例因果索引` 中移除该节点引用。

执行流程：

1. 检查基础信息和节点。
2. 读取父节点。
3. 如果父节点主信息是 `场景节点主信息类`，清理 `实例因果索引` 中同指针或同主键引用。
4. 调用 `基础信息_->删除节点(节点)`。

注意：

- 当前抽象因果模板主要挂在顶层“因果信息”根节点下面，不通过场景 `实例因果索引` 管理。
- 删除函数只清理直接父场景的实例索引，不会扫描所有场景。

## 8. 按动态创建因果信息

### 8.1 `按动态创建因果信息(...)`

签名：

```cpp
因果模板节点类* 按动态创建因果信息(
    场景节点类* 场景,
    动态节点类* 动态节点,
    枚举_因果来源类型 来源类型 = 枚举_因果来源类型::观察生成,
    基础信息节点类* 动作主体 = nullptr,
    基础信息节点类* 动作语义 = nullptr,
    const std::string& 动作语义键 = {},
    bool 设为主结果 = true);
```

用途：

- 根据一个动态节点创建或归并抽象因果模板。

必要条件：

- `基础信息_` 已绑定。
- `动态节点` 存在。
- 动态主信息存在。
- 能解析到场景。
- 动态必须有 `初始状态` 和 `结果状态`。
- 初始状态和结果状态能生成状态迁移二次特征。

执行流程：

1. 读取动态主信息。
2. 如果调用方没有传入场景，则从动态父节点解析所属场景。
3. 读取动态的初始状态和结果状态。
4. 通过 `世界树.二次特征生成().由状态比较生成二次特征` 生成状态迁移二次特征。
5. 解析动作主体、动作语义和动作模板。
6. 生成一个临时 `因果主信息类 原型`。
7. 将初始状态的状态特征加入 `原型.条件模板`。
8. 将状态迁移二次特征加入 `原型.状态迁移模板`。
9. 设置 `原型.主状态迁移`。
10. 设置来源类型、置信度、已验证标记。
11. 调用内部逻辑查找、合并或创建抽象因果模板。

原型字段写入规则：

| 字段 | 写入来源 |
| --- | --- |
| `名称` | 动态名称，或动态来源动作名。 |
| `类型` | 动态类型。 |
| `适用锚点类型` | 动态有主体时为 `存在`，否则为 `场景`。 |
| `适用层级` | 动态层级。 |
| `条件模板` | 初始状态的状态特征。 |
| `动作模板` | 动作语义、来源动作、动作主体或动态特征。 |
| `动作名称` | 来源动作名、来源动作短名或动作模板短名。 |
| `动作语义键` | 调用方传入的动作语义键。 |
| `状态迁移模板` | 初始状态到结果状态的二次特征。 |
| `来源类型` | 调用方传入。 |
| `置信度` | 固定写入 `10000`。 |
| `已验证` | 动态主信息的 `来源执行成功`。 |

### 8.2 动作模板解析顺序

`私有_取动作模板节点` 的顺序是：

1. 优先使用调用方传入的 `动作语义`。
2. 其次使用动态主信息的 `来源动作`。
3. 再使用调用方传入的 `动作主体`。
4. 最后退回 `私有_动态动作语义节点`。

`私有_动态动作语义节点` 的顺序是：

1. 动态主信息的 `来源动作`。
2. 动态主信息的 `动态特征`。
3. 都没有则返回空。

### 8.3 动作名称解析顺序

`私有_动态动作名称` 的顺序是：

1. 动态主信息的 `来源动作名`。
2. 动态主信息的 `来源动作` 对应基础节点短名。
3. 动作模板对应基础节点短名。
4. 都没有则为空。

## 9. 抽象因果根节点

### 9.1 因果信息根

内部函数 `私有_取或创建抽象因果信息根节点` 会在基础信息树顶层查找或创建一个特殊抽象因果节点。

这个根节点的特征：

- `形态 == 抽象`。
- `名称 == 因果信息`。
- `类型 == 因果信息`。
- 没有动作模板。
- 没有动作名称。
- 没有动作语义键。
- 没有条件模板。
- 没有状态迁移模板。
- 没有证据动态样本。

用途：

- 作为抽象因果模板列表的根节点。
- 不作为真实因果模板参与枚举和自然语言表达。

### 9.2 `枚举因果模板()` 对根节点的处理

`枚举因果模板()` 会枚举全部 `因果主信息类` 节点，但会跳过这个“因果信息”根节点。

## 10. 抽象因果查找、创建和归并

### 10.1 查找已有模板

内部函数 `私有_查找抽象因果模板` 用原型查找已有模板。

查找条件：

1. 必须有“因果信息”根节点。
2. 原型必须能取到主状态迁移主信息。
3. 候选节点必须是抽象因果。
4. 动作模板一致。
5. 主状态迁移可归并。

动作模板一致规则：

- 如果模板和原型都有 `动作名称`，优先比较动作名称。
- 否则比较 `动作模板` 引用键。

状态迁移可归并规则：

- 迁移结构必须一致。
- 如果任一方是值对型迁移，则双方都必须是值对型，且值对一致。
- 否则按方向型迁移比较，方向必须一致。

### 10.2 方向型迁移

方向型迁移通过以下信息推导方向：

1. 优先使用 `差值`。
2. 其次使用 `标量区间`。
3. 再使用 `方向区间`。

方向自洽要求：

- 能推导方向。
- 如果已有 `方向区间`，必须和推导方向一致。
- 如果已有 `差值`，差值方向也必须一致。
- 如果有 `标量区间`，区间必须符合方向。

### 10.3 值对型迁移

值对型迁移的判断条件：

- `离散编码 == 2`。
- 或 `离散编码 == 3`。
- 或 `离散语义键 == "判等型"`。
- 或 `离散语义键 == "差异度型"`。

值对自洽要求：

- 左右状态都存在。
- 左右状态模板结构一致。

值对一致要求：

- 迁移结构一致。
- 模板左状态值等于样本左状态值。
- 模板右状态值等于样本右状态值。

### 10.4 创建新模板

找不到可归并模板时，内部逻辑会：

1. 创建 `因果主信息类(抽象)`。
2. 复制原型的名称、类型、锚点、层级、动作、来源、置信度和验证信息。
3. 获取或创建“因果信息”根节点。
4. 把新模板挂到“因果信息”根节点下。
5. 按动态补齐条件模板、状态模板和状态迁移模板。
6. 写入证据动态样本。
7. 设置 `成立次数 = 1`。
8. 设置最近命中时间。

### 10.5 合并已有模板

找到可归并模板时，内部逻辑会：

1. 补齐模板主信息中缺失的名称、类型、动作、锚点、层级、来源和置信度。
2. 合并 `已验证` 标记。
3. 按动态补齐或扩张抽象因果模板。
4. 如果证据动态未存在，追加到 `证据动态样本`。
5. 累加 `成立次数`。
6. 更新最近命中时间。

## 11. 按动态补齐抽象因果模板

内部函数 `私有_按动态补齐抽象因果模板` 是模板结构落地的核心。

必要条件：

- 基础信息、模板节点、模板主信息和证据动态都存在。
- 动态有初始状态和结果状态。
- 初始状态和结果状态主信息都存在。
- 能得到样本迁移主信息。
- 样本迁移方向或值对自洽。

执行逻辑：

1. 从动态读取初始状态和结果状态。
2. 从原型或状态比较得到样本迁移主信息。
3. 判断样本迁移是值对型还是方向型。
4. 检查样本迁移自洽。
5. 遍历模板已有 `状态迁移模板`。
6. 如果找到同结构迁移，并且可归并，则扩张对应模板。
7. 如果没有同结构迁移，则创建新的条件模板、条件状态模板、结果状态模板和状态迁移模板。

扩张内容：

- 条件模板特征的区间会吸收样本初始状态值。
- 左右状态模板的区间会吸收样本状态值。
- 状态迁移模板的 `标量区间` 会吸收样本差值。

注意：

- 条件模板和状态模板当前以因果模板子节点形式保存。
- 这些模板节点是因果模板内部的抽象样本承载，不应理解为真实存在或场景的特征挂载入口。
- 按最新“特征只能是存在或场景子节点”的建模规则看，这部分属于需要后续审视的模板承载实现。

## 12. 追加和桥接函数

### 12.1 `按动态桥接结果状态(节点, 动态节点, 设为主结果)`

用途：

- 用一个动态补充已有因果节点的条件模板、结果迁移和证据样本。

执行流程：

1. 读取因果主信息和动态主信息。
2. 要求动态有初始状态和结果状态。
3. 把初始状态的状态特征追加为条件模板。
4. 调用 `追加结果状态` 追加结果迁移。
5. 如果因果节点是抽象因果，则追加证据动态样本。

返回：

- 返回 `追加结果状态` 的结果。

### 12.2 `追加条件模板(节点, 条件模板)`

用途：

- 向因果主信息的 `条件模板` 列表追加一个特征模板引用。

规则：

- 因果主信息和条件模板都必须存在。
- 同指针或同主键已存在时不重复追加。

### 12.3 `追加结果状态(节点, 结果状态, 设为主结果)`

用途：

- 根据结果状态为因果节点追加一个状态迁移模板。

执行流程：

1. 读取因果主信息和结果状态主信息。
2. 从已有 `条件模板` 中选择参考条件特征。
3. 优先选择和结果状态特征类型、名称一致的条件特征。
4. 用结果状态主信息和条件特征创建一个参考条件状态模板。
5. 用参考条件状态和结果状态生成状态迁移二次特征。
6. 把迁移特征加入 `状态迁移模板`。
7. 必要时写入 `主状态迁移`。

注意：

- 如果没有可用条件模板，函数返回 `false`。
- 这个函数会在因果模板节点下创建抽象状态模板。
- 它不自动创建因果根，也不自动查找可归并模板。

### 12.4 `绑定动作主体语义(...)`

用途：

- 手动给因果节点绑定动作模板、动作名称和动作语义键。

规则：

- `动作语义` 优先于 `动作主体`。
- 动作名称优先来自动作模板短名。
- 如果没有动作模板，则使用传入的动作语义键。

### 12.5 `追加证据动态样本(节点, 证据动态)`

用途：

- 给抽象因果模板追加证据动态样本。

规则：

- 只有 `取模板主信息` 成功时才追加。
- 同指针或同主键已存在时不重复追加。

注意：

- 这个函数只追加样本引用。
- 它不会累加 `成立次数`。
- `成立次数` 主要在创建或归并抽象因果模板时更新。

## 13. 二次特征匹配

### 13.1 `二次特征匹配模板(二次特征, 模板)`

用途：

- 判断一个二次特征是否满足某个二次特征模板。

实现：

- 直接调用 `二次特征满足_桥接(模板, 二次特征)`。

### 13.2 `因果模板匹配二次特征(模板, 条件候选, 结果候选)`

用途：

- 判断一条因果模板是否被当前二次特征候选命中。

现行规则：

1. 模板主信息必须是抽象因果。
2. `状态迁移模板` 不能为空。
3. 每个 `状态迁移模板` 都必须在 `结果候选` 中找到一个满足模板的二次特征。

注意：

- `条件候选` 参数当前没有参与判断。
- 条件候选由调用方收集，但现行匹配只看结果候选是否覆盖状态迁移模板。

### 13.3 `查找匹配因果模板(...)`

用途：

- 在全部抽象因果模板中查找和当前候选二次特征匹配的模板。

过滤顺序：

1. 枚举全部因果模板。
2. 如果模板限定 `适用锚点类型`，必须和传入锚点一致。
3. 如果模板限定 `适用层级`，必须和传入状态层级一致。
4. 调用 `因果模板匹配二次特征`。
5. 命中则加入结果列表。

## 14. 枚举函数

### 14.1 `枚举全部因果()`

用途：

- 返回全部当前可枚举因果。

现行实现：

- 只返回 `枚举因果模板()` 的结果。
- 不包含实例因果。

### 14.2 `枚举因果实例()`

用途：

- 设计上用于枚举实例因果。

现行实现：

- 返回空列表。

### 14.3 `枚举因果模板()`

用途：

- 枚举基础信息树里所有抽象因果模板。

规则：

- 遍历所有 `因果主信息类` 节点。
- 只保留 `形态 == 抽象` 的节点。
- 跳过特殊“因果信息”根节点。

## 15. 自然语言表达

### 15.1 `生成实例因果表达单元(...)`

用途：

- 把实例因果节点转成 `结构_因果表达单元`。

规则：

- 必须能通过 `取实例主信息`。
- 动作短语来自 `动作名称` 或动作模板短名。
- 条件短语来自 `条件模板`。
- 结果短语来自 `状态迁移模板` 的基础节点短名。
- 主结果短语来自 `主状态迁移`，为空时退回第一个结果短语。

注意：

- 当前实例因果没有完整主路径，因此这个函数更多是兼容接口。

### 15.2 `生成抽象因果表达单元(...)`

用途：

- 把抽象因果模板转成 `结构_因果表达单元`。

规则：

- 跳过特殊“因果信息”根节点。
- 写入成立次数和失败次数。
- 动作短语来自 `动作名称` 或动作模板短名。
- 条件短语来自 `条件模板`。
- 结果短语来自 `状态迁移模板` 的基础节点短名。
- 主结果短语来自 `主状态迁移`，为空时退回第一个结果短语。

### 15.3 `生成实例因果自然语言(...)`

输出模板：

- 有条件、有动作、有结果：
  - `因为[条件]，所以[动作]产生了[结果]`
- 有条件、有结果：
  - `在[条件]下，结果变为[结果]`
- 有动作、有结果：
  - `[动作]使[结果]成立`
- 只有动作：
  - `[动作]`
- 否则：
  - `[结果]`

### 15.4 `生成抽象因果自然语言(...)`

输出模板：

- 有条件、有动作、有结果：
  - `通常在[条件]时，[动作]会导致[结果]（成立 N 次）`
- 有动作、有结果：
  - `通常[动作]会使[结果]成立（成立 N 次）`
- 只有结果：
  - `这类情况下通常会出现[结果]（成立 N 次）`
- 其他：
  - `这是一条抽象因果信息（成立 N 次）`

如果 `失败次数 > 0`，会追加失败次数。

## 16. 与其它模块的关系

### 16.1 与 `世界树类`

`世界树类` 对外包装了因果类的主要入口：

- `创建因果模板`
- `按动态创建因果信息`
- `因果()`

调用方通常通过 `世界树.因果()` 或 `世界树.按动态创建因果信息(...)` 使用因果服务。

### 16.2 与 `动态类`

因果类的主要输入是动态：

- 动态必须带有初始状态和结果状态。
- 动态提供主体、特征、来源动作、来源动作名、来源执行成功、动态层级和时间。
- 因果类根据动态生成状态迁移二次特征，并把动态写入证据样本。

### 16.3 与 `二次特征应用模块`

二次特征应用模块会：

- 收集动态产生的结果二次特征。
- 收集同场景中的条件候选。
- 调用 `查找匹配因果模板`。
- 命中后可回写因果证据。

当前命中判断主要依赖 `状态迁移模板` 和结果候选。

### 16.4 与 `语言类_基础信息命名`

基础信息命名逻辑遇到因果节点时：

- 实例因果优先调用 `生成实例因果自然语言`。
- 抽象因果优先调用 `生成抽象因果自然语言`。
- 特殊“因果信息”根节点直接显示其名称。

### 16.5 与任务和自我模块

任务和自我模块在完成状态回写、治理过程记录等场景中，会调用：

- `世界树.按动态创建因果信息(...)`
- `世界树.因果().追加结果状态(...)`

这使任务执行产生的状态变化能沉淀为抽象因果模板。

## 17. 使用示例

### 17.1 从动态生成抽象因果

```cpp
auto* 因果节点 = 世界树.按动态创建因果信息(
    场景,
    动态,
    枚举_因果来源类型::执行生成,
    动作主体,
    动作语义,
    "某动作",
    true);
```

效果：

- 从动态的初始状态和结果状态生成状态迁移二次特征。
- 查找可归并的抽象因果模板。
- 找到则合并证据和扩张模板范围。
- 找不到则在“因果信息”根节点下创建新模板。

### 17.2 给已有模板追加结果状态

```cpp
世界树.因果().追加结果状态(
    因果节点,
    结果状态,
    false);
```

前提：

- 因果节点已经有条件模板。
- 结果状态主信息存在。

效果：

- 根据条件模板构造参考条件状态。
- 生成参考条件状态到结果状态的状态迁移二次特征。
- 将迁移特征加入因果模板。

### 17.3 查找匹配模板

```cpp
auto 命中模板 = 世界树.因果().查找匹配因果模板(
    枚举_因果锚点类型::场景,
    动态层级,
    条件候选,
    结果候选);
```

效果：

- 根据锚点类型、层级和结果二次特征候选查找模板。
- 返回命中的抽象因果模板列表。

### 17.4 生成自然语言

```cpp
auto 文本 = 世界树.因果().生成抽象因果自然语言(因果模板);
```

效果：

- 根据条件、动作、主结果和成立次数生成可读文本。

## 18. 重要注意事项

- 当前主路径只完整支持抽象因果模板，实例因果枚举返回空。
- `按动态创建因果信息` 要求动态有初始状态和结果状态；动作事实动态如果没有状态变化，不能通过这条路径生成因果。
- `创建因果模板` 不会自动挂到“因果信息”根节点，自动根节点只在按动态创建或归并抽象因果时使用。
- `追加结果状态` 必须依赖已有条件模板，否则返回失败。
- `追加证据动态样本` 不会累加 `成立次数`。
- `因果模板匹配二次特征` 当前没有使用 `条件候选`。
- 状态迁移归并区分方向型和值对型；值对型要求具体左右状态值一致，方向型只要求方向一致。
- 抽象模板会扩张区间，因此同一模板可能代表一个范围，而不是单个状态值。
- 因果模板内部当前会创建特征模板、状态模板和二次特征模板子节点；这些节点是模板表达结构，不是普通主体特征挂载入口。
- 如果后续严格执行“特征只能挂在存在或场景下”，因果模板内部的特征模板承载方式需要单独重构。
