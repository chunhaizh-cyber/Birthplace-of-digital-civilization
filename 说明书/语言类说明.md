# 语言类说明

本文说明 `D:\鱼巢\语言主信息类.h`、`D:\鱼巢\语言节点类型.h`、`D:\鱼巢\语言类.h`、`D:\鱼巢\语言类.cpp`、`D:\鱼巢\语言类_基础信息命名.cpp`、`D:\鱼巢\语言类_二次特征表达.cpp`、`D:\鱼巢\语言类_存在表达.cpp`。

`语言类` 是自然语言结构树和自然语言显示能力的服务层。它一方面保存文章、段落、自然句、子句这类语言结构；另一方面提供“把世界树节点转成可读名称或短句”的函数。它不是事实层，不应反向改写世界树真值。

## 1. 文件定位

当前项目中，语言系统主要承担两类职责：

- 自然语言结构存储：文章、段落、自然句、子句。
- 自然语言显示：基础信息命名、二次特征表达、存在问句式表达。

简单说：

- `语素类` 管词和词性词。
- `语言类` 管由词性词组成的文本结构，以及世界树节点的可读表达。
- `世界树类`、`基础信息类`、`状态类`、`动态类`、`因果类` 等负责结构化事实。
- 自然语言输出只用于解释和显示，不作为事实来源。

## 2. 节点别名

`语言节点类型.h` 统一把自然语言相关节点定义成链表节点别名：

```cpp
using 自然语言节点类 = 链表模板<自然语言成分基类*>::节点类;
using 文章节点类 = 自然语言节点类;
using 段落节点类 = 自然语言节点类;
using 自然句节点类 = 自然语言节点类;
using 子句节点类 = 自然语言节点类;
```

这些别名指向同一种链表节点类型，区别由节点上的 `主信息` 动态类型决定。

## 3. 树结构

`语言类` 继承自：

```cpp
链表模板<自然语言成分基类*>
```

典型结构是：

```text
语言根
  文章节点
    段落节点
      自然句节点
        子句节点
```

注意：

- 语言树是独立链表树，不是世界树的一部分。
- 自然句的词序列保存的是 `词性节点类` 引用，不直接保存分词字符串。
- 子句可以引用它对应的场景、状态、动态或因果节点。

## 4. 主信息类型

### 4.1 `自然语言成分基类`

所有语言主信息的基类。

当前只提供虚析构函数：

```cpp
virtual ~自然语言成分基类() = default;
```

### 4.2 `文章主信息类`

文章节点的主信息。

| 字段 | 默认值 | 用途 |
| --- | --- | --- |
| `标题` | 空字符串 | 文章标题。 |
| `原始文本` | 空字符串 | 文章完整原文。 |
| `作者` | 空字符串 | 作者信息，当前创建函数未填写。 |
| `来源` | 空字符串 | 来源信息，当前创建函数未填写。 |
| `对应文本世界场景` | 空引用 | 文章对应的文本世界场景。 |

### 4.3 `段落主信息类`

段落节点的主信息。

| 字段 | 默认值 | 用途 |
| --- | --- | --- |
| `在文章中的序号` | `0` | 段落在文章中的顺序号。 |
| `原始文本` | 空字符串 | 段落原文。 |

### 4.4 `自然句主信息类`

自然句节点的主信息。

| 字段 | 默认值 | 用途 |
| --- | --- | --- |
| `在段落中的序号` | `0` | 自然句在段落中的顺序号。 |
| `词序列` | 空列表 | 句子分词结果，每项是 `可解析引用<const 词性节点类>`。 |
| `原始文本` | 空字符串 | 原始句子文本。 |

### 4.5 `子句主信息类`

子句节点的主信息。

| 字段 | 默认值 | 用途 |
| --- | --- | --- |
| `原子结构类型` | `未定义` | 子句对应的原子语义结构。 |
| `起始词索引` | `0` | 在自然句词序列中的起始索引。 |
| `结束词索引` | `0` | 在自然句词序列中的结束索引，当前按半开区间使用。 |
| `局部词序列` | 空列表 | 子句自己的词序列。为空时可由起止索引从自然句复制。 |
| `对应场景` | 空引用 | 子句解析出的场景节点。 |
| `对应状态` | 空引用 | 子句解析出的状态节点。 |
| `对应动态` | 空引用 | 子句解析出的动态节点。 |
| `对应因果` | 空引用 | 子句解析出的因果节点。 |

## 5. 原子结构类型

`枚举_原子结构类型` 定义在 `基础数据类型.h`，用于标记子句的基础语义结构。

当前包含：

```text
存在与命名、属性与状态、动态与事件、时间定位、空间位置、数量与范围、
比较与排序、否定与排除、疑问与信息需求、条件与假设、因果与解释、
定义与分类、角色与参与者、评价与立场、约束与义务、来源与引述、
指代与省略恢复
```

用途：

- 给子句一个最小语义分类。
- 让后续解析模块知道该子句更可能生成场景、状态、动态、因果或需求。

## 6. 生命周期

全局对象定义在 `语言类.cpp`：

```cpp
语言类 语言集{};
```

`语言类` 禁止拷贝和移动：

```cpp
语言类(const 语言类&) = delete;
语言类& operator=(const 语言类&) = delete;
语言类(语言类&&) = delete;
语言类& operator=(语言类&&) = delete;
```

析构时会加锁并遍历语言树，删除每个节点上的 `主信息`。

### 6.1 `初始化`

```cpp
void 初始化();
```

当前为空实现。

## 7. 创建语言结构

### 7.1 `新建文章`

```cpp
文章节点类* 新建文章(const std::string& 标题, const std::string& 原文);
```

用途：

- 创建文章节点。
- 写入文章标题和原始文本。
- 把文章节点挂到语言树根下。

当前附带行为：

- 会调用 `语素集.添加词性词(标题, "专有名词")`。
- 会调用 `语素集.添加词性词("文章", "名词")`。
- 上述两个词性词当前只确保存在，未写入文章主信息字段。

### 7.2 `添加段落`

```cpp
段落节点类* 添加段落(
    文章节点类* 文章节点,
    std::size_t 在文章中的序号,
    const std::string& 段落文本
);
```

用途：

- 在指定文章下创建段落节点。
- 写入段落序号和段落原文。

行为：

- `文章节点` 为空会抛出 `std::invalid_argument`。
- 成功时返回新建段落节点。

### 7.3 `添加自然句`

```cpp
自然句节点类* 添加自然句(
    段落节点类* 段落节点,
    std::size_t 在段落中的序号,
    const std::vector<const 词性节点类*>& 词序列,
    const std::string& 原句文本
);
```

用途：

- 在指定段落下创建自然句节点。
- 保存句子序号、原句文本和词序列。

行为：

- `段落节点` 为空会抛出 `std::invalid_argument`。
- 输入的裸 `词性节点类*` 会被转换成 `可解析引用<const 词性节点类>`。

注意：

- `语言类` 当前不负责分词。
- 调用方应先通过 `语素集` 或分词模块得到词性节点序列。

### 7.4 `添加子句`

```cpp
子句节点类* 添加子句(自然句节点类* 自然句节点, 子句主信息类* 子句主信息);
```

用途：

- 在指定自然句下挂入一个子句节点。

行为：

- `自然句节点` 为空会抛出 `std::invalid_argument`。
- `子句主信息` 为空会抛出 `std::invalid_argument`。
- 如果 `子句主信息->局部词序列` 为空，并且起止索引合法，会从自然句词序列中复制 `[起始词索引, 结束词索引)` 范围。
- 成功时返回新建子句节点。

注意：

- `添加子句` 接收外部创建的 `子句主信息类*`，成功后由语言树节点持有并在析构时清理。
- 起止索引无效时不会抛异常，只是不会自动补局部词序列。

## 8. 基础信息命名

定义在 `语言类_基础信息命名.cpp`：

```cpp
std::string 获取基础信息名称(const 基础信息节点类* 节点) const;
```

用途：

- 把基础信息树中的节点转成短名称。
- 给控制面板、日志、自然语言表达提供统一显示入口。

当前处理顺序：

1. 空节点或无主信息：返回 `世界根`。
2. 指代节点：优先显示代词和指代对象。
3. 场景节点：显示场景名称和世界类型。
4. 存在节点：显示名称、类型或 `未命名存在`。
5. 抽象特征节点：显示特征类型和区间。
6. 特征节点：显示特征类型和当前值。
7. 状态节点：显示“主体的特征为值”。
8. 动态节点：显示“主体执行了动作”或动作名。
9. 二次特征节点：优先调用 `生成二次特征自然语言`。
10. 因果节点：优先调用因果类生成实例或抽象因果自然语言。
11. 其它基础信息：回退到节点名称、类型或 `节点#主键`。

特征值显示规则：

| 特征值类型 | 显示方式 |
| --- | --- |
| `I64` | 十进制整数。 |
| `指针句柄` | `指针#值`，空指针显示 `空指针`。 |
| `VecU句柄` | `VecU#主信息指针`。 |
| 其它 | `未定义`。 |

注意：

- 词性词文本通过 `语素集.获取词` 获取，失败时返回空文本。
- 动态节点的来源动作当前会把 `来源动作.指针` 解释成 `方法类::节点类*`。
- 该函数只生成显示文本，不修改节点。

## 9. 二次特征表达

定义在 `语言类_二次特征表达.cpp`。

### 9.1 `生成二次特征反义词结果`

```cpp
std::string 生成二次特征反义词结果(const 二次特征节点类* 节点) const;
```

用途：

- 根据二次特征的种类、轴键、标量值和满足状态生成结果词。

例子：

- 左右关系：正值生成 `在左侧`，负值生成 `在右侧`，零值生成 `重合`。
- 远近关系：正值生成 `更远`，负值生成 `更近`，零值生成 `同距`。
- 观察关系：生成 `观察到`。
- 没有匹配模板时，根据 `是否满足` 和 `标量值` 生成 `偏正向`、`偏反向`、`成立`、`更明显` 等。

### 9.2 `生成二次特征介词短语`

```cpp
std::string 生成二次特征介词短语(const 二次特征节点类* 节点) const;
```

用途：

- 生成二次特征的限定短语。

可能包含：

- 维度短语：如 `在左右维度上`。
- 粒度短语：如 `按当前场景`、`按相邻分步`、`按完整路径`。
- 基准短语：如 `相对前一状态`、`相对目标态`。
- 时间归一短语：如 `按总时长归一`。
- 比较对象短语：如 `相对于某对象`。
- 所属场景短语：如 `在某场景中`。

### 9.3 `生成二次特征自然语言`

```cpp
std::string 生成二次特征自然语言(const 二次特征节点类* 节点) const;
```

用途：

- 把二次特征节点组合成完整短句。

组合来源：

- 主语：优先取 `左对象`，其次取 `主体`。
- 比较对象：优先取 `右对象`，其次取 `客体`。
- 方向结果：来自 `生成二次特征反义词结果`。
- 限定短语：来自 `生成二次特征介词短语`。

常见输出形态：

```text
A在B的左侧
A相对于B更大
A在某场景中观察到B
在某维度上偏正向
```

注意：

- 二次特征主信息通过 `二次特征主信息_桥接(节点)` 获取。
- 该函数只读世界树和二次特征主信息，不生成新的事实节点。

## 10. 存在表达

定义在 `语言类_存在表达.cpp`。

这些函数面向“存在是什么、有什么、能什么、会什么、需要什么”这类控制面板或问答式表达。

| 函数 | 用途 |
| --- | --- |
| `生成存在有什么表达` | 列出存在的内部世界、子存在、子特征、需求树、任务树、方法树。 |
| `生成存在是什么表达` | 表达存在的类型和概念模板。 |
| `生成存在能什么表达` | 从方法根节点收集方法能力。 |
| `生成存在会什么表达` | 根据存在主信息中的实例因果生成趋向依据。 |
| `生成存在需要什么表达` | 从需求根节点收集需求项。 |

注意：

- 每个函数都有 `上限` 参数的重载入口，用于限制列出的项目数量。
- 当前 `语言类_存在表达.cpp` 内部的 `私有_安全词` 返回的是词性节点主键，不是 `语素集.获取词` 的词文本；这一点和其它语言分文件的同名 helper 不一致。
- 按当前可见工程文件，`语言类_存在表达.cpp` 没有出现在 `鱼巢.vcxproj` 的编译单元列表中，也没有看到被其它 `.cpp` include 接入；如果没有其它构建路径接入，使用这些函数时可能会遇到链接缺口。

## 11. 当前接线方式

项目文件中能看到：

```xml
<ClCompile Include="语言类.cpp" />
```

同时 `基础信息类.cpp` 直接 include 了两个语言分文件：

```cpp
#include "语言类_基础信息命名.cpp"
#define 私有_安全词 语言类二次_私有_安全词
#define 私有_安全文本 语言类二次_私有_安全文本
#define 私有_世界类型文本 语言类二次_私有_世界类型文本
#define 私有_二次特征种类文本 语言类二次_私有_二次特征种类文本
#define 私有_方法节点名称 语言类二次_私有_方法节点名称
#include "语言类_二次特征表达.cpp"
```

这说明：

- `获取基础信息名称` 和二次特征表达当前通过 include 源文件的方式接入。
- 宏重命名用于避免两个分文件匿名命名空间里的 helper 名称冲突。
- `语言类_存在表达.cpp` 的接入需要单独确认。

## 12. 与其它系统的关系

### 12.1 与语素类

语言类的自然句词序列保存 `词性节点类` 引用。

添加文章时，也会借助 `语素集` 确保标题词和 `文章/名词` 存在。

### 12.2 与世界树

语言类不会把自然语言文本当成世界事实直接写入世界树。

它主要做两件事：

- 读取世界树节点，生成名称或短句。
- 保存语言结构节点，并让子句可引用解析出的场景、状态、动态、因果。

### 12.3 与特征系统

语言类可以读特征节点并生成文本，例如 `安全值为10`。

但语言节点不能成为特征宿主。特征只能挂在存在、场景或虚拟存在下；任务、方法、线程需要特征时，仍通过对应虚拟存在承载。

## 13. 常用用法

### 13.1 建立文章结构

```cpp
auto* 文章 = 语言集.新建文章("测试文本", "小明在房间里。");
auto* 段落 = 语言集.添加段落(文章, 0, "小明在房间里。");

std::vector<const 词性节点类*> 词序列{
    语素集.添加词性词("小明", "专有名词"),
    语素集.添加词性词("在", "介词"),
    语素集.添加词性词("房间", "名词")
};

auto* 句子 = 语言集.添加自然句(段落, 0, 词序列, "小明在房间里。");
```

### 13.2 添加子句

```cpp
auto* 子句主信息 = new 子句主信息类{};
子句主信息->原子结构类型 = 枚举_原子结构类型::空间位置;
子句主信息->起始词索引 = 0;
子句主信息->结束词索引 = 3;

auto* 子句 = 语言集.添加子句(句子, 子句主信息);
```

### 13.3 获取基础信息显示名

```cpp
auto 名称 = 语言集.获取基础信息名称(某基础信息节点);
```

### 13.4 生成二次特征自然语言

```cpp
auto 文本 = 语言集.生成二次特征自然语言(某二次特征节点);
```

## 14. 现状注意点

- `语言类::初始化` 当前为空。
- `语言类` 当前不负责分词，只保存传入的词性节点序列。
- `新建文章` 会创建标题词和 `文章/名词`，但没有把这些词性节点保存到文章主信息。
- `添加子句` 的局部词序列补齐使用 `[起始词索引, 结束词索引)` 半开区间。
- `获取基础信息名称` 对空节点或无主信息节点返回 `世界根`，这适合显示根节点，但对异常节点也会隐藏部分问题。
- 自然语言显示函数会读取世界树、语素集、因果类、二次特征桥接模块等，属于展示层聚合能力。
- 自然语言输出不得反向新增、删除或修改世界树事实。
- 语言节点不应作为任务、方法、线程的特征宿主；相关对象要通过虚拟存在承载特征。
