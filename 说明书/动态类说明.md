# 动态类说明

本文说明 `D:\鱼巢\动态类.h`、`D:\鱼巢\动态类.cpp`，并补充 `D:\鱼巢\基础信息主信息类.h` 中 `动态节点主信息类` 与本类直接相关的字段和枚举。

`动态类` 是基础信息树中“动态节点”的服务层。动态节点用于记录状态变化、动作事实和聚合后的动态段。最常见的动态是：同一主体、同一特征，从一个状态变化到另一个状态。

## 1. 文件定位

当前项目中，动态节点位于状态和因果之间：

- 状态节点记录“某时刻某主体某特征是什么值”。
- 动态节点记录“状态之间如何变化”或“某动作在一段时间内发生过”。
- 场景主信息通过 `动态索引` 保存动态引用，并按动态时间排序。
- 动态创建后会触发动态二次特征生成和聚合动态生成尝试。
- 因果模块会读取动态，把动态转成因果模板或因果实例依据。

简单说：

- 状态是点。
- 动态是两个点之间的变化，或者一段动作事实。
- 二次特征和因果会继续利用动态。

### 3.2 `比较`

`动态节点主信息类::比较` 当前只判断对象是否也是动态主信息：

- 右侧也是 `动态节点主信息类`：返回 `0`。
- 否则返回 `-1`。

动态时间、主体、特征、来源等字段不在该 `比较` 函数里比较。

## 4. `动态类` 的职责

`动态类` 主要做七件事：

1. 绑定基础信息树。
2. 读取动态主信息。
3. 创建状态变化动态和动作动态。
4. 维护场景主信息的 `动态索引`。
5. 在状态记录后自动寻找相邻前状态并刷新动态。
6. 设置动态业务语义、主体特征、聚合信息、来源方法和来源场景。
7. 创建动态后触发二次特征生成和聚合动态生成。

私有状态：

| 字段 | 用途 |
| --- | --- |
| `基础信息_` | 指向基础信息树。为空时大多数操作返回空或失败。 |

## 7. 创建动态

### 7.1 `创建动态(场景节点类* 场景, 动态节点主信息类* 主信息)`

用途：

- 使用调用方准备好的动态主信息创建动态节点，并挂到场景下面。

流程：

1. `基础信息_` 为空或 `主信息` 为空时返回 `nullptr`。
2. 调用 `私有_自动补齐动态主体基础特征`，从初始状态和结果状态补齐动态主体、基础动态特征。
3. 调用 `私有_补齐动态名称类型`，补齐主信息的名称和类型。
4. 调用 `私有_刷新动态业务语义类型`，推导业务语义类型。
5. 调用 `基础信息_->添加子节点(场景, 主信息)` 创建动态节点。
6. 如果父场景有场景主信息，把动态按时间追加到 `动态索引`。
7. 如果节点和场景都存在，触发动态二次特征生成。
8. 尝试生成连续同向聚合动态。
9. 尝试生成同类事件段聚合动态。

注意：

- 正常动态应挂在场景下面。
- 当前实现不强制检查场景非空；场景为空时底层可能挂到世界根，这不符合正常动态建模。
- 创建成功后，主信息生命周期交给基础信息树。
- `动态特征` 字段只表示动态对应的基础状态特征，不保存生成出来的动态二次特征集合。

### 7.2 `创建动态(场景, 初始状态, 结果状态, 起, 止)`

用途：

- 由两个状态节点创建状态变化动态。

前置条件：

- 初始状态和结果状态都有效。
- 两个状态都有状态主体和状态特征。
- 两个状态主体指针相同。
- 两个状态特征指针相同。
- 初始状态时间不晚于结果状态时间。

流程：

1. 读取初始状态和结果状态主信息。
2. 调用 `私有_可形成基础动态` 检查是否可形成动态。
3. 如果 `起 == 0`，使用初始状态排序时间。
4. 如果 `止 == 0`，使用结果状态排序时间。
5. 如果起止都非零但止早于起，交换起止。
6. 创建动态主信息。
7. 写入初始状态、结果状态。
8. 动态主体取结果状态的状态主体。
9. 动态特征取结果状态的状态特征。
10. 写入时间段。
11. 计算动态路径签名。
12. 设置 `动态层级 = 0`。
13. 设置 `聚合方式 = 原子相邻`。
14. 转调基础创建函数。

注意：

- 该函数要求前后状态主体和特征的指针相同，不用主键兼容判断。
- 这是状态变化动态的核心入口。

### 7.3 `创建叶子动态(场景, 初始状态, 结果状态, now = 0)`

用途：

- 创建一个由相邻状态形成的叶子动态。

行为：

- 转调 `创建动态(场景, 初始状态, 结果状态, 0, now)`。
- 起点由初始状态时间推导。
- 终点优先使用 `now`，为 `0` 时再由结果状态时间推导。

### 7.4 `创建动作动态(场景, 动态主体, 动态特征, 开始时间 = 0, 结束时间 = 0)`

用途：

- 创建一条不依赖初始/结果状态的动作事实动态。

前置条件：

- 场景非空。
- 动态主体非空。
- 动态特征非空。

时间规则：

- 开始时间和结束时间都为 `0`：两者都设为当前微秒时间。
- 开始时间为 `0`：开始时间设为结束时间。
- 结束时间为 `0`：结束时间设为开始时间。
- 结束时间早于开始时间：交换两者。

流程：

1. 创建动态主信息。
2. 写入动态主体和动态特征。
3. 写入时间段。
4. 转调基础创建函数。

注意：

- 来源方法和动作相位不会在这里自动写入，需要后续调用 `绑定来源方法`。
- 动作动态当前不写入 `动态路径签名`。

## 8. 删除和枚举

### 8.1 `删除动态(动态节点类* 节点)`

用途：

- 删除动态节点，并从父场景的 `动态索引` 中移除引用。

流程：

1. `基础信息_` 为空或节点为空时返回 `false`。
2. 取得节点父节点。
3. 如果父节点主信息是场景主信息，从 `动态索引` 中删除同指针或同主键引用。
4. 调用 `基础信息_->删除节点(节点)`。

### 8.2 `枚举全部动态() const`

用途：

- 从基础信息树全链枚举所有动态节点。

返回：

- `基础信息_` 为空时返回空数组。
- 否则返回所有主信息为 `动态节点主信息类` 的节点。

### 8.3 `获取场景动态(const 场景节点类* 场景) const`

用途：

- 从场景主信息的 `动态索引` 中解析并返回动态节点列表。

行为：

- 场景主信息无效时返回空数组。
- 对每个索引引用，先按主键解析，再按指针地址在全局动态节点中查找。
- 只返回能解析到动态主信息的节点。

注意：

- 返回顺序来自 `动态索引` 当前顺序。
- 创建动态时会按时间维护索引顺序。

## 9. 状态记录后刷新动态

### 9.1 `查找相邻前状态(场景节点类* 场景, 状态节点类* 新状态) const`

用途：

- 在场景状态索引中，为新状态寻找同主体、同特征的相邻前状态。

流程：

1. `基础信息_`、场景或新状态为空时返回 `nullptr`。
2. 读取新状态主信息。
3. 新状态缺少主体或特征时返回 `nullptr`。
4. 读取场景主信息。
5. 从场景 `状态索引` 末尾向前遍历。
6. 先跳过当前新状态自身。
7. 对候选状态调用 `私有_可形成基础动态`。
8. 第一个可形成基础动态的候选状态即为相邻前状态。

当前状态识别规则：

- 候选节点和新状态指针相同。
- 或主键相同。
- 或主体、特征、发生时间、收到时间、状态值都相同。

注意：

- 可形成基础动态仍要求前后状态主体和特征指针相同。
- 遍历依赖场景 `状态索引`。

### 9.2 `状态记录后刷新动态(场景, 新状态, now = 0)`

用途：

- 状态记录后，自动寻找相邻前状态，并创建叶子动态。

流程：

1. 调用 `查找相邻前状态`。
2. 找不到前状态时返回 `nullptr`。
3. 找到时调用 `创建叶子动态(场景, 前状态, 新状态, now)`。

## 11. 业务语义推导

`创建动态` 和部分修改函数会调用 `私有_刷新动态业务语义类型`。

推导规则按优先级：

1. 动作名包含 `管理_`，或动态特征标题包含 `任务管理_`：
   - `治理过程动态`
2. 动作相位是 `失败`，或动作名包含 `进入收束/进入挂起/进入完成/进入失败/进入等待/停止/待机/封闭`：
   - `边界结束动态`
3. 来源类型是 `内部机制动作`，或动作名包含 `衰减/回归/回升/修复/保护/重算/校正/锚点`：
   - `被动记账动态`
4. 有动作名或来源动作指针：
   - `动作事实动态`
5. 有动态特征、初始状态或结果状态：
   - `状态变化动态`
6. 否则：
   - `未定义`

动态名称和类型补齐规则：

- 优先使用 `来源动作名`。
- 其次使用 `动态特征` 的特征类型。
- 否则使用语素 `"动态"`。

## 12. 路径签名和时间排序

### 12.1 状态变化动态路径签名

`私有_计算动态路径签名` 使用以下信息打包后哈希：

- 动态主体指针
- 动态特征指针
- 初始状态主体指针
- 结果状态主体指针
- 初始状态排序时间
- 结果状态排序时间

注意：

- 当前签名包含运行期指针地址，不是稳定持久化主键。

### 12.2 动态索引排序时间

`私有_取动态时间`：

- `时间.止 != 0` 时使用 `时间.止`。
- 否则使用 `时间.起`。

场景 `动态索引` 按该时间升序排序，时间相同时按主键升序。

| `私有_引用已存在` | 判断引用列表中是否已有同指针或同主键节点。 |

## 14. 常见使用路径

### 14.1 由两个状态创建动态

```cpp
auto* 动态 = 世界树.动态().创建动态(
    场景,
    初始状态,
    结果状态);
```

要求：

- 两个状态必须描述同一个主体。
- 两个状态必须描述同一个特征。
- 初始状态时间不能晚于结果状态时间。

### 14.2 状态记录后自动刷新动态

```cpp
auto* 动态 = 世界树.动态().状态记录后刷新动态(
    场景,
    新状态);
```

这会在场景状态索引中寻找相邻前状态，并创建叶子动态。

### 14.3 创建动作动态并绑定来源方法

```cpp
auto* 动态 = 世界树.动态().创建动作动态(
    场景,
    动态主体,
    动态特征,
    开始时间,
    结束时间);

世界树.动态().绑定来源方法(
    动态,
    方法节点,
    动作名词,
    枚举_动作事件相位::完成);
```

### 14.4 标记聚合动态

```cpp
世界树.动态().设置聚合信息(
    动态,
    1,
    枚举_动态聚合方式::连续同向段);

世界树.动态().追加来源低层动态(动态, 低层动态);
```

## 15. 与状态、场景、二次特征、因果的关系

- `状态类` 创建状态后，可调用 `状态记录后刷新动态` 自动形成动态。
- `场景类` 通过 `动态索引` 维护场景内动态。
- `动态类::创建动态` 会维护场景 `动态索引`。
- 创建动态后会调用二次特征生成模块：
  - `获取动态二次特征`
  - `尝试生成连续同向聚合动态`
  - `尝试生成同类事件段聚合动态`
- `二次特征类` 会读取动态路径签名、动态层级、时间段等信息生成动态二次特征。
- 动态二次特征是集合，不是 `动态主信息.动态特征` 的单个字段；当前自动生成的基础集合包括变化量、变化方向和持续时间。
- 查询某条动态的二次特征时，应从场景二次特征索引中按 `来源动态`、粒度、基准类型、时间归一方式和概念名称筛选。
- `因果类` 会读取动态，建立状态迁移、动作语义和因果模板。

## 16. 注意事项

- 动态节点应挂在场景下；基础创建函数不强制检查场景非空。
- 状态变化动态要求前后状态主体和特征指针相同，不按主键兼容。
- 动作动态不会自动绑定来源方法，也不会写入动作动态路径签名，需要后续调用 `绑定来源方法` 补来源动作。
- `绑定来源方法` 当前只写来源动作指针，不写来源动作主键。
- `设置主体特征` 不会重新计算动态路径签名。
- `设置主体特征` 不会重新生成二次特征。
- 动态路径签名使用运行期指针和时间，不是稳定持久化主键。
- 创建动态会触发二次特征生成和聚合尝试，调用方要意识到这是有后续副作用的入口。
- 时间变化和状态特征值变化是动态二次特征集合中的不同成员；涉及“单位时间变化量”或“一个特征单位变化对应另一个特征变化”的结果属于派生指标，不应塞回 `动态特征` 字段。
- `获取场景动态` 会尝试按主键或指针解析动态引用，比状态类的场景状态读取更主动。
