# 项目说明

## 1. 项目是什么

`HY-Ego（鱼巢）` 是一个基于 C++ 的实验工程，用来承载一套“世界树驱动的自我运行内核”。

它不是单纯的数据结构工程，也不是单纯的 UI 工程，而是把下面几层放到同一条运行主链里：

- 世界树与基础数据结构
- 自我实体与双值运行状态
- 需求、任务、方法三棵核心树
- 状态、动态、实例因果、抽象因果
- 任务治理与方法补齐回流
- 自然语言表达与运行可观测性

项目当前关注的是“最小可运行闭环”，也就是先把启动、推进、观察、自检、继续演化这几件事打通。

## 2. 当前主目标

当前这一版工程的主目标可以收成五条：

1. 以 `鱼巢.exe` 为唯一主宿主，入口源码见 [`鱼巢.cpp`](../鱼巢.cpp)，统一启动路径、线程路径、显示路径和日志路径。
2. 建立 `数据初始化 -> 自我初始化 -> 自我线程 -> 任务治理 -> 方法补齐 -> 可观测性` 的主运行闭环。
3. 让旧功能优先迁成当前结构内的模块或适配层，而不是再复制出一套平行系统。
4. 让控制面板、自检日志和线程快照共同反映当前真实运行状态。
5. 在不破坏既有领域语义边界的前提下，继续补方法生长、方法回写和自然语言显示。

## 3. 核心概念

### 3.1 世界树

世界树是项目的统一承载结构，当前主要承载这些信息：

- 场景
- 存在
- 特征
- 状态
- 动态
- 因果
- 需求
- 任务
- 方法

这些信息不是彼此孤立的，而是通过树结构、特征引用、状态回写、动态记录和因果归档串起来，构成项目的统一真值面。

### 3.2 自我

`自我类` 是当前运行时的核心锚点，负责管理：

- 自我现实场景
- 自我内部世界
- 双值状态，例如安全值、服务值、物理安全、风险安全
- 当前主需求、当前主任务、当前主方法
- 根需求、根任务和主链镜像

这里的“自我”既是世界树里的一个存在，也是运行时推进逻辑的主入口。

### 3.3 需求 / 任务 / 方法

项目当前把三棵树作为运行闭环的核心治理面：

- 需求树：表达方向与目标来源
- 任务树：表达推进过程、步骤链和结果链
- 方法树：表达动作能力、来源边界和后续成长基础

当前实现里，任务树已经不再只有头结点，而是可以稳定生成：

- 头结点
- 步骤节点
- 结果节点

并且步骤位、步骤语义、目标/结果双面、反馈类型、后果治理等信息已经接入治理主链。

### 3.4 状态 / 动态 / 因果

运行期结果当前主要通过三类信息沉淀：

- 状态：当前值和阶段性结果
- 动态：运行过程中的事实变化、方法调用、状态变化
- 因果：实例因果与抽象因果

其中抽象因果统一挂在世界树根链下的 `因果信息` 容器中，用来描述更通用的方向性规律，而不是具体一次实例的原始值。

### 3.5 方法补齐

旧学习任务账本、学习调度和学习执行模块已经删除。当前方法补齐不再走独立学习模块，而是由需求树提出 `方法是否可用=0->1` 等目标，任务树筹办普通任务，方法树中的本能方法直接执行并继续提出动作、条件、结果等缺口。

## 4. 运行主链

当前程序的主入口在 [`鱼巢.cpp`](../鱼巢.cpp)，实际运行链大致如下：

1. 初始化日志系统。
2. 执行自我环境初始化，补齐世界树、自我锚点和主链承载。
3. 注册默认本能方法，补齐根需求、根任务、任务管理任务和方法根。
4. 启动 `自我线程类`。
5. 在 Tick 主循环里推进：
   - 主循环消息归并
   - 初始化补齐与主链回填
   - 先天维护
   - 上位输入刷新
   - 任务管理治理
   - 方法补齐
   - 心跳巡检
6. 把运行结果写回线程快照、控制面板快照和运行日志。
7. 通过控制台命令、文本摘要、HTML 或 WebView2 面板回读当前状态。

这意味着当前仓库已经不是“静态代码集合”，而是一个能启动、能推进、能观察、能记录的实验运行体。

## 6. 当前可观测面

当前工程已经提供三类主要观察入口：

### 6.1 控制台与命令行

常用命令如下：

```powershell
.\x64\Debug\鱼巢.exe
.\x64\Debug\鱼巢.exe --summary
.\x64\Debug\鱼巢.exe --task-summary
.\x64\Debug\鱼巢.exe --learning-summary
.\x64\Debug\鱼巢.exe --panel-html
.\x64\Debug\鱼巢.exe --self-check-log
```

控制台运行时支持：

- `启动控制面板`
- `检查自我实现`
- `退出`

### 6.2 控制面板

控制面板当前可以承载：

- 世界树、自我、需求树、任务树、方法树
- 线程阶段事件
- 动态、因果、先天动作
- 任务管理摘要
- 学习摘要

WebView2 窗口关闭后，控制台仍可继续输入命令重新打开。

### 6.3 日志

运行期日志写入 [`../日志`](../日志)，当前已经包括：

- 程序启动与退出
- 自我实现检查
- 线程快照摘要
- 任务管理专项摘要
- 学习链相关摘要

## 9. 当前完成度与边界

当前已经形成的稳定能力：

- 世界树和自我锚点初始化
- 根需求、根任务、任务管理任务、方法根补齐
- 默认本能方法注册
- 自我线程启动与快照读取
- 任务管理一步治理最小闭环
- 任务树步骤/结果节点写回
- 控制面板、文本摘要和日志联动
- 自然语言命名链和部分表达链接入

当前仍未完全收口的部分：

- 学习模块尚未完全收成独立稳定主链
- 真实业务推进步骤仍弱于治理步骤链
- 方法成熟度、可靠度、条件桶和结果桶仍需深化
- 抽象因果的失败回流、验证回路和消费链还不完整
- 不同节点类型的自然语言完整句式仍在逐步接线
- 自我线程的消息消费、延迟事实车道和最终去向裁决仍需继续落实

因此，当前工程适合作为“持续开发中的可运行内核”，还不应视作功能完全体。
