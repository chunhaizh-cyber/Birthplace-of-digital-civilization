# 本能方法类说明

本文说明 `D:\鱼巢\本能方法类.h` 和 `D:\鱼巢\本能方法类.cpp`。

`本能方法类` 是默认本能方法的方法树初始化入口。它只保留启动所需的默认种子：本能方法 ID、动作名称和初始分类；分类等事实会写入方法虚拟存在的特征中。它不负责执行本能动作；执行入口由 `本能动作管理类` 注册和调度。

## 1. 文件定位

本能方法系统分三层：

- `本能方法类`：保存默认本能方法种子，并确保方法树中存在对应方法头。
- `本能动作管理类`：把本能方法 ID 绑定到实际执行函数。
- `方法类`：维护方法树结构和方法节点。

## 2. 核心数据

### 2.1 `枚举_本能方法ID`

用途：

- 为每个本能方法分配稳定 ID。
- 被方法动作句柄、动作管理器、执行桥和任务治理逻辑共同引用。

当前 ID 分段大致包括：

- `6020-6039`：自我读取、确认、保护和方法补齐。任务筹办与任务执行是任务系统普通函数，不再占用本能方法 ID。
- `6040-6054`：安全值、服务值、方法补齐、方法结构和方法运行账相关本能。
- `6100-6115`：任务管理最小治理动作和最小原语能力。

### 2.2 本能方法结果码

用途：

- 表示需求缺口链路中的特殊非终态结果。

当前结果码：

| 常量 | 含义 |
| --- | --- |
| `本能方法结果_任务缺条件需自我线程` | 执行发现实际缺失，应回自我线程形成对应原子需求。 |

### 2.3 `枚举_本能方法分类`

| 枚举值 | 含义 |
| --- | --- |
| `未定义` | 未分类。 |
| `内部先天动作` | 由系统内生周期或保护逻辑触发。 |
| `自我内部动作` | 自我内部用于任务、方法补齐、治理的动作。 |
| `外部动作` | 面向外部交互的动作。 |

### 2.4 默认本能方法种子

字段说明：

| 字段 | 用途 |
| --- | --- |
| `ID` | 本能方法 ID。 |
| `分类` | 启动时写入方法虚拟存在的本能方法分类特征。 |
| `动作名称` | 注册为方法动作名的文本。 |

## 3. 默认本能方法表

`本能方法类.cpp` 中的 `g_默认自我本能方法表` 是默认本能方法的启动种子表，不再作为运行期元信息事实库。

表项用于：

- `是默认自我本能方法` 判断 ID 是否属于默认本能方法。
- `查询默认动作名称` 查询启动动作名。
- `枚举默认自我本能方法ID` 返回 ID 列表。
- `注册默认自我本能方法集` 批量创建或补齐方法头。
- `默认自我本能方法已注册` 检查默认本能方法是否已经具备可执行方法头。

## 4. 公开方法

### 4.1 `是默认自我本能方法(...)`

用途：

- 判断本能方法 ID 是否在默认种子表中。

输入：

- `本能ID`：目标本能方法 ID。

返回：

- 属于默认本能方法返回 `true`。
- 不属于默认本能方法返回 `false`。

特点：

- 只读，不创建节点。
- 只表达默认集合成员关系，不返回元信息结构。

### 4.2 `查询默认动作名称(...)`

用途：

- 查询默认本能方法的启动动作名称。

返回：

- 找到返回动作名称 C 字符串。
- 找不到返回 `nullptr`。

特点：

- 只用于创建动作名词条和默认动作注册名称。
- 方法分类、运行许可等不通过该接口返回。

### 4.3 `枚举默认自我本能方法ID()`

用途：

- 返回默认本能方法 ID 列表。

返回：

- `std::vector<枚举_本能方法ID>`。

常见用途：

- 动作线程从动作管理器批量注册默认动作。
- 控制或诊断逻辑枚举默认本能。

### 4.4 `注册默认自我本能方法集(...)`

用途：

- 确保默认本能方法都已经挂入自我存在的方法树。

流程：

1. 遍历默认本能方法表。
2. 确保自我存在。
3. 在自我存在的方法根下查找对应方法首节点。
4. 找不到则创建方法首节点。
5. 补齐动作名、本能函数 ID、方法虚拟存在、条件场景、结果场景和方法结构。
6. 将本能方法分类写入方法虚拟存在特征。
7. 只统计最终具备可执行动作条件的方法头。

返回：

- 已确保可执行的默认本能方法数量。

注意：

- 该函数会修改方法树。
- 补齐具体结构时会调用 `方法类::补齐本能方法首节点`。
- 当前可执行判断是：方法首节点存在、节点种类正确、有动作骨架、条件判定索引非空。

### 4.5 `默认自我本能方法已注册(...)`

用途：

- 检查默认本能方法集是否已经全部注册并具备可执行方法头。

返回：

- 全部默认本能方法都可执行时返回 `true`。
- 任一缺失或不可执行时返回 `false`。

## 5. 关键私有逻辑

### 5.1 `私有_查找本能方法首节点(...)`

用途：

- 在自我存在的方法根直接子链中查找本能方法首节点。

匹配规则：

- 优先匹配动作句柄类型为 `本能函数ID` 且 ID 一致。
- 也支持按动作名称词匹配。

### 5.2 `私有_补齐本能方法结构(...)`

用途：

- 将方法首节点补齐成本能方法结构。

主要动作：

- 确保自我存在和自我内部世界。
- 创建方法虚拟存在。
- 创建条件场景和结果场景。
- 写入动作名、本能动作 ID、方法类型和本能方法分类特征。
- 调用 `方法类::补齐本能方法首节点` 完成方法结构补齐。

## 6. 职责边界

`本能方法类` 应负责：

- 维护默认本能方法启动种子。
- 将默认种子转换为方法树中的本能方法头。
- 把分类等身份事实写入方法虚拟存在特征。
- 检查默认本能方法是否已注册。

`本能方法类` 不应负责：

- 执行具体动作函数。
- 管理动作函数注册表。
- 处理任务运行循环。
- 解释方法虚拟存在上的具体运行特征。
