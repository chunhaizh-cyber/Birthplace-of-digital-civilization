# 本能方法类说明

本文说明 `D:\鱼巢\本能方法类.h` 和 `D:\鱼巢\本能方法类.cpp`。

`本能方法类` 是默认本能方法的元信息注册表和方法树初始化入口。它负责维护本能方法 ID、动作名称、分类、说明和调用许可，并把默认本能方法注册到自我存在的方法树中。它不负责执行本能动作；执行入口由 `本能动作管理类` 注册和调度。

## 1. 文件定位

本能方法系统分三层：

- `本能方法类`：保存默认本能方法元信息，并确保方法树中存在对应方法头。
- `本能动作管理类`：把本能方法 ID 绑定到实际执行函数。
- `方法类`：维护方法树结构和方法节点。

## 2. 核心数据

### 2.1 `枚举_本能方法ID`

用途：

- 为每个本能方法分配稳定 ID。
- 被方法动作句柄、动作管理器、执行桥和任务治理逻辑共同引用。

当前 ID 分段大致包括：

- `6020-6039`：自我读取、确认、保护、学习、任务筹办和任务执行。
- `6040-6054`：安全值、服务值、学习闭环、方法结构和方法运行账相关本能。
- `6100-6115`：任务管理最小治理动作和最小原语能力。

### 2.2 本能方法结果码

用途：

- 表示任务筹办、执行和学习链路中的特殊非终态结果。

当前结果码：

| 常量 | 含义 |
| --- | --- |
| `本能方法结果_任务筹办已建立方法头待重入` | 筹办阶段已建立方法头，需要下一轮重入。 |
| `本能方法结果_任务筹办已激活学习任务` | 筹办阶段转入学习补齐。 |
| `本能方法结果_任务执行一步已建立方法头待重入` | 执行阶段已建立方法头，需要下一轮重入。 |
| `本能方法结果_任务执行一步已激活学习任务` | 执行阶段转入学习补齐。 |
| `本能方法结果_学习任务已建立任务头待重入` | 学习任务已建立任务头，需要下一轮重入。 |
| `本能方法结果_任务缺条件需自我线程` | 缺条件，应回自我线程形成需求或补齐。 |
| `本能方法结果_任务缺方法需任务管理任务` | 缺方法，应交任务管理链路处理。 |

### 2.3 `枚举_本能方法分类`

| 枚举值 | 含义 |
| --- | --- |
| `未定义` | 未分类。 |
| `内部先天动作` | 由系统内生周期或保护逻辑触发。 |
| `自我内部动作` | 自我内部用于任务、学习、治理的动作。 |
| `外部动作` | 面向外部交互的动作。 |

### 2.4 `结构_本能方法元信息`

字段说明：

| 字段 | 用途 |
| --- | --- |
| `ID` | 本能方法 ID。 |
| `分类` | 本能方法分类。 |
| `动作名称` | 注册为方法动作名的文本。 |
| `说明` | 人类可读说明。 |
| `允许自我线程被动触发` | 是否允许自我线程被动触发。 |
| `允许任务方法调用` | 是否允许作为任务方法调用。 |
| `允许学习兜底调用` | 是否允许学习或补齐链路兜底调用。 |

## 3. 默认本能方法表

`本能方法类.cpp` 中的 `g_默认自我本能方法表` 是当前默认本能方法事实来源。

表项用于：

- `查询元信息` 查找元信息。
- `枚举默认自我本能方法` 返回完整元信息。
- `枚举默认自我本能方法ID` 返回 ID 列表。
- `注册默认自我本能方法集` 批量创建或补齐方法头。
- `默认自我本能方法已注册` 检查默认本能方法是否已经具备可执行方法头。

## 4. 公开方法

### 4.1 `查询元信息(...)`

用途：

- 按本能方法 ID 查询默认元信息。

输入：

- `本能ID`：目标本能方法 ID。

返回：

- 找到返回 `结构_本能方法元信息*`。
- 找不到返回 `nullptr`。

特点：

- 只读，不创建节点。
- 直接扫描默认本能方法表。

### 4.2 `枚举默认自我本能方法()`

用途：

- 返回默认本能方法元信息列表。

返回：

- `std::vector<结构_本能方法元信息>`。

特点：

- 返回的是表内容拷贝，调用方修改不会影响默认表。

### 4.3 `枚举默认自我本能方法ID()`

用途：

- 返回默认本能方法 ID 列表。

返回：

- `std::vector<枚举_本能方法ID>`。

常见用途：

- 动作线程从动作管理器批量注册默认动作。
- 控制或诊断逻辑枚举默认本能。

### 4.4 `注册默认自我本能方法集(...)`

用途：

- 确保默认本能方法都已经挂入自我存在的方法树。

流程：

1. 遍历默认本能方法表。
2. 确保自我存在。
3. 在自我存在的方法根下查找对应方法首节点。
4. 找不到则创建方法首节点。
5. 补齐动作名、本能函数 ID、方法虚拟存在、条件场景、结果场景和方法结构。
6. 只统计最终具备可执行动作条件的方法头。

返回：

- 已确保可执行的默认本能方法数量。

注意：

- 该函数会修改方法树。
- 补齐具体结构时会调用 `方法类::补齐本能方法首节点`。
- 当前可执行判断是：方法首节点存在、节点种类正确、有动作骨架、条件判定索引非空。

### 4.5 `默认自我本能方法已注册(...)`

用途：

- 检查默认本能方法集是否已经全部注册并具备可执行方法头。

返回：

- 全部默认本能方法都可执行时返回 `true`。
- 任一缺失或不可执行时返回 `false`。

## 5. 关键私有逻辑

### 5.1 `私有_查找本能方法首节点(...)`

用途：

- 在自我存在的方法根直接子链中查找本能方法首节点。

匹配规则：

- 优先匹配动作句柄类型为 `本能函数ID` 且 ID 一致。
- 也支持按动作名称词匹配。

### 5.2 `私有_补齐本能方法结构(...)`

用途：

- 将方法首节点补齐成本能方法结构。

主要动作：

- 确保自我存在和自我内部世界。
- 创建方法虚拟存在。
- 创建条件场景和结果场景。
- 写入动作名、本能动作 ID、方法类型。
- 调用 `方法类::补齐本能方法首节点` 完成方法结构补齐。

## 6. 职责边界

`本能方法类` 应负责：

- 维护默认本能方法元信息。
- 将本能方法元信息转换为方法树中的本能方法头。
- 检查默认本能方法是否已注册。

`本能方法类` 不应负责：

- 执行具体动作函数。
- 管理动作函数注册表。
- 处理任务运行循环。
- 解释方法虚拟存在上的具体运行特征。
