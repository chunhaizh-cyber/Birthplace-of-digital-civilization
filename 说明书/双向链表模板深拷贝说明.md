# 双向链表模板深拷贝说明

本文说明 `D:\鱼巢\双向链表模板_深拷贝.inl` 中各函数的用途、用法和注意事项。

## 1. 文件定位

`双向链表模板_深拷贝.inl` 是 `链表模板<主信息类型>` 的模板分片文件，由 `双向链表模板核心.h` 在末尾 `#include` 引入。

它提供“独立树快照”的复制与释放能力，主要用于把某个节点及其子树复制出来，脱离原链表做临时观察、比较或传递。

## 2. 核心规则

- 深拷贝复制节点对象、主键、子节点数量、父子关系和同层环。
- 深拷贝不会重建原模板实例的全链环，复制出来的每个节点默认是自己的全链自环。
- 值类型主信息要求可拷贝构造。
- 指针型主信息只复制指针值，不复制指针指向的对象。
- 释放独立树时，模板只删除节点对象；指针型主信息不会被删除。

因此，这里的“深拷贝”是对树节点结构的深拷贝，不是对所有外部主信息对象的深拷贝。

## 3. 基本用法

```cpp
链表模板<我的主信息> 树;

auto* 源子树 = 树.查找主键("1A");
auto* 快照 = 树.深拷贝树(源子树);

// 使用快照做只读分析

链表模板<我的主信息>::释放树_独立(快照);
```

如果调用方已经持有读锁或写锁，可以使用 `_已加锁` 版本：

```cpp
auto lk = 树.获取读锁();
auto* 快照 = 树.深拷贝树_已加锁(源子树);
```

## 4. 函数说明

| 函数 | 用途 | 用法和注意 |
| --- | --- | --- |
| `深拷贝树_独立(const 节点类* 源根)` | 静态入口，复制 `源根` 及其全部子树。 | 不加锁，不依赖某个模板实例。返回的独立树需要用 `释放树_独立` 释放。源根为空时返回空指针。 |
| `深拷贝树(const 节点类* 源根) const` | 加读锁复制子树。 | 外部常用入口。函数内部获取读锁，然后转调 `深拷贝树_独立`。 |
| `深拷贝树_已加锁(const 节点类* 源根) const` | 已持锁场景下复制子树。 | 调用方必须已经持有读锁或写锁。避免批量操作中重复加锁。 |
| `释放树_独立(节点类*& 独立根)` | 释放由深拷贝得到的独立树。 | 会递归删除节点，并把 `独立根` 置空。指针型主信息不释放。 |
| `私有_深拷贝主信息(const 主信息类型& src)` | 复制主信息。 | 内部函数。值类型直接拷贝；指针类型只返回原指针。 |
| `私有_深拷贝树_独立_递归(const 节点类* src)` | 递归复制节点及其子链。 | 内部函数。负责建立新节点、复制主键和主信息、复制子节点同层环。 |
| `私有_递归释放_独立(节点类* n)` | 递归释放独立树。 | 内部函数。会沿子链递归删除全部节点。 |

## 5. 结构复制细节

复制单个节点时会写入：

- `主键`
- `子节点数量`
- `主信息`
- `上/下` 初始化为自身
- `父` 初始化为空
- `子` 初始化为空
- `同层环头` 初始化为自身
- `链上/链下` 初始化为自身

复制子链时会：

1. 从源节点的 `子` 开始遍历同层环。
2. 对每个源子节点递归复制。
3. 将复制出的子节点父指针设为目标父节点。
4. 在复制出的子节点之间重建同层环。
5. 将目标父节点的 `子` 指向复制出的子链环头。
6. 用实际复制数量覆盖目标父节点的 `子节点数量`。
7. 调用核心模板中的 `私有_设置同层环头` 修正整条子链的同层环头。

## 6. 和原树的关系

独立树不会挂回原树，也不会自动进入原树的全链环。复制后：

- 原树修改不会改变复制出的节点结构。
- 复制树修改不会改变原树节点结构。
- 如果 `主信息类型` 是指针，原树和复制树会共享同一个主信息对象指针。

指针型主信息共享示例：

```cpp
链表模板<基础信息基类*> 树;
auto* 快照 = 树.深拷贝树(某节点);

// 快照->主信息 和 某节点->主信息 指向同一对象
```

这种设计符合核心模板规则：模板不拥有指针型主信息生命周期。

## 7. 适合场景

- 临时读取一棵子树做分析。
- 在不持有原树锁太久的情况下复制一份只读快照。
- 将局部树结构传给不会修改原树的逻辑。
- 做结构比较、调试输出或自检采样。

## 8. 不适合场景

- 不适合把复制结果当作完整 `链表模板` 实例使用，因为全链环没有按整棵复制树重建。
- 不适合用来复制指针型主信息对象本体。
- 不适合在复制后长期持有并假设主信息指针一定有效；指针生命周期仍由外部系统负责。

## 9. 常见错误

- 复制后忘记调用 `释放树_独立`，会泄漏节点对象。
- 对指针型主信息误以为复制树拥有对象所有权，手动释放可能造成重复释放或悬空引用。
- 在未持锁情况下调用 `深拷贝树_已加锁`，并发写入时可能复制到不一致结构。
- 把独立树节点直接插回原树前没有重新初始化，可能带入旧父子或同层关系。

## 10. 推荐检查

深拷贝本身不调用原树的 `校验结构()`。如果用于关键路径，建议：

1. 复制前对原树调用 `校验结构()`。
2. 对复制出的独立树只按父子和同层关系遍历，不依赖全链关系。
3. 使用后立即 `释放树_独立`。
