# 自我动作实现模块说明

状态：20260503 生成。

对应实现：`D:\鱼巢\自我动作实现模块.ixx`

## 1. 职责

`自我动作实现模块` 承载自我相关本能动作的具体执行体。它把本能方法元信息、方法树结构、输入参数场景、输出结果场景、自我现实场景中的特征值、状态记录和动作动态串联起来。

它主要做三类事情：

- 为自我本能方法确保方法首节点、方法虚拟存在、抽象条件场景和抽象结果场景。
- 从输入参数场景读取当前值和改变量，更新自我现实场景中的安全值或服务值。
- 为每次动作生成运行虚拟存在，回写执行情况、执行前后值、动作动态、输入输出场景引用、方法条件节点和方法结果节点。

导出的本能动作函数统一签名为：

```cpp
存在节点类* 动作函数(
    方法类::节点类* 方法首节点,
    场景节点类* 输入参数场景,
    场景节点类* 输出结果场景) noexcept;
```

参数说明：

- `方法首节点`：调用方传入的方法树锚点。函数会用对应本能 ID 校正到实际方法首节点。
- `输入参数场景`：参数来源。变化类动作要求其中存在 `条件集合`，并在集合下提供目标特征和 `改变量`。
- `输出结果场景`：执行结果写入场景。成功时会写入目标特征结果值；无论成功与否，会尽量写入执行情况和动作动态引用。

返回说明：

- 返回本轮动作的 `运行虚拟存在`。
- 返回空表示无法找到宿主、无法创建运行存在，或前置方法节点不可用。

## 2. 总体运行流程

变化类动作使用同一条执行链：

1. 调用 `确保本能方法`，按本能 ID 找到或注册方法首节点。
2. 调用 `本轮运行虚拟存在`，在方法虚拟存在或输出结果场景下创建本轮运行存在。
3. 初始化方法虚拟存在信息，并声明抽象条件特征。
4. 调用 `构建方法条件和结果节点`，保证方法树有条件节点和结果节点。
5. 调用 `变化输入已绑定`，从输入场景的 `条件集合` 读取当前值和改变量。
6. 用 `按方向改变` 计算执行后值。
7. 调用 `写入场景I64特征` 更新自我现实场景。
8. 调用 `输出变化执行` 统一生成状态、动态和结果回写。

闭环推进类动作目前是运行记录和参数声明型实现：

1. 确保对应本能方法。
2. 创建本轮运行虚拟存在。
3. 写入“无条件本能”的自描述信息。
4. 声明运行参数。
5. 若调用方传入输入场景，则检查输入场景是否包含声明参数。
6. 当前实现不做实际数值变更，最后只补齐运行存在的结束时间和输入输出场景引用。

## 4. 内部辅助函数

### 4.1 `写入I64`

用途：向方法虚拟存在写入一个 I64 特征值。

实现逻辑：检查方法首节点和特征类型非空，然后调用 `方法类::记录方法虚拟存在特征`，以 `特征值{ 值 }` 和当前时间写入。

使用方法：模块内部在需要把执行前值、执行后值或条件自描述数据写入方法虚拟存在时调用。

### 4.2 `写入存在I64`

用途：向某个存在节点写入 I64 特征。

实现逻辑：检查存在和特征类型后，把 `存在节点类*` 转为 `基础信息节点类*`，调用 `世界树.写入特征_I64`。

使用方法：运行存在需要记录开始时间、结束时间、执行前值、执行后值时调用。

### 4.3 `写入存在指针`

用途：向存在节点写入指针型特征。

实现逻辑：检查存在、特征类型和指针均非空，调用 `世界树.写入特征_指针`。

使用方法：运行存在记录方法虚拟存在、输入输出场景、动态节点、条件节点、结果节点时调用。

### 4.4 `写入基础节点指针`

用途：向任意基础信息节点写入指针型特征。

实现逻辑：检查宿主、特征类型和指针非空，然后调用 `世界树.写入特征_指针`。

使用方法：输出结果场景、方法虚拟存在等宿主需要挂接动态或执行情况指针时调用。

### 4.5 `方法虚拟存在`

用途：取得方法首节点对应的方法虚拟存在。

实现逻辑：若方法首节点已有 `主信息.方法虚拟存在.指针`，直接返回；否则调用 `方法类::取或创建_方法虚拟存在` 创建。

使用方法：所有需要把运行状态写到方法自身空间的辅助函数先通过它取得宿主。

### 4.6 `运行参数根`

用途：取得方法虚拟存在下的运行参数根特征。

实现逻辑：先取方法虚拟存在，再在该存在下按 `特征_运行参数()` 取或创建子特征。

使用方法：`声明运行参数` 和 `运行参数已从输入场景匹配` 通过它访问运行参数声明集合。

### 4.7 `抽象条件场景`

用途：取得或创建方法首节点的抽象条件场景。

实现逻辑：优先返回 `主信息.条件场景.指针`；没有时调用 `方法类::取或创建_方法条件场景`。

使用方法：变化类动作声明输入条件、构建方法条件节点时调用。

### 4.8 `抽象结果场景`

用途：取得或创建方法首节点的抽象结果场景。

实现逻辑：优先返回 `主信息.结果场景.指针`；没有时调用 `方法类::取或创建_方法结果场景`。

使用方法：构建方法结果节点时调用。

### 4.9 `抽象条件集合根`

用途：取得抽象条件场景中的 `条件集合` 根特征。

实现逻辑：先取得抽象条件场景，再在场景下按 `特征_条件集合()` 取或创建子特征。

使用方法：`声明条件特征` 和 `变化输入已绑定` 用它获得方法声明的条件集合。

### 4.10 `声明条件特征`

用途：在方法的抽象条件集合中声明一个必需条件特征。

实现逻辑：取得抽象条件集合根，然后按传入特征类型取或创建子特征。

使用方法：变化类动作在执行前声明目标值特征和 `改变量`。

### 4.11 `实际条件集合根`

用途：从调用方输入参数场景中查找实际 `条件集合`。

实现逻辑：若输入参数场景非空，则在场景下查找类型为 `特征_条件集合()` 的子特征；不创建缺失节点。

使用方法：`变化输入已绑定` 用它确认调用方是否提供了实际条件集合。

### 4.12 `读取条件特征I64`

用途：从实际条件集合中读取某个条件特征的 I64 值。

实现逻辑：先在条件集合下按类型查找子特征，再调用 `世界树.读取特征_I64`。

使用方法：读取目标当前值和改变量。读取失败时返回 `false`。

### 4.13 `变化输入已绑定`

用途：验证变化类动作的输入是否满足抽象条件，并读取目标当前值和改变量。

实现逻辑：取得抽象条件集合和实际条件集合，遍历抽象条件集合中的每个特征类型。遇到 `特征_改变量()` 时尝试读取改变量，读取失败则保留默认值；遇到目标特征时必须成功读取当前值；遇到其他条件时要求实际集合中存在对应特征。

使用方法：`增加安全值`、`减少安全值`、`保持安全值`、`增加服务值`、`减少服务值`、`保持服务值` 在执行前调用。返回 `false` 时动作按“参数绑定失败”输出。

### 4.14 `写入方法执行情况`

用途：把执行情况语义写到指定宿主。

实现逻辑：调用 `写入基础节点指针`，以 `特征_方法执行情况()` 为特征类型，情况词条为指针值。

使用方法：运行存在、输出结果场景、方法虚拟存在需要记录执行成功、失败或绑定失败时调用。

### 4.15 `构建方法条件和结果节点`

用途：在方法树下创建本次方法的条件节点和结果节点。

实现逻辑：取得抽象条件场景和抽象结果场景；条件场景存在时调用 `方法类::创建条件节点`；结果场景存在时调用 `方法类::创建结果节点`；返回两个节点指针组成的结构。

使用方法：变化类动作执行前调用一次，`输出变化执行` 也会调用并把节点引用写入运行存在。

### 4.16 `饱和增加`

用途：执行不会溢出的 I64 加法。

实现逻辑：若正向相加超过 `I64` 最大值，返回最大值；若负向相加低于最小值，返回最小值；否则返回普通加法结果。

使用方法：`按方向改变` 和 `增加场景I64特征` 用它保护数值边界。

### 4.17 `增加场景I64特征`

用途：给场景中的某个 I64 特征增加增量。

实现逻辑：取或创建场景下目标特征，读取当前值，使用 `饱和增加` 计算新值，再写回。

使用方法：当前文件未被导出动作直接调用，可作为后续累加型场景特征更新辅助。

### 4.18 `写入场景I64特征`

用途：把 I64 值写入场景中的指定特征。

实现逻辑：取或创建场景下目标特征，然后调用 `世界树.写入特征_I64`。

使用方法：变化类动作把安全值或服务值写回自我现实场景时调用。

### 4.19 `自我现实场景`

用途：取得当前自我所在的现实场景。

实现逻辑：优先返回 `世界树.自我所在场景`；若为空，则调用 `世界树.取或创建自我现实场景()`。

使用方法：变化类动作读取输出状态宿主和写入目标特征时调用。

### 4.20 `按方向改变`

用途：按动作方向计算执行后值。

实现逻辑：方向大于 `0` 时执行 `当前值 + 改变量`；方向小于 `0` 时执行 `当前值 - 改变量`；方向等于 `0` 时保持当前值。加减都通过 `饱和增加` 保护溢出。

使用方法：增加、减少、保持安全值或服务值时分别传入 `1`、`-1`、`0`。

### 4.21 `写入参数绑定失败`

用途：把参数绑定失败写入运行存在和输出结果场景。

实现逻辑：如果运行存在非空，写入 `值_参数绑定失败()`；如果输出结果场景非空，也写入同样执行情况。

使用方法：当前变化类动作改为通过 `输出变化执行(..., 值_参数绑定失败(), ...)` 统一输出，本函数保留为直接写失败状态的辅助。

### 4.22 `场景特征`

用途：取得或创建场景下的目标特征节点。

实现逻辑：场景和特征类型非空时，调用 `世界树.取或创建子特征_按类型`；否则返回空。

使用方法：`输出变化执行` 创建状态和动态前用它定位目标特征。

### 4.23 `创建场景状态`

用途：在场景中创建某个特征值对应的状态节点。

实现逻辑：调用 `世界树.状态().创建状态` 创建状态，再调用 `世界树.状态().标记变化` 标记该状态是否代表变化。

使用方法：`输出变化执行` 在成功时创建执行前状态和执行后状态。

### 4.24 `输出动作动态`

用途：创建本次方法动作对应的动态节点。

实现逻辑：检查方法首节点、场景和特征非空后，调用 `世界树.动态().创建方法动作动态`，写入方法节点、动作名、成功标志、事件相位、输入参数场景和输出结果场景。

使用方法：`输出变化执行` 在能定位场景和特征时调用。成功相位为 `完成`，失败相位为 `失败`。

### 4.25 `输出运行虚拟存在` 前向声明

用途：让 `输出变化执行` 可以在实际定义之前调用 `输出运行虚拟存在`。

实现逻辑：只声明函数签名，不包含实现。

使用方法：模块内部编译依赖处理，不由业务代码直接调用。

### 4.26 `输出变化执行`

用途：统一完成变化类动作的输出回写。

实现逻辑：先决定执行情况语义；构建方法条件和结果节点；取得自我现实场景与目标特征；成功时创建执行前状态和执行后状态；创建动作动态；向运行存在写入执行情况、执行前后值、动态、条件节点、结果节点；向输出结果场景写入执行情况、成功结果值和动态；向方法虚拟存在写入执行前后值、执行情况和动态；最后调用 `输出运行虚拟存在` 补齐结束时间和场景引用。

使用方法：变化类导出动作在绑定失败、写入成功或写入失败后都通过它返回统一格式的运行存在。

### 4.27 `声明运行参数`

用途：在方法虚拟存在的运行参数根下声明一个参数类型。

实现逻辑：取得 `运行参数根`，按参数类型取或创建子特征。

使用方法：闭环推进类动作声明自己需要的输入参数，例如安全值、服务值、待学习方法数量或可用方法数量。

### 4.28 `运行参数已从输入场景匹配`

用途：检查输入参数场景是否包含方法声明的全部运行参数。

实现逻辑：遍历运行参数根下的子特征，取出每个参数类型，并要求输入场景下存在相同类型的子特征。

使用方法：闭环推进类动作在调用方传入输入场景时执行参数匹配校验。

### 4.29 `描述无条件本能`

用途：给方法虚拟存在写入“该本能没有条件特征要求”的自描述。

实现逻辑：初始化方法虚拟存在信息，然后写入条件描述状态、条件描述时间、条件特征数量、必需条件数量、可选条件数量。其中三个数量当前都写为 `0`。

使用方法：闭环推进类动作在声明运行参数前调用。

### 4.30 `方法树根`

用途：从任意方法节点向上找到方法树根节点。

实现逻辑：沿 `父` 指针循环上溯，直到没有父节点为止。

使用方法：`确保本能方法` 用它找到搜索或注册方法首节点的根。

### 4.31 `确保本能方法`

用途：确保当前动作使用的是真正对应本能 ID 的方法首节点。

实现逻辑：校验锚点和本能 ID；构造本能函数 ID 动作句柄；若传入节点已是匹配的方法首节点则直接返回；否则找到方法树根，按动作句柄查找已有方法；仍未找到时查询本能方法元信息，创建动作名词条，注册主体局部方法首节点，并调用 `方法类::补齐本能方法首节点`。

使用方法：每个导出动作进入时第一步调用它，把调用方传入的锚点校正为对应动作的方法首节点。

### 4.32 `本轮运行虚拟存在`

用途：为本次动作运行创建或取得运行虚拟存在。

实现逻辑：优先使用方法虚拟存在作为宿主；若方法虚拟存在不可用，则使用输出结果场景作为宿主。随后按 `类型_运行虚拟存在()` 和 `特征_开始时间()` 创建或查找子存在，写入方法虚拟存在指针和开始时间。

使用方法：所有导出动作在确保方法后调用，用返回值作为本轮执行回写的核心对象。

### 4.33 `输出运行虚拟存在`

用途：补齐运行虚拟存在的结束信息和输入输出引用。

实现逻辑：写入结束时间；若输入参数场景存在，写入 `输入参数场景` 和 `返回参数信息`；若输出结果场景存在，写入 `输出结果场景` 和 `返回场景信息`；最后返回运行存在。

使用方法：变化类动作由 `输出变化执行` 间接调用；闭环推进类动作直接调用。

## 5. 导出动作函数

### 5.1 `增加安全值`

用途：按输入参数中的 `改变量` 增加自我现实场景的安全值。

实现逻辑：确保本能方法 ID 为 `自我_增加安全值`；声明条件特征 `安全值` 和 `改变量`；从输入场景的 `条件集合` 读取当前安全值和改变量；计算 `安全值 + 改变量`；写回自我现实场景；通过 `输出变化执行` 记录状态、动态和结果。

使用方法：

```cpp
auto* 运行存在 = 自我动作实现模块::增加安全值(
    方法首节点,
    输入参数场景,
    输出结果场景);
```

输入参数场景应包含 `条件集合 -> 安全值`，通常也包含 `条件集合 -> 改变量`。绑定失败时返回的运行存在会标记 `参数绑定失败`。

```cpp
auto* 运行存在 = 自我动作实现模块::减少安全值(
    方法首节点,
    输入参数场景,
    输出结果场景);
```

调用方需提供能绑定当前安全值的输入条件集合；改变量缺失时当前实现会按默认 `0` 处理。

## 6. 输入参数场景约定

变化类动作读取的是输入参数场景下的 `条件集合`，而不是直接读取输入参数场景根上的特征。

推荐结构：

```text
输入参数场景
  条件集合
    安全值 / 服务值 = 当前值
    改变量 = 本次变化量
```

闭环推进类动作读取的是输入参数场景根上的运行参数特征：

```text
输入参数场景
  安全值 / 服务值 / 待学习方法数量 / 可用方法数量
```

## 7. 输出结果约定

变化类动作成功时会尽量输出：

- 运行存在：执行情况、执行前值、执行后值、动作动态、方法条件节点、方法结果节点、开始时间、结束时间、输入输出场景引用。
- 输出结果场景：执行情况、目标特征的执行后值、动作动态。
- 自我现实场景：目标特征的执行后值。
- 方法虚拟存在：执行前值、执行后值、执行情况、动作动态。
- 状态树：执行前状态和执行后状态。
- 动态树：方法动作动态。

闭环推进类动作当前主要输出：

- 运行存在：开始时间、结束时间、方法虚拟存在引用、输入输出场景引用。
- 方法虚拟存在：无条件本能自描述和运行参数声明。

## 8. 使用示例

下面示例只展示调用形态。输入场景和输出场景的创建方式应按调用方所在模块的世界树约定完成。

```cpp
import 自我动作实现模块;

方法类::节点类* 方法锚点 = /* 方法树根或候选方法节点 */;
场景节点类* 输入参数场景 = /* 含 条件集合 -> 安全值、改变量 */;
场景节点类* 输出结果场景 = /* 调用方准备的结果场景 */;

存在节点类* 运行存在 = 自我动作实现模块::增加安全值(
    方法锚点,
    输入参数场景,
    输出结果场景);

if (!运行存在) {
    // 调用方可视为本能动作未能建立运行记录。
}
```
