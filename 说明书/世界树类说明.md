# 世界树类说明

本文说明 `D:\鱼巢\世界树类.h` 和 `D:\鱼巢\世界树类.cpp` 中的 `世界树类`。

`世界树类` 是项目当前世界模型的总入口。它组合并持有基础信息树、特征值池、特征服务、存在服务、场景服务、二次特征服务、状态服务、动态服务和因果服务，并提供一组常用的创建、读取、写入、比较和默认世界初始化接口。

## 1. 文件定位

`世界树类` 不是新的数据结构树，而是多个服务的组合协调层：

- `基础信息类`：真实的基础信息节点树。
- `特征值类`：VecU 特征值证据池。
- `特征类`：特征节点、抽象特征、特征比较和稳态相关服务。
- `存在类`：存在节点、虚拟存在、存在内部需求/任务/方法根。
- `场景类`：场景节点、内部世界、状态/动态/二次特征索引。
- `二次特征类`：二次特征节点服务。
- `二次特征生成模块`：由比较、场景等事实生成二次特征。
- `状态类`：状态节点服务。
- `动态类`：动态节点服务。
- `因果类`：因果模板和因果实例服务。

全局对象：

```cpp
extern 世界树类 世界树;
```

实现文件中定义：

```cpp
世界树类 世界树{};
```

## 3. 构造、析构和复制限制

### 3.1 `世界树类()`

用途：

- 构造世界树对象，初始化各服务与基础信息树的绑定关系。

行为：

- `特征服务_`、`存在服务_`、`场景服务_`、`二次特征服务_`、`状态服务_`、`动态服务_`、`因果服务_` 都绑定到同一个 `基础信息链_`。
- 创建 `二次特征生成服务_`，并把 `this` 传给生成模块。
- 给 `特征服务_` 设置默认差异度函数，转调 `世界树类::默认特征差异度`。

注意：

- 构造函数只绑定服务，不创建默认世界。
- 默认世界需要调用 `初始化默认世界`。

### 3.2 `~世界树类()`

用途：

- 默认析构。

说明：

- 成员对象按 C++ 生命周期自动析构。
- `二次特征生成服务_` 由 `std::unique_ptr` 管理。

### 3.3 禁止复制和移动

`世界树类` 删除了复制构造、复制赋值、移动构造、移动赋值。

用途：

- 避免多个世界树对象共享或误搬移同一批内部指针。

## 4. 初始化

### 4.1 `初始化()`

用途：

- 初始化世界树的基础组件。

流程：

1. `基础信息链_.初始化()`
2. `值池_.初始化()`
3. `语素集.初始化()`
4. `初始化度量模板注册表()`

注意：

- 该函数不会创建默认世界。
- 默认世界由 `初始化默认世界()` 创建。
- 抽象特征根会在初始化阶段确保存在。

### 4.2 `初始化默认世界()`

用途：

- 创建或取得系统默认世界场景，并写入公开缓存指针。

创建的默认世界：

行为：

- 每个默认世界都通过 `取或创建默认世界` 创建或复用。
- 默认世界挂在基础信息世界根下面。

## 7. 抽象特征入口

抽象特征不是普通实例特征。实例特征只能挂在存在或场景下面；抽象特征节点属于抽象特征树，根节点挂在基础信息世界根下面。

### 7.1 `取或创建抽象特征根()`

用途：

- 获取抽象特征树根节点，不存在时创建。

行为：

1. 如果 `抽象特征根` 已缓存且主信息是 `抽象特征主信息类`，直接返回。
2. 使用语素 `"抽象特征根"` 查找世界根下已有抽象特征根。
3. 找到则缓存并返回。
4. 找不到则创建新的 `抽象特征主信息类`。
5. 设置名称和类型为 `"抽象特征根"`。
6. 调用 `特征服务_.创建抽象特征(世界根(), 主信息)` 创建根。

### 7.2 `创建抽象特征(基础信息节点类* 父抽象特征, 抽象特征主信息类* 主信息)`

用途：

- 创建抽象特征节点。

行为：

- 父抽象特征为空时，使用 `取或创建抽象特征根()`。
- 转调 `特征服务_.创建抽象特征`。

### 7.3 `取或创建抽象特征_按类型(...)`

用途：

- 在指定父抽象特征下面按类型查找或创建子抽象特征。

行为：

- 父抽象特征为空时，使用抽象特征根。
- 转调 `特征服务_.取或创建子抽象特征_按类型`。

### 7.4 `取或创建实例特征_按抽象特征(...)`

用途：

- 在宿主下面按抽象特征创建或取得实例特征。

注意：

- 这里的 `宿主` 必须遵守特征宿主规则：只能是存在、场景，或任务/方法/线程对应的虚拟存在。
- 函数本身转调 `特征服务_`，不在世界树层强制检查宿主类型。

### 7.5 `解析实例特征命中抽象特征(特征节点类* 实例特征)`

用途：

- 根据实例特征当前值解析其命中的抽象特征状态。

行为：

- 转调 `特征服务_.解析实例特征命中抽象特征`。

### 7.6 `升格实例稳态为抽象区间(特征节点类* 实例特征)`

用途：

- 将实例特征当前稳态升格成抽象区间节点。

行为：

- 转调 `特征服务_.升格当前稳态为抽象区间`。

### 7.7 `求值组合抽象特征(...)`

用途：

- 对某个宿主上的组合抽象特征求值。

行为：

- 转调 `特征服务_.求值组合抽象特征`。

注意：

- `宿主` 同样应是存在、场景或虚拟存在。

## 8. 节点创建与取得

### 8.1 特征节点

| 函数 | 用途 |
| --- | --- |
| `创建特征(宿主, 主信息)` | 转调 `特征服务_.创建特征`。 |
| `确保特征(宿主, 特征类型)` | 转调 `特征服务_.取或创建子特征_按类型`。 |
| `取或创建子特征_按类型(宿主, 特征类型)` | 与 `确保特征` 等价。 |

规则：

- 实例特征只能挂在存在或场景下面。
- 任务、方法、线程需要特征时，必须通过虚拟存在承载。
- 当前世界树函数不强制检查宿主类型，调用方必须遵守规则。

### 8.2 存在节点

| 函数 | 用途 |
| --- | --- |
| `创建存在(父节点, 主信息)` | 转调 `存在服务_.创建存在`。 |
| `取或创建子存在_按类型(父节点, 类型)` | 在父节点直接子存在中按类型复用；未命中时创建。 |
| `取或创建子存在_按类型并按特征I64(...)` | 用“存在类型 + I64 唯一键特征”复用或创建子存在。 |

`取或创建子存在_按类型并按特征I64` 流程：

1. 类型或唯一键特征类型为空时返回 `nullptr`。
2. 遍历父节点直接子存在。
3. 当前实现用存在主信息的 `类型` 指针与传入类型比较。
4. 查找唯一键特征。
5. 读取 I64 当前值，等于目标值则复用。
6. 未命中时创建新存在。
7. 在新存在下确保唯一键特征。
8. 写入唯一键 I64 值。

注意：

- 唯一键特征挂在存在下面，符合特征宿主规则。

### 9.1 `写入特征_I64(特征节点类* 节点, I64 值, 时间戳 now)`

用途：

### 9.3 `写入特征_指针`

重载：

```cpp
bool 写入特征_指针(特征节点类* 节点, const void* 指针, 时间戳 now);
```

### 9.4 `写入特征_VecU`

重载：

```cpp
bool 写入特征_VecU(特征节点类* 节点, VecIU64 值, 枚举_轮廓维度 维度, 时间戳 now);
```

用途：

- 把向量值写入值池，取得 `VecU句柄`，再写入特征节点。

流程：

1. 调用 `值池_.获取或创建句柄(std::move(值), 维度)`。
2. 调用 `特征服务_.写入特征值_VecU句柄`。

## 10. 存在比较与二次特征刷新

### 10.1 `比较存在(...) const`

用途：

- 比较两个存在的直接子特征集合。

流程：

1. 任一存在为空时返回默认 `存在比较结果{}`。
2. 获取左存在直接子特征。
3. 获取右存在直接子特征。
4. 调用 `特征服务_.比较特征集合`。

注意：

- 只比较直接子特征。
- 不递归比较内部世界或子存在。

### 10.2 `刷新存在比较二次特征(...)`

用途：

- 比较两个存在，并把比较结果转换成场景内二次特征。

前置条件：

- 左右存在都非空。
- 左右存在不能是同一个节点。
- 至少一个存在的父节点是场景。
- 如果两个存在都有父场景，则必须在同一个场景。

流程：

1. 解析左右存在所在场景。
2. 不满足前置条件时返回空数组。
3. 调用 `比较存在` 得到比较结果。
4. 调用 `二次特征生成服务_->由存在比较结果生成二次特征`。

## 11. 默认世界、自我与内部世界

### 11.1 `取或创建默认世界(const std::string& 名称, 枚举_世界类型 世界类型)`

用途：

- 在世界根下按名称创建或取得默认世界场景。

行为：

- 用 `语素集.添加词性词(名称, "专有名词")` 生成名称词。
- 调用 `取或创建子场景_按名称(nullptr, 名称词, 世界类型)`。

### 11.2 `取或创建自我现实场景()`

用途：

- 取得现实世界中的 `自我初始场景`。

行为：

- 如果 `现实世界` 缓存为空，先调用 `初始化默认世界()`。
- 在 `现实世界` 下取或创建名称为 `自我初始场景`、世界类型为 `现实世界` 的子场景。
- 写入 `自我所在场景` 缓存。

### 11.3 `取或创建自我存在()`

用途：

- 取得自我存在节点。

行为：

- 先取得 `自我所在场景`，为空时调用 `取或创建自我现实场景()`。
- 名称词为 `自我`。
- 类型词为 `自我类型`。
- 在当前场景下调用 `存在服务_.取或创建子存在_按名称类型`。
- 写入 `自我指针` 缓存。

### 11.4 `取存在内部世界(const 存在节点类* 宿主存在) const noexcept`

用途：

- 读取存在主信息中保存的内部世界引用。

返回：

- 有效时返回内部世界场景指针。
- 不存在时返回 `nullptr`。

### 11.5 `取或创建存在内部世界(存在节点类* 宿主存在, const 词性节点类* 场景名称 = nullptr)`

用途：

- 取得或创建某个存在的内部世界。

流程：

1. 宿主存在为空时返回 `nullptr`。
2. 如果已有内部世界，直接返回。
3. 场景名称为空时使用 `"内部世界"`。
4. 在宿主存在下面创建或取得内部世界场景。
5. 调用 `场景服务_.绑定宿主(内部世界, 宿主存在)`。
6. 调用 `存在服务_.绑定内部世界(宿主存在, 内部世界)`。
7. 返回内部世界。

## 12. 存在观测位置

### 12.1 `写入存在观测位置(存在节点类* 节点, const Vector3D& 位置_mm)`

用途：

- 写入存在观测位置，并在存在所属场景中刷新场景存在型基础二次特征。

流程：

1. 节点为空时返回 `false`。
2. 调用 `存在服务_.写入观测位置`。
3. 如果写入失败，返回 `false`。
4. 判断该存在父节点是否为场景。
5. 如果父节点是场景，调用 `二次特征生成服务_->刷新场景存在型基础二次特征(场景节点)`。
6. 返回 `true`。

注意：

- 如果存在不是直接挂在场景下，只写入位置，不刷新场景二次特征。

## 13. 默认特征差异度

### 13.1 `默认特征差异度(const 特征值& 左值, const 特征值& 右值) const`

用途：

- 作为 `特征服务_` 的默认差异度函数。

规则：

| 值形态 | 差异度 |
| --- | --- |
| variant 形态不同 | `10000` |
| `I64` | 饱和绝对差。 |
| `VecU句柄` | 从值池取向量，按位差异率计算，范围约为 `0..10000`。 |
| `指针句柄` | 相等为 `0`，不等为 `10000`。 |
| 其他或空值 | `0`。 |

VecU 差异率：

- 比较两个 `VecIU64` 的所有 bit。
- 公共块按异或后 `popcount` 统计差异位。
- 多出的块按每块 64 位全部差异计算。
- 返回 `差异位数 * 10000 / 总位数`。

注意：

- 这是当前实现的默认比较语义。
- 更具体的特征语义可以通过特征服务的差异度函数扩展。

## 14. 私有辅助函数

这些函数位于 `世界树类.cpp` 的匿名命名空间，只供本文件内部使用。

| 函数 | 用途 |
| --- | --- |

## 15. 常见使用路径

### 15.1 初始化世界树

```cpp
世界树.初始化();
世界树.初始化默认世界();
```

### 15.2 取得自我存在

```cpp
auto* 自我存在 = 世界树.取或创建自我存在();
```

这会按需创建：

- 默认世界。
- 现实世界下的 `自我初始场景`。
- 自我存在。

### 15.3 写入存在特征

```cpp
auto* 自我存在 = 世界树.取或创建自我存在();
世界树.写入特征_I64(自我存在, 安全值词, 100);
```

注意：

- 这里宿主是存在，合法。
- 如果是任务、方法或线程状态，先取得对应虚拟存在，再写特征。

### 15.4 写入 VecU 特征

```cpp
世界树.写入特征_VecU(
    存在节点,
    轮廓特征词,
    VecIU64{ 1, 2, 3 },
    枚举_轮廓维度::未定义);
```

### 15.5 创建内部世界

```cpp
auto* 内部世界 = 世界树.取或创建存在内部世界(宿主存在);
```

### 15.6 比较同一场景中的两个存在

```cpp
auto 比较 = 世界树.比较存在(左存在, 右存在);
auto 二次特征列表 = 世界树.刷新存在比较二次特征(左存在, 右存在);
```

## 16. 注意事项

- `世界树类` 是总入口和协调层，真实数据仍在各服务和基础信息树中。
- 全局 `世界树` 是单例式全局对象，避免复制或移动。
- `初始化()` 不等于 `初始化默认世界()`。
- 公开缓存指针只是缓存，不拥有节点生命周期。
- 实例特征只能挂在存在或场景下面；任务、方法、线程需要特征时使用虚拟存在。
- 抽象特征节点属于抽象特征树，不是实例特征。
- `取或创建子场景_按名称` 只按名称复用，可能覆盖世界类型。
- `取或创建子存在_按类型并按特征I64` 的类型比较当前是指针比较。
- `比较存在` 只比较直接子特征，不递归。
- `刷新存在比较二次特征` 要求两个存在同场景或至少能解析出一个共同所属场景。
- `写入存在观测位置` 只有在存在直接挂在场景下时才刷新场景存在型基础二次特征。
