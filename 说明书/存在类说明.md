# 存在类说明

本文说明 `D:\鱼巢\存在类.h`、`D:\鱼巢\存在类.cpp`，并补充 `D:\鱼巢\基础信息主信息类.h/.cpp` 中 `存在节点主信息类` 与本类直接相关的字段和生命周期。

`存在类` 是基础信息树中“存在节点”的服务层。它负责创建、查找、删除存在节点，维护存在的名称、类型、内部世界、观测位置、实例因果引用，以及每个存在自带的需求、任务、方法三类独立根节点。

## 1. 文件定位

当前项目中，“存在”是世界树里的基本对象承载点：

- 存在节点可以挂在世界根、场景、存在或其他基础信息节点下面。
- 存在可以直接承载实例特征；抽象特征统一挂在世界树根下的抽象特征链。
- 任务、方法、线程等对象需要特征时，不能直接挂特征，必须先通过虚拟存在承载。
- 每个存在主信息内部持有需求根、任务根、方法根三棵独立信息树。
- 存在还可以绑定一个内部世界，内部世界是场景节点。

简单说：

- `基础信息类` 提供树和链表结构。
- `存在节点主信息类` 保存存在的语义和内部状态。
- `存在类` 提供围绕存在节点的创建、查找、绑定、观测和特征辅助能力。

## 2. 存在节点主信息

`存在节点主信息类` 定义在 `基础信息主信息类.h`。

### 2.1 字段

| 字段 | 用途 |
| --- | --- |
| `概念模板` | 指向该存在所属或参考的概念模板存在。 |
| `内部世界` | 指向该存在的内部世界场景。 |
| `需求根节点` | 该存在拥有的需求树独立根节点。 |
| `任务根节点` | 该存在拥有的任务树独立根节点。 |
| `方法根节点` | 该存在拥有的方法树独立根节点。 |
| `有位置历史` | 是否已有至少一次观测位置。 |
| `上次观测位置_mm` | 上一次观测位置，单位毫米。 |
| `最近观测位置_mm` | 最新观测位置，单位毫米。 |
| `最近位移_mm` | 最近两次观测位置之间的距离。 |
| `连续静止帧` | 连续未发生位移的观测次数。 |
| `连续未命中帧` | 连续未观测到该存在的次数。 |
| `实例因果` | 与该存在关联的因果实例引用列表。 |

### 2.2 三类根节点

每个存在有三棵独立信息树：

- 需求树根：`需求根节点`
- 任务树根：`任务根节点`
- 方法树根：`方法根节点`

这三棵根不是基础信息树的子节点，而是由 `存在节点主信息类` 直接持有裸指针。`存在类` 在读取存在主信息、创建存在、或显式确保根节点时，会补齐这三个根。

生命周期：

- `存在类` 使用 `私有_创建独立信息根节点` 创建孤立根节点。
- `存在节点主信息类::~存在节点主信息类()` 会调用 `需求类::释放树_独立`、`任务类::释放树_独立`、`方法类::释放树_独立` 释放三棵树。
- 删除存在时，`存在类::删除存在` 会先递归释放基础信息子树的主信息，再删除基础信息节点。

注意：

- 需求/任务/方法根节点是“该存在自己的内部管理树”，不是基础信息树中的普通子节点。
- 读取存在主信息的只读重载也会通过 `const_cast` 补齐三类根节点，因此它有惰性修复副作用。

### 2.3 `比较`

`存在节点主信息类::比较` 支持按字段比较：

| 比较字段 | 行为 |
| --- | --- |
| `基础信息基类_名称` / `名称` | 比较左右名称指针是否相同。 |
| `基础信息基类_类型` / `类型` | 比较左右类型指针是否相同。 |
| `评估_全局置信度` | 返回左右统计可信度差值。 |
| 其他字段 | 返回 `-1`。 |

注意：

- 这里的名称、类型比较是指针比较，不是词性主键比较。
- `存在类` 查找时使用的是 `私有_词性相同`，会兼容词性主键相同的情况。

## 3. `存在类` 的职责

`存在类` 主要做六件事：

1. 绑定基础信息树。
2. 读取存在主信息，并确保每个存在具备需求、任务、方法三类根节点。
3. 创建、查找、去重式获取、删除存在节点。
4. 提供存在或场景下特征节点的辅助创建和查找。
5. 写入存在名称、类型、概念模板、内部世界。
6. 记录观测位置、未命中状态和实例因果引用。

私有状态：

| 字段 | 用途 |
| --- | --- |
| `基础信息_` | 指向基础信息树。为空时大多数操作返回空或失败。 |

## 4. 构造与绑定

### 4.1 `存在类(基础信息类* 基础信息 = nullptr) noexcept`

用途：

- 创建存在服务对象，并可选绑定基础信息树。

用法：

```cpp
存在类 存在服务(&基础信息);
```

注意：

- 只保存指针，不拥有基础信息树生命周期。
- 传入 `nullptr` 时，后续必须调用 `绑定基础信息` 才能正常创建或查找存在。

### 4.2 `绑定基础信息(基础信息类* 基础信息) noexcept`

用途：

- 设置或替换 `存在类` 使用的基础信息树。

用法：

```cpp
存在服务.绑定基础信息(&世界树.基础信息());
```

注意：

- 不迁移已有节点。
- 调用方必须保证传入的基础信息树在使用期间有效。

## 5. 主信息读取与三类根节点

### 5.1 `取存在主信息(存在节点类* 节点) const noexcept`

用途：

- 从基础信息节点中取出可写的 `存在节点主信息类`。
- 如果主信息存在，则确保需求根、任务根、方法根已经补齐。

返回：

- 成功返回 `存在节点主信息类*`。
- `基础信息_` 为空、节点为空、或节点主信息不是存在时返回 `nullptr`。

用法：

```cpp
auto* 主信息 = 存在服务.取存在主信息(存在节点);
if (主信息) {
    主信息->类型 = 类型词;
}
```

注意：

- 这个函数有惰性补齐行为，读取时可能创建三类根节点。
- 普通业务写入优先使用 `写入名称`、`写入类型`、`绑定内部世界` 等封装函数。

### 5.2 `取存在主信息(const 存在节点类* 节点) const noexcept`

用途：

- 只读版本，用于读取存在主信息。

行为：

- 与可写版本一样，会在主信息存在时补齐三类根节点。
- 当前实现通过 `const_cast` 对主信息做惰性补齐。

### 5.3 `确保存在三类根节点(存在节点类* 节点) const noexcept`

用途：

- 显式确保某个存在拥有需求、任务、方法三类独立根节点。

返回：

- 如果补齐过至少一个根节点，返回 `true`。
- 如果节点无效、主信息无效，返回 `false`。
- 如果三类根节点本来就齐全，返回 `false`。

注意：

- 返回 `false` 不一定表示失败，也可能表示“无需补齐”。

### 5.4 `确保存在三类根节点(const 存在节点类* 节点) const noexcept`

用途：

- 只读指针版本的显式确保。

行为：

- 主信息存在时同样通过 `const_cast` 补齐三类根节点。

### 5.5 `获取需求根节点`

重载：

```cpp
需求类::节点类* 获取需求根节点(存在节点类* 节点) const noexcept;
const 需求类::节点类* 获取需求根节点(const 存在节点类* 节点) const noexcept;
```

用途：

- 获取该存在自己的需求树根节点。

行为：

- 内部先调用 `取存在主信息`，因此会惰性补齐三类根。
- 主信息无效时返回 `nullptr`。

### 5.6 `获取任务根节点`

重载：

```cpp
任务类::节点类* 获取任务根节点(存在节点类* 节点) const noexcept;
const 任务类::节点类* 获取任务根节点(const 存在节点类* 节点) const noexcept;
```

用途：

- 获取该存在自己的任务树根节点。

注意：

- 任务树根是存在主信息持有的独立根，不是基础信息树子节点。

### 5.7 `获取方法根节点`

重载：

```cpp
方法类::节点类* 获取方法根节点(存在节点类* 节点) const noexcept;
const 方法类::节点类* 获取方法根节点(const 存在节点类* 节点) const noexcept;
```

用途：

- 获取该存在自己的方法树根节点。

## 6. 创建与删除存在

### 6.1 `创建存在(基础信息节点类* 父节点, 存在节点主信息类* 主信息)`

用途：

- 使用调用者准备好的 `存在节点主信息类` 创建存在节点，并挂到基础信息树的 `父节点` 下面。

行为：

- `基础信息_` 为空或 `主信息` 为空时返回 `nullptr`。
- 创建前会确保主信息具有需求、任务、方法三类根节点。
- 创建成功后，主信息生命周期交给基础信息树。

用法：

```cpp
auto* 主信息 = new 存在节点主信息类(名称词, 类型词);
auto* 存在节点 = 存在服务.创建存在(父节点, 主信息);
```

注意：

- 不检查同一父节点下是否已有同名或同类型存在。
- 需要复用已有节点时，使用 `取或创建子存在_按名称类型` 或 `取或创建子存在_按类型`。

### 6.2 `创建存在_按名称类型(基础信息节点类* 父节点, const 词性节点类* 名称 = nullptr, const 词性节点类* 类型 = nullptr)`

用途：

- 按名称和类型快速创建存在节点。

行为：

- 内部创建新的 `存在节点主信息类`。
- 设置主信息的 `名称` 和 `类型`。
- 转调 `创建存在`。

用法：

```cpp
auto* 存在节点 = 存在服务.创建存在_按名称类型(场景节点, 名称词, 类型词);
```

注意：

- 名称和类型都允许为空。
- 该函数不去重。

### 6.3 `取或创建子存在_按名称类型(...)`

签名：

```cpp
存在节点类* 取或创建子存在_按名称类型(
    基础信息节点类* 父节点,
    const 词性节点类* 名称 = nullptr,
    const 词性节点类* 类型 = nullptr);
```

用途：

- 在父节点的直接子存在中查找匹配名称和类型的存在；找不到时创建。

匹配规则：

- `名称 == nullptr` 时，不限制名称。
- `类型 == nullptr` 时，不限制类型。
- 非空名称或类型使用 `私有_词性相同`，先比指针，再比词性主键。

用法：

```cpp
auto* 任务虚拟存在 = 存在服务.取或创建子存在_按名称类型(
    承载场景,
    任务虚拟存在名称词,
    任务虚拟存在类型词);
```

注意：

- 只查找直接子存在，不递归。
- 如果名称和类型都为空，将命中父节点下第一个存在；调用时应避免这种含糊用法。

### 6.4 `取或创建子存在_按类型(基础信息节点类* 父节点, const 词性节点类* 类型)`

用途：

- 在父节点直接子存在中按类型查找；找不到时创建名称为空、类型为指定类型的新存在。

行为：

- `类型` 为空时返回 `nullptr`。
- 命中则返回已有存在。
- 未命中则调用 `创建存在_按名称类型(父节点, nullptr, 类型)`。

### 6.5 `取或创建子存在_按类型并按特征I64(...)`

签名：

```cpp
存在节点类* 取或创建子存在_按类型并按特征I64(
    基础信息节点类* 父节点,
    const 词性节点类* 类型,
    const 词性节点类* 唯一键特征类型,
    I64 唯一键特征值);
```

用途：

- 通过“存在类型 + 某个 I64 特征值”定位唯一子存在；找不到时创建并写入该唯一键特征。

典型用途：

- 一个父节点下存在多个同类型子存在，需要用某个 I64 特征作为唯一键。
- 例如按线程编号、任务编号、外部 ID 等建立承载存在。

流程：

1. 参数 `类型` 或 `唯一键特征类型` 为空时返回 `nullptr`。
2. 遍历父节点直接子存在。
3. 先比较存在类型。
4. 再查找该存在下面的唯一键特征。
5. 如果特征当前值是 `I64` 且等于目标值，返回该存在。
6. 未命中时创建新存在。
7. 在新存在下面创建唯一键特征，并写入目标 I64 值。
8. 返回新存在。

注意：

- 唯一键特征直接挂在新存在下面，符合特征只能挂存在或场景的规则。
- 当前实现创建唯一键特征时使用 `存在类::创建特征_按类型`，不会走 `特征类::写入特征值_按参数` 的统计和稳态逻辑。

### 6.6 `删除存在(存在节点类* 节点)`

用途：

- 删除一个存在节点及其基础信息子树。

流程：

1. `基础信息_` 为空或节点为空时返回 `false`。
2. 调用 `私有_递归释放基础信息主信息`，递归删除该节点及其子节点的主信息。
3. 调用 `基础信息_->删除节点(节点)` 删除基础信息节点。

注意：

- 删除存在会释放该存在主信息持有的需求、任务、方法三棵独立树。
- 递归释放的是基础信息子树中的主信息，不只是存在自身。
- 如果其他地方保存了该存在、其子节点、或其内部三类树节点指针，删除后这些指针都会失效。

返回：

- 否则返回所有主信息为 `存在节点主信息类` 的节点。

### 7.2 `获取子存在(const 基础信息节点类* 父节点) const`

行为：

### 7.3 `查找子存在_按名称(const 基础信息节点类* 父节点, const 词性节点类* 名称) const`

用途：

- 在父节点直接子存在中查找指定名称的存在。

行为：

- `名称` 为空时返回 `nullptr`。
- 名称比较使用 `私有_词性相同`。
- 返回第一个命中的存在。

### 7.4 `查找子存在_按类型(const 基础信息节点类* 父节点, const 词性节点类* 类型) const`

用途：

- 在父节点直接子存在中查找指定类型的存在。

行为：

- `类型` 为空时返回 `nullptr`。
- 类型比较使用 `私有_词性相同`。
- 返回第一个命中的存在。

## 8. 特征辅助函数

`存在类` 内部也提供了一组特征辅助函数。它们是历史上围绕存在节点使用特征的便捷入口。

重要规则：

- 这里的特征辅助函数只处理实例特征；抽象特征请走 `世界树.取或创建抽象特征根()` 和 `特征类` 的抽象特征接口。
- 实例特征只能直接挂在存在、场景或虚拟存在下面。
- 任务、方法、线程不能直接作为特征宿主。
- 任务、方法、线程需要特征时，必须先取得对应虚拟存在，再把虚拟存在作为这里的 `宿主`。
- 当前 `存在类` 代码本身没有强制检查宿主是否为存在、场景或虚拟存在，调用方必须遵守该规则。

### 8.1 `创建特征_按类型(基础信息节点类* 宿主, const 词性节点类* 特征类型, const 特征值& 当前值 = {})`

用途：

- 在存在、场景或虚拟存在下面创建实例特征节点。

行为：

- `基础信息_` 为空或 `特征类型` 为空时返回 `nullptr`。
- 内部创建 `特征节点主信息类`。
- 设置 `主信息->类型` 和 `主信息->当前值`。
- 把特征作为 `宿主` 子节点挂入基础信息树。

用法：

```cpp
auto* 特征节点 = 存在服务.创建特征_按类型(
    存在节点,
    唯一键特征类型,
    特征值{ static_cast<I64>(1001) });
```

注意：

- 当前函数没有设置 `主信息->名称`。
- 当前函数不去重。
- 当前函数不维护特征统计、近帧候选和稳态。
- 更通用的特征写入建议走 `世界树类` 或 `特征类` 的封装。

### 8.2 `获取子特征(const 基础信息节点类* 宿主) const`

用途：

- 获取存在、场景或虚拟存在下面的直接特征子节点。

行为：

- 只枚举直接子节点。
- `基础信息_` 为空时返回空数组。

### 8.3 `查找子特征_按类型(const 基础信息节点类* 宿主, const 词性节点类* 特征类型) const`

用途：

- 在存在、场景或虚拟存在下面查找指定类型的直接特征。

行为：

- `特征类型` 为空时返回 `nullptr`。
- 类型比较使用 `私有_词性相同`。
- 返回第一个命中的特征。

示例：

```cpp
auto* 特征节点 = 存在服务.查找子特征_按类型(任务虚拟存在, 任务状态特征词);
```

## 9. 写入存在属性

### 9.1 `写入名称(存在节点类* 节点, const 词性节点类* 名称)`

用途：

- 修改存在主信息的名称。

返回：

- 主信息存在时写入并返回 `true`。
- 主信息不存在时返回 `false`。

### 9.2 `写入类型(存在节点类* 节点, const 词性节点类* 类型)`

用途：

- 修改存在主信息的类型。

返回规则同 `写入名称`。

### 9.3 `绑定概念模板(存在节点类* 节点, 存在节点类* 概念模板)`

用途：

- 把某个存在绑定到概念模板存在。

行为：

- 用 `私有_生成引用` 同时保存指针和主键。

### 9.4 `绑定内部世界(存在节点类* 节点, 场景节点类* 内部世界)`

用途：

- 给存在绑定内部世界场景。

行为：

- 用 `可解析引用<场景节点类>` 保存内部世界。
- 主信息不存在时返回 `false`。

注意：

- 该函数只写存在主信息里的引用。
- 创建内部世界、把场景绑定回宿主存在，通常由 `世界树类::取或创建存在内部世界` 统一处理。

## 10. 观测与因果

### 10.1 `写入观测位置(存在节点类* 节点, const Vector3D& 位置_mm)`

用途：

- 更新存在的空间观测位置，并维护位移、静止帧和未命中帧。

流程：

1. 读取存在主信息，失败则返回 `false`。
2. 如果已有位置历史：
   - 把原最近位置写入 `上次观测位置_mm`。
   - 计算原最近位置到新位置的距离，写入 `最近位移_mm`。
3. 如果没有位置历史：
   - `上次观测位置_mm` 设为新位置。
   - `最近位移_mm` 设为 `0`。
4. 更新 `最近观测位置_mm`。
5. 设置 `有位置历史 = true`。
6. 清零 `连续未命中帧`。
7. 如果已有历史且位移为 `0`，递增 `连续静止帧`；否则清零 `连续静止帧`。

距离计算：

- 使用三维欧氏距离。
- 输入单位是毫米。
- 结果四舍五入为 `std::int64_t`。

用法：

```cpp
存在服务.写入观测位置(存在节点, Vector3D{ 100, 0, 20 });
```

### 10.2 `标记观测未命中(存在节点类* 节点)`

用途：

- 记录一次本轮观测未发现该存在。

行为：

- `连续未命中帧 += 1`
- `连续静止帧 = 0`

返回：

- 主信息存在时返回 `true`。
- 主信息不存在时返回 `false`。

### 10.3 `追加实例因果(存在节点类* 节点, 因果实例节点类* 因果实例)`

用途：

- 给存在追加一个相关因果实例引用。

行为：

- 主信息或因果实例为空时返回 `false`。
- 如果列表中已经有同指针或同主键引用，不重复追加。
- 否则把指针和主键写入 `实例因果`。

## 11. 私有辅助函数

这些函数位于 `存在类.cpp` 的匿名命名空间，只供本文件内部使用。

| 函数 | 用途 |
| --- | --- |
| `私有_计算距离_mm` | 计算两个 `Vector3D` 位置之间的三维欧氏距离，结果为毫米整数。 |
| `私有_词性相同` | 比较两个词性节点，先比指针，再比主键。 |

| `私有_确保存在具备三类信息根节点` | 确保存在主信息中需求根、任务根、方法根都存在。 |

## 12. 常见使用路径

### 12.1 创建或取得一个场景下的存在

```cpp
auto* 存在节点 = 存在服务.取或创建子存在_按名称类型(
    场景节点,
    名称词,
    类型词);
```

### 12.2 为任务建立虚拟存在后写特征

```cpp
auto* 任务虚拟存在 = 存在服务.取或创建子存在_按名称类型(
    承载场景,
    任务虚拟存在名称词,
    任务虚拟存在类型词);

auto* 状态特征 = 存在服务.查找子特征_按类型(任务虚拟存在, 任务状态词);
```

注意：

- 特征挂在 `任务虚拟存在` 下面，而不是任务节点下面。
- 任务节点只通过主信息中的引用指向它的虚拟存在。

### 12.3 获取某个存在自己的需求/任务/方法根

```cpp
auto* 需求根 = 存在服务.获取需求根节点(自我存在);
auto* 任务根 = 存在服务.获取任务根节点(自我存在);
auto* 方法根 = 存在服务.获取方法根节点(自我存在);
```

### 12.4 给存在绑定内部世界

推荐使用世界树封装：

```cpp
auto* 内部世界 = 世界树.取或创建存在内部世界(自我存在);
```

这会同时完成：

- 创建或复用内部世界场景。
- 场景绑定宿主存在。
- 存在绑定内部世界引用。

## 13. 与世界树的关系

项目中普通业务代码通常通过 `世界树类` 使用存在服务：

- `世界树类::创建存在` 转调 `存在类::创建存在`。
- `世界树类::取或创建子存在_按类型` 提供按类型复用入口。
- `世界树类::取或创建子存在_按类型并按特征I64` 提供按类型和 I64 唯一键复用入口。
- `世界树类::取存在内部世界` 读取存在内部世界。
- `世界树类::取或创建存在内部世界` 同时处理内部世界创建、场景宿主绑定、存在引用绑定。
- `世界树类::写入存在观测位置` 在写入位置后，会刷新场景存在型基础二次特征。

因此：

- 创建、查找普通存在时可以用 `存在类`。
- 涉及内部世界、特征写入、二次特征刷新时，优先使用 `世界树类` 封装。

## 14. 注意事项

- `存在类` 不拥有基础信息树，只保存裸指针。
- `取存在主信息` 有惰性补齐三类根节点的副作用。
- 存在的需求根、任务根、方法根是独立树，不是基础信息树子节点。
- `创建存在` 和 `创建存在_按名称类型` 不做去重。
- `取或创建子存在_按名称类型` 只查找直接子存在，不递归。
- 名称和类型都为空时，`取或创建子存在_按名称类型` 会匹配第一个子存在，应避免这样调用。
- `存在类` 的特征辅助函数不会强制检查宿主类型，调用方必须保证特征只挂在存在或场景下面。
- 任务、方法、线程需要特征时，一律通过虚拟存在承载。
- `存在类::创建特征_按类型` 不维护特征统计和稳态，正式特征写入优先走 `世界树类` 或 `特征类`。
- 删除存在会释放基础信息子树主信息，也会释放该存在主信息持有的需求/任务/方法三棵独立树。
