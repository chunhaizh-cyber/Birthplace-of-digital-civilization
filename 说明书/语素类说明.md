# 语素类说明

本文说明 `D:\鱼巢\语素主信息类.h`、`D:\鱼巢\语素类.h`、`D:\鱼巢\语素类.cpp`、`D:\鱼巢\语素节点类型.h`、`D:\鱼巢\语素信息类型.h`。

`语素类` 是项目里的全局词库和词性节点服务层。它把词按 UTF-8 字符前缀组织成一棵词树，并在完整词节点下挂词性节点；基础信息、特征类型、方法名、需求类型等需要名字或类型词时，通常都会引用这里的 `词性节点类*`。

## 1. 文件定位

当前项目中，语素系统主要承担三件事：

- 保存词节点：如 `安全值`、`方法是否可用`、`需求_权重`。
- 保存词性节点：如 `安全值/名词`、`尝试学习/动词`。
- 让词性节点可选地指向一个基础信息节点，作为名字到基础信息的轻量绑定。

简单说：

- `语素类` 管词和词性。
- `语言类` 或自然语言结构负责句子、文章、段落等语言结构。
- `基础信息类`、`世界树类`、`方法类`、`任务类` 等通过 `词性节点类*` 使用词，不直接复制词字符串作为主身份。

## 2. 节点别名

`语素节点类型.h` 统一把语素相关节点都定义成链表节点别名：

```cpp
using 语素节点类 = 链表模板<语素基类*>::节点类;
using 词节点类 = 语素节点类;
using 词性节点类 = 语素节点类;
using 短语节点类 = 语素节点类;
using 短语子节点类 = 语素节点类;
```

这些别名指向同一种链表节点类型，区别主要由节点上的 `主信息` 动态类型决定。

## 3. 树结构

`语素类` 继承自：

```cpp
链表模板<语素基类*>
```

因此语素树是一棵独立链表树，不是世界树的一部分。

词节点按 UTF-8 字符前缀建立。例如添加 `安全值/名词` 后，结构大致是：

```text
语素根
  安
    安全
      安全值
        名词
```

注意：

- 每个前缀都是一个 `词节点类`。
- 词性节点挂在完整词节点下面。
- 同一个词可以有多个词性节点。
- 词性节点才是大多数外部模块引用的“词性词指针”。

## 4. 主信息类型

### 4.1 `语素基类`

所有语素主信息的基类，字段为：

| 字段 | 默认值 | 用途 |
| --- | --- | --- |
| `主信息类型` | `枚举_主信息类型::语素基类` | 标识当前主信息动态类型。 |

它声明纯虚函数：

```cpp
virtual int 比较(const 语素基类* other, 枚举_比较字段 字段) const = 0;
```

用于链表排序、检索或去重时比较指定字段。

### 4.2 `词主信息类`

词节点的主信息。

| 字段 | 默认值 | 用途 |
| --- | --- | --- |
| `词` | 构造参数 | 保存当前前缀或完整词字符串。 |

支持比较字段：

| 字段 | 含义 |
| --- | --- |
| `枚举_比较字段::语素_词_词` | 按 `词` 字符串比较。 |

### 4.3 `词性主信息类`

词性节点的主信息。

| 字段 | 默认值 | 用途 |
| --- | --- | --- |
| `词性` | `枚举_词性::未定义` | 当前词性枚举。 |
| `对应基础信息` | 空引用 | 可选绑定到一个基础信息节点。 |

支持比较字段：

| 字段 | 含义 |
| --- | --- |
| `枚举_比较字段::语素_词性_词性` | 按词性枚举值比较。 |

注意：

- `语素_词性_对应基础信息指针` 已在枚举中定义，但当前 `词性主信息类::比较` 没有实现该字段。
- `对应基础信息` 当前只保存一个指针，不是多场景、多目标绑定表。

### 4.4 `短语主信息类`

短语节点的主信息。

| `枚举_比较字段::语素_短语字节点_比较对象词` | 比较 `比较对象词` 指针、主键或地址。 |

短语子节点的主信息。

| 字段 | 默认值 | 用途 |
| --- | --- | --- |

| `结果词` | `nullptr` | 比较结果词。 |

支持比较字段：

| 字段 | 含义 |
| --- | --- |
| `枚举_比较字段::语素_短语子节点_比较词` | 比较 `比较词` 指针、主键或地址。 |

## 5. 词性枚举和转换

`枚举_词性` 定义在 `基础数据类型.h`，当前包含中文词性、分词缩写和专名识别缩写。

常见中文词性：

```text
名词、动词、形容词、副词、连词、介词、专有名词、抽象名词、代词、数词、量词、
助词、叹词、拟声词、标点符号、时间词、状态词、方位词、习用语、简称略语、
非语素字、语气词、外文字符、短语、地名、未定义
```

常见缩写：

```text
n, f, s, t, nr, ns, nt, nw, nz, v, vd, vn, a, ad, an, d, m, q, r, p, c, u, xc, w,
PER, LOC, ORG, TIME
```

`枚举_词性_工厂` 提供转换：

| 函数 | 用途 |
| --- | --- |
| `根据枚举类型获取文本` | 从 `枚举_词性` 得到文本。 |
| `根据文本获取枚举值` | 从文本得到 `枚举_词性`，找不到时返回 `未定义`。 |
| `添加动态映射` | 修改枚举到文本的输出映射。 |
| `批量添加动态映射` | 批量修改枚举到文本的输出映射。 |
| `清除动态映射` | 清空动态输出映射，并从内置文本表恢复。 |

注意：

- 动态映射只影响“枚举到文本”的输出。
- `根据文本获取枚举值` 只查内置文本表，动态映射不会让新文本变成可解析词性。
- `添加词性词` 不允许添加 `枚举_词性::未定义`。

## 6. 生命周期

全局对象定义在 `语素类.cpp`：

```cpp
语素类 语素集{};
```

`语素类` 禁止拷贝和移动：

```cpp
语素类(const 语素类&) = delete;
语素类& operator=(const 语素类&) = delete;
语素类(语素类&&) = delete;
语素类& operator=(语素类&&) = delete;
```

析构时会加锁并清理节点上的 `主信息`。

### 6.1 `初始化`

```cpp
void 初始化();
```

预热一批需求和特征相关词：

```text
存在_需求虚拟存在/名词
需求虚拟存在/名词
需求_权重/名词
二次特征_标量值/名词
安全需求/名词
服务需求/名词
自我根需求_安全/名词
自我根需求_服务/名词
```

用途：

- 确保自我初始化和需求系统常用词先存在。
- 避免后续模块重复处理这些基础词的创建问题。

## 7. 添加和查找词

### 7.1 `添加词`

```cpp
词节点类* 添加词(const std::string& 词字符串);
```

用途：

- 按 UTF-8 字符切分输入字符串。
- 逐级创建前缀词节点。
- 返回完整词对应的词节点。

行为：

- 输入空字符串会抛出 `std::invalid_argument`。
- 如果前缀节点已存在，则复用已有节点。
- 如果创建节点失败，会释放本次新建主信息并返回 `nullptr`。

示例：

```cpp
auto* 词节点 = 语素集.添加词("方法是否可用");
```

### 7.2 `查找词节点`

```cpp
词节点类* 查找词节点(const std::string& 词字符串) const;
```

用途：

- 按同样的 UTF-8 前缀路径查找完整词节点。

行为：

- 输入空字符串会抛出 `std::invalid_argument`。
- 找不到返回 `nullptr`。

### 7.3 `获取词性词指针`

```cpp
词节点类* 获取词性词指针(const std::string& 词字符串) const;
```

用途：

- 当前实现等价于“按词字符串查找词节点”。

注意：

- 函数名带“词性词指针”，但返回类型是 `词节点类*`。
- 输入空字符串时返回 `nullptr`，不会抛异常。

## 8. 添加和查找词性节点

### 8.1 `生成语素节点`

```cpp
词性节点类* 生成语素节点(结构体_分词 分词信息);
```

用途：

- 按分词结果创建或复用词性节点。

当前实现：

```cpp
return 添加词性词(std::move(分词信息));
```

### 8.2 `添加词性词`

```cpp
词性节点类* 添加词性词(词节点类* 词节点, 枚举_词性 词性);
词性节点类* 添加词性词(const std::string& 词值, const std::string& 词性值);
词性节点类* 添加词性词(结构体_分词 词信息);
```

用途：

- 在词节点下面创建或复用对应词性的词性节点。

行为：

- `词节点` 为空会抛出 `std::invalid_argument`。
- `词性 == 未定义` 会抛出 `std::invalid_argument`。
- 如果词性节点已存在，直接返回已有节点。
- 字符串重载会先 `添加词(词值)`，再把 `词性值` 转成枚举。

示例：

```cpp
auto* 名词节点 = 语素集.添加词性词("安全值", "名词");
auto* 动词节点 = 语素集.添加词性词("尝试学习", "动词");
```

### 8.3 `查找词性节点`

```cpp
词性节点类* 查找词性节点(const 词节点类* 词节点, 枚举_词性 词性值) const;
词性节点类* 查找词性节点(const 语素节点类* 词节点, const std::string& 词性值) const;
词性节点类* 查找词性节点(结构体_分词 分词信息) const;
```

用途：

- 在一个词节点的直接子节点中查找指定词性节点。

行为：

- 只查直接子链，不递归查更深层。
- 字符串词性无法解析时会变成 `未定义`，通常找不到节点。
- 分词重载会先查词节点，再查词性节点。

## 9. 词性节点和基础信息绑定

### 9.1 `绑定词性词到场景基础信息`

```cpp
const 词性节点类* 绑定词性词到场景基础信息(
    const std::string& 词字符串,
    枚举_词性 词性,
    场景节点类* 场景,
    基础信息节点类* 基础信息节点
);
```

用途：

- 创建或复用 `词字符串/词性` 节点。
- 把该词性节点的 `对应基础信息` 指向传入的基础信息节点。

当前实现特点：

- `场景` 参数目前没有参与绑定逻辑。
- `词字符串` 为空或 `基础信息节点` 为空会抛异常。
- 返回绑定后的词性节点。

### 9.2 `添加对应基础信息`

```cpp
std::int64_t 添加对应基础信息(const 词性节点类* 词节点, 基础信息节点类* 基础信息);
```

用途：

- 直接给词性节点写入 `对应基础信息`。

行为：

- `词节点` 为空会抛异常。
- `基础信息` 为空会抛异常。
- 如果节点主信息不是 `词性主信息类`，会抛 `std::runtime_error`。
- 成功返回 `1`。

注意：

- 当前是覆盖式单指针绑定。
- 写入后会清空 `对应基础信息.主键`，只保留直接指针。

### 9.3 `通过词和场景查找基础信息`

```cpp
std::vector<基础信息节点类*> 通过词和场景查找基础信息(
    const std::string& 词字符串,
    场景节点类* 当前场景,
    const 枚举_词性* 词性 = nullptr
) const;
```

用途：

- 按词字符串找到词节点。
- 遍历其直接词性子节点。
- 收集词性匹配且 `对应基础信息.指针` 非空的基础信息节点。

当前实现特点：

- `当前场景` 参数暂未参与过滤。
- 如果 `词性 == nullptr`，会收集所有词性上的基础信息指针。
- 只返回已直接保存指针的基础信息，不按主键二次解析。

### 10.1 `添加二次特征词`

```cpp
词节点类* 添加二次特征词(语素节点类* 参照物, 语素节点类* 比较对象, 语素节点类* 比较类型);
```

用途预期：

- 根据参照物、比较对象、比较类型生成二次特征相关词。

当前实现：

- 参数全部未使用。
- 固定返回 `nullptr`。

### 10.2 `获取词`

```cpp
std::string 获取词(const 词性节点类* 词性词指针) const;
```

用途：

- 从词性节点回到父词节点，并返回父词节点保存的词字符串。

行为：

- `词性词指针` 为空会抛异常。
- 父节点为空或父节点无主信息会抛异常。
- 父节点主信息不是 `词主信息类` 会抛 `std::runtime_error`。

示例：

```cpp
auto* 名词节点 = 语素集.添加词性词("安全值", "名词");
auto 词 = 语素集.获取词(名词节点); // "安全值"
```

## 12. 与其它系统的关系

### 12.1 与基础信息

基础信息节点通常用 `词性节点类*` 表示名字或类型。

例如特征类型、存在名称、场景名称等，都可以通过语素系统创建或查找对应词性节点。

### 12.2 与特征系统

特征类型常以 `词性节点类*` 表示，例如：

```text
安全值/名词
服务值/名词
方法是否可用/名词
任务推进状态/名词
```

语素类只提供“这个特征类型叫什么”的词节点，不承载特征当前值。

特征当前值必须挂在存在、场景或虚拟存在下面的特征节点上；任务、方法、线程需要特征时，必须通过对应虚拟存在承载。

### 12.3 与方法和任务

方法名、动作名、需求类型、任务状态类型等文本身份可以来自语素词性节点。

但语素节点本身不表示方法结构、任务结构或运行事实：

- 方法结构由方法树维护。
- 任务结构和任务列表由任务系统维护。
- 运行状态和特征写入世界树中的虚拟存在。

## 13. 常用用法

### 13.1 创建一个特征类型词

```cpp
auto* 类型词 = 语素集.添加词性词("方法是否可用", "名词");
```

### 13.2 查找已有词性词

```cpp
auto* 词节点 = 语素集.查找词节点("方法是否可用");
auto* 类型词 = 语素集.查找词性节点(词节点, 枚举_词性::名词);
```

### 13.3 通过分词结构创建

```cpp
结构体_分词 分词{ "尝试学习", "动词" };
auto* 动作词 = 语素集.生成语素节点(分词);
```

### 13.4 绑定基础信息

```cpp
auto* 名词节点 = 语素集.添加词性词("某存在", "名词");
语素集.添加对应基础信息(名词节点, 某基础信息节点);
```

## 14. 现状注意点

- `语素类` 是全局词库，不是事实库；不要把运行状态直接写成语素节点。
- `添加词` 会创建每个 UTF-8 前缀节点，因此长词会带来一串前缀词节点。
- `添加词性词(const std::string&, const std::string&)` 依赖内置词性文本表；未知词性会转成 `未定义` 并被拒绝添加。
- `获取词性词指针` 当前返回的是词节点，不是词性节点。
- `绑定词性词到场景基础信息` 和 `通过词和场景查找基础信息` 的场景参数当前没有实际过滤作用。
- `添加对应基础信息` 是单指针覆盖，不适合表达同一个词性在多个场景下对应多个基础信息。
- `添加二次特征词` 目前尚未实现。
- 语素节点不应成为任务、方法、线程的特征宿主；这些对象需要特征时，应通过各自虚拟存在挂载特征节点。
