# 双向链表模板序列化说明

本文说明 `D:\鱼巢\双向链表模板_序列化.inl` 中各函数的用途、用法、文件格式和注意事项。

## 1. 文件定位

`双向链表模板_序列化.inl` 是 `链表模板<主信息类型>` 的模板分片文件，由 `双向链表模板核心.h` 在末尾 `#include` 引入。

它负责把整棵链表模板保存到二进制文件，并从二进制文件恢复。和深拷贝不同，序列化会保存并恢复三套关系：

- 同层环：`上/下/同层环头`
- 父子树：`父/子`
- 全链环：`链上/链下`

## 2. 主信息序列化规则

模板支持三类主信息：

| 主信息类型 | 保存方式 | 读取方式 |
| --- | --- | --- |
| 指针类型 | 先写 `has` 标记；非空时调用 `主信息->序列化(ofs)`。 | 非空时调用 `Pointee::反序列化(ifs)` 创建对象。 |
| 值类型且提供 `序列化/反序列化` | 保存时调用 `主信息.序列化(ofs)`。 | 读取时调用 `主信息类型::反序列化(ifs)`。 |
| 可平凡拷贝值类型 | 直接按 POD 二进制写入。 | 直接按 POD 二进制读取。 |

注意：核心模板不拥有指针型主信息生命周期。读取失败时，本函数会释放本次读取中新建的指针主信息；读取成功后，后续旧树删除仍遵守核心模板规则，不自动释放旧指针主信息对象。

## 3. 基本用法

```cpp
链表模板<我的主信息> 树;

bool 已保存 = 树.保存数据("tree.bin");
bool 已读取 = 树.读取数据("tree.bin");

if (已读取) {
    bool 正常 = 树.校验结构();
}
```

批量控制锁时可以使用 `_已加锁` 版本：

```cpp
auto lk = 树.获取读锁();
bool 已保存 = 树.保存数据_已加锁("tree.bin");
```

读取会替换当前树结构，因此应持写锁：

```cpp
auto lk = 树.获取锁();
bool 已读取 = 树.读取数据_已加锁("tree.bin");
```

## 4. 文件格式

当前版本格式为二进制格式，版本号为 `1`。

文件头：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `_存档魔数` | `std::uint32_t` | 固定值 `0x4C4C4442u`，注释为 `'BDLL' little-endian`。 |
| `ver` | `std::uint16_t` | 当前为 `1`。 |
| `count` | `std::uint64_t` | 保存节点数量，包含根节点。 |

节点数据按 `nodes` 顺序保存。`nodes[0]` 固定是 `根指针`，后续节点按全链环 `根指针->链下` 顺序排列。

每个节点保存：

| 字段 | 说明 |
| --- | --- |
| `主键` | 节点主键字符串。 |
| `子节点数量` | 直接子节点数量。 |
| `上/下/父/子/同层环头/链上/链下` | 七个指针字段对应的节点索引。空指针写为 `uint64_t` 最大值。 |
| `主信息` | 按主信息类型规则保存。 |

读取时先创建全部节点，再根据保存的索引统一恢复指针关系。

## 5. 工具函数说明

| 函数 | 用途 | 用法和注意 |
| --- | --- | --- |
| `_写POD(std::ostream& os, const T& v)` | 将可平凡拷贝值按二进制写入流。 | `T` 必须满足 `std::is_trivially_copyable_v<T>`。内部工具函数。 |
| `_读POD(std::istream& is, T& v)` | 从流中读取可平凡拷贝值。 | 读取成功返回 `true`，失败返回 `false`。内部工具函数。 |
| `_写字符串(std::ostream& os, const std::string& s)` | 写入字符串长度和内容。 | 先写 `uint32_t` 长度，再写原始字节。 |
| `_读字符串(std::istream& is, std::string& out, std::uint32_t 上限)` | 读取字符串长度和内容。 | 默认上限由核心声明给出，为 16MB。长度超过上限返回 `false`。 |
| `_索引或空(const std::unordered_map<const 节点类*, std::uint64_t>& mp, const 节点类* p)` | 将节点指针转换为保存用索引。 | 空指针或不在索引表中返回 `uint64_t` 最大值。 |
| `_由索引取指针(const std::vector<节点类*>& vec, std::uint64_t idx)` | 将保存的索引还原为节点指针。 | 空索引、越界索引返回空指针。 |

## 6. 保存函数说明

| 函数 | 用途 | 用法和注意 |
| --- | --- | --- |
| `保存数据_已加锁(const std::string& 文件名) const` | 不加锁保存整棵树到文件。 | 调用方必须已持有读锁或写锁。文件打不开返回 `false`。 |
| `保存数据(const std::string& 文件名) const` | 加读锁保存整棵树到文件。 | 外部常用入口。 |

保存流程：

1. 以二进制方式打开输出文件。
2. 写入魔数和版本号。
3. 从 `根指针` 开始构造节点数组。
4. 建立节点指针到数组索引的映射。
5. 对每个节点写入主键、子节点数量、七个关系指针索引和主信息。
6. 返回输出流状态。

保存时不主动调用 `校验结构()`。若结构已经损坏，保存结果也可能不可恢复。

## 7. 读取函数说明

| 函数 | 用途 | 用法和注意 |
| --- | --- | --- |
| `读取数据_已加锁(const std::string& 文件名)` | 不加锁从文件恢复整棵树。 | 调用方必须已持有写锁。读取成功后会替换当前 `根指针`。 |
| `读取数据(const std::string& 文件名)` | 加写锁读取文件。 | 外部常用入口。 |

读取流程：

1. 以二进制方式打开输入文件。
2. 读取并校验魔数。
3. 读取并校验版本号，目前只接受 `1`。
4. 读取节点数量，要求 `1 <= count <= 10000000`。
5. 为每个节点分配节点对象。
6. 读取每个节点的主键、子节点数量、七个关系索引和主信息。
7. 第二轮根据索引恢复所有指针。
8. 删除当前旧链表。
9. 将 `nodes[0]` 设为新的 `根指针`。

读取失败时会跳转到失败清理逻辑，删除本次读取已经分配的节点。若主信息是指针类型，也会删除本次反序列化得到的主信息对象。

## 8. 错误处理

保存失败常见原因：

- 文件无法打开。
- 输出流写入失败。
- 主信息类型不满足序列化要求。

读取失败常见原因：

- 文件无法打开。
- 魔数不匹配。
- 版本号不是 `1`。
- 节点数量为 `0` 或超过 `10000000`。
- 字符串长度超过上限。
- 指针型主信息反序列化返回空。
- 文件中途截断或读取失败。

读取失败时不会替换当前旧树，因为旧树只在全部节点和关系读完后才删除。

## 9. 和深拷贝的区别

| 项目 | 序列化 | 深拷贝 |
| --- | --- | --- |
| 保存范围 | 整个模板实例，包含根节点和全链环。 | 某个节点及其子树。 |
| 全链关系 | 保存并恢复。 | 不重建整棵复制树的全链环。 |
| 输出形式 | 二进制文件。 | 内存中的独立节点树。 |
| 主信息指针 | 可通过 `序列化/反序列化` 创建新对象。 | 只复制指针值。 |
| 主要用途 | 持久化、恢复。 | 临时快照、分析。 |

## 10. 常见错误

- 对已损坏的链表直接保存，可能把损坏结构固化到文件。
- 读取后不做 `校验结构()`，无法及时发现文件中关系索引异常。
- 指针型主信息没有实现 `序列化` 或 `static 反序列化`，会导致编译或读取失败。
- 将可平凡拷贝二进制格式用于跨版本结构，字段变更后可能不兼容旧文件。
- 对指针型主信息误以为模板会统一管理旧对象生命周期，可能造成外部对象泄漏或重复释放。

## 11. 推荐流程

保存前：

1. 调用 `校验结构()`。
2. 确认主信息类型的序列化接口稳定。
3. 调用 `保存数据(...)`。

读取后：

1. 调用 `读取数据(...)`。
2. 成功后立即调用 `校验结构()`。
3. 再允许上层业务使用恢复出来的树。
