# 任务类说明

本文说明 `D:\鱼巢\任务主信息类.h`、`D:\鱼巢\任务类.h`、`D:\鱼巢\任务类.cpp`。

`任务类` 是任务树的服务层。它负责创建任务头节点、兼容性的任务步骤节点和任务结果节点，并为任务创建“任务虚拟存在”，把任务运行状态、统计和特征账本落到世界树中。当前设计里，需求树才是任务组织关系的权威来源；任务节点主要作为某个需求的执行承载和运行记录承载。

## 1. 文件定位

当前项目中，任务位于需求和方法之间：

- 需求表达“需要什么目标状态”。
- 任务承载“为了满足这个需求，当前怎么推进”。
- 方法承载“用什么动作或结果方向来推进”。
- 任务管理界面线程负责调度、派发和推进任务状态。
- 工作线程负责单步执行，结果通过消息回到任务管理界面线程。

简单说：

- 需求树组织任务归属。
- 任务类维护任务节点和任务虚拟存在。
- 方法类维护候选方法和方法虚拟存在。
- 任务管理线程决定什么时候筹办、执行、等待、挂起或重筹办。

## 2. 任务树结构

`任务类` 继承自：

```cpp
链表模板<任务主信息类>
```

因此任务节点是独立链表树节点，不是 `基础信息节点类`。

现行主线：

```text
需求节点
  任务列表
    任务头节点
```

兼容接口仍保留旧结构：

```text
任务头节点
  任务步骤节点
    任务结果节点
```

注意：

- `任务主信息类` 注释已经说明：任务不再负责组织树结构，需求树才是组织结构。
- `步骤节点`、`结果节点` 是旧任务树兼容占位，仍有创建接口和部分调用点。
- 新的任务推进应优先围绕需求树、任务列表和任务状态，而不是重新依赖旧步骤/结果树作为主调度结构。

## 3. 任务状态枚举

`枚举_任务状态` 定义在 `任务主信息类.h`。

| 枚举值 | 值 | 含义 |
| --- | --- | --- |
| `未定义` | `0` | 尚未设置状态。 |
| `未启动` | `1` | 任务已存在，但尚未进入筹办或执行。 |
| `运行中` | `2` | 任务处于治理或执行推进中。 |
| `挂起` | `3` | 任务主动暂停，等待外部条件变化。 |
| `完成` | `4` | 任务目标已经达成。 |
| `失败` | `5` | 任务执行失败，且本轮不能自动恢复。 |
| `取消` | `6` | 任务被调度或上层需求取消。 |
| `超时` | `7` | 任务超过允许时限。 |
| `就绪` | `8` | 任务条件已具备，等待调度执行。 |
| `执行中` | `9` | 任务正在调用方法执行。 |
| `筹办中` | `10` | 任务正在寻找方法、补齐条件或拆分子需求。 |
| `排队中` | `11` | 任务已可执行，正在等待调度队列。 |
| `等待中` | `12` | 任务等待子需求、方法补齐或外部结果。 |
| `无法执行` | `13` | 当前没有可用方法或必要条件，不能推进。 |
| `待重筹办` | `14` | 依赖条件变化后，需要重新筹办。 |

注意：

- `任务类.cpp` 主要负责创建节点时写初始状态。
- 状态推进规则不集中在 `任务类` 内。
- 任务管理界面线程和任务管理模块根据消息、执行结果和负载控制来改变任务状态。

### 8.7 兼容占位字段

`任务主信息类` 中保留大量旧字段，包括：

- 旧任务树节点类型和树类型。
- 旧步骤和结果字段。
- 旧父任务、来源任务、后续任务字段。
- 旧方法条件、方法结果、结果方向字段。
- 旧安全权重、服务权重字段。
- 旧 `置信度_Q10000` 字段。

这些字段当前用于兼容旧模块和过渡编译。新设计中：

- 任务组织关系以需求树和需求节点任务列表为准。
- 任务结果应优先落到结果状态和执行记录。
- 置信度类新算法不应继续扩展 `Q10000` 口径。

### 8.8 `比较(...)`

按字段比较：

- `名称`：名称指针相同返回 `1`，否则 `0`。
- `类型`：类型指针相同返回 `1`，否则 `0`。
- `值`：状态相同返回 `1`，否则 `0`。
- 其他字段返回 `-1`。

## 9. 任务类总体职责

`任务类` 封装以下能力：

1. 创建或读取任务虚拟存在。
2. 在任务虚拟存在上记录任务特征。
3. 统一写入任务根、任务控制态、任务局部运行态对应的实例特征。
4. 提供任务终结、派发许可、基础状态标记和管理对象特征访问函数。
5. 初始化任务特征账本和使用统计特征。
6. 同步任务特征使用统计。
7. 读取任务特征账本生命周期。
8. 创建任务头节点。
9. 创建兼容任务步骤节点。
10. 创建兼容任务结果节点。

`任务类` 不负责：

- 方法召回和选择。
- 方法执行。
- 任务状态完整推进。
- 负载控制和工作线程派发。
- 根需求判断。
- 需求满足结算。

这些由任务管理模块、自我线程、任务管理界面线程和执行桥承担。

## 10. 任务虚拟存在

### 10.1 定位

任务虚拟存在是任务的世界树承载节点。

它的类型词是：

```text
任务虚拟存在
```

名称词当前固定为：

```text
任务虚拟存在
```

注意：

- 任务节点不是基础信息节点，不能直接挂特征。
- 任务需要特征时，应通过任务虚拟存在承载。
- 这符合“特征只能挂在存在或场景下”的建模规则。

### 10.2 `取或创建_任务虚拟存在(...)`

用途：

- 读取或创建任务头节点对应的任务虚拟存在。

执行流程：

1. 归一到任务头节点。
2. 如果任务头主信息已有任务虚拟存在指针，直接返回。
3. 推导任务场景。
4. 推导任务虚拟存在的承载世界。
5. 在承载世界下取或创建子存在：
   - 名称：`任务虚拟存在`。
   - 类型：`任务虚拟存在`。
6. 写回任务头主信息的 `任务虚拟存在` 引用。
7. 为任务虚拟存在取或创建内部世界。

### 10.3 任务场景推导

`私有_推导任务场景` 的顺序是：

1. 如果上级任务节点已有场景，使用上级任务场景。
2. 如果来源需求有需求场景，使用需求场景。
3. 如果默认世界尚未初始化，初始化默认世界。
4. 优先使用 `世界树.自我所在场景`。
5. 否则使用 `世界树.虚拟世界`。

### 10.4 承载世界推导

`私有_推导任务虚拟存在承载世界` 的顺序是：

1. 如果任务场景有宿主存在，使用宿主存在内部世界。
2. 否则使用自我存在内部世界。
3. 否则初始化并使用虚拟世界。

## 11. 任务虚拟存在特征

### 11.1 `记录任务虚拟存在特征(...)`

用途：

- 在任务虚拟存在上写入一个特征值。

支持值类型：

- `I64`：调用 `世界树.写入特征_I64`。
- `指针句柄`：调用 `世界树.写入特征_指针`。
- `VecU句柄`：取值池中的 `VecU` 后调用 `世界树.写入特征_VecU`。

返回：

- 写入成功返回 `true`。
- 特征类型为空、任务头无效、任务虚拟存在创建失败或值类型无法写入时返回 `false`。

### 11.2 任务实体特征写入接口

任务根、任务控制态、任务局部运行态的长期语义由任务虚拟存在实例特征承载。

当前统一入口：

```cpp
任务类::写入任务根实体特征(...)
任务类::写入任务控制实体特征(...)
任务类::写入任务局部运行实体特征(...)
```

用途：

- `写入任务根实体特征` 写入任务根 ID、来源需求、世界任务根、父任务、管理对象任务、授权等级和只读状态。
- `写入任务控制实体特征` 写入队列状态、优先级、预算、绑定工作线程、派发状态、等待子结果、收束 / 停止请求和控制响应状态。
- `写入任务局部运行实体特征` 写入局部运行状态、当前步骤前沿、最近局部结果和局部完成度。

边界：

- 工作线程可以构造运行包，但不直接决定这些语义写成哪些实例特征。
- 任务虚拟存在实例特征的写入规则集中在 `任务类`。

### 11.3 `初始化任务虚拟存在信息(...)`

用途：

- 确保任务虚拟存在存在，并初始化任务标准统计特征。

会初始化的特征：

- `特征账本生命周期`，默认值为 `候选`。
- 暴露次数。
- 读取次数。
- 有效使用次数。
- 历史使用次数。
- 近期使用次数。
- 作为决策输入次数。
- 作为派生父特征次数。
- 作为需求触发次数。
- 作为风险闸门次数。
- 作为验证信号次数。
- 作为解释依据次数。
- 结果改变次数。
- 坏结果次数。
- 下游依赖特征数。
- 最近使用时间。
- 最近有效使用时间。

注意：

- 初始化只在特征不存在时写默认值。
- 已有 I64 特征不会被默认值覆盖。

### 11.4 任务基础状态函数

当前统一入口：

```cpp

任务类::任务允许派发(...)
任务类::确保任务初始状态(...)
任务类::标记任务待重筹办(...)
任务类::标记任务等待(...)
任务类::标记任务已承接(...)
任务类::标记任务已派发(...)
任务类::标记任务负载等待(...)
任务类::应用任务状态(...)
```

用途：

- `任务状态已终结` 判断任务是否进入完成、失败、取消或超时。
- `任务允许派发` 判断任务是否可进入工作线程派发。
- `确保任务初始状态` 将未定义任务状态归一为未启动。
- `标记任务待重筹办` 将任务重新放回筹办入口，并清空完成时间。
- `标记任务等待` 将非终结任务推进到等待中。
- `标记任务已承接` 将未启动、就绪、待重筹办等任务推进到排队中。
- `标记任务已派发` 将任务推进到执行中，并补写启动时间。
- `标记任务负载等待` 将可等待调度的任务推进到排队中。
- `应用任务状态` 接收工作线程或执行桥推导出的新状态，并在完成时补写完成时间。

边界：

- 这些函数只维护任务自身状态。
- 它们不负责需求树生长、方法选择、参数配对、工作线程调度或本能函数执行。

### 11.5 `同步任务特征使用统计(...)`

用途：

- 将 `结构_特征使用统计` 写入任务虚拟存在。

写入内容：

- 各类使用次数。
- 结果改变次数。
- 坏结果次数。
- 下游依赖特征数。
- 最近使用时间。
- 最近有效使用时间。
- 特征账本生命周期。

### 11.6 `读取任务特征账本生命周期标准特征(...)`

用途：

- 读取任务虚拟存在上的 `特征账本生命周期`。

规则：

1. 先初始化任务虚拟存在信息。
2. 尝试读取 I64 生命周期值。
3. 读取成功返回当前值。
4. 失败返回默认值。

注意：

- 读取会触发初始化，因此不是完全无副作用操作。

## 12. 创建任务头节点

### 12.1 `创建任务头结点(...)`

签名：

```cpp
节点类* 创建任务头结点(
    需求节点类* 来源需求,
    节点类* 上级任务节点 = nullptr,
    时间戳 now = 结构体_时间戳::当前_微秒(),
    const std::string& 调用点 = "任务类::创建任务头结点");
```

用途：

- 为一个需求创建任务头节点。
- 把任务加入来源需求的任务列表。
- 初始化任务虚拟存在。

必要条件：

- `来源需求` 必须存在。

执行流程：

1. 如果 `now == 0`，取当前时间。
2. 如果传入上级任务节点，归一得到父任务头节点。
3. 推导任务场景。
4. 创建 `任务主信息类`。
5. 写入名称、类型、状态、创建时间、统计等基础字段。
6. 写入任务树类型和根任务标记。
7. 从需求安全/服务权重推导任务基准优先级。
8. 写入对应需求、来源需求、目标状态和结果状态信息。
9. 如果有父任务头，写入父任务引用。
10. 如果上级任务是步骤或结果，写入旧来源父节点引用。
11. 按上级节点决定挂接位置。
12. 写入父任务的直属子任务列表和当前子任务引用。
13. 写入上级步骤或结果的兼容子任务引用。
14. 将新任务加入来源需求的 `任务列表`。
15. 初始化任务虚拟存在信息。

### 12.2 默认名称和类型

默认任务名称：

- 来源需求类型是 `安全根需求`：`安全根任务`。
- 来源需求类型是 `服务根需求`：`服务根任务`。
- 其他情况下使用来源需求类型。
- 没有需求类型时使用 `需求任务`。

默认任务类型：

- 来源需求类型是 `安全根需求` 或 `服务根需求`：`根任务`。
- 其他情况下是 `任务`。

### 12.3 优先级推导

`私有_推导基准优先级` 的顺序是：

1. 如果来源需求服务权重非零，使用服务权重。
2. 否则如果来源需求安全权重非零，使用安全权重。
3. 否则如果有父任务头，继承父任务虚拟存在中的 `任务基准优先级`。
4. 否则为 `0`。

创建任务头时写入任务虚拟存在实例特征：

- `任务基准优先级 = 推导值`。
- `任务调度优先级 = 任务基准优先级`。

### 12.4 目标状态和结果状态

创建任务头时：

- `目标状态` 来自 `来源需求->主信息.需求状态`。
- `结果状态信息` 来自 `来源需求->主信息.被需求当前状态`。

这表达的是：

- 任务目标是把需求对象推进到目标状态。
- 任务当前结果状态先从需求当前状态镜像而来。

## 13. 创建任务步骤节点

### 13.1 定位

`创建任务步骤节点` 是兼容接口。当前仍被部分任务管理、缺口承接和自我线程路径使用，但新逻辑不应把旧步骤树重新作为任务组织主线。

### 13.2 执行流程

`创建任务步骤节点(...)` 的主要流程：

1. 归一到任务头节点。
2. 如果 `now == 0`，取当前时间。
3. 推导挂接父节点：
   - 优先使用调用方传入的上级任务节点。
   - 否则使用当前步骤的结果节点。
   - 否则使用最近实际结果节点。
   - 否则使用任务头节点。
4. 如果挂接父节点本身是步骤节点，先为它创建初始化预测结果节点，再把新步骤挂到该结果节点后面。
5. 只允许挂在任务头节点或结果节点下。
6. 推导场景、当前状态、目标状态和当前方法。
7. 从方法条件节点或方法首节点读取方法条件模板场景。
8. 推导步骤序号。
9. 创建步骤节点主信息。
10. 写入父任务、场景、执行方法、目标状态、结果状态信息。
11. 如果有当前方法，写入动作句柄和可用方法首节点。
12. 创建步骤节点。
13. 如果有当前方法，创建任务方法条件实例场景。
14. 维护前后跳转引用。
15. 将任务头的 `当前步骤节点` 指向新步骤。
16. 为新步骤创建初始化预测结果节点。
17. 如果初始化结果节点创建失败，删除步骤节点并返回空。

### 13.3 步骤初始状态

步骤节点状态：

- 有当前方法时：`就绪`。
- 没有当前方法时：`筹办中`。

### 13.4 当前状态推导

`私有_推导当前状态` 的顺序是：

1. 如果上级任务节点有 `结果状态信息`，使用它。
2. 否则如果任务头有 `结果状态信息`，使用它。
3. 否则如果任务头有来源需求，使用来源需求的 `被需求当前状态`。
4. 否则为空。

### 13.5 目标状态推导

`私有_推导步骤目标状态` 的顺序是：

1. 调用方显式传入目标状态。
2. 当前方法的第一个有效 `结果初始状态`。
3. 任务头的 `目标状态`。
4. 来源需求的 `需求状态`。
5. 否则为空。

注意：

- 当前方法的结果初始状态优先级高于任务头目标状态。
- 这是旧步骤/结果树路径中的推导规则，新主线筹办时应确认是否符合当前需求目标语义。

## 14. 创建任务结果节点

### 14.1 定位

`创建任务结果节点` 也是兼容接口。任务结果的新主线应优先写 `结果状态信息` 和执行记录。

### 14.2 执行流程

`创建任务结果节点(...)` 的主要流程：

1. 归一到任务头节点。
2. 要求来源步骤节点存在，且节点种类是 `步骤节点`。
3. 如果 `now == 0`，取当前时间。
4. 推导任务场景。
5. 推导当前状态。
6. 调用内部 `私有_创建结果节点`。

### 14.3 内部结果节点创建

内部 `私有_创建结果节点` 会：

1. 检查所属任务头和步骤节点。
2. 如果要创建预测结果，优先复用同方法的已有预测结果。
3. 推导实际结果状态：有结果状态则使用结果状态，否则使用当前状态。
4. 查找方法结果模板节点和方法结果模板场景。
5. 创建结果节点主信息。
6. 写入来源需求、父任务、场景、执行方法、目标状态、结果状态信息。
7. 写入来源步骤、来源方法、对应方法结果节点。
8. 如果是实际结果，绑定命中的预测结果节点。
9. 创建结果节点。
10. 如果有当前方法，创建任务方法结果实例场景。
11. 创建任务结果方向二次特征。
12. 将步骤节点的 `后续任务` 指向结果节点。
13. 如果是实际结果，将任务头的 `最近实际结果节点` 指向该结果节点。

### 14.4 初始化预测结果节点

创建步骤节点时会自动创建一个预测结果节点。

预测结果状态：

- 优先使用步骤目标状态。
- 没有步骤目标状态时使用当前状态。

用途：

- 旧路径中用于表示执行前预期结果。
- 实际结果节点创建时会尝试绑定命中的预测结果节点。

## 15. 任务方法实例场景

任务步骤和结果节点会根据方法模板创建任务实例场景。

### 15.1 条件实例场景

创建步骤节点时，如果有当前方法：

- 从方法条件节点或方法首节点读取条件场景模板。
- 在任务虚拟存在内部世界下创建名为 `方法条件` 的内部世界场景。
- 将模板场景中的关系索引和二次特征索引复制到实例场景。
- 写入步骤节点的 `方法条件节点` 引用。

### 15.2 结果实例场景

创建结果节点时，如果有当前方法：

- 从方法结果节点或方法首节点读取结果场景模板。
- 在任务虚拟存在内部世界下创建名为 `方法结果` 的内部世界场景。
- 将模板场景中的关系索引和二次特征索引复制到实例场景。
- 写入结果节点的 `方法结果节点` 引用。

注意：

- 这些场景是世界树场景节点。
- 它们可以承载二次特征和关系。
- 任务节点自身仍不是特征宿主。

## 16. 结果方向二次特征

创建任务结果节点时，会调用：

```cpp
世界树.二次特征生成().创建任务结果方向二次特征(...)
```

输入包括：

- 场景。
- 当前状态。
- 结果状态。
- 任务虚拟存在作为宿主基础信息节点。

生成结果会写到结果节点的：

```cpp
结果方向二次特征
```

这是旧结果节点路径中的方向证据。新主线如果需要结果方向，应优先确认是否和需求目标方向一致，再决定是否沉淀到方法或因果证据中。

## 17. 与其它模块的关系

### 17.1 与 `需求类`

任务头节点来自需求：

- `对应需求` 指向需求。
- `来源需求` 是兼容镜像。
- 任务会加入需求主信息的 `任务列表`。
- 目标状态来自需求状态。
- 初始结果状态来自被需求当前状态。

因此需求树是任务归属的权威结构。

### 17.2 与 `方法类`

任务通过方法引用推进：

- `候选方法首节点` 表示候选方法。
- `执行方法` 表示当前选定或最近执行的方法。
- 旧字段 `可用方法首节点` 仍存在，但注释说明新逻辑使用候选方法首节点。
- 步骤兼容路径会使用方法条件节点、方法结果节点来创建任务实例场景。

### 17.3 与 `世界树类`

任务类通过世界树：

- 创建任务虚拟存在。
- 创建任务虚拟存在内部世界。
- 写入任务虚拟存在特征。
- 创建任务条件/结果实例场景。
- 创建任务结果方向二次特征。

### 17.4 与任务管理线程

任务类不负责完整状态推进。

任务管理线程负责：

- 梳理任务列表。
- 根据负载控制选择可推进任务。
- 将任务放入工作线程。
- 接收工作线程消息。
- 改变任务状态。
- 在必要时把任务切回待重筹办、等待、挂起、完成等状态。

### 17.5 与工作线程

工作线程负责单步执行。

执行中会：

- 读取任务、方法、基础数据。
- 写入必要的虚拟存在和动态事实。
- 返回任务执行情况消息。

任务类本身不直接执行方法。

### 17.6 与任务模块实体

`任务模块.实体.ixx` 定义了队列态和局部运行态：

- `枚举_任务队列状态`
- `枚举_任务局部运行状态`
- `结构_任务根节点`
- `结构_任务控制态`
- `结构_任务局部运行态`

这些是任务管理和工作线程层的数据结构，不是 `任务主信息类` 本身的字段。

## 18. 使用示例

### 18.1 为需求创建任务头

```cpp
任务类 任务服务{};
auto* 任务头 = 任务服务.创建任务头结点(需求节点);
```

效果：

- 创建任务头节点。
- 从需求拷贝目标状态和当前状态。
- 写入需求任务列表。
- 初始化任务虚拟存在。

### 18.2 记录任务虚拟存在特征

```cpp
任务类::记录任务虚拟存在特征(
    任务头,
    语素集.添加词性词("当前等待原因", "名词"),
    特征值{ 指针句柄{} });
```

效果：

- 归一到任务头节点。
- 确保任务虚拟存在。
- 在任务虚拟存在上写入特征。

### 18.3 创建兼容步骤节点

```cpp
auto* 步骤 = 任务服务.创建任务步骤节点(
    任务头,
    nullptr,
    方法首节点,
    目标状态);
```

效果：

- 创建一个旧步骤节点。
- 绑定执行方法。
- 创建方法条件实例场景。
- 创建初始化预测结果节点。

### 18.4 创建兼容结果节点

```cpp
auto* 结果 = 任务服务.创建任务结果节点(
    任务头,
    步骤,
    方法首节点,
    枚举_任务结果角色::实际结果,
    结果状态);
```

效果：

- 创建一个旧结果节点。
- 绑定来源步骤和来源方法。
- 创建方法结果实例场景。
- 创建结果方向二次特征。
- 更新任务头最近实际结果节点。

## 19. 重要注意事项

- 任务节点不是基础信息节点，不能直接挂特征。
- 任务自身特征必须通过任务虚拟存在表达。
- 当前主设计是需求树组织任务，不是任务树组织需求。
- 任务头节点是当前主线；步骤节点和结果节点是兼容接口。
- `任务类` 创建节点，但不负责完整任务状态机推进。
- 任务管理界面线程应根据消息和负载控制推进任务，而不是由任务类主动调度。
- 创建任务头会把任务写入需求的任务列表。
- 创建任务头会初始化任务虚拟存在，因此创建任务不是纯内存操作。
- 读取任务特征账本生命周期会触发任务虚拟存在初始化。
- 旧字段 `置信度_Q10000` 是兼容占位，新算法不应继续基于 Q10000 扩展。
- `可用方法首节点` 是旧字段，新逻辑应优先使用 `候选方法首节点`。
- 任务结果的新主线应落到结果状态和执行记录，旧结果节点只作为兼容结构使用。

## 20. 字段瘦身迁移记录

### 20.1 管理对象任务

`管理对象任务` 已不再是 `任务主信息类` 字段。

当前读写入口为：

```cpp
任务类::读取任务管理对象任务(...)
任务类::读取任务管理对象或自身(...)
任务类::任务管理对象指向(...)
任务类::写入任务管理对象任务(...)
```

语义落点为任务虚拟存在中的 `任务管理对象` 实例特征。

### 20.2 执行方法

外部模块不再直接读写 `主信息.执行方法`。

当前读写入口为：

```cpp
任务类::读取任务执行方法(...)
任务类::任务执行方法指向(...)
任务类::写入任务执行方法(...)
```

任务头节点的执行方法引用会写入任务虚拟存在中的 `任务执行方法` 实例特征。

需要注意：

- `执行方法` 不是任务长期权威选择。
- 权威选择是 `选中自我能力值`。
- `执行方法` 只是运行期方法引用。
- 旧步骤节点和旧结果节点仍暂时使用内部缓存保存每步来源方法事实。
- 删除该内部缓存前，必须先删除旧步骤 / 结果任务树，或把每步来源方法迁入运行虚拟存在事实链。

### 20.3 等待缺口唤醒

`等待缺口唤醒` 已不再是 `任务主信息类` 字段。

当前读写入口为：

```cpp
任务类::读取任务等待缺口唤醒(...)
任务类::写入任务等待缺口唤醒(...)
```

语义落点为任务虚拟存在中的 `任务等待缺口唤醒` 实例特征。

### 20.4 选中自我能力值

`选中自我能力值` 已不再是 `任务主信息类` 字段。

当前读写入口为：

```cpp
任务类::读取任务选中自我能力值(...)
任务类::写入任务选中自我能力值(...)
```

语义落点为任务虚拟存在中的 `任务选中自我能力` 实例特征。

任务可以拥有多个候选方法和多次方法尝试；`选中自我能力值` 只表达当前筹办结果选中的能力主键，不替代方法候选集合和运行事实链。

### 20.5 任务角色

`任务角色` 已不再是 `任务主信息类` 字段。

当前读写入口为：

```cpp
任务类::读取任务角色(...)
任务类::写入任务角色(...)
```

语义落点为任务虚拟存在中的 `任务角色` 实例特征。

任务角色只表达当前任务在需求树上的职责分类，不负责方法选择，也不重新引入旧任务管理任务中心模型。

### 20.6 当前阶段

`当前阶段` 已不再是 `任务主信息类` 字段。

当前读写入口为：

```cpp
任务类::读取任务当前阶段(...)
任务类::写入任务当前阶段(...)
```

语义落点为任务虚拟存在中的 `任务当前阶段` 实例特征。

读取阶段时优先使用任务虚拟存在特征；若尚未写入，则按任务状态推导出筹办、执行、阻塞或已完成阶段。
