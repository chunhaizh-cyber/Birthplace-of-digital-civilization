# 场景类说明

本文说明 `D:\鱼巢\场景类.h`、`D:\鱼巢\场景类.cpp`，并补充 `D:\鱼巢\基础信息主信息类.h` 中 `场景节点主信息类` 与本类直接相关的字段。

`场景类` 是基础信息树中“场景节点”的服务层。它负责创建、查找、删除场景节点，枚举场景下的子场景、存在和特征，并维护场景对状态、动态、二次特征和因果实例的索引引用。

## 1. 文件定位

当前项目中，“场景”是对象和事件的发生环境：

- 场景节点可以挂在世界根、存在、场景等基础信息节点下面。
- 场景可以直接承载存在节点。
- 场景可以直接承载特征节点；这符合“特征只能挂在存在或场景下面”的规则。
- 场景主信息不嵌入状态、动态、二次特征和因果实例的大对象副本，只保存它们的索引引用。
- 内部世界也是场景节点，只是通过 `宿主存在` 指向所属存在。

简单说：

- `基础信息类` 提供树结构。
- `场景节点主信息类` 保存场景的语义、世界类型和索引。
- `场景类` 提供围绕场景节点的创建、查找、绑定和索引维护。

## 2. 世界类型

`枚举_世界类型` 定义在 `基础数据类型.h`：

| 枚举值 | 含义 |
| --- | --- |
| `现实世界` | 现实环境。 |
| `文本世界` | 文本或语言输入环境。 |
| `想象世界` | 想象、模拟或构造环境。 |
| `记忆世界` | 记忆环境。 |
| `推理世界` | 推理过程环境。 |
| `内部世界` | 某个存在的内部世界。 |
| `虚拟世界` | 虚拟承载环境。 |
| `其它` | 未归类场景。 |

`世界树类::初始化默认世界` 会创建六个默认世界：

- 现实世界
- 文本世界
- 想象世界
- 记忆世界
- 推理世界
- 虚拟世界

## 3. 场景节点主信息

`场景节点主信息类` 定义在 `基础信息主信息类.h`。

### 3.1 字段

| 字段 | 用途 |
| --- | --- |
| `宿主存在` | 如果该场景是某个存在的内部世界，则保存宿主存在引用。 |
| `世界类型` | 场景类型，默认是 `现实世界`。 |
| `来源文章` | 场景来源文章引用。 |
| `状态索引` | 场景内状态节点的引用索引。 |
| `动态索引` | 场景内动态节点的引用索引。 |
| `关系索引` | 场景内关系型二次特征引用索引。 |
| `二次特征索引` | 场景内二次特征引用索引。 |
| `实例因果索引` | 场景内因果实例引用索引。 |

注意：

- 这些索引都是 `可解析引用`，保存指针和主键。
- 场景主信息只保存索引，不拥有这些节点的完整生命周期。
- 二次特征会同时进入 `关系索引` 和 `二次特征索引`。

### 3.2 构造与判断函数

| 函数 | 用途 |
| --- | --- |
| `场景节点主信息类()` | 默认构造，主信息类型为 `场景`，世界类型默认为 `现实世界`。 |
| `场景节点主信息类(const 词性节点类* 名, 枚举_世界类型 t)` | 按名称和世界类型构造场景主信息。 |
| `是否内部世界() const noexcept` | `宿主存在` 有效时返回 `true`。 |
| `是否现实世界() const noexcept` | `世界类型 == 现实世界` 时返回 `true`。 |

### 3.3 `比较`

`场景节点主信息类::比较` 支持按字段比较：

| 比较字段 | 行为 |
| --- | --- |
| `基础信息基类_名称` / `名称` | 比较左右名称指针是否相同。 |
| `基础信息基类_类型` / `类型` | 比较左右类型指针是否相同。 |
| 其他字段 | 返回 `-1`。 |

注意：

- 这里比较的是指针。
- `场景类::查找子场景_按名称` 使用 `私有_词性相同`，会兼容词性主键相同的情况。

## 4. `场景类` 的职责

`场景类` 主要做五件事：

1. 绑定基础信息树。
2. 创建、查找、删除场景节点。
3. 枚举场景下面的子场景、子存在和子特征。
4. 绑定内部世界宿主和来源文章。
5. 维护状态、动态、二次特征、实例因果的索引引用。

私有状态：

| 字段 | 用途 |
| --- | --- |
| `基础信息_` | 指向基础信息树。为空时大多数操作返回空或失败。 |

## 5. 构造与绑定

### 5.1 `场景类(基础信息类* 基础信息 = nullptr) noexcept`

用途：

- 创建场景服务对象，并可选绑定基础信息树。

用法：

```cpp
场景类 场景服务(&基础信息);
```

注意：

- 只保存指针，不拥有基础信息树生命周期。
- 构造时传 `nullptr` 时，后续必须调用 `绑定基础信息`。

### 5.2 `绑定基础信息(基础信息类* 基础信息) noexcept`

用途：

- 设置或替换场景服务使用的基础信息树。

用法：

```cpp
场景服务.绑定基础信息(&世界树.基础信息());
```

注意：

- 不迁移已有节点。
- 调用方必须保证基础信息树在使用期间有效。

## 6. 主信息读取

### 6.1 `取场景主信息(场景节点类* 节点) const noexcept`

用途：

- 从基础信息节点中取出可写的 `场景节点主信息类`。

返回：

- 成功返回 `场景节点主信息类*`。
- `基础信息_` 为空、节点为空、或节点主信息不是场景时返回 `nullptr`。

### 6.2 `取场景主信息(const 场景节点类* 节点) const noexcept`

用途：

- 只读版本，用于读取场景主信息。

返回规则与可写版本相同。

## 7. 创建与删除场景

### 7.1 `创建场景(基础信息节点类* 父节点, 场景节点主信息类* 主信息)`

用途：

- 使用调用者准备好的场景主信息创建场景节点，并挂到基础信息树的 `父节点` 下面。

行为：

- `基础信息_` 为空或 `主信息` 为空时返回 `nullptr`。
- 调用 `基础信息_->添加子节点(父节点, 主信息)` 创建节点。
- 创建后调用 `私有_确保场景默认特征已初始化`。

用法：

```cpp
auto* 主信息 = new 场景节点主信息类(场景名称词, 枚举_世界类型::现实世界);
auto* 场景节点 = 场景服务.创建场景(父节点, 主信息);
```

注意：

- 不检查同一父节点下是否已有同名场景。
- 创建成功后，主信息生命周期交给基础信息树。

### 7.2 `创建场景_按名称(基础信息节点类* 父节点, const 词性节点类* 名称, 枚举_世界类型 世界类型 = 其它)`

用途：

- 按名称和世界类型快速创建场景。

行为：

- 内部创建新的 `场景节点主信息类`。
- 设置 `名称` 和 `世界类型`。
- 转调 `创建场景`。

用法：

```cpp
auto* 场景 = 场景服务.创建场景_按名称(
    现实世界,
    场景名称词,
    枚举_世界类型::现实世界);
```

注意：

- 名称允许为空，但多数业务不应创建无名场景。
- 该函数不去重。

### 7.3 `取或创建子场景_按名称(...)`

签名：

```cpp
场景节点类* 取或创建子场景_按名称(
    基础信息节点类* 父节点,
    const 词性节点类* 名称,
    枚举_世界类型 世界类型 = 枚举_世界类型::其它);
```

用途：

- 在父节点直接子场景中按名称查找；找不到时创建。

行为：

- 先调用 `查找子场景_按名称`。
- 命中时，把命中场景主信息的 `世界类型` 更新为传入值，并返回该场景。
- 未命中时调用 `创建场景_按名称`。

用法：

```cpp
auto* 内部世界 = 场景服务.取或创建子场景_按名称(
    宿主存在,
    内部世界名称词,
    枚举_世界类型::内部世界);
```

注意：

- 只按名称查找，不按世界类型区分。
- 如果同一父节点下存在同名但不同世界类型的场景，会复用第一个命中场景并覆盖其 `世界类型`。
- `名称` 为空时，`查找子场景_按名称` 会返回 `nullptr`，因此会创建一个无名场景。

### 7.4 `删除场景(场景节点类* 节点)`

用途：

- 删除一个场景节点。

行为：

- `基础信息_` 为空时返回 `false`。
- 否则转调 `基础信息_->删除节点(节点)`。

注意：

- 删除行为由基础信息树负责。
- 如果其他对象保存了该场景、场景子节点或场景索引中节点的指针，调用方需要处理失效问题。

## 8. 枚举与查找

### 8.1 `枚举全部场景() const`

用途：

- 从基础信息树全链枚举所有场景节点。

返回：

- `基础信息_` 为空时返回空数组。
- 否则返回所有主信息为 `场景节点主信息类` 的节点。

### 8.2 `获取子场景(const 基础信息节点类* 父节点) const`

用途：

- 获取父节点下面的直接场景子节点。

行为：

- 只枚举直接子节点，不递归。
- `父节点` 为空时，底层会使用基础信息世界根。

### 8.3 `获取子存在(const 基础信息节点类* 父节点) const`

用途：

- 获取父节点下面的直接存在子节点。

典型用途：

- 获取某个场景中的存在。
- 获取某个存在内部世界中的子存在。

注意：

- 只枚举直接子存在。
- 不会读取或比较存在主信息内容。

### 8.4 `获取子特征(const 基础信息节点类* 父节点) const`

用途：

- 获取父节点下面的直接特征子节点。

规则：

- 合法直接父节点应为存在或场景。
- 对场景而言，这是读取场景级特征的入口。
- 任务、方法、线程不能直接作为特征父节点。

### 8.5 `查找子场景_按名称(const 基础信息节点类* 父节点, const 词性节点类* 名称) const`

用途：

- 在父节点直接子场景中按名称查找。

行为：

- `名称` 为空时返回 `nullptr`。
- 名称比较使用 `私有_词性相同`。
- 返回第一个命中的场景。

## 9. 绑定关系

### 9.1 `绑定宿主(场景节点类* 节点, 存在节点类* 宿主存在)`

用途：

- 把场景标记为某个存在的内部世界。

行为：

- 读取场景主信息。
- 主信息存在时，把 `宿主存在` 写入 `主信息->宿主存在`。
- 保存的是 `可解析引用<存在节点类>`，包含指针和主键。

用法：

```cpp
场景服务.绑定宿主(内部世界场景, 宿主存在);
```

注意：

- 该函数只写场景主信息。
- 完整的双向绑定通常由 `世界树类::取或创建存在内部世界` 完成，它还会把内部世界引用写回存在主信息。

### 9.2 `绑定来源文章(场景节点类* 节点, 文章节点类* 来源文章)`

用途：

- 记录场景来源文章。

行为：

- 主信息存在时，把来源文章写入 `主信息->来源文章`。
- 保存指针和主键。

## 10. 索引追加

场景主信息中的状态、动态、二次特征、因果实例都是索引引用。追加函数会防止同指针或同主键重复进入索引。

### 10.1 `追加状态(场景节点类* 节点, 状态节点类* 状态节点)`

用途：

- 把状态节点加入场景的 `状态索引`。

行为：

1. 主信息或状态节点为空时返回 `false`。
2. 如果 `状态索引` 中没有同指针或同主键状态，则追加引用。
3. 对 `状态索引` 做稳定排序。

排序规则：

- 优先按状态主信息的排序时间升序。
- 排序时间为 `发生时间 != 0 ? 发生时间 : 收到时间`。
- 时间相同时按主键字符串升序。

注意：

- 场景主信息只保存引用，不拥有状态节点。
- 如果状态节点主信息缺失，排序时间按 `0`。

### 10.2 `追加动态(场景节点类* 节点, 动态节点类* 动态节点)`

用途：

- 把动态节点加入场景的 `动态索引`。

行为：

- 主信息或动态节点为空时返回 `false`。
- 没有重复引用时追加。
- 不做排序。

### 10.3 `追加关系(场景节点类* 节点, 二次特征节点类* 二次特征节点)`

用途：

- 兼容关系追加入口。

行为：

- 当前实现直接转调 `追加二次特征`。

### 10.4 `追加二次特征(场景节点类* 节点, 二次特征节点类* 二次特征节点)`

用途：

- 把二次特征加入场景索引。

行为：

- 主信息或二次特征节点为空时返回 `false`。
- 内部调用 `私有_追加二次特征引用`。
- 同一个二次特征会分别去重后写入 `关系索引` 和 `二次特征索引`。

注意：

- 当前“关系”和“二次特征”共用同一个追加逻辑。

### 10.5 `追加实例因果(场景节点类* 节点, 因果实例节点类* 因果实例节点)`

用途：

- 把因果实例加入场景的 `实例因果索引`。

行为：

- 主信息或因果实例为空时返回 `false`。
- 没有重复引用时追加。

## 11. 默认场景特征

`场景类::创建场景` 创建场景后，会调用 `私有_确保场景默认特征已初始化`。

当前默认特征逻辑只针对自我所在场景：

命中条件：

- 创建出的场景指针等于 `世界树.自我所在场景`；或
- 场景主信息存在，世界类型为 `现实世界`，名称等于 `自我初始场景`。

补齐内容：

- 目标特征类型：`场景安全度`
- 兼容旧特征类型：`安全度`
- 默认值：`I64_MAX / 2`

流程：

1. 查找场景下是否已有 `场景安全度` 特征。
2. 如果没有，再查找旧的 `安全度` 特征。
3. 如果仍然没有，创建新的特征节点，类型为 `场景安全度`，当前值为 `I64_MAX / 2`。
4. 如果找到旧特征或新特征，统一把类型修正为 `场景安全度`。
5. 如果该特征没有当前值，则写入默认值。

注意：

- 这里的特征直接挂在场景下面，符合特征宿主规则。
- 该默认特征写入直接修改主信息，不经过 `特征类::写入特征值_按参数`，因此不维护统计和稳态。
- 该逻辑依赖全局 `世界树` 和 `语素集`。

## 12. 私有辅助函数

这些函数位于 `场景类.cpp` 的匿名命名空间，只供本文件内部使用。

| 函数 | 用途 |
| --- | --- |
| `私有_词性相同` | 比较两个词性节点，先比指针，再比主键。 |
| `私有_生成引用` | 由节点生成 `可解析引用`，保存指针和主键。 |
| `私有_引用已存在` | 判断引用列表中是否已有同指针或同主键节点。 |

| `私有_查找子特征_按类型` | 在指定宿主的直接子特征中按类型查找。 |
| `私有_确保场景默认特征已初始化` | 给自我初始场景补齐 `场景安全度` 特征。 |

## 13. 常见使用路径

### 13.1 取得默认世界

推荐使用世界树封装：

```cpp
世界树.初始化默认世界();
auto* 现实世界 = 世界树.现实世界;
```

### 13.2 创建或取得子场景

```cpp
auto* 场景 = 场景服务.取或创建子场景_按名称(
    父节点,
    场景名称词,
    枚举_世界类型::现实世界);
```

### 13.3 创建存在的内部世界

推荐使用世界树封装：

```cpp
auto* 内部世界 = 世界树.取或创建存在内部世界(宿主存在);
```

该封装会同时完成：

- 在宿主存在下面创建或复用内部世界场景。
- 调用 `场景类::绑定宿主`。
- 调用 `存在类::绑定内部世界`。

### 13.4 向场景追加状态

```cpp
场景服务.追加状态(场景节点, 状态节点);
```

追加后，`状态索引` 会按发生时间或收到时间排序。

### 13.5 向场景追加二次特征

```cpp
场景服务.追加二次特征(场景节点, 二次特征节点);
```

该节点会同时进入 `关系索引` 和 `二次特征索引`。

## 14. 与世界树的关系

项目中普通业务代码常通过 `世界树类` 使用场景服务：

- `世界树类::初始化默认世界` 创建默认世界场景。
- `世界树类::创建场景` 转调 `场景类::创建场景`。
- `世界树类::取或创建子场景_按名称` 转调 `场景类::取或创建子场景_按名称`。
- `世界树类::取或创建自我现实场景` 在现实世界下取得 `自我初始场景`。
- `世界树类::取或创建存在内部世界` 处理内部世界双向绑定。
- `世界树类::写入存在观测位置` 写入存在位置后，会刷新场景存在型基础二次特征。

因此：

- 单纯创建、查找、索引追加可以用 `场景类`。
- 涉及默认世界、自我场景、内部世界或二次特征刷新时，优先使用 `世界树类` 封装。

## 15. 注意事项

- `场景类` 不拥有基础信息树，只保存裸指针。
- `创建场景` 和 `创建场景_按名称` 不做去重。
- `取或创建子场景_按名称` 只按名称查找，命中后会覆盖世界类型。
- 场景主信息里的状态、动态、二次特征、因果实例都是索引引用，不是对象副本。
- `追加状态` 会排序，其他追加函数不排序。
- `追加关系` 当前等同于 `追加二次特征`。
- 场景可以直接承载特征；任务、方法、线程不能直接承载特征。
- 自我初始场景会自动补 `场景安全度` 特征，默认值为 `I64_MAX / 2`。
- 默认场景特征补齐不走特征统计和稳态逻辑。
