# 需求类说明

本文说明 `D:\鱼巢\需求主信息类.h` 与 `D:\鱼巢\需求类.h` 当前口径。

## 1. 需求节点职责

需求节点只表达目标关系和运行账本，不保存目标值类型，也不解释集合、区间、枚举或二次特征。

单个执行需求节点只表达一个原子目标：

```text
目标主体
目标特征类型
当前状态
目标状态
满足关系
```

多目标需求必须展开为管理需求的直接子链。父管理需求只负责组织、等待和汇总子需求，不作为单次方法查找的目标包来源。

## 2. 目标关系

`需求主信息类` 保留以下目标字段：

| 字段 | 说明 |
| --- | --- |
| `被需求存在` | 目标主体。 |
| `被需求当前状态` | 当前状态引用。 |
| `需求状态` | 目标状态引用。 |
| `满足关系` | 当前状态与目标状态满足哪些三向关系时视为满足。 |
| `目标特征类型缓存` | 从状态特征解析出的缓存，只用于查重和显示，不是权威语义源。 |

三向关系只包含：

```text
小于
等于
大于
```

保持就是等于，改变就是小于或大于。存在、可用、完整、已形成都是状态或特征值，不是方向。

## 3. 子链展开

例如方法补齐包含多个目标时，应表达为：

```text
方法补齐管理需求
├── 候选方法存在状态需求
├── 方法动作骨架状态需求
├── 方法条件结果配对状态需求
└── 方法运行账完整性需求
```

不要在 `需求主信息类` 中持久化目标包。目标包最多是运行时视图，由 `需求类::收集直接子需求目标视图(...)` 临时从直接子链收集。

## 4. 子链去重

新增子需求时，需求树按同父直接子链做目标重叠判断：

```text
目标主体相同或兼容
目标特征类型相同
满足关系掩码有交集
```

命中已有子需求时复用并更新归因，不再新增节点。派生归因只记录来源、自检归因、控制面板观察和重复触发统计，不决定需求身份。

## 5. 满足判断

需求满足判断不比较状态指针是否相同，而是调用特征侧状态比较：

```text
特征类::比较状态(当前状态, 目标状态)
    -> 小于 / 等于 / 大于
    -> 三向关系被接受(...)
```

如果状态不可比较，则执行需求不能判定为已满足。管理需求是否满足由直接必需子需求汇总，不伪造成单目标满足。
