# 特征类说明

本文说明 `D:\鱼巢\特征类.h` 和 `D:\鱼巢\特征类.cpp` 中的结构体和函数。

`特征类` 是世界树里的“特征系统服务层”。它不单独持有一棵树，而是绑定一个 `基础信息类`，在基础信息树中管理两类节点：

- `抽象特征主信息类`：挂在世界树根下的抽象特征链，用来定义特征维度、状态区间、组合规则和子抽象特征结构。
- `特征节点主信息类`：挂在存在、场景或虚拟存在下的实例特征，用来记录某个宿主的当前值、近帧候选、稳态列表和当前命中的抽象特征。

当前权威规则见 `D:\鱼巢\规范\特征类规范20260429\README.md`。本说明书描述代码入口和使用方式。

## 1. 文件定位

当前项目中，“特征”相关能力可以分成四层：

- `特征值`：一个轻量值，可能是 `I64`、`VecU句柄`、`指针句柄` 或无值。
- `抽象特征主信息类`：特征定义层，保存区间、组合规则、生命周期和重叠统计，不保存当前值。
- `特征节点主信息类`：实例层，保存抽象特征引用、当前值、当前命中抽象特征、局部区间覆盖、候选窗口和稳态列表。
- `特征类`：围绕基础信息树提供抽象特征、实例特征、写入、命中、升格、组合求值、比较和签名能力。

简单说：

- 抽象特征是定义，不是某个存在的当前状态，挂在世界树根下的抽象特征链。
- 实例特征是某个宿主拥有的特征，挂在存在、场景或虚拟存在下面。
- 任务、方法、线程等对象需要特征时，仍通过对应虚拟存在承载实例特征。
- 实例特征的 `主信息->抽象特征` 指向它实例化自哪个抽象特征。
- 实例特征的 `主信息->当前值` 表示“这个宿主现在的特征值”。
- 实例特征的 `主信息->当前命中抽象特征` 表示当前值落入哪个抽象状态区间。
- 稳态列表属于实例特征，用于从连续观测中提炼稳定值，再升格为抽象特征区间。

## 2. 相关数据结构

### 2.1 `特征比较明细`

用于记录两个存在在某一个特征类型上的比较结果。

| 字段 | 默认值 | 用途 |
| --- | --- | --- |
| `类型` | `nullptr` | 被比较的特征类型词。 |
| `差异度` | `0` | 当前特征类型上的差异值。数值越大表示越不同。 |
| `权重` | `10000` | 当前特征比较的权重。当前实现固定使用默认值。 |
| `缺失A` | `false` | 左侧集合缺少该特征。 |
| `缺失B` | `false` | 右侧集合缺少该特征。 |

注意：

- 当前 `比较特征集合` 不从特征节点读取自定义权重，所有明细默认权重都是 `10000`。
- 缺失一方时，差异度使用 `集合比较参数::缺失惩罚`。

### 2.2 `存在比较结果`

用于承载两个特征集合的整体比较结果。

| 字段 | 用途 |
| --- | --- |
| `总差异度` | 加权差异汇总后除以 `10000` 的结果。 |
| `归一化差异度` | 如果启用归一化，则为加权差异和除以权重和。 |
| `命中特征数` | 两边都有该特征且差异度为 `0` 的特征数量。 |
| `明细` | 每个特征类型的比较明细。 |

### 2.3 `稳态策略`

控制 `写入特征值_按参数` 如何从连续观测中判定稳态。

| 字段 | 默认值 | 用途 |
| --- | --- | --- |
| `连续N` | `5` | 至少连续多少帧接近，才认为通过候选门。实现中最低按 `2` 处理。 |
| `阈值_判稳` | `0` | 当前值和近帧历史值的差异度不能超过该阈值。 |
| `阈值_命中原型` | `0` | 当前值和已有稳态原型的差异度不能超过该阈值，才算命中已有稳态。 |
| `Kmax_points` | `32` | 稳态列表最大数量。超过后淘汰命中次数少、最后命中更早的稳态。 |
| `淘汰_超时` | `0` | 稳态长期未命中的超时淘汰阈值。`0` 表示不按时间淘汰。 |

### 2.4 `写入参数`

控制一次特征值写入的行为。

| 字段 | 默认值 | 用途 |
| --- | --- | --- |
| `ts` | `0` | 写入时间戳。为 `0` 时使用当前微秒时间。 |
| `仅更新当前值` | `false` | 为 `true` 时只写当前值、统计和近帧窗口，不进入稳态判定。 |
| `允许创建稳态` | `true` | 稳定但没有命中已有原型时，是否创建新稳态。 |
| `允许淘汰` | `true` | 是否允许按超时策略淘汰旧稳态。 |
| `稳态` | 默认 `稳态策略` | 本次写入使用的稳态策略。 |

### 2.5 `特征写入结果`

返回一次写入后的关键状态。

| 字段 | 含义 |
| --- | --- |
| `特征` | 被写入的特征节点。传入节点会先写入该字段，即使后续因主信息或值无效而提前返回。 |
| `通过候选门` | 近帧连续值是否满足稳态候选条件。 |
| `稳态发生变化` | 当前稳态索引是否从旧值切换到新的有效稳态。 |

### 2.6 `集合比较参数`

控制 `比较特征集合` 的输出和缺失处理。

| 字段 | 默认值 | 用途 |
| --- | --- | --- |
| `输出明细` | `true` | 是否把每个特征类型的比较明细写入结果。 |
| `缺失惩罚` | `10000` | 一侧缺失特征或值不可用时使用的差异度。 |
| `归一化` | `true` | 是否输出归一化差异度。 |

### 2.7 `特征比较方法结果`

表示两个特征值之间可以用什么方式比较，以及比较后的方向性结果。

| 字段 | 用途 |
| --- | --- |
| `可比较` | 两个值是否有效且类型一致。 |
| `比较方式` | 使用 `有序标量`、`相等` 或 `相等或相似度`。 |
| `左比较值` | 归一化后的左侧比较值。 |
| `右比较值` | 归一化后的右侧比较值。 |
| `差值` | 通常表示 `右 - 左` 的方向差，或相等/相似度差异。 |

### 2.8 `抽象特征主信息类`

抽象特征主信息用于描述“特征定义”和“抽象状态”，不保存当前值。

| 字段 | 用途 |
| --- | --- |
| `区间` | 可选 `I64区间`。为空表示抽象特征壳或待完善维度；有效区间表示一个抽象状态范围。 |
| `组合规则` | 多个抽象特征构成当前抽象特征时使用的来源引用和规则集合。 |
| `生命周期` | 当前抽象特征的候选、确认、弃用等生命周期状态。 |
| `重叠命中次数` | 当前值同时命中多个抽象区间时的统计。 |
| `最近重叠命中时间` | 最近一次重叠命中的时间。 |
| `来源实例特征` | 当前抽象区间由实例稳态升格时，记录来源实例特征。 |
| `升格时间` | 由实例稳态升格为抽象区间的时间。 |

注意：

- 抽象特征没有 `当前值`。
- 抽象特征不保存稳态列表。
- 同级抽象特征区间允许重叠，重叠是后续拆分或修正的证据。

### 2.9 `特征节点主信息类` 的新增实例字段

实例特征继续使用 `特征节点主信息类`，但现在它承担“抽象特征实例”的角色。

| 字段 | 用途 |
| --- | --- |
| `抽象特征` | 指向该实例特征实例化自哪个抽象特征。 |
| `当前命中抽象特征` | 当前值命中的抽象状态节点。 |
| `当前值` | 宿主当前实际特征值。 |
| `局部区间覆盖` | 宿主局部覆盖区间，优先于抽象特征区间参与命中。 |
| `稳态列表` | 实例观测形成的稳定值或稳定区间。 |
| `当前稳态索引` | 当前命中的稳态列表下标。 |

### 2.10 组合规则结构

多个抽象特征构成新的抽象特征时，不用子链表达来源，而用 `组合规则` 表达依赖。

| 类型 | 用途 |
| --- | --- |
| `结构_抽象特征组合输入` | 描述一个来源抽象特征、期望抽象状态或期望区间，以及缺失策略、权重、方向等。 |
| `结构_抽象特征组合输出` | 描述组合成立或不成立时写入组合实例特征的输出值。 |
| `结构_抽象特征组合规则` | 保存输入集合、组合方式、输出规则、优先级、启用状态和命中/失败统计。 |

当前代码已支持 `全部命中`、`任一命中`、`至少N项命中`、`命中计数` 的基础求值。`加权求和`、`有符号差值` 和 `方法计算` 已有数据结构入口，完整计算逻辑仍按计划继续补齐。

## 3. `特征类` 的职责

`特征类` 主要做八件事：

1. 绑定基础信息树。
2. 在世界树根下的抽象特征链中创建和查找抽象特征。
3. 在存在、场景或虚拟存在下面创建和查找实例特征。
4. 让实例特征引用抽象特征，而不是深度复制抽象特征树。
5. 读写实例特征当前值，并维护观测统计、候选窗口和稳态。
6. 根据 `I64` 当前值解析命中的抽象状态区间。
7. 将实例稳态升格为抽象区间，支持区间合并和重叠统计。
8. 比较两个特征值或两个特征集合，并为组合抽象特征求值。

实例特征的宿主边界是硬规则：

- 合法直接宿主：存在节点、场景节点。
- 间接承载：任务、方法、线程等对象需要特征时，先建立或取得对应虚拟存在，再把特征作为该虚拟存在的子节点。
- 非法直接宿主：任务节点、方法节点、线程节点以及其他非存在/非场景节点。
- 读取、写入、比较接口中的 `宿主` 参数，语义上都应理解为“存在、场景，或任务/方法/线程对应的虚拟存在”。

抽象特征的宿主边界不同：

- 根抽象特征挂在世界树根下的 `抽象特征根`。
- 抽象状态和结构子特征挂在父抽象特征下面。
- 抽象特征不能挂在存在或场景下作为实例状态使用。

它的私有状态只有两个：

| 字段 | 用途 |
| --- | --- |
| `基础信息_` | 指向基础信息树。为空时大多数操作失败或返回空结果。 |
| `差异度函数_` | 可选自定义差异度函数。未设置时使用内置差异计算。 |

## 4. 构造与绑定

### 4.1 `特征类(基础信息类* 基础信息 = nullptr) noexcept`

用途：

- 创建一个特征服务对象，并可选绑定基础信息树。

用法：

```cpp
特征类 特征服务(&基础信息);
```

注意：

- 只保存指针，不拥有 `基础信息类` 生命周期。
- 如果构造时传 `nullptr`，后续必须调用 `绑定基础信息`，否则创建、查询、删除等操作无法工作。

### 4.2 `绑定基础信息(基础信息类* 基础信息) noexcept`

用途：

- 重新绑定或延迟绑定基础信息树。

用法：

```cpp
特征服务.绑定基础信息(&世界树.基础信息());
```

注意：

- 该函数不迁移已有节点。
- 调用者必须保证传入的基础信息树在 `特征类` 使用期间仍然有效。

### 4.3 `设置差异度函数(差异度函数 函数) noexcept`

用途：

- 设置自定义差异度计算函数。

函数签名：

```cpp
I64 函数(const 词性节点类* 特征类型, const 特征值& 左值, const 特征值& 右值);
```

用法：

```cpp
特征服务.设置差异度函数([](const 词性节点类*, const 特征值& a, const 特征值& b) -> I64 {
    return a == b ? 0 : 10000;
});
```

注意：

- 该函数会影响稳态判定、`获取比较方法结果` 中的 VecU 相似度、以及 `比较特征集合` 的差异度。
- 对 `I64` 标量，如果没有自定义函数，默认使用绝对差。
- 对 `VecU句柄` 和 `指针句柄`，没有自定义函数时默认只做严格相等判断。

## 5. 主信息读取

### 5.1 `取特征主信息(特征节点类* 节点) const noexcept`

用途：

- 从基础信息节点中取出可写的 `特征节点主信息类`。

返回：

- 成功返回 `特征节点主信息类*`。
- `基础信息_` 为空、节点为空、或节点主信息不是特征时返回 `nullptr`。

用法：

```cpp
auto* 主信息 = 特征服务.取特征主信息(特征节点);
if (主信息) {
    主信息->设置当前值(特征值{ static_cast<I64>(1) });
}
```

注意：

- 直接修改主信息会绕过 `写入特征值_按参数` 的统计、近帧窗口和稳态维护。
- 普通写入优先使用 `写入特征值` 系列函数。

### 5.2 `取特征主信息(const 特征节点类* 节点) const noexcept`

用途：

- 只读版本，用于读取当前值、类型和稳态信息。

返回规则与可写版本相同。

### 5.3 `取抽象特征主信息(抽象特征节点类* 节点) const noexcept`

用途：

- 从基础信息节点中取出可写的 `抽象特征主信息类`。

返回：

- 成功返回 `抽象特征主信息类*`。
- `基础信息_` 为空、节点为空、或节点主信息不是抽象特征时返回 `nullptr`。

用法：

```cpp
auto* 主信息 = 特征服务.取抽象特征主信息(抽象特征节点);
if (主信息 && 主信息->区间.has_value()) {
    // 读取或修正抽象区间
}
```

注意：

- 抽象特征主信息只表达定义层信息，不应写入当前值。
- 修改 `组合规则` 后，可通过 `求值组合抽象特征` 把组合结果写入某个宿主的组合实例特征。

### 5.4 `取抽象特征主信息(const 抽象特征节点类* 节点) const noexcept`

用途：

- 只读版本，用于读取抽象特征的区间、组合规则、生命周期和重叠统计。

返回规则与可写版本相同。

## 6. 创建与删除

### 6.0 抽象特征创建入口

抽象特征的创建入口包括：

| 函数 | 用途 |
| --- | --- |
| `创建抽象特征(基础信息节点类* 父抽象特征, 抽象特征主信息类* 主信息)` | 使用调用者准备好的抽象特征主信息创建节点。 |
| `创建抽象特征_按类型(基础信息节点类* 父抽象特征, const 词性节点类* 特征类型, std::optional<I64区间> 区间)` | 按类型和可选区间创建抽象特征。 |
| `查找子抽象特征_按类型(const 基础信息节点类* 父抽象特征, const 词性节点类* 特征类型) const` | 只查找父抽象特征的直接子链。 |
| `取或创建子抽象特征_按类型(基础信息节点类* 父抽象特征, const 词性节点类* 特征类型)` | 有则返回，没有则创建一个区间为空的抽象特征壳。 |

标准用法：

```cpp
auto* 抽象根 = 世界树.取或创建抽象特征根();
auto* 任务状态 = 世界树.取或创建抽象特征_按类型(抽象根, 任务状态词);
auto* 执行中 = 世界树.特征().创建抽象特征_按类型(
    任务状态,
    执行中词,
    I64区间::点(2));
```

注意：

- 生成实例特征前应先有抽象特征壳。
- 抽象特征子链表达结构或状态划分，不表达“组合来源”。组合来源写入 `组合规则`。
- `创建抽象特征_按类型` 不做去重；需要复用时优先走 `取或创建子抽象特征_按类型`。

### 6.1 `创建特征(基础信息节点类* 宿主, 特征节点主信息类* 主信息)`

用途：

- 使用调用者准备好的主信息，在存在、场景或虚拟存在下面创建一个实例特征子节点。

用法：

```cpp
auto* 主信息 = new 特征节点主信息类(特征类型, 特征值{ static_cast<I64>(10) });
auto* 特征节点 = 特征服务.创建特征(存在或场景或虚拟存在节点, 主信息);
```

行为：

- `基础信息_` 为空或 `主信息` 为空时返回 `nullptr`。
- `宿主` 为空时，底层 `基础信息类::添加子节点` 会把节点挂到世界根下面；按当前规则，业务代码不应这样创建实例特征。
- 创建成功后，`主信息` 生命周期交给基础信息树。
- 这是偏底层入口。新代码若已经有抽象特征，应优先使用 `取或创建实例特征_按抽象特征`。

注意：

- 该函数不检查传入节点是否为存在、场景或虚拟存在，调用方必须保证宿主合法。
- 该函数不检查同一宿主下是否已有同类型特征。
- 需要避免重复特征时，应使用 `取或创建子特征_按类型`。
- 任务、方法、线程需要状态、进度、指针等特征时，应把这些特征挂到对应虚拟存在，而不是直接挂到任务、方法、线程节点上。

### 6.2 `创建特征_按类型(基础信息节点类* 宿主, const 词性节点类* 特征类型, const 特征值& 当前值 = {})`

用途：

- 按特征类型快速创建特征节点。

行为：

- `特征类型` 为空时返回 `nullptr`。
- 内部创建新的 `特征节点主信息类`。
- 新主信息的 `名称` 和 `类型` 都设置为 `特征类型`。
- `当前值` 写入主信息，但不会记录观测统计，也不会进入稳态判定。

用法：

```cpp
auto* 特征节点 = 特征服务.创建特征_按类型(存在或场景或虚拟存在节点, 安全值词, 特征值{ static_cast<I64>(0) });
```

注意：

- 该函数同样不去重。
- 该函数同样依赖调用方保证宿主是存在、场景或虚拟存在。
- 如果需要写入后维护统计和稳态，可以先创建，再调用 `写入特征值`。

### 6.3 `取或创建子特征_按类型(基础信息节点类* 宿主, const 词性节点类* 特征类型)`

用途：

- 在存在、场景或虚拟存在的子节点中查找指定类型实例特征；找不到时创建。

行为：

- 先调用 `查找子特征_按类型`。
- 命中则返回现有节点。
- 未命中则调用 `创建特征_按类型`。

用法：

```cpp
auto* 安全特征 = 特征服务.取或创建子特征_按类型(自我存在, 安全值词);
特征服务.写入特征值_I64(安全特征, 100);
```

注意：

- 查找时使用词性主键比较，不只是裸指针相等。
- 这是给存在、场景或虚拟存在补标准特征时最常用的入口。
- 若调用方手里是任务、方法或线程，应先取得其虚拟存在，再调用本函数。

### 6.4 `删除特征(特征节点类* 节点)`

用途：

- 从基础信息树删除一个特征节点。

返回：

- 删除成功返回 `true`。
- `基础信息_` 为空、节点为空、或节点是世界根时返回 `false`。

注意：

- 底层删除会影响树结构。
- 如果其他对象保存了该特征节点指针，删除后这些指针会失效。

## 7. 枚举与查找

### 7.1 `枚举全部特征() const`

用途：

- 从基础信息树全链中枚举所有 `特征节点主信息类` 节点。

返回：

- 返回 `std::vector<特征节点类*>`。
- `基础信息_` 为空时返回空数组。

注意：

- 这是全局枚举，不限制父节点；按设计规则，枚举结果应当都位于存在、场景或虚拟存在下面。
- 底层通过 `基础信息类::枚举节点_按类型<特征节点主信息类>()` 完成。

### 7.2 `获取子特征(const 基础信息节点类* 宿主) const`

用途：

- 获取某个存在、场景或虚拟存在下面的直接特征子节点。

行为：

- 只枚举直接子节点，不递归枚举孙节点。
- `宿主` 为空时，底层会枚举世界根下面的特征子节点；业务代码不应依赖这种形态。

用法：

```cpp
auto 特征集合 = 特征服务.获取子特征(存在节点);
```

### 7.3 `查找子特征_按类型(const 基础信息节点类* 宿主, const 词性节点类* 特征类型) const`

用途：

- 在存在、场景或虚拟存在的直接子特征中查找指定特征类型。

行为：

- `特征类型` 为空时返回 `nullptr`。
- 先获取目标节点的直接子特征，再逐个读取主信息。
- 类型比较先看指针是否相同，再比较词性主键。

用法：

```cpp
auto* 特征节点 = 特征服务.查找子特征_按类型(任务虚拟存在, 任务状态词);
```

注意：

- 只返回第一个命中的特征。
- 如果同一存在、场景或虚拟存在下已经出现多个同类型特征，后面的重复节点不会被返回。

### 7.4 `查找实例特征_按抽象特征(const 基础信息节点类* 宿主, const 抽象特征节点类* 抽象特征) const`

用途：

- 在宿主的直接子特征中，查找实例化自指定抽象特征的实例特征。

行为：

- 比较 `特征节点主信息类::抽象特征`，支持指针和主键引用。
- 只查找直接子节点，不递归查找。

用法：

```cpp
auto* 当前阶段实例 = 特征服务.查找实例特征_按抽象特征(任务虚拟存在, 当前阶段抽象特征);
```

### 7.5 `取或创建实例特征_按抽象特征(基础信息节点类* 宿主, 抽象特征节点类* 抽象特征)`

用途：

- 在宿主下取得指定抽象特征的实例特征；不存在时懒创建。

行为：

- 先按抽象特征引用查找。
- 未命中时创建 `特征节点主信息类`。
- 新实例主信息的 `名称` 和 `类型` 跟随抽象特征主信息。
- 新实例主信息的 `抽象特征` 绑定到传入抽象特征。

用法：

```cpp
auto* 当前阶段实例 = 特征服务.取或创建实例特征_按抽象特征(
    任务虚拟存在,
    当前阶段抽象特征);
特征服务.写入特征值_I64(当前阶段实例, 2);
```

注意：

- 这是新模型下推荐的实例特征创建入口。
- 它不会深度复制抽象特征的子链。需要子实例特征时，应按子抽象特征再次懒创建。

## 8. 写入与读取

### 8.1 `写入特征值_按参数(特征节点类* 节点, const 特征值& 值, const 写入参数& 参数 = {})`

用途：

- 写入特征当前值，并根据参数维护统计、候选窗口、稳态列表和当前稳态索引。

完整流程：

1. 创建返回结果，并把 `结果.特征` 设为传入节点。
2. 读取特征主信息。
3. 如果主信息不存在或值无效，提前返回。
4. 取写入时间：`参数.ts != 0` 时使用参数时间，否则使用当前微秒时间。
5. 更新 `主信息->当前值`。
6. 调用 `主信息->统计.记录观测(现在)`。
7. 把当前值和时间写入近帧环形缓冲。
8. 如果 `仅更新当前值` 为真，结束。
9. 计算当前值和近帧历史值的差异度。
10. 若连续帧数不足 `max(2, 连续N)`，结束。
11. 若任一近帧差异度超过 `阈值_判稳`，候选门失败。
12. 候选门通过后，尝试命中已有稳态原型。
13. 未命中且允许创建稳态时，创建新稳态。
14. 更新 `当前稳态索引` 和 `稳态发生变化`。
15. 如允许淘汰且设置了超时，删除长期未命中的稳态。
16. 若稳态数量超过 `Kmax_points`，删除最弱稳态。
17. 根据实例特征的 `当前值` 和 `抽象特征` 解析当前命中的抽象状态。

用法：

```cpp
写入参数 参数{};
参数.稳态.连续N = 3;
参数.稳态.阈值_判稳 = 1;
参数.稳态.阈值_命中原型 = 1;

auto 结果 = 特征服务.写入特征值_按参数(特征节点, 特征值{ static_cast<I64>(42) }, 参数);
if (结果.稳态发生变化) {
    // 可触发状态变化后续处理
}
```

差异度规则：

- 设置了 `差异度函数_` 时优先使用自定义函数。
- 未设置自定义函数且两边都是 `I64` 时，使用饱和减后的绝对值。
- 其他类型默认使用严格相等，等于 `0`，不等于 `10000`。
- 对 `I64`，如果特征主信息设置了有效的 `连续测量误差区间`，会按差值到误差区间的距离修正差异度。

注意：

- 传入 `std::monostate` 无效值时，不会写入。
- `创建特征_按类型` 设置初始当前值不会走这套统计和稳态逻辑。
- `结果.特征 != nullptr` 不等于写入成功，因为传入节点会先写入结果。判断写入是否有效时还要看主信息和值是否有效，或使用封装后的 `写入特征值`。
- 只有 `I64` 当前值会进入抽象区间命中；`VecU句柄` 和 `指针句柄` 如需区间，应先比较或映射成 `I64`。

### 8.2 `写入特征值(特征节点类* 节点, const 特征值& 值, 时间戳 now = 当前_微秒())`

用途：

- 使用默认写入参数写入任意 `特征值`。

返回：

- 当 `写入特征值_按参数(...).特征 != nullptr` 且传入值有效时返回 `true`。

用法：

```cpp
特征服务.写入特征值(特征节点, 特征值{ static_cast<I64>(1) });
```

注意：

- 当前返回判断没有直接检查主信息是否存在，因此传入非空节点但主信息不对时，返回语义需要谨慎理解。
- 更需要掌握稳态状态时，优先使用 `写入特征值_按参数`。

### 8.3 `写入特征值_I64(特征节点类* 节点, I64 值, 时间戳 now = 当前_微秒())`

用途：

- 便捷写入 `I64` 标量特征。

典型用途：

- 安全值、服务值。
- 枚举状态。
- 数量。
- 布尔型 `0/1`。
- 方向或完成度等整数特征。

示例：

```cpp
特征服务.写入特征值_I64(状态特征, static_cast<I64>(1));
```

### 8.4 `写入特征值_VecU句柄(特征节点类* 节点, VecU句柄 值, 时间戳 now = 当前_微秒())`

用途：

- 写入向量证据池句柄。

典型用途：

- 轮廓。
- 签名。
- 体素或点阵。
- 需要通过 `特征值类` 证据池保存的大型向量值。

注意：

- 该函数只写句柄，不负责创建证据池内容。
- 向量内容相似度需要通过自定义差异度函数接入，否则默认只比较句柄是否相等。

### 8.5 `写入特征值_指针句柄(特征节点类* 节点, 指针句柄 值, 时间戳 now = 当前_微秒())`

用途：

- 写入运行期指针句柄。

典型用途：

- 当前任务节点。
- 当前方法节点。
- 当前步骤节点。
- 其他运行期对象引用。

注意：

- 指针句柄不是稳定持久化主键。
- 指向对象删除后，句柄不会自动失效。
- 这里的“当前任务节点、当前方法节点”只是特征值指向的对象，不表示任务节点或方法节点可以直接作为特征宿主。

### 8.6 `读取特征值(const 特征节点类* 节点) const`

用途：

- 读取特征节点当前值。

返回：

- 有主信息时返回 `主信息->当前值`。
- 无主信息时返回空 `特征值{}`。

用法：

```cpp
特征值 当前 = 特征服务.读取特征值(特征节点);
if (const auto* 标量 = std::get_if<I64>(&当前)) {
    // 使用 *标量
}
```

### 8.7 `枚举命中抽象状态(const 特征节点类* 实例特征) const`

用途：

- 根据实例特征当前 `I64` 值，枚举它命中的抽象状态。

行为：

- 读取实例主信息的 `抽象特征`。
- 优先检查实例特征的 `局部区间覆盖`。
- 再检查抽象特征的直接子抽象特征区间。
- 如果没有子状态命中，再检查抽象特征自身区间。
- 当前值不是 `I64`、没有抽象特征引用、或无有效区间时返回空集合。

### 8.8 `解析实例特征命中抽象特征(特征节点类* 实例特征)`

用途：

- 把 `枚举命中抽象状态` 的结果写回实例主信息的 `当前命中抽象特征`。

行为：

- 没有命中时清空 `当前命中抽象特征`。
- 多个抽象状态同时命中时，为每个命中抽象特征累加 `重叠命中次数` 和 `最近重叠命中时间`。
- 当前实现选择区间宽度最小的命中作为主命中。

用法：

```cpp
auto* 命中状态 = 特征服务.解析实例特征命中抽象特征(当前阶段实例);
```

| `清除实例特征局部区间覆盖(特征节点类* 实例特征)` | 清除局部区间，恢复使用抽象特征区间。 |

局部区间覆盖用于允许某个宿主临时拥有更细或不同的命中边界，也用于后续拆分抽象区间时保留证据。

相关函数：

| 函数 | 用途 |
| --- | --- |
| `升格当前稳态为抽象区间(特征节点类* 实例特征)` | 把实例当前稳态的 `I64区间` 升格到抽象特征定义层。 |
| `合并抽象区间(抽象特征节点类* 目标抽象特征, const 抽象特征节点类* 来源抽象特征)` | 将来源抽象区间并入目标抽象区间。 |

## 9. 比较

### 9.1 `获取比较方式(const 词性节点类* 特征类型, const 特征值& 左值, const 特征值& 右值) const noexcept`

用途：

- 根据特征值形态选择比较模式。

当前规则：

| 条件 | 返回比较模式 |
| --- | --- |
| 任一值无效 | `相等` |
| 两个值 variant 类型不同 | `相等` |
| 两个值都是 `I64` | `有序标量` |
| 两个值都是 `指针句柄` | `相等` |
| 两个值都是 `VecU句柄` | `相等或相似度` |
| 其他情况 | `相等` |

注意：

- 该函数本身只返回比较方式。
- 真正判断是否可比较的是 `获取比较方法结果`，它会在值无效或类型不一致时返回 `可比较=false`。
- 当前实现没有按 `特征类型` 特化比较模式，参数保留给后续扩展。

### 9.2 `获取比较方法结果(...) const`

用途：

- 给两个特征值生成可用于方向判断、需求判断或差异解释的比较结果。

参数：

| 参数 | 用途 |
| --- | --- |
| `特征类型` | 当前特征类型，主要传给自定义差异度函数。 |
| `左值` | 左侧值，通常是初始状态、当前状态或基准值。 |
| `右值` | 右侧值，通常是结果状态、目标状态或候选值。 |
| `左特征主信息` | 可选，用于读取左侧连续测量误差区间。 |
| `右特征主信息` | 可选，用于读取右侧连续测量误差区间。 |

返回规则：

- 任一值无效，或两个值形态不同，返回默认结果，`可比较=false`。
- `I64` 使用有序标量比较。
- `指针句柄` 使用相等比较。
- `VecU句柄` 使用相等或相似度比较。

#### 有序标量

对 `I64`：

```cpp
原始差值 = 右值 - 左值
```

然后根据两侧主信息里的 `连续测量误差区间` 调整差值：

- 两边都有误差区间时，取两者低值的较小值和高值的较大值形成合并区间。
- 只有一边有误差区间时，使用该区间。
- 差值落在误差区间内时，调整后差值为 `0`。
- 差值低于区间时，返回 `差值 - 区间低值`。
- 差值高于区间时，返回 `差值 - 区间高值`。

结果：

- `可比较 = true`
- `左比较值 = 左值`
- `右比较值 = 左值 + 调整后差值`
- `差值 = 调整后差值`

用途：

- 判断目标方向：`差值 > 0` 可理解为增加，`差值 < 0` 可理解为减少，`差值 == 0` 可理解为不变或落在误差范围内。

#### 相等比较

用于指针句柄和其他相等型值：

- 严格相等时，`右比较值 = 0`，`差值 = 0`。
- 不相等时，`右比较值 = 1`，`差值 = 1`。

用途：

- 判断两个离散值是否一致。

#### 相等或相似度

主要用于 `VecU句柄`：

- 如果有自定义差异度函数，则使用自定义差异度。
- 如果没有自定义函数但严格相等，则差异度为 `0`。
- 否则差异度为 `10000`。

用途：

- 向量签名、轮廓、图像摘要等值的相似度判断入口。

### 9.3 `比较特征集合(const 特征集合& A, const 特征集合& B, const 集合比较参数& 参数 = {}) const`

用途：

- 比较两个特征集合，生成整体差异度和可选明细。

流程：

1. 遍历集合 A，按 `主信息->类型` 建立左表。
2. 遍历集合 B，按 `主信息->类型` 建立右表。
3. 合并左右两边出现过的全部特征类型。
4. 对每个特征类型计算明细差异度。
5. 累加加权差异和与权重和。
6. 输出总差异度、归一化差异度和命中特征数。

差异度规则：

- 左侧缺失：`缺失A=true`，差异度为缺失惩罚。
- 右侧缺失：`缺失B=true`，差异度为缺失惩罚。
- 主信息缺失或当前值无效：差异度为缺失惩罚。
- 有自定义差异度函数：使用自定义差异度。
- 两边都是 `I64`：使用饱和绝对差。
- 其他类型：严格相等为 `0`，不等为缺失惩罚。

用法：

```cpp
auto 左特征 = 特征服务.获取子特征(左存在);
auto 右特征 = 特征服务.获取子特征(右存在);

集合比较参数 参数{};
参数.输出明细 = true;
auto 比较 = 特征服务.比较特征集合(左特征, 右特征, 参数);
```

注意：

- 当前索引键是 `const 词性节点类*` 指针本身，而不是词性主键。
- 同一集合内同一个 `类型` 指针只保留第一个特征节点。
- 如果未来存在“不同词性节点指针但主键相同”的情况，这里和 `查找子特征_按类型` 的行为会不完全一致。

## 10. 度量签名

### 10.1 `生成度量签名(const 特征节点类* 节点) const`

用途：

- 为特征节点生成一个轻量字符串签名。

格式：

```text
Feature|T=<类型主键>|N=<名称主键>|V=<值形态>
```

其中值形态为：

| 当前值形态 | 输出 |
| --- | --- |
| `I64` | `V=I64` |
| `VecU句柄` | `V=VecU` |
| `指针句柄` | `V=Ptr` |
| 无值或其他 | `V=None` |

示例：

```cpp
std::string 签名 = 特征服务.生成度量签名(特征节点);
```

注意：

- 签名只描述类型、名称和值形态，不包含具体值。
- 类型或名称为空时，对应字段不会出现在签名中。
- 常用于二次特征或度量链的轻量区分。

## 11. 私有辅助函数

这些函数位于 `特征类.cpp` 的匿名命名空间，只供本文件内部使用。

| 函数 | 用途 |
| --- | --- |
| `私有_词性相同` | 比较两个词性节点是否相同。先比较指针，再比较主键。 |
| `私有_饱和加` | 做 `I64` 加法，溢出时钳制到 `I64_MAX` 或 `I64_MIN`。 |
| `私有_饱和减` | 做 `I64` 减法，避免 `I64_MIN` 取负溢出。 |
| `私有_绝对值饱和` | 取绝对值，`I64_MIN` 返回 `I64_MAX`。 |
| `私有_点到区间距离` | 计算点到有效区间外部的距离；点在区间内返回 `0`。 |
| `私有_区间外有符号差值` | 保留方向地计算点相对区间边界的差值；点在区间内返回 `0`。 |
| `私有_特征值严格相等` | 按 variant 形态比较 `I64`、`VecU句柄`、`指针句柄` 是否严格相等。 |
| `私有_节点引用相同` | 比较两个基础信息节点是否指向同一节点或同一主键。 |
| `私有_轻引用相同` | 比较轻量基础信息引用与节点是否相同。 |
| `私有_区间相同` | 判断两个 `I64区间` 是否边界相同且有效性一致。 |
| `私有_区间宽度` | 计算区间宽度，用于多个抽象状态命中时选择更窄的主命中。 |

这些辅助函数的重点是保证差值计算不会因为整数边界导致溢出。

## 12. 常见使用路径

### 12.1 新模型：先建抽象特征，再懒创建实例特征

```cpp
auto* 抽象根 = 世界树.取或创建抽象特征根();
auto* 当前阶段抽象 = 世界树.取或创建抽象特征_按类型(抽象根, 当前阶段词);

auto* 执行中抽象 = 世界树.特征().创建抽象特征_按类型(
    当前阶段抽象,
    执行中词,
    I64区间::点(2));

auto* 当前阶段实例 = 世界树.取或创建实例特征_按抽象特征(
    任务虚拟存在,
    当前阶段抽象);

世界树.写入特征_I64(当前阶段实例, 2);
auto* 命中状态 = 世界树.解析实例特征命中抽象特征(当前阶段实例);
```

适合：

- 任务状态、方法状态、运行阶段等原来容易写成结构体字段的语义。
- 布尔、枚举、等级、数量等可映射成 `I64` 的状态。
- 需要让多个存在共享同一套特征定义的场景。

### 12.2 旧兼容路径：给存在、场景或虚拟存在补一个 I64 特征

```cpp
auto* 特征节点 = 特征服务.取或创建子特征_按类型(存在或场景或虚拟存在节点, 特征类型);
特征服务.写入特征值_I64(特征节点, static_cast<I64>(1));
```

适合：

- 尚未完成抽象特征迁移的旧入口。
- 安全/服务值。
- 临时数量特征。
- 布尔型特征。

如果要给任务、方法或线程记录这些特征，应使用对应虚拟存在：

```cpp
auto* 特征节点 = 特征服务.取或创建子特征_按类型(任务虚拟存在, 特征类型);
特征服务.写入特征值_I64(特征节点, static_cast<I64>(1));
```

### 12.3 读取存在、场景或虚拟存在上的某个特征

```cpp
auto* 特征节点 = 特征服务.查找子特征_按类型(存在或场景或虚拟存在节点, 特征类型);
特征值 当前值 = 特征服务.读取特征值(特征节点);
```

### 12.4 比较两个状态是否达到目标

```cpp
auto 比较 = 特征服务.获取比较方法结果(
    特征类型,
    当前值,
    目标值,
    当前特征主信息,
    目标特征主信息);

if (比较.可比较 && 比较.差值 == 0) {
    // 当前状态和目标状态在该特征上已经一致或落入误差区间
}
```

### 12.5 比较两个存在的整体差异

```cpp
auto A = 特征服务.获取子特征(存在A);
auto B = 特征服务.获取子特征(存在B);
auto 结果 = 特征服务.比较特征集合(A, B);
```

### 12.6 组合抽象特征求值

当一个抽象特征由多个抽象特征组合而来时：

1. 在组合抽象特征主信息的 `组合规则` 中写入来源抽象特征引用、期望区间和组合方式。
2. 宿主下分别懒创建并写入这些来源实例特征。
3. 调用 `求值组合抽象特征`。
4. 组合结果会写入宿主下的组合实例特征。

```cpp
世界树.求值组合抽象特征(任务虚拟存在, 可执行抽象特征);
```

## 13. 与世界树的关系

虽然可以直接使用 `特征类`，但项目中更常见的入口是 `世界树类`：

- `世界树类::特征()` 返回内部特征服务。
- `世界树类::取或创建抽象特征根()` 返回世界树根下的抽象特征容器。
- `世界树类::创建抽象特征()` / `取或创建抽象特征_按类型()` 管理抽象特征。
- `世界树类::取或创建实例特征_按抽象特征()` 在宿主下懒创建实例特征。
- `世界树类::解析实例特征命中抽象特征()` 刷新实例特征当前命中状态。
- `世界树类::升格实例稳态为抽象区间()` 将实例稳态提升为抽象区间。
- `世界树类::求值组合抽象特征()` 对组合抽象特征求值并写回实例特征。
- `世界树类::确保特征()` 转调 `取或创建子特征_按类型`。
- `世界树类::写入特征_I64()` 会确保特征后写入 `I64`。
- `世界树类::读取特征快照()` 会查找并读取当前值。

因此普通业务代码优先走 `世界树类` 的封装，只有需要比较、稳态参数、组合规则细节或生成度量签名时，再直接使用 `世界树.特征()`。

无论走 `特征类` 还是 `世界树类` 封装，都必须遵守同一个宿主规则：抽象特征挂在抽象特征链；实例特征挂在存在、场景，或任务/方法/线程对应的虚拟存在。

## 14. 注意事项

- `特征类` 不拥有基础信息树，只保存裸指针。
- 抽象特征不能当实例状态使用；实例特征不能作为定义层本体使用。
- 实例特征只能作为存在、场景或虚拟存在的子节点；任务、方法、线程需要特征时必须使用虚拟存在。
- `创建抽象特征`、`创建特征` 和 `创建特征_按类型` 不做去重。
- `取或创建子特征_按类型` 只查找直接子节点，不递归。
- `取或创建实例特征_按抽象特征` 也只查找宿主直接子节点，并且不深度复制抽象子树。
- `写入特征值_按参数` 会维护统计、候选和稳态；直接改主信息不会。
- 抽象特征不保存当前值和稳态列表。
- V1 抽象区间只支持 `I64区间`。
- 区间重叠允许存在，当前实现会统计重叠命中，并选择较窄区间作为主命中。
- `VecU句柄` 默认只做句柄相等比较；需要内容相似度时要设置自定义差异度函数。
- `指针句柄` 是运行期引用，不能当稳定持久化事实。
- `比较特征集合` 当前按词性节点指针建表，而 `查找子特征_按类型` 使用词性主键兼容判断，这两个入口的相等语义不完全一致。
- `写入特征值` 的返回值只适合做简单成功判断；如果要知道候选门或稳态变化，应使用 `写入特征值_按参数`。
