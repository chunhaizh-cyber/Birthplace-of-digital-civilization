# 方法类说明

本文说明 `D:\鱼巢\方法主信息类.h`、`D:\鱼巢\方法类.h`、`D:\鱼巢\方法类.cpp`，并简要说明它和 `D:\鱼巢\本能方法类.h` 的关系。

`方法类` 是方法树的服务层。它的职责边界是创建、查找、修改和维护方法首节点、方法条件节点、方法结果节点，也就是组织方法树结构。每个方法自己的状态、成熟度、权限、统计和运行特征应存储在方法虚拟存在上；这些特征对方法树类应保持透明，方法树类不判断具体方法实现。

## 1. 文件定位

当前项目中，方法系统处在需求、任务和本能动作之间：

- 需求给出需要的目标特征类型和方向。
- 任务筹办根据需求查找或创建候选方法。
- 方法首节点描述“能做什么”或“主结果是什么”。
- 方法条件节点描述“在什么条件下可用”。
- 方法结果节点描述“会产生什么结果方向”。
- 方法虚拟存在承载方法自己的运行状态、成熟度、权限和统计特征；这些内容不属于方法树结构职责。
- 本能方法是带有 `本能函数ID` 动作句柄的方法。

简单说：

- 方法类负责组织方法树结构。
- 世界树负责承载事实、状态和特征。
- 方法虚拟存在是方法自身特征接入世界树的锚点，对方法树组织逻辑透明。

## 2. 方法树结构

`方法类` 继承自：

```cpp
链表模板<方法信息基类>
```

因此方法节点是独立链表树节点，不是 `基础信息节点类`。

常见结构是：

```text
方法根节点
  方法首节点
    方法条件节点
    方法结果节点
```

注意：

- 方法首节点是方法身份锚点。
- 条件节点和结果节点是方法首节点下的结构子节点。
- 条件节点和结果节点不是基础信息场景本身，而是引用条件场景、结果场景。
- 方法运行状态不直接挂在方法节点上，而是挂到方法虚拟存在。

## 7. 方法类总体职责

`方法类` 封装以下能力：

1. 在独立方法树中查找方法首节点。
2. 注册普通方法首节点。
3. 注册需求位方法首节点。
4. 注册需求方向方法首节点。
5. 查找或创建本能方法首节点。
6. 补齐本能方法的条件场景、结果场景、条件节点和结果节点。
7. 创建或读取方法虚拟存在。
8. 创建或读取方法条件场景、结果场景。
9. 在方法虚拟存在上记录方法标准特征。
10. 同步方法成熟度、召回权限、默认执行权限和特征使用统计。
11. 创建方法条件节点和结果节点。

## 8. 查找方法首节点

### 8.1 `查找方法首节点_按动作句柄`

用途：

- 在方法根节点的直接子节点中查找动作句柄一致的方法首节点。

规则：

1. 方法根节点必须存在。
2. 动作句柄必须有效。
3. 先归一到独立树根。
4. 只查找根节点直接子链。
5. 只命中 `节点种类 == 方法首节点` 的节点。
6. 动作句柄类型、本能 ID、外部实现主键、动作序列主键都必须一致。

返回：

- 命中返回方法首节点。
- 否则返回 `nullptr`。

### 8.2 `查找方法首节点_按动作名`

用途：

- 在方法根节点的直接子节点中查找动作名一致的方法首节点。

规则：

1. 方法根节点和动作名必须存在。
2. 先归一到独立树根。
3. 只查找根节点直接子链。
4. 只命中 `节点种类 == 方法首节点` 的节点。
5. 动作名按词性节点主键比较。

## 9. 注册方法首节点

### 9.1 `注册方法首节点(...)`

用途：

- 注册或复用一个方法首节点。

必要条件：

- 方法根节点存在。
- 至少具备以下任意一种出生锚点：
  - 动作名。
  - 有效动作句柄。
  - 主结果特征类型。

查找复用顺序：

1. 如果动作句柄有效，按动作句柄查找。
2. 没命中且动作名存在，按动作名查找。
3. 如果没有动作名、没有动作句柄、但有主结果特征类型，则按“仅结果特征需求位”查找。

新建时写入：

- `节点种类 = 方法首节点`。
- `动作名`。
- `动作句柄`。
- `主结果特征类型`。
- 如果动作句柄是本能函数 ID，则 `来源 = 本能`。
- `作用域`。

复用时补齐：

- 节点种类重置为方法首节点。
- 有动作名则覆盖动作名。
- 有效动作句柄则覆盖动作句柄。
- 如果主结果特征类型为空且调用方传入，则补齐。
- 如果来源为空且动作句柄是本能函数 ID，则补为 `本能`。
- 按注册入口提升方法作用域。

最后动作：

- 调用 `初始化方法虚拟存在信息`。
- 返回方法首节点。

注意：

- 方法头出生不要求当前可执行。
- 只要有动作骨架或主结果特征类型，就可以成为方法头。

### 9.2 `注册需求位方法首节点(...)`

用途：

- 为某个主结果特征类型创建或复用“仅结果特征需求位”方法头。

执行流程：

1. 调用 `注册方法首节点`，动作名为空、动作句柄为空、主结果特征类型非空。
2. 如果传入来源且节点原来源未定义，则写入来源。

注意：

- 这个函数创建的是需求位方法头，不是占位假方法。
- 它会进入方法列表，成为任务筹办可以绑定的正式方法头。

### 9.3 `注册需求方向方法首节点(...)`

用途：

- 为某个主结果特征类型和结果方向创建或复用需求方向方法头。

规则：

- 如果结果方向为空，退回 `注册需求位方法首节点`。
- 方法根节点和主结果特征类型必须存在。
- 查找时按“主结果特征类型 + 结果方向语义”匹配。

结果方向语义：

- 从二次特征中解析特征类型主键。
- 从二次特征中解析比较结果编码。
- 只有特征类型主键非空且比较结果不是 `2` 时有效。

新建时写入：

- `来源`。
- `主结果特征类型`。
- `结果方向索引`。

复用时补齐：

- 补主结果特征类型。
- 补来源。
- 如果现有结果方向不含同语义方向，则追加。

最后动作：

- 调用 `初始化方法虚拟存在信息`。

## 10. 过渡性职责边界

方法类当前仍保留少量历史兼容入口，主要服务旧运行链路：

- 本能方法首节点补齐。
- 方法虚拟存在创建。
- 条件场景、结果场景创建。
- 方法虚拟存在特征写入和默认初始化。

这些能力不是方法树增删改查的核心职责。后续清理方向是把方法虚拟存在特征读写、本能方法结果声明和本能补齐流程迁出方法类，让 `方法类.h` 只保留方法树组织接口。

## 11. 本能方法首节点

### 11.1 `查找或创建_本能方法首节点(...)`

用途：

- 根据宿主存在和本能方法 ID 查找或创建本能方法首节点。

必要条件：

- 宿主存在存在。
- 能从 `本能方法类::查询元信息` 查到元信息。
- 宿主存在有方法根节点。

执行流程：

1. 读取本能方法元信息。
2. 读取宿主存在的方法根节点。
3. 构造动作句柄：

```cpp
类型 = 本能函数ID
本能ID = static_cast<I64>(本能ID)
```

4. 用元信息动作名称创建动作名词。
5. 调用 `注册方法首节点`，作用域为 `主体局部`。
6. 调用 `补齐本能方法首节点`。
7. 创建或读取方法虚拟存在。
8. 根据本能分类写入方法类型特征。

返回：

- 成功返回方法首节点。
- 任一步失败返回 `nullptr`。

### 11.2 `补齐本能方法首节点(...)`

用途：

- 把一个方法首节点补齐成本能方法结构。

执行流程：

1. 归一到方法首节点。
2. 写入节点种类、动作名、动作句柄、本能 ID、来源。
3. 根据本能 ID 补齐结果声明元数据。
4. 初始化方法虚拟存在信息。
5. 创建或读取方法条件场景。
6. 写入本能条件二次特征。
7. 吸收条件场景的二次特征索引到 `条件判定索引`。
8. 创建方法条件节点。
9. 创建或读取方法结果场景。
10. 写入结果场景引用和结果主键。
11. 根据本能结果声明补齐结果方向索引。
12. 确保方法结果初始状态。
13. 创建方法结果节点。
14. 返回方法首节点是否具备可执行动作条件。

现行可执行动作条件：

```text
有动作骨架 && 条件判定索引非空
```

注意：

- 本能补齐流程会尝试同时创建条件节点和结果节点。
- 单独调用 `创建条件节点` 不会自动创建结果节点。
- 单独调用 `创建结果节点` 也不会自动创建条件节点。
- 如果业务规则要求“有条件节点就必须有结果节点”，应由上层流程或自检检查保证。

### 11.3 本能方法结果声明

`方法类.cpp` 内部为部分本能方法声明了主结果特征和方向。

示例：

| 本能方法 | 主结果示例 |
| --- | --- |
| `自我_任务执行一步` | 任务状态增加；安全值、服务值作为附带影响。 |
| `自我_增加安全值` | 安全值增加。 |
| `自我_减少安全值` | 安全值减少。 |
| `自我_保持安全值` | 安全值不变。 |
| `自我_增加服务值` | 服务值增加。 |
| `学习_尝试学习` | 方法是否可用增加。 |
| `学习_补齐方法结构` | 方法结构是否完整增加。 |
| `学习_验证方法稳定性` | 方法稳定状态增加。 |
| `管理_检查方法运行账完整性` | 方法运行账完整性增加。 |

声明用途：

- 补齐 `主结果特征类型`。
- 在结果场景中生成结果方向二次特征。
- 把参与方法方向索引的方向加入 `结果方向索引`。

## 12. 方法虚拟存在

### 12.1 定位

方法虚拟存在是方法的世界树承载节点。

它的类型词是：

```text
方法虚拟存在
```

名称来自方法标识文本：

1. 优先动作名主键。
2. 其次主结果特征类型主键。
3. 再其次动作句柄键。
4. 都没有则是 `未命名方法`。

方法虚拟存在必须带有可区分该方法的特征信息。不能只创建同名同类型的 `方法虚拟存在`，再写入一组完全相同的默认运行特征。至少应通过动作入口、动作语义、主结果特征、结果方向、条件结构、结果结构或来源范围中的一项或多项，让不同方法在虚拟存在层可被区分。

通用特征可以同名，例如 `方法是否可用`、`方法可靠度`、`最近运行状态`。但通用特征不能成为唯一信息。两个方法如果只有这些通用特征，且值也相同，应视为不可区分的方法虚拟存在；后续创建时应复用、合并或补充真实语义差异。

### 12.2 `取或创建_方法虚拟存在(...)`

用途：

- 读取或创建方法首节点对应的方法虚拟存在。

执行流程：

1. 归一到方法首节点。
2. 如果主信息已有方法虚拟存在指针，则直接返回。
3. 如果未传宿主存在，则遍历全部存在，寻找拥有当前方法根节点的宿主存在。
4. 如果找到宿主存在，则使用宿主存在内部世界。
5. 否则使用世界树虚拟世界；必要时初始化默认世界。
6. 在承载世界下取或创建子存在：
   - 名称：方法虚拟存在名称词。
   - 类型：方法虚拟存在。
7. 写回方法首节点主信息的 `方法虚拟存在` 引用。
8. 写入方法类型特征。

注意：

- 方法虚拟存在是存在节点。
- 方法相关特征都应挂在方法虚拟存在上。
- 这符合“特征只能挂在存在或场景下”的建模规则。
- 每个方法虚拟存在都应有方法身份或结构标识特征，不能只依赖存在节点主键区分方法。

### 12.3 `取或创建_方法条件场景(...)`

用途：

- 读取或创建方法条件场景。

执行流程：

1. 归一到方法首节点。
2. 如果已有条件场景指针，直接返回。
3. 确保方法虚拟存在。
4. 在方法虚拟存在下取或创建名为 `条件场景` 的内部世界场景。
5. 绑定宿主为方法虚拟存在。
6. 写回方法首节点主信息的条件场景引用。

### 12.4 `取或创建_方法结果场景(...)`

用途：

- 读取或创建方法结果场景。

执行流程：

1. 归一到方法首节点。
2. 如果已有结果场景指针，直接返回。
3. 确保方法虚拟存在。
4. 在方法虚拟存在下取或创建名为 `结果场景` 的内部世界场景。
5. 绑定宿主为方法虚拟存在。
6. 写回方法首节点主信息的结果场景引用。

## 13. 方法标准特征

### 13.1 `记录方法虚拟存在特征(...)`

用途：

- 在方法虚拟存在上写入一个特征值。

支持值类型：

- `I64`：调用 `世界树.写入特征_I64`。
- `指针句柄`：调用 `世界树.写入特征_指针`。
- `VecU句柄`：取值池中的 `VecU` 后调用 `世界树.写入特征_VecU`。

注意：

- 特征写入对象是方法虚拟存在，不是方法节点。
- 如果无法创建方法虚拟存在，返回 `false`。

### 13.2 `初始化方法虚拟存在信息(...)`

用途：

- 确保方法虚拟存在、条件场景、结果场景和方法标准特征存在。
- 方法虚拟存在初始化必须同时覆盖通用特征信息和区分特征信息。
- 通用特征信息用于统一运行、学习、治理和统计；区分特征信息用于说明这个方法和其他方法有什么不同。
- 初始化时不能只写其中一类。只写通用特征会造成方法不可区分；只写区分特征会造成方法无法进入统一治理链路。

会初始化的标准特征包括：

| 特征 | 默认值或来源 |
| --- | --- |
| `方法类型` | 从动作句柄推导。 |
| `方法是否可用` | `0`。 |
| `方法可靠度` | `0`。 |
| `动作句柄类型` | 当前动作句柄类型。 |
| `成熟度阶段` | 本能方法默认 `3`，其他动作骨架默认 `1`。 |
| `允许自动召回` | 成熟度大于等于 `2` 时为 `1`，否则 `0`。 |
| `允许默认执行` | 成熟度大于等于 `3` 时为 `1`，否则 `0`。 |
| `特征账本生命周期` | `候选`。 |
| `方法标识键` | 由动作名、动作句柄、主结果特征类型和结果方向等组合生成。 |
| `动作语义键` | 来自动作名或动作句柄。 |
| `本能动作ID` | 仅本能函数 ID 有效时写入。 |
| `主结果特征类型` | 方法首节点有主结果特征类型时写入引用或主键。 |
| `主结果方向标识` | 方法首节点已有结果方向索引时写入结构标识。 |
| `条件结构标识` | 条件场景或条件判定索引可生成时写入结构标识。 |
| `结果结构标识` | 结果场景或结果方向索引可生成时写入结构标识。 |
| `来源范围` | 能定位宿主存在、来源任务或方法根时写入引用或主键。 |

还会初始化一批使用统计特征：

- 暴露次数。
- 读取次数。
- 有效使用次数。
- 历史使用次数。
- 近期使用次数。
- 作为决策输入次数。
- 作为派生父特征次数。
- 作为需求触发次数。
- 作为风险闸门次数。
- 作为验证信号次数。
- 作为解释依据次数。
- 结果改变次数。
- 坏结果次数。
- 下游依赖特征数。
- 最近使用时间。
- 最近有效使用时间。

注意：

- 初始化函数只在特征不存在时写默认值。
- 已有 I64 特征不会被默认值覆盖。
- 通用特征信息应随方法虚拟存在创建同步初始化。
- 默认运行特征不能替代方法身份特征；新增方法虚拟存在时必须尽量写入可区分信息。

### 13.3 方法特征公开接口清理状态

成熟度、权限、统计和生命周期的公开读写接口已从 `方法类.h` 删除。方法类当前只保留运行链路仍在使用的兼容入口，后续应继续外移到方法虚拟存在服务或具体本能执行层。

### 13.4 生命周期特征

`枚举_特征生命周期` 的值定义在 `基础数据类型.h`：

| 枚举值 | 值 |
| --- | --- |
| `候选` | `0` |
| `试用` | `1` |
| `激活` | `2` |
| `冷却` | `3` |
| `归档` | `4` |
| `弃用` | `5` |

## 14. 创建条件节点

### 14.1 `创建条件节点(...)`

用途：

- 在方法首节点下创建或复用一个方法条件节点。

输入：

- 方法首节点或其子节点。
- 条件场景模板。
- 条件判定二次特征索引。
- 条件模板主键。
- 时间起止。

执行流程：

1. 归一到方法首节点。
2. 如果未传条件场景模板，则使用方法首节点主信息里的条件场景。
3. 如果条件模板主键为空且有条件场景模板，则使用条件场景主键。
4. 如果未传条件索引，则使用方法首节点已有条件判定索引。
5. 如果仍为空且有条件场景，则取条件场景的二次特征索引。
6. 确保二次特征列表具备默认值。
7. 查找已有条件节点。
8. 命中则补齐场景、主键、索引和时间范围。
9. 未命中则创建新的 `方法条件节点主信息类` 子节点。

复用判断：

- 条件主键或条件场景模板主键一致。
- 或条件场景引用一致。
- 或条件判定索引集合一致。

新节点继承：

- 动作名。
- 动作句柄。
- 来源。
- 作用域。
- 主结果特征类型。
- 结果初始状态。
- 条件场景。
- 条件判定索引。
- 父方法引用。

注意：

- 这个函数只创建条件节点。
- 它不会自动创建结果节点。

## 15. 创建结果节点

### 15.1 `创建结果节点(...)`

用途：

- 在指定父节点下创建或复用一个方法结果节点。

输入：

- 父节点。
- 结果场景模板。
- 结果方向二次特征索引。

执行流程：

1. 检查父节点。
2. 归一到方法首节点。
3. 如果未传结果场景模板，则使用方法首节点主信息里的结果场景。
4. 如果未传结果方向索引且有结果场景，则取结果场景的二次特征索引。
5. 在传入父节点的直接子节点中查找已有结果节点。
6. 命中则补齐结果场景、结果主键、结果方向索引。
7. 未命中则创建新的 `方法结果节点主信息类` 子节点。

复用判断：

- 结果主键一致。
- 或结果场景引用一致。
- 或结果方向索引集合一致。

新节点继承：

- 动作名。
- 动作句柄。
- 来源。
- 作用域。
- 主结果特征类型。
- 结果初始状态。
- 结果场景。
- 结果方向索引。
- 父方法引用。

注意：

- 结果节点创建在传入父节点下面。
- 如果传入父节点是方法首节点，则结果节点是方法首节点直接子节点。
- 如果传入父节点是条件节点，也可以成为条件节点下的结果节点。
- 当前本能补齐流程把结果节点挂在方法首节点下。

## 16. 结果方向索引语义

需求方向方法头和本能方法结果声明都会使用结果方向索引。

### 16.1 二次特征比较结果编码

内部逻辑会从二次特征主信息中提取比较结果：

1. 优先使用 `方向区间`。
2. 其次使用 `差值` 推导方向区间。
3. 如果值形态是枚举，且离散编码在 `[-1, 1]`，使用离散编码。
4. 如果值形态是布尔，满足为 `1`，不满足为 `0`。
5. 其他情况返回 `2`，表示未解析。

### 16.2 结果方向去重

结果方向去重不是按实时主键做简单判断，而是按：

```text
特征类型主键 + 比较结果编码
```

判断语义是否相同。

这保证同一目标特征、同一结果方向不会因为二次特征节点主键不同而重复建方法头。

## 17. 与其它模块的关系

### 17.1 与 `存在类`

宿主存在拥有方法根节点。

`方法类::查找或创建_本能方法首节点` 会：

- 从宿主存在取得方法根节点。
- 在该方法根节点下注册本能方法。

### 17.2 与 `世界树类`

方法类通过全局 `世界树` 完成：

- 创建方法虚拟存在。
- 创建条件场景和结果场景。
- 读写方法虚拟存在特征。
- 创建内部状态。
- 创建和读取二次特征。

### 17.3 与 `特征类`

方法节点不是特征宿主。

方法需要特征时：

- 方法本身的状态特征挂到方法虚拟存在。
- 条件和结果模板特征挂到条件场景、结果场景或对应存在上。
- 不应把特征直接挂到方法节点、任务节点或线程节点上。

### 17.4 与 `本能方法类`

`本能方法类` 提供本能方法元信息：

- 本能方法 ID。
- 分类。
- 动作名称。
- 说明。
- 是否允许自我线程被动触发。
- 是否允许任务方法调用。
- 是否允许学习兜底调用。

`方法类` 根据这些元信息创建本能方法首节点，并补齐条件和结果结构。

### 17.5 与任务筹办

任务筹办会根据需求目标查找或创建方法：

- 目标特征类型用于注册需求位方法头。
- 目标方向二次特征用于注册需求方向方法头。
- 找到的方法首节点会绑定到任务的候选方法或可用方法字段。

### 17.6 与自检线程

自检线程会检查方法结构：

- 方法是否有方法虚拟存在。
- 条件节点数量。
- 结果节点数量。
- 方法结构是否完整。

如果发现问题，应提交需求信息或记录事件，而不是直接在自检里修复。

## 18. 使用示例

### 18.1 创建或复用本能方法

```cpp
auto* 方法 = 方法类::查找或创建_本能方法首节点(
    自我存在,
    枚举_本能方法ID::自我_任务筹办);
```

效果：

- 从自我存在的方法根节点中查找或创建方法首节点。
- 写入本能动作句柄。
- 初始化方法虚拟存在。
- 补齐条件场景、结果场景、条件节点和结果节点。

### 18.2 创建需求方向方法头

```cpp
auto* 方法 = 方法类::注册需求方向方法首节点(
    方法根节点,
    目标特征类型,
    结果方向二次特征,
    枚举_方法来源::运行时学习沉淀);
```

效果：

- 根据目标特征类型和方向查找已有方法头。
- 找不到则创建新的需求方向方法头。
- 写入结果方向索引。
- 初始化方法虚拟存在。

### 18.3 写入方法可用特征

```cpp
方法类::记录方法虚拟存在特征(
    方法,
    语素集.添加词性词("方法是否可用", "名词"),
    特征值{ static_cast<I64>(1) });
```

效果：

- 确保方法虚拟存在。
- 在方法虚拟存在上写入 `方法是否可用 = 1`。

### 18.4 创建条件节点

```cpp
auto* 条件节点 = 方法类::创建条件节点(
    方法,
    条件场景,
    条件判定索引,
    条件场景->获取主键());
```

效果：

- 根据条件场景或条件索引复用已有条件节点。
- 找不到则创建新的方法条件节点。

### 18.5 创建结果节点

```cpp
auto* 结果节点 = 方法类::创建结果节点(
    方法,
    结果场景,
    结果方向索引);
```

效果：

- 根据结果场景或结果方向索引复用已有结果节点。
- 找不到则创建新的方法结果节点。

## 19. 重要注意事项

- 方法树节点不是基础信息节点，不能直接作为特征宿主。
- 方法自身的特征必须通过方法虚拟存在表达。
- 条件场景和结果场景是世界树场景节点，负责承载条件、结果相关事实和二次特征。
- 方法头出生不等于方法可用。
- 方法可用性当前通过方法虚拟存在上的 `方法是否可用` 等特征表达。
- 本能方法补齐流程会尝试同时创建条件节点和结果节点，但底层 `创建条件节点` 与 `创建结果节点` 是独立函数。
- 当前 `补齐本能方法首节点` 的最终成功条件是有动作骨架且条件判定索引非空，不把结果节点数量作为返回条件。
- 需求方向方法头按“特征类型 + 比较结果”语义去重，不按二次特征实时主键去重。
- 不应在方法类继续增加具体方法特征读写接口。
