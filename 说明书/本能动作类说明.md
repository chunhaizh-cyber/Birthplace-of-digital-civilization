# 本能动作类说明

本文说明 `D:\鱼巢\本能动作类.ixx`。

`本能动作类` 是本能动作的基础工具类，当前主要提供两类能力：方法条件参数补齐，以及输入场景与模式场景的结构匹配。它不保存动作注册表，也不直接执行具体本能函数。

## 1. 文件定位

模块名：

```cpp
export module 本能动作模块;
```

公开类：

```cpp
class 本能动作类
```

当前职责：

- 为方法首节点补齐条件场景中需要的参数特征类型。
- 对输入场景和模式场景做递归模式匹配。
- 提供最小回执诊断拼接工具。

## 2. 公开数据结构

### 2.1 `结构体_本能回执最小字段`

用途：

- 表示早期或兼容场景下的最小本能执行回执。

字段：

| 字段 | 用途 |
| --- | --- |
| `成功` | 动作是否成功。 |
| `错误码` | 失败或特殊返回码。 |
| `诊断` | 文本诊断信息。 |
| `耗时_us` | 执行耗时。 |
| `时间_系统` | 系统时间戳。 |

### 2.2 `结构体_场景模式匹配参数`

用途：

- 控制场景模式匹配的容忍度和差值计算。

字段：

| 字段 | 默认值 | 用途 |
| --- | --- | --- |
| `I64容忍误差` | `0` | I64 值匹配时允许的误差。 |
| `歧义分差阈值` | `50` | 前两名候选分差小于等于该值时标记歧义。 |
| `差值算法` | 空 | 可选自定义差值算法。 |

### 2.3 `结构体_场景模式匹配绑定`

用途：

- 表示一个模式节点匹配到了哪个输入节点。

字段：

| 字段 | 用途 |
| --- | --- |
| `模式节点` | 模式场景中的节点。 |
| `输入节点` | 输入场景中被选中的节点。 |
| `匹配分` | 本次绑定得分。 |

### 2.4 `结构体_场景模式匹配结果`

用途：

- 汇总一次场景模式匹配结果。

字段：

| 字段 | 用途 |
| --- | --- |
| `匹配成功` | 是否整体匹配成功。 |
| `有歧义` | 是否存在候选分差过小的歧义。 |
| `总分` | 递归匹配累计得分。 |
| `绑定表` | 模式节点到输入节点的绑定列表。 |
| `未满足模式节点` | 未匹配成功的模式节点列表。 |

## 3. 公开方法

### 3.1 `回执_拼接诊断(...)`

用途：

- 将新的诊断文本追加到已有诊断字符串。

规则：

- 追加文本为空时不做事。
- 原字符串非空且末尾不是 `;` 时，先补 `;`。
- 再追加新文本。

适用场景：

- 多阶段本能动作生成诊断信息。
- 保持诊断字段是单个字符串。

### 3.2 `方法首节点_确保条件参数类型(基础信息类&, 方法首节点, 需要类型表)`

用途：

- 确保方法首节点的条件场景下存在指定类型的特征节点。

输入：

- `基础信息`：基础信息服务。
- `方法首节点`：目标方法首节点。
- `需要类型表`：需要出现的特征类型列表。

流程：

1. 检查方法首节点存在。
2. 检查节点种类必须是 `方法首节点`。
3. 读取方法主信息中的条件场景。
4. 对每个需要类型，检查条件场景子节点中是否已有同类型特征。
5. 缺少时创建新的特征节点并挂到条件场景下。
6. 类型主键非空时，将新节点主键设为 `cond_` 加类型主键。

返回：

- 本次追加了至少一个特征节点时返回 `true`。
- 无需追加或失败时返回 `false`。

注意：

- 该方法会修改条件场景。
- 它只补特征类型，不写特征值。

### 3.3 `方法首节点_确保条件参数类型(方法首节点, 需要类型表)`

用途：

- 使用全局 `世界树.基础信息()` 调用上一重载。

适用场景：

- 调用方不需要显式传入基础信息服务时使用。

### 3.4 `场景_模式匹配(...)`

用途：

- 将输入场景与模式场景递归匹配，输出绑定关系、总分和歧义标记。

输入：

- `输入场景根`：待匹配的输入场景。
- `模式场景根`：模式场景。
- `参数`：匹配参数。

返回：

- `结构体_场景模式匹配结果`。

基本规则：

- 模式节点没有子节点时，当前节点匹配成功。
- 普通模式父节点要求所有模式子节点都能在输入父节点下找到候选。
- OR 组表示任一子模式成功即可。
- 模式特征节点有值时会参与值匹配评分。
- 模式特征节点无值但有子节点时，被视为 OR 组。

## 4. 匹配算法说明

### 4.1 差值计算

默认差值规则：

- 如果调用方提供 `差值算法` 且返回非负值，则使用该值。
- I64 对 I64 使用饱和绝对差。
- 其他特征值完全相等时差值为 `0`。
- 不相等时差值为最大 I64，表示不可匹配。

### 4.2 值匹配分

规则：

- 模式节点不是特征节点时得分 `0`。
- 模式特征没有值时得分 `100`。
- 模式值和输入值可比较且在 I64 容忍误差内时得分 `1000`。
- 否则得分为 `max(0, 1000 - min(1000, 差值))`。

### 4.3 候选挑选

对每个模式子节点：

1. 在输入父节点子链中寻找类型相同的候选。
2. 每个候选基础分为 `1000`，再叠加值匹配分。
3. 按分数降序排列。
4. 分数相同则按主键升序稳定选择。
5. 最优候选写入绑定表。
6. 前两名分差小于等于 `歧义分差阈值` 时标记歧义。

## 5. 职责边界

`本能动作类` 应负责：

- 本能动作执行前的轻量条件参数补齐。
- 场景结构和特征值的匹配工具。
- 本能回执诊断字符串拼接。

`本能动作类` 不应负责：

- 保存本能动作注册表。
- 直接调度本能动作函数。
- 修改任务状态。
- 创建需求。
- 解释方法虚拟存在的运行语义。
