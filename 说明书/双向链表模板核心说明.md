# 双向链表模板核心说明

本文说明 `D:\鱼巢\双向链表模板核心.h` 中 `链表模板<主信息类型>` 的结构、约定和每个函数的用途。

## 1. 文件定位

`双向链表模板核心.h` 是项目中多棵树的底层容器模板。它同时维护三种关系：

- 同层环：节点的 `上/下` 指针组成一个环，表示同一父节点下的一组兄弟节点。
- 父子树：节点的 `父/子` 指针表示树形结构。
- 全链环：节点的 `链上/链下` 指针把整棵树的全部节点串成一个环，便于全局扫描。

模板只管理节点对象本身。若 `主信息类型` 是指针类型，模板不会自动释放主信息指针指向的对象。

## 2. 使用约定

- `根指针` 是哨兵根节点，不承载业务数据。
- 每个普通节点都有唯一 `主键`，主键由根节点保存的最新主键递增生成。
- 对外公开函数默认加锁。
- 名称带 `_已加锁` 的函数不自行加锁，调用方必须已经持有合适的锁。
- 写入结构时使用写锁，扫描和查询时使用读锁。
- 同一个节点不能重复挂入多棵树或多条链，添加节点时会先初始化为孤立节点。

## 3. 基本用法

```cpp
链表模板<我的主信息> 树;

auto* 根 = 树.根指针;
auto* A = 树.添加子节点(根, 我的主信息{});
auto* B = 树.添加节点(A, 我的主信息{});

auto 主键 = 树.获取主键(A);
auto* 查到 = 树.查找主键(主键);

树.修改节点信息(A, 我的主信息{});
树.删除节点(B);

bool 结构正常 = 树.校验结构();
```

批量操作时可以自己持锁，然后调用 `_已加锁` 版本：

```cpp
auto lock = 树.获取锁();
auto* A = 树.添加子节点_已加锁(树.根指针, 我的主信息{});
auto* B = 树.添加节点_已加锁(A, 我的主信息{});
```

## 4. 节点类

`节点类` 是链表模板内部的节点类型。每个节点同时参与同层环、父子树和全链环。

### 4.1 字段

| 字段 | 用途 |
| --- | --- |
| `上/下` | 同层兄弟环的前后指针。 |
| `父/子` | 树形父子关系。`子` 指向子链环头。 |
| `同层环头` | 当前同层环的头节点。不是整棵树根。 |
| `链上/链下` | 全链环前后指针，用于遍历整棵树全部节点。 |
| `主信息` | 业务数据。类型由 `主信息类型` 决定。 |
| `子节点数量` | 直接子节点数量。 |
| `主键` | 私有字段，通过 `获取主键/设置主键` 访问。 |

### 4.2 节点函数

| 函数 | 用途 | 用法和注意 |
| --- | --- | --- |
| `变更子节点数量(std::int64_t delta)` | 增减直接子节点计数。 | 添加或删除子节点时由模板内部调用。外部一般不直接改。 |
| `获取主键() const` | 返回节点主键。 | 主键为空通常表示节点尚未正式挂入树。 |
| `设置主键(const std::string& 主键_)` | 设置节点主键。 | 主要给模板内部、反序列化或重建逻辑使用。普通添加节点会自动生成主键。 |
| `是否叶子() const` | 判断节点是否没有子节点。 | 等价于 `子 == nullptr`。 |
| `是否同层孤立() const` | 判断节点是否是单节点同层环。 | 要求 `上 == this && 下 == this`。 |
| `初始化同层自环()` | 初始化 `上/下/同层环头` 为自身。 | 用于把节点变成独立同层环。 |
| `初始化全链自环()` | 初始化 `链上/链下` 为自身。 | 用于把节点变成独立全链环。 |
| `初始化为孤立节点()` | 清空父子关系、子计数，并初始化两类自环。 | 添加节点前会调用，避免旧指针污染新结构。 |
| `从同层环摘除()` | 从当前同层环中移除节点。 | 摘除后节点变成同层自环。不会处理父节点的 `子` 指针。 |
| `插入到同层节点后(节点类* 位置)` | 把当前节点插到 `位置` 后面。 | `位置` 必须已在同层环中。函数会继承位置节点的 `同层环头`。 |
| `从全链环摘除()` | 从全链环中移除节点。 | 摘除后节点变成全链自环。 |
| `插入到全链尾(节点类* 全局根)` | 把当前节点插到全链环尾部。 | `全局根` 通常是 `根指针`。插入位置是 `全局根->链上` 与 `全局根` 之间。 |

## 5. 类型别名和锁

| 名称 | 用途 |
| --- | --- |
| `写锁守卫` | `std::unique_lock<std::shared_mutex>`，用于写操作。 |
| `读锁守卫` | `std::shared_lock<std::shared_mutex>`，用于读操作。 |
| `锁守卫` | 当前等同于写锁守卫，兼容旧代码。 |
| `链表锁` | 模板实例的共享互斥锁。 |
| `根指针` | 哨兵根节点。构造函数创建，析构函数删除。 |

## 9. 添加节点

添加节点会同时完成三件事：初始化节点、写入同层关系、登记全链关系。

| 函数 | 用途 | 用法和注意 |
| --- | --- | --- |
| `添加节点_已加锁(节点类* 位置节点, 节点类* 新节点)` | 把 `新节点` 插到 `位置节点` 的同层后面。 | `位置节点` 为空时使用 `根指针`。新节点的父节点设置为 `位置节点->父`。 |
| `添加节点(节点类* 位置节点, 节点类* 新节点)` | 加锁版本的同层插入。 | 外部常用。函数内部获取写锁。 |
| `添加节点_已加锁(节点类* 位置节点, 主信息类型 主信息)` | 根据主信息创建新节点，再插到同层。 | 调用方持有写锁。新节点由模板分配并拥有。 |
| `添加节点(节点类* 位置节点, 主信息类型 主信息)` | 加锁版本，根据主信息创建并插入同层节点。 | 外部常用。 |

示例：

```cpp
auto* 第二个兄弟 = 树.添加节点(第一个兄弟, 主信息);
```

## 10. 添加子节点

添加子节点会把新节点挂到父节点的子链中，并登记到全链环。

| 函数 | 用途 | 用法和注意 |
| --- | --- | --- |
| `添加子节点_已加锁(节点类* 父节点, 节点类* 子节点)` | 将节点挂为父节点的子节点。 | 父节点或子节点为空时返回 `nullptr`。如果父节点还没有子节点，新节点成为子链环头。 |
| `添加子节点(节点类* 父节点, 节点类* 子节点)` | 加锁版本的挂子节点。 | 外部常用。 |
| `添加子节点_已加锁(节点类* 父节点, 主信息类型 主信息)` | 根据主信息创建新节点，并挂到父节点子链。 | 调用方持有写锁。 |
| `添加子节点(节点类* 父节点, 主信息类型 主信息)` | 加锁版本，根据主信息创建子节点。 | 外部常用。 |

示例：

```cpp
auto* 子 = 树.添加子节点(父, 主信息);
```

## 11. 查找函数

查找分两类：全链扫描和当前同层链扫描。

| 函数 | 用途 | 用法和注意 |
| --- | --- | --- |
| `查找主键_已加锁(const std::string& 主键) const` | 在全链环中按主键查找节点。 | 扫描整棵树，不包含根节点。调用方应持有读锁或写锁。 |
| `查找主键(const std::string& 主键) const` | 加锁版本的全链主键查找。 | 外部常用。 |
| `查找主键_当前链_已加锁(节点类* 当前节点, const std::string& 主键) const` | 在当前节点所在同层环中按主键查找。 | 如果当前同层环头是根节点，则扫描根同层链。 |
| `查找主键_当前链(节点类* 当前节点, const std::string& 主键) const` | 加锁版本的当前链主键查找。 | 外部常用。 |
| `查找节点_当前链_已加锁(节点类* 当前节点, 主信息类型 查找内容, 枚举_比较字段 比较字段) const` | 在当前同层链中按主信息比较查找。 | `主信息类型` 必须提供 `比较(查找内容, 比较字段)`，指针类型则要求 `主信息->比较(...)`。 |

示例：

## 13. 删除函数

删除节点会从同层环、全链环和父子关系中移除节点。若节点有子链，会递归删除子链。

| 函数 | 用途 | 用法和注意 |
| --- | --- | --- |
| `删除节点_已加锁(节点类* 被删除节点)` | 删除一个节点及其全部子节点。 | 不能删除根节点。调用方应持有写锁。 |
| `删除节点(节点类* 被删除节点)` | 加锁版本的节点删除。 | 外部常用。 |
| `删除子链_已加锁(节点类* 父节点)` | 删除父节点的全部直接和间接子节点。 | 调用后父节点没有子节点。调用方应持有写锁。 |
| `删除子链(节点类* 父节点)` | 加锁版本的子链删除。 | 外部常用。 |
| `删除链表_已加锁()` | 删除整棵树，包括根节点。 | 删除后 `根指针 = nullptr`。调用方应持有写锁。 |
| `删除链表()` | 加锁版本的整树删除。 | 析构函数会调用。手动调用后不应再继续使用该实例，除非重新初始化根指针。 |

## 14. 结构校验

| 函数 | 用途 | 用法和注意 |
| --- | --- | --- |
| `校验结构() const` | 加读锁检查链表结构是否自洽。 | 外部常用。适合调试和自检。 |
| `校验结构_已加锁() const` | 不加锁版本的结构校验。 | 检查根节点、全链环、同层环、父子关系和 `子节点数量`。 |

校验会检查：

- 根节点同层环和全链环是否闭合。
- 每个节点的 `链上/链下` 是否互相指回。
- 每个节点的 `上/下` 是否互相指回。
- 同层环中每个节点的 `同层环头` 是否一致。
- 子节点的 `父` 是否指回父节点。
- 实际子节点数量是否等于 `子节点数量`。

## 17. 常见错误

- 在未持锁时调用 `_已加锁` 函数，会产生并发读写风险。
- 把同一个节点重复添加到不同位置，会破坏旧结构。添加前模板会初始化节点，因此旧挂接会丢失。
- 直接修改 `父/子/上/下/链上/链下` 容易破坏三套关系，应优先使用模板函数。
- 对指针型主信息调用 `修改节点信息` 后，旧主信息对象不会被释放，需要外部明确管理生命周期。
- 删除节点后，外部保存的节点指针立即失效。
- `查找节点_当前链` 要求主信息类型有 `比较(...)` 接口，否则编译期报错。

## 18. 推荐使用顺序

1. 普通业务新增节点使用 `添加子节点` 或 `添加节点`。
2. 需要同一父节点下查找时使用 `查找主键_当前链` 或 `查找节点_当前链`。
3. 需要全局查找时使用 `查找主键`。
4. 批量操作时先 `获取锁()`，再调用 `_已加锁` 版本。
5. 调试、自检、数据恢复后使用 `校验结构()`。
