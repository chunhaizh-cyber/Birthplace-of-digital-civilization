# 状态类说明

本文说明 `D:\鱼巢\状态类.h`、`D:\鱼巢\状态类.cpp`，并补充 `D:\鱼巢\基础信息主信息类.h` 中 `状态节点主信息类` 与本类直接相关的字段。

`状态类` 是基础信息树中“状态节点”的服务层。状态节点用于记录：在某个场景里，某个主体的某个特征，在某个时间点具有某个值。

## 1. 文件定位

当前项目中，状态节点是连接场景、主体、特征和值的时序事实：

- 状态节点通常挂在场景节点下面。
- 场景主信息通过 `状态索引` 保存状态引用，并按时间排序。
- 状态主体可以是存在、场景、特征，或内部任务/方法等节点。
- 状态特征必须是特征节点。
- 状态值是 `特征值`，可以是 `I64`、`VecU句柄`、`指针句柄`。
- 状态节点本身不表示特征宿主；特征仍然只能挂在存在或场景下面，任务/方法/线程相关状态应通过虚拟存在上的特征表达。

简单说：

- 特征节点表示“某主体有哪些可读写特征”。
- 状态节点表示“某时刻某特征是什么值”。
- 场景节点通过状态索引组织这些时序状态。

## 2. 状态域与事件

### 2.1 `枚举_状态域`

`枚举_状态域` 定义在 `基础信息主信息类.h`。

| 枚举值 | 值 | 含义 |
| --- | --- | --- |
| `未定义` | `0` | 未确定状态域。 |
| `内部状态` | `1` | 内部运行状态，通常主体不是存在/场景/特征。 |
| `世界状态` | `2` | 世界可观察状态，主体是存在、场景或特征。 |

`状态类::私有_推断状态域` 的规则：

- 主体为空或没有主信息：`未定义`。
- 主体主信息是存在、场景或特征：`世界状态`。
- 其他主体：`内部状态`。

### 2.2 `枚举_存在状态事件`

定义在 `状态类.h`。

| 枚举值 | 值 | 含义 |
| --- | --- | --- |
| `未定义` | `0` | 未定义事件。 |
| `创建` | `1` | 创建事件。 |
| `变化` | `2` | 变化事件。 |

注意：

- `状态创建参数` 里有 `事件` 字段。
- 当前 `状态节点主信息类` 没有对应事件字段。
- 因此事件当前只影响部分创建入口的控制逻辑，不会保存到状态节点主信息里。

## 3. 状态节点主信息

`状态节点主信息类` 定义在 `基础信息主信息类.h`。

### 3.1 字段

| 字段 | 用途 |
| --- | --- |
| `状态域` | 标记内部状态或世界状态。 |
| `收到时间` | 系统接收到该状态的时间。 |
| `发生时间` | 状态实际发生时间。 |
| `状态主体` | 状态描述的主体引用。 |
| `状态特征` | 状态描述的特征引用。 |
| `状态值` | 该状态下的特征值。 |
| `对应信息节点` | 该状态对应的信息节点，优先是特征，缺省时是主体。 |
| `是否变化` | 该状态是否表示变化。 |

### 3.2 `比较`

`状态节点主信息类::比较` 当前只判断对象是否也是状态主信息：

- 右侧也是 `状态节点主信息类`：返回 `0`。
- 否则返回 `-1`。

状态值、主体、特征和时间不在这个 `比较` 函数中比较。

## 4. 创建参数和结果结构

### 4.1 `状态创建参数`

用于通过统一入口创建状态。

| 字段 | 用途 |
| --- | --- |
| `状态域` | 可显式指定状态域；为 `未定义` 时由主体推断。 |
| `事件` | 创建侧事件，当前不会写入状态主信息。 |
| `状态主体` | 状态主体节点。 |
| `状态特征` | 状态特征节点。 |
| `状态值` | 状态值。 |
| `对应信息节点` | 可显式指定对应信息节点。 |
| `是否变化` | 写入状态主信息的变化标记。 |
| `now` | 创建时间。为 `0` 时使用当前微秒时间。 |

注意：

- 使用该参数创建状态时，发生时间和收到时间都会使用同一个 `now`。
- 如果需要分别设置发生时间和收到时间，应使用基础重载 `创建状态(场景, 主体, 特征, 状态值, 发生时间, 收到时间)`。

### 4.2 `结构_最近两次I64状态方向结果`

用于比较场景中同一主体、同一特征最近两次 I64 状态的方向。

| 字段 | 用途 |
| --- | --- |
| `可比较` | 是否找到最近两条可比较 I64 状态。 |
| `方向` | I64 比较值，等于 `最新值 - 上次值`；判断方向时只和 `0` 比较。 |
| `最新值` | 最新状态值。 |
| `上次值` | 上一次状态值。 |
| `最新时间` | 最新状态排序时间。 |
| `上次时间` | 上一次状态排序时间。 |

## 5. `状态类` 的职责

`状态类` 主要做六件事：

1. 绑定基础信息树。
2. 生成和验证状态主信息。
3. 在场景下创建状态节点。
4. 同步维护场景主信息的 `状态索引`。
5. 按时间读取场景状态并比较最近两次 I64 方向。
6. 修改状态主体、特征、值和变化标记。

私有状态：

| 字段 | 用途 |
| --- | --- |
| `基础信息_` | 指向基础信息树。为空时大多数操作返回空或失败。 |

## 8. 状态生成与有效性

### 8.1 `生成状态(...) const`

签名：

```cpp
状态节点主信息类 生成状态(
    基础信息节点类* 主体,
    特征节点类* 特征,
    特征值 状态值,
    时间戳 发生时间,
    时间戳 收到时间) const;
```

用途：

- 根据主体、特征、值和时间生成一个状态主信息对象，但不挂入基础信息树。

行为：

- `状态主体` 保存主体引用。
- `状态特征` 保存特征引用。
- `状态值` 保存传入值。
- `发生时间`、`收到时间` 保存传入时间。
- `状态域` 根据主体推断。
- `对应信息节点` 优先指向特征；没有特征时指向主体。

### 8.2 `状态有效检查(const 状态节点主信息类& 主信息) const noexcept`

用途：

- 判断状态主信息是否足够构成有效状态。

有效条件：

- `状态主体` 有效。
- `状态特征` 有效。
- `状态值` 不是 `std::monostate`。
- 如果状态值是 `指针句柄`，指针句柄必须有效。
- 如果状态值是 `VecU句柄`，VecU 句柄必须有效。
- 如果状态值是 `I64`，通过有效性检查。

注意：

- 当前状态必须有状态特征；缺少特征会被判为无效。
- 这符合“状态描述某主体某特征在某时刻的值”的模型。

### 8.3 `获取状态二次特征(...) const noexcept`

用途：

- 对两个状态的 I64 值做状态比较。

返回：

- 任一状态无效：`0`。
- 任一状态值不是 `I64`：`0`。
- 返回 `第二个状态值 - 第一个状态值`。

注意：

- 返回值是 `I64` 比较值，不压缩成 `-1/0/1`。
- 需要判断方向时，用返回值和 `0` 比较：大于 `0` 表示增加，小于 `0` 表示减少，等于 `0` 表示不变。

## 9. 创建状态

### 9.1 `创建状态(场景节点类* 场景, 状态节点主信息类* 主信息)`

用途：

- 使用调用方准备好的主信息创建状态节点，并挂到场景下面。

流程：

1. `基础信息_` 为空或 `主信息` 为空时返回 `nullptr`。
2. 如果 `状态域` 是 `未定义`，根据状态主体推断状态域。
3. 如果 `对应信息节点` 无效，优先使用状态特征，否则使用状态主体。
4. 调用 `状态有效检查`。
5. 无效时删除传入主信息并返回 `nullptr`。
6. 记录一次观测统计，时间优先用 `收到时间`，否则用当前微秒时间。
7. 调用 `基础信息_->添加子节点(场景, 主信息)` 创建节点。
8. 如果父场景有场景主信息，把状态节点按时间追加到 `状态索引`。

注意：

- 调用方应传入有效场景。
- 当前实现不强制检查 `场景` 非空；如果场景为空，底层可能挂到世界根，这不符合正常状态建模。
- 创建成功后，主信息生命周期交给基础信息树。

### 9.2 `创建状态(场景, 状态创建参数, 后处理, 调用点)`

用途：

- 使用统一参数结构创建状态，并在成功后执行可选后处理。

流程：

1. 取 `now`：参数 `now != 0` 时使用参数值，否则使用当前微秒时间。
2. 调用 `生成状态`，发生时间和收到时间都使用 `now`。
3. 如果参数显式指定 `状态域`，覆盖推断状态域。
4. 如果参数指定 `对应信息节点`，覆盖默认对应信息节点。
5. 写入 `是否变化`。
6. 转成堆上主信息，调用基础创建函数。
7. 创建成功且有后处理回调时，调用后处理。

后处理签名：

```cpp
void(场景节点类* 场景, 状态节点类* 状态节点, 时间戳 now, const std::string& 调用点)
```

注意：

- `调用点` 当前只传给后处理回调。
- `事件` 参数当前不会写入状态主信息。

### 9.3 `创建状态(场景, 主体, 特征, 状态值, 发生时间, 收到时间)`

用途：

- 直接按主体、特征、值和两个时间创建状态。

行为：

- 创建新主信息。
- 写入主体、特征、状态值、发生时间、收到时间。
- 根据主体推断状态域。
- 对应信息节点优先为特征，否则为主体。
- 转调基础创建函数。

- 强制 `状态域 = 世界状态`。

### 10.5 `记录世界状态到场景(...)`

用途：

### 10.6 `记录内部特征状态(...)`

用途：

- 只在“创建事件”或“发生变化”时记录内部特征状态。

规则：

- 如果 `事件 != 创建` 且 `是否变化 == false`，返回 `nullptr`，不创建状态。

典型含义：

- 普通未变化的内部特征不重复记账。
- 创建事件即使 `是否变化=false` 也会记录。

## 11. 删除和枚举

### 11.1 `删除状态(状态节点类* 节点)`

用途：

- 删除状态节点，并从父场景的 `状态索引` 中移除引用。

流程：

1. `基础信息_` 为空或节点为空时返回 `false`。
2. 取得节点父节点。
3. 如果父节点主信息是场景主信息，从 `状态索引` 中删除同指针或同主键引用。
4. 调用 `基础信息_->删除节点(节点)`。

注意：

- 如果状态节点没有挂在场景下，则不会从任何场景索引中移除。
- 如果其他对象保存状态节点指针，删除后指针失效。

### 11.2 `枚举全部状态() const`

用途：

- 从基础信息树全链枚举所有状态节点。

返回：

- `基础信息_` 为空时返回空数组。
- 否则返回所有主信息为 `状态节点主信息类` 的节点。

### 11.3 `获取场景状态(const 场景节点类* 场景) const`

用途：

- 从场景主信息的 `状态索引` 中取出状态节点列表，并按时间排序。

行为：

- 场景主信息无效时返回空数组。
- 只使用索引项中的指针；不会按主键重新解析丢失指针。
- 返回前再次稳定排序。

排序规则：

- 优先按状态排序时间升序。
- 排序时间为 `发生时间 != 0 ? 发生时间 : 收到时间`。
- 时间相同时按状态节点主键升序。

## 12. 最近两次方向比较

### 12.1 `比较最近两次I64状态方向(...) const`

签名：

```cpp
结构_最近两次I64状态方向结果 比较最近两次I64状态方向(
    场景节点类* 场景,
    枚举_状态域 状态域,
    基础信息节点类* 状态主体,
    特征节点类* 状态特征) const;
```

用途：

- 在场景状态列表中，找到指定状态域、主体、特征的最近两条 I64 状态，并比较方向。

流程：

1. 主体或特征为空时返回默认结果。
2. 调用 `获取场景状态` 获取按时间排序的状态列表。
3. 从后向前遍历。
4. 过滤状态域不匹配的状态。
5. 过滤主体引用不匹配的状态。
6. 过滤特征引用不匹配的状态。
7. 过滤状态值不是 `I64` 的状态。
8. 找到第一条作为最新状态。
9. 找到第二条作为上次状态。
10. 如果两条都存在，计算 `最新值 - 上次值` 并填充结果。

匹配规则：

- 指针相同算匹配。
- 指针不同但主键相同也算匹配。

方向规则：

- `方向 > 0`：最新值大于上次值。
- `方向 < 0`：最新值小于上次值。
- `方向 == 0`：两次值相等。

## 13. 修改状态

### 13.1 `绑定主体特征(状态节点类* 节点, 基础信息节点类* 主体, 特征节点类* 特征)`

用途：

- 修改状态节点的主体和特征。

行为：

- 主信息不存在时返回 `false`。
- 写入 `状态主体` 和 `状态特征`。
- 根据主体重新推断 `状态域`。
- 如果 `对应信息节点` 还无效，优先写入特征，否则写入主体。

注意：

- 该函数不重新执行 `状态有效检查`。
- 该函数不会重排场景状态索引。

### 13.2 `写入状态值(状态节点类* 节点, const 特征值& 值, 时间戳 发生时间 = 0, 时间戳 收到时间 = 0)`

用途：

- 修改状态节点的状态值和可选时间。

行为：

- 主信息不存在时返回 `false`。
- 写入 `状态值`。
- `发生时间 != 0` 时更新发生时间。
- `收到时间 != 0` 时更新收到时间。
- 记录一次观测统计，时间优先用传入 `收到时间`，否则用当前微秒时间。

注意：

- 该函数不重新执行 `状态有效检查`。
- 更新时间后不会自动重排场景状态索引。

### 13.3 `标记变化(...)`

用途：

- 修改状态节点的变化标记。

行为：

- 写入 `是否变化`。

返回：

- 主信息存在时返回 `true`。
- 主信息不存在时返回 `false`。

## 15. 常见使用路径

### 15.1 创建一个世界状态

```cpp
auto* 状态 = 世界树.状态().创建世界状态(
    场景,
    存在节点,
    特征节点,
    特征值{ static_cast<I64>(1) });
```

### 15.2 创建一个内部状态

```cpp
auto* 状态 = 世界树.状态().创建内部状态(
    场景,
    任务虚拟存在,
    状态特征,
    特征值{ static_cast<I64>(2) },
    枚举_存在状态事件::变化,
    true);
```

注意：

- 任务、方法、线程相关状态应以虚拟存在作为主体。
- 状态特征仍然是挂在虚拟存在下面的特征节点。

### 15.3 只在创建或变化时记录内部特征状态

```cpp
auto* 状态 = 世界树.状态().记录内部特征状态(
    场景,
    主体,
    特征,
    值,
    枚举_存在状态事件::变化,
    是否变化);
```

### 15.4 比较最近两次 I64 状态方向

```cpp
auto 方向 = 世界树.状态().比较最近两次I64状态方向(
    场景,
    枚举_状态域::世界状态,
    存在节点,
    特征节点);
```

## 16. 与世界树、场景、动态、因果的关系

- `世界树类::创建状态` 转调 `状态类::创建状态`。
- `场景类` 也有 `追加状态`，但 `状态类::创建状态` 会在创建状态时直接维护场景 `状态索引`。
- `动态类` 会读取状态节点，建立初始状态和结果状态之间的动态。
- `因果类` 会读取状态特征和状态值，用于条件模板、结果模板和因果推断。
- `需求类` 会读取状态特征和状态值，用于判断目标方向和需求满足情况。

## 17. 注意事项

- 状态节点应挂在场景下；当前基础创建函数不强制检查场景非空，调用方必须保证。
- 状态必须有主体、特征和值，否则创建会失败。
- 状态特征是特征节点，不是特征类型词。
- 特征只能挂在存在或场景下；任务、方法、线程状态需要通过虚拟存在的特征表达。
- `状态创建参数::事件` 当前不会写入状态主信息。
- `记录内部特征状态` 会过滤非创建且未变化的状态。
- `获取场景状态` 只使用状态索引中的指针，不按主键恢复丢失指针。
- `写入状态值` 修改时间后不会自动重排场景状态索引。
- `绑定主体特征` 修改主体或特征后不会自动重新校验状态有效性。
