# 本能动作管理类说明

状态：20260430 重生成。

对应实现：`D:\鱼巢\本能动作管理类.ixx`

## 1. 职责

本能动作管理类负责本能动作函数的登记、查询、可调用检查和执行调用。

它不负责：

- 推进任务状态。
- 创建需求或派生需求。
- 结算安全值或服务值。
- 维护任务树、需求树或控制面板状态。

任务管理侧根据本能动作返回的运行结果实例和输出结果场景，统一决定任务后续状态。

## 2. 返回信息边界

本能动作函数只直接返回运行结果：

```cpp
特征节点类* 运行结果实例
```

执行结果写入调用方传入的输出结果场景。

结构结果由执行桥写入方法树条件节点和结果节点。

动作动态由执行桥或动作执行体写入动态信息。

## 3. 本能动作函数类型

```cpp
using 本能动作函数 = std::function<特征节点类*(
    方法类::节点类* 方法首节点,
    场景节点类* 输入参数场景,
    场景节点类* 输出结果场景)>;
```

参数说明：

- `方法首节点`：本次被执行的方法。
- `输入参数场景`：参数来源场景。
- `输出结果场景`：执行结果写入场景。

返回说明：

- 返回运行结果实例。
- 返回空时，调用方应视为运行结果未知。

## 4. 运行结果辅助函数

### 4.1 `本能动作运行结果语义_按码`

用途：把运行码映射为运行结果语义文本。

输入：`std::int64_t 运行码`。

输出：`const char*`。

### 4.2 `取或创建本能动作运行结果抽象特征`

用途：在抽象特征树中取或创建本能动作运行结果语义。

输入：语义文本。

输出：抽象特征节点。

### 4.3 `本能动作运行结果宿主`

用途：确定运行结果实例挂载位置。

优先顺序：

1. 方法首节点的虚拟存在。
2. 输出结果场景。

### 4.4 `创建本能动作运行结果实例`

用途：按语义和运行码创建运行结果实例。

输入：

- 方法首节点。
- 输出结果场景。
- 运行结果语义。
- I64 运行码。
- 时间戳。

输出：运行结果实例。

### 4.5 `创建本能动作运行结果实例_按码`

用途：按运行码自动选择语义并创建运行结果实例。

### 4.6 `本能动作运行结果值`

用途：读取运行结果实例中的 I64 当前值。

空实例返回 `本能动作运行结果_未知`。

### 4.7 `本能动作运行成功`

用途：判断运行结果实例是否表示运行成功。

当前规则：运行结果 I64 等于 `0`。

### 4.8 `本能动作运行结果文本`

用途：读取运行结果实例命中的抽象特征语义文本。

## 5. 运行码

核心运行码：

| 名称 | 值 | 含义 |
| --- | ---: | --- |
| `本能动作运行结果_成功` | 0 | 运行成功 |
| `本能动作运行结果_未知` | -1 | 未能取得有效运行结果 |
| `本能动作错误_未注册` | -20001 | 动作未注册 |
| `本能动作错误_执行失败` | -20002 | 动作执行失败 |
| `本能动作错误_动作异常` | -20003 | 捕获到标准异常 |
| `本能动作错误_动作未知异常` | -20004 | 捕获到未知异常 |
| `本能动作错误_缺方法首节点` | -20005 | 方法首节点为空 |
| `本能动作错误_方法节点类型错误` | -20006 | 方法节点不是方法首节点 |
| `本能动作错误_缺动作入口` | -20007 | 方法没有本能动作入口 |
| `本能动作错误_动作入口不归本类执行` | -20008 | 动作入口不属于本能动作管理类 |
| `本能动作错误_输入参数不满足` | -20009 | 输入参数或条件不满足 |

任务相关的待重入、缺条件、缺方法等旧运行码仍可作为运行结果 I64 值保留，用于任务管理侧分流。

## 6. 管理类接口

### 6.1 `注册`

用途：注册本能动作函数。

输入：

- 动作 ID。
- 本能动作函数。
- 可选名称。

输出：是否注册成功。

### 6.2 `取消注册`

用途：移除动作 ID 对应的注册函数。

### 6.3 `查找`

用途：查找动作 ID 对应的函数。

输出：函数指针包装对象。不存在时返回空。

### 6.4 `已注册`

用途：判断动作 ID 是否已经登记。

### 6.5 `已注册本能动作ID列表`

用途：返回当前已注册动作 ID 列表。

### 6.6 `补齐默认注册占位`

用途：为默认本能动作 ID 创建注册表占位。

它不创建方法树节点，不推进任务。

### 6.7 `检查是否可调用`

用途：检查本能动作是否可调用，并返回运行结果实例。

检查内容：

- 动作是否已注册。
- 方法首节点是否存在。
- 方法节点种类是否为方法首节点。
- 方法是否具有动作入口。
- 动作入口是否归本能动作管理类执行。
- 输入参数是否满足。

### 6.8 `执行功能函数`

用途：执行已注册本能动作函数。

流程：

1. 调用 `检查是否可调用`。
2. 检查失败则直接返回失败运行结果实例。
3. 检查成功后调用动作函数。
4. 动作函数返回空时返回运行结果未知。
5. 捕获异常并返回异常运行结果实例。

## 7. 调用方职责

执行桥负责：

- 创建输入和输出场景。
- 调用 `执行功能函数`。
- 接收运行结果实例。
- 维护方法条件节点和结果节点。
- 记录动作动态。

任务管理工作线程负责：

- 根据运行结果实例判断成功、待重入、缺方法、缺条件或异常。
- 根据输出结果场景判断任务执行结果。
- 生成上行消息。
- 推动任务状态变化。

自我线程负责：

- 接收任务管理侧提交的需求消息。
- 将合法需求挂入需求树。
- 触发后续任务筹办。

## 8. 维护规则

- 不再恢复 `结构_本能动作步骤结果`。
- 不在本能动作管理类中新增混合返回结构。
- 不把任务和需求字段加入本能动作函数签名。
- 运行结果使用抽象特征实例表达。
- 执行结果写入输出结果场景。
- 本能动作管理类只判断方法入口和输入参数是否满足。
