# 写代码规则

1. 需要使用日志进行调试时，只能使用`日志模块.ixx`中已有的日志功能。
2. 不能随意增加结构体。确有需要表达新差异或新行为时，优先考虑通过虚拟函数中的特征来实现。
3. 不得再增加说明字段或说明参数；已有同类字段只作为兼容层保留。
4. 新增枚举、结构体、字符串字段如果承载长期语义，必须同步设计对应的抽象特征和实例特征槽；代码枚举或结构体只能作为局部缓存、消息包、临时参数或兼容镜像。
5. 任务阶段、需求状态、缺口类型、方法结构状态、能力值、健康状态等系统语义，权威表达必须落在特征系统里，不能只停留在 C++ 字段中。
6. 旧代码中已有的候选方法指针、字符串类型名、说明字段等可以保留为迁移期兼容层，但不得作为新的权威语义入口继续扩散。
7. 新线程模块必须先明确读写边界：摘要线程和自检线程只读；自我线程负责把报告、缺口和任务调度写入主结构；工作线程只推进单个需求节点的一步。
8. 遇到结构性问题或证据不足时，可以先暂停当前实现并提出问题；不得用绕过、白名单、默认成功、兜底通过等方式遮掩真实断点。
