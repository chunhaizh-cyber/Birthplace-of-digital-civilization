# Birthplace of Digital Civilization（鱼巢）

`Birthplace of Digital Civilization（鱼巢）` 是一个以“世界树 + 自我线程 + 需求/任务/方法 + 状态/动态/因果 + 自然语言显示”为核心的 C++ 实验工程。工程内部历史名称仍保留为 `HY-Ego（鱼巢）`，当前新仓库用于同步完整代码、规范、计划和说明文档。

云端仓库：[`chunhaizh-cyber/Birthplace-of-digital-civilization`](https://github.com/chunhaizh-cyber/Birthplace-of-digital-civilization)

更完整的架构说明见 [`说明书/项目说明.md`](./说明书/项目说明.md)。

## 项目定位

这个项目不是单一算法库，也不是单一界面程序，而是在构造一个可持续运行的最小数字自我闭环：

1. 初始化世界树、自我锚点和运行环境。
2. 补齐根需求、需求对应任务与方法根。
3. 注册默认本能方法。
4. 启动自我线程，按 Tick 推进先天维护、需求树生长、方法查找和方法补齐。
5. 把运行结果回写到状态、动态、因果、任务树、方法树和虚拟存在。
6. 通过控制面板、摘要输出和日志回读当前真实状态。

当前主线重点是让“需求节点 -> 对应任务 -> 筹办 -> 查找方法 -> 方法配对结果 -> 执行 -> 配对参数 -> 可执行输入参数场景 -> 本能函数执行 -> 运行事实回写 -> 需求满足回传”这条链路逐步闭合。

## 当前能做什么

- 以 [`鱼巢.cpp`](./鱼巢.cpp) 为单一入口启动程序、自我线程和控制面板。
- 通过 [`自我模块.初始化.ixx`](./自我模块.初始化.ixx) 补齐世界根、自我、根需求、需求对应任务、方法根等核心骨架。
- 通过 [`自我线程类.h`](./自我线程类.h) 推进运行阶段、自检、先天动作、任务治理和方法补齐。
- 通过 [`本能动作管理类.ixx`](./本能动作管理类.ixx) 管理本能动作登记、调用和返回运行虚拟存在。
- 通过 [`自我动作实现模块.ixx`](./自我动作实现模块.ixx) 实现本能函数，包括 `查找方法` 与差值驱动的 `改变特征值`。
- 通过 [`控制面板类.ixx`](./控制面板类.ixx) 输出文本摘要、HTML 和 WebView2 独立窗口观察面板。
- 通过 [`规范`](./规范)、[`计划`](./计划)、[`说明书`](./说明书) 保留实现边界、路线和接手说明。

## 近期语义基线

近期主线需要把方法查找和特征改变从“名称/能力值兜底”收口到“需求结果驱动”：

- `查找方法` 的正式问题只有一个：有没有方法能把需求关心的目标特征类型推向目标方向。
- `查找方法` 不按方法名、动作名、动作句柄、本能 ID 或固定编号选择方法；这些字段只允许作为已命中方法的执行入口和调试信息。
- `查找方法` 的权威匹配依据是 `需求结果 = 目标特征类型 + 目标方向` 与 `方法能力签名中的结果包`；过渡期的 `结果能力集` 只能作为派生索引，不替代结果包。
- 找不到就是找不到：表示当前方法树没有任何方法声明自己能造成该结果变化。后续应暴露闭环缺口、本能方法缺失或本能方法结果声明缺失，不能偷偷改按名称、动作句柄或本能 ID 取方法。
- `查找方法` 只输出 `方法配对结果存在`，不直接执行方法，也不生成可执行输入参数场景；命中本能方法时可以记录 `选中自我能力值 = static_cast<I64>(枚举_本能方法ID)` 作为执行入口缓存。
- 任务虚拟存在长期保存本轮配对结果和可解析执行入口；执行阶段必须校验该入口来自结果能力命中的方法首节点。方法首节点和方法虚拟存在只能作为缓存字段。
- 需求树是运行主结构；任务只是需求节点的运行壳，每个需求节点对应一个任务。
- 需求主信息只保存目标关系，不保存值解释；值解释、集合比较、区间比较、枚举解释和二次特征解释全部交给特征类或状态比较服务。
- 单个执行需求节点只表达一个原子目标：目标主体、目标特征类型、当前状态、目标状态和满足关系；多目标需求必须展开为管理需求的直接子需求链。
- 需求树的直接子链就是目标包的结构化版本；目标包只允许作为候选展开或方法查找时的运行时视图，不作为 `需求主信息类` 的持久字段。
- 需求方向只保留三向关系：小于、等于、大于；保持等于等于，改变等于小于或大于，存在/可用/完整/已形成都是状态或特征值，不是方向。
- 同父子需求去重以目标主体、目标特征类型和方向重叠为准；派生归因只记录来源、自检归因、控制面板观察和重复触发统计，不决定需求身份。
- 任务只有两个核心阶段：`筹办` 是任务系统普通函数，负责按执行需求的目标结果查找方法；未命中时直接创建或复用带目标结果声明的候选方法头并等待后续补齐。`执行` 负责配对参数、调用方法，并在发现动作入口、参数、条件等实际缺失后输出对应原子需求。
- 任务虚拟存在必须回写 `当前阶段`、`缺口类型` 与 `选中自我能力值`；阶段事实不能只停在代码分支里。
- 旧 `任务管理任务模块` 主体已删除；任务治理入口收敛到需求树、任务实体、工作线程协议和两阶段推进。
- 学习不再作为独立任务系统，相关缺口被表达为需求树上的知识缺口节点。自我内部方法补齐闭环由 `练习`、`尝试学习`、再回到 `练习` 推进；`定向学习` 只属于外界信息或未归因现象的方向形成链。
- 自检可以产生执行缺失或候选方法补齐事实，事实可以展开为原子需求；`练习` 的核心职责是补全候选方法虚拟存在、能力包、形参包、结果包、条件结果对和试运行限制包，并用运行事实推动候选结构升格。
- `练习` 的执行口径由任务传入参数完整度决定：目标特征类型与练习方向、执行差值或目标执行后值齐备时走指定练习；参数不闭合但目标方法可练习时走探索练习，由 `练习` 自己生成未出现过的候选实参，并观察同一动作在不同输入下是否产生新用途。
- 执行阶段在调用目标方法前必须做执行入口自检；只有方法头或缺少可调用执行入口时，当前任务进入阻塞，并输出“执行入口/动作骨架”等对应原子需求。
- 方法只有头节点且缺少可调用执行入口时，`练习` 只能输出自检结果和候选结构，不得伪造试运行结果。
- `改变特征值` 不再使用“方向 + 改变量”作为核心逻辑，而是使用 `执行后值 - 执行前值 = 实际执行差值`。
- `改变特征值` 的执行前值必须从目标宿主下的实例特征读取，不能从输入条件包伪造当前值。
- 抽象特征只定义维度、区间和细化链；当前值必须写在实例特征上。
- `特征值差值` 已明确为抽象特征，包含 `等于零`、`大于零`、`小于零`，其中 `不等于零` 表达为 `任一命中(小于零, 大于零)`，不伪装成单个连续 I64 区间。
- `自我能力` 是标准抽象特征，方法虚拟存在用它记录本能方法能力值，任务用 `选中自我能力值` 表达“本轮要调用哪项自我能力”。
- `特征类型` 可以理解为抽象特征树；任意抽象特征或实例特征都可以成为它的值，匹配时归一到抽象特征树节点再判断父子覆盖关系。

## 构建与运行

推荐环境：

- Windows 10/11
- Visual Studio 2022 或兼容 MSBuild 环境
- C++20
- x64
- 工程当前配置的 MSVC 工具集：`v145`

### Visual Studio

1. 打开 [`鱼巢.slnx`](./鱼巢.slnx)
2. 选择 `Debug|x64` 或 `Release|x64`
3. 构建并运行 [`鱼巢.vcxproj`](./鱼巢.vcxproj)

### MSBuild

```powershell
msbuild .\鱼巢.vcxproj /t:Build /p:Configuration=Debug /p:Platform=x64 /m
```

### 常用命令

```powershell
.\x64\Debug\鱼巢.exe
.\x64\Debug\鱼巢.exe --summary
.\x64\Debug\鱼巢.exe --task-summary
.\x64\Debug\鱼巢.exe --gap-summary
.\x64\Debug\鱼巢.exe --panel-html
.\x64\Debug\鱼巢.exe --self-check-log
```

补充说明：

- 文本摘要命令支持 `--summary` 和 `--panel-text`
- `--task-summary` 也支持 `--task-mgmt-summary` / `--task-management-summary`
- `--gap-summary` 也支持 `--method-gap-summary`
- `--panel` 或 `--panel-open` 会直接打开控制面板窗口
- 仓库内置了最小本地门禁脚本：`.githooks/pre-push` 与 `.githooks/pre-push.ps1`；启用方式为 `git config core.hooksPath .githooks`
- 默认控制台交互命令为：`启动控制面板`、`检查自我实现`、`退出`
