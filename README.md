# HY-Ego（鱼巢）

`HY-Ego（鱼巢）` 是一个以“世界树 + 自我线程 + 需求/任务/方法 + 状态/动态/因果 + 自然语言显示”为核心的 C++ 实验工程。

当前仓库已经从“旧代码迁移混杂态”进入“有主线、有观测、有验收入口、可持续推进”的开发阶段。主程序是一个 Windows 控制台应用，启动后可自动拉起自我线程，并通过 WebView2 控制面板、文本摘要和运行日志观察系统状态。

更完整的架构说明见 [`说明书/项目说明.md`](./说明书/项目说明.md)。

## 项目定位

这个项目关注的不是单一算法或单一界面，而是一条持续运行的最小闭环：

1. 初始化世界树、自我锚点和运行环境。
2. 补齐根需求、根任务、任务管理任务与方法根。
3. 注册默认本能方法。
4. 启动自我线程，按 Tick 推进先天维护、任务治理和学习链路。
5. 把运行结果回写到状态、动态、因果、任务树和方法树。
6. 通过控制面板、摘要输出和日志回读当前真实状态。

当前工程的目标是先把这条主线跑稳，再在既有语义边界内继续补学习、方法生长、自然语言表达和更完整的业务步骤链。

## 当前工程能做什么

- 以 [`鱼巢.cpp`](./鱼巢.cpp) 为单一入口启动程序、自我线程和控制面板。
- 通过 [`自我初始化模块.h`](./自我初始化模块.h) 补齐世界根、自我、根需求、根任务、任务管理任务、方法根等核心骨架。
- 通过 [`自我线程类.h`](./自我线程类.h) 推进运行阶段、自检、先天动作、任务治理、学习调度和学习执行。
- 通过 [`任务管理任务模块.h`](./任务管理任务模块.h) 维护宿主任务、方法挂点、步骤节点、结果节点和治理证据。
- 通过 [`控制面板类.h`](./控制面板类.h) 输出文本摘要、HTML 和 WebView2 独立窗口观察面板。
- 通过 `2000+` 编号段规范和 `语言类_*.cpp` 把部分世界树信息映射到自然语言显示。

## 构建与运行

推荐环境：

- Windows 10/11
- Visual Studio 2022 或兼容 MSBuild 环境
- C++20
- x64
- 工程当前配置的 MSVC 工具集：`v145`

### Visual Studio

1. 打开 [`鱼巢.sln`](./鱼巢.sln)
2. 选择 `Debug|x64` 或 `Release|x64`
3. 构建并运行 [`鱼巢.vcxproj`](./鱼巢.vcxproj)

### MSBuild

```powershell
msbuild .\鱼巢.vcxproj /t:Build /p:Configuration=Debug /p:Platform=x64 /m
```

### 常用命令

```powershell
.\x64\Debug\鱼巢.exe
.\x64\Debug\鱼巢.exe --summary
.\x64\Debug\鱼巢.exe --task-summary
.\x64\Debug\鱼巢.exe --learning-summary
.\x64\Debug\鱼巢.exe --panel-html
.\x64\Debug\鱼巢.exe --self-check-log
.\x64\Debug\鱼巢.exe --acceptance-report
```

补充说明：

- 文本摘要命令支持 `--summary` 和 `--panel-text`
- `--task-summary` 也支持 `--task-mgmt-summary` / `--task-management-summary`
- `--learning-summary` 也支持 `--study-summary`
- `--panel` 或 `--panel-open` 会直接打开控制面板窗口
- `--acceptance-report` 会执行 “M1-M6 六原语主体化、事件模型与最终去向验收”，当前覆盖 14 个场景：运行态任务治理、自然运行态补方法、两个半白盒补原语场景、白盒补原语/治理本能、白盒补方法正式提交闸门、`M4` 恢复重建与等待态投影、`M5` 三类 mailbox 单次消费与双事件顺序、`M6` 两类最终去向裁决与延迟事实统一裁决
- `--acceptance-repeat=<n>` 与 `--acceptance-report-repeat=<n>` 可用于重复/批量回归
- 默认控制台交互命令为：`启动控制面板`、`检查自我实现`、`退出`

### 当前进入后续阶段前状态

当前工程已经把 `M1-M6` 这条主线收成正式基线：`主体化 / 桥接字段 / 学习双通道 / 恢复重建一次性消费 / 最小 mailbox / 延迟事实车道 / 主循环最终去向裁决` 都已落地。现阶段最主要的缺口，已经不再是“事件能否回来”，而是更进一步的：

- 更细粒度的事件优先级、事件持久化与队列观测
- 更完整的业务步骤链与学习回流后的长期策略
- 把当前 `--acceptance-report` 继续沉到自动化回归门，例如本地 pre-push 或 CI

## 当前状态

当前已经稳定形成的最小闭环包括：

- 初始化世界树和自我锚点
- 补齐根需求、根任务、任务管理任务和方法根
- 注册默认本能方法
- 启动自我线程
- 推进一步任务治理并写回任务树
- 产出状态、动态、因果与运行摘要
- 通过控制面板、文本摘要和日志回读主链

当前仍在持续推进的部分包括：

- 学习链路还没有完全迁成独立、稳定的运行时主链
- 真实业务步骤链仍少于治理链
- 方法条件、结果、成熟度、可靠度和学习回写仍需继续深化
- 自然语言显示有一部分还停留在名称层或短句层
- 事件优先级、延迟事实持久化和自动化回归门仍可继续加强
