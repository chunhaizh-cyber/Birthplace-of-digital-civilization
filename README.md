# HY-Ego（鱼巢）

`HY-Ego（鱼巢）` 是一个以“世界树 + 自我线程 + 需求/任务/方法 + 状态/动态/因果 + 自然语言显示”为核心的 C++ 实验工程。

这份 `README` 记录的是 **2026-04 本次大改版之后的当前主工程状态**。  
旧资料、旧迁移记录、旧规范草稿仍有保留，但后续开发应以当前仓库中的：

- [`规范`](./规范)
- [`计划`](./计划)
- 当前工程源码

为准。

## 本次大改版的原因

这次大改版，不是为了“换一套说法”，而是为了把工程重新收成一条能持续推进的主线。

主要原因有四个：

1. 旧版说明、旧迁移代码与当前工程真实结构已经明显脱节。
2. 初始化、自我线程、任务治理、控制面板、日志、自检等链路原来分散，观察面与运行面不够统一。
3. 需要在 **不改变现有领域数据结构语义边界** 的前提下，把旧目录中的功能逐步迁入当前工程。
4. 需要先形成“能启动、能观察、能自检、能继续演化”的最小闭环，而不是继续堆新的平行系统。

这次改版的核心约束一直没有变：

- 不为了迁移功能而改坏现有 `存在 / 场景 / 特征 / 状态 / 动态 / 需求 / 任务 / 方法 / 自我` 的数据语义。
- 允许新增模块、适配层、结果结构、日志和显示层。
- 不允许另起与世界树并列的长期真值层。

## 本次改版的总体目标

本轮改版的目标，可以收成下面五条：

1. 以当前 `鱼巢.exe` 为主宿主，收口启动路径、线程路径、显示路径和日志路径。
2. 建立 `数据初始化 -> 自我初始化 -> 自我线程 -> 任务管理 -> 可观测性` 的最小运行闭环。
3. 把旧功能尽量迁成当前结构内的模块或适配层，而不是复制出一套新结构。
4. 让控制面板和自检日志能直接反映当前真实运行状态。
5. 逐步把世界树内容接入自然语言显示，并让对应规范从 `2000+` 编号段开始独立收口。

## 当前主工程与运行方式

当前主工程已经收口为：

- 解决方案：[`鱼巢.sln`](./鱼巢.sln)
- 主工程：[`鱼巢.vcxproj`](./鱼巢.vcxproj)
- 主程序：`鱼巢.exe`

当前主程序默认形态为：

- 启动控制台
- 自动启动自我线程
- 自动启动 WebView2 控制面板

控制面板关闭后，控制台仍可继续输入命令重新打开。

当前常用入口：

- 直接运行 `鱼巢.exe`
- 控制台输入：`启动控制面板`
- 控制台输入：`退出`
- 参数：`--panel-text`
- 参数：`--panel-html`
- 参数：`--self-check-log`

当前控制面板已具备：

- WebView2 独立窗口宿主
- 左侧菜单栏 + 中间分页内容区
- 刷新按钮
- 页面内容可选中、可复制
- 世界树、自我、需求树、任务树、方法树、线程事件、先天动作等观察页

## 当前已实现概况

### 1. 初始化与世界骨架

当前已经形成明确的初始化主链：

1. 基础数据初始化
2. 世界树骨架初始化
3. 自我锚点初始化
4. 自我运行环境初始化
5. 根需求 / 根任务 / 方法根初始化

当前已经可以稳定补齐并读回：

- 世界根
- 现实场景
- 自我
- 自我内部世界
- 安全根需求 / 服务根需求
- 安全根任务 / 服务根任务 / 任务管理任务
- 方法根
- 因果信息根容器

### 2. 自我线程与双值时间本能

当前 `自我线程类` 已经正式接入主工程，并承担最小运行时推进。

当前已经落地的主线包括：

- 自我初始化入口
- 自我线程启动 / 停止 / 快照读取
- 服务值 / 安全值的时间回归能力
- 双值步长重算
- 服务归零待机保护
- 风险安全回归
- 本能动作动态与因果记录

当前口径已经按规范固定为：

- `服务值 = 0` 进入待机边界，不等于死亡
- `安全值 = 0` 才进入死亡退出边界
- 低安全对服务衰减的压制只用于防止双值共振塌陷，不直接代决“继续推进”

### 3. 本能方法与方法根

当前默认本能方法已经迁入并在初始化阶段注册，不再等热路径首次补建。

当前已经具备的最小能力包括：

- 默认本能方法首节点出生
- 动作句柄绑定
- 方法根接线
- 方法虚拟存在补齐

当前默认本能方法已覆盖的主方向包括：

- 读取核心状态
- 读取最新人类输入
- 确认收到
- 请求补充信息
- 停止当前试探并进入保守态
- 服务归零待机保护
- 定时衰减服务值
- 风险安全回归
- 双值步长重算相关先天动作

### 4. 任务管理任务

这是本次大改版里推进最多的一条主线。

当前 `任务管理任务模块` 已经从“最小一步治理骨架”推进到了 `T0 ~ T6` 的最小闭环：

- `T0`：专项摘要、自检入口、控制面板任务管理页
- `T1`：稳定上下文与稳定返回面
- `T2`：等待 / 无缺口 / 保持等待 等治理分支进入主线
- `T3`：方法挂点治理骨架与方法来源记录
- `T4`：步骤位语义落地：`执行位 / 等待位 / 回接位 / 补条件位`
- `T5`：任务目标状态 / 任务结果状态双面落地
- `T6`：治理结果回流到线程快照、面板和自检日志

当前已经能稳定完成的最小治理链包括：

- 选择宿主任务
- 绑定当前方法
- 推进宿主任务状态
- 生成步骤节点
- 生成结果节点
- 写回状态 / 动态 / 因果证据
- 把治理中间态镜像回线程摘要与任务管理结果面

当前任务树已经不再只有头结点，已经能稳定看到：

- 头结点
- 步骤节点
- 结果节点

并且步骤/结果已带位语义，而不只是单纯的“治理步骤 / 治理结果”占位名。

### 5. 状态、动态、因果与抽象因果

当前工程已经形成两层因果：

- 实例因果
- 抽象因果信息

当前抽象因果的实现口径为：

- 抽象因果信息是世界通用的
- 抽象因果信息统一挂在世界树根链下 `因果信息` 节点的子链上
- 抽象因果不再直接保留具体实例值，而是尽量用方向、变化、回升、回落、切换等抽象表达
- 抽象因果是否可信，当前主要由 `成立次数 / 失败次数` 判断

当前实例因果、状态、动态的主要产出来源包括：

- 自我线程先天动作
- 任务管理治理过程
- 目标状态 / 结果状态写回

### 6. 控制面板、日志与自检

当前控制面板和日志链已经是主工程的一部分，不再是附属试验功能。

当前已经具备：

- 控制面板自动启动
- 控制台命令重新打开面板
- WebView2 独立窗口
- 左侧菜单栏和中间分页显示
- 刷新按钮
- 文本可选择、可复制
- 自我线程摘要
- 世界树、需求树、任务树、方法树、线程事件、先天动作等观察页

当前日志与自检已具备：

- `--self-check-log`
- 运行期日志写入 [`日志`](./日志)
- 任务管理专项摘要
- 线程快照与控制面板摘要共用同一批主结果面

### 7. 自然语言规范与当前接入情况

自然语言相关规范已经独立收口到 `2000+` 编号段。

当前已经生成并接入的规范主线包括：

- `2050` 状态信息自然语言表达

- `2080` 需求信息自然语言表达

当前代码里已经接上的部分包括：

## 当前已经形成的最小闭环

到目前为止，工程已经形成下面这条最小闭环：

1. 初始化世界树和自我锚点
2. 补齐根需求、根任务、任务管理任务、方法根
3. 注册默认本能方法
4. 启动自我线程
5. 每周期执行最小先天动作和一步任务治理
6. 产出状态 / 动态 / 实例因果 / 抽象因果信息
7. 由控制面板、自检日志、文本摘要读回当前主链

这条链已经足够支持“继续迁移功能、继续补规则、继续做专项验证”，不再是完全静态的代码堆叠。

## 当前尚未完成的部分

虽然主骨架已经立住，但当前工程仍然不是最终完成态。

目前最主要的未完成项包括：

1. 学习模块还没有完全迁成独立、稳定的运行时主链。
2. 自我线程距离完整的主循环职责表仍有差距，消息消费、学习车道、延迟事实车道、最终去向裁决还需继续推进。
3. 任务管理任务当前已经完成最小治理闭环，但“真实业务步骤链”仍少于“治理链”。
4. 方法系统已不再只是空壳，但条件桶、结果桶、成熟度、可靠度、学习回写仍需继续深化。
5. 抽象因果已经能生成并显示，但失败回流、验证回路和更强的复用消费链仍需继续补。
6. 自然语言显示规范已经基本成形，但不同信息节点的完整句式生成仍在逐步接线。

一句话说，当前工程已经从“旧代码迁移混杂态”进入了“有主线、有验收、有观测、有后续工位”的持续开发态，但还没有进入“功能完全体”。

## 构建与运行

推荐环境：

- Windows 10/11
- Visual Studio 2022
- MSVC v143
- Windows SDK 10.0
- x64

### Visual Studio

1. 打开 [`鱼巢.sln`](./鱼巢.sln)
2. 选择 `Debug|x64` 或 `Release|x64`
3. 构建并运行 [`鱼巢.vcxproj`](./鱼巢.vcxproj)

### MSBuild

```powershell
msbuild .\鱼巢.vcxproj /t:Build /p:Configuration=Debug /p:Platform=x64 /m
```

### 常用运行命令

```powershell
.\x64\Debug\鱼巢.exe
.\x64\Debug\鱼巢.exe --panel-text
.\x64\Debug\鱼巢.exe --panel-html
.\x64\Debug\鱼巢.exe --self-check-log
```
