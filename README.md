# Birthplace of Digital Civilization（鱼巢）

`Birthplace of Digital Civilization（鱼巢）` 是一个以“世界树 + 自我线程 + 需求/任务/方法 + 状态/动态/因果 + 自然语言显示”为核心的 C++ 实验工程。工程内部历史名称仍保留为 `HY-Ego（鱼巢）`，当前新仓库用于同步完整代码、规范、计划和说明文档。

云端仓库：[`chunhaizh-cyber/Birthplace-of-digital-civilization`](https://github.com/chunhaizh-cyber/Birthplace-of-digital-civilization)

更完整的架构说明见 [`说明书/项目说明.md`](./说明书/项目说明.md)。

## 项目定位

这个项目不是单一算法库，也不是单一界面程序，而是在构造一个可持续运行的最小数字自我闭环：

1. 初始化世界树、自我锚点和运行环境。
2. 补齐根需求、根任务、任务管理任务与方法根。
3. 注册默认本能方法。
4. 启动自我线程，按 Tick 推进先天维护、任务治理、方法查找和方法补齐。
5. 把运行结果回写到状态、动态、因果、任务树、方法树和虚拟存在。
6. 通过控制面板、摘要输出和日志回读当前真实状态。

当前主线重点是让“任务需求 -> 查找方法 -> 方法配对结果 -> 配对参数 -> 可执行输入参数场景 -> 本能函数执行 -> 运行事实回写”这条链路逐步闭合。

## 当前能做什么

- 以 [`鱼巢.cpp`](./鱼巢.cpp) 为单一入口启动程序、自我线程和控制面板。
- 通过 [`自我模块.初始化.ixx`](./自我模块.初始化.ixx) 补齐世界根、自我、根需求、根任务、任务管理任务、方法根等核心骨架。
- 通过 [`自我线程类.h`](./自我线程类.h) 推进运行阶段、自检、先天动作、任务治理和方法补齐。
- 通过 [`本能动作管理类.ixx`](./本能动作管理类.ixx) 管理本能动作登记、调用和返回运行虚拟存在。
- 通过 [`自我动作实现模块.ixx`](./自我动作实现模块.ixx) 实现本能函数，包括 `查找方法` 与差值驱动的 `改变特征值`。
- 通过 [`控制面板类.ixx`](./控制面板类.ixx) 输出文本摘要、HTML 和 WebView2 独立窗口观察面板。
- 通过 [`规范`](./规范)、[`计划`](./计划)、[`说明书`](./说明书) 保留实现边界、路线和接手说明。

## 近期语义基线

近期主线已经把方法查找和特征改变从“名称/方向驱动”推进到“特征类型 + 特征值约束 + 运行事实”驱动：

- `查找方法` 不按方法名查找，只判断 `需求条件包 ⊆ 方法能力包`。
- `查找方法` 只输出 `方法配对结果存在`，不直接执行方法，也不生成可执行输入参数场景。
- `改变特征值` 不再使用“方向 + 改变量”作为核心逻辑，而是使用 `执行后值 - 执行前值 = 实际执行差值`。
- `改变特征值` 的执行前值必须从目标宿主下的实例特征读取，不能从输入条件包伪造当前值。
- 抽象特征只定义维度、区间和细化链；当前值必须写在实例特征上。
- `特征值差值` 已明确为抽象特征，包含 `等于零`、`大于零`、`小于零`，其中 `不等于零` 表达为 `任一命中(小于零, 大于零)`，不伪装成单个连续 I64 区间。

## 构建与运行

推荐环境：

- Windows 10/11
- Visual Studio 2022 或兼容 MSBuild 环境
- C++20
- x64
- 工程当前配置的 MSVC 工具集：`v145`

### Visual Studio

1. 打开 [`鱼巢.slnx`](./鱼巢.slnx)
2. 选择 `Debug|x64` 或 `Release|x64`
3. 构建并运行 [`鱼巢.vcxproj`](./鱼巢.vcxproj)

### MSBuild

```powershell
msbuild .\鱼巢.vcxproj /t:Build /p:Configuration=Debug /p:Platform=x64 /m
```

### 常用命令

```powershell
.\x64\Debug\鱼巢.exe
.\x64\Debug\鱼巢.exe --summary
.\x64\Debug\鱼巢.exe --task-summary
.\x64\Debug\鱼巢.exe --learning-summary
.\x64\Debug\鱼巢.exe --panel-html
.\x64\Debug\鱼巢.exe --self-check-log
```

补充说明：

- 文本摘要命令支持 `--summary` 和 `--panel-text`
- `--task-summary` 也支持 `--task-mgmt-summary` / `--task-management-summary`
- `--learning-summary` 也支持 `--study-summary`
- `--panel` 或 `--panel-open` 会直接打开控制面板窗口
- 仓库内置了最小本地门禁脚本：`.githooks/pre-push` 与 `.githooks/pre-push.ps1`；启用方式为 `git config core.hooksPath .githooks`
- 默认控制台交互命令为：`启动控制面板`、`检查自我实现`、`退出`

## 当前仍在推进

- `查找方法 -> 配对参数 -> 改变特征值` 链路继续从局部可用推进到全局稳定。
- 方法能力包、形参表、结果表、效果表需要继续补齐和统一回写。
- 任务管理模块仍在内聚重构中，真实业务步骤链还要继续加深。
- 自然语言显示仍有一部分停留在名称层或短句层。
- 日志化验证、运行摘要、计数与快照检查需要继续收敛。
