日期：2026-05-10

```text

```

## 1. 定位

需求树正常生长不是单个需求节点字段能完成的事情，而是一组跨模块的确定性转换链。

```text
自我状态
→ 根需求压力
→ 需求目标
→ 需求入树
→ 去重 / 合并
→ 权重刷新
→ 管理 / 执行角色分化
→ 任务承接
→ 任务 / 方法反馈
→ 需求结算
→ 下一轮主需求选择
```

口径修订：

```text
需求目标中的“差距”只表示同一目标特征类型上的特征值未满足。
它不能表达特征类型之间的关系。

不同特征类型之间的关系，包括联动、先后、影响、转化和因果，应放入状态动态、二次特征 / 关系特征、因果实例 / 因果模板和方法结果能力中。

需求改变的不是某个存在本身，而是某个存在上的某个特征状态。
因此原子目标必须按：

    目标宿主 / 被需求存在
    目标特征类型
    当前状态
    目标状态
    满足关系

来解析。历史文本中的“目标主体”若非指需求主体，均按“目标宿主 / 被需求存在”理解。
```

本设计先完成 D0-D1b：

```text
D0：特征链详细设计落档。
D1：现有代码入口核对。
D1b：本能转换函数 / 本能转换方法分层。
```

不在本轮实现：

```text
休眠期修复动作
学习需求自动生成
外界探索需求
自检报告需求化
任务筹办 / 执行重新注册成本能方法
```

## 2. 执行层口径

需求树生长转换分三层。

```text
本能转换函数：
    C++ 内置确定性函数，保证内部循环最小需求树能生长。

本能转换方法：
    对本能转换函数的任务 / 方法系统包装，用于被任务调度、记录、复盘和因果沉淀。

学习转换方法：
    后续因果补完、学习或探索阶段形成的增强方法。
```

本轮结论：

```text
需求树最小生长必须先靠本能转换函数。
是否注册成本能转换方法要逐条判断。
会引入循环依赖的转换，暂不注册成本能方法。
```

禁止项：

```text
任务筹办不是本能转换方法。
任务执行一步不是本能转换方法。
自我线程主循环不是本能转换方法。
任务管理界面线程承接不是本能转换方法。
任务管理工作线程调度不是本能转换方法。
```

## 3. 特征所有者

| 所有者 | 必需特征 | 当前代码承载 | 状态 |
| --- | --- | --- | --- |
| 自我存在 | 安全值、服务值、风险安全值、待机状态、时序步长 | `自我类` 运行态字段和自我特征镜像 | 已有 |
| 状态节点 | 状态主体、状态特征、状态值、对应信息节点、是否变化 | `状态节点主信息类` | 已有 |
| 需求节点 | 需求主体、需求场景、被需求存在、当前状态、目标状态、满足关系、目标特征缓存、权重、对应任务、任务列表、结算账本、派生归因 | `需求主信息类` | 已有 |
| 需求树结构 | 父子链、结构角色、阻塞父任务、有效截止 | 链表节点结构和 `需求主信息类` | 已有 |
| 任务信息节点 | 对应需求、目标状态、优先级、承接权重、已折算值 | `任务主信息类` 及任务类访问器 | 已有 |
| 任务虚拟存在 | 任务语义身份、任务目标、任务状态、完成条件等语义镜像 | 任务管理界面线程绑定 | 已有，不由需求直接持有 |
| 方法 / 本能动作 | 方法结果、阻塞原因、能力缺口、派生来源 | 方法树、筹办回执、任务上行消息 | 已有 |
| 因果实例 | 条件、动作、结果、来源任务 / 方法、置信度、验证状态 | `因果实例主信息类` | 已有，后续学习阶段使用 |
| 场景 / 内部世界 | 状态索引、动态索引、关系索引、因果索引 | 场景节点主信息 | 已有 |

边界：

```text
需求节点仍是目标记录 / 动因节点。
需求.对应任务 仍指向任务信息节点。
需求不直接指向任务虚拟存在。
任务虚拟存在由任务建立流程和任务管理界面线程保证。
```

## 5. 转换细节

### T1：自我状态 -> 根需求压力

输入：

```text
安全值
服务值
根任务权重
目标服务值
安全根方向差值
```

输出：

```text
安全压力
服务压力
安全根需求权重
服务根需求权重
```

代码入口：

```text
自我线程模块.impl.cpp:2337 私有_从需求树构建需求列表
自我线程模块.impl.cpp:2432 私有_选择当前主需求候选_按根权重
```

现状：

```text
已有本能转换函数。
安全根需求 / 服务根需求权重会在构建需求列表时按安全压力 / 服务压力刷新。
根需求当前状态和目标状态镜像会随需求列表刷新。
根权重刷新、子权重刷新和控制面板需求树读取共用需求树全局互斥，避免半帧权重诊断。
```

注意：

```text
这只证明根需求压力可选择。
它不等于需求树正常生成成立。
```

本能方法注册建议：

```text
暂缓。
原因：这是自我线程主裁决的内核函数，过早包装成本能方法会让主需求选择依赖方法系统。
```

## 6. 业务缺口与非正常逻辑错误

### 6.1 非正常逻辑错误

以下不允许静默兜底：

```text
运行期显式创建需求但父需求为空。
派生需求父需求解析为需求树结构根。
派生需求缺当前状态或目标状态却试图入树。
派生需求缺目标特征类型却试图入树。
执行需求任务化完成后仍没有任务信息节点。
需求同时指向任务信息节点和任务虚拟存在。
任务建立流程完成后仍缺任务虚拟存在。
实例入口缺主体或场景却试图落树。
```

当前状态：

```text
派生需求父需求、结构根和状态 / 特征缺失闸门已在自我线程中实现。
普通需求更新指令仍保留根需求初始化兼容路径，D2 需要按调用入口区分初始化和运行期显式创建。
```

### 6.2 业务缺口

以下不是逻辑错误，应走状况回执、派生需求、学习需求或等待：

```text
方法缺动作骨架。
方法缺可执行入口。
方法缺条件节点。
方法缺结果节点。
方法缺条件结果对。
参数不可配且筹办 / 执行选择由学习承接。
方法运行账不足。
方法结果不稳定。
因果链缺关键节点。
```

### 6.3 暂不需求化

```text
自检报告型候选。
休眠期修复动作。
外界探索需求。
```

自检线程只发现缺失并报送自我线程，不生成修复需求。
