# 特征类型串联字典

## 1. 串联口径

- `状态` 是特征在某一时刻的值。
- `特征` 归属于某个存在，存在处于某个场景中；存在既包括真实存在，也包括虚拟存在。
- 本文只串联 `特征类型`，不串联具体状态实例。
- 串联起点固定为 `安全值`、`服务值` 两个根。
- “直接链接”按当前代码里的直接参与关系判定，包含 4 种情况：
  - `直接写回`：A 的当前值直接参与 B 的新值写入。
  - `直接选取`：A 的当前值直接参与 B 的候选选择。
  - `直接派生`：A 的当前值直接参与 B 的枚举/摘要/签名派生。
  - `直接镜像`：A 的当前值直接写成 B 的治理镜像或账本镜像。
- 当前系统存在天然回环，所以本文分两层看：
  - `严格直接影响图`：严格按“谁直接影响上层节点”展开，回到根时不重复展开，只记为回环边。
  - `全量主链字典`：为了把 49 个特征类型全部挂上去，按“首次进入主控制链的位置”分层；多上游时保留多个上游说明，不强行压成单父树。


## 2. 当前必须承认的 3 个回环

### 2.1 双根调节回环

- `服务值 -> 时序正向步长 -> 安全值`
- `服务值 -> 时序反向步长 -> 安全值`
- `安全值 + 服务值 -> 服务时序衰减步长 -> 服务值`
- 额外还有一条保护性跨根边：
  - `服务值(归零) -> 待机状态`
  - `服务值(归零) -> 安全值=1`

### 2.2 治理状态机回环

- `任务管理_最近功能域 + 任务管理_当前缺口类型 + 任务管理_当前下一步去向 + 门控结果 + 锚点指针 -> 任务管理_当前方法需求位`
- `任务管理_当前方法需求位 + 双面状态/轻量证据 -> 任务管理_当前本能能力缺口类型`
- `任务管理_当前本能能力缺口类型 -> 任务管理_当前功能域 / 任务管理_当前缺口类型 / 任务管理_当前下一步去向`
- 下一轮再把这批“当前态”读回去，继续派生新的 `任务管理_当前方法需求位`

### 2.3 双面反馈回环

- `任务管理_当前治理状态迁移 -> 任务目标状态`
- `任务管理_宿主任务状态 -> 任务结果状态`
- `任务目标状态 + 任务结果状态 -> 任务管理_当前宿主目标结果比较 -> 任务管理_当前后果治理 -> 任务管理_最近反馈摘要`
- `任务管理_最近反馈摘要` 又会在下一轮回填为轻量因果摘要，参与 `任务管理_当前方法需求位 / 任务管理_当前本能能力缺口类型`


## 3. 严格直接影响图

这部分只保留“能直接影响上层状态”的边，不为了覆盖面强行补语义边。

### 3.1 双根

- `安全值`
  - `时序正向步长`
    - `[回环] 服务值`
  - `时序反向步长`
    - `[回环] 服务值`
  - `[跨根直写] 服务值`
    - 仅在 `服务归零待机保护` 时把 `安全值` 压到 `1`
- `服务值`
  - `服务时序衰减步长`
    - `[回环] 服务值`
    - `[回环] 安全值`

### 3.2 紧贴双根但不直接充当根控制器的耦合特征

- `待机状态 <- 服务值`
  - 用途：把“服务归零”翻译成可持续的待机保护态
- `当前主需求 <- 安全值 + 服务值`
  - 用途：把双根压力翻译成主需求锚点


## 4. 全量主链字典

## 4.1 L0 双根

| 特征类型 | 直接上游特征 | 直接下游特征 / 用途 |
| --- | --- | --- |
| 安全值 | `时序正向步长`、`时序反向步长`、`服务值(仅归零保护)` | 下游到 `当前主需求`、`服务时序衰减步长`；用途是根安全真值 |
| 服务值 | `服务时序衰减步长` | 下游到 `当前主需求`、`时序正向步长`、`时序反向步长`、`待机状态`；用途是根服务真值 |


## 4.2 L1 根控制与安全护栏

| 特征类型 | 直接上游特征 | 直接下游特征 / 用途 |
| --- | --- | --- |
| 时序正向步长 | `服务值` | 直接写回 `安全值`；用途是安全回升力度 |
| 时序反向步长 | `服务值` | 直接写回 `安全值`；用途是安全回落力度 |
| 服务时序衰减步长 | `服务值`、`安全值` | 直接写回 `服务值`；用途是服务衰减力度 |
| 待机状态 | `服务值(归零触发)` | 下游到线程门控、主体风险保守判定；用途是零服务保护镜像 |
| 物理安全 | `无（当前仅初始化/外部输入）` | 当前主要进入线程摘要；用途是物理层安全护栏 |
| 风险安全 | `无（线程周期回归写回）` | 下游到学习输入、线程摘要；用途是风险预算护栏 |
| 当前主需求 | `安全值`、`服务值` | 直接下游是 `当前主任务`；用途是把双根压力翻译成主需求锚点 |

## 5. 一句话总括

- `安全值`、`服务值` 是根真值。
- `时序步长/衰减步长/待机状态` 是紧贴双根的控制护栏。
- `当前主需求 -> 当前主任务 -> 当前主方法` 是把双根压力翻译成治理锚点的主链。
- `任务管理_当前方法需求位 -> 任务管理_当前本能能力缺口类型` 是治理判断核心。
- `任务管理_当前功能域 / 任务管理_当前缺口类型 / 任务管理_当前下一步去向 / 任务管理_当前治理状态迁移` 是治理状态机核心。
- `任务目标状态 / 任务结果状态 / 任务管理_当前宿主目标结果比较 / 任务管理_当前后果治理` 是双面校核核心。
- `任务管理_最近反馈摘要` 是跨轮回流最关键的轻量因果摘要。
- `待学习方法数量 / 尝试学习状态` 是学习压力镜像。
- `物理安全 / 风险安全 / 情绪 / 外显情绪 / 外设可用性` 目前属于侧翼或保留支路，还没完全接入主治理闭环。
