## 3. 主要差距

### B0 差距：任务结果没有统一生成因果候选

当前代码已有任务价值反馈、需求结算、推进回执和结果状态消息，但缺少一个统一结构：

```text
任务工作结果 / 推进回执 / 结算快照
→ 因果候选记录
```

需要新增：

```cpp
struct 结构_因果候选记录 {
    任务节点类* 来源任务 = nullptr;
    需求节点类* 来源需求 = nullptr;
    方法节点类* 来源方法 = nullptr;
    存在节点类* 任务虚拟存在 = nullptr;

    std::vector<可解析引用<状态节点类>> 前置状态集{};
    std::vector<可解析引用<状态节点类>> 结果状态集{};
    可解析引用<状态节点类> 主结果状态{};

    I64 安全收益 = 0;
    I64 服务收益 = 0;
    I64 缺口类型值 = 0;
    bool 样本完整 = false;
    bool 执行成功 = false;
    bool 需要学习 = false;
};
```

第一轮只记录候选，不直接升格因果模板。

### B1 差距：推进状况到因果缺口类型没有稳定映射

需要把推进状况映射为因果缺口：

| 推进状况 | 因果缺口 |
| --- | --- |
| 方法运行账不足 | 学习方法：补运行账 |
| 方法结果不稳定 | 学习方法：验证可靠度 |
| 方法缺条件结果对 | 学习因果：补条件结果对 |
| 参数不可配且源于场景条件不足 | 学习条件：确认前置条件 |
| 目标结果不符合 | 学习因果：解释结果偏差 |
| 动作执行成功无可比较结果 | 学习状态：补可比较结果状态 |

当前这些映射尚未有统一函数。

建议新增：

```cpp
enum class 枚举_因果缺口类型 : std::uint8_t {
    未定义 = 0,
    缺前置条件 = 1,
    缺结果验证 = 2,
    方法运行账不足 = 3,
    方法结果不稳定 = 4,
    结果偏差无法解释 = 5,
    缺可比较结果 = 6,
    缺外部信息 = 7,
    缺风险判断 = 8,
};
```

以及：

```cpp
结构_因果缺口记录 从任务推进回执识别因果缺口(
    const 任务模块::筹办::结构_任务推进回执& 回执);
```

### C0 差距：因果缺口尚不能生成学习需求


学习需求目标不得复制原父需求目标。主体应是方法虚拟存在、任务虚拟存在、外部对象占位或因果候选记录承载对象。
