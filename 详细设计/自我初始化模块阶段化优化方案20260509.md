# 自我初始化模块阶段化优化方案 20260509

## 定位

自我初始化模块是数字生命核心的点火器，不是任务治理发动机。它只负责让自我核心可存在、可定位、可感知自身、可拥有根需求，并让自我线程能够接管后续治理。

冷启动最小闭环：

```text
数据层可用
→ 语素层可用
→ 自我实例存在可用
→ 现实场景 / 内部世界可用
→ 自我核心特征槽可用
→ 安全根需求 / 服务根需求可用
→ 本能方法可用
→ 运行态恢复
→ 当前主需求可被自我线程接管
```

任务信息节点治理、任务虚拟存在绑定、任务工作线程派发、任务结果回流，不进入冷启动硬条件。

## 阶段

```text
数据层
语素层
自我锚点
核心特征
本能方法
根需求与结构根按需入口
主链镜像
运行态同步
验收
故障收口
```

每个阶段都写入 `结构_自我初始化步骤记录`，包括阶段、名称、成功状态和摘要。初始化失败时，结果对象必须能看到缺失项和故障摘要。

## 结果口径

初始化结果分四层：

```text
冷启动完成：
    数据层、语素层、自我锚点、核心特征、运行控制、根需求就绪。

治理入口就绪：
    冷启动完成，默认本能方法可用，当前主需求已回填。

任务治理就绪：
    治理入口就绪，任务治理入口可承接。

完全运行就绪：
    任务治理就绪，且根任务已就绪或允许延后。
```

`初始化完成` 当前等同于 `治理入口就绪`。根任务、当前主任务、当前主方法允许由自我线程和任务管理界面线程后续补齐。

## 根任务延后

初始化模块必须确保：

```text
安全根需求
服务根需求
```

`需求根节点 / 任务根节点 / 方法根节点` 属于 `存在节点主信息类` 的结构字段，默认可以为空。初始化模块可以因创建根需求而按需确保需求根，但冷启动完成不以三类根字段全部非空作为硬条件。

初始化模块不强制确保：

```text
安全根任务
服务根任务
当前主任务
当前主方法
```

`根任务已就绪` 是诊断字段，必须如实反映实际根任务是否存在；`根任务允许延后` 是验收放行条件。
`方法根已就绪` 也只作为诊断字段；方法根在注册或创建方法时由方法创建链确保，缺失本身不是冷启动异常。

## 禁止项

本优化不引入以下结构：

```text
枚举_存在形态
抽象存在主信息类
特征节点主信息类::当前值信息
```

存在仍然只有一种结构 `存在节点主信息类`。存在或基础信息节点作为特征值时，继续使用 `特征值::指针句柄`。

## 旧兼容链删除

`私有_补齐初始化任务解决方法集` 不再保留。任务解决方法补齐已经迁移到新链：

```text
自我线程
→ 任务管理界面线程
→ 任务管理工作线程
```

初始化模块不再写任务执行方法、不再为根任务追加默认本能方法，也不再暴露“初始化任务解决方法已补齐”诊断位。冷启动只保证自我核心、根需求、本能方法和治理入口可用。

## 后续概念层

等基础信息层提供稳定接口后，可增加概念归属阶段：

```text
确保存在概念：存在、数字生命、自我
自我存在.概念集 = [存在, 数字生命, 自我]
按概念扩充自我实例记忆
```

在接口未稳定前，不在初始化模块硬编码概念树创建逻辑。
