# 特征双根树

## 1. 收口规则

- 本文目标不是再保留“全部直接因果边”，而是把已经识别出的全部链路压成 `一颗双根树`。
- 固定双根：`安全值`、`服务值`。
- 每个非根特征只保留 `一个主父节点`，在树中只出现 `一次`。
- `任务完成` 在当前仓内还不是独立特征类型，因此本文把它视为 `双根共同影响概念`：
  - 不计入 `49` 个特征类型
  - 当前主要由 `任务结果状态=完成` 与 `任务管理_当前宿主目标结果比较=已对齐` 这条现有特征链承载
- 父子关系含义统一为：
  - `子节点主要服务于、供给、支撑父节点`
  - 它可以是直接写值、直接选取、直接派生，也可以是为了把整套链路压成树而采用的 `主归属挂接`
- 所有被压掉的“第二父节点、回流边、跨根边”，统一放到文末 `非树反馈边`。
- 本树共收口 `49` 个特征类型，与 [状态字典.md](</D:/鱼巢/详细设计/状态字典.md>) 保持一致。

## 3. 挂树说明

### 3.1 为什么 `当前主需求` 挂在 `安全值` 下

- `当前主需求` 实际由 `安全值` 与 `服务值` 共同裁决。
- 这里把它挂到 `安全值` 下，不是说它只受安全影响，而是为了把双根合成成树。
- 这么挂有 2 个代码侧理由：
  - `选择当前主需求候选_()` 里安全压力与服务压力并列比较，平局时安全侧优先。
  - 安全根比服务根更像“停损/保底”的总锚点，放在树上更稳。
- `服务值 -> 当前主需求` 保留到文末 `非树反馈边`。

### 3.2 为什么治理主链挂在 `当前主任务 -> 当前宿主任务指针` 下

- 这条链是整套治理的实际工作面。
- 从用途上看，几乎所有治理镜像最终都在回答 3 个问题：
  - 现在治理哪一个宿主任务
  - 现在挂了哪个方法
  - 当前步骤/最近结果推进到了哪里
- 所以把 `任务管理_当前宿主任务指针` 当成治理树干最稳。

### 3.3 为什么 `任务管理_最近反馈摘要` 挂在 `任务管理_当前后果治理` 下

- 它并不只是“文字摘要”，而是当前轮治理结论的轻量因果封装。
- 从生成顺序看，它主要是对 `后果治理 / 总控结果 / 反馈类型` 的压缩。
- 但它又会在下一轮反向参与 `任务管理_当前方法需求位` 与 `任务管理_当前本能能力缺口类型` 的判断，所以它同时也是一条重要回流边。

### 3.4 为什么 `待学习方法数量`、`尝试学习状态` 挂在 `任务管理_当前本能能力缺口类型` 下

- 学习压力不是自我独立目的，而是当前治理判断“缺能力、缺证据、需补齐”后的外溢结果。
- 从用途看，它们更像 `当前本能能力缺口类型` 的后续行动镜像，而不是独立根。

### 3.5 为什么 `情绪/外显情绪/外设可用性` 挂在 `服务值` 下

- 这几类特征当前没有稳定接入治理主链，也没有直接写回安全主链。
- 但从用途归属上，它们更接近“服务侧的活动表达与执行条件”。
- 所以统一收口到 `服务值` 根下，作为当前版本的服务侧保留支路。

### 3.6 为什么还要单独补一个“任务完成”概念

- 你指出的是对的：从控制意义上，`任务完成` 确实是影响 `安全值 / 服务值` 的上位因子之一。
- 但当前代码里它还没有被升格成独立特征名；现在真正可观测、可挂树的承载链是：
  - `任务结果状态=完成`
  - `任务管理_当前宿主目标结果比较=已对齐`
  - `任务管理_当前后果治理=无额外后果/回接后保持等待`
- 规范层也已经把这件事说得很清楚：
  - `子任务完成后必须能回投到根差值变化`
  - 但 `不能只因为执行完成了，就判定服务或安全应正向加分`
  - 也 `不能只因任务完成而直接生成正向增量`
- 所以本文的处理是：
  - 树内继续用现有特征链承载 `任务完成`
  - 树外把 `任务完成` 补成 `双根共同影响概念边`
  - 等后续仓内真的出现 `任务完成真值` 或同类正式特征时，再把它升成树内正式节点


## 4. 非树反馈边

这些边真实存在，但为了保持“一颗双根树”的结构，没有再作为主父边保留。

- `服务值 -> 时序正向步长`
- `服务值 -> 时序反向步长`
- `安全值 -> 服务时序衰减步长`
- `服务值 -> 当前主需求`
- `任务管理_当前方法指针 -> 任务管理_当前方法需求位`
- `任务管理_当前步骤指针 -> 任务管理_当前方法需求位`
- `任务管理_最近结果指针 -> 任务管理_当前方法需求位`
- `任务结果状态 -> 任务管理_当前宿主目标结果比较`
- `任务管理_当前下一步去向 -> 任务管理_当前后果治理`
- `任务管理_当前步骤位类型 -> 任务管理_当前治理态型`
- `任务管理_当前方法需求位 -> 任务管理_当前治理态型`
- `任务管理_最近反馈摘要 -> 任务管理_当前方法需求位`
- `任务管理_最近反馈摘要 -> 任务管理_当前本能能力缺口类型`
- `任务管理_当前方法来源 -> 任务管理_最近反馈摘要`
- `任务管理_当前主体ID -> 任务管理_最近反馈摘要`
- `任务管理_当前分身ID -> 任务管理_最近反馈摘要`
- `任务管理_最近上层反馈摘要 -> 任务管理_最近反馈摘要`
- `任务管理_当前方法指针 -> 任务管理_当前方法位专项动态`
- `任务管理_当前方法指针 -> 任务管理_当前最小原语动态`
- `任务结果状态=完成 + 任务管理_当前宿主目标结果比较=已对齐 -> 任务完成（概念）`
- `任务完成（概念） -> 安全值`
- `任务完成（概念） -> 服务值`
