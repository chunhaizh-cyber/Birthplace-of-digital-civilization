## 1. 主循环到任务管理任务

关键入口：

- `自我线程类::执行主循环一轮_`
- `任务管理任务模块::执行一步治理`

```mermaid
flowchart TD
    A["自我线程::执行主循环一轮"] --> B{"mailbox 有待消费事件?"}
    B -->|是| C["弹出队首事件<br/>记录进行中事件"]
    B -->|否| D["构造普通推进心跳事件"]
    C --> E["写入任务管理显式事件控制"]
    D --> E
    E --> F["补齐初始化 / 主链回填 / 先天维护"]
    F --> G["根层重判 + 执行前门控"]
    G --> H["写入上位输入镜像"]
    H --> I["任务管理任务::执行一步治理"]
    I --> J["收集治理结果"]
    J --> K["学习调度 / 学习执行"]
    K --> L["归并延迟事实"]
    L --> M["主循环统一裁决最终去向"]
    M --> N["必要时把学习回流 / 执行结果 / 外部反馈重新入队"]
```

说明：

- `mailbox` 为空时，不会停机，而是退化为一次 `普通推进` 心跳。
- 即使是 `恢复重建`，也先回到主循环，再进入 `任务管理任务` 的单步治理。
- `最终去向` 不在 `任务管理任务` 内部拍板，而在主循环里统一裁决。

## 2. 任务管理任务一步治理

关键入口：

- `任务管理任务模块::执行一步治理`
- `私有_M9_执行前段治理阶段`
- `私有_M9_执行后段治理动作阶段`
- `私有_M9_执行学习承接与业务派生阶段`
- `私有_M2_组装一步结果阶段`

```mermaid
flowchart TD
    A["执行一步治理"] --> B["前段治理阶段"]
    B --> B1["构造任务管理请求"]
    B1 --> B2["读取上下文"]
    B2 --> B3["消费显式事件 / 恢复重建"]
    B3 --> B4["装配请求骨架<br/>请求ID / 触发信息 / 补充材料"]
    B4 --> C["确保任务管理主体虚拟存在"]
    C --> D["初始化分身继承面"]
    D --> E["形成治理特征面"]
    E --> F["命中当前方法需求位"]
    F --> G["评估本能能力缺口"]
    G --> H{"桥接补层裁决允许继续?"}
    H -->|否| I["早停<br/>保持等待 / 转学习 / 补结构"]
    H -->|是| J["按宿主状态形成单步决策"]
    J --> K["写治理证据<br/>宿主 / 方法 / 状态"]
    K --> L["后段治理动作阶段"]
    L --> M["学习承接与业务派生阶段"]
    I --> N["组装一步结果阶段"]
    M --> N
    N --> O["治理轮收口<br/>主体回并 / 稳定字段 / 恢复快照 / 摘要"]
```

说明：

- `任务管理任务` 每轮只围绕一个宿主推进一步。
- 桥接补层不允许继续时，本轮会早停，但仍然会进入统一收口。
- 收口阶段会同步：`一步结果`、`稳定字段`、`主体回并摘要`、`恢复快照`。

## 3. 桥接补层

关键入口：

- `私有_形成治理特征面`
- `私有_命中方法需求位`
- `私有_评估本能能力缺口`
- `私有_桥接补层裁决`

```mermaid
flowchart TD
    A["形成治理特征面"] --> B["命中方法需求位"]
    B --> C["评估本能能力缺口"]
    C --> D{"无缺口?"}
    D -->|是| E["放行后段"]
    D -->|否| F{"缺口类型"}
    F -->|缺观察 / 写回 / 挂点原语| G["转补原语 / 治理本能"]
    F -->|缺状态差分 / 因果 / 轻量链| H["退回补结构或等待补证据"]
    F -->|需学习补齐| I["放行学习承接"]
    G --> J["本轮不继续业务扩张"]
    H --> J
    I --> J
```

说明：

- 当前最小原语固定为 6 个，桥接层消费的是：
  - `方法需求位`
  - `本能能力缺口`
  - `最小原语位图`
  - `主观察特征`
- 桥接字段是治理证据和承接锚点，不等于最终裁决真值。

## 4. 主循环最终去向裁决

关键入口：

- `私有_裁决主循环最终去向`

```mermaid
flowchart TD
    A["主循环拿到 治理结果 + 已消费事件 + 延迟事实"] --> B{"停止边界?"}
    B -->|是| C["最终去向 = 停止"]
    B -->|否| D{"收束优先 或 任务管理要求收束?"}
    D -->|是| E["最终去向 = 进入收束"]
    D -->|否| F{"任务管理要求回到重筹办?"}
    F -->|是| G["最终去向 = 回到重筹办"]
    F -->|否| H{"待机保持 / 仅允许等待 / 等待正式提交放行?"}
    H -->|是| I["最终去向 = 待机"]
    H -->|否| J{"学习调度 / 学习执行 / 学习回流待再判?"}
    J -->|是| K["最终去向 = 转入学习"]
    J -->|否| L["最终去向 = 继续提交"]
```

说明：

- `任务管理任务` 可以给出 `当前下一步去向`，但不能直接替代主循环拍板。
- `延迟事实` 与 `显式事件` 会一起进入最终去向裁决。
- `等待正式提交放行` 会直接把最终去向压到 `待机`。

## 5. 当前代码分层

按当前实现，可以把任务管理相关流程分成 4 层：

1. `事件入口层`
   mailbox、恢复重建、普通心跳
2. `一步治理层`
   请求、单步决策、写回结果、一步结果
3. `桥接裁决层`
   方法需求位、本能能力缺口、最小原语、桥接补层裁决
4. `主循环裁决层`
   延迟事实归并、最终去向统一裁决

## 6. 当前收口结论

当前代码已经形成下面这条稳定主链：

`mailbox/恢复重建/普通心跳`
-> `主循环重判`
-> `任务管理任务单步治理`
-> `学习与回流`
-> `延迟事实归并`
-> `主循环统一裁决最终去向`

这意味着：

- `任务管理任务` 已经不是单纯的大函数，而是显式阶段链。
- `mailbox` 已经成为真实事件入口，而不是纯摘要概念。
- `最终去向` 仍然保持在主循环统一裁决，没有下沉到任务管理内部。
