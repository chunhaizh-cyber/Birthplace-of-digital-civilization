# 任务执行函数与流程梳理

本文说明当前“执行这一步”的新口径。

任务执行不是本能函数，也不再通过任务步骤节点壳间接调用。任务执行是任务系统普通函数：工作线程在完成门控和筹办后，直接读取任务头绑定的目标方法并调用执行桥。

## 1. 当前定位

```text
自我线程主循环
  -> 任务管理工作线程门控
  -> 普通筹办函数：按执行需求目标查找或创建目标方法头
  -> 普通执行函数：读取任务头绑定方法并调用
  -> 方法动作函数 / 本能动作函数
  -> 运行事实、结果场景、执行缺失原子需求消息
  -> 自我线程回流判断需求是否满足
```

本能函数只代表真实动作能力，例如改变特征值、练习、尝试学习和定向学习。任务执行本身只是调度和调用已选方法，不是方法树中可被查找命中的动作能力。

## 2. 执行入口

当前执行入口是：

```text
任务模块.管理工作线程
  -> 自我初始化模块::筹办任务(...)
  -> 自我_执行桥模块::执行任务(...)
```

`执行任务(...)` 接收任务头，读取任务头上的执行方法和选中自我能力值。`结构_任务步骤执行结果_最小::当前步骤节点` 仅作兼容字段，当前应等同于被执行任务头。

## 3. 执行前置

进入执行前必须满足：

```text
任务头存在
任务头绑定具体执行需求
普通筹办已选中或创建目标方法头
任务头已写入执行方法
任务头已写入选中自我能力值
```

如果缺少执行方法，执行函数返回执行缺失信号，由自我线程展开原子需求；它不能再派生“缺方法需求”，也不能绕回任务执行本能方法。

## 4. 执行结果

执行函数只负责一次真实调用和运行账落地：

```text
调用目标方法
记录任务执行记录
写任务状态 / 错误码 / 结果场景
生成真实方法调用动作动态
发现实际缺失时输出原子需求消息
```

需求是否满足不由执行函数直接宣布。执行完成后，自我线程读取状态比较服务和需求目标关系来判断是否满足。

## 5. 废弃口径

以下口径已废弃：

```text
自我_任务执行一步 本能方法
任务执行一步待重入结果码
筹办后创建执行步骤节点再调用
通过步骤节点读取当前方法
任务执行作为方法查找可命中的能力
```
