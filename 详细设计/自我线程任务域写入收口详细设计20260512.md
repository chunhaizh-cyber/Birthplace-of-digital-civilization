# 自我线程任务域写入收口详细设计 20260512

## 1. 设计结论

按 2026-05-12 最新线程边界：

```text
自我线程：
    管需求树、需求生长、需求估权、任务发起意图、调度意图、需求回写。

任务管理线程：
    管任务创建提交、任务虚拟存在维护、任务状态推进、任务筹办、任务执行、任务回执同步、任务状态动作动态。
```

硬规则：

```text
只有任务管理线程可以写任务状态。
```

自我线程只能发出任务域意图或请求，不能直接提交任务域状态。

本详细设计是后续实施计划的约束依据。实施时必须先对照本文件确认每个写入点归属。

## 2. 当前代码越界点

基于 `自我线程模块.impl.cpp` 2026-05-12 检查，当前仍有以下任务域写入混在自我线程内。

| 位置 | 当前行为 | 冲突点 | 目标归属 |
| --- | --- | --- | --- |

| `私有_确保任务承接权重已固化_线程侧` | 写任务承接权重上限、任务评分和权重版本 | 自我线程直接写任务权重字段 | 自我线程计算建议，任务管理线程固化 |

## 3. 目标边界

### 3.1 自我线程可写

```text
需求节点：
    需求树父子关系
    需求状态
    需求权重
    派生归因
    需求结算账本

自我存在：
    安全值 / 服务值等自我核心状态

自我线程运行态：
    当前主需求
    调度意图
    最近请求 / 回执快照
```

### 3.2 自我线程只读或只生成请求

```text
任务信息节点：
    只读任务状态、任务来源需求、任务主键、任务列表引用。

任务虚拟存在：
    只读任务状态、当前方法、候选方法集、筹办/执行回执。

任务状态推进：
    只能生成请求，不能直接写。
```

### 3.3 任务管理线程可写

```text
任务信息节点
任务虚拟存在
任务状态
当前方法
任务方法候选集
可执行输入参数场景
筹办回执同步
执行回执同步
任务状态动作动态
任务承接权重
任务折算值
```

## 4. 请求与回执设计

以下请求是线程协议，不是世界树语义结构，不进入概念树。

### 4.1 任务发起请求

用途：

```text
自我线程发现有效执行需求，但没有承接任务时，提交给任务管理线程。
```

字段建议：

```cpp
struct 结构_任务发起请求 {
    需求节点类* 来源需求 = nullptr;
    存在节点类* 发起存在 = nullptr;
    场景节点类* 所属场景 = nullptr;
    I64 建议安全权重 = 0;
    I64 建议服务权重 = 0;
    I64 建议调度优先级 = 0;
    时间戳 当前时间 = 0;
};
```

任务管理线程收到后负责：

```text
创建任务信息节点。
创建任务虚拟存在。
写任务状态 = 未启动。
绑定来源需求。
初始化任务语义。
返回任务创建回执。
```

### 4.2 任务调度请求

用途：

```text
自我线程选出已有主任务后，只表达调度意图。
```

字段建议：

```cpp
struct 结构_任务调度请求 {
    任务节点类* 任务信息节点 = nullptr;
    需求节点类* 来源需求 = nullptr;
    I64 调度权重 = 0;
    时间戳 当前时间 = 0;
};
```

任务管理线程收到后负责：

```text
判断任务状态是否允许推进。
必要时排队、筹办或执行。
提交任务状态变化。
```

### 4.3 任务状态推进请求

用途：

```text
自我线程因派生需求入树、派生需求满足、需求回写等原因，建议某任务进入等待或待重筹办。
```

字段建议：

```cpp
struct 结构_任务状态推进请求 {
    任务节点类* 任务信息节点 = nullptr;
    枚举_任务状态 建议任务状态 = 枚举_任务状态::未定义;
    枚举_任务阶段 建议任务阶段 = 枚举_任务阶段::未定义;
    const 语素入口节点类* 原因类型 = nullptr;
    时间戳 当前时间 = 0;
};
```

任务管理线程收到后负责：

```text
验证状态迁移是否合法。
提交任务状态。
生成任务状态动作动态。
```

### 4.4 任务权重固化请求

用途：

```text
自我线程计算需求承接权重，但不直接写任务字段。
```

字段建议：

```cpp
struct 结构_任务权重固化请求 {
    任务节点类* 任务信息节点 = nullptr;
    需求节点类* 来源需求 = nullptr;
    I64 建议承接安全权重上限 = 0;
    I64 建议承接服务权重上限 = 0;
    I64 建议任务权重评分 = 0;
    时间戳 当前时间 = 0;
};
```

任务管理线程收到后负责：

```text
写任务承接安全权重上限。
写任务承接服务权重上限。
写任务权重评分。
写任务权重版本号。
```

### 4.5 任务价值结算回执

用途：

```text
自我线程结算需求与自我值后，把任务折算账建议回送给任务管理线程。
```

字段建议：

```cpp
struct 结构_任务价值结算回执 {
    任务节点类* 任务信息节点 = nullptr;
    需求节点类* 来源需求 = nullptr;
    I64 实际安全增量 = 0;
    I64 实际服务增量 = 0;
    时间戳 当前时间 = 0;
};
```

任务管理线程收到后负责：

```text
写任务已折算安全值。
写任务已折算服务值。
写任务价值反馈镜像。
```

## 5. 阶段流程设计

### 5.1 需求整理阶段

旧流程：

```text
刷新需求列表
顺手确保每个需求相关任务
缺主任务时直接创建任务头
```

新流程：

```text
刷新需求列表
刷新需求权重
选择主导需求
只查找已有承接任务
生成任务发起意图快照
```

不得创建任务。

### 5.2 主派发阶段

旧流程：

```text
如果有需求但无任务，自我线程创建任务头。
如果有任务，自我线程提交任务治理请求。
```

新流程：

```text
if (有主导需求 && 无承接任务) {
    构造任务发起请求;
    投递任务管理线程;
    return;
}

if (有主导任务) {
    构造任务调度请求或任务治理请求;
    投递任务管理线程;
    return;
}
```

自我线程不创建任务头。

### 5.3 派生需求入树

旧流程：

```text
派生需求入树
直接为派生需求创建任务
直接把关联任务写为等待中
```

新流程：

```text
派生需求入树
刷新需求结构角色
为新需求投递任务发起请求
为关联任务投递等待状态推进请求
```

任务状态由任务管理线程写。

### 5.4 派生需求解除等待

旧流程：

```text
派生需求满足或已调度
自我线程直接标记父任务待重筹办
```

新流程：

```text
派生需求满足或已调度
自我线程投递任务重筹办请求
任务管理线程验证并提交 待重筹办
```

### 5.5 成功结果回流

旧流程：

```text
任务成功结果回流
写来源需求当前状态
写任务结果状态信息
执行需求结算和任务折算
唤醒父任务
```

新流程：

```text
任务成功结果回流
自我线程只写来源需求当前状态和需求结算
任务结果状态信息由任务管理线程写
任务折算值由任务管理线程写
父任务唤醒改为重筹办请求
```

### 5.6 任务价值反馈

旧流程：

```text
自我线程写自我安全 / 服务值
自我线程写来源需求累计结算
自我线程写任务已折算安全 / 服务值
```

新流程：

```text
自我线程写自我安全 / 服务值
自我线程写来源需求累计结算
自我线程发送任务价值结算回执
任务管理线程写任务已折算安全 / 服务值
```

## 6. 函数迁移表

| --- | --- |
| `私有_按需求创建任务头_任务管理统一挂接` | 迁入任务管理线程，改为 `任务管理线程_按需求创建任务并初始化虚拟存在` |
| `私有_按需求创建任务头_任务管理新链` | 删除或改为构造任务发起请求的薄壳 |

## 7. 验收规则

### 7.1 静态反查

自我线程模块中不得再出现直接任务域写入：

```text
任务类::写入任务状态
任务类::标记任务待重筹办
任务类::写入任务当前阶段
任务类::同步任务运行镜像特征
任务类::写入任务已折算安全值
任务类::写入任务已折算服务值
任务头->主信息.结果状态信息 =
```

允许出现：

```text
任务状态读取
任务来源需求读取
任务主键读取
构造任务请求
投递任务请求
```

### 7.2 行为验收

1. 有需求但无任务时，自我线程提交任务发起请求，任务由任务管理线程创建。
2. 派生需求入树后，自我线程不直接写关联任务等待中，而是投递等待请求。
3. 派生需求满足后，自我线程不直接标记父任务待重筹办，而是投递重筹办请求。
4. 任务成功结果回流后，自我线程只更新需求域和自我值；任务结果状态和折算值由任务管理线程更新。
5. `rg` 静态反查清零后，Debug x64 构建通过。

## 8. 与其他计划的顺序

本设计必须先于以下计划落地：

```text
需求树真实生长自动化剧本
任务筹办 / 任务执行三通道回执完整接入
内部循环闭合验收
自我苏醒验收
因果补完与学习需求正式入树
```

原因：

```text
若自我线程仍直接写任务状态，后续需求树生长和任务闭环验收无法判断状态变化来自哪个线程。
```
