# 20260411 M1 到 M4 桥接后续实施计划

## 目标

在现有 `P0.5` 桥接补层基础上，继续把 `方法需求位 -> 本能能力缺口 -> 最小原语` 的链路从“可见”推进到“可验”，优先完成 `M1：P0.6 半白盒真实触发`，再为 `M2/M3/M4` 留出稳定结构面。

## 当前判断

- `P0.5` 已完成：桥接三阶段、早停闸门、四层薄壳、六原语与派生规则分层都已接入。
- `M1` 已完成：`触发来源 / 来源主观察特征 / 来源最小原语位图 / 影子验证状态 / 是否允许正式资产提交` 已贯通到请求、决策、一步结果、恢复快照，并完成 5 场景验收。
- `M2` 已完成：`执行一步治理(...)` 已显式切成 6 段，普通尾段也统一走阶段收口；桥接字段不再只靠兼容壳和摘要存活。
- `M3` 已完成：任务管理侧已经补出 `学习承接决策` 结构，并把“补方法 / 补原语或治理本能”显式放行接进主治理链；学习同步输入、学习实体、学习执行摘要已贯通 `触发来源 / 承接类型 / 影子验证 / 正式提交 / 承接摘要`，并新增“等待正式提交放行”闸门与白盒门槛验收场景。
- `M4` 已完成：恢复快照已补齐等待态与学习回流投影；线程启动自故障恢复时会一次性激活 `恢复重建` 请求，首轮重回根层重判，随后自动回落到普通主循环，不越权连续推进。

## 实施顺序

### M1：P0.6 半白盒真实触发

1. 贯通字段
   - 把 `触发来源 / 来源主观察特征 / 来源最小原语位图 / 影子验证状态 / 是否允许正式资产提交` 接进：
   - `结构_任务管理请求`
   - `结构_任务管理单步决策`
   - `结构_任务管理一步结果`
   - `结构_治理恢复快照`

2. 加半白盒控制
   - 新增 `设置半白盒治理控制(...) / 清除半白盒治理控制()`
   - 只允许两类局部遮蔽：
   - `缺状态差分取证`
   - `缺轻量因果链组织`
   - 遮蔽只发生在桥接层，不改学习链和线程池语义

3. 让真实治理链消费半白盒控制
   - 在 `执行一步治理(...)` 的前半段顺序上保持：
   - `形成治理特征面`
   - `命中方法需求位`
   - `应用半白盒桥接屏蔽`
   - `评估本能能力缺口`

4. 扩展一键验收
   - `--acceptance-report` 扩成 5 场景：
   - 运行态任务治理摘要
   - 自然运行态补方法
   - 半白盒补原语：缺状态差分取证
   - 半白盒补原语：缺轻量因果链组织
   - 白盒补原语/治理本能

5. 恢复快照入验收
   - 半白盒场景必须同时校验：
   - `触发来源=半白盒`
   - `影子验证状态` 保留
   - `允许正式资产提交=否`
   - 恢复后字段不丢、不串、不误升级

### M2：P0 结构拆分

1. 维持外部接口不变
2. 让桥接字段稳定留在新四层结构里，而不是只活在兼容汇总壳和摘要中
3. 把 `执行一步治理(...)` 优先显式切成 6 段：
   - 受理请求
   - 宿主锚点确认
   - 上位输入约束应用
   - 形成单步决策
   - 形成写回结果
   - 组装一步结果

### M3：P1 学习承接正式化

1. 从“流程尾部尝试学习”改成“显式放行学习”
2. 正式区分：
   - `补方法`
   - `补原语/治理本能`
3. 继续保持：
   - `影子验证 -> 回滚`
   - 默认禁止正式资产提交

### M4：P2 运行模型升级

1. 已完成等待态与恢复语义投影
   - `结构_治理恢复快照` 现已补齐：
   - `最近学习承接摘要`
   - `最近学习等待摘要`
   - `最近学习回流摘要`
   - `存在待消费学习回流`
   - `存在待消费外部反馈`
2. 已完成恢复重建一次性消费
   - 新增 `设置治理恢复重建控制(...) / 清除治理恢复重建控制()`
   - 自我线程若“本次启动来自故障恢复”，会把恢复快照装配成一次性 `恢复重建` 请求
   - 首轮请求命中 `恢复重建` 后立即消费，第二轮自动回到普通事件类型
3. 已完成线程侧恢复投影
   - `结构_自我线程快照` 已补出：
   - `任务管理恢复点类型`
   - `任务管理恢复投影摘要`
   - `任务管理恢复存在待消费学习回流`
   - `任务管理恢复存在待消费外部反馈`
4. 当前不前置 mailbox/额外等待模型
   - 仍保持“恢复后先回主循环根层重判，再围绕一个宿主推进一步”的边界
