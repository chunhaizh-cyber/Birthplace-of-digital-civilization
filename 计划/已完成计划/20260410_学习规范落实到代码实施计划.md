## 3. 当前实现结论

按当前代码，学习并不是“完全没有”，而是已经存在一层明显的治理侧挂接，但还没有成为独立学习运行时。

### 3.1 当前已经具备的部分

1. `任务管理任务模块` 已经能在治理轮中派生学习子任务：
   - 能创建或复用 `学习子任务` 头结点
   - 能在 `任务管理学习树 -> 定向学习分支` 下挂学习子任务

2. `任务管理任务模块` 已经能写出最小学习方法骨架：
   - 创建或复用 `学习方法头`
   - 创建学习方法 `条件节点`
   - 创建学习方法 `结果节点`
   - 把来源任务、动作名、条件场景、结果场景挂到方法树

3. 自我特征里已经有学习相关占位：
   - `待学习方法数量`
   - `尝试学习状态`

4. 控制面板和文本摘要已经能读到一部分学习痕迹：
   - `待学习方法数量`
   - 最近一轮是否创建/复用学习子任务
   - 最近一轮是否创建/复用学习方法头与条件/结果节点

### 3.2 当前最关键的现实判断

当前学习实现本质上仍属于：

`任务管理任务在治理时顺手补学习挂点`

它还不是规范要求的：

`独立学习任务实体 -> 学习任务表 -> 就绪队列/等待表 -> 共享学习执行器 -> 学习结果回流 -> 外层重判`

### 3.3 当前主要缺口

1. 还没有独立 `学习模块` 源码目录与运行时实体。
2. 还没有把 `学习任务最小状态面` 明确成独立结构。
3. 还没有 `学习任务表 / 就绪队列 / 等待表 / 去重键 / 合并键`。
4. 还没有 `采样 -> 更新 -> 验证 -> 提交 / 回滚` 阶段机。
5. 还没有把 `学习对象原料 -> 待学习方法骨架` 独立成对象适配器。
6. 还没有把 `因果信息` 作为对象来源之一接成独立适配通道。
7. 还没有“学习结果先回学习任务实体，再回任务管理与主循环”的正式闭环。
8. 当前 `待学习方法数量` 还是从树上数出来的，不是从独立学习任务账本读出来的。
9. 外层主循环还没有正式消费“学习结果事件”，也没有做学习结果后的重判与再筹办。

## 4. 规范收口后的最小正确链

后续代码实现必须统一按这条链理解，不再把学习简化成“直接在因果里找一下然后试试”：

`任务管理推进`
-> `上浮学习衍生需求`
-> `形成学习目标`
-> `选择学习对象原料`
-> `对象适配器抽成待学习方法骨架`
-> `独立创建/更新学习任务实体`
-> `进入学习任务表`
-> `调度器判定就绪/等待/合并`
-> `共享学习执行器执行`
-> `学习结果先回学习任务实体`
-> `再回任务管理/方法侧/宿主任务侧`
-> `返回外层重判`

必须特别固定：

1. `学习任务` 不是第三根任务，而是 `任务管理内部治理子任务`。
2. `因果信息` 是学习原料来源之一，不是唯一主流程。
3. 若当前只有 `需求信息 / 目标特征类型`，只能形成 `需求位方法头`，不得直接进入正式学习执行。
4. 可执行学习任务的最小前提是：至少能落成 `待学习方法骨架`，且已有 `动作骨架 + 条件侧`。
5. 学习执行器不能替代外层最终裁决。


## 5. 实现红线

后续落实到代码时，必须一直遵守以下红线：

1. 不改现有 `需求 / 任务 / 方法 / 存在 / 自我` 的领域语义边界。
2. 可以新增模块、运行时容器、状态结构、日志与面板快照，但不得另起并行真值层。
3. `任务管理任务模块` 负责生成和更新学习任务，不负责独吞学习全流程执行。
4. `学习调度器` 负责排队、等待、合并、分派，不负责越权裁决最终去向。
5. `共享学习执行器` 负责 `采样 / 更新 / 验证 / 提交 / 回滚`，不负责替代学习任务实体保存来源语义。
6. 任一学习结果被本轮消费后，仍必须回到外层做 `根层重判 -> 上下文刷新 -> 再决定下一轮`。
7. 学习写回必须按“可补偿、可重试、可解释”的补建式链路实现，不能假设跨多步天然事务化。

## 8. 推荐执行顺序

建议严格按以下顺序推进：

1. `L0 固化当前学习挂接基线`
2. `L1 独立学习任务实体与状态面`
3. `L2 任务管理改成创建/更新学习任务实体`
4. `L3 学习对象选择器与对象适配器`
5. `L4 学习任务表/就绪队列/等待表`
6. `L5 学习阶段机与共享学习执行器`
7. `L6 学习结果回流与外层重判`
8. `L7 观测与审计`

原因：

1. 没有 `L0`，后续拆学习模块时容易把当前已有学习挂接面拆丢。
2. 没有 `L1`，后面所有学习调度、学习执行、学习回流都会继续散落在布尔值和树节点里。
3. `L2` 先做，能尽早把“任务管理生成学习需求”和“学习模块执行学习”切开。
4. `L3` 必须早于 `L4/L5`，否则调度器和执行器会拿不到统一对象输入。
5. `L4/L5` 应晚于 `L1/L2/L3`，否则会先变成无语义的空调度框架。
6. `L6` 必须在调度和执行都跑通后再接，不然外层重判会接到半成品结果。

## 10. 本轮结论

当前最合理的推进方式，不是继续把学习逻辑塞进 `任务管理任务模块.cpp`，也不是直接跳到“完整学习算法”。
