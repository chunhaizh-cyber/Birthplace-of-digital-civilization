# 方法主信息角色面拆分计划 v0.1

日期：2026-05-06

## 1. 背景

`20260506_方法首节点结果能力面拆分计划_v0.1.md` 只先拆出结果能力面，避免一次性重写方法树。完成该计划后，方法查找已经可以只依赖 `首节点能力.结果能力集`。

下一步要处理的是 `方法信息基类` 仍然过胖的问题：方法首节点、方法条件节点、方法结果节点仍共用动作面、条件面、结果面、任务挂接面和统计字段。

## 2. 目标

将 `方法信息基类` 中的字段按角色面重新归组，但暂时仍保留在同一个主信息类内，先形成明确访问边界。

目标结构：

```cpp
struct 结构_方法节点公共信息;
struct 结构_方法首节点信息;
struct 结构_方法条件节点信息;
struct 结构_方法结果节点信息;
```

`方法信息基类` 临时承载：

```cpp
结构_方法节点公共信息 公共{};
结构_方法首节点信息 首节点{};
结构_方法条件节点信息 条件节点{};
结构_方法结果节点信息 结果节点{};
```

本计划不立即启用 `std::variant`，先以结构分组降低迁移风险。

## 3. 字段归属

### 3.1 公共信息

```text
描述信息
节点种类
统计
```

### 3.2 首节点信息

```text
方法虚拟存在
动作名
动作句柄
来源
作用域
来源任务
父方法
前置方法
后续方法
首节点能力
可靠度
成熟度阶段
是否允许自动查找
```

### 3.3 条件节点信息

```text
条件场景
条件主键
条件场景模板主键
条件判定索引
时间起
时间止
```

### 3.4 结果节点信息

```text
结果场景
结果主键
结果能力集
结果初始状态
```

旧字段先保留为兼容镜像，但所有新代码必须通过角色面访问。

## 4. 实施步骤

1. 在 `方法主信息类.h` 新增四个角色面结构。
2. 在 `方法信息基类` 中加入 `公共 / 首节点 / 条件节点 / 结果节点`。
3. 增加访问 helper：

```text
取首节点信息()
取条件节点信息()
取结果节点信息()
是否方法首节点()
是否方法条件节点()
是否方法结果节点()
```

4. 把 `方法类.cpp` 中写入动作、来源、作用域、方法虚拟存在、结果能力的逻辑迁移到首节点面。
5. 把条件场景和条件判定索引写入迁移到条件节点面。
6. 把结果场景、结果能力集、结果初始状态写入迁移到结果节点面。
7. 保持旧字段同步写入，作为兼容镜像。
8. 编译并清理无用旧访问点。

## 5. 验收项

1. 方法首节点新增、补齐、查找仍可运行。
2. 方法条件节点不再需要读取动作句柄。
3. 方法结果节点不再需要读取条件场景。
4. 结果能力查找仍只读首节点能力面。
5. 控制面板仍能展示方法首节点、条件节点、结果节点信息。
6. Debug x64 构建通过。

## 6. 暂不做

```text
删除旧字段
改链表模板
改为 std::variant
迁移运行账
```

这些由后续计划承接。
