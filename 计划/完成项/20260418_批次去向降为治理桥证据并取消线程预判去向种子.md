# 20260418 批次去向降为治理桥证据并取消线程预判去向种子

## 本步目标

继续削弱 `自我线程` 在主派发前半段对“去向种子”的本地预设，让 `批次执行去向` 直接作为治理桥证据输入，而不是先在线程里映射成 `当前最终去向 / 当前下一步去向候选`。

## 本步完成内容

1. 在 `自我_主循环治理模块` 的 `结构_一步治理输入_最小` 中新增：
   - `已携带主派发批次执行去向`
   - `主派发批次执行去向编码`
2. 治理桥内部新增“批次执行去向 -> 主循环去向”的最小解释函数，并在 `当前下一步去向候选` 仍未定义时，优先使用这份批次证据作为桥层初始去向。
3. 线程侧主派发预判装配不再先用 `批次去向` 映射出线程最终去向，再推导主循环去向；而是直接以：
   - `当前缺口 = 未定义`
   - `当前下一步去向候选 = 未定义`
   - `主派发批次执行去向编码 = 批次去向`
   交给治理桥。
4. `主派发快照` 正式带上了 `批次执行去向`，让桥层快照重读和运行时主链保持同一套证据面。
5. 线程侧的候选去向初始化也改成：
   - 优先使用治理桥预判建议
   - 只有在桥层没有给出建议时，才回落到本地批次去向映射

- `D:\鱼巢\自我线程模块.ixx`

## 当前效果

- `批次去向` 已经不是线程内部先吃掉、再转述给治理桥的隐式种子，而是桥层正式消费的证据。
- 主派发预判阶段的“初始去向”开始从线程侧搬到治理桥侧。
- 线程本地映射还保留，但已经退到“桥层无建议时的兜底回落”。
