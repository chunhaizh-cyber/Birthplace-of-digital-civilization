# 因果补完 C0c 学习需求目标构造器计划 v0.1

日期：2026-05-10

前置计划：

```text
20260510_因果补完C0b学习需求原子状态词典计划_v0.1.md
```

## 1. 目标

为学习需求正式入树前补一个独立目标构造层。

```text
因果缺口 / 学习候选
    -> 学习需求目标草案
    -> 原子当前状态
    -> 原子目标状态
    -> 完整性校验
```

本计划只构造和诊断学习目标，不写入需求树。

## 2. 边界

```text
1. 不创建学习需求。
2. 不修改需求树。
3. 不生成探索需求。
4. 不复制父需求目标。
5. 不使用人类词性直落树。
```

## 3. 目标主体规则

```text
学习条件：
    主体 = 当前任务虚拟存在 / 当前方法虚拟存在 / 缺口目标主体

学习状态：
    主体 = 缺失状态对应主体 / 任务虚拟存在

学习对象：
    主体 = 外部对象占位或任务虚拟存在

学习方法：
    主体 = 当前方法虚拟存在，若无则为目标方法头虚拟存在

学习风险：
    主体 = 当前动作 / 方法虚拟存在 / 任务虚拟存在

学习因果：
    主体 = 因果候选承载对象，第一版可暂用任务虚拟存在
```

目标主体不得默认使用父需求的被需求存在。

## 4. 特征类型规则

```text
学习条件 -> 学习信息补齐状态
学习状态 -> 学习信息补齐状态
学习对象 -> 学习信息补齐状态
学习方法 -> 学习方法可靠度状态
学习风险 -> 学习风险评估状态
学习因果 -> 学习因果验证状态
```

## 5. 状态规则

```text
信息补齐类：
    当前状态 = 未补齐
    目标状态 = 已补齐

因果验证类：
    当前状态 = 未验证
    目标状态 = 已验证

方法可靠类：
    当前状态 = 不可靠
    目标状态 = 可靠

风险评估类：
    当前状态 = 未评估
    目标状态 = 已评估
```

满足关系统一为：

```text
关系_等于
```

## 6. 拟新增结构

```cpp
struct 结构_学习需求目标草案 {
    bool 有效 = false;

    可解析引用<基础信息节点类> 目标主体{};
    const 语素入口节点类* 目标特征类型 = nullptr;

    可解析引用<状态节点类> 当前状态{};
    可解析引用<状态节点类> 目标状态{};

    三向关系掩码 满足关系 = 关系_等于;

    std::string 学习候选类型{};
    std::string 缺口类型{};
};
```

## 7. 拟新增函数

```cpp
结构_学习需求目标草案 构造学习需求目标草案(
    const 结构_因果补完诊断快照& 因果诊断,
    时间戳 now
);

bool 校验学习需求目标草案(
    const 结构_学习需求目标草案& 草案
);
```

校验失败属于正常业务缺口，不弹错；但如果进入入树流程时草案仍不完整，应按非正常逻辑错误报错并记录日志。

## 8. 验收

```text
1. Debug x64 构建通过。
2. `--self-loop-summary` 可展示最近学习候选及目标构造是否完整。
3. 不新增需求节点。
4. 不改变现有需求树数量。
5. 目标构造失败时能明确显示缺少主体 / 特征 / 状态中的哪一项。
```
