# 20260418 主派发逐类路由细则前推为治理桥原生消费

## 本步目标

让 `自我_主循环治理模块` 不只消费 `主派发判据二次特征` 和 `主派发路由原语摘要`，而是开始直接消费“覆盖后的逐类路由细则”。

## 本步完成内容

1. 在 `结构_一步治理输入_最小` 中新增主派发逐类路由细则输入：
   - `主派发细则允许继续提交级`
   - `主派发细则允许重筹办级`
   - `主派发细则允许待机级`
   - `主派发细则允许收束级`
   - `主派发逐类路由摘要`
   - `主派发逐类路由细则摘要`
   - `主派发多证据覆盖摘要`
   - `主派发多证据联合裁决摘要`
2. 在 `结构_一步治理结果_最小` 中新增 `已消费主派发逐类路由细则`，并把逐类路由摘要/细则摘要回写到桥层结果。
3. 治理桥在 `当前去向 == 执行方法` 的分支下，开始原生消费几类确定型细则：
   - 只允许 `重筹办级` 时，桥层直接回 `继续治理`
   - 只允许 `待机级` 时，桥层直接落到 `待机`
   - 只允许 `收束级` 时，桥层直接改入 `收束链`
4. 线程运行态与快照重读两条链都已喂入相同的逐类路由细则输入，避免 runtime 与 snapshot 口径分叉。

## 主要改动文件

- `D:\鱼巢\自我_主循环治理模块.ixx`
- `D:\鱼巢\自我线程模块.impl.cpp`

## 当前效果

- 治理桥第一次开始直接理解“覆盖后的主派发逐类路由细则”，而不是只看一段总摘要。
- 对于 `只许重筹办 / 只许待机 / 只许收束` 这类确定型路由细则，桥层已经可以直接给出原生裁决。
- 复杂、仍需线程局部补充判据的场景暂时保留原口径，没有在本轮强行硬改。
