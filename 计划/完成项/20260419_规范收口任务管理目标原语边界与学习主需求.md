## 收口内容

### 1. 任务管理任务的目标

正式收口为：

- `任务管理任务` 与普通任务遵循同一条主流程：
  `筹办条件 -> 条件满足 -> 运行方法 -> 比对结果与目标 -> 满足则完成，不满足则继续完善任务树`
- `任务管理任务` 的唯一终局目标，是推动 `宿主任务状态进入完成`

### 2. 原语 / 本能函数边界

正式收口为：

- `任务管理任务` 改写 `宿主任务虚拟存在` 中的内部治理特征
- `普通任务` 改写 `世界树` 中的对象/场景特征
- `任务管理任务` 使用的最小内部方法统一称 `原语`
- `普通任务` 使用的外部最小方法统一称 `本能函数`

并把 `1115` 中残留的 `最小本能方法集` 词面统一改为 `最小原语集`

### 3. 学习主需求

正式收口为：

- 当 `任务管理任务` 发现当前任务没有可直接命中需求的方法时，应先绑定或创建一个“仅含结果(需求)”的 `待学习的方法首节点`
- `任务管理任务` 对外生成的顶层学习需求，应优先收口为：
  `让当前待学习方法头可用`
- `样本收集 / 差异比较 / 共同条件提取 / 候选动作生成 / 结果预测 / 沙箱验证 / 风险评估 / 边界标记`
  这些不再充当顶层需求，而下沉为满足主需求的 `学习动作` 或 `原语级内部最小方法组合`
