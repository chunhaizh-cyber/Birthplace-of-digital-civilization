# 2026-04-18 主派发判据原语收口并消除线程内重复推导

## 本次目标

把 `自我线程` 中“主派发判据”的重复私算收口成一套统一原语，让：

- `步骤_生成主派发判定二次特征_`
- `步骤_生成并执行主派发决议_`

都建立在同一份主派发判据计算结果之上，再由二次特征做覆盖，而不是线程内各算一套。

## 已完成

1. 在 [自我线程模块.impl.cpp](/D:/鱼巢/自我线程模块.impl.cpp) 中新增并落稳 `结构_主派发判据原语结果` 与 `私有_计算主派发判据原语(...)`。
2. 统一收口了主派发前半段的核心证据推导：
   - 来源需求候选数 / 方法禁止项数
   - 需求联合压力值 / 禁止联合压力值
   - 绝对禁止 / 已验证降根 / 目标反向
   - 收益估计值 / 禁止付出值
   - 长期根方向否决 / 可改路由证据 / 仅能降级证据
   - 联合需求主导 / 联合禁止主导 / 联合僵持
   - 决定性需求候选摘要 / 决定性方法禁止项摘要 / 联合候选摘要
3. `步骤_生成主派发判定二次特征_` 已改为直接消费这份原语结果来写第一批主派发判据二次特征。
4. `步骤_生成并执行主派发决议_` 已改为先读取同一份原语结果，再叠加场景二次特征覆盖，后续裁决继续沿原有链路执行。
5. 主派发函数里原先那一整段本地重复的“前3候选/联合压力/禁止类别/收益付出”推导已经移除，不再和二次特征生成阶段各算一套。

## 本次口径

- 现在 `主派发判据二次特征` 和 `主派发决议` 的底层母式已经统一。
- 二次特征阶段负责把原语结果写成证据节点。
- 主派发阶段负责读取同一套原语结果，并允许二次特征在场景层做最终覆盖。
- 这一步的重点是“先统一证据生成母式”，不是改写现有主派发高层裁决规则。
