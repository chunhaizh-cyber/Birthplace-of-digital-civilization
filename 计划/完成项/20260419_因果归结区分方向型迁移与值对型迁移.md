# 20260419 因果归结区分方向型迁移与值对型迁移

## 本次完成

- 在 `因果类.cpp` 中把状态迁移归结正式拆成两类：
  - `方向型迁移`
  - `值对型迁移`
- 对于 `差值型` 状态迁移：
  - 继续使用 `方向区间 / 差值 / 标量区间` 做方向一致性验证。
- 对于 `判等型 / 差异度型` 状态迁移：
  - 不再强行解释为“增加 / 减少”；
  - 改为按 `因值 -> 果值` 这组状态对进行归结；
  - 只有值对完全一致，才允许并入同一条抽象因果。

## 关键修正

- `私有_由状态生成状态迁移特征(...)`
  - 对 `值对型` 迁移不再保留伪方向；
  - 会清空差值/方向，并统一把迁移概念改成 `变为`。
- `私有_按动态补齐抽象因果模板(...)`
  - 对 `方向型` 迁移继续做方向一致性校验；
  - 对 `值对型` 迁移改成“值对一致性校验”；
  - `值对型` 迁移如果值对不同，不再错误并入既有模板。

## 当前口径

- 数值型特征：
  - `条件范围 + 动作 + 方向一致的状态迁移`
  - 才归入同一条抽象因果。
- 枚举型 / 无序型特征：
  - `条件范围 + 动作 + 因值对果值一致`
  - 才归入同一条抽象因果。
- 因此：
  - 数值型因果看方向；
  - 枚举型因果看值对。
