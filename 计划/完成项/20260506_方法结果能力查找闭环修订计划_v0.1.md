# 20260506 方法结果能力查找闭环修订计划 v0.1

状态：待实施。

## 1. 目标口径

方法查找只回答一个问题：

```text
有没有方法能够把目标特征类型，按需求要求的方向改变？
```

固定定义：

```text
需求结果 = 目标特征类型 + 目标方向
方法结果 = 主结果特征类型 + 结果方向索引
```

找不到就是找不到。查找失败表示当前方法树没有任何方法声明能造成该需求要求的结果变化。不得改按方法名称、动作名称、动作句柄、本能 ID、固定编号或学习方法白名单取方法。

## 2. 当前实现检查结论

### 2.1 已具备的结构基础

`方法主信息类.h` 已有结果面字段：

```cpp
const 词性节点类* 主结果特征类型 = nullptr;
std::string 结果主键{};
std::vector<可解析引用<二次特征节点类>> 结果方向索引{};
```

`方法类.cpp` 已有 `注册需求位方法首节点`、`注册需求方向方法首节点`，并且内部已经有按 `主结果特征类型 + 结果方向索引` 找需求方向方法头的私有逻辑。

### 2.2 三个学习本能的结果声明不符合新口径

`方法类.cpp` 的 `私有_查询本能方法结果声明集` 当前把：

```text
自我_练习
自我_尝试学习
自我_定向学习
```

合并声明为：

```text
方法是否可用 -> 增加
方法可靠度 -> 增加
可用方法数量 -> 增加
```

其中只有 `方法是否可用 -> 增加` 参与方法方向索引。结果是三者无法按各自真实结果能力区分，`候选方法存在状态` 等内部闭环需求不会稳定命中对应学习方法，外界信息链里的 `学习方向状态` 也无法被清晰约束到定向学习。

内部方法补齐闭环还需要特别约束：自我内部环境里的因果信息已经有动作主体、动作函数、起始状态和结果状态，所以尝试学习阶段已经具备形成方法候选所需的方向材料，不应该再生成 `学习方向状态 从未形成到已形成` 的需求去触发定向学习。

### 2.3 `查找方法` 本能仍是能力包匹配

`自我动作实现模块.ixx` 的 `查找方法` 当前流程是：

```text
绑定需求条件包
枚举候选方法
读取方法能力包
匹配需求包与能力包
按距离/成功率/耗时排序
输出方法配对结果
```

这仍是 `需求条件包 ⊆ 方法能力包` 口径。它应改为先解析 `需求结果`，按方法首节点的 `主结果特征类型 + 结果方向索引` 命中候选，再用能力包和上下文条件过滤。

### 2.4 执行桥存在半成品结果查找，但有绕路

`自我_执行桥模块.ixx` 已有 `私有_查找方法首节点_按结果特征_执行桥_最小_v0`，会按目标特征类型和需求方向扫描方法树。

问题：

1. 方向未命中时会退回同特征 `特征候选`，这违反“目标方向必须覆盖需求方向”。
2. 找不到时会调用 `方法类::注册需求方向方法首节点` 创建占位方法头，不能等价于“找到可执行方法”。
3. 任务执行阶段还可通过 `选中自我能力值` 调 `本能方法类::取或确保本能方法首节点_按能力值` 解析方法，必须限定为“已由结果查找命中的执行入口缓存”，不能作为查找兜底。

### 2.5 名称/动作句柄查找入口仍存在

`方法类::查找方法首节点_按动作句柄` 和 `方法类::查找方法首节点_按动作名` 仍是公开接口，并被本能方法确保、注册和执行入口解析使用。

这些接口可以保留给本能注册、执行入口解析和调试，但不得被 `查找方法`、任务筹办或缺口分派当作语义查找依据。

## 3. 目标结果声明

### 3.1 定向学习

```text
主结果特征类型：
  学习方向状态

结果方向：
  已形成

补充结果：
  目标特征类型识别状态 -> 已识别
  目标变化方向识别状态 -> 已识别
  学习约束状态 -> 从无到有
  方法需求位识别状态 -> 已命中
```

触发域：

```text
只属于外界信息处理链：
  外界信息
  未归因外部现象
  动作主体未知
  动作函数未知
  需要观察需求 / 确认需求 / 信息需求
```

自我内部方法补齐闭环不得生成定向学习可满足的需求。

### 3.2 尝试学习

```text
主结果特征类型：
  候选方法存在状态

结果方向：
  已存在

补充结果：
  方法动作骨架状态 -> 已形成
  方法条件结果配对状态 -> 已配对
  方法结构是否完整 -> 更完整
  方法最小出生锚点 -> 具备
```

### 3.3 练习

```text
主结果特征类型：
  方法可靠度

结果方向：
  增加

补充结果：
  方法稳定状态 -> 更稳定
  方法是否可用 -> 趋向可用
  方法运行账完整性 -> 更完整
  方法成熟度阶段 -> 上升
  方法新用途发现状态 -> 发现候选
```

## 4. 修改计划

### 阶段一：补齐结果特征字典

1. 在方法/自我特征定义侧补齐固定特征词：
   `学习方向状态`、`目标特征类型识别状态`、`目标变化方向识别状态`、`候选方法存在状态`、`方法动作骨架状态`、`方法条件结果配对状态`、`方法新用途发现状态`。
2. 复用已有 `方法结构是否完整`、`方法可靠度`、`方法稳定状态`、`方法是否可用`、`方法运行账完整性`、`方法成熟度阶段`。
3. 固定方向语义：`增加`、`已形成`、`已识别`、`已存在`、`已配对`、`更完整`、`更稳定`、`趋向可用`、`上升`。

### 阶段二：拆分本能方法结果声明

1. 修改 `方法类.cpp::私有_查询本能方法结果声明集`。
2. `自我_练习` 主索引改为 `方法可靠度 -> 增加`。
3. `自我_尝试学习` 主索引改为 `候选方法存在状态 -> 已存在`。
4. `自我_定向学习` 主索引改为 `学习方向状态 -> 已形成`。
5. 补充结果写入结果场景；练习的新用途发现必须写为同一动作下的新增条件结果对候选，包含 `输入参数区间 + 新结果特征类型 + 新结果方向`。
6. 只有主结果或明确允许的结果进入 `结果方向索引`，避免一个方法用过宽索引抢占其他需求。
7. 定向学习的方法条件必须带上外界信息处理上下文，或未归因外部现象，或动作主体/动作函数未知；内部因果样本来自自我内部动作记录且动作主体、动作函数已知时，不满足定向学习条件。

### 阶段三：新增公开结果查找接口

在 `方法类` 增加只读查找接口：

```cpp
static std::vector<节点类*> 查找方法首节点_按需求结果(
    节点类* 方法根节点,
    const 词性节点类* 需求目标特征类型,
    const 二次特征节点类* 需求目标方向);
```

匹配规则：

```text
1. 节点必须是方法首节点。
2. 主结果特征类型必须覆盖需求目标特征类型。
3. 结果方向索引必须覆盖需求目标方向。
4. 没有方向索引时不能命中带方向需求。
```

### 阶段四：改造 `查找方法` 本能

1. `结构_查找方法输入帧` 增加 `需求目标特征类型`、`需求目标方向`。
2. 输入绑定从 `需求条件包` 解析需求结果；保留上下文条件包。
3. 候选枚举改为先调用 `查找方法首节点_按需求结果`。
4. `方法能力包` 仅过滤上下文条件，不再作为第一匹配依据。
5. 失败输出统一为“当前方法树无结果能力声明可满足该需求”，并记录目标特征、目标方向、扫描数量。
6. 删除或隔离任何按学习方法 ID 兜底写入 `选中自我能力值` 的路径。

### 阶段五：收紧执行桥和任务筹办

1. `私有_查找方法首节点_按结果特征_执行桥_最小_v0` 删除同特征无方向退回。
2. 找不到时不得把新注册的需求方向方法头当作命中方法；只允许生成缺口或占位，并返回“未找到可执行方法”。
3. `取或确保本能方法首节点_按能力值` 只允许执行阶段使用，并且必须有来自配对结果的缓存能力值。
4. 任务筹办不再因缺口类型直接写 `自我_练习`、`自我_尝试学习`、`自我_定向学习`，只能生成结果需求重新查找。
5. 自我内部方法补齐闭环不得生成 `学习方向状态 -> 已形成` 需求；练习缺候选方法时只能生成 `候选方法存在状态 -> 已存在` 需求，命中尝试学习。
6. 尝试学习失败时不得转向定向学习；失败原因必须细化为内部因果样本缺失、样本与方法补齐入口不匹配、结果特征类型无法对应当前需求、条件不足、动作函数不能转成方法动作骨架，或方法结果方向无法稳定归纳。

### 阶段六：验证

1. 启动后检查三个学习本能方法首节点：
   - `练习`：`主结果特征类型 = 方法可靠度`，方向包含 `增加`。
   - `尝试学习`：`主结果特征类型 = 候选方法存在状态`，方向包含 `已存在`。
   - `定向学习`：`主结果特征类型 = 学习方向状态`，方向包含 `已形成`。
2. 在自我内部方法补齐上下文中，构造 `候选方法存在状态 -> 已存在`，应命中 `尝试学习`。
3. 在自我内部方法补齐上下文中，构造 `方法可靠度 -> 增加`，应命中 `练习`。
4. 在自我内部方法补齐上下文中，不得生成或命中 `学习方向状态 -> 已形成`。
5. 在外界信息处理上下文中，构造 `学习方向状态 -> 已形成`，才允许命中 `定向学习`。
6. 构造不存在的结果需求，应返回查找失败，不产生本能 ID 兜底。
7. 运行控制面板或摘要线程，确认方法树显示 `主结果特征类型` 与 `结果方向索引`。

## 5. 风险与注意

1. 旧链路大量使用 `选中自我能力值` 作为长期权威，改造时要先保留字段，但把语义降级为执行入口缓存。
2. 一些管理本能仍需要按 ID 确保本能方法首节点，这属于注册/执行，不属于语义查找；必须在代码注释和调用层隔离。
3. 结果方向当前用二次特征表达，需要统一“已形成/已存在/已识别”等离散方向与差值方向的编码，否则会出现文本相同但比较语义不命中的问题。
4. `注册需求方向方法首节点` 可以继续用于创建缺口占位，但不得被任务筹办解释为已经找到满足需求的方法。
