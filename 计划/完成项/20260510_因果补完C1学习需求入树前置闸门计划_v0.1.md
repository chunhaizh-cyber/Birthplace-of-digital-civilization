# 因果补完 C1 学习需求入树前置闸门计划 v0.1

日期：2026-05-10

前置计划：

```text
20260510_因果补完C0d学习目标构造触发验收计划_v0.1.md
```

## 1. 目标

建立学习需求正式入树前的预检闸门，验证学习目标草案是否已经具备转换为 `结构_需求树更新指令` 的最低条件。

## 2. 边界

```text
1. 不正式创建学习需求。
2. 不修改需求树。
3. 不生成探索需求。
4. 不复制父需求目标。
5. 不把 I64 当前值 / 目标值伪装成状态节点。
```

## 3. 闸门规则

学习需求入树前必须满足：

```text
1. 存在学习候选。
2. 学习目标草案完整。
3. 目标主体有效。
4. 目标特征类型有效。
5. 当前状态节点有效。
6. 目标状态节点有效。
7. 满足关系非零。
8. 能明确上级需求。
```

当前 C0c/C0d 只具备：

```text
目标主体
目标特征类型
当前 I64 值
目标 I64 值
满足关系
```

尚缺：

```text
当前状态节点
目标状态节点
上级学习需求挂接策略
```

因此 C1 第一版应只输出预检拒绝原因，不进入需求树。

## 4. 实现方式

新增控制面板命令：

```text
--causal-learning-demand-gate-summary
--learning-demand-gate-summary
```

输出内容：

```text
学习候选类型
学习目标草案完整性
目标主体
目标特征类型
当前值
目标值
满足关系
当前状态节点状态
目标状态节点状态
上级需求状态
预检结论
拒绝原因
```

## 5. 验收

```text
1. Debug x64 构建通过。
2. `--causal-learning-demand-gate-summary` 0 码返回。
3. 默认剧本没有学习候选时，输出“不通过 / 无学习候选或目标不完整”。
4. 即使学习目标草案完整，只要当前状态节点和目标状态节点缺失，也必须输出“不通过 / 缺状态节点构造”。
5. 输出明确“不入树、不修改需求树、不生成探索需求”。
```
