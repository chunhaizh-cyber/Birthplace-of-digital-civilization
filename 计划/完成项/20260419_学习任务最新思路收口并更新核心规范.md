# 20260419 学习任务最新思路收口并更新核心规范

## 本次收口目标

把近期关于学习链的最新思路，正式写回规范：

- 学习任务优先固定为 `唯一可复用的执行壳`
- 真正排队和切换的是 `学习工单`
- 学习工单优先写入 `任务虚拟存在`
- 工单可用 `头节点缓存` 暂存进度，需要时再复制回学习任务头节点
- 方法成长主口径改为 `有没有 / 能不能用 / 好不好用`
- `稳定度` 退出显性主概念，只保留为后台统计证据

## 本次更新的规范

### 1. `1226_学习目的分解与长期学习任务规范.md`

新增并写死：

- 学习任务是复用执行壳
- 学习工单队列才是排队对象
- 学习工单以任务虚拟存在中的 I64 特征与引用特征承载
- 允许头节点缓存学习进度，再复制回学习任务头节点执行
- 学习工单终态改为 `已补出 / 已可用 / 已优选 / 暂停等待 / 废弃`

### 2. `1207_学习任务作为任务管理内部治理子任务规范.md`

新增并写死：

- 学习任务在语义上是内部治理子任务执行壳
- 任务管理任务负责管理学习工单队列与当前激活工单
- 学习工作线程只执行当前激活工单
- 可在方法 A 学到一半时暂停并切到方法 B
- 学习结果主判词改为 `有无 / 可用 / 优选`

### 3. `1229_学习方法生成链与任务管理编排规范.md`

新增并写死：

- 任务管理任务先绑定或创建待学习的方法首节点
- 学习工作线程从方法列表中找可执行学习方法
- 学习工单达到可执行后，直接带入待学习方法首节点开始学习
- 抽象因果优先作为补方法骨架的燃料
- 方法成长主链改为 `方法头 -> 可用方法 -> 优选方法`

同时清理了多处旧的 `稳定度 / 稳定转正 / 稳定方法` 词面。

## 当前统一口径

一句话收口：

`任务管理任务发现方法缺口后，先绑定待学习方法首节点，再把学习目标、学习需求和候选学习程序收口到学习工单；学习工单排在任务虚拟存在中，学习任务只作为当前激活工单的执行壳，学习结果继续回写方法树，并按 有无 / 可用 / 优选 三层推进。`
