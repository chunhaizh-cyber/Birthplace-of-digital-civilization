# 20260419 证据动态归结抽象因果前增加方向一致性验证

## 本次完成

- 在 `因果类.cpp` 为 `状态迁移模板` 新增了“结构签名 + 方向一致性”校验。
- 证据动态归结到抽象因果前，现在会先验证：
  - 证据动态由 `初始状态 -> 结果状态` 生成的状态迁移二次特征，方向是否可明确推导。
  - 该迁移的 `方向区间 / 差值 / 标量区间` 是否自洽。
  - 若抽象因果中已存在同结构迁移模板，则证据动态的迁移方向必须与模板方向一致。
- 若方向不一致：
  - 不再把该证据动态并入当前抽象因果；
  - 不再错误增加 `成立次数`；
  - 不再错误追加 `证据动态样本`。
- 新建抽象因果模板时，也改成先完成方向校验；如果样本迁移方向不可用于归结，则直接撤销新建模板。

## 当前口径

- 证据动态归结到抽象因果，不只看“条件模板 + 动作模板 + 迁移结构签名”。
- 还必须满足：
  - `证据动态` 的结果特征由 `初始值 -> 结果值` 生成的状态迁移二次特征方向，
  - 与 `抽象因果` 中对应特征的状态迁移模板方向一致。
- 只有方向一致，才允许把该证据动态视为同一条抽象因果的证据样本。

## 影响

- 避免了“同一个抽象因果下混入相反方向证据动态”的问题。
- 避免了方向不一致时仍然累计 `成立次数` 和 `证据动态样本` 的脏统计。
- 后续控制面板里看到的抽象因果统计，会更接近真实成立方向。
