# 执行缺失原子需求与筹办补齐闭环修订计划 v0.1

制定日期：2026-05-07

归档日期：2026-05-07

完成记录：

```text
1. 任务筹办已从本能方法迁出为 `自我初始化模块::筹办任务` 普通函数，工作线程执行前调用。
2. `自我_任务筹办` 本能 ID、元信息、运行时注册和执行桥允许入口已删除。
3. 执行桥不再生成 `任务形成方法结构缺口`，无当前方法/缺执行入口改为执行缺失信号。
4. 找不到方法由筹办路径创建带目标结果声明的目标方法头，后续由执行发现动作/入口/条件等实际缺失。
5. 工作线程派生消息改为执行缺失原子需求，不再以“补条件”作为总称。
6. 自我线程派生需求入树要求当前状态、目标状态和目标特征类型齐备；缺失时拒绝落树。
7. Debug x64 构建通过，`--summary` 正常返回。
```

## 1. 背景

当前链路已经具备若干部件：

```text
执行方法函数
    -> 在运行输出中写入失败原因 / 派生需求
    -> 任务管理工作线程构造派生需求消息
    -> 自我线程把派生需求入树
    -> 自我线程为新需求创建任务
```

但实现仍混有旧口径：

```text
无执行方法 -> 方法结构缺口 / 挂起
执行输出派生需求 -> 统一包装成补条件派生需求
缺口语义需求 -> 可能没有当前/目标状态，导致同父子链目标去重不稳定
```

这不符合当前规范：

```text
缺方法不叫缺口。
发现没有能满足需求结果的方法时，筹办阶段直接创建目标方法头。
目标方法头不是空方法头；它至少声明目标特征类型和目标方向。
任务筹办是任务系统函数，不属于本能方法。
执行方法函数发现实际缺什么，就输出对应原子需求。
自我线程创建任务。
下一轮由筹办补齐方法、由执行补齐条件。
```

## 2. 最终链路

目标链路固定为：

```text
需求任务进入筹办
    ↓
按需求结果查找方法
    ↓
找不到方法：直接创建目标方法头
    ↓
任务进入执行
    ↓
执行方法函数
    ↓
发现实际缺失
    ↓
输出对应原子需求
    ↓
任务管理工作线程上报执行缺失原子需求
    ↓
自我线程按同父子链目标去重入树
    ↓
为原子需求创建任务
    ↓
下一轮筹办/执行继续补齐
```

其中：

```text
筹办负责：
    找方法 / 创建目标方法头 / 选择可执行方法 / 创建执行输入。

执行负责：
    执行动作函数 / 发现缺动作、缺条件、缺参数、缺实例、缺当前值等实际缺失 / 输出原子需求。

自我线程负责：
    派生需求入树 / 同父子链去重 / 创建任务 / 唤醒来源任务。
```

## 3. 必须删除的旧逻辑

### 3.0 删除“任务筹办本能方法”口径

位置：

```text
自我初始化模块.cpp
自我_执行桥模块.ixx
本能方法类.cpp
本能方法类.h
```

现状：

```text
任务筹办被注册成 自我_任务筹办 本能方法。
执行桥允许把 自我_任务筹办 当作方法动作分发。
任务筹办返回“已建立方法头待重入”等本能运行结果。
```

目标：

```text
任务筹办是任务系统普通函数，不属于本能方法。
任务筹办不进入方法树查找。
任务筹办不通过动作句柄执行。
任务筹办不产生本能动作运行结果。
```

任务筹办函数负责：

```text
读取当前原子执行需求；
解析需求目标特征类型和目标方向；
按结果能力查找方法；
找不到时创建目标方法头；
绑定候选方法 / 执行方法；
创建或刷新任务步骤和执行输入。
```

目标方法头的最低结构：

```text
节点种类 = 方法首节点
来源 = 需求筹办生成 / 运行时方法沉淀
目标特征类型 = 来自需求目标状态
目标方向 = 来自当前状态与目标状态的三向比较
能力签名.结果包 = 至少包含该目标结果项
动作骨架 = 可以缺失
条件结果对 = 可以缺失
可执行入口 = 可以缺失
```

删除或迁移：

```text
枚举_本能方法ID::自我_任务筹办
自我_任务筹办 的本能方法元信息
任务筹办运行时注册函数
任务筹办要求待重入判断
任务筹办本能模块状态 / 运行码 / 缺口类型账
```

保留但迁移到普通筹办函数：

```text
解析任务来源需求
解析目标特征类型和目标方向
查找或创建目标方法头
绑定任务执行方法
创建任务步骤
```

### 3.1 删除“无方法 -> 方法结构缺口”分支

位置：

```text
自我_执行桥模块.ixx
```

现状：

```text
当前任务没有执行方法
    -> 本能方法结果_任务形成方法结构缺口
    -> 枚举_任务执行记账结果::方法结构缺口
    -> 任务挂起
```

目标：

```text
执行桥不再负责把“无方法”转换成结构缺口。
无方法应由筹办阶段处理：
    查找方法失败 -> 创建目标方法头 -> 待重筹办 / 待执行。
```

验收：

```powershell
rg -n "任务形成方法结构缺口|建议形成方法结构缺口|方法结构缺口" 自我_执行桥模块.ixx
```

业务分支应清零；如果枚举名仍在协议历史中保留，不能再被执行桥作为运行结果生成。

### 3.2 删除“缺方法/方法结构缺口”作为需求身份

位置：

```text
自我线程模块.impl.cpp
任务模块.工作线程消息协议.ixx
任务模块.管理工作线程.ixx
```

目标：

```text
不再生成“缺方法”或“方法结构缺口”需求。
方法结构问题如果来自执行函数实际发现，应拆成原子需求：
    缺动作骨架
    缺可执行入口
    缺条件节点
    缺结果节点
    缺形参规格
    缺实参绑定
```

这些都是原子目标，不是泛化缺口。

## 4. 执行输出原子需求

### 4.1 统一输出结构

执行函数输出缺失时，至少写出：

```text
失败原因
派生需求类型 / 需求特征类型
当前状态
目标状态
被需求主体
来源方法
```

需求主信息仍保持原子目标：

```text
被需求存在
被需求当前状态
需求状态
满足关系
目标特征类型缓存
```

禁止只输出：

```text
缺口类型枚举
失败原因词
无状态的派生需求
```

否则无法稳定去重。

### 4.2 原子需求类型首批固定

先固定这些执行缺失原子需求：

```text
方法动作骨架状态 -> 已形成
方法可执行入口状态 -> 已形成
方法条件节点存在状态 -> 已存在
方法结果节点存在状态 -> 已存在
方法形参规格状态 -> 已形成
方法实参绑定状态 -> 已绑定
目标实例特征存在状态 -> 已存在
目标特征当前值状态 -> 已知
执行差值或目标值状态 -> 已确定
输入参数场景状态 -> 已形成
```

这些特征和值必须能形成状态节点，供需求去重和满足判断使用。

### 4.3 特征类能力补齐

需要增补或确认：

```text
特征类型词
状态值词
状态创建入口
状态比较入口
三向关系比较结果
```

需求类不解释值。所有“已形成、已存在、已绑定、已知、已确定”的比较仍交给特征/状态比较服务。

## 5. 工作线程消息修订

### 5.1 改名

将：

```text
私有_构造补条件派生需求消息
私有_派生需求特征_参数条件缺口
执行输出需要派生条件需求
```

改为：

```text
私有_构造执行缺失原子需求消息
私有_派生需求特征_执行缺失
执行输出需要派生原子需求
```

语义变更：

```text
不是所有执行缺失都叫补条件。
补条件只是其中一种执行缺失。
```

### 5.2 消息内容

执行缺失消息必须优先携带执行函数输出的：

```text
需求特征类型
当前状态
目标状态
被需求主体
来源方法
```

如果输出只有失败原因词，则工作线程可以映射到原子需求类型，但必须补齐状态后再上报。

## 6. 自我线程入树修订

### 6.1 去重入口统一

派生需求入树时，应优先构造 `需求主信息类` 候选，然后调用：

```cpp
需求类::查找直接子需求_按目标重叠(...)
```

或通过 `执行更新指令` 进入 `需求类::添加子需求节点_已加锁` 的目标重叠逻辑。

不得另起一套只按：

```text
需求类型
缺口枚举
来源方法
```

去重的分支作为权威身份。

### 6.2 派生归因只记录来源

以下字段只作为来源与观察信息：

```text
派生来源方法主键
派生需求类型值
派生方法需求位值
派生本能能力缺口类型值
```

不能决定需求身份。

需求身份只由：

```text
同父子链 + 目标主体 + 目标特征类型 + 方向掩码
```

决定。

## 7. 筹办阶段修订

### 7.0 筹办函数定位

新增或明确普通函数：

```cpp
结构_任务筹办结果 筹办任务(
    自我类& 自我对象,
    任务节点* 任务头,
    时间戳 now
) noexcept;
```

它属于任务系统，不属于本能方法系统。

函数职责：

```text
1. 解析任务绑定的原子执行需求。
2. 从执行需求得到目标特征类型和目标方向。
3. 调用方法查找逻辑。
4. 找到方法则绑定到任务。
5. 找不到方法则创建目标方法头。
6. 创建或刷新任务执行步骤。
7. 不执行动作函数。
8. 不生成缺方法需求。
```

返回值只描述筹办结果：

```text
已绑定可执行方法
已创建目标方法头
缺少需求目标状态
缺少目标特征类型
缺少目标方向
缺少执行输入
```

这些不是本能动作运行结果。

### 7.1 查找不到方法时直接创建目标方法头

筹办阶段按需求结果查找方法：

```text
命中方法：
    绑定方法并准备执行。

找不到方法：
    新建目标方法头。
    写入目标结果能力签名。
    标记方法尚缺动作/条件/结果事实。
    不生成“缺方法需求”。
```

### 7.2 下一轮继续补齐

创建目标方法头后，后续执行函数如果发现：

```text
缺动作骨架
缺可执行入口
缺条件节点
缺结果节点
缺参数绑定
```

再输出对应原子需求，由自我线程创建任务进入下一轮。

## 8. 分阶段实施

### 阶段一：清理执行桥旧结构缺口分支

1. 删除执行桥中 `无当前方法 -> 任务形成方法结构缺口`。
2. 删除 `建议形成方法结构缺口` 在执行结果中的业务用途。
3. 缺当前方法时返回“待筹办/不可执行”类结果，交给筹办处理。
4. 保持任务不会因此失败或挂起成旧缺口。

### 阶段一补充：任务筹办从本能方法迁出

1. 删除 `自我_任务筹办` 运行时本能注册。
2. 删除执行桥对 `自我_任务筹办` 的 dispatch。
3. 删除 `任务筹办已建立方法头待重入` 作为执行结果的状态流。
4. 将 `自我初始化模块.cpp` 中的任务筹办模块迁成普通任务筹办函数。
5. 工作线程推进前先调用筹办函数，筹办完成后再执行任务步骤。

### 阶段二：工作线程派生消息泛化

1. `补条件派生需求消息` 改为 `执行缺失原子需求消息`。
2. `执行输出需要派生条件需求` 改为 `执行输出需要派生原子需求`。
3. 根据执行输出派生需求词/失败原因词映射原子需求。
4. 保留 `缺条件` 作为一种具体类型，不作为总称。

### 阶段三：补齐原子状态生成

1. 为首批原子需求补状态创建函数。
2. 执行输出缺失时写当前状态和目标状态。
3. 工作线程上报时若状态缺失，按失败原因补齐状态。
4. 自我线程入树时不再接受无状态的泛化缺口作为长期需求身份。

### 阶段四：自我线程入树去重收口

1. 派生需求消息先归一成 `需求主信息类` 候选。
2. 走同父子链目标重叠。
3. 命中已有子需求则更新归因、统计、权重，不新增节点。
4. 只有目标键无效时才记录诊断，不创建无状态缺口需求。

### 阶段五：筹办找不到方法新建方法头

1. 检查筹办查找方法失败路径。
2. 改为创建候选方法首节点。
3. 候选方法首节点只具备目标结果能力意图，不假装可执行。
4. 后续执行发现缺动作/条件/参数时再派生原子需求。

## 10. 完成判定

满足以下条件后，本计划可移动到完成项：

```text
1. 执行桥不再生成“方法结构缺口”。
2. 任务筹办已从本能方法迁出，成为任务系统普通函数。
3. 找不到方法由筹办直接创建目标方法头，不生成缺方法需求。
4. 目标方法头至少带目标特征类型、目标方向和能力签名结果项。
5. 执行函数发现缺失后输出具体原子需求。

```
