## 1. 当前结论

当前代码已经具备任务筹办普通函数、任务执行普通函数和若干核心提交方法：

```text
已存在：
    自我_同步筹办回执到任务
    自我_同步执行回执到任务
    自我_提交任务状态变化
    自我_回写需求满足
    自我_查找特征类型
    自我_配对参数
    自我_改变特征值
    自我_检查需求是否满足
    自我_尝试学习
    自我_定向学习
    自我_练习
```

缺口是：规范中一部分协作子动作仍由普通函数或线程流程表达，还没有成为可查找、可调度、可记录的本能方法入口。

任务筹办 / 任务执行自身不属于这个缺口；它们按任务管理生命周期普通函数执行。

本计划目标不是一次性把全部逻辑搬成独立方法，而是先把“动作来源必须是方法”的缺口按优先级补齐。

## 3. 实施顺序

### P1. 枚举与默认方法表

修改：

```text
本能方法类.h
本能方法类.cpp
```

新增本能方法 ID 与默认方法种子。

要求：

```text
1. 只新增方法身份，不改变当前主链行为。
2. 新 ID 必须连续放在现有 6067 之后。
3. 默认方法名使用规范名，不使用临时名。
4. 不注册到执行闭环的方法，不能被任务主链误调用。
```

### P2. 专属规格确认骨架

修改：

```text
自我动作实现模块.ixx
```

新增规格确认函数：

```text
确认查找候选方法规格
确认选择当前方法规格
确认检查方法条件是否满足规格
确认挂起任务等待子需求规格
确认唤醒任务重筹办规格
确认派生需求入树规格
确认登记任务发起意图规格
确认承接需求创建任务规格
确认建立条件结果对规格
确认提交方法状态变化规格
确认归并子结果到父任务筹办规格
确认检查父任务是否可重筹办规格
```

第一版规格只声明输入槽、回执槽和结果槽，不做业务写入。

### P3. 注册第一批可安全执行的过程方法

优先注册：

```text
自我_查找候选方法
自我_选择当前方法
自我_检查方法条件是否满足
```

这些方法只读取输入、生成运行存在回执，不直接写任务状态、需求树或世界事实。

暂停条件：

```text
如果某方法需要写任务状态、需求树或方法状态，则不得在本步注册，必须等对应提交方法明确。
```

### P4. 迁移任务筹办内部普通函数的动作来源

现有内部函数：

```text
填充任务方法候选集
从任务候选集中选择当前方法
候选方法不可就绪状况
```

第一阶段保留内部函数，但在日志和计划口径中明确对应本能方法：

```text
查找候选方法 -> 填充任务方法候选集 / 填充候选集_按结果能力
选择当前方法 -> 从任务候选集中选择当前方法
检查方法条件是否满足 -> 候选方法不可就绪状况 / 条件节点匹配
```

第二阶段不再把任务筹办主链整体改成本能方法链。
任务筹办普通函数可以继续调用内部 helper；协作本能方法只在需要作为可调度动作、
可记录动作来源或跨线程提交入口时使用。

### P5. 方法学习提交动作

新增并注册：

```text
自我_建立条件结果对
自我_提交方法状态变化
```

约束：

```text
1. 尝试学习 / 定向学习 / 练习 仍是过程方法。
2. 条件结果对集合写入必须归到 建立条件结果对。
3. 方法状态变化必须归到 提交方法状态变化。
4. 方法状态动作动态来源必须是 提交方法状态变化。
```

### P6. 任务等待与重筹办动作

新增并注册：

```text
自我_挂起任务等待子需求
自我_唤醒任务重筹办
```

约束：

```text
1. 它们不直接替代 提交任务状态变化。
2. 它们形成等待 / 重筹办过程回执。
3. 最终任务状态仍由 自我_提交任务状态变化 提交。
```

### P7. 需求树协作动作

新增并注册：

```text
自我_派生需求入树
自我_登记任务发起意图
自我_承接需求创建任务
```

暂停条件：

```text
如果实现需要改变自我线程和任务管理线程的权责边界，先暂停确认。
需求树写入必须留在需求域方法，不得由任务筹办直接写。
```

### P8. 父任务结果归并

新增：

```text
自我_归并子结果到父任务筹办
自我_检查父任务是否可重筹办
```

它们用于把子任务、学习产物、方法状态变化回流到父任务筹办。

## 4. 验收标准

```text
1. Debug x64 v143 构建通过。
2. 本能方法默认表能查询新增方法名。
3. 已注册方法进入 注册本能函数执行闭环。
4. 未注册方法不会被任务主链误调用。
5. 任务状态变化仍只由 自我_提交任务状态变化 提交。
6. 方法状态变化不得由尝试学习 / 定向学习 / 练习直接裸写成最终事实。
7. 日志能看出：
   查找候选方法 -> 选择当前方法 -> 检查方法条件 -> 同步筹办回执 -> 提交任务状态变化。
```

## 5. 当前执行建议

先执行：

```text
P1 + P2 + P3
```

原因：

```text
它们不会改变当前需求树生长主逻辑，但能把筹办协作动作先方法化，为后续日志和动作动态归因打基础。
```

暂缓：

```text
P5-P8
```

原因：

```text
涉及方法状态提交、需求树入树、任务创建和父任务重筹办，容易碰到上级规范边界，需要逐步接入。
```
