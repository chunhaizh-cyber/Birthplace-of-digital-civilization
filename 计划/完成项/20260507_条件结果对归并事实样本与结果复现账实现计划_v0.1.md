# 20260507 条件结果对归并事实样本与结果复现账实现计划 v0.1

## 1. 背景

最新口径已经收束：

```text
学习动作改变的是方法树，不直接改变需求，也不直接判定任务成功。
方法执行成功后产生条件结果对事实样本。
条件结果对进入方法树归并。
归并时没有等价结构就添加，已有等价结构就融合。
后续这个方法是否可用、能不能满足需求，由自我线程在下一轮筹办和需求满足检查中判断。
```

不再让执行样本携带：

```text
旧隔离学习字段
旧执行前目标验证场景
旧本次目标验证标志
```

这些字段属于旧的隔离学习/影子验证口径，会把学习事实样本和普通需求满足判断混在一起。

## 2. 目标

1. 条件结果对样本只保存来源方法、来源任务、执行前条件场景、执行后实际结果场景和执行结束时间。
2. 条件结果对归并只处理实际条件和实际结果。
3. 同条件下复现已有等价结果时，更新结果符合次数。
4. 同条件下出现新的非等价结果分支时，更新结果不符合次数。
5. 需求是否被满足，由需求/自我线程根据当前状态和目标状态判断，不写进学习样本。
6. 方法选择不再读取旧评分率作为主排序；结果能力命中后，优先使用事实样本数、匹配距离、能力宽度和运行账排序。

## 3. 非目标

1. 不恢复旧隔离学习验证字段。
2. 不把学习动作执行结果等同于需求满足。
3. 不把动作执行失败写入结果符合/不符合账。
4. 不把多个方法组合能力塞进单次查找方法。

## 4. 关键概念

### 4.1 条件结果对样本

第一阶段以场景承载：

```cpp
struct 结构_方法条件结果对样本 {
    方法类::节点类* 来源方法 = nullptr;
    任务节点类* 来源任务 = nullptr;
    场景节点类* 执行前条件场景 = nullptr;
    场景节点类* 执行后实际结果场景 = nullptr;
    时间戳 执行结束时间 = 0;
};
```

后续可把条件场景和结果场景展开为状态集，但语义仍然是事实样本，不是预期验证包。

### 4.2 添加与融合

```text
添加：方法树里没有等价条件节点或等价结果节点，需要长出新节点。
融合：方法树里已有等价条件节点或等价结果节点，只更新统计和证据。
```

四种组合固定为：

| 条件节点 | 结果节点 | 操作 |
| --- | --- | --- |
| 不存在 | 不存在 | 添加条件节点，添加结果节点。 |
| 不存在 | 结果在别的条件下存在 | 添加条件节点，并在新条件下添加结果节点。 |
| 存在 | 不存在 | 融合条件节点，添加结果节点。 |
| 存在 | 存在 | 融合条件节点，融合结果节点。 |

结果节点必须挂在具体条件节点下。同一个结果出现在不同条件下，不能因为结果相同就合账。

### 4.3 结果符合/不符合账

结果符合/不符合账有两个来源：

```text
同条件复现账：
    同一条件下实际结果复现已有等价结果 -> 结果符合次数 +1
    同一条件下实际结果出现新分支     -> 结果不符合次数 +1

需求满足账：
    普通任务完成后，由需求满足检查根据当前状态和目标状态判断。
```

学习任务只写方法树事实，不直接写需求满足账。

## 5. 实施项

1. 删除 `结构_方法条件结果对样本` 中的旧隔离学习验证字段。
2. 删除归并阶段的旧验证分支创建逻辑。
3. 保留并修正 `结果符合次数 / 结果不符合次数`：
   - 它们只用于同条件结果复现/分裂，或需求满足检查。
   - 不再表示旧执行前验证目标是否被覆盖。
4. 方法虚拟存在记录事实样本数和最近样本时间。
5. 方法选择排序改为：
   - 结果能力覆盖需求。
   - 当前条件满足。
   - 参数可配。
   - 非根写入原语。
   - 方法事实样本数更多。
   - 匹配距离更小。
   - 能力包宽度更窄。
   - 平均运行时长更短。
6. 规范和说明书清理所有旧隔离学习验证和旧评分率排序口径。

## 6. 验收

1. 代码中不存在旧隔离学习验证字段的方法归并路径。
2. 方法执行成功后能归并实际条件和实际结果。
3. 同条件同结果复现时增加结果符合次数。
4. 同条件新结果分支出现时增加结果不符合次数。
5. 方法选择不再调用旧评分率或旧有效样本读取函数。
6. 工程能构建并运行 `--summary`。
