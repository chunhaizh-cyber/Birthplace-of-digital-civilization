# 20260418 主派发最终去向改写前推为治理桥主裁决

## 本步目标

将 `自我线程` 在主派发阶段形成的本地去向，改成“线程候选去向”；真正写入 `当前最终去向 / 任务管理当前去向` 的最终裁决，前推为 `自我_主循环治理模块` 的治理桥结果。

## 本步完成内容

1. 在线程主派发阶段引入 `线程候选最终去向 / 线程候选当前去向`，将线程内部的本地路由推导先收口为候选值。
2. 将治理桥输入中的 `主循环缺口 / 主循环去向`，改为基于线程候选去向推导，而不是直接基于线程已写回的最终去向。
3. 在治理桥结果回写阶段改成“双层裁决”：
   - 治理桥若给出明确最终去向，则优先采用治理桥建议。
   - 治理桥若未给出明确最终去向，则回落到线程候选去向。
4. 同步收口 `任务管理当前去向` 的文本映射逻辑，让它和线程最终去向保持统一来源。

## 主要改动文件

- `D:\鱼巢\自我线程模块.impl.cpp`

## 当前效果

- 线程本地路由不再直接覆盖最终去向，而是先形成候选。
- 治理桥开始承担“最终去向主裁决层”的职责。
- 后续如果继续前推，可以再把主派发更早阶段的局部去向推导继续并入治理桥输入，而不是在线程内部先定再改。
