# 控制面板任务树改为显式树形并展开任务节点字段

## 本次完成

控制面板里的“任务树”页面，原来只是把每个任务节点压成一行：

`任务标题 | 状态 | 节点种类 | 调度优先级 ...`

这次改成了真正的树形显式：

- 每个任务节点仍按真实父子关系挂在树上
- 每个任务节点下面会继续展开自己的字段子树
- 字段子树和真实下级节点同时可见

对应修改在：

- `D:\鱼巢\控制面板类.cpp`

## 现在的显示结构

每个任务节点现在会展开成：

1. `树下节点(n)`
2. `基本信息`
3. `运行信息`
4. `关系锚点`
5. `头结点字段 / 步骤节点字段 / 结果节点字段`

也就是说，控制面板不再只显示“任务头标题”，而是会把：

- 主键
- 名称
- 类型
- 节点种类
- 状态
- 任务树类型
- 创建/启动/完成时间
- 优先级
- 动作句柄
- 场景
- 任务虚拟存在
- 来源需求
- 父任务
- 后续任务
- 执行方法
- 目标状态
- 结果状态信息

这些字段显式挂到树里。

同时按节点种类，还会继续展开专属字段：

- 头结点：
  - 当前步骤节点
  - 当前子任务头结点
  - 最近实际结果节点
  - 等待学习方法首节点
  - 直属子任务头结点列表
  - 目标维度签名
- 步骤节点：
  - 所属任务头结点
  - 可用方法首节点
  - 方法条件节点
  - 并发子任务头结点列表
  - 步骤序号
  - 重试/超时/允许切换
- 结果节点：
  - 结果角色
  - 来源步骤节点
  - 来源方法首节点
  - 方法结果节点
  - 对应方法结果节点
  - 结果方向二次特征
  - 命中预测结果节点
  - 命中后动作
  - 置信度
  - 错误码
  - 派生子任务头结点列表

## 这次的关键语义

这次不是“把任务列表做长”，而是把任务树页面改成了：

`真实任务树结构 + 每个节点自己的字段树`

所以现在在控制面板里，能同时看见：

- 节点在任务树中的位置
- 节点和谁相连
- 节点当前挂了什么方法、场景、状态、结果

更适合排查任务管理、学习子任务、步骤推进、结果回接这些链路问题。
