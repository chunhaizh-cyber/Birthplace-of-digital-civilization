# 任务筹办按方法能力查找到就绪流程修正计划 v0.1

日期：2026-05-13

状态：已执行并按最新口径修订。

最新修订：

```text
任务筹办 / 任务执行是任务管理生命周期普通函数，不再经由 自我_任务筹办 / 自我_任务执行 本能方法。
本计划仍有效的是任务筹办内部流程：
    候选集填充优先；
    按方法结果能力召回候选；
    从候选集中选择当前方法；
    方法已有动作入口但不可用时优先派生 方法状态 = 可用；
    目标宿主 / 目标特征 / 目标状态缺失必须记录日志。
```

依据规范：

```text
规范/任务筹办父方法与协作子方法链规范20260513.md
规范/任务筹办按方法能力查找到就绪因果链规范20260513.md
```

## 1. 当前代码状态

当前 `任务模块.筹办.ixx` 已具备以下正确方向：

```text
1. 任务筹办先填充任务方法候选集，再从候选集中选择当前方法。
2. 改变特征值 已声明参数化结果能力，可按任意目标特征类型召回。
3. 候选方法已有动作入口但不可用时，优先派生 方法状态 = 可用。
4. 只有明确缺动作骨架 / 缺执行入口时，才派生动作骨架或入口类缺口。
5. 学习补齐本能方法不要求自身先有条件结果对，避免自阻塞。
```

仍需修正和验证的是：

```text
1. 候选集填充要成为筹办第一动作事实，不能被占位方法抢先。
2. 候选集数量、选择原因、目标特征、目标状态必须有日志。
3. 无候选 / 候选不可用 / 缺入口 / 缺参数 / 条件不满足 必须落到不同缺口。
4. 需求目标必须同时具备目标宿主、目标特征类型、目标状态或明确满足关系。
5. 缺目标状态时必须有日志，不能静默兜底。
```

## 2. 目标流程

标准链路：

```text
来源需求 D
    -> 解析目标宿主 X、目标特征 F、目标状态 V、满足关系 R
    -> 查找结果能力命中 F 的候选方法集
    -> 写入任务方法候选集
    -> 从候选集中选择当前方法 M
    -> 检查 M 的方法状态、动作入口、条件结果对、参数和条件
    -> 形成可就绪回执
    -> 同步筹办回执到任务
    -> 提交任务状态变化
    -> 任务状态 = 就绪
```

缺口链路：

```text
无候选方法
    -> 派生：存在可改变 F 的方法

候选方法不可用
    -> 派生：M.方法状态 = 可用

候选方法缺动作 / 缺入口
    -> 派生：M.动作骨架 / 可执行入口 已形成

候选方法缺条件结果对
    -> 派生：M.方法状态 = 可用

参数不可配
    -> 派生：缺失参数已获得 / 已绑定

条件节点不满足
    -> 派生：方法前置条件成立
```

## 3. 实施步骤

### P1. 目标解析日志补强

位置：

```text
任务模块.筹办.ixx::筹办任务
```

新增日志字段：

```text
任务
来源需求
目标宿主
目标特征
目标状态
满足关系
当前状态
```

要求：

```text
缺目标特征类型：现有日志保留。
缺目标状态：新增日志。
缺目标宿主且不能从状态主体补齐：新增日志，并进入不可筹办缺口。
```

暂停条件：

```text
如果当前状态节点无法稳定解析目标宿主，先暂停，不允许凭空用任务或自我当目标宿主。
```

### P2. 候选集填充第一步固化

位置：

```text
解析或创建任务目标方法
填充任务方法候选集
填充候选集_按结果能力
```

修正要求：

```text
1. 每次筹办必须先填充候选集。
2. 候选集填充完成前不得创建占位方法头。
3. 候选集非空时不得新建占位方法头。
4. 单候选直接选择当前方法。
5. 多候选走候选评分。
```

验收日志：

```text
候选集填充
候选集填充后选择当前方法
候选集填充后仍无当前方法
按需求目标创建占位方法头并加入候选集
```

### P3. 方法能力命中规则复核

位置：

```text
方法结果项命中目标特征
方法结果能力命中目标特征
确保改变特征值方法可按任意特征召回
```

规则：

```text
1. 固定结果特征类型必须与需求目标特征类型一致。
2. 参数化结果能力可命中任意目标特征，但必须要求输入包能绑定目标特征类型。
3. 根写入原语不得参与普通方法召回。
4. 改变特征值必须能作为“任意特征类型变化”的候选方法。
```

验收：

```text
安全值 / 服务值 / 方法状态 等需求都能召回 改变特征值。
```

### P4. 当前方法可就绪判断收口

位置：

```text
候选方法不可就绪状况
返回方法结构缺口
返回方法状态可用性缺口
```

规则：

```text
1. 没有动作骨架且没有本能入口：方法缺动作骨架。
2. 有动作骨架但没有可执行入口：方法缺可执行入口。
3. 有本能入口或可执行入口，但没有条件结果对：方法状态 = 可用。
4. 学习补齐本能方法只要入口存在，不因自身缺条件结果对被阻塞。
```

重要约束：

```text
如果一个任务已绑定的方法有动作入口，不得派生 方法动作骨架状态。
此时应派生 方法状态 = 可用 或更细条件结果对 / 参数 / 条件需求。
```

### P5. 目标状态缺失日志与派生输入检查

位置：

```text
筹办任务
返回方法状态可用性缺口
返回方法结构缺口
```

要求：

```text
1. 缺口回执必须携带缺口特征类型。
2. 缺口回执必须携带缺口目标主体。
3. 若能构造目标状态，必须记录目标状态主键。
4. 若不能构造目标状态，必须写日志说明缺哪个字段。
```

### P6. 派生需求与父任务等待关系复核

目标：

```text
筹办缺口被上报后，自我线程生成子需求；
父任务等待子需求；
子需求满足后父任务重筹办。
```

检查点：

```text
1. 筹办回执.需要创建或复用子需求 = true。
2. 筹办回执.需要自我线程 = true。
3. 派生需求消息包含父任务、来源需求、目标宿主、目标特征、目标状态。
4. 子需求满足后能触发父任务重筹办请求。
```

### P7. 禁止旧执行桥回退

要求：

```text
1. 任务筹办不得输出会让工作线程回到旧执行桥状态写入的结果。
2. 任务执行经任务管理生命周期普通函数执行，不经 自我_任务执行。
3. 当前方法执行失败不能直接推出任务完成。
4. 需要派生需求时不能同轮输出最终结果返回。
```
