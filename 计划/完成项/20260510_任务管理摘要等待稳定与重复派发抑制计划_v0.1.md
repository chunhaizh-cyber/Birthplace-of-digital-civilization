# 任务管理摘要等待稳定与重复派发抑制计划 v0.1

日期：2026-05-10

承接：

```text
计划/20260510_自我内部循环P3P4任务化与任务语义验收计划_v0.1.md
```

## 1. 问题

执行 P3-P4 动态验收时发现：

```text
.\x64\Debug\鱼巢.exe --task-mgmt-summary
```

在 120 秒内未返回。已停止残留进程。

日志显示同一任务持续重复派发：

```text
自我线程/任务派发目标 | 任务=8 | 对应需求=5 | 需求类型=方法条件结果配对状态 ...
```

这说明任务管理摘要等待稳定条件无法满足，至少存在以下一种情况：

```text
1. 自我线程每轮重复提交同一任务治理请求。
2. 任务管理界面线程幂等复用结果没有让自我线程进入稳定态。
3. 当前任务处于等待 / 缺口状态，但主派发仍持续提交。
4. 等待稳定函数只看“空闲拍”，没有识别“同一任务同一等待态已稳定”。
```

## 2. 最新边界

必须保持：

```text
自我线程只提交任务治理请求给任务管理界面线程。
任务管理界面线程负责绑定任务虚拟存在、初始化任务语义、生成工作包和归并上行。
任务管理工作线程只处理任务工作包。
```

禁止回退：

```text
不恢复自我线程直连工作线程。
不恢复自我线程创建任务虚拟存在。
不让任务虚拟存在替代任务信息节点。
```

## 3. 排查计划

### 步骤 1：定位重复提交条件

检查：

```text
自我线程主派发决议
任务管理界面线程幂等键和承接缓存
任务状态 / 下一步去向
当前主任务等待态
当前主需求是否仍适合任务化
```

输出：

```text
同一任务重复提交的直接条件。
```

### 步骤 2：补任务治理请求占用 / 等待判定

候选策略：

```text
如果当前主任务已进入等待中，且等待原因来自派生需求 / 缺口承接，则自我线程本轮不重复提交工作包。
如果任务管理界面线程已记录同幂等键的等待承接结果，则返回复用承接结果，并明确下一步为保持等待。
```

### 步骤 3：修正等待稳定函数

候选策略：

```text
把“同一任务同一等待态连续出现 N 拍”视为稳定，而不是必须工作线程完全无推进。
保留故障、排队、执行中、持续新派生需求为不稳定条件。
```

### 步骤 4：验证

验证命令：

```text
msbuild "鱼巢.vcxproj" /p:Configuration=Debug /p:Platform=x64 /p:LinkIncremental=false /m
.\x64\Debug\鱼巢.exe --task-mgmt-summary
.\x64\Debug\鱼巢.exe --summary
```

验收：

```text
1. `--task-mgmt-summary` 能在有限时间内返回。
2. 不再持续刷同一任务同一需求的派发日志。
3. 控制面板摘要仍能显示内部循环 P3-P4 断点。
4. 自我线程不恢复旧直连 / 旧创建任务虚拟存在能力。
```
