# 本能函数按抽象方法概念初始化方法虚拟存在修订计划 v0.1

日期：2026-05-11

## 1. 背景

当前抽象存在树中已经存在：

```text
存在
└─ 虚拟存在
   └─ 方法
```

并且 `方法` 概念已经补齐通用方法抽象特征槽。后续每个本能函数在初始化自己的方法虚拟存在时，应优先复用抽象存在树中的 `方法` 概念模板，而不是在各个本能函数里重复拼接通用方法特征。

本计划只修订本能函数自身方法环境的建立方式，不判定自我内部循环已经闭合。

## 2. 总原则

```text
本能函数执行前，先确保自身方法环境。

自身方法环境中的通用方法部分：
    由抽象存在树中的“方法”概念扩充。

自身方法环境中的专属执行规格：
    由各自的 确认XXX规格 函数建立。
```

角色边界：

```text
方法首节点：
    大范围索引、候选召回、动作名 / 动作句柄入口。

方法虚拟存在：
    方法语义记忆、统计、学习产物索引。

方法概念：
    方法虚拟存在的通用模板来源。

条件节点：
    本能函数执行时的输入参数槽、参数匹配和前置条件依据。

结果节点：
    本能函数输出槽、结果状态和校验依据。
```

禁止项：

```text
1. 不新增抽象存在类。
2. 不新增存在形态枚举。
3. 不新增语义结构体承载方法状态。
4. 不在本能函数里重复手写通用方法特征槽。
5. 不用方法头节点替代条件节点做参数匹配。
```

## 3. 现有基础

已具备：

```text
世界树：
    确保核心概念_方法
    添加实例概念
    根据概念扩充实例记忆

方法虚拟存在服务：
    确保方法虚拟存在
    补齐方法虚拟存在概念集
    同步方法节点到虚拟存在
    初始化方法虚拟存在基础信息
    同步方法首节点结构特征
    初始化方法统计特征

自我动作实现模块：
    确保本能方法自身环境
    确认本能方法专属规格
    各本能函数四段式入口雏形
```

## 4. 修订目标

### 4.1 统一方法虚拟存在初始化入口

目标：

```text
所有本能函数都通过同一个自身环境入口完成方法虚拟存在初始化。
```

实现：

```cpp
方法虚拟存在服务类::同步方法节点到虚拟存在(
    方法首节点,
    宿主存在,
    now);
```

该入口必须保证：

```text
1. 方法虚拟存在存在。
2. 方法虚拟存在概念集包含：存在、虚拟存在、方法、具体方法概念。
3. 方法虚拟存在按“方法”概念扩充通用特征槽。
4. 方法首节点结构摘要同步到方法虚拟存在。
5. 方法统计特征初始化。
```

### 4.2 专属规格只负责本能函数自己的条件 / 结果接口

`确认XXX规格` 只负责：

```text
1. 建立本能函数自己的条件节点。
2. 建立本能函数自己的结果节点。
3. 建立条件节点输入槽。
4. 建立结果节点输出槽。
5. 建立必要的条件结果对或规格摘要。
```

不再负责：

```text
方法来源
方法作用域
方法状态
方法是否可用
方法运行账完整性
方法统计特征
允许自动召回
允许默认执行
```

这些属于通用方法环境。

## 5. 分步计划

### P0：复核抽象方法概念模板

检查项：

```text
1. 存在概念树中有 虚拟存在 / 方法。
2. 方法概念已经拥有通用方法抽象特征槽。
3. 常用抽象特征初始化验收覆盖方法状态、方法来源、方法作用域、动作句柄类型和能力签名摘要。
```

验收：

```text
Debug x64 构建通过。
常用抽象特征已初始化 返回 true。
```

### P1：强化 `方法虚拟存在服务类`

修改点：

```text
1. 确保方法虚拟存在时，必须补齐方法概念集。
2. 补齐概念集时使用 添加实例概念(..., true)，触发概念记忆扩充。
3. 同步方法节点到虚拟存在时，先完成概念扩充，再写结构和统计镜像。
```

验收：

```text
任意本能方法虚拟存在创建后，概念集包含 方法。
方法虚拟存在具备方法概念通用特征槽。
```

### P2：改造 `确保本能方法自身环境`

修改点：

```text
1. 确保本能方法首节点。
2. 调用 方法虚拟存在服务类::同步方法节点到虚拟存在。
3. 读取同步后的方法虚拟存在。
4. 再调用 确认本能方法专属规格。
5. 若同步失败，记录日志并报错，不静默兜底。
```

验收：

```text
每个本能函数进入核心逻辑前，方法虚拟存在已按方法概念扩充。
```

### P3：清理本能函数内重复通用初始化

删除旧代码范围：

```text
1. 本能函数或规格确认函数中重复写入通用方法特征的逻辑。
2. 重复初始化 方法状态、方法是否可用、方法来源、方法作用域、成熟度、允许自动召回、允许默认执行 的逻辑。
3. 重复初始化方法统计账的逻辑。
4. 绕过 方法虚拟存在服务类 直接拼方法概念集的逻辑。
5. 本能函数里把输入包直接当作参数匹配依据的旧路径。
```

保留范围：

```text
1. 本能函数专属条件节点输入槽。
2. 本能函数专属结果节点输出槽。
3. 条件结果对生成。
4. 专属运行事实写入。
5. 核心动作逻辑。
```

验收：

```text
通用方法特征只由方法概念扩充和方法虚拟存在服务统一保证。
本能函数不再重复造通用方法特征槽。
```

### P4：逐个确认九件套本能函数

范围：

```text
自我_查找特征类型
自我_配对参数
自我_改变特征值
自我_检查需求是否满足
自我_尝试学习
自我_定向学习
自我_练习
自我_安全服务闭环安全推进
自我_安全服务闭环服务推进
```

每个函数验收：

```text
1. 进入核心逻辑前已确保自身方法环境。
2. 参数匹配依据来自自身条件节点。
3. 输出依据来自自身结果节点。
4. 通用方法特征不在函数内重复初始化。
```

### P5：增加代码注释

注释位置：

```text
1. 方法虚拟存在服务类::同步方法节点到虚拟存在。
2. 自我动作实现模块::确保本能方法自身环境。
3. 各 确认XXX规格 函数入口。
```

注释口径：

```text
通用方法环境来自抽象存在树中的“方法”概念。
专属规格只建立本能方法自己的条件节点和结果节点接口。
方法头节点只做候选召回，不做参数匹配。
```


## 6. 删除旧代码清单

执行时重点删除：

```text
1. 本能函数中重复写入通用方法状态的旧逻辑。
2. 本能函数中重复写入方法可用、结构完整、稳定、运行账完整性的旧逻辑。
3. 本能函数中重复写入方法来源、方法作用域、动作句柄类型、成熟度的旧逻辑。
4. 本能函数中重复创建方法统计账默认槽的旧逻辑。
5. 绕过方法虚拟存在服务直接创建 / 拼接方法虚拟存在概念集的旧逻辑。
6. 用方法头节点或裸输入包直接替代条件节点做参数绑定的旧逻辑。
7. 旧注释中暗示“方法头就是执行接口”的表述。
```

删除边界：

```text
不能删除条件节点 / 结果节点规格确认。
不能删除核心本能函数逻辑。
不能删除方法虚拟存在服务。
不能删除方法首节点。
```

## 7. 与当前主线关系

本计划属于：

```text
内部动作闭环前置修订
```

它不替代：

```text
需求树真实生长
执行需求任务化
任务反馈结算
自我苏醒验收
初步成熟验收
```

当前真实第一阻塞点仍然是：

```text
需求树不会根据自我状态差距稳定自动生长。
```

本计划可以并行推进，但完成后只能说明本能函数自身方法环境更干净，不能说明自我内部循环已经闭合。
