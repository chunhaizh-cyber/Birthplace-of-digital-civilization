# 语素入口节点命名与人类词性入口边界计划 v0.1

日期：2026-05-10

## 1. 背景

最新规范已经将系统内部入口收口为：

```text
人类词性 = 语言数字化工具。
信息入口类型 = 系统语义入口和落树路由。
```

因此旧名 `词性节点类` 容易误导。该节点实际已经不是单纯的人类语法词性节点，而是：

```text
词面 + 人类词性 + 信息入口类型 + 对应基础信息引用
```

更准确的权威名应为：

```text
语素入口节点类
```

## 2. 命名规则

```text
新权威名：语素入口节点类
旧兼容名：词性节点类
```

第一阶段保留兼容别名：

```cpp
using 语素入口节点类 = 语素节点类;
using 词性节点类 = 语素入口节点类;
```

业务代码应使用 `语素入口节点类`；旧名只允许留在兼容别名、历史说明和少量迁移注释中。

## 3. `添加词性词` 边界

`添加词性词` 不删除，但语义固定为：

```text
添加一个“人类词性入口”。
用于自然语言分词、词性标注、候选信息入口推断。
不用于业务语义节点落树。
```

业务语义入口必须使用：

```cpp
添加信息入口词(...)
```

例如：

```text
内部特征槽：添加信息入口词(..., 特征模板入口)
动作模板：添加信息入口词(..., 动态模板入口)
任务虚拟存在：添加信息入口词(..., 任务虚拟存在入口)
方法虚拟存在：添加信息入口词(..., 方法虚拟存在入口)
治理主体实例：添加信息入口词(..., 存在实例入口)
```

## 4. 实施步骤

```text
P1. 在语素节点类型中新增 `语素入口节点类`，保留 `词性节点类` 兼容别名。
P2. 将业务源码中的 `词性节点类` 类型使用迁移为 `语素入口节点类`。
P3. 在 `语素类` 接口处补充 `添加词性词` 的边界注释。
P4. 检查业务源码中是否仍有 `添加词性词(..., "名词/动词/专有名词")` 直接语义落树。
P5. Debug x64 构建验证。
```

## 6. 最终口径

```text
语素入口节点是系统入口节点；
人类词性只是它的语言标注之一；
信息入口类型才决定基础信息类型和存储树路由。
```
