# 三个学习本能函数按方法状态链修订计划 v0.1

日期：2026-05-11

> 2026-05-11 口径废止说明：本计划中的“练习承接 方法状态=可用”已被废止。当前权威口径为：`方法状态=可用` 等于目标方法已有至少一组条件结果对，由 `自我_尝试学习` 承接；`自我_练习` 只处理“好用 / 稳定 / 细化”，承接 `方法运行账完整性`、`方法稳定状态` 等目标。本文件仅保留历史记录，不作为当前实现依据。

## 1. 背景

当前安全根任务已经能查到多个和“安全值改变”相关的方法候选，但候选方法大多只有方法头，`方法状态` 尚未到 `可用`。

最新口径要求：

```text
方法头节点 = 大范围索引和候选召回
条件节点 = 执行时参数匹配和前置条件依据
结果节点 = 执行后结果结构和校验依据
条件结果对 = 条件节点和结果节点之间的真实知识单元
方法状态 = 方法完善顺序的唯一状态入口，最后一个值是“可用”
```

因此三个学习本能函数不能都声明或写出“方法状态=可用”。它们应形成顺序链：

```text
练习
  目标：让目标方法最终达到 方法状态=可用
  发现缺真实条件结果对时，产生需要尝试学习满足的缺口

尝试学习
  目标：为目标方法形成真实条件结果对
  发现缺候选动作 / 动作导向时，产生需要定向学习满足的缺口

定向学习
  目标：为目标方法形成候选动作 / 方法动作骨架
```

## 2. 总目标

修订 `自我动作实现模块.ixx` 中三个学习本能函数：

```text
自我_练习
自我_尝试学习
自我_定向学习
```

让它们按方法状态链工作：

```text
定向学习 -> 候选动作 / 动作骨架
尝试学习 -> 真实条件结果对
练习 -> 方法状态=可用
```

并同步修正方法能力声明，让筹办查找时能按目标选择正确学习方法。

## 3. 边界

本计划只处理三个学习本能函数及其直接能力声明 / 状态写回。

暂不处理：

```text
1. 外界探索需求。
2. 复杂因果模板升格。
3. 长期学习策略。
4. 休眠期修复动作。
5. 控制面板展示优化。
```

但若任务筹办无法按 `方法状态 / 方法条件结果配对状态 / 方法动作骨架状态` 选择对应学习方法，则必须在本计划内修正最小选择分流，否则三个函数即使改好也不会被用上。

## 4. 分步计划

### P0. 复核并阻断旧错误路径

目标：

```text
确认没有在方法头创建阶段强行制造条件结果对；
确认没有把“条件节点 + 结果节点存在”直接等同于“真实条件结果对”；
确认没有继续使用 方法是否可用 作为可用性权威。
```

检查点：

```text
1. 方法首节点创建只建立索引、虚拟存在、条件 / 结果场景和规格槽。
2. 真实条件结果对只能由学习 / 执行中带证据的流程生成。
3. 可用性只读写 方法状态。
```

### P1. 修正三个学习方法的结果能力声明

目标：

```text
筹办按目标查找学习方法时，找到的是对应阶段的方法，而不是三个方法都声称能让方法可用。
```

修订：

```text
自我_练习：
    结果能力保留 方法状态 -> 可用

自我_尝试学习：
    删除 方法状态 -> 可用
    声明 方法条件结果配对状态 -> 已配对

自我_定向学习：
    删除 方法状态 -> 可用
    声明 方法动作骨架状态 / 候选动作状态 -> 已形成
```

约束：

```text
不要新增过多特征类型。
优先复用已有 方法状态、方法条件结果配对状态、方法动作骨架状态。
```

### P2. 修正 `自我_定向学习`

目标：

```text
定向学习只产出候选动作 / 动作骨架，不越级产出条件结果对，也不标记目标方法可用。
```

实现：

```text
1. 先确保自身方法环境。
2. 通过自身条件节点绑定输入。
3. 读取目标方法头、目标特征、来源观察 / 因果证据。
4. 为目标方法写入候选动作节点 / 方法动作骨架状态。
5. 目标方法状态推进到“待条件结果对”或等价阶段。
6. 不创建假的真实条件结果对。
7. 不写 方法状态=可用。
```

验收：

```text
定向学习成功后：
    目标方法有候选动作 / 动作骨架；
    目标方法仍不可用；
    下一阶段需求应指向 条件结果对补齐。
```

### P3. 修正 `自我_尝试学习`

目标：

```text
尝试学习只负责形成目标方法的真实条件结果对。
```

实现：

```text
1. 先确保自身方法环境。
2. 通过自身条件节点绑定输入。
3. 必须有来源因果、来源观察或可接受的运行证据。
4. 用证据为目标方法归并条件节点和结果节点。
5. 写入 条件结果对。
6. 写入 方法条件结果配对状态=已配对。
7. 目标方法状态推进到“待练习 / 待验证 / 待可用确认”阶段。
8. 不写 方法状态=可用。
```

验收：

```text
尝试学习成功后：
    目标方法至少有一组真实条件结果对；
    目标方法仍不因尝试学习直接可用；
    下一阶段由练习承接可用性验证。
```

### P4. 修正 `自我_练习`

目标：

```text
练习承接“方法状态=可用”的需求，是学习链最后一步。
```

实现：

```text
1. 先确保自身方法环境。
2. 通过自身条件节点绑定输入。
3. 检查目标方法是否已有候选动作 / 动作骨架。
4. 检查目标方法是否已有真实条件结果对。
5. 缺候选动作时，输出需要定向学习承接的缺口。
6. 缺条件结果对时，输出需要尝试学习承接的缺口。
7. 条件齐备后构造候选实参 / 可执行输入参数场景。
8. 试运行成功后，写入 目标方法.方法状态=可用。
9. 不再写 方法是否可用。
```

验收：

```text
练习成功后：
    目标方法.方法状态=可用；
    目标方法至少有一组真实条件结果对；
    目标方法具备后续筹办可执行的输入场景或参数绑定记录。
```

### P5. 修正任务筹办对学习方法的最小选择分流

目标：

```text
目标需求进入任务筹办时，能按目标特征选择正确学习方法。
```

规则：

```text
目标 = 方法状态=可用
    -> 候选学习方法优先 自我_练习

目标 = 方法条件结果配对状态=已配对
    -> 候选学习方法优先 自我_尝试学习

目标 = 方法动作骨架状态=已形成
    -> 候选学习方法优先 自我_定向学习
```

约束：

```text
学习方法仍然通过方法头召回，但执行时必须经条件节点绑定输入。
```

### P6. 删除 / 禁用旧代码

删除或禁用：

```text
1. 尝试学习 / 定向学习中直接写 方法状态=可用 的逻辑。
2. 尝试学习 / 定向学习的结果能力声明中“方法状态 -> 可用”。
3. 以 方法是否可用 作为权威可用性的读写路径。
4. 方法构建阶段强行生成真实条件结果对的旧逻辑。
```

保留：

```text
方法头大范围索引；
方法虚拟存在语义记忆；
条件节点输入规格；
结果节点输出规格；
条件结果对真实知识单元。
```

### P7. 验证

最小验证路径：

```text
1. 安全根任务筹办能查到安全值相关方法候选。
2. 候选方法不可用时，生成的缺口目标是 方法状态=可用。
3. 方法状态=可用 任务筹办应选择 自我_练习。
4. 练习发现缺条件结果对时，应转出 条件结果对补齐 缺口。
5. 条件结果对补齐任务筹办应选择 自我_尝试学习。
6. 尝试学习发现缺候选动作时，应转出 动作骨架 / 候选动作 缺口。
7. 动作骨架缺口筹办应选择 自我_定向学习。
8. 三者按 定向学习 -> 尝试学习 -> 练习 回填后，目标方法状态才能到 可用。
```

构建验证：

```text
Debug x64 构建至少通过编译。
若链接失败但错误为 exe 被占用，记录为运行环境阻塞，不判定代码失败。
```

## 5. 完成判定

本计划完成必须同时满足：

```text
1. 三个学习本能函数不再都声称能直接产生 方法状态=可用。
2. 练习是唯一承接 方法状态=可用 的学习方法。
3. 尝试学习承接 条件结果对 / 方法条件结果配对状态。
4. 定向学习承接 候选动作 / 方法动作骨架状态。
5. 真实条件结果对不在方法头初始化阶段凭空生成。
6. 方法是否可用 不再作为三个学习函数的写入目标。
7. 任务筹办能按学习目标选择对应学习方法。
```

## 6. 当前状态

```text
已完成。
```

执行记录：

```text
1. 自我_定向学习 不再越级构造真实条件结果对，不写 方法状态=可用；只推进目标方法动作骨架状态，并把目标方法推进到待条件结果对阶段。
2. 自我_尝试学习 只在有证据且目标方法已有动作骨架时构造真实条件结果对；不写 方法状态=可用；成功后推进到待运行账 / 待练习阶段。
3. 自我_练习 成为唯一写入 方法状态=可用 的学习本能；它要求目标方法已有真实条件结果对并试运行成功。
4. 任务筹办对方法结构补齐需求改为按来源需求目标结果选择学习本能：
   方法状态=可用 -> 练习；
   方法条件结果配对状态 -> 尝试学习；
   方法动作骨架状态 / 方法动作骨架数量 / 方法可执行入口数量 -> 定向学习。
5. 任务筹办普通目标方法头创建不再使用筹办侧“需求方向”路径，改为只按来源需求目标结果特征召回 / 创建方法头；目标值、满足关系和参数匹配继续由需求、条件节点和结果节点承载。
6. Debug x64 非增量链接构建通过。
```

后续阻塞仍在：

```text
需求树真实生长、根任务缺口到子需求的稳定入树、学习链运行期验证、任务执行回写和需求结算仍未完成。
本计划完成只表示三个学习本能函数和筹办分流口径已修正，不表示内部循环闭合。
```
