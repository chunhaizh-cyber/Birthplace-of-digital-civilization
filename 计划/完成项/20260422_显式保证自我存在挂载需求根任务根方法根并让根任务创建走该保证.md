# 20260422_显式保证自我存在挂载需求根任务根方法根并让根任务创建走该保证

本次收口了“安全/服务根需求与任务根都挂在自我存在内、每个存在显式具备任务树”这条规则。

## 改动
- 给 `存在类` 增加了公开入口 `确保存在三类根节点(...)`，把原来私有的“需求根/任务根/方法根补齐”能力显式化。
- 在 `自我类::确保自我根需求与根任务已初始化_()` 中，先显式确保自我存在具备三类根节点，再初始化安全根需求、服务根需求和任务管理任务。
- 在 `自我线程` 按需求创建根级任务头之前，也先显式确保 `自我存在` 已挂载任务根，避免后续再依赖隐式补齐。

## 结果
- 安全根需求、服务根需求继续挂在 `自我存在 -> 需求根节点` 下。
- 根任务继续挂在 `自我存在 -> 任务根节点` 下。
- 三类根节点不再只靠读取主信息时的隐式副作用补齐，而是有显式 API 和明确调用点。
