## 1. 目标

补一个专门测试夹具，验证完整学习目标具备时，`自我类::创建学习I64原子状态` 能构造当前状态节点和目标状态节点，并能让 C1 入树前置闸门通过。

## 2. 边界

```text
1. 只做测试夹具。
2. 不伪造自我线程治理主链。
3. 不创建学习需求。
4. 不执行需求树更新指令。
5. 不修改需求树。
6. 不生成探索需求。
7. 夹具失败必须记录错误日志并返回非 0。
```

## 3. 实现

新增控制面板命令：

```text
--causal-learning-state-fixture-summary
--learning-state-fixture-summary
```

夹具动作：

```text
1. 初始化自我环境，不启动自我线程。
2. 使用当前自我内部世界作为状态场景。
3. 使用当前自我存在作为学习目标主体。
4. 使用 学习因果验证状态 作为学习目标特征。
5. 构造 当前状态：未验证。
6. 构造 目标状态：已验证。
7. 使用服务根需求作为上级需求预检字段。
8. 调用 C1 闸门摘要展示结果。
```

## 4. 验收

```text
1. Debug x64 构建通过。
2. `--causal-learning-state-fixture-summary` 0 码返回。
3. 输出中当前状态节点和目标状态节点非空。
4. 输出中闸门通过=是。
5. 输出中仍显示：入树=否，修改需求树=否，生成探索需求=否。
6. `--causal-learning-demand-gate-summary` 默认无学习候选时仍拒绝入树。
```
