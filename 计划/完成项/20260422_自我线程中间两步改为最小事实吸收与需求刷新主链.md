# 2026-04-22 自我线程中间两步改为最小事实吸收与需求刷新主链

## 目标

继续收窄自我线程主链，删除“从世界读取二次特征并重建治理帧、根值与禁止项长链”的旧逻辑，只保留：

- 按消息事实吸收本轮执行结果
- 刷新需求树与需求列表
- 选出主导需求与主导任务候选

## 本次改动

### 1. 简化 `步骤_刷新线程上下文并生成治理帧_`

文件：

- `D:\鱼巢\自我线程模块.impl.cpp`

改为：

- 不再读取自我内部世界中的二次特征
- 不再计算服务/安全净变化
- 直接遍历 `本轮冻结消息`
- 汇总：
  - 线程状态消息数
  - 结果状态消息数
  - 派生需求消息数
  - 成功/等待/失败结果数
  - 阻塞事实数
  - 风险事实数
- 只生成最小 `最近影响项摘要`
- `本轮最高优先级候选` 直接按事实消息抬高

### 2. 简化 `步骤_整理需求并做根层重判_`

文件：

- `D:\鱼巢\自我线程模块.impl.cpp`

改为：

- 不再做旧的服务值/安全值回写
- 不再调整旧根权重
- 不再生成禁止项、依赖权重、需求候选长链
- 直接调用 `私有_刷新需求列表并确保需求相关任务(...)`
- 再用现有 helper 选：
  - `本轮主导需求候选`
  - `本轮主导任务候选`
- 如果主需求适合任务化但任务缺失，则立即按需求补任务
- 仅生成最小的 `需求与权重治理输出快照`

## 当前主链

自我线程当前主链进一步收成：

`收消息 -> 规范化落账 -> 重算二次特征 -> 吸收事实消息 -> 刷新需求树/需求列表 -> 选主需求/主任务 -> 主派发`

其中“吸收事实消息”和“刷新需求树/需求列表”已经不再依赖旧治理帧重建。
