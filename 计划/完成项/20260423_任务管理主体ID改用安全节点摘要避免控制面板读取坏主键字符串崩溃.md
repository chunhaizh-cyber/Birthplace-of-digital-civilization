标题：任务管理主体ID改用安全节点摘要避免控制面板读取坏主键字符串崩溃

时间：2026-04-23

变更：
- 修改 [任务管理任务模块.impl.cpp](D:\鱼巢\任务管理任务模块.impl.cpp) 的 `私有_节点主键摘要(...)`。
- 不再通过 `节点->获取主键()` 读取字符串后做哈希。
- 改为直接使用节点指针地址生成安全摘要，用于主体/分身锚定。

原因：
- 控制面板读取任务管理专项快照时，会走 `读取任务管理上下文 -> 读取任务管理主体虚拟存在 -> 私有_主体ID_按宿主任务 -> 私有_节点主键摘要`。
- 当任务节点的主键字符串损坏或引用失效时，继续读 `获取主键()` 会在 `std::string` 分配路径上崩溃。
- 这里的摘要只用于主体/分身的运行期锚定，不需要强依赖主键字符串。

结果：
- 控制面板和专项快照读取不会再因为坏主键字符串在主体ID计算阶段崩溃。

验证：
- `msbuild D:\鱼巢\鱼巢.vcxproj /p:Configuration=Debug /p:Platform=x64 /p:TargetName=鱼巢_tmp_fix_0423_subjectid /m /clp:ErrorsOnly`
