# 方法运行账迁移计划 v0.1

日期：2026-05-06

## 1. 背景

当前方法主信息中仍混有运行态字段：

```text
最近执行时间
最近完成时间
运行次数
成功次数
失败次数
平均运行时长
最近运行虚拟存在
```

这些字段描述的是方法最近怎么运行，不是方法静态结构。继续放在方法主信息里，会让“方法是什么”和“方法运行账”混在一起。

## 2. 目标

将方法运行账从方法静态主信息迁出，优先落到：

```text
方法虚拟存在特征
运行虚拟存在
动态节点
因果记录
```

方法主信息只保留必要的静态结构与索引。

## 3. 迁移范围

优先迁移：

```text
最近执行时间
最近完成时间
最近运行虚拟存在
运行次数
成功次数
失败次数
平均运行时长
```

保留：

```text
结构_统计
```

但 `结构_统计` 也要降级为兼容镜像，后续再评估是否完全迁出。

## 4. 实施步骤

1. 搜索所有读取和写入运行账字段的位置。
2. 确认当前方法虚拟存在中已有对应特征：

```text
最近执行时间
最近完成时间
最近运行虚拟存在
运行次数
成功次数
失败次数
平均运行时长
```

3. 如果缺少特征定义，补齐特征词与读写 helper。
4. 将运行统计更新逻辑改为写入方法虚拟存在。
5. 将读取成功率、平均运行时长等逻辑改为从方法虚拟存在读取。
6. 方法主信息旧字段只作为兼容镜像同步一段时间。
7. 控制面板展示改为优先读方法虚拟存在。
8. 编译并运行摘要线程或控制面板检查。

## 5. 验收项

1. 方法执行后，运行次数、成功次数、失败次数写入方法虚拟存在。
2. 查找候选优先级读取方法虚拟存在统计，不依赖方法主信息运行字段。
3. 控制面板仍能显示方法运行摘要。
4. 方法静态结构不再因运行一次而必须修改主信息字段。
5. Debug x64 构建通过。

## 6. 暂不做

```text
物理删除主信息旧运行字段
完整运行账数据库化
运行账与因果记录彻底合并
```

这些在兼容窗口结束后另起清理计划。
