# 存在概念与实例创建最小闭环实现计划 v0.1

日期：2026-05-09

状态：已完成并归档。本文保留早期执行记录；其中“当前代码现状 / 还缺”反映计划制定时状态，不再代表当前代码现状。后续核心概念树、自我初始化概念层、任务 / 方法虚拟存在概念集统一入口已由 `20260509_存在概念抽象存在最小启动链实施计划_v0.2.md` 完成承接。

## 1. 目标

本计划实现“存在概念”的最小闭环：

```text
存在概念 = 用于解释词义、辅助创建实例的存在模板
实例存在 = 实际世界、场景、内部世界中的具体存在
```

只落两个用途：

```text
1. 自然语言理解：
   名词、专名、抽象名词等语素，优先指向一个概念存在。

2. 新建实例存在：
   根据一个或多个概念存在创建实例存在，并把概念中的基础记忆扩充到实例上。
```

不引入复杂本体系统。存在仍然只有一种结构：

```cpp
存在节点主信息类
```

## 2. 非目标

本轮明确不做：

```text
1. 不新增 抽象存在主信息类。
2. 不新增 枚举_存在形态。
3. 不新增复杂的概念归属记录。
4. 不记录概念归属置信度、来源、状态、命中特征、冲突特征。
5. 不把能力、用途、需求、方法、行为模板塞进概念归属关系。
6. 不新增“集合存在”形态。
7. 不做动态继承、逻辑合并、冲突集。
```

能力、用途、默认需求、默认方法等后续统一视为：

```text
新建实例时的记忆扩充内容
```

## 4. 数据结构调整

### 4.1 存在节点主信息类

文件：

```text
基础信息主信息类.h
```

新增字段：

```cpp
std::vector<可解析引用<存在节点类>> 概念集{};
```

保留旧字段：

```cpp
可解析引用<存在节点类> 概念模板{};
```

兼容语义：

```text
概念模板 = 主概念 / 默认概念 / 历史兼容字段
概念集 = 新逻辑权威字段
```

同步规则：

```text
1. 添加第一个概念时，如果 概念模板 为空，则同步写入 概念模板。
2. 调用旧 绑定概念模板 时，内部改为添加 / 替换主概念，并确保 概念集 包含该概念。
3. 新逻辑读取概念归属时优先读取 概念集。
4. 如果 概念集 为空但 概念模板 有效，则视为单概念集合。
```

## 6. 存在服务接口

文件：

```text
存在类.h
存在类.cpp
世界树类.h
世界树类.cpp
```

按当前代码分层，核心实现放在 `存在类`，`世界树类` 提供对外门面。

### 6.1 确保存在概念

接口：

```cpp
存在节点类* 确保存在概念(
    const 词性节点类* 名称,
    存在节点类* 父概念 = nullptr
);
```

行为：

```text
1. 父概念为空时，使用 世界树.取或创建存在概念根()。
2. 在父概念下按名称查找子存在。
3. 找不到则创建存在节点。
4. 不创建抽象存在，不写存在形态。
5. 返回概念存在。
```

### 6.2 创建实例存在

接口：

```cpp
存在节点类* 创建存在实例(
    const 词性节点类* 名称,
    const std::vector<存在节点类*>& 概念集,
    场景节点类* 所属场景 = nullptr
);
```

行为：

```text
1. 所属场景为空时，使用现实世界或调用方传入的世界 / 内部世界。
2. 在所属场景下创建存在实例。
3. 逐个写入 概念集。
4. 第一概念同步到 概念模板 兼容字段。
5. 调用 根据概念扩充实例记忆。
```

### 6.3 添加实例概念

接口：

```cpp
bool 添加实例概念(
    存在节点类* 实例,
    存在节点类* 概念,
    bool 是否扩充记忆 = true
);
```

行为：

```text
1. 概念为空或已存在时不重复添加。
2. 概念加入 概念集。
3. 概念模板 为空时写入该概念。
4. 是否扩充记忆 为 true 时，调用 根据概念扩充实例记忆。
```

### 6.4 根据概念扩充实例记忆

接口：

```cpp
void 根据概念扩充实例记忆(
    存在节点类* 实例,
    const std::vector<存在节点类*>& 概念集
);
```

第一版扩充范围：

```text
1. 复制 / 实例化概念存在下的直接特征。
2. 递归补充概念存在自己的上位概念集。
3. 同类型特征按词性类型去重。
```

暂不做：

```text
1. 不动态继承。
2. 不做冲突判定。
3. 不复制运行中的任务。
4. 不复制已发生的实例因果。
```

后续扩充范围预留：

```text
1. 默认需求模板 -> 实例需求。
2. 默认方法模板 -> 实例方法根。
3. 默认能力 -> 实例能力特征。
```

这些不进入 概念集 字段，只作为实例创建时的扩充内容处理。

## 7. 语素入口调整

文件：

```text
语素类.h
语素类.cpp
基础信息类.cpp
```

### 7.1 名词性词性默认绑定概念存在

名词、专名、抽象名词、地名、时间词、方位词、n、nr、ns、nt、nz、PER、LOC、ORG、TIME 默认仍映射到：

```text
枚举_主信息类型::存在
```

但当它们作为自然语言模板创建时，应优先创建到：

```text
存在概念根
```

调用链定型为：

```text
语素集.确保词性对应基础信息("飞机", 名词)
    ↓
词性节点
    ↓
确保存在概念("飞机")
    ↓
词性节点.对应基础信息 = 存在概念根 / 飞机
```

### 7.2 专名 / 实例名绑定

专名可能有两种情况：

```text
1. “飞机”这种类名专名：绑定概念存在。
2. “B-1234”这种实例名：绑定实例存在。
```

规则：

```text
默认 确保词性对应基础信息 仍创建概念存在。
如果调用方明确已有实例，应使用 绑定词性词到场景基础信息(...) 绑定到实例存在。
```

这样不在语素层猜“专名到底是类还是实例”，避免误建实例。

## 11. 风险点

| 风险 | 处理 |
| --- | --- |
| `语素类` 当前只拿 `基础信息类`，不知道 `世界树` 的概念根 | 优先通过调用方传入父节点；必要时在 `世界树类` 提供名词概念门面，避免强行引入循环依赖 |
| 旧 `概念模板` 调用点很多 | 保留字段，先同步到 `概念集`，表达层双显示 |
| 复制方法 / 需求容易复制运行态 | 第一版只复制特征；需求 / 方法只留默认模板扩充钩子 |
| 专名可能是概念也可能是实例 | 默认自然语言入口走概念，明确场景绑定才指向实例 |
| 集合需求容易诱导新节点类型 | 坚持用概念集 + 特征条件查询 |

## 12. 验收口径

本轮完成后应满足：

```text
1. 存在仍只有 存在节点主信息类。
2. 没有新增存在形态枚举。
3. 没有新增抽象存在主信息类。
4. 名词性语素默认指向概念存在。
5. 实例存在可以拥有多个概念存在。
6. 创建实例时可以把概念特征扩充到实例。
7. 旧概念模板字段仍兼容可用。
8. 集合查询不依赖集合存在节点。
```

一句话：

```text
概念存在只是词义锚点 + 实例创建模板；
实例创建出来以后，概念中的基础记忆变成实例自己的记忆。
```
