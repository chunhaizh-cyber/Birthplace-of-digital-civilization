# 20260508 任务推进状况回执闭环与目标方法头待重入计划 v0.1

## 1. 背景

当前任务闭环已经收束为：

```text
筹办和执行只返回状况
任务管理线程解释状况
自我线程改树、结账、重调度
子任务完成后父任务待重筹办
```

但“方法查找未命中”的处理口径需要进一步固定：

```text
方法查找未命中
    ≠ 立即创建候选方法存在状态子需求

方法查找未命中
    = 筹办阶段创建或复用待学习目标方法头，并返回待重入
```

## 2. 目标

1. 新增统一结构化任务推进回执，承载筹办 / 执行状况。
2. 固定三层职责边界：

```text
本能函数产出状况
任务管理线程解释状况
自我线程改需求树、任务树、价值账和风险账
```

3. 将“方法查找未命中”改为：

```text
创建或复用待学习目标方法头
写入父需求目标结果能力
父任务待重筹办
返回已建立方法头待重入
```

4. 方法头存在但不可执行时，才生成可执行性学习子需求。
5. 子需求目标主体必须是方法头、方法虚拟存在、任务或参数绑定场景，不得复制父需求目标。
6. 同一方法树下不得重复创建相同目标结果能力的待学习方法头。

## 3. 非目标

1. 不恢复旧任务步骤节点。
2. 不恢复 `自我_查找方法`、`自我_任务执行一步` 等旧本能路径。
3. 不让本能函数直接改父需求树或结算父需求。
4. 不把方法查找未命中解释成方法命中。

### 阶段四：方法查找未命中改为目标方法头待重入

状态：已完成。

修改筹办逻辑：

```text
查找满足父任务目标的方法
    ↓
未命中
    ↓
查找是否已有待学习方法头覆盖同一目标结果
    ↓
已有则绑定
没有则创建
    ↓
写入目标结果能力
    ↓
标记待学习 / 不可执行
    ↓
父任务待重筹办
    ↓
返回已建立方法头待重入
```

待学习方法头至少写入：

```text
来源需求
来源任务
目标特征类型
目标状态
目标状态关系 / 方向
成熟度 = 待学习
可执行状态 = 不可执行
```

验收：

```text
方法查找未命中时不创建子需求。
下一轮筹办能按结果能力命中新建方法头。
命中后仍因不可执行进入结构缺失判断。
```

### 阶段五：待学习方法头去重

状态：已完成。

建立或复用待学习方法头查找：

```text
目标特征类型
目标状态关系 / 方向
目标主体类型或目标主体范围
来源需求或适用范围
```

不得按名称、动作句柄或固定本能 ID 命中。

验收：

```text
同一目标结果能力不会重复创建多个待学习方法头。
```

### 阶段六：方法头不可执行才生成学习子需求

状态：已完成。

将缺口状况映射为可执行性子需求：

| 状况 | 子需求目标 |
| --- | --- |
| 方法缺动作骨架 | 方法动作骨架状态 |
| 方法缺可执行入口 | 方法可执行入口状态 |
| 方法缺条件节点 | 方法条件节点存在状态 |
| 方法缺结果节点 | 方法结果节点存在状态 |
| 方法缺条件结果对 | 方法条件结果配对状态 |
| 参数不可配 | 参数配对状态 / 任务参数绑定状态 |
| 方法运行账不足 | 方法运行账完整性 |
| 方法结果不稳定 | 方法稳定状态 |

学习 / 练习只是可能匹配这些方法状态目标的普通方法，不作为硬性派发通道。

验收：

```text
子需求目标主体是方法头、方法虚拟存在、任务或参数绑定场景。
父子同目标硬阻断仍然有效。
```


## 5. 验收标准

1. 方法查找未命中不创建候选方法存在状态子需求。
2. 方法查找未命中创建或复用待学习目标方法头。
3. 待学习方法头写入父需求目标结果能力。
4. 创建方法头后父任务待重筹办，不完成父需求。
5. 下一轮筹办能按结果能力命中该方法头。
6. 方法头不可执行时才创建学习子需求。
7. 子需求目标不复制父需求目标。
8. 筹办 / 执行状况有结构化回执。
9. 编译通过，摘要运行正常。
