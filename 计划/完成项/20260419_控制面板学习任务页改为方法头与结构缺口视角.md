# 20260419 控制面板学习任务页改为方法头与结构缺口视角

## 本次完成

- 调整控制面板中“学习任务”页面的显示口径，不再优先展示旧的阶段机摘要。
- 学习任务列表、就绪队列、等待表的标题改为优先显示：
  - 当前学习方法
  - 学习方法
  - 兜底方法
  - 最后才回退到学习子任务标题
- 每个学习项的摘要改为围绕：
  - 当前学习目标
  - 当前学习需求
  - 当前候选学习程序
  - 当前方法是否可执行
  - 是否可直接进入学习
- 每个学习项新增详情行，显示：
  - 子任务 / 阶段 / 状态 / 等待原因 / 影响面
  - 当前方法 / 兜底方法 / 兜底可执行 / 是否已切换兜底 / 是否已由兜底进采样
  - 待学习方法骨架摘要
  - 学习目标
  - 学习需求
  - 原函数需求
  - 方法选择
  - 候选程序
  - 抽象因果
  - 最近反馈
- 学习页面副标题改为：
  - 查看待学习方法、结构缺口、候选学习程序与当前工单去向
- 页面三栏标题改为：
  - 学习工单
  - 可调度工单
  - 等待工单
- 页面三栏渲染从简单表格切换为支持详情行的卡片队列视图。
- 线程学习摘要与终端学习摘要同步补充“当前摘要”显示，便于快速看当前学习主线。

## 结果

- 控制面板里的学习任务显示，现在更贴近“让方法可用”的简化学习主线。
- 旧的“子任务/阶段机”不再作为第一视角，而退到详情行中。
- 当前待学习方法、结构缺口和候选学习程序已经成为学习页的主要阅读入口。
