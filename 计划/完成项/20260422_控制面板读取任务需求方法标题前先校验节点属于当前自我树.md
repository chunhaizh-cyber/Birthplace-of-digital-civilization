# 20260422_控制面板读取任务需求方法标题前先校验节点属于当前自我树

本次修复了控制面板读取快照时的一条展示层崩溃路径。

## 问题
控制面板会直接把 `当前主任务/当前主需求/当前主方法` 以及治理快照里的节点传给 `私有_任务标题/需求标题/方法标题`，这些标题函数会继续解引用节点主信息和词指针。如果传入的是旧裸指针或失效节点，就会在 `语素类::获取词(...)` 或 `节点->获取主键()` 中崩溃。

## 修复
- 在 `控制面板类.cpp` 中新增“节点是否属于当前自我需求树/任务树/方法树”的校验 helper。
- `私有_需求标题(...) / 私有_任务标题(...) / 私有_方法标题(...)` 现在都会先校验节点是否属于当前自我树；不属于就直接返回 `失效需求引用 / 失效任务引用 / 失效方法引用`。
- `读取控制面板快照(...)` 读取 `当前主需求 / 当前主任务 / 当前主方法` 时，也先过树内成员校验，再写入快照。

## 结果
控制面板展示层不再直接信任旧裸指针；即使主链快照里还有残留失效引用，也会在展示层被截断成“失效引用”，不再崩溃。
