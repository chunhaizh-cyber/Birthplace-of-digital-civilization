# 20260422_修复自我线程按需求创建根任务时的悬空任务节点崩溃

1. 崩溃根因：`私有_刷新需求列表并确保需求相关任务(...)` 和新的主派发最小链在创建根级任务时，使用了局部 `任务类` 对象的默认临时根指针。
2. `任务类::创建任务头结点(...)` 在 `上级任务节点 == nullptr` 时会把新任务挂到 `this->根指针` 下，因此函数返回后局部 `任务类` 析构，根任务节点随临时树一起失效。
3. 表现：`自我线程类::自我初始化(...)` 中回填 `当前主任务` 时，访问悬空任务节点的 `获取主键()` 触发 `std::_Xlen_string()` 崩溃。
4. 修复：在 `自我线程模块.impl.cpp` 新增 `私有_按需求创建根级任务头(...)`，显式把局部 `任务类` 的 `根指针` 临时切到 `世界树` 的真实任务根，再创建根级任务头。
5. `按需求补任务` 与 `步骤_生成并执行主派发决议_()` 的根任务创建入口都已统一改走该 helper。
6. 验证通过：`msbuild D:\鱼巢\鱼巢.vcxproj /p:Configuration=Debug /p:Platform=x64 /p:TargetName=鱼巢_tmp /m /clp:ErrorsOnly`
