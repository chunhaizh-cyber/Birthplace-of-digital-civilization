# 方法主信息 variant 承载改造计划 v0.1

日期：2026-05-06

## 1. 背景

当前 `方法类` 继承：

```cpp
class 方法类 : public 链表模板<方法信息基类>
```

如果链表模板按值保存 `方法信息基类 主信息`，那么现有：

```cpp
方法首节点主信息类
方法条件节点主信息类
方法结果节点主信息类
```

只起到构造时设置 `节点种类` 的作用，不能真正形成不同承载结构。前置计划会先把角色面归组到同一个主信息里。本计划在这些前置完成后，再评估并改造为 `std::variant`。

## 2. 前置条件

必须先完成：

```text
20260506_方法首节点结果能力面拆分计划_v0.1.md
20260506_方法主信息角色面拆分计划_v0.1.md
20260506_方法运行账迁移计划_v0.1.md
```

并确认：

```text
查找方法只读结果能力集
条件节点逻辑只读条件节点信息
结果节点逻辑只读结果节点信息
运行账已迁出静态主信息
```

## 3. 目标结构

```cpp
using 方法角色信息 = std::variant<
    std::monostate,
    结构_方法首节点信息,
    结构_方法条件节点信息,
    结构_方法结果节点信息
>;

class 方法主信息类 {
public:
    结构_方法节点公共信息 公共{};
    方法角色信息 角色{};
};
```

或在保留名称兼容时：

```cpp
class 方法信息基类 {
public:
    结构_方法节点公共信息 公共{};
    方法角色信息 角色{};
};
```

## 4. 实施步骤

1. 检查 `链表模板` 对主信息的构造、拷贝、移动和比较要求。
2. 确认 `std::variant` 放入链表节点后不会破坏序列化、控制面板展示和复制逻辑。
3. 将 `方法信息基类` 的角色面字段替换为 `方法角色信息 角色`。
4. 提供访问 helper：

```text
首节点信息()
条件节点信息()
结果节点信息()
确保首节点信息()
确保条件节点信息()
确保结果节点信息()
```

5. 将全部直接字段访问替换为 helper 访问。
6. 删除派生主信息类或改成工厂函数。
7. 删除旧兼容镜像字段。
8. 编译并运行方法树摘要/控制面板验收。

## 5. 风险点

```text
链表模板按值复制主信息的行为
控制面板直接读取主信息字段
方法类比较函数依赖旧字段
旧说明书和测试入口依赖字段平铺
```

需要先做小范围编译检查，再逐批替换。

## 6. 验收项

1. 方法首节点无法误读条件节点专属字段。
2. 方法条件节点无法误读动作句柄。
3. 方法结果节点无法误读条件场景。
4. 查找方法仍通过结果能力集命中。
5. 方法创建、补齐、结果节点创建、条件节点创建均通过。
6. Debug x64 构建通过。

## 7. 暂不做

```text
改变方法树链表结构
改变基础信息节点体系
引入运行时反射系统
重写控制面板所有展示模型
```
