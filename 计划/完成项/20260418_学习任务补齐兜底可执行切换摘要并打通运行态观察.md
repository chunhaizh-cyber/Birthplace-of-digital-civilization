# 20260418 学习任务补齐兜底可执行切换摘要并打通运行态观察

## 本步目标

在前一刀已经打通“当前候选不可执行时，学习任务显式切到兜底尝试学习方法”的基础上，
继续把这次切换做成运行态可观察事实，
让后续查看学习账本、调度摘要、控制面板时，能直接看见：

- 当前是否切到了兜底可执行方法
- 为什么切
- 切完之后是否进入可调度

## 完成内容

1. 给学习任务同步输入与学习实体补入显式切换信息
   - 新增：
     - `已切换到兜底可执行方法`
     - `当前学习方法选择摘要`
   - 位置：
     - `D:\鱼巢\学习任务模块.ixx`
     - `D:\鱼巢\学习任务模块.cpp`

2. 任务管理同步学习任务时正式生成“方法选择摘要”
   - 在 `D:\鱼巢\任务管理任务模块.impl.cpp` 中：
     - 区分 `当前候选已可执行`
     - `兜底方法已可执行`
     - `当前仍缺可执行方法头`
   - 当“当前候选不可执行、兜底可执行”时：
     - `已切换到兜底可执行方法 = true`
     - `当前学习方法选择摘要 = 当前候选未就绪，已切换到兜底可执行方法=...`

3. 学习对象适配器与学习调度摘要开始消费这份切换信息
   - `D:\鱼巢\学习对象适配器.cpp`
     - 来源摘要与适配摘要会带上 `方法选择=...`
   - `D:\鱼巢\学习调度模块.cpp`
     - 调度摘要会带上：
       - `已切兜底可执行方法`
       - `选择=...`

4. 学习实体摘要与任务管理最近反馈同步补齐
   - `D:\鱼巢\学习任务模块.cpp`
     - 学习实体摘要新增：
       - `已切兜底执行=是/否`
       - `方法选择=...`
   - `D:\鱼巢\任务管理任务模块.impl.cpp`
     - 最近反馈会明确写：
       - `学习方法选择=当前候选不可执行，已切换到兜底尝试学习方法`

## 结果

现在“尝试学习真正可运行”这条链不再只是内部逻辑成立，而是可以在运行态直接观察到：

- 当前选中的学习方法是不是可执行
- 是否因为当前候选不可执行而切到了兜底尝试学习方法
- 调度摘要是否已经认定其为可直接进入学习

这为下一步做真正运行态验证铺平了路：
当出现“当前候选不可执行、兜底尝试学习可执行”的场景时，
应该能在学习账本与调度快照里直接看到它进入 `可调度 / 待调度`。
