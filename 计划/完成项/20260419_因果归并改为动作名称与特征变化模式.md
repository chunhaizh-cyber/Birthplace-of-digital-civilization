# 20260419 因果归并改为动作名称与特征变化模式

## 本次目标

把抽象因果的归并口径从“实例化动作语义 + 过细迁移结构”收紧为：

- 动作侧：`动作名称`
- 迁移侧：`变化特征 + 方向` 或 `变化特征 + 因值对果值`

不再让 `动作语义键` 和实例级主体/客体继续主导归并。

## 已完成

1. `因果主信息类` 新增 `动作名称` 字段，作为抽象动作信息承载。
2. `因果类` 的动作归并逻辑改为优先比较 `动作名称`，仅在动作名称缺失时才回退到动作模板引用。
3. `动态 -> 因果` 时，先从 `来源动作名 / 来源动作节点 / 动作模板节点` 抽取 `动作名称`，不再退回到动态名称。
4. `状态模板结构一致(...)` 不再要求 `状态主体` 一致。
5. `迁移结构一致(...)` 不再比较 `主体 / 客体 / 概念名称` 的实例引用，改为只看左右状态的特征结构与值形态。
6. 控制面板因果标题、因果摘要、先天动作匹配优先改读 `动作名称`。
7. 语言命名链里，因果节点名称优先读 `动作名称`，`动作语义键` 只作为兼容回退。

## 当前边界

- `动作语义键` 还保留在因果主信息里，主要用于兼容旧链和兜底显示。
- 归并主链已经不再依赖 `动作语义键`。
- 迁移归并现在更接近“动作名称 + 特征变化模式”，但运行态是否显著提升归并率，还需要后续回放验证。
