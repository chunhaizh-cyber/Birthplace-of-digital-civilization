# 20260418 派生需求待实例化迁入治理桥并削弱批次去向本地映射

## 本步目标

继续削弱 `自我线程` 在主派发阶段对 `批次执行去向 -> 最终去向` 的本地映射，让治理桥更早接管 `继续提交 / 回到重筹办` 这类软分流。

## 本步完成内容

1. 将 `私有_映射批次执行去向到线程最终去向(...)` 收口成只保留硬边界：
   - `停止 -> 停止`
   - `进入收束 -> 进入收束`
   - `待机 -> 待机`
   - `继续提交 -> 继续提交`
2. 去掉了该映射中对 `有派生需求待实例化` 的本地分支，不再由线程直接把 `继续提交` 改写成 `回到重筹办`。
3. 在 `自我_主循环治理模块` 的 `结构_一步治理输入_最小` 中新增 `主派发有派生需求待实例化`。
4. 治理桥在 `当前去向 == 执行方法` 分支下，若命中 `主派发有派生需求待实例化`，会改判为：
   - 去向：`继续治理`
   - 缺口：`待实例化派生需求`
   - 摘要：返回治理侧实例化入口
5. `主派发输出快照` 同步新增 `有派生需求待实例化`，让桥层快照重读时也能复现这份口径。

## 主要改动文件

- `D:\鱼巢\自我线程模块.impl.cpp`
- `D:\鱼巢\自我线程模块.ixx`
- `D:\鱼巢\自我_主循环治理模块.ixx`

## 当前效果

- `批次去向` 对线程最终去向的影响进一步降成“边界种子”。
- `派生需求待实例化` 这类软分流开始由治理桥显式消费，而不是由线程本地映射提前裁决。
- 治理桥第一次拿到了一个更正式的“返回治理侧实例化入口”的缺口类型：`待实例化派生需求`。
