# 标准原子状态词典与自检报告候选承接计划 v0.2

日期：2026-05-10

## 1. 背景

`20260510_自检候选需求化原子状态承载计划_v0.1.md` 的 P3 暂缓原因是：自检候选虽然已经有“报告型 / 需求型”分流和原子目标字段，但边界需要改为“自检线程只报送缺失，自我线程决定是否需求化”。

不能继续用：

```text
缺什么 -> 随手造一个泛化状态
```

因为这会违反当前全局错误处理规则：非正常逻辑错误必须弹错 / 报错并记录日志，不能被业务需求兜底吞掉。

---

## 2. 目标

建立最小标准原子状态词典，使自检报告在被自我线程决策为可修复需求时具备明确目标：

```text
目标主体
+ 目标特征类型
+ 当前状态
+ 目标状态
+ 满足关系
```

第一阶段只提供词典和边界，不允许自检线程把关键结构缺失自动需求化。

---

状态：已完成。


```cpp
自检原子状态词典::已就绪
```


## 4. 自我线程需求化边界

### 自我线程可以转为业务需求的情况

```text
1. 目标主体是基础信息节点或存在节点。
2. 当前状态和目标状态都可创建为状态节点。
3. 缺口属于可恢复业务缺口，而不是初始化 / 线程边界 / 权威入口错误。
4. 修复动作已经有明确承接方法或允许由学习任务承接。
```

例如：

```text
任务虚拟存在绑定状态：未就绪 -> 已就绪
```

但这还需要明确主体承载。任务信息节点不是基础信息节点，因此不能直接作为状态主体；应由界面线程错误或任务语义对象承接。自检线程不得硬造需求，只能报送缺失。

### 必须报错而非需求化的情况

```text
1. 当前自我实例缺失。
2. 自我内部世界缺失。
3. 请求创建需求根 / 任务根 / 方法根失败。
4. 任务建立流程完成后仍未绑定任务虚拟存在。
5. 目标主体不是基础信息节点，无法创建合法状态节点。
```

注意：

```text
需求根 / 任务根 / 方法根 是存在信息结构字段，默认可以为空。
未使用时为空不是错误。
任务虚拟存在在建立任务时由程序建立；未建立任务时为空不是错误。
```

上述错误属于工程创建失败或模块职责越界，必须：

```text
弹错 / 报错
记录日志
阻断当前候选入需求树
进入待完成项或错误面板
```

## 5. 后续分步

### P2：创建原子状态节点工具

状态：已完成。

动作：

```text
1. 封装“主体 + 特征类型 + I64状态值 -> 状态节点”的小工具。
2. 要求主体必须是基础信息节点。
3. 缺场景时不静默落默认根；必须使用自我内部世界或明确场景。
4. 创建失败要报错 / 记录日志。
```

落地：

```cpp
自我类::创建自检I64原子状态(
    场景节点类* 场景,
    基础信息节点类* 主体,
    const 语素入口节点类* 特征类型,
    I64 状态值,
    时间戳 now);
```

失败时写入 `项目运行错误日志`，不兜底创建场景或泛化主体。

### P3：首批需求型候选试点

状态：已改为自我线程侧后续计划。

候选：

```text
核心状态可读写状态：未就绪 -> 已就绪
```

说明：

```text
主体可用当前自我存在；
特征类型可用 类型_自检_核心状态可读写状态；
但只能由自我线程在收到自检报告后，按修复时机和动作边界决定是否生成需求。
若修复会扰动运行中任务 / 方法 / 状态索引，应等自我休眠期再处理。
```

### P4：结构根缺失处理

状态：第一轮已完成。

结论：

```text
需求根 / 任务根 / 方法根 属于存在信息结构字段，默认可以为空。
缺失本身不是非正常逻辑错误。
只有当流程需要使用并请求创建失败时，才按非正常逻辑错误处理。
```

需要补：

```text
1. 创建链失败时弹错 / 记录错误日志。
2. 控制面板显示“根字段为空 / 未使用”与“创建失败”的区别。
3. 自检线程不因根字段默认空而自动生成需求，只报送缺失事实。
```

已落地：

```text
1. `自我类::确保自我根需求与根任务已初始化_` 在明确执行根创建链后复核需求根 / 任务根 / 方法根。
2. 若请求创建后仍缺任一根，调用 `项目运行错误日志` 并弹出 `鱼巢 - 自我结构根逻辑错误`。
3. 控制面板快照新增自我需求根 / 任务根 / 方法根字段建立状态；`--summary` 显示 `根字段=需/任/方`。
4. 自检线程仍只报送缺失事实，不因根字段默认空自动需求化。
```

保留待完成：

```text
`存在类::取存在主信息 / 获取需求根节点 / 获取任务根节点 / 获取方法根节点`
当前仍会在读取时隐式补齐三类根节点。它与“结构字段默认可空，按需确保”的最终口径存在实现层张力。
后续应另起小计划拆分“只读获取”和“显式确保”，避免诊断读取触发创建。
```

### P5：任务虚拟存在缺失处理

状态：第一轮已完成。

结论：

```text
任务虚拟存在在建立任务时由程序建立。
未建立任务时为空不是错误。
任务建立流程完成后仍为空，属于任务创建链或任务管理界面线程职责错误。
```

后续应接入：

```text
任务管理界面线程错误日志
任务管理摘要故障字段
控制面板弹错 / 报错
不由自检线程直接创建需求。
若属于可恢复一致性问题，由自我线程在休眠期修复门控中决策。
```

已落地：

```text
1. 任务管理界面线程在任务治理请求承接阶段确保任务虚拟存在。
2. 任务建立流程完成后仍无法创建 / 解析任务虚拟存在时，拒绝派发工作线程。
3. 该错误会写入 `项目运行错误日志`，并弹出 `鱼巢 - 任务语义绑定逻辑错误`。
4. 任务管理界面线程快照标记故障，记录最近任务、需求、调度动作和失败说明。
5. `--task-mgmt-summary` 已显示界面线程故障、最近任务、任务虚拟存在和最近说明。
6. 自检线程不直接创建任务虚拟存在需求；可恢复一致性问题仍等待自我线程休眠期修复门控映射表。
```
