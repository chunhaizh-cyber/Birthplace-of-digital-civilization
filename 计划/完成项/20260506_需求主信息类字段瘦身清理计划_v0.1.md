# 需求主信息类字段瘦身清理计划 v0.1

状态：已完成，2026-05-06 归档。

## 目标

对照当前代码删除 `需求主信息类` 中无用或旧逻辑占位字段，保留现行需求树、任务关联、派生需求去重、需求结算仍在使用的字段。

## 字段结论

### 保留

1. 基础描述：`描述信息`、`类型`、`统计`。
2. 需求树语义：`结构角色`、`需求主体`、`需求场景`、`被需求存在`、`被需求当前状态`、`需求状态`。
3. 任务关联：`对应任务`、`任务列表`。
4. 生命周期与权重：`需求有效截止`、`安全权重`、`服务权重`。
5. 结算锚：`结算锚已初始化`、`初始状态值`、`目标状态值`、`已结算状态值`、`累计安全结算`、`累计服务结算`、`最近结算任务主键`、`最近结算时间`。
6. 派生需求语义：`来源方法主键`、`派生需求类型`、`方法需求位`、`本能能力缺口类型`、`主观察特征`。

### 删除

这些字段没有现行写入，或只剩旧控制面板展示/旧来源判断读取：

1. `需求虚拟存在`
2. `相关任务`
3. `相关任务列表`
4. `来源任务头结点`
5. `来源步骤节点`
6. `来源任务头主键`
7. `来源步骤主键`
8. `来源需求主键`
9. `来源类型`
10. `任务挂接方式`

同步删除 `结构_需求树更新指令` 中不再被执行链消费的旧来源/挂接字段：

1. `来源任务头指针`
2. `来源任务头主键`
3. `来源步骤指针`
4. `来源步骤主键`
5. `任务挂接方式`

## 执行步骤

1. 修改 `需求主信息类.h` 和 `需求类.h`。
2. 清理自我线程中旧来源字段判断。
3. 清理控制面板旧字段展示。
4. 编译验证。
5. 写入执行记录并归档。
