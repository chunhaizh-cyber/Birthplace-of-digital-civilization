# 学习子任务自动生成当前步骤节点并接入需求驱动步骤映射

## 本次完成

本次在 `D:\鱼巢\任务管理任务模块.impl.cpp` 补齐了学习子任务的最小任务树闭环。

- 新增“学习步骤动作类型”推导函数：
  - 优先从 `学习动作集合` 中挑选最高优先级且必须执行的动作
  - 若动作集合为空，则回退到“候选学习程序裁决 + 方法骨架缺口 + 抽象因果状态”
- 新增“学习步骤类型词”生成函数：
  - 将学习步骤显式区分为 `学习步骤_候选动作生成`、`学习步骤_共同条件提取`、`学习步骤_结果预测`、`学习步骤_沙箱验证` 等
- 新增“学习步骤执行方法”推导函数：
  - 学习步骤的语义目标方法优先保持为“当前待学习方法”
  - 当目标方法暂不可执行时，允许执行方法切到“兜底尝试学习方法”
- 新增“确保学习子任务当前步骤”函数：
  - 当学习子任务已经创建、学习方法骨架已经写回后，自动为学习子任务创建或复用一个当前步骤节点
  - 若现有步骤的步骤类型、语义方法、执行方法都匹配，则直接复用
  - 若不匹配，则自动追加新步骤，形成 `学习子任务头 -> 学习步骤 -> 学习结果` 结构
- 在学习承接主链里接入了上述 helper：
  - 位置在“创建/复用学习子任务头 + 写回学习方法骨架 + 同步学习任务实体”之后
  - 会把“已创建学习步骤/已复用学习步骤”写进最近反馈片段

## 语义收口

这次改动明确了两层方法语义：

- `可用方法首节点`
  - 表示当前学习步骤真正服务的“待学习方法”
- `执行方法`
  - 表示当前学习步骤这轮实际使用的执行入口
  - 当待学习方法尚不可执行时，可以切到兜底尝试学习方法

这样可以避免“学习步骤一旦切到兜底执行，任务树里就丢失真正学习对象”的问题。
