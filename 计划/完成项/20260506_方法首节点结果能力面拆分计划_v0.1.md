# 方法首节点结果能力面拆分计划 v0.1

日期：2026-05-06

## 1. 背景

当前 `方法信息基类` 同时承载方法首节点、方法条件节点、方法结果节点和运行账相关字段。字段存在并不等于字段对当前节点有效，导致查找、学习、练习逻辑需要反复依赖 `节点种类` 做语义判断。

当前阶段已经把方法查找口径收束为：

```text
需求结果 = 目标特征类型 + 目标方向
方法结果 = 可改变特征类型 + 可造成方向
```

因此最先需要从胖基类中拆出来的是“方法结果能力面”。本计划不立即重写链表模板，也不一次性改为 `variant`，先在现有 `方法信息基类` 中增加结构化能力面，让查找方法只读取能力集。

## 2. 目标

1. 在 `方法主信息类.h` 中新增 `结构_方法结果能力`。
2. 在 `方法信息基类` 中临时增加 `结构_方法首节点能力面 首节点能力`。
3. 将 `主结果特征类型 / 结果方向索引` 迁移为兼容镜像，不再作为查找主入口。
4. `查找方法` 只遍历 `首节点能力.结果能力集`。
5. 三个学习本能方法通过结果能力集声明多种产出。
6. `改变特征值` 标记为根写入原语，当前阶段不作为普通方法查找命中。

## 3. 新增结构

```cpp
struct 结构_方法结果能力 {
    const 词性节点类* 目标特征类型 = nullptr;
    std::vector<可解析引用<二次特征节点类>> 方向索引{};
    I64 最小变化方向 = 0;
    I64 置信度_Q10000 = 0;
    I64 成熟度阶段 = 0;
    bool 可被方法查找命中 = true;
    bool 是否根写入原语 = false;
};

struct 结构_方法首节点能力面 {
    std::vector<结构_方法结果能力> 结果能力集{};
};
```

临时承载：

```cpp
class 方法信息基类 {
public:
    结构_方法首节点能力面 首节点能力{};
};
```

## 4. 实施步骤

### 4.1 新增能力面

修改 `方法主信息类.h`：

1. 增加 `结构_方法结果能力`。
2. 增加 `结构_方法首节点能力面`。
3. 在 `方法信息基类` 中加入 `首节点能力`。
4. 保留现有 `主结果特征类型` 和 `结果方向索引`，标记为旧兼容镜像。

### 4.2 建立迁移 helper

在 `方法类.cpp` 中新增内部 helper：

```text
确保结果能力集存在
从旧字段生成默认结果能力
追加或合并结果能力
按 目标特征类型 + 方向索引 去重
```

规则：

```text
如果 结果能力集 非空：
    以结果能力集为准。

如果 结果能力集 为空，但 主结果特征类型 + 结果方向索引 存在：
    生成一个默认结果能力。
```

### 4.3 本能方法补齐写入能力集

调整本能方法结果声明补齐逻辑，使其写入 `首节点能力.结果能力集`。

练习：

```text
方法可靠度 ↑
方法稳定状态 ↑
方法是否可用 → 可用
方法运行账完整性 ↑
方法成熟度阶段 ↑
方法新用途发现状态 → 发现候选
```

尝试学习：

```text
候选方法存在状态 → 已存在
方法动作骨架状态 → 已形成
方法条件结果配对状态 → 已配对
方法结构是否完整 ↑
```

定向学习：

```text
学习方向状态 → 已形成
目标特征类型识别状态 → 已识别
目标变化方向识别状态 → 已识别
```

`改变特征值`：

```text
是否根写入原语 = true
当前普通方法查找跳过
```

### 4.4 查找方法改读能力集

修改 `自我动作实现模块.ixx` 的 `方法能满足需求结果_查找方法`：

```text
方法必须是方法首节点
遍历 首节点能力.结果能力集
    跳过 可被方法查找命中 == false
    跳过 是否根写入原语 == true
    匹配 目标特征类型
    匹配 方向索引
命中则找到
否则找不到
```

旧字段只能通过迁移 helper 进入能力集，不再直接作为主判断入口。

### 4.5 执行桥同步

修改 `自我_执行桥模块.ixx` 中按结果特征查找方法的最小实现，使其也读取 `首节点能力.结果能力集`，保持与 `查找方法` 一致。

### 4.6 文档同步

同步更新：

```text
README.md
规范/方法规范20260425/README.md
规范/学习相关本能函数规范20260505.md
说明书/方法类说明.md
```

说明：

```text
主结果特征类型 / 结果方向索引 是旧兼容镜像。
结果能力集 是方法查找的权威结构。
动作名、动作句柄、本能 ID 只能作为执行入口或调试信息，不参与查找命中。
```

## 5. 验收项

1. 只有动作句柄但没有结果能力的方法，查找不到。
2. `方法可靠度 ↑` 能命中 `练习`。
3. `候选方法存在状态 → 已存在` 能命中 `尝试学习`。
4. `学习方向状态 → 已形成` 在外界信息上下文可命中 `定向学习`。
5. 内部方法补齐入口闭环不生成学习方向需求，因此不会命中 `定向学习`。
6. `改变特征值` 标为根写入原语后，不作为普通查找命中。
7. 旧字段存在但能力集为空时，可以迁移生成默认能力；迁移后查找仍走能力集。
8. Debug x64 构建通过。

## 6. 暂不做

本计划暂不执行：

```text
链表模板改为 variant 承载
一次性拆完方法首节点 / 条件节点 / 结果节点主信息
删除主结果特征类型 / 结果方向索引旧字段
迁移运行账到动态或方法虚拟存在
```

这些作为后续结构瘦身计划承接。
