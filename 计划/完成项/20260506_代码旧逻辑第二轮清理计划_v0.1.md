# 20260506_代码旧逻辑第二轮清理计划_v0.1

## 背景

第一轮旧逻辑残留清理已完成，`调用点`、运行审计模块、阶段事件和调试快照等审计链路已经清零并通过 Debug x64 构建。

本轮对照代码继续发现以下旧逻辑残留：

1. 仓库根目录仍存在一次性探针文件 `_probe_root.cpp`。
2. 顶层需求状态结算仍保留直接给自我安全值 / 服务值加值的路径。
3. 需求主信息类中仍保留部分“状态结算锚”字段。
4. 任务树内部仍存在“任务管理任务”和“任务管理树闸门”运行路径。
5. 任务 / 方法结果节点仍作为旧结构被创建、统计和传递。
6. 控制面板仍保留“历史宿主残留”未实现专项入口。
7. `鱼巢.cpp` 中“自我实现检查日志”仍作为大块运行诊断逻辑存在。

## 清理原则

```text
1. 旧逻辑可以直接删除，但每一步必须保证能编译。
2. 自我安全值 / 服务值只允许通过价值结算模块落账。
3. 任务树、需求树、方法树的显示只呈现自我节点中的对应根链信息；治理服务类节点不混入普通业务树。
4. 任务结果优先落到结果状态、任务结果消息、因果或运行事实，不再依赖任务结果节点作为正式结构。
5. 控制面板中已经废止或未实现的专项命令不保留假入口。
6. 诊断逻辑如果保留，只能作为显式自检命令路径，不进入普通启动主链。
```
