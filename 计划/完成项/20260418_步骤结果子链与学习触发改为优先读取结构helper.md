# 步骤结果子链与学习触发改为优先读取结构helper

完成时间：2026-04-18

## 本步目标

在上一轮已经把以下 4 个判据前移成正式结构字段之后：

- `当前步骤可复用`
- `最近结果命中当前治理`
- `当前动作长时间无推进`
- `当前动作需重试`

继续把主链里仍直接读取治理证据的地方，收成统一 helper，让：

- `步骤结果子链`
- `等待判定`
- `稳定等待态复盘学习`

都优先读取 `结构字段 + 上下文 + 治理证据` 的统一收敛值，而不是各自直接翻同一批治理证据字段。

## 本步改动

### 1. 新增结构优先 helper

在 [D:\鱼巢\任务管理任务模块.impl.cpp:2839](D:/鱼巢/任务管理任务模块.impl.cpp:2839) 附近新增：

- `私有_P0_当前步骤可复用(...)`
- `私有_P0_最近结果命中当前治理(...)`
- `私有_P0_当前动作长时间无推进(...)`
- `私有_P0_当前动作需重试(...)`
- `私有_P0_当前需等待(...)`

这些 helper 的口径是：

1. 先读 `当前需求位特征`
2. 再读 `上下文`
3. 最后再读 `当前治理二次特征证据`
4. 对 `当前需等待` 再保留最小枚举/语义兜底

这样后续调用方不再需要自己拼一遍“结构优先、证据兜底”。

### 2. 等待分支改读 helper

[D:\鱼巢\任务管理任务模块.impl.cpp:10212](D:/鱼巢/任务管理任务模块.impl.cpp:10212) 的 `私有_可进入等待分支(...)`，已经改成直接使用：

- `私有_P0_当前需等待(结果)`

不再直接依赖 `当前治理二次特征证据.当前需等待`。

### 3. 结果命中当前治理改读 helper

[D:\鱼巢\任务管理任务模块.impl.cpp:10607](D:/鱼巢/任务管理任务模块.impl.cpp:10607) 的 `私有_结果节点匹配当前治理(...)`，已经改成使用：

- `私有_P0_最近结果命中当前治理(结果)`

不再直接读取治理证据字段。

### 4. 稳定等待态复盘学习改读 helper

[D:\鱼巢\任务管理任务模块.impl.cpp:11376](D:/鱼巢/任务管理任务模块.impl.cpp:11376) 的 `私有_稳定等待态复盘学习(...)`，现在：

- 用 `私有_P0_当前需等待(结果)` 判断等待
- 用 `私有_P0_当前动作长时间无推进(结果)` 排除停滞态
- 用 `私有_P0_当前动作需重试(结果)` 排除重试态

这让“广泛学习”更像稳定等待下的学习承接，而不是把停滞/失败重试也混进同一条触发口。

### 5. 步骤结果子链改读 helper

[D:\鱼巢\任务管理任务模块.impl.cpp:12848](D:/鱼巢/任务管理任务模块.impl.cpp:12848) 的 `私有_推进步骤结果子链(...)`，已把：

- `最近结果命中当前治理`
- `当前步骤可复用`

改成统一读取：

- `私有_P0_最近结果命中当前治理(结果)`
- `私有_P0_当前步骤可复用(结果)`

这样步骤结果子链不再直接碰 `当前治理二次特征证据` 的同名字段。

## 结果

这一轮完成后，`任务管理任务模块` 里这条链进一步收口成：

`结构字段 -> helper -> 编排/学习触发/步骤结果子链`

而不是：

`治理证据字段 -> 各调用方各自直接读取`

好处是：

- 结构字段优先的口径现在集中在一处
- 后面继续削治理证据生成层时，不需要同时改很多调用点
- `步骤结果子链` 和 `学习触发` 已经更接近“只做编排/选择”，而不是“自己解释证据”

## 仍保留的边界
