# 因果补完 C0a 学习映射候选诊断计划 v0.1

日期：2026-05-10

前置计划：

```text
20260510_因果补完C0学习需求映射前置复核计划_v0.1.md
```

## 1. 目标

在不生成学习需求的前提下，把 B0/B1 的因果缺口映射为学习候选类型，供控制面板观察。

```text
因果缺口
→ 学习候选类型
→ 控制面板诊断
```

## 2. 边界

允许：

```text
1. 自我线程诊断快照记录最近学习候选类型。
2. 控制面板展示最近学习候选类型。
```

禁止：

```text
1. 创建学习需求。
2. 修改需求树。
3. 生成探索需求。
4. 绕过学习需求原子状态词典。
```

## 3. 第一版映射

```text
缺资源 -> 学习条件
缺确认 -> 学习状态
缺信息 -> 学习对象/状态
方法结构 -> 学习方法
风险处置 -> 学习风险
异常恢复 -> 学习因果
补偿动作 -> 学习方法
缺条件 -> 学习条件
执行缺失 -> 学习方法
因果样本不完整 -> 学习因果
无缺口 -> 无
```

## 4. 验收

```text
1. Debug x64 构建通过。
2. --self-loop-summary 展示“学习候选”。
3. 无学习需求入树行为。
```
