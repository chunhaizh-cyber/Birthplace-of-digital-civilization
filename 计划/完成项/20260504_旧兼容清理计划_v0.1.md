# 旧兼容清理计划 v0.1

## 1. 目标

本计划不以立刻删除旧逻辑为目标，而是把旧链路标记、隔离，并逐步退出主链。

最终目标：

```text
自我线程只保留最小状态快照。
控制面板只读摘要线程快照。
任务长期保存选中自我能力值，不把方法首节点作为权威字段。
学习只作为缺口需求可选中的尝试学习能力。
```

## 2. 清理对象

```text
旧任务管理任务中心化入口
旧学习任务特殊链路
旧控制面板从自我线程读取大摘要接口
旧方法头预绑定逻辑
旧候选方法首节点长期保存逻辑
旧字符串语义字段作为权威状态的使用
```

## 3. 迁移原则

```text
先标记，后隔离，再退出主链。
有替代路径前不删除旧入口。
旧字段只能作为兼容镜像或运行期缓存。
长期语义必须迁移到抽象特征和实例特征槽。
方法首节点只能作为运行期解析结果，权威选择字段是选中自我能力值。
```

## 4. 子计划

### P0 旧入口盘点

盘点并建立清单：

```text
任务管理任务中心化入口
学习任务特殊判断
学习子任务创建
控制面板读取自我线程大摘要接口
需求方向方法头预创建函数
任务主信息中的候选方法首节点 / 可用方法首节点 / 执行方法
```

验收：

```text
每个旧入口都有保留、隔离、替换或删除结论。
每个仍保留的旧入口都有兼容说明和退出条件。
```

### P1 控制面板旧读取接口降级

目标：

```text
控制面板展示只读取控制面板摘要线程快照。
自我线程的大摘要读取接口只作为旧兼容或调试入口。
```

验收：

```text
控制面板主展示路径不再触发自我线程深扫树。
旧读取接口命名或注释明确其兼容身份。
```

### P2 任务管理中心入口降级

目标：

```text
任务管理任务不再作为需求树上方的中心管理者。
任务管理工作线程只推进一个需求节点对应任务的一步。
```

验收：

```text
任务创建只建立需求和任务的对应关系。
任务筹办由查找方法获得选中自我能力值。
任务执行由选中自我能力值解析方法首节点。
```

### P3 学习特殊链路迁移

目标：

```text
旧学习任务特殊分支迁移为缺口需求链路。
尝试学习作为自我能力参与查找方法和任务执行。
```

验收：

```text
代码中不再以学习任务作为独立调度系统的权威分类。
方法结构缺口、条件结果对缺口、证据不足缺口可以生成需求节点。
这些缺口需求可通过查找方法选中尝试学习能力。
```

### P4 方法指针缓存清理

目标：

```text
任务长期语义保存选中自我能力值。
方法首节点、方法虚拟存在只作为运行期缓存。
```

验收：

```text
执行前会用选中自我能力值重新解析或校验缓存方法首节点。
缓存不匹配时丢弃缓存并重新解析。
旧候选方法首节点字段不再作为权威执行依据。
```

### P5 字符串语义降级

目标：

```text
缺口类型、任务阶段、需求状态、方向、功能域等长期语义迁移到特征槽。
字符串字段只保留为显示摘要、日志或兼容镜像。
```

验收：

```text
新增长期语义必须先有抽象特征槽。
旧字符串语义字段不再扩散到新接口。
```

## 5. 完成口径

```text
旧入口仍可兼容运行，但不再是主链。
新主链由需求树、任务两阶段、自我能力值、本能函数闭环承担。
控制面板和自检不再增加自我线程负担。
学习变成缺口需求的可选能力，不再是独立任务系统。
```
