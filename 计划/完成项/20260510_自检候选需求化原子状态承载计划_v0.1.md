# 自检报告原子状态承载与自我线程需求化边界计划 v0.2

日期：2026-05-10

## 1. 背景

自检线程可以发现结构缺口和镜像不一致等问题，并通过自检缺口候选消息上报。自检线程只负责发现和报送，不生成修复需求。

本轮动态验证发现：

```text
可用方法数量镜像不一致
当前=0
统计=32
```

这类候选只有报告语义，没有完整原子需求语义。如果直接转成任务管理派生需求，会缺少：

```text
目标主体
当前状态
目标状态
满足关系
```

自我线程现在会拒绝缺少当前状态或目标状态的执行缺失原子需求入树。这是正确的，不能用默认状态或泛化对象兜底。

## 2. 最新规则

```text
自检报告进入自我线程后分两类处理：

1. 报告型候选
   只进入日志、控制面板、自检报告。
   不进入需求树。

2. 自我线程决策需求型候选
   必须携带完整原子目标。
   只能由自我线程在允许时机内转成正式需求。
```

禁止：

```text
1. 自检候选缺少当前状态或目标状态时仍强行入树。
2. 用需求树结构根作为派生需求父需求。
3. 用泛化默认状态掩盖具体缺口。
4. 把统计镜像不一致直接包装成补偿动作需求。
```

允许：

```text
1. 报告型候选继续写日志和摘要。
2. 自我线程可在明确目标状态和修复时机后进入需求树。
3. 无法需求化的候选后续由诊断界面或自检报告展示。
```

## 3. 目标

让自检报告需求化具备硬边界：

```text
能需求化的候选，必须由自我线程决策，且原子目标完整；
不能需求化的候选，只能作为报告，不允许伪造成需求。
自检线程不生成修复需求。
```

## 4. 分步计划

### P0：候选分类口径固定

状态：已开始。

动作：

```text
1. 将“可用方法数量镜像不一致”改成报告型候选。
2. 报告型候选不增加已提交需求计数。
3. 自检日志显示“提交报告”，不再显示“提交需求”。
```

验收：

```text
--self-check-log 不再因该候选生成执行缺失原子需求而报错。
```

### P1：扩展自检候选消息

状态：已完成第一版。

动作：

```text
给自检候选消息补可选原子目标字段：
1. 目标主体指针 / 主键。
2. 当前状态指针 / 主键。
3. 目标状态指针 / 主键。
4. 满足关系。
5. 目标特征类型。
```

要求：

```text
字段是需求型候选的必填项；
报告型候选可以为空。
```

### P2：自我线程需求化闸门

状态：已修订。

动作：

```text
1. 自我线程收到自检候选时先判断候选类型。
2. 报告型候选只落日志和报告。
3. 若自检线程越界提交需求型 / 建议生成需求候选，自我线程记录错误日志并拒绝入树。
4. 后续若需要需求化，必须由自我线程自己的修复决策流程重新构造需求。
5. 修复决策还必须检查运行期 / 休眠期门控。
```

### P3：首批自我线程修复决策候选

状态：后续另起自我线程侧计划。

优先支持：

```text
1. 当前自我核心状态缺失。
2. 当前自我内部世界缺失。
3. 需求根 / 任务根 / 方法根请求创建失败。
4. 任务建立完成后任务虚拟存在仍未绑定。
```

这些候选有明确目标：

```text
某结构存在状态 = 已建立
某核心状态可读写状态 = 已就绪
```

暂缓原因：

```text
当前状态类已有创建状态能力，但首批需求型自检候选还缺少统一的“结构存在状态 / 核心状态可读写状态”特征词和标准目标状态词。
在这些原子状态词典明确前，不允许用泛化默认状态硬造需求。
```

2026-05-10 补充：

```text
标准原子状态词典入口已由
`20260510_标准原子状态词典与自检需求型候选承接计划_v0.1.md`
完成 P0/P1。

但首批候选仍不能直接入树：
1. 当前自我、内部世界缺失属于非正常逻辑错误，不应由自检需求兜底。
2. 需求根 / 任务根 / 方法根 是存在信息结构字段，默认可以为空；
   未使用时为空不是错误，创建失败才是非正常逻辑错误。
3. 任务虚拟存在在建立任务时由程序建立；
   未建立任务时为空不是错误，任务建立后仍未绑定才是任务创建链错误。
4. 任务信息节点不是基础信息节点，不能直接作为状态主体。
5. 核心状态可读写缺失可作为后续试点，但只能由自我线程在明确修复动作边界后决策。
6. 有些缺失只能在自我休眠期修复；运行期只能记录和等待修复窗口。
```
