承接：

## 1. 目标

本计划执行内部循环总计划的第一段：

```text
P0：验收仪表和诊断基线
P1：冷启动验收
P2：需求树生成成立的第一段：根需求压力评估和主需求选择
```

目标是先让系统能回答：

```text
当前自我内部循环处于哪一级？
断点在哪里？
安全压力和服务压力分别是多少？
当前主需求为什么被选中？
```

本阶段不改任务执行闭环、不清理任务虚拟存在边界残留，只补诊断与根需求选择公共口径。

阶段口径修订：

```text
需求树正常生成属于内部循环建立阶段的“需求层闭合”，不是初始化阶段，也不是自我苏醒阶段。
本计划已完成的是 P2 第一段：根需求压力评估和当前主需求选择。
完整 P2 还需要补：根据内部状态 / 任务缺口 / 方法缺口生成子需求、去重合并、管理需求 / 执行需求角色刷新。
```

## 2. 实施范围

### 2.1 自我类

新增：

```text
枚举_自我内部循环完成度
结构_自我根需求压力评估
结构_自我内部循环诊断
自我类::评估根需求压力()
自我类::诊断内部循环()
```

实现要求：

```text
1. 压力评估复用当前安全 / 服务根需求口径。
2. 诊断可映射冷启动完成为 1 级。
3. 当前主需求有效或可由压力评估推出时，只能达到 P2 第一段。
4. 若要正式判定“需求树生成成立”，还必须具备动态需求生成、子需求组织、去重合并和结构角色刷新验收。
5. 后续 P3-P7 字段先预留，但不强行判满。
```

### 2.2 控制面板

## 3. 分步

### 步骤 1：新增诊断类型

在 `自我类.ixx` 中新增公共类型。

验收：

```text
其他模块可导入自我类后读取诊断结构。
```

### 步骤 2：实现压力评估

在 `自我类.cpp` 中实现：

```text
自我类::评估根需求压力()
```

输出：

```text
安全差值
服务差值
安全压力
服务压力
选中需求
选择原因
```

验收：

```text
安全压力 >= 服务压力 时选安全根需求。
服务压力 > 安全压力 时选服务根需求。
安全 / 服务根需求缺失时按已有根需求退化选择。
```

### 步骤 3：实现内部循环诊断

在 `自我类.cpp` 中实现：

```text
自我类::诊断内部循环()
```

本阶段等级判定：

```text
0 级：冷启动条件不成立。
1 级：冷启动完成。
2 级第一段：根需求有效且能得到当前主需求或压力评估候选。
2 级完整验收：需求树能根据自身状态和内部缺口动态生成 / 更新 / 组织需求，并稳定选出当前主需求。
```

如果系统已经具备任务、任务虚拟存在、结果回写、需求结算，则诊断可以继续给出对应布尔值，但不作为本阶段必过项。

### 步骤 4：控制面板接入

在 `控制面板类.ixx` 增加字段，在 `控制面板类.cpp` 读取诊断并渲染摘要。

验收：

```text
控制面板摘要能显示内部循环完成度、压力和断点。
```

### 步骤 5：构建验证

执行 Debug x64 构建。

验收：

```text
Debug x64 构建通过。
```
