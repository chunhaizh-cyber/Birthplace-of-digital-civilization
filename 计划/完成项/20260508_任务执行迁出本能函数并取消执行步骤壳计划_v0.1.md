# 任务执行迁出本能函数并取消执行步骤壳计划 v0.1

制定日期：2026-05-08

## 1. 背景

任务筹办已经从本能方法迁出为任务系统普通函数。任务执行也应采用同一口径：

```text
任务执行是任务系统普通函数。
任务执行不作为本能方法。
任务执行不需要先创建任务步骤节点再调用。
```

本能函数只代表真实动作能力，例如改变特征值、练习、尝试学习、定向学习等。任务执行只是调度和调用已选方法。

## 2. 目标

1. 删除 `自我_任务执行一步` 本能方法入口。
2. 删除任务执行一步待重入运行码。
3. 工作线程直接调用普通任务执行函数。
4. 筹办阶段只在任务头绑定目标方法和选中自我能力值，不再创建执行步骤节点。
5. 执行桥把传入任务归一到任务头后，直接读取任务头执行方法。
6. 执行结果里的 `当前步骤节点` 只作为兼容字段，填任务头。
7. 文档规范同步：筹办、执行都是任务系统函数，不是本能方法。

## 3. 修改项

### 阶段一：代码删除

- 删除 `枚举_本能方法ID::自我_任务执行一步`。
- 删除 `本能方法结果_任务执行一步已建立方法头待重入`。
- 删除本能方法元信息中的任务执行一步。
- 删除本能动作管理文本映射。
- 删除执行桥中识别任务执行一步本能方法与待重入的逻辑。
- 删除自我初始化模块中注册 `自我_任务执行一步` 的整段本能函数。

### 阶段二：执行路径收口

- `自我初始化模块::筹办任务(...)` 不再创建任务执行步骤节点。
- `自我_执行桥模块::执行任务(...)` 不再优先读取当前步骤，直接以任务头为执行承载。
- 工作线程日志不再打印当前步骤作为执行入口，只打印任务头和任务头绑定方法。

### 阶段三：文档和验收

- 更新 README、方法规范、本能方法说明、任务类说明。
- 扫描活代码中 `自我_任务执行一步 / 任务执行一步 / 待重入` 残留。
- Debug x64 构建通过。
- `鱼巢.exe --summary` 通过。

## 4. 验收标准

1. 活代码不再出现 `枚举_本能方法ID::自我_任务执行一步`。
2. 本能方法元信息不再注册任务执行一步。
3. 任务筹办不再创建执行步骤节点。
4. 工作线程仍能推进任务，并通过普通执行函数调用目标方法。
5. Debug x64 构建通过。
6. `鱼巢.exe --summary` 返回 `EXIT_CODE=0`。
