## 5. P6 改造工作线程过程方法调用

P6 是本子计划的真正主任务。

目标：

```text
工作线程主链不再直接调用任务模块::筹办::筹办任务。
工作线程主链不再直接调用执行任务。
工作线程主链不再直接调用执行桥。
工作线程主链改为调用自我_任务筹办 / 自我_任务执行。
```

### P6.0 建立工作线程调用索引

命令：

```powershell
rg -n "任务模块::筹办::筹办任务|执行任务|执行桥|自我_任务筹办|自我_任务执行" -S .
rg -n "任务类::写入任务状态|任务类::应用任务状态|任务类::写入任务当前阶段" -S .
```

产出：

```text
P6.0.output:
1. 工作线程主链文件。
2. 只筹办模式分支。
3. 只执行模式分支。
4. 普通筹办后执行模式，如果存在。
5. 任务模块::筹办::筹办任务 的调用点。
6. 执行任务 的调用点。
7. 执行桥调用点。
8. 工作线程可达的任务状态裸写点。
```

暂停条件：

```text
C-P6.0：
如果无法明确哪个函数是工作线程主链入口，暂停。
不要靠函数名猜主链。
```

### P6.1 固定主链允许 / 禁止关系

允许：

```text
工作线程主链
    -> 自我_任务筹办
        -> 任务模块::筹办::筹办任务

工作线程主链
    -> 自我_任务执行
        -> 执行任务 或底层执行核心
```

不允许：

```text
工作线程主链 -> 任务模块::筹办::筹办任务
工作线程主链 -> 执行任务
工作线程主链 -> 执行桥
工作线程主链 -> 任务类::写入任务状态
工作线程主链 -> 任务类::应用任务状态
工作线程主链 -> 任务类::写入任务当前阶段
```

### P6.2 改造只筹办模式

目标：

```text
只筹办模式：
    构造任务筹办输入参数场景
    -> 调用 自我_任务筹办
    -> 读取运行存在生成工作结果
```

验收：

```text
P6.2.accept.1：
只筹办模式分支不再直接调用任务模块::筹办::筹办任务。

P6.2.accept.2：
工作结果来自自我_任务筹办生成的运行存在 / 回执。

P6.2.accept.3：
只筹办模式不直接写任务状态，不直接写任务虚拟存在最终事实。
```

暂停条件：

```text
C-P6.2：
自我_任务筹办不能暴露运行存在，或不能提供生成工作结果所需字段。
```

### P6.3 改造只执行模式

目标：

```text
只执行模式：
    构造任务执行输入参数场景
    -> 调用 自我_任务执行
    -> 读取运行存在和输出结果场景生成工作结果
```

验收：

```text
P6.3.accept.1：
只执行模式分支不再直接调用执行任务。

P6.3.accept.2：
只执行模式分支不再直接调用执行桥。

P6.3.accept.3：
只执行模式分支不再直接触发任务类::写入任务状态。

P6.3.accept.4：
工作结果由自我_任务执行的运行存在和输出结果场景生成。
```

暂停条件：

```text
C-P6.3：
自我_任务执行无法提供运行存在、输出结果场景、目标是否达成或执行缺口。
```

### P6.4 处理普通筹办后执行模式

```text
    -> 自我_任务执行
```

```text

    执行任务
```

## 6. P7 隔离旧执行桥任务状态写入

P7 是 P6 后的必要保险丝。

目标：

```text
旧执行桥不得绕过提交任务状态变化本能方法。
```

### P7.1 定位旧桥状态写入

命令：

```powershell
rg -n "任务类::写入任务状态|任务类::应用任务状态|任务类::写入任务当前阶段" -S 任务模块.执行.ixx
rg -n "执行桥|执行任务" -S 任务模块.执行.ixx
```

产出：

```text
P7.1.output:
1. 旧执行桥中所有任务状态写入点。
2. 哪些写入来自执行成功。
3. 哪些写入来自执行失败。
4. 哪些写入来自等待 / 缺口 / 中断。
5. 哪些写入同时生成动作动态。
```

### P7.2 选择隔离策略

优先级：

```text
策略 A：
    把旧执行桥内状态写入迁出，改为生成执行回执字段。

策略 B：
    给旧执行桥增加“禁止写任务状态”标志。
    工作线程 / 自我_任务执行 调用时强制传入 禁止写任务状态 = true。

策略 C：
    自我_任务执行 不再调用旧执行桥，直接调用底层执行核心。
```

禁止：

```text
旧执行桥继续写任务状态，同时判定 P7 完成。
```

### P7.3 旧桥回执化

旧桥原本写状态的位置，改为写运行存在回执字段：

```text
建议任务状态
建议任务阶段
目标是否达成
执行缺口类型
等待原因
派生需求提示
输出结果场景
```

后续由：

```text
同步执行回执到任务
-> 提交任务状态变化
```

完成提交。

### P7.4 P7 验收

命令：

```powershell
rg -n "任务类::写入任务状态|任务类::应用任务状态|任务类::写入任务当前阶段" -S 任务模块.执行.ixx
rg -n "执行桥" -S .
```

通过条件：

```text
1. 任务主链不会触发任务模块.执行.ixx 中的任务状态直写。
2. 旧执行桥如果仍存在，只能生成执行回执，不写任务状态。
3. 旧桥产生的状态建议必须经提交任务状态变化提交。
```
