# 因果补完 C3 学习需求更新指令预览计划 v0.1

日期：2026-05-10

前置计划：

```text
20260510_因果补完C2b学习目标状态节点夹具验收计划_v0.1.md
```

## 1. 目标

在不执行需求树更新的前提下，把学习目标状态节点、上级需求、需求主体、需求场景和学习需求类型组织成 `结构_需求树更新指令` 预览，确认后续正式入树所需字段齐备。

## 2. 边界

```text
1. 只生成预览。
2. 不调用 `需求类::执行更新指令`。
3. 不创建学习需求。
4. 不修改需求树。
5. 不生成探索需求。
6. 不恢复旧学习主需求链。
7. 预览字段缺失必须记录错误日志并返回非 0。
```

## 3. 实现

新增控制面板命令：

```text
--causal-learning-demand-preview-summary
--learning-demand-preview-summary
```

预览字段：

```text
指令类型 = 新增需求
父需求 = 服务根需求
需求主体 = 当前自我存在
需求场景 = 自我内部世界
被需求存在 = 学习目标主体
需求类型 = 学习因果需求
当前状态 = 学习目标当前状态节点
目标状态 = 学习目标目标状态节点
满足关系 = 等于
```

## 4. 验收

```text
1. Debug x64 构建通过。
2. `--causal-learning-demand-preview-summary` 0 码返回。
3. 输出中预览完整=是。
4. 输出中当前状态、目标状态、父需求、需求主体、需求场景、被需求存在均非空。
5. 输出中仍显示：执行=否，入树=否，修改需求树=否，生成探索需求=否。
6. 闸门预检通过，但不执行更新指令。
```
