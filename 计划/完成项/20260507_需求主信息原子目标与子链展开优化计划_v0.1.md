# 需求主信息原子目标与子链展开优化计划 v0.1

制定日期：2026-05-07

## 1. 背景

`需求主信息类` 目前已经较瘦，核心字段是：

```text
需求主体
需求场景
被需求存在
被需求当前状态
需求状态
对应任务
任务列表
权重与结算账本
派生归因字段
```

这与“需求是某个对象从当前状态到目标状态”的目标关系基本一致。

新的优化重点不是加入目标值类型，也不是加入持久化目标包，而是把口径进一步收紧：

```text
单个执行需求节点 = 单个目标特征 + 单个方向 / 满足关系
多目标需求 = 管理需求 + 直接子需求链
```

## 2. 核心原则

1. 需求类不保存值类型。
2. 需求类不判断集合、区间、枚举、二次特征。
3. 需求类只保存状态引用、目标关系和账本信息。
4. 具体怎么比较，由特征类 / 状态比较服务完成。
5. 需求方向只保留三向关系：小于、等于、大于。
6. 保持等于 `等于`。
7. 改变等于 `小于 | 大于`。
8. 存在、可用、完整、已形成是状态或特征值，不是方向。
9. 区间由特征比较服务解释，不进入需求字段。
10. 目标包不作为需求主信息持久字段。

## 3. 不做的事

本计划明确不做：

```text
新增 结构_需求目标包 作为 需求主信息类 字段
新增 I64 / VecIU64 / 枚举 / 布尔 / 指针等目标值字段
在需求类中实现集合比较
在需求类中实现区间比较
在需求类中解释枚举值
在需求类中解释二次特征
用派生归因决定需求身份
用缺口签名替代目标重叠去重
```

## 4. 目标结构

### 4.1 三向关系

在合适的公共头或 `需求主信息类.h` 中新增：

```cpp
enum class 枚举_三向关系 : std::int8_t {
    小于 = -1,
    等于 = 0,
    大于 = 1,
};

using 三向关系掩码 = std::uint8_t;

inline constexpr 三向关系掩码 关系_小于 = 0b001;
inline constexpr 三向关系掩码 关系_等于 = 0b010;
inline constexpr 三向关系掩码 关系_大于 = 0b100;
inline constexpr 三向关系掩码 关系_保持 = 关系_等于;
inline constexpr 三向关系掩码 关系_改变 = 关系_小于 | 关系_大于;
inline constexpr 三向关系掩码 关系_大于等于 = 关系_大于 | 关系_等于;
inline constexpr 三向关系掩码 关系_小于等于 = 关系_小于 | 关系_等于;
```

### 4.2 单目标面

不引入目标包，只新增单目标关系字段：

```cpp
struct 结构_需求目标面 {
    可解析引用<存在节点类> 被需求存在{};
    可解析引用<状态节点类> 被需求当前状态{};
    可解析引用<状态节点类> 需求状态{};
    三向关系掩码 满足关系 = 关系_等于;

    const 词性节点类* 目标特征类型缓存 = nullptr;
};
```

第一阶段可直接把当前旧字段视为目标面来源：

```text
被需求存在
被需求当前状态
需求状态
```

只补：

```text
满足关系
目标特征类型缓存
```

### 4.3 分面整理

后续分面可以逐步整理为：

```text
基础面：
    描述信息、类型、统计、结构角色

归属面：
    需求主体、需求场景

目标面：
    被需求存在、被需求当前状态、需求状态、满足关系、目标特征类型缓存

任务承接面：
    对应任务、任务列表

生命周期面：
    需求有效截止、最近评估时间、等待承接/等待回流状态

价值账本面：
    安全权重、服务权重、累计安全结算、累计服务结算、最近结算任务、最近结算时间

派生归因面：
    来源任务、来源方法、派生类型、重复触发统计、观察显示字段
```

但第一阶段不需要一次性把类改成多层嵌套结构，避免大面积破坏调用点。

## 5. 子链展开规则

多目标需求不写目标包，而是创建管理需求和子需求。

例：

```text
方法补齐管理需求
├── 候选方法存在状态需求
├── 方法动作骨架状态需求
├── 方法条件结果配对状态需求
├── 方法运行账完整性需求
└── 方法是否可用需求
```

父需求：

```text
结构角色 = 管理需求
负责组织、等待和汇总
不需要持久化目标包
```

子需求：

```text
结构角色 = 执行需求
每个节点只表达一个目标特征和方向关系
```

## 6. 子链去重规则

添加子需求时，按同父直接子链做目标重叠判断。

目标键：

```cpp
struct 结构_需求目标键 {
    std::string 目标主体主键{};
    std::string 目标特征类型主键{};
    三向关系掩码 方向掩码 = 0;
};
```

去重逻辑：

```text
同父
目标主体相同
目标特征类型相同
方向掩码有交集
    => 复用或更新已有子需求
否则
    => 创建新子需求
```

缺口签名和派生归因只用于说明来源、观察和重复触发统计，不作为需求身份。

## 7. 特征比较服务

新增或明确状态比较服务：

```cpp
struct 结构_特征比较结果 {
    bool 可比较 = false;
    枚举_三向关系 关系 = 枚举_三向关系::等于;
    std::string 说明{};
};

struct 特征比较服务 {
    static 结构_特征比较结果 比较状态(
        const 状态节点类* 当前状态,
        const 状态节点类* 目标状态
    ) noexcept;
};
```

需求满足判断只做：

```text
调用状态比较服务
读取三向关系
判断三向关系是否被满足关系掩码接受
```

需求类不解释状态值。

## 8. 方法查找对接

任务筹办处理执行需求时：

```text
读取执行需求的目标状态关系
调用特征比较服务得到三向关系
如果已满足，则回传满足
如果未满足，则生成方法查找请求
```

任务筹办处理管理需求时：

```text
收集直接子需求作为运行时目标视图
推进未满足子需求
汇总必需子需求状态
```

方法查找输入包可以存在，但权威输入应来自当前要处理的执行需求。管理需求的直接子需求目标视图只作为调度、汇总和选择下一批执行需求的运行时视图，不直接作为单次方法查找目标包。

## 9. 派生归因注释修订

修改 `需求主信息类.h` 中派生归因注释：

旧口径：

```cpp
// 派生需求归因：用于缺口需求去重、自检归因和控制面板观察。
```

新口径：

```cpp
// 派生需求归因：用于记录来源、自检归因、控制面板观察和重复触发统计。
// 需求去重不使用派生归因。
// 需求去重以同父子链上的目标特征类型和方向重叠为准。
```

## 10. 执行阶段

### 阶段一：规范和注释落地

1. 新增需求主信息规范。
2. 更新 README 近期语义基线。
3. 修订 `需求主信息类.h` 注释。
4. 不新增目标包字段。

### 阶段二：三向关系和单目标面

1. 新增三向关系枚举和掩码。
2. 新增 `满足关系` 字段，默认 `关系_等于`。
3. 可选新增 `目标特征类型缓存`。
4. 保持旧 `被需求存在 / 被需求当前状态 / 需求状态` 调用兼容。

### 阶段三：运行时目标视图

1. 新增 `生成需求目标键`。
2. 新增 `收集直接子需求目标视图`。
3. 新增 `获取或创建直接子需求_按目标重叠`。
4. 把派生需求去重从归因字段迁到目标重叠判断。

### 阶段四：状态比较服务

1. 新增或明确 `特征比较服务::比较状态`。
2. 将需求满足判断改为三向关系掩码。
3. 删除需求侧 I64 差值判断或迁到比较服务。

### 阶段五：执行缺失原子需求展开

1. 执行函数发现实际缺失后输出对应原子执行需求。
2. 候选方法存在、动作骨架、条件结果配对、运行账完整性、方法可用性等目标按实际缺失生成子执行需求。
3. 子需求按目标重叠去重。
4. 父需求如存在，只按必需子需求满足状态汇总，不直接进入方法查找。

### 阶段六：文档与验收

1. 更新任务类说明中“需求状态/当前状态”的描述。
2. 更新控制面板需求展示说明。
3. 确认 README 不再把目标包作为持久结构。

## 11. 验收标准

1. `需求主信息类` 没有持久化目标包字段。
2. `需求主信息类` 没有新增目标值类型字段。
3. 单个执行需求只表达一个目标状态关系。
4. 多目标需求通过管理需求子链表达。
5. 子需求去重使用目标主体、目标特征类型、方向重叠。
6. 派生归因不再作为需求身份。
7. 需求满足判断只接收三向比较结果。
8. Debug x64 构建通过。
9. `鱼巢.exe --summary` 通过。
