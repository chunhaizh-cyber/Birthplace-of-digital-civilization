# 因果补完 C2 学习目标状态节点构造计划 v0.1

日期：2026-05-10

前置计划：

```text
20260510_因果补完C1学习需求入树前置闸门计划_v0.1.md
```

## 1. 目标

把 C0c 的学习目标草案中的 I64 当前值 / 目标值构造成真实 `状态节点类`，让后续学习需求入树闸门可以使用真实当前状态和目标状态。

## 2. 边界

```text
1. 允许在自我内部世界创建学习目标当前状态节点和目标状态节点。
2. 不创建学习需求。
3. 不修改需求树。
4. 不生成探索需求。
5. 不把 I64 当前值 / 目标值直接伪装成需求更新指令的状态字段。
6. 状态节点创建缺主体、缺特征、缺内部世界时必须记录错误日志并拒绝创建。
```

## 3. 实现方式

新增自我类基础函数：

```cpp
状态节点类* 创建学习I64原子状态(
    场景节点类* 场景,
    基础信息节点类* 主体,
    const 语素入口节点类* 特征类型,
    I64 状态值,
    时间戳 now) noexcept;
```

使用位置：

```text
自我线程因果补完诊断生成学习目标草案后；
若草案完整，则在自我内部世界创建当前状态节点和目标状态节点；
把两个状态节点主键写入因果补完诊断快照。
```

## 4. 验收

```text
1. Debug x64 构建通过。
2. `--causal-learning-demand-gate-summary` 0 码返回。
3. 默认无学习候选时不创建状态节点，闸门仍拒绝入树。
4. 有完整学习目标时，闸门能读取当前状态节点和目标状态节点主键。
5. 全流程仍不创建学习需求、不修改需求树、不生成探索需求。
```
