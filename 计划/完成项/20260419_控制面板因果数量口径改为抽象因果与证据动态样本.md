# 2026-04-19 控制面板因果数量口径改为抽象因果与证据动态样本

## 本次目的

修正控制面板把 `抽象因果模板数` 与 `证据动态样本数` 并排显示成 `因果模板数 / 因果实例数` 的误导口径，避免把两种不同层级的数据误读成“抽象因果和实例因果数量总是一样”。

## 本次改动

1. 在 `结构_控制面板快照` 中新增 `因果证据动态样本数`。
2. `读取控制面板快照(...)` 中：
   - `因果模板数` 继续表示抽象因果模板数量。
   - `因果实例数` 改为读取真实实例节点数量。
   - `因果证据动态样本数` 改为统计所有抽象因果下挂接的证据动态样本数量。
3. 控制面板页面、概览、菜单、卡片、终端摘要统一改成：
   - `抽象因果`
   - `证据动态样本`
4. 因果列表预览也改成 `因果证据动态预览 / 因果证据动态列表`。

## 结果

控制面板现在不会再把“模板数量”和“证据样本数量”误写成“模板数量”和“实例数量”。  
如果后续仍看到抽象因果和证据动态样本数量接近或相等，那就应该继续追查抽象因果归并率，而不是先怀疑控制面板统计字段写错。
