# 2026-04-27 方法执行前置失败改为结构化缺口

## 背景

方法真正执行前的失败不应生成方法条件/结果样本，但必须作为任务筹办或任务执行阶段发现的需求缺口返回。

## 已完成

- `方法不可用` 继续返回 `方法是否可用: 0 -> 1`，由任务管理任务组织学习。
- `没有条件场景` 继续返回 `方法条件节点数量: 当前值 -> 1`，由自我线程按缺条件需求处理。
- `句柄不支持` 不再返回普通 `-2` 错误码，改为返回 `方法可执行入口数量: 0 -> 1`。
- `方法可执行入口数量` 使用 `本能方法结果_任务缺方法需任务管理任务` 上报，使工作线程能转成派生需求消息。
- 这些前置缺口不会触发方法执行样本沉淀；只有已经进入本能函数调用的真实方法执行才创建条件/结果节点。

## 验证

```text
msbuild 鱼巢.vcxproj /p:Configuration=Debug /p:Platform=x64 /m
结果：0 警告，0 错误
```
