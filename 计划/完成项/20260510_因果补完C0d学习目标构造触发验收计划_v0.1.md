## 1. 目标

补一个可重复触发的诊断验收入口，验证学习候选存在时，学习目标草案能完整构造。

## 2. 边界

```text
1. 不创建学习需求。
2. 不修改需求树。
3. 不生成探索需求。
4. 不伪造任务执行结果进入治理主链。
5. 只验证候选类型 -> 目标草案 的纯映射和展示。
```

## 3. 待实现方式

优先选择：

```text
新增控制面板命令：
    --causal-learning-target-summary
```

输出内容：

```text

当前值
目标值
满足关系

是否需要任务虚拟存在
是否需要方法虚拟存在
```

该命令只做映射表验收，不启动正式学习需求入树。

## 4. 验收

```text

4. 输出明确学习需求仍未入树。
```
