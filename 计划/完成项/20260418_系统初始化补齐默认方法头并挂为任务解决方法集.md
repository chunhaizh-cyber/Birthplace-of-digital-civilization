# 20260418 系统初始化补齐默认方法头并挂为任务解决方法集

## 本步目标

落实这条口径：

- 对自我来说，先有可调用的方法，比方法可靠度更重要
- 可靠度可以后续继续验证
- 所有本能函数，以及当前已映射到本能函数 ID 的最小管理原语，应在系统初始化时就出生正式方法头
- 这些方法头应进入任务解决方法集，作为根任务与治理任务的初始可用方法集合

## 本次实现

### 1. 保留现有方法头出生机制，不另起新树

没有重写方法树出生逻辑，继续复用：

- `本能方法类::注册默认自我本能方法集(...)`
- `方法类::查找或创建_本能方法首节点(...)`

这样能保证：

- 默认本能方法仍按现有正式入口出生
- 条件场景 / 结果初始状态 / 方法虚拟存在 / 来源 / 成熟度默认值继续沿用现有链路

### 2. 在系统初始化阶段补齐任务解决方法集

在 `自我类.初始化.cpp` 中新增最小初始化桥：

- 收集全部默认本能方法头
- 将它们挂到以下任务头的 `可用方法首节点列表`
  - `任务管理任务`
  - `安全根任务`
  - `服务根任务`

这样初始化完成后，自我至少已经拥有一批“可用于解决任务的正式方法头”，不需要等学习或可靠度验证之后才承认“有方法”。

### 3. 当前“元函数”的落账口径

当前仓内还没有单独成体系的“元函数注册表 + 元函数方法头批量出生”合同。

本次先按现有代码口径处理：

- 已纳入默认本能表的管理类最小原语
  - `管理_确保单承载存在`
  - `管理_读取单特征当前值`
  - `管理_写入单特征当前值`
  - `管理_写入单挂点指针`
  - `管理_记录动作开始`
  - `管理_记录动作结束`

这些当前都已经通过 `枚举_本能方法ID` 进入默认本能方法集，所以这一步也一并进入了初始化方法头与任务解决方法集。

也就是说：

- 当前仓里“已正式落账的元函数”，先按“已映射到本能方法 ID 的管理类最小原语”处理
- 若后续出现独立的元函数注册表，再扩这一层初始化桥
