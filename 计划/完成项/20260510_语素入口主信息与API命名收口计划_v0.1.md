# 语素入口主信息与 API 命名收口计划 v0.1

日期：2026-05-10

## 1. 背景

上一轮已经完成：

```text
词性节点类 -> 语素入口节点类
```

并明确：

```text
添加词性词 = 人类词性入口。
添加信息入口词 = 系统语义入口。
```

当前还剩一层旧名：

```text
词性主信息类
确保词性对应基础信息
查找词性模板
查找引用词性节点的基础信息
```

这些名字已经不再准确。节点主信息实际承载的是：

```text
人类词性 + 信息入口类型 + 对应基础信息类型 + 对应基础信息引用
```

因此权威名应收口到：

```text
语素入口主信息类
语素入口对应基础信息
语素入口模板
引用语素入口节点
```

## 2. 命名原则

```text
凡是表达系统语义入口、基础信息模板、落树路由的 API，使用“语素入口 / 信息入口”。
凡是只表达自然语言标注、人类语法词性候选的 API，允许保留“词性”。
```

保留：

```text
添加词性词
查找人类词性入口节点
推断人类词性默认基础信息类型
```

迁移：

```text
词性主信息类 -> 语素入口主信息类
确保词性对应基础信息 -> 确保语素入口对应基础信息
查找词性对应基础信息 -> 查找语素入口对应基础信息
获取词性对应基础信息类型 -> 获取语素入口对应基础信息类型
获取词性信息入口类型 -> 获取语素入口信息入口类型
查找词性模板 -> 查找语素入口模板
确保词性模板 -> 确保语素入口模板
查找引用词性节点的基础信息 -> 查找引用语素入口节点的基础信息
基础信息节点引用词性节点 -> 基础信息节点引用语素入口节点
```

## 3. 兼容边界

第一阶段保留兼容别名：

```cpp
using 词性主信息类 = 语素入口主信息类;
```

旧公开函数不作为推荐入口；本轮直接迁移仓库内调用，不再保留同名旧公开函数。

## 4. 实施步骤

```text
P1. 新增本计划并确认命名边界。
P2. 将 `词性主信息类` 改为 `语素入口主信息类`，保留兼容别名。
P3. 将泛化语义入口 API 从“词性”命名迁到“语素入口”命名。
P4. 保留 `添加词性词` 作为人类词性入口，并避免误改。
P5. 检查本轮迁移目标旧名残留，只允许兼容别名和人类词性边界说明。
P6. Debug x64 非增量构建验证。
P7. 更新计划索引。
```

## 6. 最终口径

```text
语素入口主信息类是系统入口记录；
人类词性只是入口记录中的语言标注；
信息入口类型才决定主信息类型、存储树和默认根。
```
