# 代码旧逻辑残留清理计划 v0.1

## 1. 目标

对照当前代码，继续清理已经退出主链但仍留在接口、协议、语素和诊断中的旧逻辑。

本计划的目标不是按关键字机械删除，而是逐项确认：

```text
1. 已有现行替代路径的旧逻辑，物理删除。
2. 仍被现行路径借用的旧命名，先改名或迁移语义，再删除旧入口。
3. 只是状态迁移含义的“旧状态 / 旧值”不纳入删除范围。
4. 删除后必须保持 Debug x64 可编译。
```

## 2. 当前代码残留清单

### P0 任务两步壳与步骤结果树兼容逻辑

范围：

```text
任务类.h
自我线程模块.impl.cpp
任务模块.管理工作线程.ixx
```

现状：

```text
任务类仍保留创建任务步骤节点、创建任务结果节点等兼容占位接口。
自我线程仍保留历史任务两步壳清理、诊断和当前步骤引用清理。
管理工作线程仍有“兼容待删：工作线程不再把筹办/执行阶段当普通本能方法步骤绑定”的路径。
```

执行要求：

```text
1. 确认当前任务主链是否已经完全由需求对应任务头、任务虚拟存在特征、工作线程消息协议承担。
2. 若结果节点仍用于工作线程回报，先改为正式结果记录口径，不再称为步骤结果树兼容接口。
3. 删除不再创建、不再消费的历史两步壳诊断和清理代码。
4. 删除或收窄任务类中的旧步骤/结果节点占位接口。
```

验收：

```text
rg “兼容待删|历史任务两步壳|创建任务步骤节点|创建任务结果节点” 无主链残留。
任务仍能创建、推进、回报并完成价值反馈结算。
```

### P1 需求任务挂接方式旧协议清理

范围：

```text
需求任务挂接方式.h
任务模块.工作线程消息协议.ixx
鱼巢.cpp
```

现状：

```text
需求主信息类中的旧挂接字段已经清理。
但工作线程消息协议仍包含任务挂接方式、任务挂接方式抽象特征指针和值同步逻辑。
控制面板或终端摘要中仍可能显示兼容回流口径。
```

执行要求：

```text
1. 搜索任务挂接方式的所有读写点。
2. 若只用于展示或兼容摘要，删除字段和展示。
3. 若仍承担需求到任务关系判断，改为读取任务对应需求 / 需求对应任务头 / 任务列表。
4. 物理删除需求任务挂接方式.h。
```

验收：

```text
rg “需求任务挂接方式|任务挂接方式|兼容回流” 无旧协议残留。
需求到任务的挂接关系只由现行结构表达。
```

### P2 派生需求旧命名字段迁移

范围：

```text
需求主信息类.h
自我线程模块.impl.cpp
自检线程模块.impl.cpp
控制面板相关读取代码
```

现状：

```text
需求主信息类中仍有来源方法主键、派生需求类型、方法需求位、本能能力缺口类型、主观察特征等旧命名字段。
这些字段虽然标注为旧占位，但仍用于派生需求去重、自检判断和界面展示。
```

执行要求：

```text
1. 不直接删除这些字段，先梳理每个字段的实际调用用途。
2. 若字段表达的是派生需求生成归因，改名或收口为明确的派生需求归因口径。
3. 若字段只用于显示摘要，迁移为摘要生成逻辑或特征读取。
4. 删除旧注释和旧命名调用点。
```

验收：

```text
需求派生、自检缺口去重和控制面板展示仍正常。
需求主信息类不再出现“旧来源方法主键占位”等旧注释。
```

### P3 需求虚拟存在语素残留清理

范围：

```text
语素类.cpp
可能引用需求虚拟存在的初始化、控制面板、自检代码
```

现状：

```text
语素注册中仍保留“存在_需求虚拟存在”和“需求虚拟存在”。
当前需求运行态已经主要由需求链、任务链、特征和结算字段表达。
```

执行要求：

```text
1. 搜索需求虚拟存在是否仍有运行时读写。
2. 若无运行时依赖，删除语素注册。
3. 若仅为历史显示名称，改为需求链正式名称或特征名称。
```

验收：

```text
rg “需求虚拟存在|存在_需求虚拟存在” 无无效残留。
初始化与控制面板启动正常。
```

### P4 旧安全度兼容读取清理

范围：

```text
场景类.cpp
```

现状：

```text
场景安全读取仍兼容旧“安全度”特征。
当前安全值 / 服务值已经落到自我价值与需求结算路径。
```

执行要求：

```text
1. 确认当前初始化和运行中不再写入旧“安全度”。
2. 删除读取旧“安全度”的 fallback。
3. 若需要数据迁移，先在初始化阶段做一次显式迁移，不在读取路径长期兼容。
```

验收：

```text
rg “安全度” 只剩非旧兼容语义或无残留。
场景安全展示和自我安全值展示不回退旧字段。
```

### P5 动作实现兼容特征类型解析清理

范围：

```text
自我动作实现模块.ixx
```

现状：

```text
动作实现模块中仍有兼容特征类型解析逻辑。
```

执行要求：

```text
1. 确认外部动作输入是否仍传旧格式。
2. 若旧格式已经没有入口，删除兼容解析。
3. 若仍有入口，先把入口转换到正式特征类型，再删除动作实现内部兼容分支。
```

验收：

```text
rg “兼容特征类型|兼容.*特征类型” 无内部兼容解析残留。
本能动作执行和方法动作动态仍正常。
```

### P6 治理协议过渡壳收口

范围：

```text
任务模块.治理协议.ixx
```

现状：

```text
治理协议中仍有 P-1 主体化过渡类型、P0 前向请求壳和兼容回填说明。
部分结构可能已经成为正式协议，但命名和注释仍停留在迁移期。
```

执行要求：

```text
1. 判断每个过渡壳是正式协议还是兼容桥。
2. 已正式化的结构改名、改注释，不再称为过渡。
3. 已不再使用的兼容回填字段删除。
```

验收：

```text
治理协议不再出现无退出条件的“过渡 / 兼容字段最终回填”说明。
任务治理请求和结果摘要仍能编译通过并被现行调用消费。
```

## 3. 非清理对象

以下内容不作为旧逻辑删除目标：

```text
旧状态、旧值、宿主任务旧状态等状态迁移快照字段。
因果类中表示状态迁移或特征变化的“迁移”概念。
临时树作为局部构造和安全转移技术使用时，不因名称包含临时而删除。
```

## 4. 执行顺序

```text
1. P1 需求任务挂接方式旧协议清理。
2. P0 任务两步壳与步骤结果树兼容逻辑。
3. P2 派生需求旧命名字段迁移。
4. P3 需求虚拟存在语素残留清理。
5. P4 旧安全度兼容读取清理。
6. P5 动作实现兼容特征类型解析清理。
7. P6 治理协议过渡壳收口。
```

优先 P1、P0 的原因：

```text
它们最接近任务主链和需求到任务关系。
清理后可以减少后续字段迁移时的歧义。
```

