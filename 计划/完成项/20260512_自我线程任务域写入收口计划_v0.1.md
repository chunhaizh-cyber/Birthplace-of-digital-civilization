# 自我线程任务域写入收口计划 v0.1

状态：已完成  
日期：2026-05-12  
详细设计：`详细设计/自我线程任务域写入收口详细设计20260512.md`

## 1. 目标

按最新权责边界收口 `自我线程模块.impl.cpp`：

```text
自我线程：
    只管需求树、需求估权、任务发起意图、调度意图、需求回写。

任务管理线程：
    负责任务创建、任务虚拟存在、任务状态推进、筹办 / 执行回执同步。
```

计划实施时必须参照详细设计中的函数迁移表和验收规则。

## 2. 非目标

本计划不做：

```text
1. 不重写自我线程主循环骨架。
2. 不改变需求树数据结构。
3. 不改变任务信息节点和任务虚拟存在组合关系。
4. 不改变任务筹办 / 任务执行三通道本能方法边界。
5. 不直接实现需求树真实生长剧本。
```

## 3. 实施顺序

### P0：建立请求协议和投递入口

目标：

```text
让自我线程能表达任务发起、调度、等待、重筹办、权重固化、价值结算等意图。
```

动作：

1. 在任务管理线程协议中补充或扩展：
   - 任务发起请求；
   - 任务调度请求；
   - 任务状态推进请求；
   - 任务权重固化请求；
   - 任务价值结算回执。
2. 若复用现有 `结构_任务治理请求`，必须允许：
   - `任务信息节点 = nullptr`；
   - `来源需求` 有效；
   - `允许创建任务信息节点 = true`。
3. 新增任务管理界面线程投递入口，先只做接收和日志，后续再接创建/状态提交。

验收：

```text
自我线程可以在无任务头时投递任务发起请求。
接口命名不混用“创建任务头”。
```

### P1：迁移按需求创建任务逻辑到任务管理线程

目标：

```text
任务创建、任务状态初始化、任务阶段初始化、任务虚拟存在初始化都归任务管理线程。
```

动作：

1. 将 `私有_按需求创建任务头_任务管理统一挂接` 的任务域写入逻辑迁入任务管理界面线程或任务管理线程。
2. 新函数建议命名：
   - `任务管理线程_按需求创建任务并初始化虚拟存在`。
3. 自我线程保留：
   - `私有_构造任务发起请求_线程侧`。
4. 自我线程不再调用 `任务服务.创建任务头结点`。

验收：

```text
自我线程模块中不再创建任务头。
有任务发起请求时，任务管理线程能创建任务并回执。
```

### P2：拆分需求列表刷新和任务确保

目标：

```text
需求整理阶段只刷新需求列表和主导需求，不提前创建任务。
```

动作：

1. 将 `私有_刷新需求列表并确保需求相关任务_任务管理新链` 改为：
   - `私有_刷新需求列表_线程侧`。
2. 删除其中 `确保相关任务` 的创建逻辑。
3. `步骤_整理需求并做根层重判_` 只选择：
   - 主导需求；
   - 已存在承接任务。
4. 若主导需求没有承接任务，只记录任务发起意图，不创建任务。

验收：

```text
需求整理阶段不再调用按需求创建任务头。
```

### P3：主派发阶段改为请求投递

目标：

```text
主派发阶段不再直接创建任务，只投递任务发起或调度请求。
```

动作：

1. `步骤_生成并执行主派发决议_` 中：
   - 有主导需求、无承接任务：投递任务发起请求；
   - 有主导任务：投递任务调度 / 治理请求。
2. 删除主派发阶段 `私有_按需求创建任务头_任务管理新链` 调用。
3. 保留当前主需求回填。
4. 当前主任务回填只在已有任务或任务管理线程回执后进行。

验收：

```text
主派发阶段不直接创建任务。
无任务时任务管理当前去向显示为“已提交任务发起请求”或等价状态。
```

### P4：派生需求入树只写需求域

目标：

```text
派生需求入树仍由自我线程负责，但派生任务创建和关联任务等待状态交给任务管理线程。
```

动作：

1. `私有_收到派生需求消息时完善需求树` 保留：
   - 解析关联任务；
   - 确保派生需求节点；
   - 刷新结构角色。
2. 删除：
   - 派生需求任务头直接创建；
   - `派生需求任务头->主信息.结果状态信息 = ...`；
   - `任务类::写入任务状态(...等待中...)`。
3. 新增：
   - 派生需求任务发起请求；
   - 关联任务等待状态推进请求。

验收：

```text
派生需求入树后，自我线程日志显示已投递任务请求，不直接写任务等待中。
```

### P5：派生需求解除等待改为重筹办请求

目标：

```text
自我线程只建议父任务重筹办，不直接标记任务待重筹办。
```

动作：

1. 改造 `私有_唤醒派生需求来源任务_任务管理新链`。
2. 删除：
   - `任务类::标记任务待重筹办`；
   - `任务类::写入任务当前阶段(...筹办...)`。
3. 新增：
   - `私有_构造任务重筹办请求_线程侧`；
   - 投递任务状态推进请求。

验收：

```text
自我线程不再直接写待重筹办或任务阶段。
```

### P6：成功结果回流只回写需求域

目标：

```text
任务成功结果回流时，自我线程只更新来源需求和需求结算。
```

动作：

1. 将 `私有_按成功结果满足来源需求并唤醒父任务_任务管理新链` 改名为：
   - `私有_根据任务结果回写来源需求_线程侧`。
2. 保留：
   - 来源需求当前状态回写；
   - 来源需求统计时间；
   - 需求结算。
3. 删除：
   - `任务头->主信息.结果状态信息 = ...`。
4. 父任务唤醒改为 P5 的重筹办请求。

验收：

```text
成功结果回流不再写任务结果状态信息。
```

### P7：任务价值反馈拆账

目标：

```text
自我线程写自我值和需求结算；任务折算值由任务管理线程写。
```

动作：

1. 在 `私有_处理任务价值反馈_线程侧` 中保留：
   - 应用自我安全 / 服务值；
   - 来源需求累计安全 / 服务结算；
   - 最近结算任务主键和时间。
2. 删除或迁移：
   - `任务类::写入任务已折算安全值`；
   - `任务类::写入任务已折算服务值`。
3. 新增任务价值结算回执投递给任务管理线程。

验收：

```text
自我线程不再写任务已折算安全值 / 服务值。
```

### P8：任务承接权重固化迁移

目标：

```text
自我线程只计算建议权重，任务管理线程固化任务权重字段。
```

动作：

1. 改造 `私有_确保任务承接权重已固化_线程侧`。
2. 拆成：
   - `私有_计算任务承接权重建议_线程侧`；
   - `任务管理线程_固化任务承接权重`。
3. 自我线程删除任务权重字段写入。

验收：

```text
自我线程不再写任务承接权重上限、任务评分、权重版本。
```

## 4. 风险与处理

| 风险 | 处理 |
| --- | --- |
| 任务管理线程当前请求协议不支持无任务发起 | P0 先扩展协议，允许来源需求驱动创建任务 |
| 自我线程当前主派发依赖立即得到任务头 | P3 改为“请求已提交”状态，下一轮由任务创建回执回填当前主任务 |
| 派生需求入树后父任务无法立即待重筹办 | P5 使用任务状态推进请求，任务管理线程负责合法迁移 |
| 任务折算值迁出后结算幂等风险 | P7 的任务价值结算回执必须携带任务主键、需求主键和反馈事件主键 |
| 迁移过程中任务管理线程职责膨胀 | 只迁任务域写入，不迁需求树生长和需求估权 |

## 5. 与当前主线顺序

本计划应插入在：

```text
任务筹办 / 任务执行三通道本能方法入口完成
```

之后，以及：

```text
需求树真实生长剧本
内部循环闭合验收
自我苏醒验收
因果补完正式入树
```

之前。

理由：

```text
任务状态写入权不收口，就无法判断后续任务状态推进到底来自任务管理线程还是自我线程。
```

## 6. 完成定义

本计划完成必须同时满足：

```text
1. 自我线程不再直接创建任务头。
2. 自我线程不再直接写任务状态 / 任务阶段。
3. 自我线程不再直接写任务结果状态信息。
4. 自我线程不再直接写任务折算值。
5. 自我线程不再直接固化任务承接权重。
6. 任务管理线程能接收任务发起、调度、状态推进、权重固化和价值结算回执。
7. Debug x64 构建通过。
```
