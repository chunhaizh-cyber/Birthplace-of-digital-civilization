# 已完成功能说明日志

## 事项
- 显式收集同需求轴可比候选并收口升阶触发条件

## 完成时间
- 2026-04-18

## 这一步做了什么
- 扩展了 `结构_原语候选项`，在保留 `节点 / 候选角色 / 候选主键 / 是否同向` 的基础上，补入了 `需求轴键 / 比较特征键 / 比较基准类型 / 结果时间戳 / 结果序号 / 来源锚点类型`，并为方法结果锚点补了 `方法节点` 指针位。
- 把 `任务管理任务模块.impl.cpp` 里的升阶候选收集器改成了显式的“同需求轴候选收集器”，当前只收三类锚点：
  - 步骤结果锚点
  - 宿主最近实际结果锚点
  - 方法结果锚点
- 候选收集时不再只看“有没有节点”，而是同步填充比较尺子：
  - 优先取结果状态对应的特征键作为 `比较特征键`
  - 取不到时再回退到方法结果特征或结果方向二次特征
  - 同时把来源锚点、时间戳、序号写入候选摘要面
- 把 `升阶判定原语` 的口径继续收紧成三条件同时成立才升阶：
  - 同一需求轴
  - 至少两个同向候选
  - 存在可比对
- 同轴但反向的候选不再触发升阶，而是明确交回二次层处理冲突。
- 同向但缺可比基准的候选不再硬判升阶，只会在摘要里写出“缺什么比较基准”。
- 这一步没有直接生成三次特征实体，而是先把高阶排序请求、入选候选、比较基准、排序原因、未决原因收进现有决策摘要链路，保证链路先可观察、可复盘。

## 结果
