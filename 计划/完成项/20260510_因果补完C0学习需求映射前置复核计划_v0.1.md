## 1. 目标

复核 B0/B1 输出的因果缺口是否足够进入 C0：

```text
因果缺口
→ 学习需求类型
→ 学习需求目标
→ 原子目标状态
→ 需求树入树
```

本计划的目标不是直接生成学习需求，而是确认生成学习需求所需的目标主体、目标特征、当前状态、目标状态和满足关系是否齐备。

## 2. 当前代码基础

已具备：

```text
1. 任务派生需求消息携带派生需求类型。
2. 派生需求入树已有原子目标硬闸门。
3. 任务筹办已有方法运行账不足、方法结果不稳定、缺条件结果对等状况。
4. 本能方法中已有“尝试学习”入口。
5. B0/B1 已能在控制面板显示因果候选、缺口和建议学习。
```

## 3. 当前缺口

暂不直接实现学习需求生成，原因：

```text
1. 还没有权威的学习需求目标特征词典。
2. 还没有“学习需求当前状态 / 目标状态”的标准原子状态集合。
3. 还没有缺口类型到学习目标主体的统一规则。
4. 还没有明确哪些学习需求由内部学习承接，哪些等待外部探索前置。
5. 直接用原父需求的当前状态 / 目标状态入树，会再次造成父子需求目标重叠。
```

## 4. 第一版映射候选

只作为诊断候选，不入树：

| 因果缺口 | 学习需求候选 |
| --- | --- |
| 缺资源 | 学习条件 |
| 缺确认 | 学习状态 |
| 缺信息 | 学习对象 / 学习状态 |
| 方法结构 | 学习方法 |
| 风险处置 | 学习风险 |
| 异常恢复 | 学习因果 |
| 补偿动作 | 学习方法 |
| 缺条件 | 学习条件 |
| 执行缺失 | 学习方法 |
| 因果样本不完整 | 学习因果 |

