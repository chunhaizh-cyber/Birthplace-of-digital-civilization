# 20260426 已知对象名称改为通用命名并补齐名称字段

## 背景

当前需求树和控制面板展示中仍能看到较多模块前缀，例如 `任务筹办_目标方法数量`、`任务管理本体步骤_筹办`、`任务虚拟存在_...`、`本能方法_...`。这些前缀属于旧组织方式，不利于后续用通用因果关系组织需求、任务、方法、特征和动态。

## 本轮完成

- 特征创建时同步补齐 `名称=类型`。
- 动态创建时自动补齐 `名称/类型`：
  - 有来源动作时使用动作名。
  - 有动态特征时使用动态特征类型。
  - 否则使用通用名 `动态`。
- 方法虚拟存在、方法条件场景、方法结果场景改为通用名称。
- 本能方法虚拟存在、条件场景、结果场景改为通用名称。
- 任务虚拟存在、任务内部世界、任务步骤、任务结果、任务方法实例场景改为通用名称。
- 任务管理任务本体步骤改为 `筹办`、`执行` 等通用名称和类型，不再生成 `任务管理本体步骤_...`。
- 任务管理任务实例改为通用类型 `任务管理任务`，实例名称使用 `管理自身`、`管理学习`、`管理任务`。
- 任务筹办本能的运行特征改为通用特征名，例如 `目标方法数量`、`本轮动作`，不再生成 `任务筹办_...`。
- 本能动作运行记录特征改为通用特征名，例如 `运行状态`、`运行阶段`、`最近错误码`、`缺方法结束次数`。

## 验证

- 已执行 `MSBuild D:\鱼巢\鱼巢.vcxproj /t:ClCompile /p:Configuration=Debug /p:Platform=x64 /m`。
- 编译结果：0 错误，0 警告。
