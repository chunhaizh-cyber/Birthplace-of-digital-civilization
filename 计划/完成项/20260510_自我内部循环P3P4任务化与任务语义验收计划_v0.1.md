# 自我内部循环 P3-P4 任务化与任务语义验收计划 v0.1

日期：2026-05-10

承接：

```text
计划/20260510_自我内部循环最小生命闭环阶段总计划_v0.1.md
计划/20260510_自我内部循环P0P2诊断与根需求评估计划_v0.1.md
```

## 1. 目标

本计划执行内部循环总计划的第二段：

```text
P3：主需求任务化
P4：任务语义和方法入口
```

目标是让诊断能证明：

```text
当前主需求 D
    ↓
对应任务信息节点 N
    ↓
任务虚拟存在 T
    ↓
方法或本能动作入口
```

本阶段优先做验收和边界收口，不重构任务管理工作线程。

## 2. 实施范围

### 2.1 自我类诊断增强

补充或确认：

```text
当前主需求对应任务是否有效
当前主任务来源需求是否匹配当前主需求
当前主任务是否重复创建风险
当前主任务虚拟存在是否有效
当前主任务虚拟存在概念集是否包含任务概念
当前主任务虚拟存在是否具备任务目标 / 完成条件 / 任务状态等语义槽
当前主方法或默认本能动作是否可调用
```

### 2.2 控制面板展示增强

摘要中补充：

```text
当前主任务主键
当前任务虚拟存在主键
任务化断点
任务语义断点
```

### 2.3 旧边界复查

复查并清理：

```text
自我线程是否仍直接创建任务虚拟存在
自我线程是否绕过任务管理界面线程直连工作线程
任务虚拟存在是否替代任务信息节点
```

发现仍有残留时，按最新边界修订；如果只剩兼容只读解析，记录为已符合。

## 3. 分步

### 步骤 1：代码差距检查

检查以下落点：

```text
自我类.cpp
自我线程模块.impl.cpp
任务模块.管理界面线程.*
任务类.*
控制面板类.*
```

验收：

```text
列出 P3-P4 已满足项和缺口。
```

### 步骤 2：增强内部循环诊断

在 `结构_自我内部循环诊断` 中补充必要字段，或复用已有字段生成更明确摘要。

验收：

```text
诊断能区分：
1. 主需求未任务化；
2. 任务信息节点有效但缺任务虚拟存在；
3. 任务虚拟存在有效但语义槽不足；
4. 方法 / 本能动作入口不足。
```

### 步骤 3：任务化重复风险检查

最小实现：

```text
诊断当前主需求的任务列表数量。
识别当前主需求是否已有未结束任务。
若存在多个未结束任务，输出重复任务风险。
```

验收：

```text
控制面板或诊断摘要能显示重复任务风险。
```

### 步骤 4：控制面板接入

补齐摘要字段。

验收：

```text
`--summary` 能看到当前主任务、任务虚拟存在和 P3-P4 断点。
```

### 步骤 5：构建和记录

执行 Debug x64 构建，运行控制面板摘要。

验收：

```text
Debug x64 构建通过。
控制面板摘要能显示 P3-P4 关键字段。
计划索引更新。
```

## 4. 不做项

```text
不实现动作结果回写。
不实现需求结算闭环。
不实现连续多轮稳定轨迹。
不新增任务虚拟存在类。
不让任务虚拟存在替代任务信息节点。
```

这些由 P5-P8 承接。
