# 学习子任务在补骨架步骤完成后自动生长下一学习步骤

## 本次完成

这次补的是“树的成长”，不是再补一层摘要。

现在学习子任务的补骨架步骤一旦完成，会直接以当前结果节点为父节点，自动接出下一学习步骤，而不是停在原地等待下一轮任务管理重算后再补。

对应修改在：

- `D:\鱼巢\学习执行模块.cpp`

## 生长规则

当前实现走的是“按目标方法还缺什么，立即决定下一步”的最小闭环：

1. 当前补骨架步骤完成
2. 先把当前步骤和当前结果收口为已完成/待适配
3. 读取当前步骤绑定的语义目标方法
4. 按目标方法当前骨架状态推导下一学习步骤
5. 直接在当前结果节点下创建下一步骤和对应结果节点

当前缺口到下一步骤的映射是：

- 没动作骨架：`学习步骤_候选动作生成`
- 没条件侧：`学习步骤_共同条件提取`
- 没结果侧：`学习步骤_结果预测`
- 三者齐备后，再按学习程序裁决进入：
  - `学习步骤_样本收集`
  - `学习步骤_沙箱验证`
  - `学习步骤_可用层进入判断`
  - `学习步骤_优选层进入判断`
  - `学习步骤_边界标记`

## 这次真正改变了什么

以前的行为更像：

`当前步骤完成 -> 学习实体挂起 -> 等任务管理下一轮重算 -> 再补当前步骤`

现在改成了：

`当前步骤完成 -> 当前结果节点下直接长出下一步骤 -> 下一轮只负责复用/继续推进`

也就是说，学习子任务树现在会真的沿着：

`学习子任务头 -> 步骤 -> 结果 -> 下一步骤 -> 下一结果`

这条链往下生长。

## 为什么这一步重要

这解决的是“学习已经知道下一步该干什么，但树上还没有节点”的断层。

在“让方法可用”的路径里，学习最后一步才应该进入尝试学习；
在那之前，应该先把：

- 动作骨架
- 条件侧
- 结果侧

一步一步补出来。

如果树不生长，学习链虽然在账本里推进了阶段，但任务树上仍然只有一个当前步骤，后续步骤无法真正成为可见、可挂接、可继续执行的节点。
