# 20260419 清理因果兼容壳并改为统一主信息加形态过滤

## 本次目标

清理 `因果实例主信息类 / 因果模板主信息类` 这组兼容壳，不再把它们当运行时独立主信息类型继续扩散；统一收口到 `因果主信息类 + 枚举_因果形态`，并把创建、枚举、命名、显示、任务管理写回这些主链全部切到统一口径。

## 本次完成

1. 在 `基础信息主信息类.h` 中移除 `因果实例主信息类` 和 `因果模板主信息类` 的运行时派生类定位，改为兼容别名；运行时真正区分实例/抽象因果的方式改为 `因果主信息类::形态`。
2. `因果类` 与 `世界树类` 的创建接口统一改为接收 `因果主信息类*`，实例/抽象因果通过形态自动校正，不再依赖兼容壳类型。
3. `因果类.cpp` 中实例因果枚举、抽象因果枚举、抽象因果根节点查找、抽象因果模板查找与创建，全部改成 `因果主信息类 + 是实例因果/是抽象因果` 过滤。
4. `任务管理任务模块.impl.cpp` 中剩余的 `new 因果实例主信息类{}` 已全部切换为 `new 因果主信息类(枚举_因果形态::实例)`。
5. `控制面板类.cpp`、`语言类_基础信息命名.cpp`、`语言类_二次特征表达.cpp` 中原本依赖兼容壳类型判断实例/抽象因果的逻辑，全部改为 `取因果主信息(...) + 形态判断`，避免统一后出现运行时误判。

## 当前口径

- `因果主信息类` 是唯一正式运行时承载。
- `因果实例主信息类 / 因果模板主信息类` 只保留兼容语义名，不再作为独立运行时主信息层扩散。
- 实例因果和抽象因果的判断，统一通过 `枚举_因果形态` 完成。
