# 需求树权威组织任务主信息合同调整

完成时间：2026-04-25

## 结构结论

- 需求树是任务组织结构的权威来源。
- 任务不再强行构建独立树形结构。
- 每个需求节点挂自己的任务头；叶子需求对应执行任务，非叶子需求对应管理任务。
- 原步骤节点不再作为任务树结构必需项；长过程由方法结构或任务执行记录表达。

## 本次调整

- `需求主信息类` 改为表达需求树角色、主任务与任务列表；父子需求关系完全由节点物理树表达。
- `需求主信息类` 移除步骤挂接语义字段，不再保留 `来源步骤节点` 和 `任务挂接方式`。
- `需求主信息类` 移除 `需求虚拟存在`，需求只表达被需求对象的当前状态与目标状态。
- `需求主信息类` 移除来源追踪字段，来源复盘后续改走日志或审计记录。
- `需求主信息类` 移除派生原因与缺口描述字段，缺口解释后续由需求类型、状态差额或审计记录表达。
- `任务主信息类` 改为单一任务实例合同，新增 `任务角色`、`完成口径` 和 `执行记录列表`。
- `任务主信息类` 删除旧的头/步骤/结果分载合同，保留任务目标、执行方法、结果状态和运行统计。
- 删除单一值的 `任务节点种类` 与已失去节点意义的 `任务结果角色`，结果改由 `结果状态信息` 和 `执行记录列表` 表达。
- 已给需求主信息和任务主信息的保留字段补充用途注释，作为后续实现迁移合同。

## 注意

- 本次按最新模型先改主信息合同，未要求也未执行编译验证。
- 后续需要把实现代码从旧字段迁移到 `需求.主任务/任务列表` 与 `任务.对应需求/执行记录列表`。
