# 任务执行升级为本能方法计划 v0.1

状态：历史完成项，2026-05-13 已被新口径废止  
日期：2026-05-12

## 0. 2026-05-13 废止说明

最新规范已修正为：

```text
任务筹办 / 任务执行不是本能方法。
它们是任务管理生命周期普通函数，不注册到方法树，不进入候选方法集。
同步筹办回执、同步执行回执、提交任务状态变化仍是本能方法。
```

本文件只保留为历史记录，不再作为当前实现依据。

## 1. 修订口径

本计划当时按三通道边界，把“任务执行”升级为本能方法；该口径已废止：

```text
运行存在：
    任务层回执，给任务管理层读取。

输出结果场景：
    世界事实输出，只承载真实特征变化、状态节点、动态事实、结果状态。

方法虚拟存在：
    本方法自身记忆，只记录运行账、最近运行、学习产物和索引。
```

这不是恢复旧的 `自我_任务执行一步` 步骤壳。旧步骤壳、旧工作线程直改任务账、旧任务状态写入输出结果场景的做法仍然废止。

## 2. 已完成代码

1. 旧 `枚举_本能方法ID::自我_任务执行` 仅允许作为历史残留 ID 被识别和排除。
2. “任务执行”不再注册为自我内部动作本能方法种子。
3. `任务模块.执行.ixx` 的普通 `执行任务(...)` 是任务管理生命周期函数。
4. `自我动作实现模块.ixx` 中旧 `自我_任务执行` 本能函数和规格确认已删除。
5. 历史实现曾采用四段模式：
   - 确保自身方法环境；
   - 通过自身条件节点绑定输入；
   - 调用当前方法；
   - 写任务层执行回执并更新本方法运行账。
6. 当前有效口径是：任务执行普通函数不直接写任务虚拟存在，不把任务状态写入输出结果场景，只产出过程结果和任务层回执。

## 3. 当前行为

`任务执行` 读取就绪任务的当前方法和可执行输入参数场景，调用当前方法后，根据运行结果和需求满足证据写入运行存在：

```text
建议任务状态 = 完成 / 待重筹办 / 失败
执行缺口类型 = 无执行缺口 / 当前方法缺失 / 当前方法执行失败 / 需求未满足 等
当前方法运行存在 = 本轮被调用方法返回的运行存在
输出结果场景 = 本轮世界事实输出场景
```

任务虚拟存在的真实状态提交仍由任务管理层读取运行回执后执行。
