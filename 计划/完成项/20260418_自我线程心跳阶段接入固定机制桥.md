# 20260418 自我线程心跳阶段接入固定机制桥

## 本次目标

将 `自我线程` 的主循环心跳/巡检阶段正式接到 `自我_固定机制模块`，形成：

`主循环心跳阶段 -> 固定机制桥 -> 核心状态快照 -> 线程摘要`

## 本次完成

1. 在 `自我线程模块.impl.cpp` 中新增 `import 自我_固定机制模块;`
2. 在 `步骤_本轮结算学习触发故障收口_(...)` 进入心跳阶段后，先执行 `执行固定机制一步_最小_v0(...)`
3. 固定机制执行前后都读取 `读取自我核心状态快照_最小_v0(...)`，将 `服务值 / 安全值 / 物理安全值 / 风险安全值` 的前后变化写入固定机制摘要
4. 将固定机制摘要追加进线程的 `最近根值摘要`
5. 扩展 `结构_本轮结算封口` 与 `结构_结算治理输出快照`，新增：
   - `已执行固定机制`
   - `固定机制摘要`
6. 即使本轮没有结算输入，心跳阶段也会产出“巡检 + 固定机制”摘要，不再直接提前返回

## 本次效果

现在 `自我线程` 不只是“读自我桥”，而是已经开始通过桥模块执行固定机制并把结果投影回线程摘要与结算快照。

这意味着后续读取 `最近结算治理输出快照()` 时，已经可以区分：

- 本轮是否存在结算输入
- 本轮固定机制是否命中
- 固定机制命中后，自我双值是否发生变化
