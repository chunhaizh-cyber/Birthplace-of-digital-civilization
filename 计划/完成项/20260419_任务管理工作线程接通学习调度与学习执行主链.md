# 2026-04-19 任务管理工作线程接通学习调度与学习执行主链

## 本步目标

让学习不再停留在“创建学习任务实体/刷新调度快照”，而是真正由任务管理工作线程在每轮推进时：

- 选择一条可执行学习任务
- 执行一步学习任务
- 回写学习执行摘要与学习反馈

## 完成内容

### 1. 修正学习调度选择口径

修改 `学习调度模块::选择一条可执行学习任务(...)`：

- 先从 `执行中列表` 选择，保证进入 `采样/执行中` 的学习任务后续还能继续推进
- 其次再从 `就绪队列` 选择

这样学习链不再只会挑“新就绪任务”，而能延续已有执行中的学习任务。

### 2. 新增任务管理侧学习推进薄包装

在 `任务管理任务模块` 中新增：

- `结构_任务管理学习推进结果`
- `执行一步学习推进(...)`

该薄包装负责：

- 构建学习调度快照
- 选择一条可执行学习任务
- 执行一步学习任务
- 返回最小结果：
  - 是否构建调度快照
  - 是否选中学习任务
  - 是否执行学习任务
  - 当前学习账本 ID
  - 学习调度摘要
  - 学习执行摘要
  - 学习反馈摘要

这样 `任务模块.管理工作线程` 不再直接 import `学习调度模块/学习执行模块`，避免模块接口膨胀和编译器内部错误。

### 3. 接入任务管理工作线程主链

在 `任务模块.管理工作线程` 中：

- 每轮治理和最小回写后
- 若未被控制请求阻断
- 则调用 `任务管理任务模块::执行一步学习推进(...)`

并将结果回填到：

- 局部运行摘要
- 最近反馈摘要
- 工作线程壳快照中的最近学习摘要
- 累计学习执行次数

### 4. 解决编译问题

最初把 `学习调度快照/学习调度项/学习执行结果` 直接塞进 `任务模块.管理工作线程` 导出结构时，触发了 MSVC 模块内部编译错误。

本步改为：

- 学习模块只在 `任务管理任务模块.impl.cpp` 中使用
- 工作线程导出结构只保留最小摘要字段

同时避免了 `任务管理任务模块 -> 学习调度模块 -> 学习对象适配器 -> 学习任务模块 -> 任务管理任务模块` 的接口循环依赖。

## 当前结果

学习链现在正式从：

`任务管理任务 -> 创建/更新学习任务实体`

推进到：

`任务管理工作线程 -> 学习调度 -> 选任务 -> 学习执行一步 -> 回写学习反馈`

这意味着“学习为什么还没有真正开始”的主因已经被消除。
