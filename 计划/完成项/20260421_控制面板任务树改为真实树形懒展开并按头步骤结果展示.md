# 2026-04-21 控制面板任务树改为真实树形懒展开并按头步骤结果展示

## 目标

把控制面板中的任务树从“一次性展开全部节点字段子树”的展示方式，改成更贴近真实任务树结构的树形展示：

- 头节点先显示任务目标、步骤数量
- 点击头节点后再加载该头节点的详细字段和下一级真实子节点
- 下一级优先看到步骤节点
- 再下一级看到步骤对应的结果节点
- 不再把 `节点字段` 当成和真实任务节点并列的固定可见子树

## 本次改动

### 1. 任务树节点数据模型

在 `D:\鱼巢\控制面板类.ixx` 的 `结构_控制面板树节点` 中增加了：

- `详情行`

用于承载每个任务节点自己的详细字段文本，供 HTML 页在点击节点时懒加载显示。

### 2. 任务树节点摘要改造

在 `D:\鱼巢\控制面板类.cpp` 中新增了任务树摘要 helper：

- `私有_统计任务直接子节点种类(...)`
- `私有_统计任务后代节点种类(...)`
- `私有_任务目标文本(...)`
- `私有_任务树显示摘要(...)`

当前展示规则：

- 头节点：显示 `状态 / 任务目标 / 步骤数量`
- 步骤节点：显示 `状态 / 执行方法 / 结果数量`
- 结果节点：显示 `状态 / 结果角色 / 下一步数量`

### 3. 真实树与详细字段分离

`私有_构建任务树详情节点(...)` 不再把 `节点字段` 作为真实子节点挂到树上显示，而是：

- 继续按真实父子关系递归构建任务节点
- 把原来 `节点字段` 里的内容扁平化收进当前节点的 `详情行`

这样任务树可见层级只保留真实任务节点：

- 头节点
- 步骤节点
- 结果节点
- 结果节点下继续可能长出的步骤节点

### 4. 任务树 HTML 改成懒展开

在 `D:\鱼巢\控制面板类.cpp` 中新增：

- `私有_追加任务树详情HTML(...)`
- `私有_追加任务树节点HTML(...)`
- `私有_任务树HTML(...)`

任务树页改为专用渲染，不再复用通用 `私有_树HTML(...)`。

当前 HTML 结构：

- 首屏只渲染任务节点摘要
- 每个节点把自己的详情块和子节点树放在 `<template>` 中
- 用户第一次展开时，再由前端脚本把模板内容插入 DOM

### 5. 前端点击后再加载

在控制面板 HTML 内嵌脚本中增加：

- `初始化任务树懒加载(...)`

行为：

- 首次点击任务节点时，才加载该节点的详细字段和下一层子节点
- 已加载过的节点不会重复构造

## 结果

控制面板任务树页现在满足：

- 使用真实任务树父子关系显示
- 头节点先看任务目标和步骤数量
- 点击后再看到详细字段
- 点击后再看到步骤节点/结果节点
- 不会一上来把全部字段子树全部铺开
