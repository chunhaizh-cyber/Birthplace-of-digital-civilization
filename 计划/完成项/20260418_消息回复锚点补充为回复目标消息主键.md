# 20260418 消息回复锚点补充为回复目标消息主键

## 本次完成

- 在 `自我线程模块.消息协议.ixx` 的 `结构_治理消息头` 中新增 `回复目标消息主键`
- 在 `任务管理任务模块.消息协议.ixx` 的 `结构_任务管理消息头` 中新增 `回复目标消息主键`
- 在 `任务管理任务模块.消息协议.ixx` 的 `结构_任务管理控制请求` 中新增 `回复目标消息主键`
- `自我线程` 下发控制请求时，正式把当前已消费治理消息的 `消息主键` 写入控制请求的 `回复目标消息主键`
- `任务管理工作线程` 在消费控制请求并回报线程状态消息时，会把该 `回复目标消息主键` 写回消息头
- `任务管理线程状态消息 -> 治理消息` 的转换链现在会保留并透传 `回复目标消息主键`

## 本次口径

- 回复消息不应只有“这是什么结果”，还必须明确“这是在回复哪条消息”
- 这次先收口在最小闭环：
  `自我线程控制请求 -> 工作线程线程状态回报 -> 自我线程治理消息`
- 当前实现没有把所有类型消息都一并改完，先优先打通已经跑实的控制回报链

## 结果

- `自我线程` 与 `任务管理工作线程` 之间的控制请求/状态回报，已经具备正式的回复锚点
- 后续如果其它线程要接入同一消息合同，可以直接复用这套 `回复目标消息主键` 口径

## 验证

- 已执行：
  `msbuild D:\鱼巢\鱼巢.vcxproj /t:Build /p:Configuration=Debug /p:Platform=x64 /m`
- 结果：
  `0 error / 0 warning`

## 后续建议

- 在控制面板或治理摘要中显式展示 `消息主键 -> 回复目标消息主键` 关系，方便观察回路
