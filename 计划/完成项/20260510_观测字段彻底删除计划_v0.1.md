# 观测字段彻底删除计划 v0.1

## 0. 背景

当前项目在推进自我内部循环、自我苏醒、因果补完和初步成熟时，为了临时观察进度，加入了大量诊断、快照、摘要、完成度、预览、闸门、夹具和 summary 字段。

这些字段曾经有用，但现在已经产生明显副作用：

```text
根需求存在
被显示成
需求树生成成立；

诊断快照存在
被误读成
内部循环已经闭合；

学习需求预览存在
被误读成
学习需求已经入树；

苏醒诊断字段存在
被误读成
自我已经苏醒。
```

因此需要彻底删除生产代码中的观测字段，回到最新口径：

```text
没有真实运行闭环，就不能用观测字段假装完成。
```

## 1. 删除目标

本计划目标不是优化控制面板，而是清理生产代码里只为观察而存在的状态字段。

删除后，系统只能通过三类东西判断状态：

```text
1. 权威结构字段
   需求、任务、方法、存在、状态、因果等真实业务结构。

2. 运行消息 / 回执 / 工作结果
   任务推进回执、任务工作结果、任务管理上行消息、错误返回。

3. 日志和报错
   非正常逻辑错误必须日志 / 弹错 / 报错，不允许被观测字段吞掉。
```

## 2. 观测字段定义

本计划中的“观测字段”指：

```text
为了控制面板、命令行摘要、阶段验收、临时调试而新增，
但不参与真实需求、任务、方法、因果运行决策的字段。
```

典型名称包括：

```text
诊断
快照
摘要
完成度
苏醒等级
当前自我报告
内部循环轨迹
最近因果补完
学习目标草案
学习需求闸门
需求树生成成立
最近生成事件类型
预览
夹具
summary
```

## 3. 不删除范围

以下内容虽然能被显示，但不是观测字段，不能按本计划删除：

```text
1. 需求主信息字段
   需求主体、需求场景、被需求存在、当前状态、目标状态、满足关系、结构角色、
   安全权重、服务权重、对应任务、任务列表、结算账本、派生归因。

2. 任务主信息字段
   来源需求、任务虚拟存在、运行状态、任务阶段、调度字段、承接权重、折算值。

3. 方法主信息字段
   方法首节点、条件节点、结果节点、动作句柄、能力签名、方法虚拟存在。

4. 任务推进结构
   筹办回执、执行结果、任务工作包、任务工作结果、任务管理上行消息。

5. 线程运行必需状态
   队列、请求表、幂等索引、停止标志、线程生命周期、错误状态。

6. 日志与错误上报
   项目运行日志、弹错、命令行报错、非正常逻辑错误记录。
```

原则：

```text
显示层可以读取权威结构并实时计算视图；
但权威对象不再为了显示长期保存观测字段。
```

## 4. 删除优先级

本计划必须排在 `P2b 需求树真实生长实现` 之前。

原因：

```text
如果旧观测字段还在，P2b 实现后很难判断是需求树真的生长，
还是控制面板 / summary 又把观测字段误报成生长。
```

修订后的顺序：

```text
1. 观测字段彻底删除计划
2. 自我内部循环 P2b 需求树生成成立计划
3. 需求树正常生长特征链与转换落地计划
4. 内部任务闭环 / 结果回写 / 结算
5. 自我苏醒
6. 因果补完和初步成熟
```

### P1：删除自我类观测面

```text
自我类不再保存内部循环完成度、苏醒等级、当前自我报告、最近回写快照、最近结算快照、多轮轨迹等观测状态。
```

```text

结构_自我内部循环轮次轨迹
结构_自我当前报告
```

保留：


```text

```

### P2：删除自我线程观测快照


## 6. 验收标准

### 6.1 删除验收

生产代码中不得存在以下阶段判定型字段：

```text
内部循环完成度
自我苏醒等级
需求树生成成立
因果补完诊断快照
学习目标草案
学习需求入树闸门
学习需求更新指令预览
```

### 6.2 业务不变验收

删除观测字段后，以下能力必须保留：

```text
自我初始化
根需求创建
当前主需求读写
需求更新指令
派生需求入树
任务信息节点创建
任务虚拟存在绑定
任务管理界面线程派发
任务工作线程工作包处理
错误日志 / 弹错
```

### 6.3 防误判验收

不允许再出现：

```text
有根需求 -> 需求树生成成立
有诊断快照 -> 内部循环成立
有苏醒等级字段 -> 自我苏醒
有学习目标预览 -> 学习需求形成
有探索摘要 -> 初步成熟
```

阶段完成只能由真实运行链路证明。

## 7. 风险与处理

### 风险 1：控制面板页面大面积失效

处理：

```text
允许临时缩减控制面板页面，只保留权威树浏览和错误展示。
```

### 风险 2：某些业务逻辑误用了诊断字段

处理：

```text
发现后必须迁移到权威结构读取；
不能为了兼容保留诊断字段。
```

### 风险 3：已有完成项文档与生产代码不一致

处理：

```text
完成项作为历史记录保留；
计划索引明确旧观测完成项不代表阶段完成。
```

### 风险 4：短期调试不方便

处理：

```text
用日志、错误码、运行消息、真实树结构代替观测字段；
需要临时调试时使用局部变量或测试夹具，不进生产主信息和线程状态。
```

## 8. 最终定稿

```text
观测字段是过渡期脚手架。
现在它已经开始遮住真实结构，必须拆掉。

删除后：
自我是否闭环，看真实需求、任务、方法、结果和结算；
自我是否苏醒，看连续多轮真实治理；
是否初步成熟，看学习需求和探索需求是否真实入树、执行、回写。
```

一句话：

```text
不再用“看起来像活着”的字段证明生命闭环；
只用真实结构变化和真实运行结果证明。
```
