# 2026-04-19 删除因果签名并改为动作迁移直比归并

## 本次目标

- 删除因果信息中的 `因果签名` 口径
- 停止使用签名驱动 `动态 -> 抽象因果` 的查找与归并
- 改为：
  - 先由动态生成状态迁移方向或值对
  - 再按 `动作 + 状态迁移结构 + 方向/值对` 查找抽象因果
  - 命中则归并，未命中则新建

## 已完成

1. 删除 `因果主信息类` 中的 `因果签名` 字段。
2. 删除 `因果类.cpp` 中全部签名辅助函数：
   - `私有_状态特征签名`
   - `私有_特征模板签名片段`
   - `私有_状态模板签名片段`
   - `私有_迁移模板签名片段`
   - `私有_迁移模板结构签名片段`
   - `私有_刷新因果签名`
3. 将抽象因果查找逻辑改为直接比较：
   - 动作模板/动作语义
   - 主状态迁移结构
   - 方向型迁移的一致方向
   - 值对型迁移的一致因值/果值
4. 将抽象因果归并逻辑改为直接比较同结构迁移，不再做签名精确命中。
5. 删除控制面板因果树中的 `因果签名=` 显示。
6. 删除 `追加条件模板/追加结果状态/绑定动作主体语义` 这些路径里残留的签名刷新。

## 当前结果

现在 `动态归并为因果` 的主链已改为：

`动态 -> 生成状态迁移 -> 按动作和迁移查找抽象因果 -> 归并或新建`

不再依赖签名做语义判定。

## 备注

- 这一步的重点是去掉签名污染，让因果归并回到“动作 + 迁移关系”本身。
