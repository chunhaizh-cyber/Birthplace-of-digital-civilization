# 方法信息基类删除与 variant 主信息落地计划 v0.1

日期：2026-05-07

## 目标

彻底删除 `方法信息基类` 以及 `方法首节点主信息类`、`方法条件节点主信息类`、`方法结果节点主信息类` 这组三个派生提示类。

方法节点主信息只保留：

```cpp
using 方法角色信息 = std::variant<
    std::monostate,
    结构_方法首节点信息,
    结构_方法条件节点信息,
    结构_方法结果节点信息
>;

class 方法主信息类 {
public:
    结构_方法节点公共信息 公共{};
    方法角色信息 角色{};

    枚举_方法节点种类 节点种类() const noexcept {
        return 公共.节点种类;
    }
};
```

核心原则：

```text
字段存在必须代表字段有效。
首节点字段只存在于 结构_方法首节点信息。
条件字段只存在于 结构_方法条件节点信息。
结果字段只存在于 结构_方法结果节点信息。
方法能力不是单结果槽，而是 条件面 -> 结果包 的能力签名集合。
不再通过胖基类平铺字段表达节点语义。
```

---

## 当前问题

当前 `方法主信息类.h` 虽然已经有 `方法角色信息`，但仍然保留：

```cpp
class 方法信息基类 {
public:
    结构_方法节点公共信息 公共{};
    结构_方法首节点信息 首节点{};
    结构_方法条件节点信息 条件节点{};
    结构_方法结果节点信息 结果节点{};
    方法角色信息 角色{};

    const 自然句节点类* 描述信息 = nullptr;
    枚举_方法节点种类 节点种类 = 枚举_方法节点种类::未定义;
    ...
};
```

这说明 `variant` 还没有成为唯一承载；它只是被同步出来的一份镜像。实际代码仍大量读取：

```text
节点->主信息.动作名
节点->主信息.动作句柄
节点->主信息.条件判定索引
节点->主信息.结果场景
节点->主信息.首节点能力
节点->主信息.节点种类
```

这些访问必须改成角色访问：

```text
主信息.公共.节点种类
取首节点信息(主信息)->动作句柄
取条件节点信息(主信息)->条件判定索引
取结果节点信息(主信息)->结果场景 / 结果能力集
```

---

## 新结构要求

### 0. 方法能力签名

本计划同步修订方法能力模型。方法不是：

```text
一个条件特征 + 一个主结果特征
```

而是：

```text
方法能力签名 = 条件面 -> 结果包
方法 = 能力签名集
```

结构落点：

```cpp
struct 结构_方法条件项 {
    const 词性节点类* 特征类型 = nullptr;
    枚举_方法作用对象 作用对象 = 枚举_方法作用对象::任意;
    std::string 绑定名{};
    枚举_条件比较方式 比较方式 = 枚举_条件比较方式::未定义;
    特征值 期望值{};
    I64 区间下界 = 0;
    I64 区间上界 = 0;
    bool 是否必需 = true;
};

struct 结构_方法条件组 {
    std::vector<结构_方法条件项> 条件项集{};
};

struct 结构_方法条件面 {
    std::vector<结构_方法条件组> 可用条件组集{};
};

struct 结构_方法结果项 {
    const 词性节点类* 特征类型 = nullptr;
    枚举_方法作用对象 作用对象 = 枚举_方法作用对象::任意;
    std::string 绑定名{};
    枚举_结果变化方向 方向 = 枚举_结果变化方向::未定义;
    特征值 目标值{};
    I64 最小变化量 = 0;
    I64 最大变化量 = 0;
    bool 是否主结果 = false;
    bool 是否必要结果 = true;
    bool 是否副结果 = false;
    bool 是否风险结果 = false;
    I64 置信度_Q10000 = 0;
};

struct 结构_方法结果包 {
    std::vector<结构_方法结果项> 结果项集{};
};

struct 结构_方法能力签名 {
    std::string 能力键{};
    结构_方法条件面 条件面{};
    结构_方法结果包 结果包{};
    I64 可靠度_Q10000 = 0;
    I64 成熟度阶段 = 0;
    bool 可被方法查找命中 = true;
    bool 是否根写入原语 = false;
};

struct 结构_方法能力面 {
    std::vector<结构_方法能力签名> 能力签名集{};
};
```

含义固定为：

```text
同一条件组内：条件项全部满足。
多个条件组之间：任意一组满足即可。
同一结果包内：可以包含多个结果项。
查找方法时：需求结果集必须被同一个结果包覆盖。
多个方法组合不属于单次查找方法职责，应交给任务筹办或方法序列构造。
```

当前过渡：

```text
结构_方法首节点能力面 中暂时保留 结果能力集 作为旧查找索引。
最终结果能力索引应由 能力签名集 中每个结果包的结果项派生。
```

### 1. 方法主信息类

新增或改名为：

```cpp
class 方法主信息类 {
public:
    结构_方法节点公共信息 公共{};
    方法角色信息 角色{};

    枚举_方法节点种类 节点种类() const noexcept {
        return 公共.节点种类;
    }

    bool 是否方法首节点() const noexcept;
    bool 是否方法条件节点() const noexcept;
    bool 是否方法结果节点() const noexcept;

    结构_方法首节点信息* 取首节点信息() noexcept;
    const 结构_方法首节点信息* 取首节点信息() const noexcept;

    结构_方法条件节点信息* 取条件节点信息() noexcept;
    const 结构_方法条件节点信息* 取条件节点信息() const noexcept;

    结构_方法结果节点信息* 取结果节点信息() noexcept;
    const 结构_方法结果节点信息* 取结果节点信息() const noexcept;
};
```

允许保留便捷查询函数，但它们只能读 `variant`：

```cpp
bool 有动作骨架() const noexcept;
bool 有结果能力() const noexcept;
const 词性节点类* 首个结果能力特征类型() const noexcept;
bool 有最小出生锚点() const noexcept;
bool 是仅结果特征需求位() const noexcept;
const std::vector<结构_方法能力签名>* 取能力签名集() const noexcept;
```

### 2. 构造函数替代派生类

删除：

```cpp
class 方法首节点主信息类 : public 方法信息基类
class 方法条件节点主信息类 : public 方法信息基类
class 方法结果节点主信息类 : public 方法信息基类
```

改为静态构造函数或自由函数：

```cpp
static 方法主信息类 创建方法首节点主信息(...);
static 方法主信息类 创建方法条件节点主信息(...);
static 方法主信息类 创建方法结果节点主信息(...);
```

或者：

```cpp
方法主信息类::首节点(...)
方法主信息类::条件节点(...)
方法主信息类::结果节点(...)
```

### 3. 链表模板承载类型切换

当前：

```cpp
class 方法类 : public 链表模板<方法信息基类>
```

目标：

```cpp
class 方法类 : public 链表模板<方法主信息类>
```

这一步是本计划的验收核心。完成后，代码里不应再出现 `方法信息基类`。

## 实施步骤

### 阶段一：新增角色访问 helper

在不改业务逻辑前，先补齐统一 helper，减少后续改动面：

```cpp
inline 枚举_方法节点种类 方法节点种类(const 方法主信息类& 主信息);
inline 结构_方法首节点信息* 方法首节点信息(方法主信息类& 主信息);
inline const 结构_方法首节点信息* 方法首节点信息(const 方法主信息类& 主信息);
```

并补充只读便捷函数：

```text
方法动作名
方法动作句柄
方法来源
方法作用域
方法虚拟存在
方法结果能力集
方法能力签名集
方法条件判定索引
方法结果场景
```

要求：

```text
helper 只读 variant，不再读取平铺镜像字段。
```

### 阶段二：迁移读路径

逐文件迁移直接读字段：

```text
方法类.cpp
方法虚拟存在服务类.cpp
本能方法类.cpp
自我动作实现模块.ixx
自我_执行桥模块.ixx
控制面板类.cpp
控制面板摘要线程模块.impl.cpp
自检线程模块.impl.cpp
自我类.初始化.cpp
动态类.cpp / 因果类.cpp / 语言类相关命名代码
```

迁移规则：

```text
主信息.节点种类          -> 主信息.节点种类()
主信息.描述信息          -> 主信息.公共.描述信息
主信息.统计              -> 主信息.公共.统计
主信息.动作名            -> 取首节点信息()->动作名
主信息.动作句柄          -> 取首节点信息()->动作句柄
主信息.方法虚拟存在      -> 取首节点信息()->方法虚拟存在
主信息.首节点能力        -> 取首节点信息()->能力
主信息.首节点能力.结果能力集 -> 取首节点信息()->能力.能力面.能力签名集 派生结果索引
主信息.条件场景          -> 取条件节点信息()->条件场景
主信息.条件判定索引      -> 取条件节点信息()->条件判定索引
主信息.结果场景          -> 取结果节点信息()->结果场景
主信息.结果初始状态      -> 取结果节点信息()->结果初始状态
```

对于“首节点也持有默认条件场景 / 结果场景”的历史写法，需要明确落点：

```text
首节点只持有方法级执行信息与能力签名集。
条件场景属于条件节点。
结果场景属于结果节点。
如果首节点需要缓存最近模板，也应新增明确字段，不能复用条件/结果节点字段。
```

### 阶段三：迁移写路径

所有构造和补齐路径改成写入角色对象：

```text
注册方法首节点
注册需求位方法首节点
注册需求方向方法首节点
补齐本能方法首节点
创建条件节点
创建结果节点
取或创建方法虚拟存在
初始化方法虚拟存在信息
```

写入规则：

```cpp
方法主信息类 主信息{};
主信息.公共.节点种类 = 枚举_方法节点种类::方法首节点;

结构_方法首节点信息 首节点{};
首节点.动作名 = 动作名;
首节点.动作句柄 = 动作句柄;
首节点.能力.能力面.能力签名集 = ...;

主信息.角色 = std::move(首节点);
```

禁止继续出现：

```text
主信息.同步角色面镜像()
主信息.首节点 = ...
主信息.条件节点 = ...
主信息.结果节点 = ...
```

### 阶段四：删除镜像字段与同步函数

从主信息里删除：

```text
首节点
条件节点
结果节点
同步角色面镜像
所有平铺字段
```

删除后全仓搜索应无：

```text
方法信息基类
方法首节点主信息类
方法条件节点主信息类
方法结果节点主信息类
同步角色面镜像
主信息.节点种类
主信息.动作名
主信息.动作句柄
主信息.条件判定索引
主信息.结果主键
主信息.首节点能力
主信息.首节点能力.结果能力集
```

注意：`结构_方法首节点信息::动作句柄` 仍然存在，因为动作句柄是执行入口；禁止的是用动作句柄做需求结果查找。

### 阶段五：编译与语义验收

至少完成：

```text
Debug x64 编译通过。
方法查找路径只读能力签名集或由能力签名集派生的结果项索引。
方法执行路径通过首节点角色读取动作句柄。
条件节点只读条件角色。
结果节点只读结果角色。
单次查找不得把多个方法拼接成需求结果集覆盖。
控制面板能显示方法树字段。
自检线程能统计方法结构。
```

## 风险点

### 1. 链表模板按值保存主信息

这是本次改造的目标之一。切到 `方法主信息类` 后，不再有派生切片问题。

### 2. 首节点历史上混用了条件/结果字段

当前首节点上仍可能保存条件场景、结果场景、结果初始状态等历史字段。迁移时不能简单搬进首节点角色，需要判断：

```text
执行入口需要的放首节点。
条件约束放条件节点。
结果样本放结果节点。
跨节点索引如确实需要，新增明确的首节点索引字段。
```

### 3. 结果能力集是过渡索引

`结果能力集` 已经比单主结果字段前进了一步，但仍然缺少条件面和结果包边界。迁移时应避免继续扩展它的语义：

```text
新增能力写入：写 能力签名集。
旧查找索引：可从 能力签名集 派生结果项索引。
验收完成后：删除 结构_方法结果能力 和 结果能力集，或仅保留为私有缓存而非主信息字段。
```

### 3. 控制面板与命名代码容易散读字段

这类代码不影响核心执行，但引用面广。应通过统一 helper 降低改动量。

## 一句话结论

本计划要完成的不是再整理一次字段，而是把方法主信息从：

```text
胖基类 + variant 镜像
```

改成：

```text
公共信息 + variant 唯一角色承载
```

这样方法首节点、条件节点、结果节点才真正成为三种不同结构，而不是共用一套大字段。
