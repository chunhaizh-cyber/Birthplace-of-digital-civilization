## 1. 总目标

`本能动作管理类` 只作为任务管理界面线程可调用的能力供应层，承担五项功能：

1. 注册功能函数。
2. 查询功能函数。
3. 检查是否可调用。
4. 执行功能函数。
5. 返回标准结果。

本类不承担任务管理、需求生成、需求满足、方法评价、方法虚拟存在更新、动态记录、执行样本记录。

核心原则：

- 方法就是方法，不是任务。
- 方法执行只面对方法首节点、输入参数和输出结果。
- 任务、需求、线程、控制面板、调度状态都不能成为本能动作函数的输入依赖。
- 不保留旧上下文兼容路径；调用方一起重构。

## 3. 目标接口方案

不新增复杂结构体，直接使用现有节点、枚举和返回结果。

### 3.1 动作函数签名

目标签名：

```cpp
using 本能动作函数 = std::function<结构_本能动作步骤结果(
    方法类::节点类* 方法首节点,
    场景节点类* 输入参数场景,
    场景节点类* 输出结果场景)>;
```

含义：

- `方法首节点`：本次执行的方法。
- `输入参数场景`：方法参数来源。
- `输出结果场景`：方法结果写入或承载位置，可空。

动作函数不能读取任务节点、需求节点、任务列表、需求树、控制面板或线程状态。

### 3.2 标准结果

保留 `结构_本能动作步骤结果`，但收口为方法执行结果。

保留字段：

| 字段 | 含义 |
| --- | --- |
| `动作ID` | 本次执行的动作 ID |
| `成功码` | 0 成功，非 0 为失败或特殊状态 |
| `置信度` | I64，越大表示越可信 |
| `质量` | I64，越大表示结果质量越好 |
| `成本_us` | I64 或无符号耗时，越小成本越低 |
| `结果存在集场景` | 方法执行产出的结果场景 |
| `结果事件集场景` | 方法执行事件结果场景 |
| `动作动态` | 真实方法动作动态，可由执行桥填写，不由管理类创建 |
| `备注` | 必要诊断文本 |

删除字段：

- `成功`：由 `成功码 == 0` 判断。
- `质量_Q10000`。
- `成本_Q10000`。
- `方法虚拟存在`。
- `需求当前状态`。
- `需求目标状态`。
- `需求方向`。
- `需求方向说明`。

说明：

- 任务管理界面线程可以根据 `成功码` 和结果场景改变任务状态。
- 自我线程可以根据任务管理界面线程提交的消息处理需求。
- 本能动作管理类不直接表达需求。

### 3.3 注册功能函数

目标接口：

```cpp
bool 注册(
    枚举_本能动作ID 动作ID,
    本能动作函数 动作函数,
    std::string 函数名称 = {},
    方法类::节点类* 方法首节点 = nullptr);
```

规则：

- `动作ID` 不能是 `未定义`。
- `动作函数` 必须有效。
- 函数名称为空时从 `本能方法类` 默认元信息取。
- 注册只登记函数入口，不创建需求，不修改任务，不更新方法虚拟存在。

### 3.4 查询功能函数

保留：

- `有(...)`
- `查询(...)`
- `查询_按函数名(...)`
- `查询函数名称(...)`
- `查询ID(...)`
- `枚举已注册ID()`

调整：

- `结构体_本能动作上下文` 改名为更准确的登记信息类或直接保留但语义改为“动作登记信息”。
- 登记信息只保留：动作函数、方法首节点、函数名称。

### 3.5 检查是否可调用

新增核心检查入口：

```cpp
bool 检查是否可调用(
    枚举_本能动作ID 动作ID,
    方法类::节点类* 方法首节点,
    场景节点类* 输入参数场景,
    结构_本能动作步骤结果* 输出 = nullptr) const noexcept;
```

检查内容：

1. 动作 ID 有效。
2. 动作函数已注册。
3. 方法首节点存在。
4. 方法首节点的节点种类是 `方法首节点`。
5. 方法首节点具备动作函数或动作指针：
   - 本能函数 ID 必须能查到注册函数。
   - 外部实现主键和动作序列主键不在本能动作管理类中执行。
6. 输入参数满足方法条件节点：
   - 无条件索引时视为无需输入参数。
   - 有条件索引或条件场景时，必须提供输入参数场景。
   - 复用 `本能动作类::场景_模式匹配` 做条件场景与输入参数场景匹配。

不检查：

- 任务状态。
- 需求状态。
- 调度队列。
- 控制面板。
- 方法虚拟存在评价特征。

### 3.6 执行功能函数

目标接口：

```cpp
bool 执行功能函数(
    枚举_本能动作ID 动作ID,
    方法类::节点类* 方法首节点,
    场景节点类* 输入参数场景,
    场景节点类* 输出结果场景,
    结构_本能动作步骤结果* 输出 = nullptr) const noexcept;
```

流程：

1. 调用 `检查是否可调用`。
2. 检查失败则返回标准结果。
3. 调用动作函数。
4. 补齐动作 ID。
5. 捕获异常并转换为标准结果。
6. 返回是否成功。

删除：

- 旧 `运行(...)`。
- 旧 `运行_兼容(...)`。
- 旧执行上下文构造路径。

## 4. 错误码方案

- `本能动作错误_未注册`
- `本能动作错误_执行失败`
- `本能动作错误_动作异常`
- `本能动作错误_动作未知异常`

新增常量，不新增结构体：

| 错误码 | 含义 |
| --- | --- |
| `本能动作错误_方法首节点缺失` | 没有提供方法首节点 |
| `本能动作错误_方法节点类型错误` | 传入节点不是方法首节点 |
| `本能动作错误_缺动作入口` | 方法头没有可执行动作函数或动作指针 |
| `本能动作错误_动作入口不归本类执行` | 外部实现或动作序列不由本能动作管理类执行 |
| `本能动作错误_输入参数不满足` | 输入参数不满足方法条件 |

错误结果只表达方法执行层问题，不表达任务下一步。
