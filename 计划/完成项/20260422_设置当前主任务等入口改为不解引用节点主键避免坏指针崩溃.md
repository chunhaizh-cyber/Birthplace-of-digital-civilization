# 完成项

- 时间：2026-04-22
- 主题：设置当前主需求/任务/方法时不再为记录标题解引用节点主键

# 背景

自我线程初始化阶段虽然已经删除了“设置当前主任务”的主动调用，但旧运行体或失效指针一旦仍然流入 `自我类::设置当前主任务(...)`，原实现会先调用 `节点->获取主键()`，从而在坏指针场景下直接崩溃。

# 修改

- 修改文件：[D:\鱼巢\自我类.cpp](D:/鱼巢/自我类.cpp)
- 调整 `设置当前主需求(...)`、`设置当前主任务(...)`、`设置当前主方法(...)`
- 这三个入口现在不再为了记录变化标题去解引用传入节点的主键
- 当传入节点非空时，改为写入固定安全标题：
  - `已切换到需求节点`
  - `已切换到任务节点`
  - `已切换到方法节点`

# 结果

- 即便还有旧路径或坏指针误调这三个入口，也不会再在“获取主键”阶段崩溃
- 初始化阶段“不设置当前主任务”的边界继续保留
