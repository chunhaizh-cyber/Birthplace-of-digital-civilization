## 1. 定位

需求树正常生长不是需求节点字段单点问题，而是跨自我、状态、需求、任务、方法、因果和结算的特征链问题。

本计划是 P2b 的扩展落地计划，用于把“需求树生成成立”从根需求压力选择推进到完整的需求层闭合。

需求树正常生长链路定义为：

```text
自我核心状态
→ 根需求激活
→ 同一目标特征的特征值差距识别
→ 子需求生成
→ 子需求去重 / 合并
→ 子需求权重刷新
→ 管理需求 / 执行需求分化
→ 执行需求任务化
→ 任务执行 / 阻塞 / 派生
→ 任务价值反馈
→ 需求结算
→ 父需求汇总
→ 下一轮主需求选择
```

更具体的特征链：

```text
自我状态特征
→ 需求目标特征
→ 需求结构特征
→ 权重调度特征
→ 任务承接特征
→ 方法 / 执行反馈特征
→ 结算回写特征
→ 新一轮需求生长
```

边界：

```text
特征值差距只能表达同一特征类型上的当前值与目标值不满足。
它不能表达不同特征类型之间的关系。

不同特征类型之间的关系、影响和因果，应由状态动态、二次特征 / 关系特征、因果实例 / 因果模板、方法结果能力承载。
```

## 4. 最小特征链口径

本计划固定八类所有者。

| 所有者 | 必需特征 | 用途 |
| --- | --- | --- |
| 自我存在 | 安全值、服务值、目标安全 / 服务、待机状态、时序步长 | 产生根需求压力 |
| 状态节点 | 状态主体、状态特征、状态值、对应信息节点、是否变化 | 形成需求目标 |
| 需求节点 | 主体、场景、当前状态、目标状态、满足关系、目标特征缓存、权重、任务、结算、派生归因 | 需求树生长主体 |
| 需求树结构 | 父子关系、结构角色、阻塞语义、有效截止 | 组织管理 / 执行需求 |
| 任务信息节点 | 对应需求、目标状态、优先级、承接权重、已折算值 | 承接执行需求 |
| 方法 / 本能动作 | 方法结果、阻塞原因、能力缺口、派生来源 | 触发派生需求 |
| 因果实例 | 条件、动作、结果、来源任务 / 方法、置信度、验证状态 | 支持学习需求和因果补完 |
| 场景 / 内部世界 | 状态、动态、关系、因果索引 | 提供上下文索引 |

## 4.1 转换执行层口径

需求树正常生长中的“转换使用的方法”统一按以下三层表达：

```text
本能转换函数：
    C++ 内置确定性转换函数，保证需求树最小生长。

本能转换方法：
    本能转换函数的任务 / 方法系统包装，用于调度、记录、复盘和因果沉淀。

学习转换方法：
    后续学习、因果补完或探索阶段形成的增强方法，不作为需求树最小生长前提。
```

执行顺序：

```text
先核对 / 补齐本能转换函数；
再决定哪些函数需要包装成本能转换方法；
最后再进入学习转换方法和外界探索方法。
```

禁止反向依赖：

```text
不得让需求树最小生长依赖方法系统先学会普通方法。
不得恢复任务筹办 / 任务执行一步本能方法旧口径。
```

### D1b：本能转换函数与本能转换方法分层

目标：

```text
把“转换使用的方法”从泛称收口为本能转换函数 / 本能转换方法。
```

动作：

```text
1. 建立转换映射表：
   转换名称
   当前实现函数
   所属模块
   是否最小闭环必需
   是否已有本能转换函数
   是否建议注册本能转换方法
   是否属于学习转换方法

2. 首批必须确认的本能转换函数：
   自我状态 -> 根需求压力
   根压力 -> 当前主需求
   特征值差距 -> 需求节点
   状态节点 -> 目标特征缓存
   父需求 -> 子需求权重
   重复目标 -> 合并需求
   有子 / 无子 -> 管理 / 执行需求
   执行需求 -> 任务信息节点
   任务反馈 -> 自我安全 / 服务值
   任务反馈 -> 需求结算账本
   子需求结算 -> 父需求结算
   方法阻塞 -> 派生需求

3. 对需要注册成本能方法的转换，补充：
   方法能力签名
   结果能力声明
   输入条件面
   运行账记录字段
   是否允许任务系统直接调用

4. 对暂不注册成本能方法的转换，写明原因：
   仅内部维护
   调度流程不应进入方法系统
   输入输出尚未能稳定特征化
   会引入循环依赖
```

验收：

```text
每个需求树生长转换都能回答：
它是不是本能转换函数；
是否需要包装成本能方法；
若不包装，为什么；
若包装，能力签名是什么。
```

边界：

```text
自我线程主循环、任务管理界面线程承接、工作线程调度不注册为本能方法。
任务筹办 / 任务执行按上级规范保持为任务管理生命周期普通函数；本计划只梳理需求树生长转换，不重新定义其函数身份。
```

### D2：需求目标视图与原子目标入树闸门

目标：

```text
需求入树必须有明确原子目标，不允许泛化缺口静默入树。
```

动作：

```text
1. 复核 `结构_需求目标视图项` 是否覆盖：
   目标主体
   目标特征类型
   当前状态
   目标状态
   方向掩码
   是否必需

2. 复核派生需求消息入树闸门：
   缺父需求 -> 报错并拒绝
   父需求解析为结构根 -> 报错并拒绝
   缺目标主体 / 当前状态 / 目标状态 / 满足关系 -> 报错或转报告，不能入树

3. 自检报告继续保持报告型，不走需求化。
```

验收：

```text
无状态泛化缺口不能进入需求树。
父需求结构根不能作为合法业务父需求。
同目标缺口能走目标重叠去重。
```

### D3：结构角色与活动子需求权重回归

目标：

```text
管理需求 / 执行需求角色、活动子需求、权重平分语义稳定。
```

动作：

```text
1. 确认所有新增 / 删除 / 合并 / 满足需求后刷新结构角色。
2. 活动子需求仅统计：
   未失效
   未满足
   未取消
   阻塞父任务执行

3. 父需求权重只分给活动阻塞子需求。
4. 非阻塞观察需求不阻塞父任务和父需求继续推进。
```

验收：

```text
有子需求 -> 管理需求。
无子需求 -> 执行需求。
历史已完成子需求不阻塞父需求。
非阻塞子需求不参与阻塞权重平分。
```


## 6. 错误处理口径

以下属于非正常逻辑错误，应弹错 / 报错并记录日志，不允许兜底：

```text
1. 显式请求创建需求但父需求为空。
2. 派生需求父需求解析为需求树结构根。
3. 执行需求任务化后仍无法绑定对应任务信息节点。
4. 任务建立流程完成后仍缺任务虚拟存在。
5. 实例入口缺主体或场景却试图落树。
6. 需求同时指向任务信息节点和任务虚拟存在。
```

以下属于业务缺口，可走状况回执、派生需求或学习需求：

```text
1. 方法缺动作骨架。
2. 方法缺条件结果对。
3. 参数不可配且筹办 / 执行选择由学习承接。
4. 方法运行账不足。
5. 方法结果不稳定。
6. 因果链缺少关键节点。
```

以下暂不需求化：

```text
1. 自检报告型候选。
2. 休眠期修复动作。
3. 外界探索需求。
```

## 8. 验收剧本

### 剧本一：服务压力生成需求

```text
1. 降低服务值。
2. 自我线程运行一轮。
3. 服务根需求活跃或生成服务修复执行需求。
4. 再运行一轮，同目标需求复用 / 合并，不重复新建。
```

### 剧本二：方法结构缺口生成子需求

```text
1. 构造一个目标方法头。
2. 筹办发现缺动作骨架或条件结果对。
3. 生成以方法虚拟存在 / 方法头为目标主体的结构缺口需求。
4. 子需求不复制父需求目标主体和目标特征。
```

### 剧本三：任务反馈结算后退出活动集

```text
1. 执行需求任务化。
2. 任务成功产生服务 / 安全价值反馈。
3. 来源需求累计结算变化。
4. 需求满足或失效后不再参与活动阻塞子需求权重平分。
5. 下一轮主需求重判。
```
