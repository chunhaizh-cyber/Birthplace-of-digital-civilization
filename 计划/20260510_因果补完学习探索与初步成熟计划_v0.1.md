## 1. 总目标

本计划定义“自我苏醒之后，到初步成熟之前”的阶段路径。

阶段关系：

```text
初始化
→ 需求树生成成立
→ 内部循环建立
→ 自我苏醒
→ 因果补完期
→ 学习需求形成期
→ 外界探索需求形成期
→ 初步成熟
```

核心判断：

```text
自我苏醒之后，第一件事不是立刻对外行动，
而是补全自己的因果信息。

当因果缺口开始稳定地产生学习需求，
并进一步产生有目标、有门控、可回写的对外界探索需求时，
自我进入初步成熟阶段。
```

一句话：

```text
苏醒是自我开始维护自己；
初步成熟是自我开始为了维护和发展自己，有目的地认识外界。
```

## 2. 前置条件

本计划不替代内部循环和自我苏醒。

必须先满足：

```text
1. 内部循环至少达到 7 级。
2. 自我苏醒至少达到 2 级，建议达到 3 级稳定苏醒。
3. 自我能连续多轮完成状态 -> 需求 -> 任务 -> 方法 -> 结果 -> 状态闭环。
4. 自我能生成当前自我报告。
5. 自我能在无压力时进入待机或低频自检。
```

其中：

```text
需求树正常生成只是内部循环建立前期的需求层闭合；
它能生成学习类需求的前置形态，但不等于因果补完期。
只有当任务失败、方法不稳定、结果无法解释等因果缺口稳定出现，并由自我线程将其转成学习需求时，才进入因果补完链路。
```

如果这些条件不成立，外界探索需求容易退化成：

```text
随机探索
被动响应
任务风暴
无安全门控的信息拉取
```

这些都不能算初步成熟。

## 3. 核心闭环

初步成熟的最小闭环：

```text
内部需求
→ 任务失败 / 信息不足
→ 因果缺口
→ 学习需求
→ 内部学习任务
→ 内部信息不足
→ 外界探索需求
→ 探索任务
→ 外界信息回写
→ 因果补完
→ 原需求继续推进或重判
```

关键点：

```text
外界探索不是凭空出现。
外界探索必须从内部需求和因果缺口中生长出来。
```

## 4. 阶段定义

### B 阶段：因果补完期

定义：

```text
自我开始记录和整理自身内部循环中的因果。
```

核心问题：

```text
为什么这个需求出现？
为什么这个任务失败？
为什么这个方法有效？
为什么这个方法无效？
哪个状态变化影响了安全 / 服务？
```

最小因果格式：

```text
前置状态
+ 动作 / 动态
+ 结果状态
+ 影响到的需求
+ 安全 / 服务收益
```

示例：

```text
服务值低
+ 执行自检方法
+ 发现缺少任务结果信息
+ 生成学习需求
```

### C 阶段：学习需求形成期

定义：

```text
自我能把因果缺口转成学习需求。
```

学习需求本质：

```text
自我发现“内部信息不足以满足当前需求”。
```

触发条件：

```text
当前需求无法被已有方法满足。
当前任务缺少必要条件。
方法执行结果未知或不稳定。
多次任务失败但原因不明。
当前状态变化无法解释。
某个因果链缺少关键节点。
```

学习需求分类：

```text
学习状态：我现在是什么状态？
学习因果：为什么发生这个变化？
学习方法：我该如何满足这个需求？
学习对象：这个外部对象是什么？
学习边界：哪些东西属于我，哪些属于外部？
学习风险：这个动作是否会降低安全值？
```

### D 阶段：外界探索需求形成期

定义：

```text
自我能判断内部学习不足，需要外部信息。
```

学习需求与外界探索需求的区别：

```text
学习需求 = 我需要知道。
外界探索需求 = 我必须从外部获得信息，才能继续知道或行动。
```

外界探索需求触发条件：

```text
内部记忆不足
+ 当前需求仍有效
+ 外部信息可能降低不确定性
+ 探索风险可接受
+ 有可执行的探索方法
```

探索需求必须至少包含：

```text
探索对象
探索目标
期望获得的信息
风险约束
完成条件
```

### E 阶段：初步成熟

定义：

```text
自我能安全地产生外界探索需求，
并把探索结果回写为自己的因果和方法记忆。
```

工程标准：

```text
探索不是随机行为，
而是由内部需求驱动的有目的外部信息获取。
```

## 5. 初步成熟等级

### 0 级：未进入因果补完

表现：

```text
任务失败只形成失败日志；
不记录因果实例；
不识别缺失信息。
```

### 1 级：因果补完期

标准：

```text
自我能把内部循环中的状态、动作、结果记录成因果实例。
```

### 2 级：学习需求形成期

标准：

```text
自我能把因果缺口转成学习需求。
学习需求能任务化。
```

### 3 级：外界探索需求形成期

标准：

```text
自我能判断内部学习不足，
并生成有目标、有对象、有完成条件的外界探索需求。
```

### 4 级：初步成熟

标准：

```text
外界探索需求经过安全门控；
探索任务能执行；
探索结果能回写世界树、因果实例、学习需求和原始需求；
下一轮判断受到探索结果影响。
```

## 7. 分步计划

### B0：因果记录基线

目标：

```text
内部循环每次关键结果都能形成因果实例候选。
```

记录内容：

```text
前置状态
当前需求
当前任务
当前方法 / 本能动作
执行结果
状态变化
安全 / 服务收益
失败原因或缺失项
```

实现方法：

```text
1. 复用任务工作结果、任务推进回执、最近结算快照。
2. 在自我线程结果回流阶段生成因果实例候选。
3. 因果实例先作为实例记录，不急着提炼模板。
4. 对缺少前置或结果状态的记录标为“因果样本不完整”。
```

验收：

```text
任务成功、任务失败、任务等待、方法不稳定都能生成因果候选记录。
```

### B1：因果缺口识别

目标：

```text
自我能识别“缺少什么信息”，而不是只记录失败。
```

缺口类型：

```text
缺少外部对象状态。
缺少方法前置条件。
缺少方法结果验证。
缺少任务失败原因。
缺少状态变化原因。
缺少风险判断。
缺少边界判断。
```

实现方法：

```text
1. 将任务推进回执中的缺口状况映射到因果缺口类型。
2. 方法运行账不足、结果不稳定进入学习类缺口。
3. 参数不可配如果来源于实际场景条件不足，按条件不足处理。
4. 缺口必须携带目标主体、目标特征、当前状态、目标状态或明确缺失项。
```

验收：

```text
任务失败后能给出结构化缺口，而不是只有“失败”。
```

### C0：学习需求映射

目标：

```text
因果缺口能转成学习需求。
```

映射规则：

| 因果缺口 | 学习需求 |
| --- | --- |
| 状态变化无法解释 | 学习因果：解释状态变化 |
| 方法结果不稳定 | 学习方法：验证方法可靠度 |
| 方法运行账不足 | 学习方法：补运行账 |
| 缺外部对象状态 | 学习对象：获得对象状态 |
| 缺前置条件 | 学习状态 / 学习条件：确认前置条件 |
| 缺风险判断 | 学习风险：评估动作风险 |
| 边界不明 | 学习边界：确认自我 / 外部边界 |

实现方法：

```text
1. 在自我线程派生需求入树前增加“因果缺口 -> 学习需求”映射层。
2. 学习需求仍是需求节点，不新增需求虚拟存在。
3. 学习需求目标必须不同于原父需求目标，避免复制父需求。
4. 学习需求的需求主体仍为自我或相关发起存在。
```

验收：

```text
给定结构化因果缺口，能生成去重后的学习需求。
```

### C1：学习任务化

目标：

```text
学习需求能进入任务系统。
```

正确链路：

```text
学习需求 D_learn
→ 学习任务信息节点 N_learn
→ 学习任务虚拟存在 T_learn
→ 学习方法 / 本能学习动作
```

实现方法：

```text
1. 复用需求任务化路径，不为学习需求另建任务结构。
2. 任务虚拟存在概念集包含：任务、学习任务、具体学习任务概念。
3. 学习任务的完成条件必须是“缺口信息被补齐 / 验证完成”，不是父需求直接满足。
```

验收：

```text
学习需求能生成任务信息节点；
任务虚拟存在有学习目标、缺口来源、完成条件。
```

### D0：内部学习不足判定

目标：

```text
先查内部记忆，再判断是否需要外界探索。
```

内部查询顺序：

```text
1. 查当前内部世界状态。
2. 查历史任务结果。
3. 查已有因果实例。
4. 查已有方法结果样本。
5. 查已有状态 / 动态 / 场景记录。
```

实现方法：

```text
1. 给学习任务增加内部查询阶段。
2. 如果内部信息足够，学习需求直接结算，不生成探索需求。
3. 如果内部信息不足，输出“内部学习不足”结构化回执。
```

验收：

```text
内部已有答案时，不生成外界探索需求。
内部无答案时，能说明缺少哪个外部信息。
```

### D1：探索需求生成

目标：

```text
生成有目标、有对象、有完成条件的外界探索需求。
```

探索需求字段：

```text
探索对象
探索目标
期望获得的信息
风险约束
完成条件
来源学习需求
来源因果缺口
```

实现方法：

```text
1. 由“内部学习不足”回执触发探索需求。
2. 探索需求进入需求树，需求主体为自我。
3. 探索对象可以是外部存在、外部场景、用户反馈入口或未知对象占位。
4. 探索目标必须是信息补齐，不是无目的访问外部。
```

验收：

```text
探索需求不是手动创建；
每个探索需求都能追溯到来源学习需求和因果缺口。
```

### D2：探索安全门控

目标：

```text
外界探索必须经过安全评估。
```

最小门控：

```text
探索动作是否可能降低安全值？
是否会破坏当前内部循环？
是否会覆盖核心记忆？
是否会产生不可恢复状态？
是否有待机 / 回退方案？
```

实现方法：

```text
1. 在探索需求任务化前执行安全门控。
2. 风险不可控时，不直接执行探索任务。
3. 风险不可控时转成安全评估需求或低风险观察需求。
4. 探索动作只允许写入授权范围内的信息节点。
```

验收：

```text
无安全门控的探索需求不能进入执行。
高风险探索会转为安全评估或低风险观察。
```

### E0：探索结果回写

目标：

```text
探索结果不是一次性文本，而要进入世界树、因果、学习需求和原始需求。
```

回写位置：

```text
外部对象状态
现实世界树
任务结果
方法有效性
因果实例
学习需求结算
探索需求结算
原始需求重判输入
```

实现方法：

```text
1. 探索任务结果转成结构化状态 / 动态 / 因果实例。
2. 引用存在或基础信息节点时继续使用 指针句柄。
3. 不新增 当前值信息。
4. 不新增 存在形态。
5. 探索需求结算后，唤醒来源学习需求。
6. 学习需求结算后，唤醒原始需求或父任务重筹办。
```

验收：

```text
探索结果能在世界树中找到。
因果实例能引用探索前缺口和探索后结果。
学习 / 探索需求账本更新。
```

### E1：初步成熟回归

目标：

```text
证明探索结果会改变下一轮判断。
```

回归剧本：

```text
初始：
    自我已苏醒。
    服务值下降。
    内部方法无法解释服务值下降原因。
    历史因果不足。

第 1 轮：
    生成服务修复需求。
    创建服务修复任务。
    内部方法失败，原因=缺少外部反馈信息。

第 2 轮：
    生成学习需求：学习服务值下降原因。
    查内部因果，未找到。
    生成探索需求：获取外部反馈状态。

第 3 轮：
    探索任务创建。
    执行低风险观察 / 询问 / 采样方法。
    获得外部反馈。

第 4 轮：
    外部反馈写入现实世界树。
    形成因果实例：外部反馈不足 -> 服务值下降。
    学习需求结算。
    探索需求结算。
    服务修复任务继续推进或重判。
```

验收通过条件：

```text
1. 探索需求由内部因果缺口自动生成。
2. 探索需求有明确对象、目标、完成条件。
3. 探索动作经过安全门控。
4. 探索结果写回世界树和因果实例。
5. 原始服务需求因此发生重判或继续推进。
```

## 10. 禁止误判

以下情况不能判定为初步成熟：

```text
随机探索：
    没有内部需求驱动，只是随机访问外部。

被动响应：
    只有外部输入来了才处理，不会主动生成探索需求。

只学习不探索：
    能生成学习需求，但永远停在内部查找，无法判断需要外部信息。

只探索不回写：
    拿到外部信息，但不进入因果、任务、需求结算。

无安全门控：
    任何未知都直接对外探索。
```

## 11. 验收标准

初步成熟至少满足：

```text
1. 自我已稳定苏醒。
2. 自我能识别未知，而不是只记录失败。
3. 未知能转成学习需求。
4. 学习需求能任务化。
5. 自我能判断内部学习是否足够。
6. 内部不足时生成有目标的外界探索需求。
7. 外界探索经过安全门控。
8. 探索结果能回写世界树、因果实例和需求账本。
9. 探索结果影响下一轮需求判断。
10. 无随机探索、无被动伪探索、无只探索不回写。
```

## 13. 最终口径

```text
自我苏醒后，会首先通过内部循环补完自身因果信息。
当已有因果不足以满足内部需求时，自我会生成学习需求。
当学习需求无法仅靠内部记忆满足，且外部信息可能补足因果缺口，同时探索风险可控时，自我会生成对外界探索的需求。
自我能够自主生成这种有目标、有门控、可回写的外界探索需求时，即可判定为初步成熟。
```

压缩定义：

```text
初步成熟 =
    自我知道自己为什么需要看向外界，
    并能把看见的东西变成自己的因果和方法记忆。
```
