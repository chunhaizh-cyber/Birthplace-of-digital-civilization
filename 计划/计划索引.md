## 全局错误处理规则

```text
非正常逻辑错误不走静默兜底。

适用范围：
1. 模块职责越界，例如自我线程重新创建任务虚拟存在、工作线程直达自我线程等旧链路误入。
2. 关键结构在应存在阶段为空，例如当前自我、内部世界缺失，请求创建需求根 / 任务根 / 方法根失败，任务建立流程完成后仍缺任务虚拟存在。
3. 状态机非法迁移，例如等待态反复派发、已完成任务继续执行、未知去向被当作成功。
4. 信息入口 / 存储树路由不满足上下文约束，例如实例入口缺主体或场景却试图落树。
5. 本应唯一的权威入口出现双入口或并列写入，例如需求同时指向任务信息节点和任务虚拟存在。

处理要求：
1. 弹窗或命令行报错，明确错误位置、错误类型、关键主键和当前阶段。
2. 同步写日志，至少包含模块名、函数名、主键 / 指针、状态枚举、输入摘要和失败原因。
3. 不用“找不到就创建一个泛化对象”“状态未知就当等待”“缺上下文就落默认根”等方式掩盖错误。
4. 允许业务可恢复缺口继续走状况回执、派生需求、学习需求；但工程逻辑错误必须显式暴露。
5. 后续修复计划若需要临时兜底，必须在计划正文标明“临时兼容”和清理条件。
```
