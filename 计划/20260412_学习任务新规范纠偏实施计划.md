# 2026-04-12 学习任务新规范纠偏实施计划

## 1. 计划目标

本文用于把“学习任务最新规范”正式落成一份可执行的代码纠偏计划，回答以下问题：

- 当前规范已经把学习主链改成了什么
- 当前代码还停留在哪套旧口径上
- 哪些偏差必须先改，哪些可以后置
- 应按什么顺序推进，才能既不打断现有验收链，又能把实现拉回新规范

本文不是规范重述，而是后续代码整改的正式施工计划。


## 2. 规范基线

当前规范已统一固定为以下 4 条：

1. `任务管理任务`
   - 负责识别学习目标、生成学习任务、推进学习任务、接收学习反馈。
2. `学习任务`
   - 负责围绕学习目标直接完善 `学习对象方法`。
3. `学习完成后`
   - 只向 `任务管理任务` 反馈 `学习情况`，不再由学习任务自己直接决定“回给谁、立刻怎么用”。
4. `尝试学习方法`
   - 只是学习任务的一种可选执行方法，不是学习任务本体，也不是唯一执行入口。

当前规范主链已经固定为：

`任务管理任务上浮学习需求`
-> `学习任务头节点生成`
-> `下一轮筹办命中学习任务兜底方法`
-> `生成学习方法选项`
-> `学习任务就绪`
-> `选择当前轮学习方法`
-> `执行并直接完善学习对象方法`
-> `条件-结果对融合写入方法`
-> `形成学习情况`
-> `反馈给任务管理任务`
-> `任务管理任务决定继续学习 / 挂起观察 / 使用已完善方法 / 再派生学习任务`


## 3. 当前代码事实判断

当前代码并不是“完全没接上”，而是处于“新旧两套语义并存、主链仍偏旧”的状态。

### 3.1 已经接上的部分

1. `广泛学习`
   - 已从枚举值变成真实分支，相关挂载已接入 `任务管理学习树`。
2. `尝试学习`
   - 已被单独立成本能方法 ID，不再和正式方法资产句柄混用。
3. `学习执行器`
   - 已具备 `采样 / 更新 / 验证 / 提交 / 回滚` 阶段机。
4. `正式资产提交`
   - 已和学习期方法头做了基础分流，不再直接污染长期策略句柄。

### 3.2 仍停留在旧口径的部分

1. `学习任务实体`
   - 仍以 `回流目标方法位 / 回流目标摘要 / 最近回流摘要` 为主字段。
2. `学习执行结果`
   - 仍以 `已回流 / 回流摘要` 为主输出。
3. `治理邮箱事件`
   - 仍以 `学习回流` 为事件类型和唤醒原因。
4. `恢复快照 / 主体汇总`
   - 仍以 `待消费学习回流 / 最近学习回流摘要 / 学习回流次数` 为主统计口径。
5. `学习方法模型`
   - 仍是“当前学习方法单指针”模型，没有正式长出“兜底方法 -> 学习方法选项 -> 当前轮选中方法”这层结构。


## 4. 核心偏差清单

按阻断程度，当前偏差可分成 `P0-P4` 五层。

### 4.1 P0 语义主轴未切换

最核心的问题不是“字段名旧”，而是：

- 代码主链仍把学习结果理解成 `回流`
- 规范主链已要求把学习结果理解成 `学习情况反馈`

这会直接影响：

- 学习执行结果怎么表达
- 自我线程怎样投递治理事件
- 任务管理任务如何做后续消费裁决

### 4.2 P1 学习任务实体缺结构面

当前学习任务实体还缺少规范要求的关键结构：

- `学习任务兜底方法`
- `学习方法选项`
- `当前轮选中学习方法`
- `学习情况消费策略`
- `方法服务范围`

这意味着：

- `尝试学习只是一个选项` 还只停留在规范文字里
- 代码里还没有真正的“多学习方法选项”模型

### 4.3 P2 执行器职责仍然偏旧

当前执行器的主输出仍然偏向：

- `已提交正式方法资产`
- `等待外层正式回流裁决`

而不是规范要求的：

- `已直接完善学习对象方法`
- `已形成学习情况`
- `等待任务管理任务消费`

### 4.4 P3 消费链仍走旧 mailbox 语义

当前自我线程与任务管理事件链仍按：

- `学习回流事件`
- `学习回流待再判`
- `学习回流待消费`

来组织主循环再判。

这会导致：

- 规范里的四种消费动作没有正式落点
- 自我线程和恢复快照继续把旧词当真语义

### 4.5 P4 主体/M10/恢复层仍统计旧语义

主体长期统计和恢复摘要里仍大量使用：

- `累计学习回流次数`
- `最近学习回流方法位`
- `最近学习回流摘要`

如果不改，这些壳会持续把新实现拉回旧理解。


## 5. 纠偏总原则

本轮整改必须遵守以下 6 条：

1. `先桥接、再切主链、再清旧壳`
   - 不建议一上来全量硬改名。
2. `先补结构，再改行为`
   - 先把学习任务实体、事件和结果结构补齐，再动执行主链。
3. `双写过渡`
   - 新旧字段允许短期并存，但主路径必须逐步切到新字段。
4. `不破坏现有验收链`
   - 验收场景应分批迁移，不在同一轮重写所有口径。
5. `先任务管理消费，再主体层汇总`
   - 主体/M10 只应跟着任务管理消费动作变化，不应先独立发明新语义。
6. `尝试学习继续保留`
   - 但只能作为学习方法选项之一，不得重新被拉回“总入口唯一方法”。

## 8. 推荐推进顺序

建议按 3 轮推进：

1. `第一轮`
   - 完成 `P0 + P1`

2. `第二轮`

- 但底层学习执行和治理事件仍然还是旧 `学习回流` 语义

## 10. 最终收口

后续统一按以下判断执行：

1. 当前规范纠偏的真正重点，不是“把回流两个字批量替换掉”。
2. 真正重点是把学习主链从“学习结果自动回流”改成“学习任务完善方法后，向任务管理任务反馈学习情况，由任务管理任务决定后续消费动作”。
3. 因此整改顺序必须先动 `结构`，再动 `执行`，再动 `消费`，最后再清 `统计与旧壳`。

一句话收口：

`先把学习任务补成真正的“可选学习方法 + 直接完善方法 + 反馈学习情况”模型，再去清掉旧回流语义；不要反过来先改名词、后补主链。`
