# 20260403 自我最小循环实现大纲

## 1. 目标

这份大纲只回答一件事：

`一个符合当前规范口径的“自我最小循环”，最少必须由哪些承载面组成，这些承载面按什么顺序闭环，以及它们应分别落在哪一层模块。`

它是总蓝图，不替代阶段计划；后续所有阶段计划、施工清单和代码落点都应服从这份大纲。

## 3. 一句话总纲

`自我最小循环 = 根需求 + 双值核 + 主循环 + 治理中轴 + 任务树承接 + 执行/学习分流 + 结果消息回流 + 双值结算 + 再提交。`

只要缺其中任一环，就还只是“局部机制”，不能称为正式自我闭环。


## 4. 硬边界

以下边界在当前工程阶段必须固定，不允许后续实现再漂移：

1. `需求` 不得直接连到 `方法执行`，正式主链必须经过 `治理中轴`。
2. `自我主循环` 是唯一的总协调层，负责根层重判、双值结算和最终去向决定。
3. `任务管理任务` 是当前自我闭环中的治理承接中轴，不是可有可无的兼容壳。
4. `学习任务` 属于 `任务管理内部治理子任务`，不是第三根任务。
5. `执行层` 和 `学习层` 只能回流结果、证据和建议，不能越权决定最终继续、待机或停止。
6. `正式真值` 优先落在 `自我特征 / 任务事实 / 结果节点`，不能让本地缓存升格成并列真值层。
7. `外部输入` 在当前阶段仍不进入自我闭环，验证以日志为准。
8. `主循环` 负责统一消费回流，不允许叶子线程私自连跑下一轮主闭环。


## 5. 最小定义

`自我最小循环` 不是“线程能 Tick 一次”，而是：

`在根约束下，以双值核维持自我存在与服务能力，并由主循环统一完成根层重判、需求治理、任务管理承接、执行/学习分流、结果回收、双值结算、后果回写与再提交的最小正式闭环。`

因此最小循环至少同时具备：

- `根需求`
- `双值核`
- `主循环`
- `治理中轴`
- `任务树承接`
- `执行或学习分流`
- `结果消息回流`
- `双值结算与后果回写`
- `最终去向决定`


## 6. 稳定承载面

### 6.1 自我真值面

必须稳定承载：

- `安全值`
- `服务值`
- `物理安全`
- `风险安全`
- `待机状态`
- `时序正向步长`
- `时序反向步长`
- `服务衰减步长`

这些值的正式口径以 `自我存在特征` 为准。

### 6.2 根锚点面

必须稳定承载：

- `安全根需求`
- `服务根需求`
- `安全根任务`
- `服务根任务`
- `治理任务`
- `必要时的学习承接任务`

这里的“确保存在”不等于每轮重建，而是保证可复用、可补建、可继续推进。

### 6.3 治理中间态

每轮主循环至少要显式承载：

- `最近根层重判结果`
- `当前主需求`
- `当前主任务`
- `当前管理对象`
- `当前主方法或方法挂点`
- `当前缺口类型`
- `当前下一步去向候选`
- `最近一步治理结果`

这些字段属于 `治理承接层的稳定中间态`，不是叶子执行局部变量。

### 6.4 任务事实面

必须能稳定镜像：

- `当前步骤`
- `当前方法`
- `最近结果节点`
- `执行开始/结束记账`
- `等待学习标记`
- `学习来源字段`
- `学习任务状态字段`
- `结果回挂`

任务事实统一走 `任务虚拟存在` 与任务节点结构，不另造平行账本。

### 6.5 学习调度运行时

在当前最小循环里，学习侧至少要有：

- `学习任务表`
- `学习就绪队列`
- `学习等待表`
- `当前调度学习任务`

学习调度运行时属于承接层运行结构，不属于根裁决层。

### 6.6 本地运输面

允许存在但不得升格为正式真值：

- `尝试学习参数队列`
- `工作线程临时返回项`
- `局部排序缓存`
- `原子镜像缓存`

这些结构只服务运行时运输和效率，不可冒充正式业务真值。


## 7. 最小运行对象

自我最小循环最少要能同时看到以下对象：

- `自我存在`
- `安全根需求 / 服务根需求`
- `安全根任务 / 服务根任务 / 治理任务`
- `当前主需求`
- `当前主任务`
- `当前步骤`
- `当前方法`
- `最近实际结果`
- `学习任务实体`
- `学习目标 / 学习对象 / 学习产出 / 回流策略`
- `最近根层重判结果`
- `最近一步治理结果`


## 8. 统一主循环顺序

从规范口径推演，当前自我最小循环的统一顺序应固定为：

1. `初始化承载`
2. `进入运行态`
3. `统一消费回流消息`
4. `执行固定先天机制`
5. `根层重判`
6. `需求治理`
7. `确保根任务、治理任务和必要学习承接任务存在`
8. `装配治理上下文`
9. `提交一步治理`
10. `根据治理结果选择去向`
11. `若进入执行链，则挂任务头/步骤/结果并做执行记账`
12. `若进入学习链，则形成学习任务实体并进入学习调度运行时`
13. `统一回收执行结果与学习结果消息`
14. `按实际结果方向 vs 需求方向做双值结算`
15. `回写经验项 / 禁止项 / 治理临时记忆`
16. `刷新需求、治理镜像和下一轮去向`

这 16 步里，真正的闭环核心是：

`5 -> 9 -> 10 -> 13 -> 14 -> 16`


## 9. 各层职责

### 9.1 初始化层

负责：

- 确保自我存在
- 确保自我核心特征
- 确保根需求与根任务锚点
- 确保默认本能注册
- 确保工作者承载可用

不负责：

- 抢跑运行期治理
- 抢跑叶子执行

### 9.2 主循环层

负责：

- 消费回流消息
- 执行固定机制
- 做根层重判
- 整理需求
- 装配治理上下文
- 消费一步治理结果
- 统一双值结算
- 决定最终去向

一句话：`主循环是总协调层，不是叶子执行器。`

### 9.3 治理层

负责：

- 判当前治理域
- 判当前主缺口
- 判当前下一步去向
- 组织任务树结构
- 下发执行候选或学习候选
- 形成一步治理结果

一句话：`治理层是主循环和叶子承接层之间的正式中轴。`

### 9.4 执行层

负责：

- 在任务头下承接步骤
- 调方法执行一步动作
- 记录开始/结束记账
- 生成结果节点
- 回流实际结果消息

不负责：

- 自行改写双值
- 自行拍板继续/停止

### 9.5 学习层

负责：

- 学习任务实体化
- 学习目标判定
- 学习对象选择
- 学习产出定义
- 回流策略判定
- 学习调度运行时维护
- 学习结果回流

一句话：`学习层是主闭环内的正式分流，不是外挂补丁。`

### 9.6 结果层

负责：

- 形成实践结果消息
- 比较 `实际结果方向` 与 `需求方向`
- 结算安全值和服务值
- 回写经验项后果
- 回写禁止项后果
- 刷新治理面临时记忆


## 10. 最小合法去向

当前自我最小循环每轮末尾只允许进入以下去向之一：

- `继续治理`
- `进入执行`
- `转入学习`
- `进入等待`
- `进入待机`
- `进入停止`

禁止：

- 直接从主需求跳到方法执行且绕开治理中轴
- 执行失败后静默吞掉，不产生命中缺口
- 学习结果直接越权唤醒宿主任务而不回到主循环
- 叶子线程自行决定下一轮主闭环


## 11. 学习分流的最小正式口径

当主循环命中 `缺方法 / 缺条件 / 缺结果 / 健壮性缺口` 时，学习侧最少要形成：

- `学习任务实体`
- `学习目标`
- `学习对象`
- `学习产出摘要`
- `回流策略`
- `结果归属范围`
- `当前状态`
- `等待原因`

对于 `任务子任务生成的学习任务`，目标粒度必须至少细到以下之一：

- `在需求位 X 上新增 1 个可用方法`
- `为父任务当前步骤补齐条件`
- `为父任务当前步骤补齐结果`
- `为父任务当前路径补一个结构方法`
- `为父任务当前方法提升健壮性`

禁止只写：

- `新增可用方法`
- `学习一下`

这也是当前阶段继续完善学习链时必须守住的最小粒度。

## 13. 当前阶段定义

当前工程仍处于：

`自我内部循环建设阶段`

因此当前总纲必须额外满足：

- 外部输入不进入闭环
- 前端输入只记日志
- 验证以日志和运行状态为准
- 优先做内部自洽，不抢开外部交互

当前阶段的正式目标不是“人机闭环”，而是：

`内部自洽的最小治理闭环`


## 14. 与当前代码阶段对齐

按当前实现状态，这份大纲对应的工程阶段可理解为：

- `主循环骨架` 已具备
- `根需求 / 根任务 / 治理任务锚点` 已具备
- `治理中轴的最小返回面` 已具备
- `执行链挂任务头、步骤、结果、记账` 已具备
- `学习任务正式化` 已进入中段

  - 已有目标/对象/产出/回流最小链
  - 已有学习任务入表与等待表
  - 仍待继续补“学习对象补位、排队/执行/完成的完整运行时流转”

3. `最后再接恢复、写回和外部输入`


## 15. 最小验收标准

当以下条件同时成立时，才可以说“自我最小循环已经形成”：

- 启动后能稳定确保安全根需求、服务根需求、对应根任务和治理任务存在
- 主循环每轮都统一消费回流、执行固定机制、根层重判和治理承接
- 当前主需求、主任务、主方法、缺口类型、下一步去向都能稳定承载
- 普通执行必须经过任务头、步骤节点、结果节点和执行记账
- 执行失败、无推进、缺方法都能显性化成治理缺口
- 学习不是外挂，而是正式分流
- 学习任务至少进入学习任务表，并能区分就绪/等待
- 执行或学习结果都能回流主循环
- 双值结算不再依赖“成功/失败固定奖惩”
- 后果层至少区分：`正向推进 / 零增量 / 经验项后果 / 禁止项后果`
- 在未开放外部输入前，日志已经能完整证明这条闭环在运转

- `下一轮去向`


## 17. 收口结论

后续所有实现都统一按以下口径收口：

`自我最小循环不是“选一个需求去执行”，而是把根约束、需求、任务、学习、结果和双值重新组织成一条可持续再提交的统一治理闭环。`
