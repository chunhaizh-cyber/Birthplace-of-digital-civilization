## 3. 当前仍未真正解决的问题

### 3.1 自我真值型学习已接通最小真实影子执行

当前 `影子验证` 已具备：

1. 正式字段
2. 正式阶段名
3. 验证摘要
4. 挂在世界树根链上的纯想象镜像世界
5. 镜像自我快照
6. 首批镜像自我绑定执行路径
7. 独立验收 `H2 PASS`

当前已经明确的收口方向是：

1. 影子验证所需的执行载体，统一固定为 `挂在世界树根链上的纯想象镜像世界`
2. 该镜像世界可以随时建立、删除或复用
3. 镜像世界中的内部数据只能影响其内部规则，不得直接带回现实世界
4. 镜像世界允许回带到现实治理链的，只能是：
   - 方法资产
   - 待验证方法
   - 与方法验证有关的证据摘要

当前仍保留的增强点是：

1. 现有一部分本能动作并不只读写 `输入场景/输出场景`，还会直接读写 `世界树.自我指针`
2. 当前最小镜像执行只覆盖首批动作，尚未形成“所有自我绑定动作都能走镜像上下文”的通用桥
3. 后续若继续扩覆盖，仍应坚持：
   - 不在学习线程里直接切换全局 `世界树.自我指针`
   - 继续走显式镜像自我执行上下文
   - 或镜像自我专用执行桥
   - 或把直接依赖 `世界树.自我指针` 的本能动作改成可注入宿主自我

当前实现本质上仍属于：

`最小真实影子执行已接通 + 后续动作覆盖仍待扩展`

因此 `H2` 不再是当前学习闭环主阻塞。

### 3.2 现实影响型学习仍未实现真实受控微试探隔离

当前 `受控微试探` 已具备：

1. 约束字段
   - 作用对象
   - 幅度
   - 时长
   - 次数
   - 停止条件
2. 正式验证门
3. 正式摘要
4. 任务管理任务侧受控判定入口
5. 真实单次试探执行路径
6. 最小综合分口径
7. `受控` 与 `试探` 的概念分层
8. 最小受控白名单
9. 最小时长/幅度限流
10. 停止条件命中摘要
11. 对象级结果隔离校验
12. 强制中止摘要
13. 失败后的两级最小补偿链
14. 多步强可中止执行器

但尚未真正具备：

1. 更完整的对象白名单与对象级风险隔离
2. 面向不可逆现实影响的完整补偿链
3. 能稳定覆盖多步真实试探的正式验收样本
4. `整次试探结果状态集合` 的正式收集器
5. `中间产生且最终保留下来的结果状态` 入集合规则
6. 基于结果状态集合的正式综合分判定

当前已经新增并验证通过的一步是：

1. `自我_确认收到` 这类不需要镜像自我绑定的现实影响型方法，已经能在独立验收入口里走通真实受控微试探
2. 受控边界、结果状态集合、综合分与正式回流建议，已经能形成稳定输出
3. 为防止来源任务权重过大导致综合分溢出，当前已补：
   - 受控权重钳制
   - 综合分安全累加
4. 当前最小真实控制链已补：
   - 动作白名单检查
   - 幅度/时长限流
    - 对象级结果隔离
   - 停止条件命中判定
   - 强制中止摘要
   - 最小保守态补偿
   - 风险上升/隔离失败时的升级补偿
   - 多步强可中止执行

因此当前真实状态应理解为：

`真实受控微试探最小路径已接通，且已具备最小控制链与多步强可中止执行；但更广对象白名单、不可逆现实影响的完整补偿链和稳定覆盖多步场景的验收样本仍未完成`

因此当前实现仍属于：

`受控试探约束显式化`

而不是真正的：

`受控试探执行隔离`

并且必须继续固定：

- `受控` 应由任务管理任务先完成治理判定
- `试探` 必须是真实执行，而不是新的影子占位
- 受控微试探的可接受性，不得只看最后一步返回状态
- 整次试探过程中中间产生、且最终保留下来的结果状态，也必须纳入综合评判

### 3.3 方法资产正式账本写回已接通，但仍是最小正式落账

当前已完成：

1. 学习提交阶段不再只写 `已写入方法资产=true` 摘要，而会正式写入：
   - `方法条件节点`
   - `方法结果节点`
   - `方法虚拟存在`
   - `方法内部状态场景`
2. 方法侧会同步最小正式资产：
   - 条件场景模板与条件状态列表
   - 结果场景模板与结果状态列表
   - 方法样本数 / 新结果次数 / 最近输入条件主键
   - 方法虚拟存在上的最近错误码、最近成功标记、最后调用时间
3. 学习提交阶段若正式写回失败，当前学习任务会直接进入失败/回滚，而不再把“摘要存在”冒充成“账本已落”

当前仍未完整解决：

1. 条件节点/结果节点仍按“最小模板场景 + 首个匹配节点复用”处理，还没有完整的条件桶/结果桶匹配规则
2. `禁用条件` 还没形成正式写回链
3. `验证证据` 目前优先写到方法内部状态场景，还没有形成更细的桶级证据账本
4. `稳定度增量` 目前主要复用 `样本数 / 稳定出现次数 / 可靠度` 这组既有账本，还没有新增独立“稳定复现桶”口径

因此当前应理解为：

`方法资产正式账本写回已接通，但还属于最小正式落账版本`

### 3.4 外层正式回流判定已接通，但还是最小正式裁决表

当前已完成：

1. 学习线程给出的 `建议直接回流 / 建议匹配后回流 / 建议仅沉淀资产` 不再被直接当成最终结论
2. 自我主循环消费学习回流时，会重新经过一次正式裁决
3. 正式裁决当前至少会联合判断：
   - 方法资产是否已正式写回
   - 学习任务是否真正成功完成
   - 当前主循环是否处于待机/停止保护
   - 受控微试探综合分是否转正
   - 来源任务是否仍处于可接受回流状态

当前仍未完整解决：

1. 匹配后回流仍主要面向来源任务自身，没有真正展开到“同类任务族匹配”
2. 正式裁决结果还没有写成独立的可查询回流放行表
3. 回流失败时还缺少更细的补偿/降级路径

因此当前应理解为：

`外层正式回流裁决已接通，但还是最小正式裁决表`
