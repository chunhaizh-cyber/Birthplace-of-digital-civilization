# 20260403 阶段 B 学习子任务正式化详细计划

## 1. 目标

本计划用于把当前 `等待学习 -> 尝试学习参数队列 -> 消费一个队列项` 的最小兜底链，
升级成符合规范的 `学习子任务治理链`。

本阶段只做 `内部学习承接正式化`，不做外部输入接入，不做快照恢复，不做元层学习写回总整合。


## 2. 规范边界

本阶段直接服从以下规范：

- [1208_学习任务最小状态面与来源字段规范.md](D:/鱼巢/规范/1208_学习任务最小状态面与来源字段规范.md)
- [1215_学习任务目标-对象-产出-回流规范.md](D:/鱼巢/规范/1215_学习任务目标-对象-产出-回流规范.md)

先固定 8 条边界：

1. `自我主循环` 只负责外层协调，不负责把学习目标、学习对象、学习产出、回流策略写死在自己体内。
2. `学习任务` 不属于第三根任务，仍属于 `任务管理内部治理子任务`。
3. `学习承接链` 是缺口补齐承载者，不允许继续散落成 `自我类` 里的若干临时 if/queue 分支。
4. `学习结果` 首先属于 `内部治理结果`，不能直接越权唤醒原任务。
5. `回流是否允许正式生效` 仍由外层主循环重判决定。
6. `来源字段 / 目标字段 / 结果字段 / 回流字段` 必须有稳定承载面，不能只留在局部变量里。
7. `任务事实镜像` 仍统一走 [任务类.ixx](D:/鱼巢/任务类.ixx) 的任务虚拟存在特征，不另造平行账本。
8. 当前阶段继续封禁外部输入，验证只看日志。

## 4. 模块落点总原则

本阶段代码落点固定如下。

### 4.1 [自我类.ixx](D:/鱼巢/自我类.ixx)

只允许放：

- 主循环外层协调
- 根层重判后的学习缺口显性化
- 学习原始上下文装配
- 调用学习模块得到学习任务实体/学习决议/学习回流摘要
- 学习结果回到主循环后的统一消费
- 外层日志

不再继续扩张：

- 学习类型判定细节
- 学习对象选择细节
- 学习目标生成细节
- 回流策略默认表
- 学习实体去重/调度排序规则

一句话：`自我类只做调度总控，不做学习业务规则库。`

### 4.2 [学习类.ixx](D:/鱼巢/学习类.ixx)

作为本阶段主改造模块，负责承载：

- 学习任务实体定义

- 学习任务去重键、合并键
- 学习调度选择
- 学习对象选择规则
- 学习目标判定
- 学习产出定义
- 学习回流策略判定
- 学习结果归属范围判定
- 学习运行状态与等待原因标签

本阶段新增/收口逻辑优先落这里，不再继续塞到 `自我类.ixx`。

一句话：`学习类是学习治理规则层，不是主循环层。`

### 4.3 [任务类.ixx](D:/鱼巢/任务类.ixx)

- `结果归属范围`

这些字段的“事实写入接口”落这里或复用这里；
但字段值的“业务判定逻辑”不落这里，仍归 `学习类.ixx`。

一句话：`任务类只做统一记账和镜像，不做学习判定。`

### 4.4 [方法类.ixx](D:/鱼巢/方法类.ixx)

继续只负责：

- 候选方法召回

本阶段不把学习调度、学习目标、学习回流塞进 `方法类.ixx`。

一句话：`方法类是能力对象层，不是学习治理总控层。`

### 4.5 [自我本能方法_学习兜底模块.ixx](D:/鱼巢/自我本能方法_学习兜底模块.ixx)

继续只负责：

一句话：`这里是动作承接层，不是目标裁决层。`

### B1. 冻结学习链模块边界

目标：

- 明确哪些学习逻辑留在 [自我类.ixx](D:/鱼巢/自我类.ixx)
- 哪些逻辑迁到 [学习类.ixx](D:/鱼巢/学习类.ixx)

- 在 [自我类.ixx](D:/鱼巢/自我类.ixx) 中识别以下函数属于“应下沉到学习模块的业务规则”：

- `学习类` 里形成学习治理入口

### B2. 把等待学习正式映射成学习任务实体

目标：

- 不再让 `等待学习` 只是一条任务标记
- 把它提升成正式 `学习任务实体`

- [学习类.ixx](D:/鱼巢/学习类.ixx)

实施点：

- `自我类` 装配 `结构_定向学习任务原始上下文_v0`
- 来源字段至少补齐：
  - 来源根任务头指针
  - 来源宿主任务头指针
  - 来源步骤指针
  - 来源方法指针
  - 来源结果指针
  - 来源方法需求位
  - 回流目标任务头指针
  - 回流目标步骤指针
- 调 [学习类.ixx](D:/鱼巢/学习类.ixx) 的 `构建定向学习任务实体_v0()`
- 把实体同步进学习调度运行时，而不是只保留 `roots_.学习` 计数

交付：

- 每个等待学习都能映射到一条正式学习任务实体
- 日志中能看到学习任务实体创建

### B3. 接通 `目标 -> 对象 -> 产出 -> 回流` 四段链

目标：

- 让学习不再只是“拿到参数就试”

主落点：

- [学习类.ixx](D:/鱼巢/学习类.ixx)

实施点：

- 以 `学习任务实体` 为输入，补一条统一入口：
  - 先选学习对象
  - 再判学习目标
  - 再定义本轮产出
  - 最后判定回流策略
- 复用已有函数：
  - `补全基础学习对象规则_v0()`
  - `判定学习目标_v0()`
  - `取学习对象默认产出建议()`
  - `取学习回流策略默认值_v0()`
  - `取学习结果归属范围默认值_v0()`
- 本阶段允许“产出定义”先落最小版，但必须显式有字段：
  - 产出摘要
  - 回流策略
  - 结果归属范围
  - 是否允许直接回流

交付：

- 学习请求有完整四段链
- 无对象时也能明确记录“对象缺失”，而不是静默返回

### B4. 学习任务事实镜像正规化

目标：

- 让学习子任务的关键字段落到统一任务事实，而不是只活在 self 私有结构里

- [任务类.ixx](D:/鱼巢/任务类.ixx)

  - `尝试学习_来源方法指针`
  - `尝试学习_来源结果指针`
  - `尝试学习_来源方法需求位`
  - `学习任务_学习类型`
  - `学习任务_回流策略`
  - `学习任务_结果归属范围`
  - `学习任务_当前状态`
  - `学习任务_等待原因`

交付：

- 学习任务从日志和任务事实两边都可追溯

### B5. 学习承接消费与回流分离

目标：

- 把“学习承接执行”和“外层是否接回当前闭环”分开

主落点：

- [自我类.ixx](D:/鱼巢/自我类.ixx)
- [学习类.ixx](D:/鱼巢/学习类.ixx)

实施点：

- `消费一个尝试学习参数队列项_()` 继续承担当前最小执行器角色
- 但它执行后返回的结果，不直接等于“允许唤醒原任务”
- 先把学习结果写成：
  - 本轮学习结果摘要
  - 学习产出摘要
  - 回流策略
  - 结果归属范围
  - 是否建议直接回流
- 再由 `TickOnce_()` 外层统一做：
  - 根层重判
  - 上下文刷新
  - 决定是否允许正式回流

交付：

- 学习结果先成为承接层结果，再回到外层裁决

### B6. 日志验收固定化

目标：

- 不靠猜测判断学习链是否成立

主落点：

- [自我类.ixx](D:/鱼巢/自我类.ixx)

- `回流策略判定`
- `学习回流结果`

交付：
