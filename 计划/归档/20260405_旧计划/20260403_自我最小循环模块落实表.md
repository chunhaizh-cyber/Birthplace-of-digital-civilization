# 20260403 自我最小循环模块落实表

## 1. 目标

本表用于把 [20260403_自我最小循环实现大纲.md](D:/鱼巢/计划/20260403_自我最小循环实现大纲.md) 进一步拆成：

- `规范最终应落在哪个模块`
- `当前代码实际上落在哪个模块`
- `当前已经做到什么程度`
- `下一步应继续改哪里`

它不替代总纲，也不替代阶段计划；它是后续做代码时的“模块级施工对照表”。

自我侧模块边界的最新口径，统一以 [20260404_自我相关模块细分与实施收口计划.md](D:/鱼巢/计划/20260404_自我相关模块细分与实施收口计划.md) 为准。


## 2. 使用方法

后续每做一刀改造，都应优先回答四件事：

1. 这项能力在规范上属于哪一层
2. 它最终应落在哪个模块
3. 当前代码是否只是临时落点
4. 这一刀做完后，是否把边界收得更清楚

若一项实现找不到对应条目，应先补表，再补代码。


## 3. 落点总原则

### 3.1 最终落点原则

- `自我相关模块` 负责自我侧正式能力；当前以 [自我类.ixx](D:/鱼巢/自我类.ixx) 为总控入口，后续新增实现统一收口到以 `自我_` 开头的自我相关模块。
- [自我类.ixx](D:/鱼巢/自我类.ixx) 只保留主循环总协调、最终去向、线程拥有权与自我侧真值拥有权。
- `学习类.ixx` 负责学习任务实体、学习目标/对象/产出/回流规则、学习调度运行时。
- `任务类.ixx` 负责任务树结构、任务事实镜像、执行记账、等待学习与结果回挂。
- `任务执行类.ixx` 负责叶子执行和执行失败转学习的执行器级承接。
- `方法类.ixx` 负责候选方法召回、尝试学习参数生成和方法动作承接。
- `需求类.ixx` 负责需求登记、需求状态比较、需求方向基底和需求头承接。
- `学习写回模块_v0.ixx` 暂不作为最小循环主施工面，只保留后续接入位。

### 3.2 当前阶段原则

- 当前允许 [自我类.ixx](D:/鱼巢/自我类.ixx) 继续作为总控入口，但新增自我实现细节不再默认继续堆回这里，而是优先落到 `自我_*.ixx`。
- 当前不开放外部输入，因此 `鱼巢控制面板.cpp` 不属于自我最小循环主链，只是宿主与日志界面。
- 当前一切验证优先依赖日志，不以交互能力作为验收前提。

## 6. 当前最该继续补的施工序列

按这张落实表，后续最优先顺序固定为：

1. `P0：自我相关模块边界冻结`
   - 明确 `自我类.ixx` 只保留总控、线程与最终拍板
   - 明确新增自我实现统一落到 `自我_*.ixx`

2. `学习类.ixx`
   - 补“对象缺失时的最小对象补位”
   - 补学习任务 `等待 -> 就绪 -> 排队 -> 执行 -> 完成/失败` 的完整状态流转

3. `自我_学习承接模块.ixx`
   - 把学习结果回流逐步收成更正式的结果消息消费
   - 不再继续内联新的学习业务规则

4. `自我_结果治理模块.ixx + 自我_治理记忆模块.ixx`
   - 正规化多特征双值结算
   - 正规化经验项/禁止项/治理临时记忆

5. `任务类.ixx`
   - 只在需要新的稳定事实镜像字段时补字段，不主动扩业务逻辑

6. `任务执行类.ixx`
   - 在确定主闭环执行路径后，再决定是否继续保留双轨最小执行链

7. `学习写回模块_v0.ixx`
   - 放到内部闭环稳定之后再接


## 7. 当前明确不要做的事

为避免再次边界漂移，当前明确不要做：

- 不把外部输入解封
- 不把 `学习参数队列` 当学习总控
- 不把学习目标/对象/回流规则继续堆回 `自我类.ixx`
- 不新增无 `自我_` 前缀的平行“自我小类”
- 不把任务事实镜像逻辑散落到各个模块私写
- 不在还没收口内部闭环前就跳去做元层写回总整合


## 8. 最小验收方式

后续每做一刀，都至少要能回答：

- 这刀让哪一个模块更接近“规范最终落点”
- 这刀有没有把临时落点继续往正式落点收
- 日志里是否能看到新增闭环证据

当前最小日志验收仍固定看：

- `一步治理结果`
- `学习子任务创建`
- `学习目标判定`
- `学习任务入表`
- `学习调度结果`
- `学习回流结果`
- `双值结算摘要`
- `经验项 / 禁止项后果`


## 9. 收口结论

后续做代码时，统一以这张表为准：

`先判断能力属于哪一层，再判断最终落在哪个模块；如果当前只是临时落点，就让下一刀把它继续往正式落点收。`
