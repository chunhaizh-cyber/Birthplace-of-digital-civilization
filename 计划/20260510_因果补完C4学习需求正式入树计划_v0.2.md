## 3. 入树范围

```text
第一版只承认以下四类学习需求触发。
其他因果补完和外界探索需求仍后置。
```

### 3.1 无确定候选方法

```text
触发：
    任务筹办按目标特征类型查找方法能力；
    没有方法结果能力确定命中。

派生需求：
    需求类型 = 学习方法需求
    目标宿主 = 方法仓库 / 待学习方法头
    目标特征类型 = 可改变目标特征类型的方法存在性
    目标状态 = 存在

因果句式：
    要父任务就绪，需要一个能改变目标特征 F 的方法。
    因为当前没有确定候选方法，
    所以派生学习需求：存在一个可改变 F 的方法。
```

### 3.2 候选方法不可用

```text
触发：
    候选方法结果能力命中；
    但方法状态未达到可用。

派生需求：
    目标宿主 = 候选方法虚拟存在
    目标特征类型 = 方法状态
    目标状态 = 可用

限制：
    不得在缺口不明确时直接派生“动作骨架状态”。
    只有回执明确证明缺执行入口，才派生执行入口相关需求。
```

### 3.3 明确缺执行入口

```text
触发：
    筹办回执明确候选方法缺执行入口或缺动作骨架。

派生需求：
    目标宿主 = 候选方法虚拟存在
    目标特征类型 = 执行入口状态 / 动作骨架状态
    目标状态 = 已形成
```

### 3.4 明确缺条件结果对

```text
触发：
    候选方法已有方法头或动作；
    但缺有效条件结果对。

派生需求：
    目标宿主 = 候选方法虚拟存在
    目标特征类型 = 条件结果对状态
    目标状态 = 至少一组有效条件结果对已建立

后续闭合：
    尝试学习
    建立条件结果对
    提交方法状态变化
```

## 4. 筹办条件需求边界

```text
参数 / 条件补齐需求第一版不归入学习需求。

例如：
    缺参数 P
        -> 需求：参数 P 已获得 / 已绑定

    方法条件 C 不满足
        -> 需求：条件 C 成立

它们可以支撑筹办，但不作为“学习需求正式入树”的验收项。
```

## 5. 权重来源

```text
学习需求权重来自触发它的父需求 / 父任务剩余压力。
权重只作为调度压力。
不直接作为安全 / 服务结算上限。

禁止：
    I64_MAX 作为真实服务结算值。

轻量结算口径：
    叶子任务结算安全 / 服务值；
    叶子剩余权重减少；
    父任务剩余权重同步减少；
    父任务不重复结算子任务已结算部分。
```

## 6. 父需求挂接策略

```text
学习需求挂在触发它的父需求下面。

挂接规则：
1. 必须有触发父需求。
2. 父需求不能是单纯结构根。
3. 子需求必须有明确目标宿主、目标特征类型、当前状态 / 目标状态、满足关系。
4. 子需求必须带派生归因。
5. 不直接挂到根需求，也不单独挂成全局学习树。
```

## 7. 禁止项

```text
1. 不允许从测试夹具直接执行入树。
2. 不允许复制父需求目标作为学习需求目标。
3. 不允许缺目标宿主 / 目标特征类型 / 当前状态 / 目标状态 / 满足关系时入树。
4. 不允许同时生成外界探索需求。
5. 不允许恢复旧学习主需求链。
6. 不允许把“候选方法不可用”默认派生成“方法动作骨架状态”。
7. 不允许把参数 / 条件补齐需求误归为学习需求。
```

## 8. 硬回归剧本

### 剧本 1：无候选方法 -> 学习方法 -> 父任务就绪

```text
父需求：目标宿主 X 的特征 F 达到目标状态。
父任务筹办：按方法能力查找，找不到候选。
派生子需求：存在可改变 F 的方法。
子任务：学习 / 生成方法头。
方法结果能力形成。
父任务待重筹办。
父任务再次筹办，候选方法进入候选集。
单候选直接绑定。
参数可绑定。
任务进入就绪。
```

### 剧本 2：候选方法不可用 -> 方法状态可用 -> 父任务就绪

```text
父任务筹办查到候选方法 M。
M 结果能力命中目标特征。
M 方法状态不是可用。
派生子需求：M.方法状态 = 可用。
学习任务补齐条件结果对。
提交方法状态变化：M = 可用。
父任务重筹办。
父任务就绪。
```

### 剧本 3：缺执行入口 -> 补齐入口 -> 父任务就绪

```text
候选方法 M 结果能力命中。
筹办回执明确 M 缺执行入口。
派生需求：M.执行入口状态 = 已形成。
定向学习 / 补齐方法结构。
入口形成后，若满足可用条件，提交 M.方法状态 = 可用。
父任务重筹办。
父任务就绪。
```

### 剧本 4：参数缺失 -> 参数子需求 -> 父任务就绪

```text
候选方法 M 可用。
方法条件满足。
但参数 P 无法绑定。
派生子需求：参数 P 已获得 / 已绑定。
子任务完成参数获取。
父任务重筹办。
配对参数成功。
可执行输入参数场景形成。
父任务就绪。
```
