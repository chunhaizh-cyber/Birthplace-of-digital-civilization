# 计划目录归档一致性清理计划 v0.1

## 1. 目标

让计划目录状态和计划正文状态保持一致：

```text
已经验收的计划进入 完成项。
尚未开始或仍有主体工作未完成的计划留在 待完成项。
剩余工作不得继续塞回已验收计划，应拆成新的待完成计划。
```

## 2. 清理范围

```text
计划/待完成项
计划/完成项
已写入总体验收记录但仍停留在待完成项的计划
被新计划覆盖但未标记状态的旧计划
重复表达同一目标的计划
```

## 3. 子计划

### P1 已验收计划归档

动作：

```text
扫描待完成项中带有验收记录、完成记录、收口记录的计划。
核对是否已有独立后续计划承接剩余工作。
满足条件后移动到完成项。
```

验收：

```text
已验收计划不再停留在待完成项。
待完成项中保留的计划都有仍待执行的主体任务。
```

### P2 重复计划合并

动作：

```text
扫描同名或近似主题计划。
保留最新有效计划作为主计划。
旧计划补充被承接说明，或移动到完成项作为历史记录。
```

验收：

```text
待完成项中同一主题只有一个主计划。
历史计划能追溯到新的承接计划。
```

### P3 建立计划索引

动作：

```text
建立计划索引文档。
记录计划名、状态、主提交号、验收记录、后续承接计划。
```

验收：

```text
从计划索引能看出当前主线、已完成计划和后续计划。
```

## 4. 完成口径

```text
完成项 / 待完成项 与计划正文状态一致。
已完成计划有验收依据。
剩余工作有独立后续计划承接。
```
