## 2. 删除原则

1. 先删除核心类型和核心接口，不保留兼容别名。
2. 本能动作函数不接收任务、需求、线程、控制面板、世界树、请求 ID。
3. 任务信息只留在任务管理界面线程或执行桥内部。
4. 方法参数必须通过输入参数场景表达。
5. 方法结果通过返回结构和输出结果场景表达。
6. 本能动作管理类不负责方法虚拟存在评价更新。
7. 本能动作管理类不负责创建需求、推进任务、解释任务状态。

## 3. 目标接口

### 3.1 本能动作函数

替换为：

```cpp
using 本能动作函数 = std::function<结构_本能动作步骤结果(
    方法类::节点类* 方法首节点,
    场景节点类* 输入参数场景,
    场景节点类* 输出结果场景)>;
```

参数含义：

| 参数 | 含义 |
| --- | --- |
| `方法首节点` | 当前被执行的方法 |
| `输入参数场景` | 方法条件参数和输入值所在场景 |
| `输出结果场景` | 方法写入或承载结果的场景 |

### 3.2 执行入口

新增并作为唯一执行入口：

```cpp
bool 执行功能函数(
    枚举_本能动作ID 动作ID,
    方法类::节点类* 方法首节点,
    场景节点类* 输入参数场景,
    场景节点类* 输出结果场景,
    结构_本能动作步骤结果* 输出 = nullptr) const noexcept;
```

删除：

- `运行(...)`
- `运行_兼容(...)`
- `注册_兼容(...)`

### 3.3 可调用检查

新增：

```cpp
bool 检查是否可调用(
    枚举_本能动作ID 动作ID,
    方法类::节点类* 方法首节点,
    场景节点类* 输入参数场景,
    结构_本能动作步骤结果* 输出 = nullptr) const noexcept;
```

检查项：

1. 动作 ID 不是 `未定义`。
2. 动作函数已注册。
3. 方法首节点不为空。
4. 方法首节点类型是 `枚举_方法节点种类::方法首节点`。
5. 方法头存在可执行动作入口：
   - `动作句柄.类型 == 枚举_动作句柄类型::本能函数ID`，并且本能 ID 与动作 ID 一致或可解析为注册动作。
   - `动作名` 存在时可反查到注册动作。
6. 输入参数满足条件：
   - 无条件索引且无条件场景时，视为不需要输入参数。
   - 有条件索引或条件场景时，必须有输入参数场景。
   - 参数匹配使用现有 `本能动作类::场景_模式匹配`。

不检查：

- 任务是否应该推进。
- 需求是否满足。
- 任务队列和工作线程负载。
- 方法稳定性、成功次数、失败次数。

## 4. 结果结构收口
