# 2026-05-05 任务主信息类字段瘦身清理计划 v0.1

## 1. 计划目标

本计划目标是清理 `D:/鱼巢/任务主信息类.h` 中不再符合新任务模型的字段。

新任务模型固定为：

```text
需求树是主结构。
每个需求节点对应一个任务头。
任务只有筹办和执行两个核心阶段。
筹办阶段选择自我能力值。
执行阶段根据自我能力值配对参数并调用本能函数。
缺口生长为子需求。
```

因此，`任务主信息类` 最终应只保留：

```text
任务身份
对应需求
任务虚拟存在
当前阶段
选中自我能力值
配对结果缓存
可执行输入缓存
最近运行事实缓存
任务状态镜像
调度优先级镜像
执行统计镜像
```

旧的步骤节点、结果节点、任务树父子链、方法首节点长期缓存和旧学习任务字段，应删除或降级为兼容待删字段。

## 2. 上位规则

1. 不用字段删除掩盖真实断点。
2. 字段删除前必须先找到所有读写点。
3. 可以先暂停并提出结构问题，不允许用默认成功、白名单或兜底通过遮蔽问题。
4. 语义字段不得用新枚举或新结构体替代；长期语义应迁移到抽象特征和实例特征。
5. 结构体字段只允许作为运行期缓存、消息包、局部算法缓存或兼容镜像。
6. 方法首节点只能作为运行期缓存，不能作为任务长期权威语义。
7. 删除字段后必须通过编译，并用日志模块确认新的断点和旧断点变化。

## 8. 暂停条件

遇到以下情况必须暂停并提出问题：

```text
1. 某字段同时被新主链和旧兼容链依赖，无法判断权威来源。
2. 删除字段会导致需求树无法生成任务。
3. 删除字段后只能通过默认成功或特殊放行恢复运行。
4. 字段语义找不到对应抽象特征或实例特征承载位置。
5. 工作线程必须重新创建旧步骤节点才能推进。
```
