# 20260503 自我线程瘦身与五线程分责迁移计划 v0.1

## 1. 计划目标

本计划用于把当前“重型自我线程 + 任务管理任务中心”的过渡结构，迁移为：

```text
自我线程轻调度
需求树两阶段任务
控制面板摘要线程只读展示
自检线程只读发现缺口
任务管理工作线程推进单个需求任务
```

最终目标是：

```text
自我线程负责调度生命方向；
摘要线程负责观察；
自检线程负责发现缺口；
工作线程负责推进一个需求任务；
需求树负责生长；
自我能力枚举负责执行定位。
```

## 2. 当前已具备基础

当前代码已经具备以下基础，不应推倒重来：

| 已有基础 | 当前位置与价值 |
| --- | --- |
| 自我线程主循环骨架 | 已有固定 Tick 步骤、消息冻结、治理帧和派发决议。 |
| 需求节点对应任务 | 需求可创建或复用任务头，并写入对应需求与当前阶段。 |
| 筹办 / 执行两步壳 | 已有 `自我_任务筹办` 与 `自我_任务执行一步`。 |
| 派生需求入树 | 工作线程回流可生成子需求并挂接父需求。 |
| 任务虚拟存在阶段回写 | 已开始写入 `当前阶段`、`缺口类型`、`选中自我能力值`。 |
| 方法结构自检雏形 | 执行前已能记录方法自检结果和方法结构缺口。 |
| 尝试学习候选补全 | `尝试学习` 已能补候选能力包、形参包、结果包、条件结果对和试运行限制包。 |

## 3. 当前主要差距

| 差距 | 说明 |
| --- | --- |
| 自我线程仍过重 | 自我线程仍保存大量控制面板级快照和深层治理摘要。 |
| 控制面板摘要线程缺失 | 控制面板仍依赖自我线程或主线程侧摘要接口。 |
| 自检线程缺失 | 结构审计仍混在自我线程内部逻辑中。 |
| 消息路由过粗 | 当前消息路由仍偏向全部上浮自我线程。 |
| 任务管理任务仍偏中心化 | 需求树尚未完全替代任务管理任务中心。 |
| 学习仍有特殊任务痕迹 | 学习应收敛为缺口需求可查到的 `尝试学习` 能力。 |
| 任务创建仍可能预建方向方法头 | 主链应改为任务筹办时查找方法。 |
| 长期语义仍有字符串和结构体缓存 | 缺口类型、阶段、方向等应逐步迁移为实例特征。 |

## 4. 目标线程结构

目标线程结构为五类：

| 线程 | 职责 |
| --- | --- |
| 自我线程 | 接收消息、整理需求、确认方向、生成禁止项、生成任务、派发任务、控制任务、调度任务。 |
| 任务管理任务界面线程 | 接收用户输入，发送控制消息，展示摘要线程生成的控制面板快照。 |
| 任务管理任务工作线程 | 推进一个需求节点对应任务的筹办或执行。 |
| 控制面板摘要线程 | 只读读取世界树、需求树、任务树、方法树、运行事实树，生成控制面板摘要快照。 |
| 自检线程 | 只读检查世界树、需求树、任务树、方法树、特征系统，输出自检报告和缺口候选。 |

硬边界：

```text
自我线程负责“怎么活”；
摘要线程负责“怎么看”；
自检线程负责“哪里缺”；
工作线程负责“干一步”。
```

## 5. 总阶段计划

| 阶段 | 名称 | 目标 | 结果 |
| --- | --- | --- | --- |
| P0 | 基线冻结 | 明确当前可用链路，不破坏已有运行。 | 保留消息归并、需求派生、两步壳、工作线程桥接。 |
| P1 | 摘要外移 | 新增控制面板摘要线程。 | 控制面板不再要求自我线程深扫树。 |
| P2 | 自检外移 | 新增自检线程。 | 自检只读发现缺口，自我线程负责把缺口变需求。 |
| P3 | 消息分流 | 扩展消息协议和路由。 | 控制面板刷新、自检报告、工作线程回流各走正确路径。 |
| P4 | 自我线程瘦身 | 自我线程只做方向、禁止项、需求、任务调度。 | 大快照和显示统计从自我线程剥离。 |
| P5 | 任务创建收敛 | 需求节点只创建对应任务，不提前创建方法头。 | 方法选择全部交给筹办阶段。 |
| P6 | 两阶段任务闭合 | 筹办查能力，执行配参数与本能函数。 | 任务长期保存 `选中自我能力值`。 |
| P7 | 学习收敛 | 学习任务改为缺口需求 + 尝试学习能力。 | `尝试学习` 成为方法结构缺口的处理能力。 |
| P8 | 特征语义迁移 | 长期状态迁移为实例特征。 | 阶段、缺口、能力、方法结构状态不再只靠字符串或结构体。 |
| P9 | 验收固化 | 建立摘要、自检、任务推进验收标准。 | 可稳定观察、可回归、可继续拆旧模块。 |

```cpp
export bool 请求刷新控制面板摘要();
export 控制面板摘要线程::结构_控制面板摘要快照 读取最新控制面板摘要快照();
```

### 6.4 快照结构

```cpp

struct 结构_控制面板摘要快照 {
    std::uint64_t 快照序号 = 0;

    std::string 自我线程摘要{};
    std::string 需求树摘要{};
    std::string 任务树摘要{};
    std::string 方法树摘要{};
    std::string 运行事实摘要{};
    std::string 自检摘要{};

    std::size_t 活跃需求数 = 0;
    std::size_t 等待子需求数 = 0;
```

### 6.5 硬规则

```text

不得调用任何 取或创建、确保、补齐、注册、写入、升格、修正 接口。
```

### 6.6 验收

```text
控制面板能从摘要线程读取快照；
自我线程不再为控制面板生成大摘要。
```


```text

```

## 11. 子计划六：筹办阶段闭合

### 11.1 目标

筹办只找“我会什么”。

### 11.2 标准流程

```text
读取需求条件包
读取查找限制包
调用 查找方法
得到 选中自我能力值
写入任务虚拟存在
切换任务阶段 = 执行
```

### 11.3 失败流程

```text
生成方法缺口 / 方法结构缺口需求
任务阶段 = 等待子需求或阻塞
```

### 11.4 验收

```text
筹办结果的权威输出是 自我能力枚举值；
方法首节点只作为运行期缓存。
```

## 12. 子计划七：执行阶段闭合

### 12.1 目标

执行验证“这次能不能做”。

### 12.2 标准流程

```text
读取选中自我能力值
按能力值解析方法首节点
执行前方法结构自检
配对参数
生成可执行输入参数场景
调用本能函数
检查需求是否满足
满足则回传
失败则生成条件 / 结果缺口需求
```

### 12.3 硬规则

```text
执行阶段不重新查方法。
执行阶段不按名称理解方法。
执行阶段不得默认补参数。
```


## 15. 建议执行顺序

1. 控制面板摘要线程。
2. 自检线程。
3. 消息协议与路由。
4. 自我线程最小状态快照。
5. 任务创建不再提前绑定方法头。
6. 筹办写入 `选中自我能力值`。
7. 执行按能力值解析方法并配对参数。
8. 学习任务改为缺口需求 + 尝试学习。
9. 长期语义特征化。
10. 清理旧任务管理中心逻辑。

## 16. 验收总线

本计划完成时，应满足：

1. 控制面板摘要线程可独立生成摘要快照。
2. 自检线程可独立生成缺口候选报告。
3. 自我线程接收自检报告后能派生需求。
4. 任务创建不再主链预建方向方法头。
5. 筹办阶段写入 `选中自我能力值`。
6. 执行阶段按能力值解析方法首节点。
7. 方法结构缺口由 `尝试学习` 处理。
8. 任务阶段、缺口类型、能力值均可从任务虚拟存在实例特征读取。
9. 旧任务管理任务只作为兼容壳，不再作为需求树上方的中心管理任务。
