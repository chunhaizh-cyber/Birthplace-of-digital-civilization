## 1. 总目标

把本能动作函数返回信息从 `结构_本能动作步骤结果` 收敛为：

```text
运行结果：返回抽象特征的实例
执行结果：写入输出结果场景
```

任务管理任务或任务管理界面线程读取这两类信息后，统一推进任务状态、需求满足、派生需求和父任务唤醒。

本轮不新增新的混合结构体，不恢复执行上下文，不把任务和需求字段放入本能动作管理类。

## 2. 当前问题

| 问题 | 当前表现 | 修订方向 |
| --- | --- | --- |
| 返回结构混合 | `结构_本能动作步骤结果` 同时包含成功码、质量、成本、结果场景、动作动态和备注 | 删除核心接口中的混合结果结构 |
| 运行结果不够语义化 | 调用方主要看 `成功码` | 改为返回运行结果抽象特征实例 |
| 执行结果入口重复 | 返回结构里保存结果场景指针 | 调用方直接传入并持有输出结果场景 |
| 任务管理解释分散 | 执行桥和任务管理工作线程读取结构字段推导缺口 | 统一由任务管理侧读取运行结果实例和输出结果场景 |
| 备注可能被误用 | `备注` 容易成为结构依据 | 备注只进入日志事件，不参与权威判断 |

## 3. 目标接口

### 3.1 本能动作函数

目标：

```cpp
using 本能动作函数 = std::function<特征节点类*(
    方法类::节点类* 方法首节点,
    场景节点类* 输入参数场景,
    场景节点类* 输出结果场景)>;
```

规则：

1. 返回 `特征节点类*`，表示运行结果实例。
2. 执行结果全部写入 `输出结果场景`。
3. 函数内部不得读取任务、需求、控制面板或线程调度状态。
4. 函数内部不得创建任务或需求。

### 3.2 本能动作管理类

目标：

```cpp
特征节点类* 检查是否可调用(
    枚举_本能动作ID 动作ID,
    方法类::节点类* 方法首节点,
    场景节点类* 输入参数场景,
    场景节点类* 输出结果场景) const noexcept;

特征节点类* 执行功能函数(
    枚举_本能动作ID 动作ID,
    方法类::节点类* 方法首节点,
    场景节点类* 输入参数场景,
    场景节点类* 输出结果场景) const noexcept;
```

说明：

1. `检查是否可调用` 也返回运行结果实例。
2. 检查成功时返回“可调用”或“运行准备完成”类实例。
3. 检查失败时返回具体失败类实例。
4. `执行功能函数` 调用动作函数后返回最终运行结果实例。

### 3.3 调用方判断

调用方不再判断 `成功码 == 0`。

调用方应通过以下方式判断：

1. 运行结果实例命中的抽象特征。
2. 运行结果实例的 I64 当前值。
3. 输出结果场景中的执行结果事实。

## 4. 运行结果实例创建策略

优先创建或复用方法虚拟存在上的运行结果实例。

若无法定位方法虚拟存在，则写入输出结果场景上的运行结果实例。

需要补齐一个小型辅助函数，但不新增结构体：

```text
输入：方法首节点、输出结果场景、运行结果抽象特征、I64 当前值
输出：特征节点类* 运行结果实例
```

该辅助函数只负责选择宿主并取或创建实例特征。

宿主选择顺序：

1. 方法首节点的虚拟存在。
2. 输出结果场景。
3. 无法创建时返回空。

## 5. 运行结果抽象特征

第一批需要支持的运行结果语义：

| 语义 | I64 |
| --- | ---: |
| 运行成功 | 0 |
| 运行结果未知 | -1 |
| 输入参数不满足 | -20009 |
| 方法首节点缺失 | -20005 |
| 方法节点类型错误 | -20006 |
| 动作入口缺失 | -20007 |
| 动作入口不归本类执行 | -20008 |
| 本能动作未注册 | -20001 |
| 执行异常 | -20003 |
| 执行未知异常 | -20004 |

实现时优先复用已有抽象特征机制：

1. 通过世界树抽象特征根查找或创建运行结果抽象特征。
2. 通过 `取或创建实例特征_按抽象特征` 创建实例。
3. 实例当前值使用 `I64`。

## 8. 风险点

| 风险 | 处理 |
| --- | --- |
| 检查失败时没有方法虚拟存在 | 将运行结果实例落到输出结果场景 |
| 调用方仍需要耗时、质量、动态入口 | 从方法虚拟存在、输出结果场景或动态信息读取，不放回混合返回结构 |
| 大量调用方依赖 `成功码` | 分阶段迁移，先建立抽象特征实例到 I64 的对应关系 |
| 方法规范仍有旧描述 | 阶段 6 一并修订 |
