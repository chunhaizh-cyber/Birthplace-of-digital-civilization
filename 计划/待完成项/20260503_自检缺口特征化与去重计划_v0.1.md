# 自检缺口特征化与去重计划 v0.1

## 1. 目标

自检线程已经能独立发现缺口并上报。本计划负责继续收口：

```text
用自检缺口条件包替代旧派生需求消息作为主载体。
实现方法结构学习积压检测。
防止重复生成同一方法结构缺口需求。
将缺口语义迁移到抽象特征和实例特征槽。
```

## 2. 子计划

### P1 自检缺口条件包

目标结构：

```text
自检缺口条件包
  缺口类型
  来源线程
  来源对象
  来源方法能力值
  来源方法头节点
  目标特征类型
  缺失结构类型
  建议需求类型
  证据集合
  严重程度
  是否建议生成需求
```

验收：

```text
缺口类型使用抽象特征点区间值。
来源对象使用可解析引用。
自检线程只生成缺口条件包。
自我线程负责把缺口条件包转换成需求节点。
```

### P2 旧派生需求消息降级

目标：

```text
旧任务管理派生需求消息只保留为兼容入口。
自检主链不再以字符串消息包作为权威语义。
```

验收：

```text
自检报告载荷包含特征化缺口候选。
字符串摘要只用于显示和调试。
```

### P3 方法结构学习积压检测

判断依据：

```text
需求树中已有同目标方法的结构缺口需求。
已有对应任务处于筹办、执行或等待子需求。
已有选中自我能力值为自我_尝试学习的任务正在处理该方法。
已有未完成的学习尝试运行事实。
已有候选条件结果对处于候选或试用状态。
```

验收：

```text
同一目标方法已有学习积压时，自检线程不重复提交同类方法结构缺口。
```

### P4 去重账与需求树协同

目标：

```text
内存去重账只做短期抑制。
需求树和任务状态负责长期去重。
```

验收：

```text
重启或长时间运行后，仍能通过需求树和任务状态识别已存在的缺口处理。
```

## 3. 完成口径

```text
自检缺口有特征化主载体。
旧派生需求消息降级为兼容。
方法结构缺口不会对同一方法反复生成重复需求。
```
