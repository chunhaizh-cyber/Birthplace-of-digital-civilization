目标：

在现有特征类实现基础上，落地“抽象特征定义层 + 实例特征观测层 + 稳态升格区间 + 组合规则”的最小闭环。旧 `特征节点主信息类` 暂作为实例特征兼容壳，避免一次性推翻现有任务、方法、状态、动态、因果链路。

## 1. 总原则

本计划只做特征系统的骨架迁移，不在第一阶段迁移所有业务语义结构体。

硬规则：

1. 抽象特征挂世界树根下的抽象特征子链。
2. 抽象特征没有当前值。
3. 实例特征挂存在、场景或虚拟存在下。
4. 实例特征有当前值、稳态、当前命中抽象特征。
5. 抽象特征区间只支持 `I64区间`。
6. 多抽象特征组合生成新抽象特征时，使用组合规则，不使用子链表示来源。
7. 子链只表达结构组成。
8. 组合规则只表达来源依赖。

### 7.2 组合求值入口

落点：

- `D:\鱼巢\特征类.h`
- `D:\鱼巢\特征类.cpp`
- `D:\鱼巢\世界树类.h`
- `D:\鱼巢\世界树类.cpp`

新增接口：

```text
求值组合抽象特征(宿主, 组合抽象特征)
```

V1 支持：

- `全部命中`
- `任一命中`
- `至少N项命中`
- `命中计数`

V1 可暂缓：

- `加权求和`
- `有符号差值`
- `方法计算`

求值流程：

```text
读取组合规则
-> 找到宿主下每个来源抽象特征的实例特征
-> 读取当前值
-> 判断期望区间或期望抽象状态命中
-> 按组合方式得到中间结果
-> 按输出策略得到输出值
-> 写入组合抽象特征对应实例特征
-> 执行区间命中
```

验收：

- 在任务虚拟存在下已有：
  - `方法是否可用 = 1`
  - `条件是否齐备 = 1`
  - `风险是否阻断 = 0`
- 求值 `任务是否可执行` 后，能得到实例特征当前值 `1`，并命中 `可执行 [1,1]`。

### 7.3 组合依赖环检测

V1 最小要求：

- 后续补完整 DAG 环检测。

验收：

## 10. 风险与边界

风险：

1. 现有代码大量使用 `特征节点主信息类`，不能直接替换。
2. `枚举_主信息类型` 增加类型后，控制面板和命名分支可能遗漏。
3. 抽象特征根若初始化过晚，业务初始化可能找不到抽象定义。
4. 组合求值若不限制环，后续可能出现递归求值死循环。
5. 区间重叠允许存在，选择函数不能假设单命中。

边界：

1. V1 不迁移所有语义结构体。
2. V1 不实现 VecU 值域区间。
3. V1 不实现完整自动区间拆分治理。
4. V1 不实现完整组合 DAG 环检测，只禁止直接自依赖。
5. V1 不删除旧特征接口，只新增新入口。
