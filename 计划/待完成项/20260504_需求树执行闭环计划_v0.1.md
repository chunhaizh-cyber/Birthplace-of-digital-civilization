# 需求树执行闭环计划 v0.1

## 1. 目标

本计划负责把需求节点、任务虚拟存在、筹办、执行、缺口生长、叶子需求执行和父需求回传串成闭环。

核心定版：

```text
需求树是主结构。
任务不是管理者，而是需求节点的运行壳。
每个需求节点都有一个对应任务。
任务只有两个主阶段：筹办、执行。
缺口不作为失败终点，而是生长为子需求。
```

## 2. 总结构

```text
需求节点
  需求条件包
  限制包
  父需求
  子需求集合
  需求状态
  对应任务

任务虚拟存在
  所属需求节点
  当前阶段
  选中自我能力值
  方法配对结果
  参数配对结果
  可执行输入参数场景
  最近运行虚拟存在
```

这些长期语义必须落到抽象特征和实例特征槽；结构体字段只能作为运行期缓存或兼容镜像。

## 3. 子计划

### P1 需求节点与任务一一对应

目标：

```text
每个需求节点都有对应任务。
任务记录所属需求节点。
需求节点记录对应任务。
```

验收：

```text
创建需求时可以创建或复用对应任务。
任务不存在时生成结构缺口或补齐需求。
任务不再承担管理所有任务的中心角色。
```

### P2 筹办阶段

筹办只回答：

```text
这个需求有没有可用方法能力？
```

流程：

```text
读取需求条件包
读取查找限制包
构建查找方法输入场景
调用查找方法
读取选中自我能力值
写回任务虚拟存在
阶段切换为执行
```

失败：

```text
生成方法缺口、方法结构缺口、能力包缺口或抽象路径缺口需求。
当前需求进入等待子需求。
```

验收：

```text
筹办不配参数。
筹办不执行方法。
筹办不提前创建需求方向方法头。
筹办结果的权威字段是选中自我能力值。
```

### P3 执行阶段

执行只回答：

```text
这个已选能力现在能不能真正运行并满足需求？
```

流程：

```text
读取选中自我能力值
根据能力值解析方法首节点
自检方法结构
调用配对参数
获得可执行输入参数场景
调用本能函数
检查需求是否满足
```

失败：

```text
方法结构缺口 -> 子需求
参数条件缺口 -> 子需求
执行结果缺口 -> 子需求
```

验收：

```text
执行阶段不重新查方法，除非选中能力值缺失或缓存失效。
方法首节点只作为运行期解析结果。
配对参数失败不会默认补值。
本能函数失败会输出可派生的缺口信息。
```

### P4 缺口生长

目标：

```text
筹办或执行发现的缺口都能转为需求树子节点。
```

缺口类型：

```text
方法缺口
方法结构缺口
能力包缺口
形参包缺口
条件结果对缺口
参数条件缺口
实例特征缺口
执行结果缺口
证据不足缺口
```

验收：

```text
缺口需求保留来源需求、来源任务、来源运行事实和缺口类型。
缺口类型有抽象特征槽，不只存在于字符串字段。
```

### P5 叶子需求执行

叶子需求定义：

```text
当前已经不需要继续派生子需求，
因为它有可执行方法，并且方法条件满足。
```

验收：

```text
叶子需求能生成闭合输入参数场景。
叶子需求成功执行后写满足证据。
执行后结果必须通过检查需求是否满足确认。
```

### P6 父需求回传

目标：

```text
子需求满足后，父需求可以重新筹办或重新执行。
根需求满足后形成闭环完成记录。
```

验收：

```text
子需求满足会更新父需求缺口状态。
父需求可重新进入筹办或执行。
根需求满足后写完成记录和运行事实链。
```

## 4. 非目标

```text
不重新引入任务管理任务管理所有任务的旧模式。
不把学习作为独立任务系统。
不让自我线程承担控制面板摘要或深度树扫描。
```

## 5. 完成口径

```text
每个需求节点都有对应任务。
任务只有筹办和执行两个主阶段。
筹办失败生成方法缺口。
执行失败生成条件缺口或结果缺口。
缺口生长为子需求。
子需求满足后父需求重新筹办或执行。
根需求满足后形成闭环完成记录。
```
