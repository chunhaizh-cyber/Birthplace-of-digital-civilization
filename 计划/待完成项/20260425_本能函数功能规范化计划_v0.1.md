# 2026-04-25 本能函数功能规范化计划 v0.1

## 1. 目标

建立一套新的本能函数功能规范，让所有本能函数运行一次时都自动完成内部四段式流程：

1. 确保
2. 解析绑定
3. 执行
4. 输出

本计划以最新口径为准：本能函数返回信息分两类存储。函数功能结果，例如执行结果、缺口、特征变化和结果变化，记录在结果场景中；函数运行情况，例如运行成功、运行失败、错误码、成本、质量和本轮运行摘要，记录在方法的虚拟存在中。返回结构只作为本轮调用的轻量传递，不作为权威存储。

## 2. 最新口径

1. 本能函数是方法函数的最小运行单元。
2. 本能函数内部自带四段式执行壳，外部执行桥不再拆分这四步。
3. 工作线程只负责调用任务管理任务实例上的当前方法，并回传方法实际结果与任务管理结果。
4. 条件场景定义方法需要哪些参数；具体参数来源由任务层级决定。
5. 管理自身时，参数来源是任务管理任务虚拟存在。
6. 管理学习时，参数来源是被学习方法的虚拟存在。
7. 管理普通任务时，参数来源是被管理任务的虚拟存在或任务场景。
8. 普通方法真正执行时，才使用方法输入参数中的条件场景完成取值匹配。
9. 缺条件时，本能函数在结果场景记录缺口，由上行消息交给自我线程转为需求。
10. 缺方法时，本能函数在结果场景记录缺口，由任务管理任务组织补方法或学习。
11. 本能函数运行成功、失败、错误码、成本、质量等运行情况写入方法虚拟存在。
12. 本能函数、执行桥、工作线程、界面线程、自我线程都不能再恢复旧上下文镜像逻辑。

## 3. 四段式规范

### 3.1 确保

职责：

1. 确保结果场景存在。
2. 确保目标存在或目标场景可解析。
3. 确保方法头、动作句柄和条件场景入口可解析。
4. 只做运行前基础保证，不直接创建任务头。

输出到结果场景：

1. 本能函数 ID。
2. 执行尝试主键。
3. 目标存在主键。
4. 方法主键。
5. 输入场景主键。
6. 输出结果场景主键。
7. 确保阶段缺口。

写入方法虚拟存在：

1. 确保阶段运行状态。
2. 确保阶段错误码。
3. 确保阶段是否通过。
4. 确保阶段运行摘要。

### 3.2 解析绑定

职责：

1. 从方法条件场景读取所需特征类型。
2. 根据本轮任务层级选择参数来源。
3. 将所需特征类型绑定到具体特征节点、状态节点或实际值。
4. 对无法绑定的条件生成缺条件事实。

输出到结果场景：

1. 条件特征类型。
2. 参数来源类型。
3. 参数来源主键。
4. 实际绑定的特征节点或状态节点。
5. 缺失条件列表。
6. 缺失条件对应的当前状态和目标状态。

写入方法虚拟存在：

1. 解析绑定阶段运行状态。
2. 解析绑定阶段错误码。
3. 参数绑定成功或失败。
4. 解析绑定阶段运行摘要。

### 3.3 执行

职责：

1. 只有解析绑定通过时才执行真实动作。
2. 执行过程可以改变目标世界树状态，但必须同时把变化事实写入结果场景。
3. 如果执行发现新缺口，不直接新建任务，而是在结果场景记录缺口。
4. 执行阶段不负责调度下一任务。

输出到结果场景：

1. 方法实际返回值。
2. 变化前状态。
3. 变化后状态。
4. 变化的特征类型。
5. 变化主体。
6. 是否产生缺条件。
7. 是否产生缺方法。

写入方法虚拟存在：

1. 执行阶段运行状态。
2. 执行是否开始。
3. 执行阶段成功或失败。
4. 执行阶段错误码。
5. 执行阶段成本。
6. 执行阶段质量。
7. 执行阶段运行摘要。

### 3.4 输出

职责：

1. 汇总前三段结果。
2. 生成简短返回结构。
3. 保证 `结果存在集场景` 和 `结果事件集场景` 指向结果场景。
4. 将函数运行情况汇总写入方法虚拟存在。
5. 不把功能细账或运行账复制到返回结构中。

结果场景保留：

1. 功能执行结果。
2. 缺条件和缺方法事实。
3. 特征变化事实。
4. 需求当前状态。
5. 需求目标状态。
6. 需求方向摘要。

方法虚拟存在保留：

1. 本轮运行状态。
2. 成功或失败。
3. 错误码。
4. 质量和成本。
5. 运行开始和结束时间。
6. 运行摘要。

返回结构临时携带：

1. 结果场景指针。
2. 方法虚拟存在指针或主键。
3. 调用桥接码。
4. 备注摘要。

### 3.5 方法虚拟存在运行特征清单

本节只定义本能函数运行情况特征。函数功能结果、缺口明细、特征变化明细仍以结果场景为权威。

1. `方法类型`

3. `方法_本能动作ID`
4. `方法_成熟度阶段`
5. `方法_允许自动召回`
6. `方法_允许默认执行`
7. `方法_影响签名数量`
8. `方法_特征账本生命周期`
9. `方法_特征暴露次数`
10. `方法_特征读取次数`
11. `方法_特征有效使用次数`
12. `方法_特征历史使用次数`
13. `方法_特征近期使用次数`
14. `方法_特征作为决策输入次数`
15. `方法_特征作为派生父特征次数`
16. `方法_特征作为需求触发次数`
17. `方法_特征作为风险闸门次数`
18. `方法_特征作为验证信号次数`
19. `方法_特征作为解释依据次数`
20. `方法_特征结果改变次数`
21. `方法_特征坏结果次数`
22. `方法_特征下游依赖特征数`
23. `方法_特征最近使用时间`
24. `方法_特征最近有效使用时间`

本能函数运行时最小必填特征：

1. `方法_本能运行状态`：`I64` 枚举，表示未启动、运行中、成功、失败、异常、超时、中断。
2. `方法_本能运行阶段`：`I64` 枚举，表示确保、解析绑定、执行、输出、完成。
3. `方法_本能最近请求ID`：`I64`，对应本轮执行请求。
4. `方法_本能最近开始时间`：`I64` 时间戳。
5. `方法_本能最近结束时间`：`I64` 时间戳。
6. `方法_本能最近耗时_us`：`I64`。
7. `方法_本能最近错误码`：`I64`，只表达函数运行错误，不表达功能结果明细。
8. `方法_本能最近调用桥接码`：`I64`，用于执行桥轻量判断。
9. `方法_本能最近质量_Q10000`：`I64`。
10. `方法_本能最近成本_Q10000`：`I64`。
11. `方法_本能最近成本_us`：`I64`。
12. `方法_本能最近结果场景`：`指针句柄`，指向本轮功能结果账本。
13. `方法_本能最近输入场景`：`指针句柄`。
14. `方法_本能最近目标存在`：`指针句柄`。
15. `方法_本能最近宿主任务`：`指针句柄`。

四段式阶段运行特征：

1. `方法_本能确保阶段状态`：`I64` 枚举。
2. `方法_本能解析绑定阶段状态`：`I64` 枚举。
3. `方法_本能执行阶段状态`：`I64` 枚举。
4. `方法_本能输出阶段状态`：`I64` 枚举。
5. `方法_本能确保阶段耗时_us`：`I64`。
6. `方法_本能解析绑定阶段耗时_us`：`I64`。
7. `方法_本能执行阶段耗时_us`：`I64`。
8. `方法_本能输出阶段耗时_us`：`I64`。
9. `方法_本能最近失败阶段`：`I64` 枚举，成功时为未定义。

运行统计特征：

1. `方法_本能运行次数`：`I64`。
2. `方法_本能成功次数`：`I64`。
3. `方法_本能失败次数`：`I64`。
4. `方法_本能异常次数`：`I64`。
5. `方法_本能超时次数`：`I64`。
6. `方法_本能连续失败次数`：`I64`。
7. `方法_本能最近成功时间`：`I64` 时间戳。
8. `方法_本能最近失败时间`：`I64` 时间戳。
9. `方法_本能累计成本_us`：`I64`。
10. `方法_本能累计成功耗时_us`：`I64`。

派生统计特征：

1. `方法_本能缺条件结束次数`：`I64`，由结果场景派生，不作为缺条件权威。
2. `方法_本能缺方法结束次数`：`I64`，由结果场景派生，不作为缺方法权威。
3. `方法_本能结果场景变更次数`：`I64`，用于观察结果账本是否持续产生变化。

存储约束：

1. 运行状态、阶段、错误码、成本、质量、次数和时间统一用 `I64`。
2. 场景、存在、任务、方法引用统一用 `指针句柄`。
3. 不在方法虚拟存在中写字符串摘要；需要可读摘要时从结果场景或节点主信息读取。
4. 缺条件、缺方法、功能执行结果和特征变化明细不在方法虚拟存在中重复存储。

## 4. 阶段计划

### 阶段 0：现状盘点

目标：

确认所有已注册本能函数和兼容壳的实际形态。

任务：

1. 盘点 `本能动作管理类.ixx` 中的注册接口和兼容注册接口。
2. 盘点 `自我初始化模块.cpp` 中运行时注册的本能函数。
3. 盘点 `任务管理任务模块.impl.cpp` 中任务管理治理本能动作。
4. 盘点 `自我_执行桥模块.ixx` 中对本能动作返回结果的转换逻辑。
5. 标出仍把功能结果写在返回结构、旧上下文或输入场景中的代码。
6. 标出仍把运行情况只放在返回结构而没有同步方法虚拟存在的代码。

产物：

1. 现有本能函数清单。
2. 未符合四段式规范的函数列表。
3. 旧上下文残留列表。

### 阶段 1：定义统一数据合同

目标：

先确定本能函数执行壳的输入、阶段结果和输出，不急着迁移业务逻辑。

任务：

1. 明确 `结构_本能动作执行上下文` 的字段含义。
2. 明确 `结构_本能动作步骤结果` 只作为轻量返回。
3. 设计结果场景中统一记录的事实类型。
4. 设计缺条件、缺方法、执行结果、特征变化的结果场景记录格式。
5. 设计成功、失败、错误码、成本、质量的方式，将其写入方法虚拟存在。
6. 约定结果场景主键和方法虚拟存在主键如何传给任务管理结果状态消息。

产物：

1. 本能函数执行上下文规范。
2. 本能函数轻量返回规范。
3. 结果场景功能事实记录规范。
4. 方法虚拟存在运行情况记录规范。

### 阶段 2：实现四段式执行壳

目标：

新增统一执行壳，让本能函数实现只填四段逻辑，不再散落手写流程。

任务：

1. 新建或扩展本能函数执行壳 helper。
2. 提供阶段回调：确保、解析绑定、执行、输出。
3. 提供结果场景写入 helper。
4. 提供方法虚拟存在运行情况写入 helper。
5. 提供缺条件和缺方法的标准写入 helper。
6. 保证兼容旧注册方式时也会经过统一输出口径。

约束：

1. 执行壳不创建任务。
2. 执行壳不直接派发消息。
3. 执行壳不读取旧上下文镜像。
4. 执行壳只保证本能函数内部四段顺序、结果场景功能记账和方法虚拟存在运行记账。

### 阶段 3：迁移任务管理三类本能函数

目标：

先迁移当前最关键的三类本能函数。

范围：

1. `自我_任务筹办`
2. `自我_任务执行一步`
3. `自我_学习任务`

任务：

1. `任务筹办` 改为四段式：根据需求找方法，找不到方法则在结果场景写缺方法。
2. `任务执行一步` 改为四段式：根据当前方法条件绑定参数，缺条件则在结果场景写缺条件。
3. `学习任务` 改为四段式：根据方法虚拟存在判断缺动作骨架、缺条件、缺结果。
4. 三者均不直接创建任务头。
5. 三者均不直接补旧上下文。

通过标准：

1. 三类本能函数每次执行都有结果场景。
2. 三类本能函数每次执行都会更新方法虚拟存在运行情况。
3. 缺条件和缺方法可以从结果场景读出。
4. 成功、失败、错误码、成本和质量可以从方法虚拟存在读出。
5. 工作线程结果消息能带回结果场景主键和方法虚拟存在主键。

### 阶段 4：接通消息与需求生成

目标：

让结果场景中的缺口成为自我线程和任务管理任务生成需求的依据。

任务：

1. 工作线程上行结果状态消息携带结果场景入口、方法虚拟存在入口和轻量状态。
2. 任务管理界面线程收取上行消息，并按结果场景与方法虚拟存在改变任务管理任务状态。
3. 自我线程根据结果场景中的缺条件事实创建需求。
4. 任务管理任务根据结果场景中的缺方法事实创建补方法或学习需求。
5. 同一父需求链上同类型子需求只出现一次。

通过标准：

1. 缺条件不会直接创建任务，只会先形成需求。
2. 缺方法不会由普通任务直接创建学习任务，只会先形成需求。
3. 新需求出现后，原任务暂停等待需求满足或被调度。

### 阶段 5：迁移普通本能函数

目标：

把非任务管理类本能函数也迁移到统一四段式。

任务：

1. 按风险从低到高迁移兼容场景本能函数。
2. 每个本能函数补齐条件场景读取。
3. 每个本能函数补齐结果场景变化记录。
4. 每个本能函数补齐方法虚拟存在运行情况记录。
5. 移除仍绕过执行壳的旧注册路径。

通过标准：

1. 所有本能函数都通过统一执行壳运行。
2. 本能函数运行一次必有四段式记录。
3. 控制面板可从结果场景看到功能执行细账。
4. 控制面板可从方法虚拟存在看到运行成功、失败、错误码、成本和质量。

### 阶段 6：清理旧逻辑与观察面

目标：

删除与新规范冲突的旧代码和旧文案。

任务：

1. 删除旧上下文镜像字段写入通道。
2. 删除旧摘要推断逻辑中依赖本能函数返回细节的部分。
3. 控制面板显示结果场景真实字段，不再加工旧摘要。
4. 更新计划完成记录。

通过标准：

1. 搜索旧上下文关键词无有效调用。
2. 编译通过。
3. 控制面板能沿结果场景展示真实执行事实。

## 5. 执行顺序

1. 阶段 0：现状盘点。
2. 阶段 1：统一数据合同。
3. 阶段 2：四段式执行壳。
4. 阶段 3：迁移任务管理三类本能函数。
5. 阶段 4：接通消息与需求生成。
6. 阶段 5：迁移普通本能函数。
7. 阶段 6：清理旧逻辑与观察面。

## 6. 验证矩阵

### 6.1 构建验证

命令：

```powershell
MSBuild.exe 鱼巢.vcxproj /p:Configuration=Debug /p:Platform=x64 /m:1 /v:minimal
```

通过标准：

1. 0 error。
2. 模块无循环 import。
3. 本能函数注册数量符合预期。

### 6.2 摘要验证

命令：

```powershell
.\x64\Debug\鱼巢.exe --task-mgmt-summary
.\x64\Debug\鱼巢.exe --learning-summary
```

观察项：

1. 需求数量不再因同类缺口重复增长。
2. 任务数量只随需求任务映射增长。
3. 缺条件时出现自我线程需求。
4. 缺方法时出现补方法或学习需求。
5. 原任务进入等待，不继续空转增长。

### 6.3 日志验证

观察项：

1. 每个本能函数执行都有结果场景主键。
2. 每个本能函数执行都有方法虚拟存在主键。
3. 四段式阶段均有记录。
4. 缺条件和缺方法能定位到具体特征类型。
5. 特征变化有变化前和变化后状态。
6. 成功、失败、错误码、成本和质量写入方法虚拟存在。
7. 每次本能函数执行都会在方法树中生成或复用一组条件节点和结果节点。
8. 条件节点按条件方向融合，结果节点按结果方向融合，不按旧上下文摘要增长。

### 6.4 控制面板验证

观察项：

1. 任务树显示真实任务字段。
2. 需求树与任务列表对应。
3. 结果场景可展开查看细账。
4. 方法虚拟存在可展开查看运行情况。
5. 不再出现旧上下文摘要文案。

## 7. 完成定义

本计划完成时，应满足：

1. 所有本能函数运行一次都经过内部四段式。
2. 执行桥不再理解四段式细节，只调用本能函数并接收轻量结果。
3. 工作线程不再解析本能函数细账，只上报结果场景入口和方法虚拟存在入口。
4. 自我线程和任务管理任务根据结果场景生成功能需求，根据方法虚拟存在判断函数运行情况。
5. 旧上下文镜像逻辑不再参与本能函数执行。
6. 学习任务可以基于缺方法结果场景进入真实调度。
