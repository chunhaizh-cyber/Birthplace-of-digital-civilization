## 0. 状态修订

```text
当前状态：未完成。

此前“P0-P8 第一版完成”只能说明：
1. 诊断字段、摘要命令和局部任务承接观测已建立；
2. 部分任务 / 方法 / 结算路径有可见样本；
3. 控制面板能展示内部循环状态。

但这不等于最小生命闭环成立。

当前第一阻塞点：
    需求树尚不能真实自动生长。

因此本总计划必须重新回到：
    特征值差距 -> 需求生长 -> 需求任务化 -> 方法执行 -> 状态回写 -> 需求结算 -> 下一轮判断
的真实闭环验收。
```

这里的“特征值差距”只表示同一目标特征类型上的当前值与目标值未满足；不同特征类型之间的关系不由差距表达，应由状态动态、二次特征 / 关系特征、因果实例 / 因果模板和方法结果能力表达。

## 1. 总目标

本计划用于把“自我只有内部循环”的工程验收从初始化成功推进到最小生命闭环成立。

总目标：

```text
自我能在无外部输入的情况下，
从自身安全值 / 服务值变化出发，
生成和选择当前主需求，
建立任务信息节点并绑定任务虚拟存在，
选择方法或本能动作，
执行动作并回写结果，
结算来源需求，
并让下一轮主需求判断受到上一轮结果影响。
```

闭环链路：

```text
自我状态
→ 安全 / 服务压力
→ 根需求
→ 需求树生成 / 更新 / 组织
→ 当前主需求
→ 任务信息节点
→ 任务虚拟存在
→ 方法 / 本能动作
→ 执行结果
→ 状态回写
→ 需求结算
→ 下一轮状态判断
```

完成定义：

```text
至少达到内部循环完成度 6 级。
工程验收达到内部循环完成度 7 级。
```

## 2. 当前边界

本计划不改变当前系统边界：

```text
自我线程 = 治理协调层
任务管理界面线程 = 任务系统入口和结果归并层
任务管理工作线程 = 一步任务推进层
任务信息节点 = 任务运行壳
任务虚拟存在 = 任务语义身份和记忆对象
方法信息节点 = 方法运行结构
方法虚拟存在 = 方法语义身份和长期记忆
```

禁止回退：

```text
不恢复自我线程直连任务管理工作线程。
不让任务虚拟存在替代任务信息节点。
不让方法虚拟存在替代方法首节点。
不新增抽象存在主信息类。
不新增存在形态枚举。
不新增特征节点当前值信息。
不以外部输入输出作为自我建立完成标准。
```

## 3. 阶段总览

| 阶段 | 完成度 | 阶段名称 | 目标 |
| --- | --- | --- | --- |
| P0 | 0 -> 1 | 验收仪表和诊断基线 | 能诊断当前自我内部循环处于哪一级 |
| P1 | 1 | 冷启动验收 | 自我锚点、内部世界、核心状态、根需求、本能注册稳定 |
| P2 | 2 | 需求树生成成立 | 根据安全 / 服务压力和内部状态生成 / 更新 / 组织需求树，并选择当前主需求 |
| P3 | 3 | 主需求任务化 | 当前主需求能稳定创建 / 复用任务信息节点 |
| P4 | 3 -> 4 | 任务语义和方法入口 | 任务信息节点绑定任务虚拟存在，并能选择方法或本能动作 |
| P5 | 4 -> 5 | 动作执行与状态回写 | 本能动作或内部方法能改变自我核心状态 |

## 4. 分步计划

### P0：验收仪表和诊断基线

目标：

```text
新增一套内部循环完成度诊断入口。
先能判断，再谈修复。
```

建议实现：

```cpp
enum class 枚举_自我内部循环完成度 : std::uint8_t {
    未初始化 = 0,
    冷启动完成 = 1,
    需求树生成成立 = 2,
    任务化可用 = 3,
    本能动作可用 = 4,
    结果回写可用 = 5,
    需求结算可用 = 6,
    连续多轮稳定 = 7,
};

struct 结构_自我内部循环诊断 {
    枚举_自我内部循环完成度 完成度 = 枚举_自我内部循环完成度::未初始化;
    bool 自我锚点有效 = false;
    bool 核心状态可读写 = false;
    bool 根需求有效 = false;
    bool 当前主需求有效 = false;
    bool 当前主任务有效 = false;
    bool 当前主任务虚拟存在有效 = false;
    bool 方法或本能动作可调用 = false;
    bool 最近结果已回写 = false;
    bool 最近需求已结算 = false;
    bool 最近多轮稳定 = false;
    std::vector<std::string> 缺失项{};
    std::string 摘要{};
};
```

候选落点：

```text
自我类：提供自我相关诊断公共函数。
自我线程：在每轮上下文中刷新当前循环等级。
控制面板：只读展示诊断结果。
```

验收：

```text
能在冷启动后输出 0-7 级中的一个等级。
缺失项能指出最先断开的链路。
不依赖外部输入输出判断。
```

### P1：冷启动验收

目标：

```text
确认自我点火成功。
```

检查项：

```text
自我存在有效。
自我内部世界有效。
自我现实场景有效。
安全值 / 服务值 / 风险安全值可读写。
待机状态和时序步长可同步。
安全根需求、服务根需求有效。
本能动作 / 本能方法已注册。
```

口径修正：

```text
需求根 / 任务根 / 方法根 是存在信息结构字段，默认可以为空。
冷启动验收不把三类根字段非空作为硬条件。
需要创建需求、任务或方法时，由对应创建链按需确保结构根；
请求创建失败才属于非正常逻辑错误。
```

实施要点：

```text
1. 复用现有自我初始化阶段化结果。
2. 给初始化结果和内部循环诊断建立映射。
3. 当前主任务、当前主方法允许为空，不影响冷启动等级。
```

验收：

```text
初始化完成后，内部循环完成度至少达到 1 级。
缺任一锚点时，诊断不能误报 1 级。
```

### P2：需求树生成成立

目标：

```text
自我能根据自身状态生成、更新、组织需求树，并选择当前主需求。
```

检查项：

```text
安全压力可计算。
服务压力可计算。
安全根需求和服务根需求可参与候选。
安全 / 服务压力能激活或生成对应需求。
任务缺条件、方法不可用、结果不确定等内部缺口能转成对应需求。
同目标需求能复用 / 合并 / 刷新权重，而不是每轮重复新建。
有子需求的需求能刷新为管理需求，无子需求的需求能刷新为执行需求。
当前主需求能写回自我运行态镜像。
```

最小规则：

```text
安全压力 >= 服务压力 -> 安全根需求
服务压力 > 安全压力 -> 服务根需求
```

实施要点：

```text
1. 复查 `自我类::选择当前主需求候选_()` 的压力口径。
2. 复查安全 / 服务值读写与运行态同步。
3. 复查需求树更新入口，确认动态需求来自自我状态或任务 / 方法回执，而非初始化预置。
4. 复查需求去重 / 合并和结构角色刷新。
5. 让自我线程一轮循环后能稳定刷新当前主需求。
6. 诊断记录最近主需求选择原因和需求树生成断点。
```

验收剧本：

```text
降低安全值，应选安全根需求。
降低服务值，应选服务根需求。
两者都低时，选择压力更大的根需求。
任务缺条件时，应生成或复用补条件需求。
方法不可用时，应生成或复用学习 / 补方法需求。
连续两轮同一缺口不应生成重复需求风暴。
有子需求的父需求应为管理需求，叶子需求应为执行需求。
```

完成后：

```text
内部循环完成度至少达到 2 级。
```

### P3：主需求任务化

目标：

```text
当前主需求能稳定进入任务信息节点。
```

正确链路：

```text
当前主需求 D
→ 对应任务信息节点 N
```

实施要点：

```text
1. 自我线程只确保或选择任务信息节点，不直接调用任务管理工作线程。
2. 任务创建 / 复用必须走需求绑定统一入口。
3. 同一未结束需求不得重复创建多个当前任务。
4. 当前主任务写回自我线程上下文和运行态快照。
```

验收：

```text
给定安全根需求或服务根需求；
执行一轮自我线程；
能得到当前主任务信息节点或任务治理请求；
重复执行不会制造同一需求的任务风暴。
```

完成后：

```text
内部循环完成度至少达到 3 级。
```

### P4：任务语义和方法入口

目标：

```text
任务不仅有运行壳，还具备语义身份，并能找到方法 / 本能动作入口。
```

正确链路：

```text
任务信息节点 N
→ 任务虚拟存在 T
→ 候选方法 / 本能动作
```

实施要点：

```text
1. 任务管理界面线程确保 N.任务虚拟存在。
2. 任务虚拟存在概念集包含任务 / 具体任务概念。
3. 任务虚拟存在补齐任务目标、任务状态、完成条件等特征槽。
4. 任务筹办能基于来源需求和任务目标找到候选方法。
5. 没有方法时创建 / 复用目标方法头并待重入，不生成缺方法子需求。
6. 本能动作作为最小阶段的可执行方法来源。
```

验收：

```text
当前主任务信息节点有效。
任务虚拟存在有效。
任务虚拟存在有任务目标 / 完成条件 / 任务状态。
任务筹办能得到可执行方法、本能方法，或得到目标方法头待重入回执。
```

完成后：

```text
内部循环完成度至少达到 4 级。
```

### P5：动作执行与状态回写

目标：

```text
方法或本能动作执行后，能改变自我内部状态。
```

最小动作集：

```text
更新安全值
更新服务值
进入待机
退出待机
重置时序步长
同步运行态
生成任务结果
结算需求价值
```

实施要点：

```text
1. 任务管理工作线程只处理任务工作包并返回任务工作结果。
2. 动作执行结果先进入任务工作结果。
3. 任务管理界面线程归并工作结果和上行消息。
4. 自我线程消费上行消息后写回自我内部世界或自我特征。
5. 核心状态回写后刷新自我运行态镜像。
```

验收剧本：

```text
执行前：服务值低，服务根需求被选中。
执行动作：提升服务值。
执行后：服务值变化被读回。
下一轮：服务压力降低。
```

完成后：

```text
内部循环完成度至少达到 5 级。
```

### P6：需求结算闭环

目标：

```text
任务结果能结算来源需求，并影响下一轮压力。
```

检查项：

```text
需求累计安全结算可更新。
需求累计服务结算可更新。
最近结算任务主键可记录。
最近结算时间可记录。
需求状态能进入部分满足或已满足。
```

实施要点：

```text
1. 任务结果和价值账分账：动作失败、无可比较结果、目标符合 / 不符合各自处理。
2. 来源需求结算由自我线程完成，不由工作线程直接结账。
3. 已结算任务不重复对同一需求无限记账。
4. 结算后触发下一轮主需求重判。
```

验收：

```text
任务完成后，来源需求安全 / 服务账本发生变化。
需求压力降低。
下一轮不重复创建同一未结束任务。
```

完成后：

```text
内部循环完成度至少达到 6 级。
```

### P7：连续多轮稳定

目标：

```text
验证内部循环不是单次成功，而是可持续运行。
```

最小剧本：

```text
安全值 = 100
服务值 = 20
目标服务值 = 100
待机状态 = false
连续运行 N 轮
```

期望：

```text
第 1 轮：服务压力最大，选择服务根需求，创建 / 复用服务任务，执行提升服务值动作。
第 2 轮：服务压力降低，不重复创建同一未结束任务。
第 3 至 N 轮：服务值逐渐接近目标；当服务压力低于安全压力时可切换主需求；无明显压力时进入待机或低频自检。
```

稳定性约束：

```text
无重复任务风暴。
无无限派生需求。
无空转。
无自我线程直连工作线程旧通道。
无任务虚拟存在替代任务信息节点。
```

完成后：

```text
内部循环完成度达到 7 级。
```

## 8. 验收标准

总体验收：

