# 特征值差距刷新成子需求闭环剧本计划 v0.1

日期：2026-05-14

## 1. 阶段状态修订

```text
需求树真实生长硬回归：已通过。
内部循环多轮稳定态硬回归：本计划引入差距子需求后已重新复核通过。
当前不得提前验收：自我苏醒、初步成熟。

新的第一阻塞点：
    差距消失后子需求能否退出活动集，子任务完成后父任务能否稳定回流。
```

本计划只处理“差距 -> 子需求”这条窄链，不扩展到学习需求四类剧本和初步成熟验收。

## 2. 主剧本

```text
当前状态 / 目标状态之间存在差距
    -> 自动生成或刷新一个更细原子执行子需求
    -> 子需求具备完整目标宿主、目标特征类型、当前状态、目标状态、满足关系
    -> 同目标刷新时复用 / 合并
    -> 子需求任务化
    -> 子任务完成后父需求 / 父任务回流
```

## 3. 分步计划

### P1. 根需求差距执行子需求生成

在根需求状态镜像刷新后，若根需求仍未满足，则为根需求确保一个同目标执行子需求。

硬要求：

```text
1. 子需求必须挂在触发它的根需求下。
2. 子需求必须有被需求存在、当前状态、目标状态、目标特征类型缓存、满足关系。
3. 根驱动需求允许生成同目标执行子需求。
4. 不得生成模糊需求。
```

### P2. 同目标刷新复用 / 合并

再次刷新同一差距时必须复用同父同目标子需求，而不是重复创建。

硬要求：

```text
1. 同父直接子链不得出现重复目标活动需求。
2. 派生归因不能作为身份键。
3. 复用旧子需求时要刷新当前状态镜像。
```

### P3. 子需求权重刷新

子需求权重跟随父需求压力刷新。

硬要求：

```text
权重只分给 active && 阻塞父任务执行 的子需求。
已满足或非活跃子需求不继续分配权重。
```

### P4. 回归命令

新增：

```text
--demand-difference-child-check
--demand-diff-child-check
--need-difference-child-check
```

验收：

```text
1. 根需求差距镜像可读。
2. 自动观察到差距执行子需求。
3. 子需求完整字段齐备。
4. 同目标直接子需求不重复。
5. 子需求已任务化。
6. 二次缩小差距后，必须复用同一子需求并刷新当前状态镜像。
7. 子需求权重不是 I64_MAX 哨兵时，二次缩小差距后权重必须下降。
```
