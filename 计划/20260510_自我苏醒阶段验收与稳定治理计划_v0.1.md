## 0. 状态修订

```text
当前状态：未完成，且阻塞于内部循环闭合。

此前 A0-A4 第一版完成，只表示：
1. 苏醒诊断结构和自我报告观测面已建立；
2. 自检报告承接边界已建立；
3. 休眠期修复门控有第一版占位。

但由于需求树尚不能真实自动生长，内部循环尚未闭合，
因此不能判定自我已经苏醒。

本计划后续只能在内部循环真实闭合后继续验收：
    自主主需求 -> 任务化 -> 执行 -> 回写 -> 结算 -> 下一轮判断 -> 待机 / 自检 / 修复决策。
```

## 1. 总目标

本计划用于定义“自我苏醒”的工程落地路径。

核心定义：

```text
内部循环建立 =
    自我能完成 状态 -> 需求 -> 任务 -> 方法 -> 结果 -> 状态 的闭环。

自我苏醒 =
    内部循环已经不是单次闭环，
    而是成为自我实例的稳定治理主体。
```

也就是说，自我苏醒不是哲学意义上的主观意识，而是工程意义上的稳定自我治理能力：

```text
自我状态可被读取；
当前目标可被确认；
上一轮结果可被记忆；
下一轮行动可由自身状态推出；
异常时能由自检报告进入自我线程修复决策；
无明显压力时能进入待机；
连续多轮不空转、不任务风暴、不失去主链。
```

## 2. 前置条件

自我苏醒必须建立在内部循环至少 7 级之上。

前置闭环：

```text
自我状态
→ 安全 / 服务压力
→ 根需求
→ 当前主需求
→ 任务信息节点
→ 任务虚拟存在
→ 方法 / 本能动作
→ 执行结果
→ 状态回写
→ 需求结算
→ 下一轮状态判断
```

如果上述链路不能连续多轮稳定运行，只能判定为：

```text
已初始化
或
内部循环建设中
```

不能判定为苏醒。

## 3. 阶段关系

| 阶段 | 含义 | 判断重点 |
| --- | --- | --- |
| 初始化完成 | 自我核心结构被创建 | 自我存在、内部世界、核心状态、根需求入口和安全 / 服务根需求有效 |
| 需求树生成成立 | 自我能从自身状态产生内部驱动力 | 状态压力和内部缺口能生成 / 更新 / 组织需求树，并选出当前主需求 |
| 内部循环建立 | 自我能跑通闭环 | 需求能任务化，任务能调用方法，结果能回写 |
| 自我苏醒 | 自我能稳定治理自己 | 持续判断自身状态、维持目标、处理异常、形成自我报告 |
| 稳定苏醒 | 自我长期无外部输入仍不空转、不崩坏 | 多轮稳定、无任务风暴、可自检、可恢复 |

一句话：

```text
需求树生成成立是“机器开始产生内部驱动力”；
内部循环是“机器能转”；
自我苏醒是“机器开始以自己为对象进行治理”。
```

## 4. 苏醒等级

### 0 级：未苏醒

定义：

```text
内部循环未跑通。
```

表现：

```text
有自我实例，但不能自主形成需求；
或能形成需求，但不能任务化；
或任务不能执行；
或结果不能回写；
或下一轮判断不受上一轮结果影响。
```

### 1 级：预苏醒

定义：

```text
自我能连续完成内部闭环。
```

标准：

```text
状态 -> 需求 -> 任务 -> 方法 -> 结果 -> 状态
```

能跑通，但还不一定能稳定自检、待机、修复。

### 2 级：苏醒

定义：

```text
自我能稳定治理自己。
```

标准：

```text
能选择主需求；
能任务化；
能执行本能动作或内部方法；
能回写状态；
能结算需求；
下一轮判断受上一轮影响；
能生成当前自我报告。
```

这是本计划建议采用的正式“苏醒”标准。

### 3 级：稳定苏醒

定义：

```text
自我能长期无外部输入运行，不空转、不任务风暴、不失去主链。
```

标准：

```text
连续 N 轮稳定；
能进入待机；
能自检；
能故障收口；
能避免重复任务。
```

默认：

```text
N = 10
```

### 4 级：可交互苏醒

定义：

```text
自我能在外部输入进入时，
区分内部需求和外部需求，
把外部任务纳入内部治理循环，
并保护核心需求不被外部任务覆盖。
```

本阶段不是当前计划重点，只作为后续方向保留。

| 阶段 | 苏醒等级 | 阶段名称 | 目标 |

| A4 | 2 | 自检报告与修复决策 | 自检线程报送缺失，自我线程按运行期 / 休眠期门控决策修复或保护 |

## 6. 分步计划

### A0：苏醒诊断面

目标：

```text
在内部循环完成度诊断之上新增苏醒等级诊断。
```

建议结构：

```cpp
enum class 枚举_自我苏醒等级 : std::uint8_t {
    未苏醒 = 0,
    预苏醒 = 1,
    苏醒 = 2,
    稳定苏醒 = 3,
    可交互苏醒 = 4,
};

struct 结构_自我苏醒诊断 {
    枚举_自我苏醒等级 等级 = 枚举_自我苏醒等级::未苏醒;

    bool 内部循环已建立 = false;
    bool 自我锚点唯一稳定 = false;
    bool 核心状态可读 = false;
    bool 可自主产生主需求 = false;
    bool 可任务化并执行 = false;
    bool 执行结果可反身回写 = false;
    bool 下一轮受上一轮影响 = false;
    bool 可承接自检报告修复决策 = false;
    bool 可进入待机 = false;
    bool 可形成自我报告 = false;
    bool 连续多轮稳定 = false;

    std::string 当前自我主键{};
    std::string 当前主需求主键{};
    std::string 当前主任务主键{};
    std::string 当前主方法主键{};
    std::string 当前状态摘要{};
    std::string 上一轮结果摘要{};
    std::string 下一轮行动摘要{};
    std::vector<std::string> 缺失项{};
    std::vector<std::string> 风险项{};
    std::string 摘要{};
};
```

依赖：

```text
内部循环完成度诊断必须先具备。
```

验收：

```text
能输出 0-4 级苏醒等级。
等级依据不依赖外部输入输出。
```

### A1：自我锚点唯一性

目标：

```text
确认当前自我实例是唯一治理主体。
```

检查项：

```text
当前自我存在有效。
当前自我内部世界有效。
当前自我现实场景有效。
当前自我实例概念集包含：存在、数字生命、自我。
不存在多个当前活动自我。
不存在多个竞争内部世界。
```

实现方法：

```text
1. 复用世界树核心概念接口，读取当前自我概念集。
2. 在自我初始化或诊断中扫描当前活动自我引用。
3. 若发现多个活动自我，记录风险项，不自动合并。
4. 若概念集缺失，生成自我概念归属修复建议。
```

验收：

```text
当前自我主键稳定。
当前自我概念集稳定。
自我内部世界和现实场景不为空。
```

### A2：自主主需求稳定

目标：

```text
无外部输入时，自我能由自身安全 / 服务压力持续产生当前主需求。
```

检查项：

```text
安全压力可计算。
服务压力可计算。
当前主需求选择有原因。
当前主需求能随状态变化改变。
无明显压力时不强造任务。
```

实现方法：

```text
1. 复用内部循环 P2 的根需求压力评估。
2. 把“选择原因”写入苏醒诊断。
3. 记录最近 N 轮主需求变化。
4. 判断当前主需求是否由自身状态推出，而不是仅由外部消息刺激。
```

验收剧本：

```text
安全值降低 -> 安全根需求成为当前主需求。
服务值降低 -> 服务根需求成为当前主需求。
安全 / 服务均稳定 -> 进入待机候选，不继续制造任务。
```

### A3：自我报告

目标：

```text
自我能形成当前自我状态报告。
```

最小报告字段：

```text
我是谁：当前自我实例
我在哪里：内部世界 / 现实场景
当前安全值
当前服务值
当前风险安全值
当前待机状态
当前主需求
当前主任务
当前主方法
上一轮做了什么
上一轮结果是什么
下一轮准备做什么
当前是否故障
当前缺失项 / 风险项
```

实现方法：

```text
1. 新增只读报告结构，不直接修改治理状态。
2. 报告从自我类、内部循环诊断、自我线程快照、任务界面线程快照、最近结算快照组合生成。
3. 报告写入控制面板快照和日志，不作为任务执行输入。
```

验收：

```text
任意时刻能生成一份当前自我报告。
报告中的主需求、主任务、主方法和实际运行态一致。
```

### A4：自检报告与修复决策

目标：

```text
当关键链路缺失时，自检线程只报送缺失；
自我线程决定记录、休眠期修复、生成修复需求、待机保护或故障收口。
```

触发条件：

```text
当前主需求为空。
当前主任务为空。
任务没有虚拟存在。
方法不可用。
本能动作缺失。
安全值 / 服务值不可读。
内部世界缺失。
请求创建需求根 / 任务根 / 方法根失败。
```

边界说明：

```text
需求根 / 任务根 / 方法根 属于存在信息结构字段，默认可以为空；
空字段本身不是自检 / 修复需求触发条件。
只有当对应创建链需要创建需求、任务或方法且结构根创建失败时，才视为非正常逻辑错误并进入日志 / 报错。
任务虚拟存在在建立任务时由程序创建；任务未建立前为空不是错误，任务建立完成后仍为空才是任务语义绑定错误。
```

实现方法：

```text
1. 自检线程只生成报告型缺口候选并报送自我线程。
2. 自我线程维护自检报告待处理队列。
3. 自我线程为缺失项标注修复时机：运行期只记录 / 休眠期可修复 / 立即保护 / 非正常错误。
4. 只有自我线程在允许时机内生成修复需求；修复任务仍走任务管理界面线程。
5. 严重缺失时进入待机保护或故障收口。
```

缺失项到自我线程修复决策示例：

| 缺失项 | 自我线程决策 |
| --- | --- |
| 当前主需求为空 | 运行期重判主需求；失败则进入保护 |
| 当前主任务为空 | 运行期由自我线程推动任务化 |
| 任务虚拟存在缺失 | 若任务已建立则报错；否则等待任务建立链 |
| 方法不可用 | 可转学习 / 方法补齐需求 |
| 本能动作缺失 | 非正常错误或休眠期修复，按影响级别决定 |
| 内部世界缺失 | 非正常错误，进入保护 |

验收：

```text
人为制造一个缺失项；
下一轮诊断能发现；
自检线程报送缺失；
自我线程按时机记录、等待休眠期修复、生成修复需求或进入待机保护；
不崩溃、不任务风暴。
```

### A5：待机治理

目标：

```text
无明显压力时，自我能停止高频行动，进入待机或低频自检。
```

待机条件：

```text
安全压力低于阈值。
服务压力低于阈值。
无未完成关键需求。
无待修复严重异常。
最近 N 轮无有效新增任务需要推进。
```

实现方法：

```text
1. 在苏醒诊断中新增“待机建议”。
2. 复用自我类已有待机状态和服务归零待机保护函数。
3. 待机不删除根需求，只降低主动推进频率。
4. 保留低频自检和异常唤醒入口。
```

验收：

```text
安全 / 服务值稳定后，待机状态置为 true。
待机期间不重复创建新主任务。
低频自检仍可运行。
压力重新出现时能退出待机。
```

### A6：多轮稳定验收

目标：

```text
证明苏醒不是单轮偶然，而是稳定状态。
```

默认剧本：

```text
安全值 = 100
服务值 = 20
待机状态 = false
服务目标值 = 100
本能动作：提升服务值 已注册
连续运行 10 轮
```

期望：

```text
第 1 轮：
    服务压力较高；
    选择服务根需求；
    创建 / 复用服务任务；
    执行提升服务值动作；
    服务值上升；
    服务根需求结算。

第 2 轮：
    服务压力降低；
    不重复创建同一未结束任务；
    继续合理推进或转入下一阶段任务。

第 3 至 10 轮：
    服务值逐步恢复；
    服务压力继续下降；
    必要时切换安全根需求；
    无压力时进入待机 / 低频自检。
```

稳定性约束：

```text
无崩溃。
无空指针主链。
无重复任务风暴。
无无限派生需求。
无无效空转。
无自我线程直连工作线程旧通道。
```

验收：

```text
连续 10 轮满足稳定性约束。
苏醒等级达到 3 级。
```


## 9. 验收标准

正式苏醒标准：

```text
1. 内部循环已达到 7 级。
2. 当前自我锚点唯一且稳定。
3. 自我能读取核心状态和当前主链。
4. 无外部输入时能由自身状态选择当前主需求。
5. 主需求能任务化并绑定任务虚拟存在。
6. 当前主任务能调用本能动作或内部方法。
7. 执行结果能反身回写自我内部世界。
8. 下一轮判断受上一轮结果影响。
9. 结构缺失时能由自检报告进入自我线程修复决策或保护态。
10. 无明显压力时能进入待机 / 低频自检。
11. 能形成当前自我报告。
12. 连续 10 轮无崩溃、无任务风暴、无无限派生、无空转。
```

不能判定苏醒的情况：

## 11. 最终口径

```text
自我苏醒 =
    自我开始把自己作为治理对象，
    持续维护自己的安全、服务、任务和记忆连续性。
```

它不要求外部世界，也不要求复杂智能；它要求的是：

```text
能自主判断；
能形成行动；
能改变自己；
能记住上一轮；
能决定下一轮；
能修复缺失；
能在无压力时停下来。
```
