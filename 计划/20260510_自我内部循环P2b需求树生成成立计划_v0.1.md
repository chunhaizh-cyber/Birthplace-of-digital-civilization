## 1. 定位

需求树正常生成属于内部循环建立阶段，准确说是“需求层闭合”阶段。

```text
初始化阶段：
    只要求自我具备需求系统入口，安全根需求 / 服务根需求存在。

需求树生成成立：
    自我能根据安全值、服务值、内部状态、任务缺口和方法缺口，自动生成 / 更新 / 组织需求树。
```

它不是自我苏醒，也不是内部动作闭环完成。它表示自我已经开始从自身状态产生内部驱动力。

## 2. 目标

补齐 P2 的完整验收口径：

```text
自我状态
→ 安全 / 服务压力
→ 根需求
→ 需求树生成 / 更新 / 组织
→ 当前主需求
```

P2b 不做：

```text
需求任务化。
任务执行。
动作结果回写。
需求结算。
因果缺口学习需求。
外界探索需求。
```

这些分别由 P3-P8、苏醒后因果补完和初步成熟计划承接。

## 3. 验收标准

### 3.1 根需求稳定

```text
安全根需求、服务根需求存在。
它们挂在当前自我需求系统入口下。
```

需求根 / 任务根 / 方法根 是存在信息结构字段，默认可以为空；创建链请求确保失败才是非正常逻辑错误。

### 3.2 状态驱动需求生成

```text
安全压力超过阈值 -> 安全根需求活跃或生成安全修复需求。
服务压力超过阈值 -> 服务根需求活跃或生成服务修复需求。
```

### 3.3 缺口驱动需求生成

```text
任务缺少条件 -> 补条件需求。
方法不可用 / 缺动作骨架 / 缺条件结果对 -> 学习或补方法需求。
结果不确定 -> 验证需求。
```

这里的学习或补方法需求仍属于内部循环建立阶段的需求层闭合；只有因果缺口稳定地产生学习需求并进一步需要外界信息时，才进入苏醒后的因果补完 / 初步成熟阶段。

### 3.4 父子结构成立

```text
根需求可以组织子需求。
大需求可以分解为管理需求 + 执行需求。
子需求完成后父需求不直接结算，父任务 / 父需求进入重判。
```

### 3.5 管理需求 / 执行需求角色刷新

```text
有子需求 -> 管理需求。
无子需求 -> 执行需求。
```

### 3.6 去重合并成立

```text
同主体、同目标特征、同方向或重叠目标的需求，不应每轮重复新建。
已有未完成需求应刷新权重、状态或有效期。
```

### 3.7 权重、有效期和阻塞语义成立

需求至少能维护：

```text
安全权重
服务权重
有效期
是否阻塞父任务执行
满足状态
```

### 3.8 当前主需求可选择

```text
需求树生成后，能选出当前最该处理的需求。
安全压力最大 -> 当前主需求偏向安全根或其子需求。
服务压力最大 -> 当前主需求偏向服务根或其子需求。
```

## 4. 实施步骤

### P2b-1：梳理需求生成入口

检查并记录以下入口：

```text
自我状态压力生成需求。
任务筹办缺口生成需求。
任务执行缺失生成需求。
方法结构缺口生成需求。
自检报告不生成需求，只报送自我线程。
```

输出：

```text
详细设计中记录每个入口的生成条件、父需求归属、去重口径和状态字段。
```

### P2b-2：补需求树生成诊断

在自我内部循环诊断中补充：

```text
需求树生成成立
动态需求生成入口可用
最近需求生成原因
最近需求去重 / 合并结果
管理需求 / 执行需求角色刷新状态
需求树生成断点
```

### P2b-3：补控制面板展示

控制面板内部循环摘要新增：

```text
需求树生成成立
最近需求生成原因
最近去重 / 合并摘要
管理需求数
执行需求数
需求树生成断点
```

### P2b-4：补回归剧本

最小剧本：

```text
1. 降低服务值。
2. 自我线程运行一轮。
3. 服务根需求活跃，或生成 / 刷新服务修复需求。
4. 再运行一轮，不重复创建同目标服务修复需求。
5. 触发一个方法结构缺口，生成或复用补方法需求。
6. 检查父子结构、管理 / 执行角色、权重和有效期。
```

### P2b-5：更新完成度口径

当前代码中的 `根需求循环可用` 可作为历史枚举名保留，但文档和控制面板显示应解释为：

```text
P2 第一段：根需求压力评估和主需求选择。
```

若后续改代码枚举，推荐显示名改为：

```text
需求树生成成立
```

## 5. 非目标和边界

```text
1. 不把初始化阶段的安全根需求 / 服务根需求存在，误判为需求树正常生成。
2. 不把因果缺口学习需求提前归入初始化或普通需求树生成。
3. 不让自检线程生成修复需求；自检线程只报送缺失。
4. 不把派生需求父需求缺失兜底挂到根需求；这是逻辑错误，应报错并记录日志。
5. 不静默兜底生成泛化需求。
```
